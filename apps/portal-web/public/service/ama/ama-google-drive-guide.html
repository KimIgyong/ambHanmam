ℹ️
왜 Service Account를 사용하나요? 일반 OAuth(개인 로그인) 방식은 특정 직원의 계정에 종속되어, 해당 직원 퇴사 시 연동이 끊기는 문제가 발생합니다. Service Account는 구글 Cloud의 서비스 전용 계정으로, 특정 개인에 종속되지 않아 안정적으로 운영할 수 있습니다.

3

Google Cloud 설정

Service Account 생성 및 API 활성화
STEP 3-1

Google Cloud 프로젝트 생성 또는 선택

  1. console.cloud.google.com에 접속합니다
  2. 상단의 프로젝트 선택기 클릭 → [새 프로젝트] 또는 기존 프로젝트 선택
  3. 새 프로젝트 생성 시 프로젝트 이름 입력 (예: amoeba-ama-drive)
  4. [만들기] 클릭 후 해당 프로젝트가 선택된 상태로 진행
💡
이미 Google Cloud를 사용 중이라면 기존 프로젝트를 재활용해도 됩니다. 단, 프로젝트는 청구 계정이 연결되어 있어야 합니다 (Drive API는 무료 사용 한도 이내에서 비용이 발생하지 않습니다).
STEP 3-2

Google Drive API 활성화

  1. 좌측 메뉴 → API 및 서비스 → 라이브러리
  2. 검색창에 "Google Drive API" 입력
  3. 검색 결과에서 Google Drive API 클릭
  4. [사용 설정] 버튼 클릭 (이미 활성화된 경우 "관리" 버튼으로 표시)
STEP 3-3

Service Account (서비스 계정) 생성

  1. 좌측 메뉴 → API 및 서비스 → 사용자 인증 정보
  2. 상단 [+ 사용자 인증 정보 만들기] → 서비스 계정 선택
  3. 서비스 계정 이름 입력: 
    # 이름 예시
서비스 계정 이름: ama-drive-service
서비스 계정 ID:  ama-drive-service  (자동 생성됨)
설명 (선택):    AMA Management Drive Integration
  4. [만들고 계속하기] 클릭
  5. 역할 설정 단계는 건너뛰기 → [완료]
  6. 생성된 서비스 계정 이메일 주소를 복사해 두세요
    예: [email protected]
✅
서비스 계정 생성이 완료되면 사용자 인증 정보 목록에 서비스 계정이 표시됩니다. 이메일 주소는 AMA 설정에 사용되므로 반드시 메모해 두세요.
STEP 3-4

서비스 계정 JSON 키 생성 및 AMA 서버 등록

  1. 생성된 서비스 계정 이메일 클릭 → [키] 탭 이동
  2. [키 추가] → [새 키 만들기]
  3. 키 유형: JSON 선택 → [만들기]
  4. JSON 파일이 자동 다운로드됩니다 — 안전한 곳에 보관하세요
  5. 다운로드된 JSON 파일을 AMA 시스템 관리자에게 전달하여 서버에 등록해야 합니다
# JSON 파일 구조 예시 (서버 등록용)
{
  "type": "service_account",
  "project_id": "your-project-id",
  "private_key_id": "abc123...",
  "private_key": "-----BEGIN PRIVATE KEY-----\n...",
  "client_email": "[email protected]",
  "client_id": "1234567890",
  ...
}
🔒
보안 주의 JSON 키 파일은 서비스 계정의 인증 정보입니다. Slack, 이메일, Git 저장소에 절대 업로드하지 마세요. 서버 관리자에게만 안전한 채널로 전달하세요.
4

도메인 위임 설정

Google Admin Console에서 서비스 계정에 Drive 접근 권한 부여
ℹ️
도메인 위임(Domain-wide Delegation)은 서비스 계정이 회사 도메인의 특정 사용자를 대신하여(impersonate) Google Drive에 접근할 수 있도록 허용하는 설정입니다. AMA 서버는 이 위임받은 계정의 권한으로 Drive API를 호출합니다.
STEP 4-1

서비스 계정에 도메인 위임 활성화

  1. Google Cloud Console → IAM 및 관리자 → 서비스 계정
  2. 생성한 서비스 계정 클릭 → [세부정보] 탭
  3. "도메인 위임 표시" 섹션 클릭하여 펼치기
  4. [G Suite 도메인 전체 위임 사용 설정] 체크박스 활성화
  5. OAuth 클라이언트 ID를 복사해 두세요 (13~22자리 숫자)
  6. [저장]
STEP 4-2

Google Admin Console에서 API 범위 승인

  1. admin.google.com에 Workspace 관리자 계정으로 로그인
  2. 보안 → API 제어 → 도메인 전체 위임 관리
  3. [새로 추가] 버튼 클릭
  4. 클라이언트 ID: 앞서 복사한 OAuth 클라이언트 ID 입력
  5. OAuth 범위 입력: 
    https://www.googleapis.com/auth/drive
  6. [승인] 클릭
💡
위임 이메일(Impersonate Email)이란? 서비스 계정이 실제로 Drive에 접근할 때 사용할 회사 계정 이메일입니다. Drive 관련 파일 소유권 및 접근 권한이 이 계정 기준으로 적용되므로, Drive 전체에 접근 가능한 관리자 또는 전용 계정 이메일을 사용하세요.

예: [email protected]
STEP 4-3

위임 이메일을 공유 드라이브 멤버로 추가

Shared Drive(공유 드라이브)를 사용하는 경우, 위임 이메일 계정을 해당 드라이브의 멤버로 추가해야 합니다.

  1. Google Drive 웹(drive.google.com)에서 공유 드라이브 오른쪽 클릭 → [멤버 관리]
  2. 위임 이메일 주소 추가 (예: [email protected])
  3. 역할: 관리자 또는 콘텐츠 관리자 선택
  4. [보내기]

TIP

Google Drive 폴더 ID 찾는 방법

  1. Google Drive(drive.google.com)에서 원하는 폴더 열기
  2. 브라우저 주소창의 URL 확인
  3. URL에서 /folders/ 뒤의 문자열이 폴더 ID입니다
# URL 예시
https://drive.google.com/drive/folders/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs

# 폴더 ID
1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs

공유 드라이브의 경우 URL 구조가 다를 수 있습니다:

https://drive.google.com/drive/u/0/shared-drives/0AKnpMm8KPwUEUk9PVA

연결 테스트 실행

  3. 결과 확인:
    • ✓ 성공 연결 성공 메시지와 함께 공유 드라이브 목록이 표시됩니다
    • ✗ 실패 오류 메시지 확인 후 아래 FAQ의 문제 해결 방법을 참고하세요
STEP 5-2

연결된 공유 드라이브 탐색 및 확인

연결 테스트 성공 후 설정 화면 하단에 공유 드라이브 탐색기가 표시됩니다.

  1. 연결된 공유 드라이브 목록 확인
  2. 드라이브 클릭 → 하위 폴더 트리 탐색
  3. 원하는 폴더의 ID를 확인하여 설정에 반영 가능
✅
공유 드라이브가 정상적으로 표시되면 Google Cloud → AMA 서버 간 연동이 완료된 것입니다. 다음 단계로 팀원들이 접근할 폴더를 등록하세요.

폴더 등록 (문서 메뉴)

팀원들이 접근할 공유 폴더를 AMA 문서 메뉴에 등록하는 방법
📍
경로: 사이드바 → 문서 (/documents)  | 폴더 등록은 ADMIN 권한만 가능합니다.
STEP 6-1

팀 공유 폴더 등록

  1. 사이드바에서 문서 메뉴 클릭
  2. 루트 화면에서 [+ 폴더 등록] 버튼 클릭 (ADMIN에게만 표시)
  3. 입력:

    항목내용필수
    폴더 ID Google Drive 폴더 URL의 /folders/ 뒤 문자열 필수
    설명 팀원에게 표시될 폴더 설명 (예: "베트남 법인 계약서 보관함") 선택
  4. [등록] 버튼 클릭
  5. 루트 화면에 폴더 카드가 추가되어 모든 팀원이 접근 가능
💡
폴더 등록 해제 시 폴더 카드의 [해제] 버튼을 클릭하세요. Google Drive의 실제 파일은 삭제되지 않으며, AMA에서의 연결만 해제됩니다.
STEP 6-2

파일 업로드 및 탐색

기능방법제한
폴더 탐색 폴더 카드 클릭 → 브레드크럼으로 경로 이동 무한 스크롤
파일 업로드 폴더 내부에서 [업로드] 버튼 → 파일 선택 (다중 가능) 최대 50MB/파일
서브폴더 생성 폴더 내부에서 [새 폴더] 버튼 —
파일 검색 상단 검색창에 키워드 입력 (300ms 디바운스) 등록된 폴더 전체 대상
파일 열기 파일 클릭 → 상세 모달 → Google Drive 링크로 원본 열기 —

자주 묻는 질문

문제 해결 및 FAQ
Q
연결 테스트에서 "403 Forbidden" 오류가 발생합니다
도메인 위임 설정이 완료되지 않았거나, Google Admin Console에서 API 범위 승인이 되지 않은 경우입니다.
확인 방법: Google Admin Console → 보안 → API 제어 → 도메인 전체 위임 목록에서 서비스 계정 클라이언트 ID와 https://www.googleapis.com/auth/drive 범위가 등록되어 있는지 확인하세요. 위임 설정 후 반영까지 최대 24시간이 소요될 수 있습니다.
Q
공유 드라이브 목록이 비어 있습니다
위임 이메일 계정이 공유 드라이브의 멤버로 등록되지 않은 경우입니다.
Google Drive에서 공유 드라이브 → [멤버 관리]에서 위임 이메일을 관리자 또는 콘텐츠 관리자로 추가하세요.
Q
일반 팀원이 [+ 폴더 등록] 버튼을 볼 수 없습니다
정상입니다. 폴더 등록/해제는 ADMIN 권한만 가능합니다. ADMIN이 폴더를 등록하면 모든 팀원이 해당 폴더에 접근할 수 있습니다. 팀원에게 관리자 권한을 부여하려면 설정 → 멤버 관리에서 역할을 변경하세요.
Q
개인 Gmail(@gmail.com) 계정으로도 연동이 가능한가요?
불가능합니다. 도메인 위임은 Google Workspace(기업용 G Suite) 환경에서만 지원됩니다. 개인 Gmail 계정은 Google Admin Console이 없어 도메인 위임 설정이 불가합니다. 회사 Google Workspace 구독이 필요합니다.
Q
파일 업로드 후 Google Drive에서 소유자가 이상하게 표시됩니다
AMA를 통해 업로드된 파일은 위임 이메일 계정 소유로 등록됩니다. 이는 정상적인 동작입니다. 필요한 경우 Google Drive에서 직접 소유자를 변경할 수 있습니다.
Q
서비스 계정 JSON 키를 분실했습니다
JSON 키는 생성 시 한 번만 다운로드되며, 이후 다시 다운로드할 수 없습니다. Google Cloud Console → 서비스 계정 → [키] 탭에서 기존 키를 삭제하고 새 키를 생성한 후, 서버 관리자에게 재등록을 요청하세요.
SUMMARY

전체 설정 항목 한눈에 보기

단계작업 위치주요 작업권한
1. Cloud 프로젝트 Google Cloud Console 프로젝트 생성 / Drive API 활성화 GCP 관리자
2. 서비스 계정 Google Cloud Console 서비스 계정 생성 + JSON 키 발급 GCP 관리자
3. 도메인 위임 Google Admin Console API 범위 승인 (Drive 범위) Workspace 관리자
4. AMA 설정 AMA → 설정 → Drive

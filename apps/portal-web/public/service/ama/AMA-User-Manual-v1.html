시스템 접속 및 로그인 Login

기능 설명 / Feature Description

AMB Management(AMA)는 웹 브라우저에서 접속하는 클라우드 기반 업무 관리 시스템입니다.

AMB Management (AMA) is a cloud-based work management system accessed via web browser.

사용법 / How to Use

  1. 웹 브라우저에서 시스템 URL에 접속합니다 / Open the system URL in a web browser
  • 개발 환경: http://localhost:5179
  • 스테이징: https://stg-ama.amoeba.site
  • 프로덕션: https://ama.amoeba.site
  1. 회사 이메일과 비밀번호로 로그인합니다 / Log in with your company email and password
  2. 로그인 후 오늘의 미션(Today) 대시보드가 표시됩니다 / After login, the Today dashboard appears

💡 사용 팁 / Usage Tips

  • 자동 로그아웃: 30분간 미활동 시 자동 로그아웃되므로, 중요 작업 중에는 주기적으로 활동하세요.

Auto logout occurs after 30 minutes of inactivity. Stay active during important work.

  • PWA 설치: 모바일/데스크톱에서 "앱 설치" 기능을 사용하면 네이티브 앱처럼 사용할 수 있습니다.

Install as PWA on mobile/desktop for a native app-like experience.

  • 다국어 전환: 헤더 우측의 언어 선택기로 한국어/영어/베트남어 중 원하는 언어를 선택할 수 있습니다.

Use the language selector in the header to switch between Korean/English/Vietnamese.

조직 구조 이해 Organization Structure

기능 설명 / Feature Description

AMA는 매트릭스 조직 구조를 지원합니다. 데이터는 Entity(법인) 단위로 격리되며, Unit(유닛/부서) 은 상설 조직, Cell(셀/프로젝트팀) 은 프로젝트 기반 유연한 조직입니다.

AMA supports a matrix organization structure. Data is isolated per Entity (company), Unit (department) is a permanent structure, and Cell (project team) is a flexible project-based structure.

| 계층 / Level | 설명 / Description |

|------|------|

| Entity (법인/회사) | 최상위 단위. 모든 데이터가 법인별로 격리됩니다. / Top-level unit. All data is isolated per entity. |

| Unit (유닛/부서) | 1인 1 유닛 소속. 하위 유닛(Section) 가능. / One person belongs to one unit. Sub-units (Section) possible. |

| Cell (셀/프로젝트팀) | 1인 다수 셀 소속 가능. TF(Task Force) 형태. / One person can belong to multiple cells. Task Force style. |

💡 사용 팁 / Usage Tips

  • 법인 전환: 여러 법인에 소속된 경우, 헤더의 법인 선택기로 소속 법인을 전환할 수 있습니다.

If affiliated with multiple entities, use the entity selector in the header to switch.

  • 데이터 격리: 프로젝트, 거래처, 회계 등의 데이터는 법인 단위로 분리되어 있으므로, 올바른 법인을 선택했는지 확인하세요.

Data for projects, billing, accounting, etc. is isolated per entity — ensure you've selected the correct one.

0. 오늘의 미션 Today

> 경로 / Path: 로그인 후 첫 화면 / Login landing page (/)

기능 설명 / Feature Description

로그인 직후 표시되는 통합 대시보드 페이지입니다. 오늘 처리해야 할 미션, 지연 작업, 이슈, 일정, 회의록을 한 화면에서 확인할 수 있습니다. 범위 탭(My Work / Unit / Cell / All)으로 조회 범위를 전환할 수 있습니다.

Integrated dashboard displayed immediately after login. View today's missions, overdue tasks, issues, schedules, and meeting notes in one screen. Switch scope with tabs (My Work / Unit / Cell / All).

화면 구성 / Screen Layout

  • 범위 탭: 내 업무 · 유닛(부서) · 셀(프로젝트팀) · 전체 4개 탭
  • 오늘의 미션: 오늘과 어제 미완료 미션 표시
  • 작업: 지연 작업(빨간색 강조) + 진행 중 작업
  • 이슈: "확인 필요"(OPEN) / "작업 중"(IN_PROGRESS) / "관계자" 3개 탭
  • 오늘 일정: 사용자 타임존 기반 일정
  • 회의록: 최근 5건
  • 요약 바: 각 항목의 통계

사용법 / How to Use

  1. 로그인 → 자동으로 오늘의 미션 페이지가 표시됩니다 / Login → Today page appears automatically
  2. 범위 탭 전환 → 내 업무 탭이 기본이며, 유닛(부서) · 셀(프로젝트) · 전체로 다른 멤버의 업무도 확인할 수 있습니다 / My Work is default; switch to Unit · Cell · All to see others' work
  3. 오늘의 미션 클릭 → 해당 항목의 상세 페이지로 이동합니다 / Click a mission item → Navigate to the detail page
  4. 이슈 탭 전환 → "확인 필요", "작업 중", "관계자" 탭으로 이슈 상태별 확인 / Switch issue tabs to check by status
  5. AI 업무 코칭 → ✨ 버튼 클릭 → AI가 오늘 업무에 대한 분석과 코칭을 제공합니다 / Click ✨ button → AI provides coaching for today's work
  6. 저장된 리포트 → 📄 버튼 → 이전에 생성된 업무 리포트를 열람합니다 / Click 📄 → Browse saved work reports
  7. 히스토리 캘린더 → 내 업무 탭 우측 사이드바에서 과거 날짜별 업무 내역을 확인합니다 / Use the right sidebar calendar to view past work history

💡 사용 팁 / Usage Tips

  • 아침 루틴: 매일 출근 후 가장 먼저 이 페이지를 확인하여 당일 해야 할 일을 정리하세요.

Check this page first every morning to plan your day.

  • AI 코칭 활용: 업무가 많을 때 ✨ 버튼을 눌러 AI의 우선순위 제안을 참고하세요.

When busy, press ✨ for AI's priority suggestions.

  • 지연 작업 주의: 빨간색으로 강조된 항목은 마감일이 지난 작업입니다. 즉시 처리하거나 마감일을 조정하세요.

Red-highlighted items are overdue — handle immediately or adjust the deadline.

  • 히스토리 캘린더: 과거 날짜를 클릭하면 그날의 업무 기록을 확인할 수 있어, 업무 보고 작성 시 유용합니다.

Click past dates in the history calendar to review work records — useful for writing reports.

1. 할일 TODO

> 경로 / Path: 사이드바 > 할일 / Sidebar > TODO (/todos)

기능 설명 / Feature Description

개인 및 유닛 단위의 할일을 칸반 보드로 관리합니다. 마감일 기준으로 완료(Done) · 오늘(Today) · 예정(Upcoming) 3개 컬럼으로 자동 분류됩니다. 태그, 참여자, 이슈 연결을 지원하며, 그룹/전사 범위의 할일도 조회 가능합니다.

Manage personal and unit tasks on a Kanban board. Auto-classified into 3 columns (Done / Today / Upcoming) by due date. Supports tags, participants, issue linking, and group/all scope viewing.

사용법 / How to Use

할일 생성 / Create Task

  1. 내 할일 탭에서 우상단 [+ 할일 추가] 클릭 / Click [+ Add Task] in My Tasks tab
  2. 제목(필수), 설명, 마감일(필수) 입력 / Enter Title (required), Description, Due Date (required)
  3. 필요 시 태그(쉼표 구분), 참여자(다중 선택), 연결 이슈 설정 / Add Tags, Participants, Linked Issue as needed
  4. [저장] 클릭 → 자동 번역 대화상자가 나타나면 확인 / Click [Save] → Confirm auto-translation dialog

상태 변경 / Change Status

  • 카드의 체크 아이콘 클릭 → 예정(SCHEDULED) → 진행중(IN_PROGRESS) → 완료(COMPLETED) 순으로 순환

Click the check icon on a card → Cycles through SCHEDULED → IN_PROGRESS → COMPLETED

상세보기·수정·삭제 / View Detail · Edit · Delete

  1. 카드 클릭 → 상세 모달 열림 (제목, 설명, 상태, 참여자, 연결 이슈 확인) / Card click → Detail modal opens
  2. 카드 메뉴(⋯) → 수정 또는 삭제 선택 / Card menu (⋯) → Select Edit or Delete
  3. 히스토리 아이콘 클릭 → 상태 변경 이력 확인 / Click history icon → View status change log

그룹·전사 보기 / Group · All View

  • 그룹 탭: 같은 유닛(부서) 멤버들의 할일 조회 (읽기 전용) / View unit members' tasks (read-only)
  • 전사 탭: 전체 회사의 할일 조회 (읽기 전용) / View all company tasks (read-only)

💡 사용 팁 / Usage Tips

  • 마감일 설정: 마감일을 정확히 설정하면 자동으로 "오늘"과 "예정" 컬럼에 분류되어 우선순위 관리에 편리합니다.

Set due dates accurately for automatic column classification and easy priority management.

  • 이슈 연결: 이슈 트래커의 항목과 할일을 연결하면, 이슈 상세에서도 관련 할일을 확인할 수 있습니다.

Linking tasks to issues enables cross-referencing from the issue detail page.

  • 태그 활용: 태그를 #긴급, #회의, #개발 등으로 분류하면 빠른 시각적 구분이 가능합니다.

Use tags like #urgent, #meeting, #dev for quick visual identification.

  • 자동 번역: 저장 시 다국어 자동 번역이 제공되므로, 다국적 팀에서 별도 번역 작업이 불필요합니다.

Auto-translation on save eliminates the need for manual translation in multinational teams.

2. 노트 Meeting Notes

> 경로 / Path: 사이드바 > 노트 / Sidebar > Meeting Notes (/meeting-notes)

기능 설명 / Feature Description

회의록(Meeting Note) 과 메모(Memo) 두 가지 유형의 노트를 작성·관리합니다. 공개 범위를 PRIVATE(본인만) / GROUP(유닛/그룹) / ENTITY(법인) / PUBLIC(전사)으로 설정하여, 개인 메모부터 전사 공유 회의록까지 다양한 용도로 활용할 수 있습니다. 리치 텍스트 에디터와 자동 번역 기능을 제공합니다.

Create and manage two types of notes: Meeting Notes and Memos. Set visibility from PRIVATE (self only) to PUBLIC (company-wide). Rich text editor and auto-translation included.

사용법 / How to Use

노트 생성 / Create Note

  1. 우상단 [+ 새 노트] 클릭 / Click [+ New Note] at top-right
  2. 유형 선택: 회의록 또는 메모 / Select Type: Meeting Note or Memo
  3. 제목(필수) 입력 / Enter Title (required)
  4. 내용 작성 (리치 텍스트 에디터로 서식 편집 가능) / Write Content using rich text editor
  5. 회의록인 경우 회의 일자 선택 / Select Meeting Date for Meeting Notes
  6. 공개 범위 선택: 비공개 / 그룹 / 법인 / 공개 / Select Visibility
  7. 그룹 범위 선택 시 대상 유닛(부서) 지정 / Specify target Unit when GROUP scope selected
  8. [저장] 클릭 / Click [Save]

노트 조회 / Browse Notes

  1. 상단의 유형 필터로 전체 / 회의록 / 메모 필터링 / Use Type Filter to filter
  2. 공개범위 필터로 비공개 / 그룹 / 법인 / 공개 필터링 / Use Scope Filter to filter
  3. 카드 클릭 → 상세 페이지 이동 / Click card → Navigate to detail page

상세 보기 / Detail View

  • HTML 서식 렌더링된 노트 내용 확인 / View HTML-formatted note content
  • 번역 패널: 다국어(KO/EN/VN) 자동 번역 확인 / Translation Panel: Auto-translation (KO/EN/VN)
  • 작성자만 수정 / 삭제 가능 / Only author can Edit / Delete

💡 사용 팁 / Usage Tips

  • 회의록 작성 순서: 회의 전에 미리 제목과 아젠다를 메모로 만들고, 회의 중 내용을 업데이트하면 효율적입니다.

Create a memo with title and agenda before the meeting, then update it during the meeting.

  • 공개 범위 활용: 초안은 비공개로 작성하고, 완성 후 법인 또는 공개로 변경하면 안전합니다.

Start drafts as PRIVATE, then change to ENTITY or PUBLIC once finalized.

  • 번역 패널: 다국적 팀과 공유할 회의록은 공개로 저장하면 자동 번역 패널이 제공됩니다.

Notes shared with multinational teams get auto-translation panels when set to PUBLIC.

3. 출근예정표 Attendance

> 경로 / Path: 사이드바 > 출근예정표 / Sidebar > Attendance (/attendance)

기능 설명 / Feature Description

유닛(부서) 멤버들의 주간 근무 일정을 표 형식으로 확인하고 등록하는 도구입니다. 행은 멤버, 열은 월~일(7일)로 구성되며, 각 셀에 근무유형과 시간이 표시됩니다. 본인 행만 수정 가능하며, 일괄 등록으로 주간 일정을 한 번에 입력할 수 있습니다.

Weekly schedule table for unit members. Rows represent members (Mon–Sun columns) showing work type and hours. Only your own row is editable. Batch registration available for weekly entry.

사용법 / How to Use

일정 등록 / Register Schedule

  1. 본인 행의 원하는 요일 셀 클릭 / Click a day cell in your row
  2. 하단 편집 폼에서 근무유형 선택 / Select Work Type in the bottom form
  • 출근(WORK), 재택(REMOTE), 휴가(DAY_OFF), 오전반차(AM_HALF), 오후반차(PM_HALF), 생리휴가(MENSTRUATION)
  1. 시작시간 선택 (06:00~12:00, 30분 간격) / Select Start Time (06:00–12:00, 30-min intervals)
  2. 종료시간은 근무유형에 따라 자동 계산 / End time is auto-calculated by type
  3. [저장] 클릭 / Click [Save]

주간 일괄 등록 / Batch Registration

  1. [일괄 등록] 버튼 클릭 / Click [Batch Register] button
  2. 월~금 각 요일별 근무유형과 시작시간 설정 / Set work type and start time for each weekday
  3. [저장] 클릭 → 5일치 일정 한 번에 등록 / Click [Save] → All 5 days registered at once

일정 삭제 / Delete Schedule

  • 기존 일정이 있는 셀 선택 → [삭제] 클릭 / Select a cell with existing schedule → Click [Delete]

일정 조회 / View Schedule

  • < > 버튼으로 주간 이동 / Navigate weeks with < > buttons
  • 다른 멤버의 일정은 조회만 가능 / Other members' schedules are view-only

💡 사용 팁 / Usage Tips

  • 일괄 등록 활용: 매주 동일 패턴으로 근무한다면, 일괄 등록으로 한 번에 입력하면 시간을 절약할 수 있습니다.

Use batch registration to save time if you have a consistent weekly schedule.

  • 편집 가능 범위: 다음 주 월요일부터 1개월 후까지만 편집 가능합니다. 현재 주나 과거 주는 수정할 수 없으므로, 미리미리 등록하세요.

Only editable from next Monday to 1 month ahead. Register in advance since past weeks can't be modified.

  • 비고 컬럼: NOTE 컬럼에 재택 근무 요일이나 특이사항을 메모해 두면 팀원 간 소통에 도움됩니다.

Use the NOTE column to record remote work days or special notes for team communication.

  • 재택 근무: 재택(REMOTE) 유형은 녹색으로 표시되어, 한눈에 누가 재택인지 확인할 수 있습니다.

REMOTE type is shown in green, making it easy to identify who's working remotely at a glance.

4. 휴가 신청 My Leave

  3. 소속 법인: 법인명, 코드, 국가, 역할, 상태 확인 / Entities: Name, code, country, role, status
  4. 소속 유닛: 유닛명, 역할, Primary 여부, 법인명 확인 / Units: Unit name, role, ⭐Primary, entity
  5. 소속 셀: 셀명, 법인명 확인 / Cells: Cell name, entity
  6. 휴가 잔액: 기본 지급, 사용, 잔여, 이월, 근속년수 5개 카드 / Leave Balance: 5 metric cards
  7. 지출 요청: 최근 5건 + [신규 신청] 버튼으로 빠른 지출결의 / Expense Requests: Recent 5 + [New Request] button

💡 사용 팁 / Usage Tips

  • 소속 확인: 여러 법인/유닛에 소속된 경우, 마이페이지에서 현재 소속 현황을 정확히 확인하세요.

Verify your current affiliations when belonging to multiple entities/units.

  • 빠른 지출 신청: 마이페이지의 [신규 신청] 버튼으로 지출결의 페이지로 빠르게 이동할 수 있습니다.

Quickly navigate to expense request via the [New Request] button on My Page.

  • 휴가 잔액 확인: 휴가 신청 전에 마이페이지에서 잔여 일수를 확인하는 습관을 들이세요.

Make it a habit to check remaining leave days on My Page before requesting leave.

28. 공통 기능 Common Features

28.1 커스텀 앱 Custom Apps

> 경로 / Path: 사이드바 > 커스텀 앱 / Sidebar > Custom Apps (/apps/:appCode)

기능 설명 / Feature Description

외부 웹 애플리케이션을 iframe 또는 새 탭으로 호스팅하는 기능입니다. JWT, API Key, None 3가지 인증 모드를 지원합니다.

Host external web applications via iframe or new tab. 3 authentication modes: JWT, API Key, None.

사용법 / How to Use

  1. 사이드바에서 커스텀 앱 선택 / Select Custom App from sidebar
  2. iframe 모드: 페이지 내에서 바로 표시 / iframe mode: displayed inline
  3. 새 탭 모드: 새 브라우저 탭에서 열림 / new tab mode: opens in new browser tab
  4. iframe 로드 차단 시 "새 탭으로 열기" 대체 버튼 표시 / Fallback "Open in New Tab" button when iframe blocked
  5. 현재 언어가 ?locale= 파라미터로 자동 전달 / Current language auto-passed as ?locale= parameter

28.2 다국어 지원 Multi-Language Support

기능 설명 / Feature Description

전체 UI가 한국어 / 영어 / 베트남어 3개 언어를 지원합니다.

Full UI supports Korean / English / Vietnamese.

사용법 / How to Use

  1. 헤더 우측의 언어 선택기 클릭 / Click language selector in header right
  2. 원하는 언어 선택 → UI 전체 즉시 전환 / Select language → Entire UI switches immediately
  3. 노트, 할일, 공지사항 등에서 자동 번역 패널 활용 / Use auto-translation panel in notes, TODOs, notices

28.3 법인 전환 Entity Switching

사용법 / How to Use

  • 헤더의 법인 선택기 클릭 → 소속 법인 전환 / Click entity selector in header → Switch entity
  • 프로젝트, 거래처, 회계 등은 법인 단위로 데이터 격리 / Data isolated per entity for projects, billing, accounting

28.4 글로벌 검색 Global Search

사용법 / How to Use

  • 헤더 중앙 검색바에 키워드 입력 → 시스템 전체 통합 검색 / Enter keyword in header search bar → System-wide integrated search

28.5 AI 어시스턴트 AI Assistant

기능 설명 / Feature Description

업무 관련 질문, 데이터 조회, 작업 지원이 가능한 AI 어시스턴트입니다. 로그인 후 자동으로 한 번 표시됩니다.

AI assistant for work-related questions, data queries, and task support. Auto-displayed once after login.

사용법 / How to Use

  1. 우하단 플로팅 버튼 클릭 → AI 어시스턴트 모달 열림 / Click bottom-right floating button → AI assistant modal opens
  2. 질문 입력 → AI 응답 확인 / Enter question → View AI response

💡 사용 팁 / Usage Tips

  • 즉석 질문: 별도의 AI 에이전트 페이지로 이동하지 않아도, 어디서든 플로팅 버튼으로 빠르게 AI에게 질문할 수 있습니다.

Ask AI quick questions from anywhere via the floating button — no need to navigate to the AI Agents page.

28.6 타임존 Timezone

  • 사용자별 타임존 설정 (기본: Asia/Ho_Chi_Minh) / Per-user timezone setting (default: Asia/Ho_Chi_Minh)
  • 캘린더, 오늘의 업무 등 시간 관련 기능에 자동 반영 / Auto-applied to calendar, today's work, etc.

28.7 자동 로그아웃 Auto Logout

  • 30분간 미활동 시 자동 로그아웃 / Auto logout after 30 minutes of inactivity
  • 중요 작업 중에는 주기적으로 마우스/키보드 활동을 유지하세요 / Maintain periodic activity during important work

28.8 PWA (Progressive Web App)

  • 푸시 알림 지원 / Push notification support

💡 사용 팁 / Usage Tips

Add to home screen on mobile browsers for native app-like quick access.

  • 푸시 알림: PWA 설치 후 푸시 알림을 허용하면 중요한 업무 알림을 실시간으로 받을 수 있습니다.

Enable push notifications after PWA installation to receive real-time work alerts.

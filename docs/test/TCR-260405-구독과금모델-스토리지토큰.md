## 2. 통합 테스트 시나리오

### 2.1 FREE 구독 → BASIC 업그레이드 플로우

```
1. POST /subscriptions (plan_code=FREE, user_count=5)
   → 구독 생성, BASE 토큰 50,000 지급, 스토리지 1GB
2. GET /subscriptions
   → status=FREE, planCode=FREE, plan 정보 확인
3. GET /subscriptions/tokens
   → BASE: 50000, ADDON: 0, REFERRAL: 0
4. GET /subscriptions/storage
   → baseGb=1, addonGb=0, totalGb=1, usedPct=0
5. POST /subscriptions/checkout (plan_code=BASIC, user_count=10)
   → checkoutUrl 반환 (Polar redirect)
6. [Polar Webhook] subscription.active
   → 구독 상태 ACTIVE로 변경, 월간 토큰 지급 (200,000)
7. GET /subscriptions
   → status=ACTIVE, planTier=BASIC
```

### 2.2 AI 사용 → 토큰 차감 플로우

```
1. POST /conversations/:id/messages (채팅 전송)
   → TokenBalanceGuard: 잔액 확인 (> 0)
   → Claude API 스트리밍 응답
   → recordUsageIfContext → AiUsageService.recordUsage()
   → 이벤트 'ai-usage.recorded' emit
   → TokenWalletService.handleAiUsageRecorded()
   → deductAiUsage (BASE → ADDON → REFERRAL 순서)
2. GET /subscriptions/tokens
   → BASE 잔액 감소 확인
```

### 2.3 스토리지 업로드 → 쿼터 체크 플로우

```
1. POST /files/upload (파일 업로드)
   → StorageQuotaService.checkUpload(entityId, fileSize)
   → FileService.saveFile()
   → StorageQuotaService.addUsage(entityId, fileSize)
2. POST /drive/upload (Drive 업로드)
   → StorageQuotaService.checkUpload(entityId, fileSize)
   → DriveService.uploadFile()
   → StorageQuotaService.addUsage(entityId, fileSize)
3. GET /subscriptions/storage
   → usedBytes 증가, usedPct 업데이트 확인
```

### 2.4 Polar Webhook 이벤트 처리

```
1. POST /webhooks/polar (subscription.created)
   → 서명 검증 → PgWebhookEvent 저장 → PgSubscription upsert
2. POST /webhooks/polar (subscription.active)
   → 구독 활성화, 토큰 월간 지급
3. POST /webhooks/polar (subscription.canceled)
   → 구독 취소 상태 변경
4. POST /webhooks/polar (order.created)
   → Token/Storage addon 처리
5. 중복 이벤트 전송 → pgw_polar_event_id 중복 → IGNORED 처리
```

## 3. 엣지 케이스

### 3.1 토큰

| # | 시나리오 | 기대 동작 |
|---|---|---|
| E1 | 잔액 정확히 0에서 AI 사용 시도 | TokenBalanceGuard에서 차단 (E29001) |
| E2 | 차감 중 동시 요청 (Race condition) | pessimistic_write 잠금으로 순차 처리 |
| E3 | 월 리셋 중 AI 사용 요청 | 리셋 트랜잭션 완료 후 차감 (잠금 대기) |
| E4 | 이벤트 리스너 실패 | logger.warn 출력, AI 서비스는 중단 안 함 |
| E5 | 구독 없는 엔티티에서 AI 사용 | TokenBalanceGuard 바이패스 (기존 쿼터 시스템 적용) |

### 3.2 스토리지

| # | 시나리오 | 기대 동작 |
|---|---|---|
| E6 | 정확히 100%에서 업로드 | 경고 반환, 업로드 허용 |
| E7 | 정확히 120%에서 업로드 | 차단 (E29002 ForbiddenException) |
| E8 | sqt_used_bytes 음수 시도 | Math.max(0) 처리 |
| E9 | 쿼터 레코드 미존재 (구독 미가입) | checkUpload try/catch로 무시, 업로드 허용 |
| E10 | 동시 파일 업로드 (Race condition) | pessimistic_write 잠금으로 순차 처리 |

### 3.3 결제/웹훅

| # | 시나리오 | 기대 동작 |
|---|---|---|
| E11 | 웹훅 서명 위조 | 403 Forbidden (E29010) |
| E12 | 웹훅 타임스탬프 5분 초과 | 403 Forbidden (E29010) |
| E13 | 중복 웹훅 이벤트 | 200 OK 반환, 처리 건너뜀 (IGNORED) |
| E14 | rawBody 파싱 실패 | 400 Bad Request |
| E15 | Polar API 장애 시 체크아웃 | 503 에러 전달 |

### 3.4 데이터 격리

| # | 시나리오 | 기대 동작 |
|---|---|---|
| E16 | USER_LEVEL A가 B 법인 구독 조회 | OwnEntityGuard에서 차단 |
| E17 | ADMIN_LEVEL이 타법인 구독 조회 | entity_id 파라미터로 조회 가능 |

# TC-20260328 — 리치텍스트에디터 UX개선 테스트케이스

## 1. 스티키 툴바 (R1)

| # | 테스트 항목 | 절차 | 기대결과 |
|---|-----------|------|---------|
| 1.1 | 툴바 고정 - Issues | Issues → 새 이슈 생성 → 설명란에 긴 텍스트 입력 (에디터 maxHeight 초과) | 에디터 내부 스크롤 발생, 툴바는 에디터 상단에 고정 |
| 1.2 | 툴바 고정 - Todos | Todos → 새 할일 생성 → 설명란에 긴 텍스트 입력 | 동일하게 툴바 고정 |
| 1.3 | 툴바 고정 - Meeting Notes | Meeting Notes → 새 노트 생성 → 본문에 긴 텍스트 입력 | 동일하게 툴바 고정 |
| 1.4 | 툴바 고정 상태에서 서식 적용 | 에디터 스크롤 후 텍스트 선택 → 툴바 Bold/Italic 클릭 | 서식 정상 적용, 에디터 포커스 유지 |
| 1.5 | 이미지 메뉴 동작 | 스크롤 후 툴바의 이미지 버튼 클릭 | 드롭다운 메뉴가 툴바 아래에 정상 표시 |

## 2. 기본 높이 및 maxHeight (R2, R3)

| # | 테스트 항목 | 절차 | 기대결과 |
|---|-----------|------|---------|
| 2.1 | 빈 에디터 최소 높이 | Issues 새 이슈 모달 열기 | 에디터 높이 최소 120px |
| 2.2 | maxHeight 제한 | 에디터에 텍스트를 maxHeight 이상 입력 | 300px(Issues) / 400px(MeetingNotes) 높이에서 스크롤 발생, 모달 밀리지 않음 |
| 2.3 | 콘텐츠 적을 때 | 에디터에 1~2줄만 입력 | minHeight 유지, 내부 스크롤 없음 |
| 2.4 | Source 모드 높이 | HTML Source 토글 → 긴 HTML 입력 | textarea에 minHeight 적용, maxHeight 제한 동작 |

## 3. 에디터 내부 스크롤 격리 (R3)

| # | 테스트 항목 | 절차 | 기대결과 |
|---|-----------|------|---------|
| 3.1 | 에디터 내부 스크롤 | 에디터 내부에서 마우스 휠 스크롤 | 에디터 콘텐츠만 스크롤 |
| 3.2 | 스크롤 전파 차단 | 에디터 끝까지 스크롤 후 계속 휠 | 모달/페이지 스크롤 전파 차단 (overscroll-behavior: contain) |
| 3.3 | 이중 스크롤 없음 | 모달 내 에디터가 maxHeight 도달 | 에디터 내부만 스크롤, 모달 자체는 별도 스크롤 가능 |

## 4. 코멘트 에디터 유지 (R4)

| # | 테스트 항목 | 절차 | 기대결과 |
|---|-----------|------|---------|
| 4.1 | 코멘트 에디터 높이 | Issue 상세 → 코멘트 입력란 확인 | minHeight 80px, maxHeight 미적용 (제한 없음) |
| 4.2 | 코멘트 멘션 동작 | 코멘트 에디터에서 @ 입력 | 멘션 자동완성 드롭다운 정상 표시 |
| 4.3 | Todo 코멘트 | Todo 상세 → 코멘트 입력 | 기존과 동일하게 동작 |

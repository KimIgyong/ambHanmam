## 1. 업로드 버튼 위치 변경

### TC-1.1 폴더 내부 진입 시 업로드 버튼 표시

| 항목 | 내용 |
|------|------|
| 사전조건 | 사용자가 로그인된 상태이며 문서함 페이지(`/documents`)에 접근 가능 |
| 테스트 단계 | 1. 문서함 페이지로 이동한다<br>2. 임의의 폴더를 클릭하여 내부로 진입한다<br>3. 브레드크럼 영역을 확인한다 |
| 기대 결과 | 브레드크럼 영역에 업로드 버튼이 표시된다 |
| 우선순위 | 높음 |

### TC-1.2 최상위 경로에서 업로드 버튼 미표시

| 항목 | 내용 |
|------|------|
| 사전조건 | 사용자가 로그인된 상태 |
| 테스트 단계 | 1. 문서함 페이지로 이동한다<br>2. 최상위 경로(루트)에 위치한다<br>3. 헤더 및 브레드크럼 영역을 확인한다 |
| 기대 결과 | 폴더 내부가 아닌 상태에서는 업로드 버튼이 표시되지 않는다 |
| 우선순위 | 높음 |

### TC-1.3 업로드 버튼 클릭 시 파일 업로드 동작

| 항목 | 내용 |
|------|------|
| 사전조건 | 폴더 내부에 진입한 상태 |
| 테스트 단계 | 1. 브레드크럼 영역의 업로드 버튼을 클릭한다<br>2. 파일 선택 다이얼로그에서 파일을 선택한다<br>3. 업로드가 완료될 때까지 대기한다 |
| 기대 결과 | 파일이 현재 폴더에 정상적으로 업로드되고 파일 목록에 나타난다 |
| 우선순위 | 높음 |

### TC-1.4 헤더 영역에서 업로드 버튼 제거 확인

| 항목 | 내용 |
|------|------|
| 사전조건 | 사용자가 로그인된 상태 |
| 테스트 단계 | 1. 문서함 페이지로 이동한다<br>2. 폴더 내부로 진입한다<br>3. 페이지 헤더 영역을 확인한다 |
| 기대 결과 | 기존 헤더 영역에는 업로드 버튼이 존재하지 않는다 |
| 우선순위 | 중간 |

| 사전조건 | 일반(non-admin) 사용자로 로그인 |
| 테스트 단계 | 1. 문서함 페이지로 이동한다<br>2. 임의의 폴더 내부로 진입한다<br>3. 브레드크럼 영역을 확인한다 |

## 3. 카드/그리드 뷰 전환

### TC-3.1 리스트 뷰에서 그리드 뷰로 전환

| 항목 | 내용 |
|------|------|
| 사전조건 | 문서함 페이지에 파일이 있는 폴더로 진입한 상태 |
| 테스트 단계 | 1. 기본 리스트 뷰 상태를 확인한다<br>2. 그리드(카드) 뷰 토글 버튼을 클릭한다 |
| 기대 결과 | 파일 목록이 그리드(카드) 형태로 전환되어 표시된다. 각 파일이 `FileGridItem` 컴포넌트로 렌더링되며 썸네일이 표시된다 |
| 우선순위 | 높음 |

### TC-3.2 그리드 뷰에서 리스트 뷰로 전환

| 항목 | 내용 |
|------|------|
| 사전조건 | 그리드 뷰가 활성화된 상태 |
| 테스트 단계 | 1. 리스트 뷰 토글 버튼을 클릭한다 |
| 기대 결과 | 파일 목록이 기존 리스트 형태로 복원된다 |
| 우선순위 | 높음 |

### TC-3.3 뷰 상태 localStorage 영속성 확인

| 항목 | 내용 |
|------|------|
| 사전조건 | 문서함 페이지에 접근 가능한 상태 |
| 테스트 단계 | 1. 그리드 뷰로 전환한다<br>2. 브라우저에서 다른 페이지로 이동한다<br>3. 다시 문서함 페이지로 돌아온다 |
| 기대 결과 | 이전에 선택한 그리드 뷰가 유지되어 있다 (localStorage에 저장된 값이 복원됨) |
| 우선순위 | 높음 |

### TC-3.4 새 브라우저 탭에서 뷰 상태 유지

| 항목 | 내용 |
|------|------|
| 사전조건 | 그리드 뷰를 선택한 상태에서 localStorage에 값이 저장됨 |
| 테스트 단계 | 1. 새 탭에서 문서함 페이지를 연다 |
| 기대 결과 | 동일 브라우저의 새 탭에서도 그리드 뷰가 기본으로 표시된다 |
| 우선순위 | 중간 |

### TC-3.5 localStorage 초기화 시 기본 뷰

| 항목 | 내용 |
|------|------|
| 사전조건 | localStorage에 뷰 관련 값이 없는 상태 (시크릿 모드 또는 수동 삭제) |
| 테스트 단계 | 1. 문서함 페이지로 이동한다 |
| 기대 결과 | 기본값인 리스트 뷰로 표시된다 |
| 우선순위 | 중간 |

### TC-3.6 그리드 뷰에서 썸네일 표시

| 항목 | 내용 |
|------|------|
| 사전조건 | 이미지 파일(jpg, png 등)이 포함된 폴더에 진입한 상태 |
| 테스트 단계 | 1. 그리드 뷰로 전환한다<br>2. `FileGridItem` 컴포넌트의 썸네일 영역을 확인한다 |
| 기대 결과 | 이미지 파일은 실제 이미지 썸네일이 표시되고, 비이미지 파일은 파일 유형에 맞는 아이콘이 표시된다 |
| 우선순위 | 중간 |

### TC-3.7 빈 폴더에서의 뷰 전환

| 항목 | 내용 |
|------|------|
| 사전조건 | 파일이 없는 빈 폴더에 진입한 상태 |
| 테스트 단계 | 1. 리스트 뷰와 그리드 뷰를 번갈아 전환한다 |
| 기대 결과 | 오류 없이 빈 상태 메시지가 양쪽 뷰 모두에서 정상적으로 표시된다 |
| 우선순위 | 낮음 |

## 4. KMS 태그 자동 분석

### TC-4.1 파일 업로드 시 이벤트 발행 확인

| 항목 | 내용 |
|------|------|
| 사전조건 | API 서버가 실행 중이고 EventEmitter2가 설정된 상태 |
| 테스트 단계 | 1. 문서함에서 파일을 업로드한다<br>2. API 서버 로그를 확인한다 |
| 기대 결과 | `module.data.created` 이벤트가 EventEmitter2를 통해 발행된다 |
| 우선순위 | 높음 |

### TC-4.2 이벤트 페이로드 정확성

| 항목 | 내용 |
|------|------|
| 사전조건 | KMS 모듈이 `module.data.created` 이벤트를 수신하도록 구성됨 |
| 테스트 단계 | 1. 파일을 업로드한다<br>2. 이벤트 리스너에서 수신된 페이로드를 확인한다 |
| 기대 결과 | 페이로드에 파일 ID, 파일명, 모듈 출처 정보 등 태그 분석에 필요한 데이터가 포함되어 있다 |
| 우선순위 | 높음 |

### TC-4.3 KMS 자동 태깅 처리

| 항목 | 내용 |
|------|------|
| 사전조건 | KMS 모듈이 이벤트를 수신하고 자동 태깅 로직이 활성화된 상태 |
| 테스트 단계 | 1. 텍스트가 포함된 문서 파일을 업로드한다<br>2. KMS에서 해당 파일의 태그 정보를 조회한다 |
| 기대 결과 | 업로드된 파일에 대해 자동으로 태그가 분석 및 할당된다 |
| 우선순위 | 높음 |

### TC-4.4 이벤트 발행 실패 시 업로드 영향 없음

| 항목 | 내용 |
|------|------|
| 사전조건 | KMS 이벤트 리스너에서 오류가 발생하도록 설정 (또는 KMS 모듈 비활성화) |
| 테스트 단계 | 1. 파일을 업로드한다<br>2. 업로드 결과를 확인한다 |
| 기대 결과 | KMS 태그 분석 실패와 관계없이 파일 업로드 자체는 정상적으로 완료된다 (이벤트는 비동기 처리) |
| 우선순위 | 높음 |

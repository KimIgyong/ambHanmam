## 1. 백엔드 API 테스트

### TC-B01: GET /members/:id 상세 조회 (확장)
| 항목 | 내용 |
|------|------|
| 전제조건 | MANAGER 이상 권한의 사용자 로그인 |
| 테스트 | 멤버 상세 조회 API 호출 |
| 기대결과 | 기존 필드(userId, email, name, department, role, groups, createdAt) + 신규 필드(companyEmail, entityRoles[], deptRoles[], hrEmployees[]) 반환 |
| 확인사항 | 기존 MemberListTab에서 사용하는 필드가 정상 동작하는지 확인 (하위 호환성) |

### TC-B02: POST /members/:id/entity-roles 법인 역할 배정
| 항목 | 내용 |
|------|------|
| 전제조건 | ADMIN 권한 |
| 테스트 | `{ entity_id: UUID, role: "STAFF" }` 요청 |
| 기대결과 | 201, 배정된 법인 역할 정보 반환 |
| 에러케이스 | 존재하지 않는 entity_id → 404, 중복 배정 → 409 |

### TC-B03: DELETE /members/:id/entity-roles/:eurId 법인 역할 해제
| 항목 | 내용 |
|------|------|
| 전제조건 | ADMIN 권한, 해당 법인 역할 존재 |
| 테스트 | 법인 역할 해제 API 호출 |
| 기대결과 | 204 No Content |
| 에러케이스 | 존재하지 않는 eurId → 404 |

### TC-B04: 권한 제어
| 항목 | 내용 |
|------|------|
| 테스트 | USER 권한으로 법인 배정/해제 시도 |
| 기대결과 | 403 Forbidden |

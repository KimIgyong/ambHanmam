## 2. 상세 테스트케이스

### TC-01: 포털 회원가입 완료 화면 회사 정보 표시

**사전 조건**: 미가입 이메일 주소 보유

**테스트 절차**:
1. `https://stg-www.amoeba.site/register` 접속
2. Step 1: 이름, 이메일, 비밀번호 입력 → 이메일 인증코드 발송 → 인증 완료
3. Step 2: 회사명 입력 (예: "Test Company 0407"), 국가 선택 (예: Vietnam), 약관 동의 → 가입 완료
4. 완료 화면 확인

**기대 결과**:
- [ ] 성공 아이콘 + "가입 완료" 메시지 표시
- [ ] **회사 정보 카드** 표시:
  - 회사명: "Test Company 0407" (또는 auto-provision에서 생성된 entityName)
  - 회사코드: `VN????` 형식 (4자리 숫자)
- [ ] "이 코드를 저장해 두세요. Ạ 로그인 시 필요합니다." 안내 표시
- [ ] AMA CTA 카드 하단에 `ama.amoeba.site/VN????/login` URL 표시

**실패 조건**: 회사 정보 카드 미표시, 회사코드 미표시

---

### TC-02: AMA 로그인 URL에 Entity Code 포함

**사전 조건**: TC-01 완료 화면 상태

**테스트 절차**:
1. 완료 화면에서 "Ạ 로그인하러 가기" 버튼 클릭
2. 브라우저 URL 확인

**기대 결과**:
- [ ] URL이 `https://stg-ama.amoeba.site/{entityCode}/login` 형식
  - 예: `https://stg-ama.amoeba.site/VN1234/login`
- [ ] EntityLoginPage 로드 → entity 자동 조회됨 (entity 이름 표시)
- [ ] ~~기존 `/user/login`이 아님~~ entity code 기반 로그인 페이지

**실패 조건**: URL이 `/user/login`으로 이동, entity 미로딩

---

### TC-03: AMA 첫 로그인 → 비밀번호 변경 강제 리다이렉트

**사전 조건**: TC-01에서 생성된 계정, 이메일로 수신한 임시 비밀번호

**테스트 절차**:
1. AMA EntityLoginPage에서 이메일 + 임시 비밀번호 입력
2. 로그인 버튼 클릭
3. 페이지 리다이렉트 확인

**기대 결과**:
- [ ] 로그인 성공 후 `/force-change-password` 페이지로 자동 리다이렉트
- [ ] "비밀번호 변경" 폼 표시 (현재 비밀번호 + 새 비밀번호 + 확인)
- [ ] 일반 대시보드(`/today`)로 바로 이동하지 않음

**실패 조건**: `/today`로 바로 이동 (mustChangePw 미동작)

---

### TC-04: 비밀번호 변경 후 정상 로그인

**사전 조건**: TC-03에서 비밀번호 변경 페이지 진입 상태

**테스트 절차**:
1. 현재 비밀번호 (임시) 입력
2. 새 비밀번호 (8자 이상, 강도 충족) 입력 + 확인
3. "변경" 버튼 클릭
4. 리다이렉트 확인

**기대 결과**:
- [ ] 비밀번호 변경 성공 메시지/토스트
- [ ] `/today` (대시보드)로 자동 리다이렉트
- [ ] 정상적으로 AMA 서비스 이용 가능
- [ ] 로그아웃 후 새 비밀번호로 재로그인 성공

**실패 조건**: 비밀번호 변경 실패, 리다이렉트 미작동

---

### TC-05: 임시 비밀번호 이메일에 Entity Code 포함

**사전 조건**: TC-01 회원가입 완료 후 이메일 수신

**테스트 절차**:
1. 가입 시 사용한 이메일 수신함 확인
2. "[AMB Management] Your account has been created" 메일 열기
3. 이메일 내용 확인

**기대 결과**:
- [ ] **Company Code: VN????** 행 포함
- [ ] Login URL이 `stg-ama.amoeba.site/VN????/login` 형태
- [ ] "Login Now" 버튼 클릭 시 entity 코드 포함 URL로 이동

**실패 조건**: Company Code 행 누락, Login URL이 `/login` (entity code 없음)

---

### TC-06: i18n 다국어 확인

**테스트 절차**:
1. 포털 사이트에서 언어를 EN으로 변경 → 회원가입 완료 화면 확인
2. 포털 사이트에서 언어를 VI로 변경 → 회원가입 완료 화면 확인

**기대 결과**:
- [ ] EN: "Company Information", "Company Name", "Company Code", "Save this code..."
- [ ] VI: "Thông tin công ty", "Tên công ty", "Mã công ty", "Lưu mã này..."
- [ ] 모든 텍스트가 해당 언어로 표시 (번역 키 누락 없음)

## 3. 엣지 케이스

| # | 케이스 | 기대 동작 |
|---|--------|----------|
| EC-01 | Auto-provision 실패 시 | entityCode 없음 → 회사 정보 카드에 companyName만 표시, AMA URL은 `/user/login` fallback |

| EC-05 | 비밀번호 변경 없이 다른 페이지 시도 | AuthGuard가 `/force-change-password`로 재리다이렉트 |

# TCR-260413 이슈 코멘트 고객사 공개 제어 테스트 케이스

## 1. 단위 테스트

### TC-01: 코멘트 생성 시 기본 client_visible = false
- **입력**: `POST /issues/:id/comments { content: "test" }` (client_visible 미전달)
- **기대**: `isc_client_visible = false`로 저장됨
- **검증**: DB 조회 시 해당 코멘트의 `isc_client_visible` 값 확인

### TC-02: 코멘트 생성 시 client_visible = true
- **입력**: `POST /issues/:id/comments { content: "test", client_visible: true }`
- **기대**: `isc_client_visible = true`로 저장됨
- **검증**: 응답 `clientVisible: true` 확인

### TC-03: 대댓글은 부모 코멘트의 공개 상태를 상속
- **전제**: 부모 코멘트 `client_visible = true`
- **입력**: `POST /issues/:id/comments { content: "reply", parent_id: "parent-uuid" }`
- **기대**: 대댓글의 `isc_client_visible = true` (부모 상속)
- **검증**: DB 조회 확인

### TC-04: 코멘트 공개 상태 토글
- **입력**: `PATCH /issues/:id/comments/:commentId/client-visible { client_visible: true }`
- **기대**: 해당 코멘트 + 하위 대댓글 모두 `true`로 변경
- **검증**: 응답 `{ commentId, clientVisible: true }` + DB 대댓글 확인

### TC-05: 존재하지 않는 코멘트 토글 시 에러
- **입력**: `PATCH /issues/:id/comments/invalid-uuid/client-visible { client_visible: true }`
- **기대**: 404 Not Found 응답

## 2. 통합 테스트 (클라이언트 포털)

### TC-06: 클라이언트가 이슈 상세 조회 시 공개 코멘트만 표시
- **전제**: 이슈에 코멘트 3개 (2개 비공개, 1개 공개)
- **입력**: `GET /client/issues/:id` (CLIENT_LEVEL 사용자)
- **기대**: `comments` 배열에 공개 코멘트 1개만 포함
- **검증**: 응답 `comments.length === 1`

### TC-07: 클라이언트가 작성한 코멘트는 자동 공개
- **입력**: `POST /client/issues/:id/comments { content: "client comment" }`
- **기대**: `isc_client_visible = true`로 자동 저장
- **검증**: DB 조회 + 이슈 상세 조회 시 해당 코멘트 표시

### TC-08: 내부 사용자 이슈 코멘트 조회 시 전체 표시
- **전제**: 이슈에 공개/비공개 코멘트 혼재
- **입력**: `GET /issues/:id/comments` (USER_LEVEL 사용자)
- **기대**: 모든 코멘트 표시 (필터링 없음), `clientVisible` 필드 포함
- **검증**: 응답에 비공개 코멘트도 포함됨

## 3. 엣지 케이스

### TC-09: 비공개 부모 코멘트 + 대댓글 → 클라이언트 조회
- **전제**: 부모 비공개 + 대댓글도 비공개 (상속)
- **입력**: `GET /client/issues/:id`
- **기대**: 부모 + 대댓글 모두 미표시

### TC-10: 부모 코멘트 공개 토글 시 대댓글도 동기 전환
- **전제**: 부모 비공개 + 대댓글 2개
- **입력**: `PATCH ...client-visible { client_visible: true }`
- **기대**: 부모 + 대댓글 2개 모두 공개로 변경

### TC-11: 기존 코멘트 마이그레이션 후 상태 확인
- **전제**: 마이그레이션 SQL 실행 후
- **기대**: 
  - CLIENT_LEVEL 사용자 작성 기존 코멘트: `true`
  - 내부 사용자 작성 기존 코멘트: `false`

### TC-12: AI 코멘트 기본 비공개
- **전제**: AI Review로 생성된 코멘트 (`isc_author_type = 'AI'`)
- **기대**: `isc_client_visible = false` (기본값)

## 4. 데이터 격리 검증

### TC-13: 타 법인 사용자가 코멘트 공개 토글 불가
- USER_LEVEL 사용자는 EntityGuard로 보호됨 (기존 동작 유지)

### TC-14: CLIENT_LEVEL 사용자가 다른 프로젝트 이슈 접근 불가
- 기존 ClientGuard + 프로젝트 접근 권한 검증 유지

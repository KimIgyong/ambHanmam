## 1. 요구사항 요약

로그인한 사용자의 온라인/오프라인 상태를 실시간으로 표시하여, 팀원 간 협업 효율성을 높인다.
웹 PC, 모바일, PWA 모든 환경에서 동일하게 동작해야 한다.

### 2.3 핵심 문제점

**하트비트 초기화 위치가 AmoebaTalkPage에만 한정됨**

```
AmoebaTalkPage.tsx (line 27-28)
  usePresenceHeartbeat();  ← 여기서만 호출
  usePresenceSSE();        ← 여기서만 호출
```

- 사용자가 채팅 페이지(`/amoeba-talk`)에 접속해야만 online 상태로 전환
- 다른 페이지(Today, Todo, Issues 등)에 있으면 offline으로 표시됨
- 모바일/PWA 사용자도 채팅 페이지를 방문해야 online 처리

| ID | 요구사항 | 우선순위 |

## 5. 사용자 플로우

```
1. 사용자 로그인
2. MainLayout 마운트 → usePresenceHeartbeat() 시작
3. 30초마다 POST /talk/presence/heartbeat 전송
4. 서버: presenceMap에 {userId: 'online'} 저장
5. 서버: SSE로 presence:update 이벤트 브로드캐스트
6. 다른 사용자의 ChannelList.tsx에서 녹색 점 표시
7. 5분 유휴 또는 탭 비활성 → 하트비트 중단
8. 60초 후 서버 Cron이 offline 전환 → SSE 이벤트 발행
9. 다른 사용자에게 회색 점으로 변경
```

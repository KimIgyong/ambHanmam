# REQ-멤버관리강화 요구사항 분석서

**작성일**: 2026-02-19
**요청자**: 관리자
**대상 페이지**: https://mng.amoeba.site/settings/members

---

## 1. 배경 및 문제

현재 관리자 멤버 관리 페이지(`/settings/members`)에서는:
- 멤버 목록에서 역할 변경만 가능하고 상세 화면이 없음
- 사용자에게 소속 법인(Entity)을 배정하는 UI가 없음
- 사용자에게 소속 부서(Department)를 배정하는 UI가 없음
- HR 직원 정보와 사용자 계정 간의 연결을 확인/관리할 방법이 없음

## 2. 요구사항 목록

### 2.1 핵심 요구사항

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| R-01 | 멤버 목록에서 사용자명 클릭 시 상세 화면으로 이동 | 필수 |
| R-02 | 멤버 상세 화면: 기본 프로필 정보 표시 | 필수 |
| R-03 | 멤버 상세 화면: 소속 법인 배정/해제 기능 | 필수 |
| R-04 | 멤버 상세 화면: 소속 부서 배정/해제 기능 | 필수 |
| R-05 | 멤버 상세 화면: HR 직원 정보 연결 목록 확인 | 필수 |
| R-06 | 상세 화면에서 목록으로 돌아가기 | 필수 |

### 2.2 상세 화면 구성 섹션

| 섹션 | 내용 |
|------|------|
| 기본 정보 | 이름, 이메일, 역할, 부서, 회사이메일, 가입일 |
| 소속 법인 | 현재 배정된 법인 목록 + 법인 추가/제거 기능 |
| 소속 부서 | 현재 배정된 부서 목록 + 부서 추가/제거, 역할/주요부서 설정 |
| HR 직원 연결 | 해당 법인에 등록된 HR 직원 목록에서 매칭 확인 |

## 3. 기존 시스템 분석

### 3.1 이미 존재하는 백엔드 API

| 기능 | 엔드포인트 | 활용 |
|------|-----------|------|
| 멤버 상세 조회 | `GET /members/:id` | 확장 필요 (법인/부서 포함) |
| 부서 역할 배정 | `POST /departments/roles` | 재사용 가능 |
| 부서 역할 수정 | `PUT /departments/roles/:id` | 재사용 가능 |
| 부서 역할 삭제 | `DELETE /departments/roles/:id` | 재사용 가능 |
| HR 직원 목록 | `GET /hr/employees` | 재사용 가능 |
| 법인 목록 | `GET /settings/entities` (or HrEntity) | 재사용 가능 |

### 3.2 추가 개발 필요

| 기능 | 엔드포인트 | 설명 |
|------|-----------|------|
| 멤버 상세 확장 | `GET /members/:id` 응답 확장 | 법인/부서/HR 정보 포함 |
| 법인 역할 배정 | `POST /members/:id/entity-roles` | 사용자-법인 연결 |
| 법인 역할 해제 | `DELETE /members/:id/entity-roles/:eurId` | 사용자-법인 해제 |

### 3.3 관련 데이터 모델

```
UserEntity (amb_users)
  ├─ 1:N → EntityUserRoleEntity (amb_hr_entity_user_roles) → HrEntityEntity
  ├─ 1:N → UserDeptRoleEntity (amb_user_dept_roles) → DepartmentEntity
  └─ (매칭) → EmployeeEntity (emp_full_name == usr_name, 같은 법인)
```

## 2. AS-IS 현황 분석

### 2.1 검색 (R1)

**현재 구현**: `IssueFilters.tsx:33-42`
- `useDebounce(searchInput, 500)` 적용 → 500ms 후 자동 검색
- 사용자 입력 중간에도 API 호출 발생
- 검색 버튼 없음 — 타이핑 즉시 반응

**문제점**:
- 한글 입력 중 조합 완성 전 검색 발생 가능
- 의도하지 않은 중간 결과로 네트워크 요청 낭비
- 다른 필드(vendor, description 등)와 달리 검색은 명시적 실행이 더 적합

### 2.2 스코프 필터 (R2)

**현재 구현**: `IssueFilters.tsx:89-94`
```
[전체이슈] [내가등록한이슈] [관련이슈] [내Cell]
```
- 4개 고정 스코프 버튼 (radio 선택)
- 작성자별 필터: 없음 (scope='my'는 로그인 사용자 본인만)
- Cell별 선택: 없음 (scope='my_cell'은 내 Cell만, 특정 Cell 선택 불가)

**백엔드 현황**: `issue.service.ts:74-87`
- `scope=my`: `issReporterId = userId`
- `scope=my_involved`: reporter OR assignee
- `scope=my_cell`: UserCellEntity에서 사용자의 Cell 조회 → `issCellId IN (userCellIds)`
- **작성자별 필터(reporter_id)**: 미지원 — 별도 파라미터 필요
- **특정 Cell 선택(cell_id)**: 미지원 — 별도 파라미터 필요

### 2.3 유형/상태 필터 레이아웃 (R3)

**현재 구현**: `IssueFilters.tsx:184-218`
- 유형 칩: 별도 라인 (`<div>` 1줄)
- 상태 칩: 별도 라인 (`<div>` 1줄)
- 2줄로 분리되어 세로 공간 과다 사용

**요구사항**: 같은 라인에 배치, 화면 축소 시 자연스러운 줄바꿈(flex-wrap)

### 2.4 간트차트 계층 표현 (R4)

**현재 구현**: `IssueGanttView.tsx:32-34`
- 이슈를 **플랫 리스트**로 렌더링 (priority 정렬)
- parent-child 관계 시각화 없음
- 좌측 라벨에 제목만 표시, 들여쓰기 없음

**데이터 현황**:
- **DB**: `iss_parent_id` 컬럼 존재 (`issue.entity.ts:61-62`)
- **API**: `parentIssueId`, `parentIssueTitle` 응답에 포함 (`domain.types.ts:1438-1439`)
- **폼**: 등록/수정 시 상위이슈 선택 기능 구현 완료 (`IssueFormModal.tsx:275-307`)
- **상세 모달**: 상위이슈 표시 (`IssueDetailModal.tsx:341-344`)
- **간트차트**: 계층 표현 미구현

## 3. TO-BE 요구사항

### 3.1 R1: 검색 방식 변경

| 항목 | 내용 |
|------|------|
| **변경 내용** | debounce 자동검색 → 검색 버튼 클릭 또는 Enter 키 시 검색 실행 |
| **UI 변경** | 검색 입력 우측에 검색 버튼 추가 |
| **동작** | 텍스트 입력 중에는 API 미호출, 버튼 클릭 or Enter 시 `filters.search` 업데이트 |
| **초기화** | 텍스트 삭제 후 Enter/버튼 → 검색 초기화 |

### 3.2 R2: 작성자별/Cell별 이슈 보기

| 항목 | 내용 |
|------|------|
| **작성자별 필터** | 스코프 버튼 우측에 "작성자" select 드롭다운 추가 |
| **Cell별 필터** | "내Cell" 스코프 대신 Cell select 드롭다운으로 변경 |
| **멤버 목록** | `useMemberList()` 훅 재사용 |
| **Cell 목록** | `useCellList()` 훅 재사용 |
| **백엔드** | `reporter_id`, `cell_id` 필터 파라미터 추가 필요 |

**필터 스코프 재설계**:
```
[전체이슈] [내가등록한이슈] [관련이슈] | 작성자: [select] | Cell: [select]
```
- "내Cell" 스코프 버튼 → Cell select로 대체
- 작성자 select: 전체(기본) / 특정 사용자 선택
- Cell select: 전체(기본) / 특정 Cell 선택
- 작성자/Cell 필터는 스코프와 독립적으로 동작 (AND 조건)

### 3.3 R3: 유형/상태 필터 같은 라인

| **바 표현** | 상위이슈 바 아래에 하위이슈 바 정렬 |

## 5. 기술 제약사항

1. **검색 방식**: debounce 제거 시 한글 조합 이슈 해소, Enter/버튼 방식이 안정적
2. **작성자 필터**: 멤버 수가 많을 경우 select에서 검색 기능 필요 (기존 멤버 select 패턴 재활용)
3. **간트차트 계층**: 무한 깊이 방지 — 현재 DB 구조상 1단계(부모-자식)만 지원
4. **간트차트 성능**: 이슈 수 999개 제한으로 성능 이슈 없음 (기존 limit=999)
5. **Cell 필터**: `useCellList()`는 현재 엔티티의 Cell만 반환 — 요구사항과 일치

## 6. 사용자 플로우

### 6.1 검색 플로우 (R1)
1. 사용자가 검색 입력란에 키워드 입력
2. Enter 키 또는 검색 버튼 클릭
3. `filters.search` 업데이트 → API 재호출
4. 결과 리스트 갱신

### 6.2 작성자별 이슈 보기 (R2)
1. 스코프 "전체이슈" 선택 상태
2. "작성자" 드롭다운에서 특정 사용자 선택
3. 해당 사용자가 등록한 이슈만 필터링
4. 다른 필터(유형, 상태 등)와 AND 조합

### 6.3 Cell별 이슈 보기 (R2)
1. "Cell" 드롭다운에서 특정 Cell 선택
2. 해당 Cell에 속한 이슈만 필터링
3. 작성자 필터와 동시 적용 가능

### 6.4 간트차트 계층 보기 (R4)
1. 간트차트 탭 선택
2. 상위이슈가 있는 이슈는 부모 아래에 들여쓰기로 표시
3. 부모-자식 관계가 시각적으로 연결
4. 상위이슈가 없는 이슈는 최상위에 표시

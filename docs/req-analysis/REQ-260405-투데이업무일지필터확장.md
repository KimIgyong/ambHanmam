## 1. 요구사항 요약

### 요구사항 1: 내 리포트를 업무일지 안에 탭으로 통합
- 현재 **별도 모달**로 열리는 "내 리포트"(AI 분석 리포트)를 **업무일지 페이지 내 탭**으로 이동
- 업무일지 페이지의 탭 순서: **[내 리포트] [일간 리포트] [주간 리포트]**
- 내 리포트 탭에서는 **본인 리포트만** 노출 (scope=me)

### 요구사항 2: Today 페이지 조직별 필터 확장
현재 4개 스코프(나만 | 부서 | 내셀 | 전체) + **"조직별"** 스코프 추가

| 스코프 | 하위 필터 | 동작 |
|--------|----------|------|
| **나만** | 없음 | 기존 그대로 (MyTodayPanel) |
| **부서** | 소속부서명 표시 | 아래 라인에 `소속부서 : {부서명}` 표시 |
| **내셀** | 셀 이름 나열 (선택식) | 처음엔 모두 보기 → 셀 이름 클릭 시 해당 셀만 필터 |
| **전체** | 없음 | 기존 그대로 (AllTodayPanel) |
| **조직별** (신규) | 서브탭 구조 (부서 \| 셀) | 아래 상세 |

**조직별 상세:**
- 서브탭 `[부서]` `[셀]` 전환 방식으로 한 번에 하나만 표시
- **부서 서브탭**: 회사 전체 부서명 나열 (경영 \| 기획 \| 개발 \| 마케팅...)
  - 부서별 **카드뷰**: 부서 소속 매니저, 멤버 리스트
  - 카드 클릭 시 해당 부서 Today 보기
  - "전체부서" 선택 → 모든 부서 카드 표시
  - 특정 부서 선택 → 해당 부서 카드만 표시 + 멤버 Today 리스트
- **셀 서브탭**: 회사 전체 셀 카드뷰 나열
  - 셀 소속 멤버 리스트
  - 셀 카드 클릭 시 해당 셀 Today 보기
  - 중복 선택 가능

**현재 스코프 탭**: `ViewScopeTab` 공유 컴포넌트
- 타입: `ViewScope = 'mine' | 'unit' | 'cell' | 'all'`
- 아이콘: UserCircle2 / Users / MyCellIcon / LayoutGrid

### 2.2 업무일지 페이지 (WorkReportPage.tsx)

- **경로**: `/work-reports` (별도 라우트, MenuGuard: TODAY)
- **탭 구조**: [일간 리포트] [주간 리포트] (2개 탭)
- **레이아웃**: 좌측 사이드바(날짜 선택 + 저장 리포트 목록) + 우측 마크다운 뷰
- **데이터**: 본인의 업무 데이터만 (userId 기반)
- **호출 API**: `POST /api/v1/reports/work/daily/generate`, `GET /api/v1/reports/work`

### 2.3 내 리포트 (ReportsModal)

- **현재 위치**: TodayPage의 모달 (`TodayAiAnalysis.tsx`의 `ReportsModal`)
- **기능**: 저장된 AI 분석 리포트 리스트 + 상세 보기 + 삭제 + 번역
- **데이터**: `GET /api/v1/today/reports?scope=me|team|cell|all`
- scope에 따라 필터 (Today 페이지의 activeScope를 전달)
- 본인에게만 본인 리포트 노출 (usr_id 조건)

### 2.4 Team/All 패널의 필터 현황

| 패널 | 부서 필터 | 셀 필터 | 숨김 필터 |
|------|----------|---------|----------|
| TeamTodayPanel | 없음 (Unit 소속 멤버만) | 없음 | MASTER: showHidden 토글 |
| CellTodayPanel | 없음 (Cell 소속 멤버만) | 없음 | 없음 (TeamTodayPanel 재사용) |
| AllTodayPanel | **있음** (data.members에서 추출한 동적 드롭다운) | 없음 | MASTER: showHidden 토글 |

### 2.5 조직 데이터 구조

**Unit (부서)**:
- 테이블: `amb_units` (unt_id, ent_id, unt_name, unt_parent_id, unt_level, unt_is_active)
- API: `GET /api/v1/entity-settings/organization/units`
- 관계: UserUnitRoleEntity (`amb_user_unit_roles` - usr_id, unt_id, uur_role, uur_is_primary)

**Cell (셀)**:
- 테이블: `amb_cells` (cel_id, ent_id, cel_name, cel_description)
- API: `GET /api/v1/entity-settings/organization/cells`
- 관계: UserCellEntity (`amb_user_cells` - usr_id, cel_id)

**Employee (직원)**:
- `emp_department` (문자열, 50자) — MemberTodaySummary.department 출처
- `emp_position` (문자열, 50자) — MemberTodaySummary.position 출처

**MemberTodaySummary 응답 필드**:
- `department?: string` — Employee 테이블의 `emp_department`
- `unitName?: string` — UserUnitRole → UnitEntity의 `unt_name` (primary unit)
- `position?: string` — Employee 테이블의 `emp_position`

### 2.6 관련 백엔드 API

### 3.2 Today 스코프 탭 확장

**기존**: `나만 | 부서 | 내셀 | 전체`
**TO-BE**: `나만 | 부서 | 내셀 | 전체 | 조직별`

#### 부서 탭 개선
```
[나만] [부서*] [내셀] [전체] [조직별]

소속부서 : 경영
────────────────────────────────

```
[나만] [부서] [내셀*] [전체] [조직별]

```
[나만] [부서] [내셀] [전체] [조직별*]
────────────────────────────────
   [부서]  [셀]               ← 서브탭: 부서 | 셀 전환
────────────────────────────────

== 부서 서브탭 선택 시 ==
[전체부서] 경영 | 기획 | 개발 | 마케팅     ← 부서 칩 필터
────────────────────────────────
┌─ 경영 ────┐  ┌─ 기획 ────┐             ← 부서별 카드뷰
│ 매니저: 홍  │  │ 매니저: 김  │
│ 멤버: 3명  │  │ 멤버: 5명  │
│ [클릭→팀] │  │ [클릭→팀] │
└───────────┘  └───────────┘

== 셀 서브탭 선택 시 ==
[전체셀] 프로젝트1 | 파트1 | 디자인팀      ← 셀 칩 필터
────────────────────────────────
┌─ 프로젝트1 ─┐  ┌─ 파트1 ────┐          ← 셀별 카드뷰
│ 멤버: 4명    │  │ 멤버: 3명  │
│ [클릭→셀]   │  │ [클릭→셀] │
└──────────────┘  └───────────┘
```

**부서 서브탭:**
- 회사 전체 부서(Unit) 목록 조회 (`GET /entity-settings/organization/units`)
- 부서별 카드: 부서명, 부서 소속 매니저/멤버 수, 클릭 시 부서별 Today
- "전체부서" 선택 → 모든 부서 카드 표시

**셀 서브탭:**
- 회사 전체 셀 목록 조회 (`GET /entity-settings/organization/cells`)
- 셀별 카드: 셀명, 소속 멤버 리스트 표시
- 셀 카드 클릭 시 해당 셀 Today 보기
- 중복 선택 가능

## 6. 기술 제약사항

1. **ViewScopeTab 공유 컴포넌트**: 다른 모듈(이슈, 할일 등)에서도 사용 중 → 직접 수정 시 전체 영향
   - 해결: TodayPage 내에서 자체 탭 컴포넌트 사용 또는 ViewScopeTab에 `scopes` prop 추가

2. **백엔드 getCellToday()**: 현재 사용자 소속 전체 셀 멤버 반환 (셀 단위 필터 없음)
   - 셀별 필터링은 프론트에서 처리 가능 (MemberTodaySummary에 cell 정보 부재)
   - 해결: 프론트 필터링 또는 백엔드에 `cellId` 쿼리 파라미터 추가

3. **MemberTodaySummary에 cell 정보 없음**: 현재 응답에 멤버가 속한 셀 목록이 없음
   - 해결: 백엔드 응답에 `cellIds: string[]` 또는 `cellNames: string[]` 필드 추가

4. **조직별 탭의 "부서별 매니저/멤버 수"**: 현재 AllToday API에 Unit별 그룹핑 없음
   - 해결: AllToday 데이터를 프론트에서 unitName/department 기준으로 그룹핑

5. **셀별 멤버 목록**: `GET /cells/:id/members` API 존재하나, 카드뷰에서는 요약만 필요
   - 해결: 셀 목록 + AllToday 데이터를 조합하여 셀별 그룹핑

6. **i18n**: 신규 키 다수 추가 필요 (ko/en/vi 3개 언어)

7. **성능**: 조직별 탭에서 전체 부서 + 전체 셀 + 전체 멤버 데이터 → 대규모 엔티티에서 부하
   - 해결: AllToday 데이터 재사용 + 프론트 그룹핑 (추가 API 호출 최소화)

## 1. AS-IS 현황 분석

### 1.1 현재 사이드바 구조

```
MainLayout.tsx
└── renderMenuItems()
    ├── [카테고리 헤더] Work Tools
    │   ├── 제목 버튼 (클릭 시 /menu/work-tools 이동) ← 구현됨
    │   ├── ChevronUp/Down 버튼 (접기/펼치기) ← 구현됨
    │   └── 메뉴 아이템 목록
    ├── [카테고리 헤더] Work Modules
    │   ├── 제목 버튼 (클릭 시 /menu/work-modules 이동) ← 구현됨
    │   ├── ChevronUp/Down 버튼 (접기/펼치기) ← 구현됨
    │   └── 메뉴 아이템 목록
    └── [카테고리 헤더] Custom Apps
        ├── 제목 버튼 (클릭 시 /menu/custom-apps 이동) ← 구현됨
        ├── ChevronUp/Down 버튼 (접기/펼치기) ← 구현됨
        └── Custom App 목록
```

### 1.4 AS-IS 문제점 및 개선 포인트

#### 🔴 버그/누락

| # | 문제 | 위치 | 영향 |
|---|------|------|------|
| B-01 | 모바일 드로어에서 카테고리 제목 클릭 시 드로어가 닫히지 않음 | `renderMenuItems()` — `inDrawer: true` 케이스 | 모바일 사용성 저하 |
| B-02 | 사이드바 `collapsed` 상태에서 카테고리 섹션 헤더 전체가 숨겨져 카테고리 구분 불가 | `renderMenuItems()` — `isCollapsed: true` 케이스 | 접힌 사이드바에서 카테고리 접근 불가 |

#### 🟡 UX 개선

| # | 개선점 | 현재 | 목표 |
|---|--------|------|------|
| U-01 | 카드뷰 페이지 간 내비게이션 없음 | 뒤로가기(`navigate(-1)`)만 제공 | 3개 카테고리 탭 전환 UI 추가 |
| U-02 | 카드뷰 페이지 진입 시 카테고리 식별이 약함 | 제목 텍스트만 표시 | 카테고리별 아이콘/색상 배지 추가 |
| U-03 | 카드뷰에서 현재 활성 메뉴 표시 미흡 | `isActive()` 체크는 있으나 시각 강조 약함 | 활성 카드에 뚜렷한 강조 스타일 적용 |
| U-04 | 카드뷰 페이지 홈(/) 역할 없음 | 메뉴별 개별 페이지만 존재 | 전체 메뉴를 한 번에 보는 통합 홈뷰 고려 |
| U-05 | 모바일 카드뷰 페이지에서 뒤로가기 UX | `navigate(-1)` 히스토리 의존 | 명시적 "메인으로" 버튼 개선 |

#### 🟢 신규 기능 (선택)

| # | 기능 | 설명 |
|---|------|------|
| N-01 | 카드뷰 통합 홈페이지 (`/menu`) | 3개 카테고리를 섹션으로 묶어 한 화면에서 탐색 |
| N-02 | 메뉴 즐겨찾기 | 자주 쓰는 메뉴를 카드뷰 상단에 고정 |

## 2. TO-BE 요구사항

### 2.1 핵심 요구사항 (필수)

#### R-01: 사이드바 카테고리 헤더 개선
- **카테고리 제목 클릭** → 해당 카테고리 카드뷰 페이지로 이동 *(현재 구현됨, 모바일 버그 수정 필요)*
- **접기/펼치기 버튼** → 카테고리 우측에 `ChevronUp/Down` 버튼 *(현재 구현됨)*
- **모바일 드로어** 카테고리 제목 클릭 시 드로어 닫힘 처리 추가 *(버그 수정)*

#### R-02: 카테고리 카드뷰 페이지 3개 독립 운영
- `/menu/work-tools` → Work Tools 카드뷰
- `/menu/work-modules` → Work Modules 카드뷰
- `/menu/custom-apps` → Custom Apps 카드뷰
- *(이미 구현됨 — UX 개선 필요)*

#### R-03: 카드뷰 페이지 간 탭 내비게이션
- 카드뷰 페이지 상단에 **3개 카테고리 탭** 추가
- 탭 클릭으로 Work Tools ↔ Work Modules ↔ Custom Apps 전환
- 현재 카테고리 탭이 활성 상태로 강조됨

#### R-04: 카드뷰 권한 노출 기준 — 사이드바와 동일
- `useMyMenus()` 훅 기반 필터링 유지
- Custom Apps: `useMyCustomApps()` 훅 기반 필터링 유지
- 유저 등급(Admin/Master/User)/유닛/셀 기반 노출은 API 응답 기반이므로 별도 프론트 처리 불필요

### 2.2 개선 요구사항 (권장)

#### R-05: 사이드바 Collapsed 상태 개선
- 접힌 상태에서도 카테고리 구분선 또는 카테고리 아이콘 표시
- 카테고리 구분선(divider)으로 시각적 분리

#### R-06: 카드뷰 페이지 헤더 개선
- 뒤로가기 → 명시적으로 "홈으로" 또는 "사이드바로" 이동
- 카테고리별 색상/아이콘 배지로 시각 식별성 강화

## 3. 사용자 플로우

### 3.1 데스크탑 사이드바 클릭 플로우

```
사이드바 [Work Tools 제목 클릭]
  → /menu/work-tools (카드뷰)
    → 탭: [Work Tools★] [Work Modules] [Custom Apps]
      → 카드 클릭 → 해당 메뉴 페이지로 이동

사이드바 [▼ 버튼 클릭]
  → Work Tools 섹션 접힘 (하위 메뉴 숨김)
  → [▲ 버튼 클릭] → 펼침
```

### 3.2 모바일 드로어 플로우 (버그 수정 후)

```
드로어 열기 [≡ 버튼]
  → [Work Tools 제목 클릭]
    → 드로어 닫힘 (setDrawerOpen(false) 추가)
    → /menu/work-tools (카드뷰) 이동
```

### 3.3 카드뷰 내 탭 전환 플로우

```
/menu/work-tools
  → 탭 [Work Modules] 클릭
    → /menu/work-modules 이동 (navigate, 히스토리 유지)
  → 뒤로가기
    → 이전 페이지 복귀
```

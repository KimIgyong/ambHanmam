# REQ-260406 — a.amoeba.site 도메인 추가

## 1. 요구사항 요약

프로덕션 서버 nginx에 `a.amoeba.site` 도메인을 추가한다. `ama.amoeba.site`와 동일한 설정으로 생성하며, 향후 `ama.amoeba.site`를 삭제하고 `a.amoeba.site`만 사용할 예정이다.

---

## 2. AS-IS 현황 분석

### 2-1. 프로덕션 서버 인프라

| 항목 | 값 |
|------|-----|
| 서버 IP | 18.138.206.18 (AWS Singapore) |
| SSH | `ssh amb-production` |
| nginx conf 위치 | `/etc/nginx/conf.d/` |
| SSL 인증서 | `/etc/nginx/ssl/amoeba.site.crt` (와일드카드 `*.amoeba.site`) |
| 인증서 만료 | 2026-04-13 |

### 2-2. 현재 ama.amoeba.site nginx 설정

**파일**: `/etc/nginx/conf.d/ama.amoeba.site.conf`

```
HTTP (80)  → 301 HTTPS 리다이렉트
HTTPS (443):
  ├─ /api   → 127.0.0.1:3009 (amb-api 컨테이너)
  └─ /      → 127.0.0.1:8080 (amb-web 컨테이너)
```

- Security headers: X-Content-Type-Options, X-Frame-Options, CSP, HSTS 등
- client_max_body_size: 50M
- proxy timeout: 120s (API만)

### 2-3. DNS 현황

| 도메인 | IP | 상태 |
|--------|-----|------|
| `ama.amoeba.site` | 18.138.206.18 | ✅ 등록됨 |
| `a.amoeba.site` | 18.138.206.18 | ✅ 이미 등록됨 |

### 2-4. 백엔드 환경변수 (도메인 관련)

| 변수 | 현재 값 | 영향 |
|------|---------|------|
| `CORS_ORIGINS` | `https://ama.amoeba.site,https://stg-apps.amoeba.site,https://apps.amoeba.site` | 🔴 `a.amoeba.site` 미포함 → API 호출 CORS 차단 |
| `FRONTEND_URL` | `https://ama.amoeba.site` | 🟡 이메일 링크, 리다이렉트 URL (당장 미변경) |
| `PORTAL_CORS_ORIGINS` | `https://www.amoeba.site,https://ama.amoeba.site` | 🔴 `a.amoeba.site` 미포함 |

### 2-5. 쿠키 설정

```typescript
// auth.service.ts - setTokenCookies()
res.cookie('access_token', token, {
  httpOnly: true,
  secure: true,       // production
  sameSite: 'lax',
  path: '/',
  // domain 미지정 → 요청 도메인에 자동 귀속
});
```

- `domain` 명시 없음 → `a.amoeba.site`에서 로그인하면 쿠키가 `a.amoeba.site`에 귀속
- `ama.amoeba.site`와 `a.amoeba.site` 간 쿠키 공유 불가 (각각 독립 세션)
- **문제 없음**: 두 도메인은 병행 운영이지 동시 세션 공유 필요 없음

### 2-6. SSL 인증서 커버리지

```
Subject: CN=*.amoeba.site
SAN: DNS:*.amoeba.site, DNS:amoeba.site
```

- `a.amoeba.site` → `*.amoeba.site` 와일드카드에 매칭 ✅
- 추가 인증서 불필요

## 3. TO-BE 요구사항

### 3-1. nginx 설정 추가
- `/etc/nginx/conf.d/a.amoeba.site.conf` 신규 생성
- `ama.amoeba.site.conf`와 동일 구조 (server_name, 로그 파일명만 변경)
- 동일 SSL 인증서 사용

### 3-2. 백엔드 CORS 허용
- `CORS_ORIGINS`에 `https://a.amoeba.site` 추가
- `PORTAL_CORS_ORIGINS`에 `https://a.amoeba.site` 추가

### 3-3. 향후 마이그레이션 계획 (이번 작업 범위 외)
- `ama.amoeba.site` → `a.amoeba.site` 완전 전환 시:
  - `FRONTEND_URL` 변경
  - GA4 도메인 설정 변경
  - 포털(`www.amoeba.site`) 내 AMA 링크 변경
  - `ama.amoeba.site` → `a.amoeba.site` 301 리다이렉트 설정

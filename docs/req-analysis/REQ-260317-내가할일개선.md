## 1. AS-IS 현황 분석

### 1.1 데이터 모델 (amb_todos)

| 컬럼 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `tdo_id` | UUID | PK | |
| `tdo_status` | VARCHAR(20) | 상태 (SCHEDULED / IN_PROGRESS / COMPLETED) | **수동 관리** |
| `tdo_due_date` | DATE | 마감일 | NOT NULL, 필수 입력 |
| `tdo_completed_at` | TIMESTAMP | 완료 시각 | ✅ 이미 존재 |
| `tdo_started_at` | TIMESTAMP | 최초 진행 시각 | ✅ 이미 존재 |
| `tdo_recurrence_type` | VARCHAR(20) | 반복 유형 | DAILY/WEEKLY/MONTHLY/YEARLY |
| `tdo_visibility` | VARCHAR(20) | 공개 범위 | PRIVATE/CELL/ENTITY |
| `tdo_parent_id` | UUID | 부모 할일 (서브태스크) | |
| `iss_id` | UUID | 연결된 이슈 | |
| `pjt_id` | UUID | 연결된 프로젝트 | |

**핵심 발견**: `tdo_completed_at`, `tdo_started_at` 컬럼이 **이미 존재**하며 실제로 사용 중. 초안에서 제안한 `tdo_start_date`(시작 예정일)와 `tdo_cancelled_at`만 미존재.

### 1.2 현재 상태 관리 방식

- **상태 3종**: SCHEDULED → IN_PROGRESS → COMPLETED (수동 순환)
- **상태 변경 로그**: `amb_todo_status_logs` 테이블에 모든 변경 이력 기록
- **완료 시각 추적**: status = COMPLETED 전환 시 `tdo_completed_at` = NOW() 설정, 되돌리기 시 NULL
- **시작 시각 추적**: 최초 IN_PROGRESS 전환 시 `tdo_started_at` = NOW() (1회성)
- **반복 할일**: 완료 시 다음 반복 할일 자동 생성

### 1.3 현재 API 구조

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `GET /todos` | GET | 내 할일 목록 (status, date_from, date_to 필터) |
| `GET /todos/cell` | GET | 그룹 할일 |
| `GET /todos/unit` | GET | 유닛 할일 |

## 2. TO-BE 요구사항 (초안 기반 + 시스템 적합성 분석)

### 2.1 핵심 변경: Computed Status 도입

> **DB의 `tdo_status` 컬럼을 조회 시점에 날짜 기반으로 자동 계산하여 반환한다.**

| Status | 라벨 | 조건 | 배지 |
|--------|------|------|------|
| `COMPLETED` | 완료 | `tdo_completed_at IS NOT NULL` | ✓ 완료 |
| `OVERDUE` | 지연 | `due_date < today` AND 미완료 | D+N |
| `DUE_TODAY` | 오늘 마감 | `due_date = today` AND 미완료 | D-Day |
| `IN_PROGRESS` | 진행 중 | `start_date ≤ today (또는 null)` AND `due_date > today (또는 null)` | D-N |
| `UPCOMING_SOON` | 곧 시작 | `start_date > today` AND `start_date ≤ today + 3일` | N일 후 시작 |
| `UPCOMING` | 시작 예정 | `start_date > today + 3일` | 시작일 표시 |

### 2.2 데이터 모델 변경

#### 신규 추가 컬럼

| 컬럼 | 타입 | Nullable | 설명 |
|------|------|----------|------|
| `tdo_start_date` | DATE | YES | 시작 예정일. NULL = 생성일 기준 즉시 시작 |
| `tdo_cancelled_at` | TIMESTAMPTZ | YES | 취소 시각 |

#### 기존 컬럼 활용 (변경 불필요)

| 컬럼 | 현재 상태 | 활용 |
|------|----------|------|
| `tdo_completed_at` | ✅ 이미 존재 & 사용 중 | 그대로 활용 |
| `tdo_started_at` | ✅ 이미 존재 & 사용 중 | 참고용 유지 (computed status와 무관) |
| `tdo_due_date` | ✅ 이미 존재 (NOT NULL) | **NULL 허용으로 변경** (기한 없는 할일 지원) |
| `tdo_status` | ✅ 이미 존재 | Phase 1: 읽기 전용 유지, Phase 2: 삭제 예정 |

### 2.3 초안과의 갭 분석

| 초안 제안 | 현재 시스템 | 적용 판단 |
|----------|------------|----------|
| `tdo_start_date` 추가 | 미존재 | ✅ 추가 필요 |
| `tdo_completed_at` 추가 | **이미 존재** | ⏭️ 스킵 — 추가 불필요 |
| `tdo_cancelled_at` 추가 | 미존재 (`tdo_deleted_at`으로 소프트 삭제) | ⚠️ 선택적 — 취소≠삭제 구분이 필요한 경우만 |
| `tdo_status` 삭제 | 3종 상태 수동 관리 중 | ✅ Phase 1: 유지(호환성), Phase 2: 삭제 |
| `due_date` 필수 → 선택 | 현재 NOT NULL | ✅ NULL 허용으로 변경 |
| 상태 선택 UI 제거 | 현재 폼에서 드롭다운 제공 | ✅ 제거 |
| 완료/되돌리기 전용 API | 현재 PATCH로 status 변경 | ⚠️ 기존 PATCH 방식 유지하되 내부 로직 변경 |
| `computeTodoStatus()` 유틸 | 미존재 | ✅ 추가 필요 |
| 5섹션 UI 재구성 | 현재 4컬럼 | ✅ 변경 필요 |

## 3. 사용자 플로우

### 3.1 할일 생성 (변경 후)

```
사용자 → [할일 작성 폼]
  ├─ 제목 * (필수)
  ├─ 설명 (선택)
  ├─ 시작일 [날짜 선택] → 비우면 "오늘부터 바로"
  ├─ 마감일 [날짜 선택] → 비우면 "기한 없음"
  ├─ 태그 (선택)
  ├─ 이슈/프로젝트 연결 (선택)
  ├─ 참여자 (선택)
  └─ 반복 설정 (선택)

상태 선택 UI 없음 → 시스템이 날짜 기반 자동 결정
```

### 3.2 상태 자동 결정 (Decision Tree)

```
할일 1개 조회 시점
  │
  ├─ tdo_cancelled_at IS NOT NULL? ──Y──▶ CANCELLED (목록에서 숨김)
  │
  ├─ tdo_completed_at IS NOT NULL? ──Y──▶ COMPLETED
  │
  └─ 미완료
      ├─ due_date < today? ──────────Y──▶ OVERDUE (D+N 경과)
      │
      ├─ due_date = today? ──────────Y──▶ DUE_TODAY (D-Day)
      │
      ├─ start_date ≤ today (or null)? ─Y──▶ IN_PROGRESS (D-N 잔여)
      │
      ├─ start_date ≤ today + 3일? ──Y──▶ UPCOMING_SOON (N일 후 시작)
      │
      └─ ──▶ UPCOMING (시작일 표시)
```

### 3.3 할일 완료/되돌리기

```
[할일 카드] 체크박스 클릭
  → tdo_completed_at = NOW()
  → tdo_status = 'COMPLETED' (레거시 호환)
  → 반복 설정 시 다음 할일 자동 생성

[완료 항목] 되돌리기 클릭
  → tdo_completed_at = NULL
  → tdo_status → computeTodoStatus()로 재계산하여 레거시 값 갱신
```

## 5. 기술 제약사항 및 사이드 임팩트

### 5.1 호환성 유지 (Phase 1)

| 항목 | 처리 방식 |
|------|----------|
| `tdo_status` 컬럼 | 읽기 전용 유지. 완료/되돌리기 시 레거시 값도 동시 갱신 |
| 기존 API 필터 `?status=IN_PROGRESS` | computed status 기반으로 내부 변환 (날짜 조건으로 재해석) |
| `tdo_due_date` NOT NULL 제약 | NULL 허용으로 ALTER 필요 |
| 반복 할일 생성 로직 | `start_date` 고려하여 다음 반복 시작일도 계산 |

### 5.2 Today 페이지 연동 영향

| 컴포넌트 | 영향 | 필요 작업 |
|----------|------|----------|
| MyTodayPanel | overdue + inProgress 조회 쿼리 | computed status 기반으로 전환 |
| Team/Cell/All 카드뷰 | todayDueCount, overdueCount 등 집계 | 날짜 기반 조건으로 전환 |
| TodayService (백엔드) | `WHERE tdo_status = 'IN_PROGRESS'` 등 | 날짜 조건 쿼리로 변경 |

### 5.3 상태 로그 호환성

- `amb_todo_status_logs` 테이블의 `tsl_from_status`, `tsl_to_status`에 기존 3종 값(SCHEDULED/IN_PROGRESS/COMPLETED) 사용 중
- computed status 도입 후에도 완료/되돌리기 시에만 로그 기록 (COMPLETED ↔ 이전상태)
- 마감일 변경 로그는 기존 방식 유지

### 5.4 Validation 규칙

| 규칙 | 설명 |
|------|------|
| `start_date` > `due_date` | 시작일이 마감일보다 늦으면 오류 |
| `start_date`, `due_date` 모두 NULL | 허용 — 기간 없는 상시 할일 (IN_PROGRESS) |
| `start_date` NULL, `due_date` 설정 | 허용 — 즉시 시작, 기한 있음 |
| `start_date` 설정, `due_date` NULL | 허용 — 시작일만 있고 기한 없음 |

## 9. computeTodoStatus() 핵심 로직 (초안 채택)

```typescript
export type TodoComputedStatus =
  | 'COMPLETED'
  | 'CANCELLED'
  | 'OVERDUE'
  | 'DUE_TODAY'
  | 'IN_PROGRESS'
  | 'UPCOMING_SOON'
  | 'UPCOMING';

export function computeTodoStatus(
  todo: {
    tdo_start_date: Date | null;
    tdo_due_date: Date | null;
    tdo_completed_at: Date | null;
    tdo_cancelled_at?: Date | null;
  },
  today: Date = new Date(),
  upcomingSoonDays = 3,
): TodoComputedStatus {
  const t = startOfDay(today);

  if (todo.tdo_cancelled_at) return 'CANCELLED';
  if (todo.tdo_completed_at) return 'COMPLETED';

  const due = todo.tdo_due_date ? startOfDay(todo.tdo_due_date) : null;
  const start = todo.tdo_start_date ? startOfDay(todo.tdo_start_date) : null;

  if (due) {
    if (isBefore(due, t)) return 'OVERDUE';
    if (isEqual(due, t)) return 'DUE_TODAY';
  }

  if (!start || !isBefore(t, start)) return 'IN_PROGRESS';

  if (!isBefore(addDays(t, upcomingSoonDays), start)) return 'UPCOMING_SOON';
  return 'UPCOMING';
}
```

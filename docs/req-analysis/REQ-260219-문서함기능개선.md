- **대상 페이지**: `/documents` (문서함)

## 1. 개선 항목

### 1.1 업로드 버튼 미표시 (버그/UX)
- **현상**: 폴더 진입 후에도 업로드 버튼이 보이지 않음
- **원인 분석**: 브라우저 캐시로 인해 이전 버전 렌더링되거나, 업로드 버튼의 조건부 표시 위치가 사용자 시선에서 벗어남
- **해결**: 업로드 버튼을 브레드크럼 영역 근처에 더 눈에 띄게 배치

### 1.2 하위 폴더 생성 (Admin 전용)
- **요구사항**: Admin 사용자가 현재 탐색 중인 폴더에 하위 폴더를 생성할 수 있어야 함
- **AS-IS**: 폴더 생성 기능 없음 (Google Drive에서 직접 생성해야 함)
- **TO-BE**: Admin에게 "새 폴더" 버튼 제공 → 이름 입력 → Google Drive에 폴더 생성

### 1.3 카드형 보기 지원
- **요구사항**: 리스트형 외에 카드(그리드)형 보기 지원
- **AS-IS**: 리스트형만 지원
- **TO-BE**: 리스트/카드 토글 버튼, 카드형에서 썸네일·파일명·크기·수정일 표시

### 1.4 KMS 태그 분석 적용
- **요구사항**: 문서함에 업로드된 파일에 KMS 키워드/태그 자동 분석 적용
- **AS-IS**: Drive 모듈에서 `ModuleDataEvent` 미발행, KMS 연동 없음
- **TO-BE**: 파일 업로드 시 `module.data.created` 이벤트 발행 → WorkItem 생성 → 자동 태깅
- **참고**: `ModuleIntegrationService.tagFromDrive()` 메서드 이미 구현됨 (파일명/폴더명에서 태그 추출)

## 2. 기술 분석

### 2.1 백엔드
- `GoogleDriveService.createFolder()` 이미 구현 → 컨트롤러 엔드포인트만 추가
- `EventEmitter2` 글로벌 설정 완료 → DriveService에서 emit만 추가
- `tagFromDrive()` 구현 완료 → 이벤트 발행하면 자동 연결

### 2.2 프론트엔드
- 뷰 모드 토글: `useState<'list' | 'grid'>` + localStorage 저장
- 카드 컴포넌트: 기존 `FileListItem` 패턴을 카드형으로 확장
- 폴더 생성: prompt 또는 모달로 이름 입력

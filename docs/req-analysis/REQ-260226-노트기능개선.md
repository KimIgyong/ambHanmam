## 1. 요구사항 요약

노트(Meeting Notes) 모듈에 다음 6가지 기능을 추가한다:

| # | 기능 | 설명 |
|---|------|------|
| R1 | 관련 프로젝트 선택 | 노트에 관련 프로젝트를 연결 (복수 선택 가능) |
| R2 | 담당자 지정 | 노트에 책임 담당자를 1명 지정 |
| R3 | 관련자 태그 | 노트에 관련자를 복수로 태그 |
| R4 | 관련 이슈 선택 | 노트에 관련 이슈를 연결 (복수 선택 가능) |
| R5 | 코멘트 작성 | 노트에 대한 코멘트(댓글) 작성/삭제 |
| R6 | 좋아요(별점) | 노트에 별 1~5점 평가 (사용자당 1회) |

## 2. AS-IS 현황 분석

### 2.1 데이터베이스

| 현재 테이블 | 설명 |
|------------|------|
| `amb_meeting_notes` | 단일 테이블, 자식 테이블 없음 |

**현재 컬럼 구조:**
- `mtn_id` (UUID PK), `ent_id` (FK nullable), `usr_id` (FK, 작성자)
- `mtn_title` (varchar 200), `mtn_content` (text), `mtn_type` (varchar 20)
- `mtn_meeting_date` (date), `mtn_visibility` (varchar 20), `mtn_department` (varchar 30)
- `mtn_group_id` (UUID nullable), `mtn_original_lang` (varchar 5)
- `mtn_created_at`, `mtn_updated_at`, `mtn_deleted_at`

**미지원 기능:**
- ❌ 프로젝트 연결 (N:M 관계 없음)
- ❌ 담당자 지정 (작성자만 존재, 별도 담당자 없음)
- ❌ 관련자 태그 (참여자 테이블 없음)
- ❌ 이슈 연결 (N:M 관계 없음)
- ❌ 코멘트 (댓글 테이블 없음)
- ❌ 별점/좋아요 (평가 테이블 없음)

## 3. TO-BE 요구사항 상세

### 3.1 R1: 관련 프로젝트 선택 (N:M)

**설명:** 노트 작성/수정 시 관련 프로젝트를 복수 선택하여 연결한다.

**데이터 모델:**
```
amb_meeting_note_projects (Junction Table)
├── mnp_id (UUID PK)
├── mtn_id (UUID FK → amb_meeting_notes, CASCADE)
└── pjt_id (UUID FK → kms_projects, CASCADE)
UNIQUE(mtn_id, pjt_id)
```

**UI:**
- Form Modal: 프로젝트 드롭다운 (멀티 셀렉트, 현재 법인 프로젝트 목록)
- Detail Page: 연결된 프로젝트명 태그 표시 (클릭 시 프로젝트 상세 이동)
- Card: 프로젝트 카운트 아이콘 표시

### 3.2 R2: 담당자 지정

**설명:** 노트에 담당 책임자를 1명 지정한다. 작성자와 다를 수 있다.

**데이터 모델:**
```
amb_meeting_notes 컬럼 추가
└── mtn_assignee_id (UUID FK → amb_users, nullable)
```

**비즈니스 로직:**
- 담당자는 현재 법인 소속 사용자 중 선택
- 작성자 ≠ 담당자 가능
- 담당자 미지정 허용 (nullable)

**UI:**
- Form Modal: 사용자 드롭다운 (단일 선택)
- Detail Page: 담당자 이름 표시 (작성자 옆)
- Card: 담당자 아바타/이름 표시

### 3.3 R3: 관련자 태그 (N:M)

**설명:** 노트에 관련자를 복수로 태그한다. Todo의 참여자와 동일 패턴.

**데이터 모델:**
```
amb_meeting_note_participants (Junction Table)
├── mnpt_id (UUID PK)
├── mtn_id (UUID FK → amb_meeting_notes, CASCADE)
└── usr_id (UUID FK → amb_users, CASCADE)
UNIQUE(mtn_id, usr_id)
```

**동기화 방식:** Todo 모듈의 `syncParticipants()` 패턴 사용 (delete-all + re-insert)

**UI:**
- Form Modal: 사용자 멀티 셀렉트 (현재 법인 소속 사용자)
- Detail Page: 관련자 목록 표시 (아바타 + 이름)
- Card: 관련자 수 표시

### 3.4 R4: 관련 이슈 선택 (N:M)

**설명:** 노트에 관련 이슈를 복수 선택하여 연결한다.

**데이터 모델:**
```
amb_meeting_note_issues (Junction Table)
├── mni_id (UUID PK)
├── mtn_id (UUID FK → amb_meeting_notes, CASCADE)
└── iss_id (UUID FK → amb_issues, CASCADE)
UNIQUE(mtn_id, iss_id)
```

**UI:**
- Form Modal: 이슈 드롭다운 (멀티 셀렉트, 현재 법인 이슈 목록)
- Detail Page: 연결된 이슈 목록 (제목 + 상태 배지, 클릭 시 이슈 상세 이동)
- Card: 이슈 카운트 아이콘 표시

### 3.5 R5: 코멘트 작성

**설명:** 노트 상세 페이지에서 코멘트(댓글)를 작성/삭제한다.

**데이터 모델:**
```
amb_meeting_note_comments
├── mnc_id (UUID PK)
├── mtn_id (UUID FK → amb_meeting_notes, CASCADE)
├── mnc_author_id (UUID FK → amb_users)
├── mnc_content (text)
├── mnc_created_at (timestamp)
├── mnc_updated_at (timestamp)
└── mnc_deleted_at (timestamp, soft delete)
```

**비즈니스 로직:**
- 노트 접근 권한이 있는 사용자만 코멘트 가능
- 코멘트 삭제는 작성자만 가능 (soft delete)
- 코멘트 작성 시 이벤트 발생 (향후 알림 연동)

**API 엔드포인트:**
| Endpoint | 메서드 | 설명 |
|----------|--------|------|
| `/api/v1/meeting-notes/:id/comments` | GET | 코멘트 목록 |
| `/api/v1/meeting-notes/:id/comments` | POST | 코멘트 작성 |
| `/api/v1/meeting-notes/:id/comments/:commentId` | DELETE | 코멘트 삭제 |

**UI:**
- Detail Page 하단: 코멘트 섹션 (목록 + 입력 폼)
- 각 코멘트: 작성자명, 날짜, 내용, 삭제 버튼(본인만)

### 3.6 R6: 좋아요(별점 1~5)

**설명:** 노트에 대해 사용자가 별 1~5점 평가를 남긴다. 사용자당 노트당 1회.

**데이터 모델:**
```
amb_meeting_note_ratings
├── mnr_id (UUID PK)
├── mtn_id (UUID FK → amb_meeting_notes, CASCADE)
├── usr_id (UUID FK → amb_users, CASCADE)
├── mnr_rating (smallint, CHECK 1~5)
├── mnr_created_at (timestamp)
└── mnr_updated_at (timestamp)
UNIQUE(mtn_id, usr_id)
```

**비즈니스 로직:**
- 1~5점 범위 (서버 유효성 검사)
- 이미 평가한 경우 업데이트 (UPSERT)
- 평점 삭제 가능
- 노트 상세 응답에 평균 평점 + 평가 수 + 내 평점 포함

**API 엔드포인트:**
| Endpoint | 메서드 | 설명 |
|----------|--------|------|
| `/api/v1/meeting-notes/:id/rating` | PUT | 평점 등록/수정 |
| `/api/v1/meeting-notes/:id/rating` | DELETE | 평점 삭제 |

**UI:**
- Detail Page: 별점 행 (★★★★☆ 형태, 클릭 시 즉시 반영)
- Card: 평균 별점 표시 (소수 1자리)

## 9. 기술 제약사항

1. **TypeORM Entity 등록**: 새 Entity는 모듈의 `TypeOrmModule.forFeature()` + `app.module.ts` entities 배열 양쪽에 등록 필요
2. **DB 마이그레이션**: TypeORM synchronize:true 사용 시 자동 생성, 프로덕션은 마이그레이션 필요
3. **i18n**: 3개 언어(ko/en/vi) 모두 번역 키 추가 필요
4. **접근 제어**: 기존 visibility 기반 접근 제어가 코멘트/별점에도 적용되어야 함
5. **이벤트 시스템**: 코멘트 작성 시 EventEmitter2 이벤트 발행 (향후 알림 연동)

## 10. 사용자 플로우

### 10.1 노트 작성 플로우
```
노트 작성 버튼 → FormModal 열기
  → 제목 입력
  → 타입 선택 (메모/회의록)
  → 내용 작성 (RichTextEditor)
  → [NEW] 담당자 선택 (선택)
  → [NEW] 관련자 태그 (복수 선택)
  → [NEW] 관련 프로젝트 선택 (복수 선택)
  → [NEW] 관련 이슈 선택 (복수 선택)
  → 공개 범위 설정
  → 저장
```

### 10.2 노트 상세 플로우
```
카드 클릭 → 상세 페이지
  → 제목/타입/공개범위
  → [NEW] 담당자 / 관련자 태그 표시
  → [NEW] 관련 프로젝트/이슈 링크
  → 본문 콘텐츠
  → [NEW] 별점 입력 (★★★★☆)
  → [NEW] 코멘트 섹션
    → 코멘트 목록 (작성자, 날짜, 내용)
    → 코멘트 입력 폼
```

## 1. 요구사항 요약

ambManagement/ambKMS에서 **법인별·사용자별 AI API 사용량을 관리**하고, 법인별 토큰 쿼터를 설정·모니터링하는 기능.

### 핵심 요구사항
1. **시스템 관리자** (`/settings`) - AI 사용량 카드 추가, 법인별 쿼터 설정
2. **법인 관리자** (`/entity-settings`) - AI 사용량 페이지에 충전 목업 기능
3. **잔량 기반 3단계 안내** - 10%, 5%, 0% 경고 시스템

## 3. TO-BE 요구사항

### 3.1 시스템 관리자 페이지 (`/settings/ai-usage`)

**접근 권한**: `ADMIN_LEVEL` (SUPER_ADMIN / ADMIN)

#### 3.1.1 카드 추가
- `SettingsPage.tsx`의 SETTINGS_CARDS 배열에 **"AI 사용량 관리"** 카드 추가
- menuCode: `SETTINGS_AI_USAGE`
- 아이콘: `BarChart3` (차트 아이콘)

#### 3.1.2 법인별 AI 사용량 대시보드 페이지

**상단 요약 카드 (3개)**:
| 카드 | 내용 |
|------|------|
| 전체 월간 토큰 | 모든 법인 합계 총 토큰, 전월 대비 증감률 |
| 전체 월간 요청 수 | 모든 법인 합계 요청 건수 |
| 활성 법인 수 | 이번 달 AI 사용한 법인 수 / 전체 법인 수 |

**법인별 사용량 테이블**:
| 컬럼 | 설명 |
|------|------|
| 법인명 | 법인 코드 + 이름 |
| 월간 총 토큰 | 이번 달 총 사용량 |
| 월간 요청 수 | 이번 달 API 호출 건수 |
| 일일 한도 | 설정된 일일 토큰 한도 (미설정 시 "제한 없음") |
| 월간 한도 | 설정된 월간 토큰 한도 (미설정 시 "제한 없음") |
| 사용률 | 월간 한도 대비 사용률 Progress Bar |
| 초과 시 처리 | WARN / BLOCK 배지 |
| 작업 | [쿼터 설정] 버튼 |

**쿼터 설정 모달**:
- 법인명 표시 (읽기 전용)
- 일일 토큰 한도: 숫자 입력 + "제한 없음" 체크박스
- 월간 토큰 한도: 숫자 입력 + "제한 없음" 체크박스
- 초과 시 처리: WARN(경고만) / BLOCK(차단) 라디오
- [저장] [취소] 버튼

### 3.2 법인 관리자 페이지 (`/entity-settings/usage`)

**접근 권한**: `USER_LEVEL` + `MASTER` 역할

#### 3.2.1 기존 EntityUsagePage 개선

**현재**: 일별·월별 사용량 + 쿼터 정보 표시
**추가 사항**:

1. **잔량 기반 3단계 경고 배너** (기존 80%/95% → 변경):

| 잔량 | 단계 | 배너 색상 | 메시지 |
|------|------|-----------|--------|
| ≤ 10% | 주의 | 황색(amber) | "AI 쿼터가 10% 남았습니다. 충전을 고려해 주세요." |
| ≤ 5% | 경고 | 주황색(orange) | "AI 쿼터가 5% 남았습니다. 충전하셔야 합니다." |
| 0% | 중단 | 적색(red) | "AI 어시스턴트 사용이 일시 중단되었습니다." |

2. **충전하기 버튼** (목업):
   - 쿼터 정보 카드 영역에 [충전하기] 버튼 배치
   - 클릭 시 **충전 요청 모달** 표시 (목업)
   - 모달 내용:
     - 현재 잔량 표시
     - 충전 패키지 선택 (예: 1M / 5M / 10M / 50M 토큰)
     - 예상 비용 표시 (목업 가격)
     - [충전 요청] 버튼 → "충전 요청이 접수되었습니다. 관리자 승인 후 적용됩니다." 토스트 표시
     - 실제 결제/충전 로직은 미구현 (목업만)

### 3.3 잔량 경고 시스템

#### 3.3.1 백엔드 경고 로직 수정

**현재 `getEntityUsageSummary()`**:
- `dailyWarning` (≥80%), `dailyCritical` (≥95%)
- `monthlyWarning` (≥80%), `monthlyCritical` (≥95%)

**변경: 잔량 기반 3단계**:
```
remainingPercent = 100 - usagePercent

stage1: remainingPercent ≤ 10% (사용률 ≥ 90%) → "주의"
stage2: remainingPercent ≤ 5%  (사용률 ≥ 95%) → "경고"
stage3: remainingPercent ≤ 0%  (사용률 ≥ 100%) → "중단"
```

**응답 필드 추가**:
```typescript
warnings: {
  daily: {
    remainingPercent: number;
    stage: 'NORMAL' | 'CAUTION' | 'WARNING' | 'SUSPENDED';
  };
  monthly: {
    remainingPercent: number;
    stage: 'NORMAL' | 'CAUTION' | 'WARNING' | 'SUSPENDED';
  };
}
```

#### 3.3.2 프론트엔드 표시
- `EntityUsagePage`에서 stage에 따라 배너 색상·메시지·아이콘 변경
- stage가 'SUSPENDED'일 때 충전하기 버튼 강조 (pulse 애니메이션)

### 3.4 AI 토큰 사용량 실제 기록 연동

#### 3.4.1 ChatService 연동 (필수 선행 작업)
- `ChatService.sendMessage()` 메서드에서 AI 응답 완료 후 `AiUsageService.recordUsage()` 호출
- Claude API의 `usage` 응답에서 실제 `inputTokens`, `outputTokens` 추출
- 현재 `Math.ceil(fullContent.length / 4)` 임시값 → 실제 값으로 교체

#### 3.4.2 ApiQuotaInterceptor 적용
- AI 에이전트 API 요청 시 쿼터 초과 여부를 사전 차단
- `entityApiQuota.eaqActionOnExceed === 'BLOCK'` 인 경우 API 호출 전 차단
- SUSPENDED 상태의 법인에 대해 AI 관련 API 호출 차단

## 5. 사용자 흐름

### 5.1 시스템 관리자 (SUPER_ADMIN / ADMIN)
```
[설정 페이지] → [AI 사용량 관리 카드 클릭]
    → [법인별 사용량 대시보드]
        → 법인별 사용량 테이블 확인
        → [쿼터 설정] 버튼 클릭
            → 쿼터 설정 모달
            → 일일/월간 한도 입력 또는 "제한 없음" 선택
            → 초과 시 처리 방식 선택 (WARN / BLOCK)
            → [저장] → 성공 토스트
```

### 5.2 법인 관리자 (MASTER)
```
[법인 설정] → [AI 사용량 클릭]
    → [현재 사용량 확인] (일별/월별 + QuotaBar)
    → 잔량 ≤ 10%: 황색 배너 "충전을 고려해 주세요"
    → 잔량 ≤ 5%: 주황색 배너 "충전하셔야 합니다"
    → 잔량 = 0%: 적색 배너 "사용이 중단됩니다"
    → [충전하기] 버튼 클릭
        → 충전 요청 모달 (목업)
        → 패키지 선택
        → [충전 요청] → "요청이 접수되었습니다" 토스트
```

### 5.3 일반 사용자 (AI 사용 시)
```
[AI 에이전트 채팅]
    → 메시지 전송
    → (백엔드) ApiQuotaInterceptor 쿼터 확인
        → 잔량 있음: 정상 응답 + 토큰 사용량 기록
        → WARN 상태: 정상 응답 + 로그 기록
        → BLOCK 상태 (0%): "AI 사용이 일시 중단되었습니다" 에러 반환
```

## 6. 기술 제약사항

1. **토큰 기록 선행 필수**: Chat/Agent 서비스에서 실제 토큰 사용량을 기록해야 쿼터 모니터링 의미가 있음
2. **쿼터 "제한 없음"**: `daily_token_limit = NULL` 또는 `monthly_token_limit = NULL`일 때 한도 없음으로 처리 (기존 로직 유지)
3. **충전 기능 목업**: 실제 결제/충전 로직 없이 UI만 구현 → 추후 PG 연동 시 확장
4. **경고 체계 변경**: 기존 80%/95% 기준을 유지하되, 잔량 관점(10%/5%/0%)으로 UI 메시지 변경
5. **권한 분리**: `/settings/ai-usage`는 ADMIN_LEVEL만 접근, `/entity-settings/usage`는 법인 MASTER 접근

### 3.2 핵심 갭 상세

#### 갭 1: 개인 드라이브 (내 드라이브) 폴더 탐색 미지원

**현재**: 설정 페이지에서 "찾아보기" 클릭 시 `listSharedDrives()`만 호출하여 공유 드라이브만 표시.
내 드라이브 루트(`root`)의 폴더는 조회 불가.

**필요**: 위임 이메일 사용자의 "내 드라이브" 루트 폴더도 탐색 가능해야 함.
Google Drive API에서 `folderId = 'root'`로 내 드라이브 접근 가능.

**해결 방향**:
- 드라이브 브라우저에 "내 드라이브" 항목 추가
- `listSharedDrives()` 결과에 "내 드라이브 (My Drive)" 가상 항목 prepend
- 선택 시 `folderId = 'root'`로 하위 폴더 조회

#### 갭 2: Billing 전용 필드명 → 범용 "회사 공용 폴더"로 확장

**현재**: `drsBillingRootFolderId`, `drsBillingRootFolderName` → Billing 모듈 전용 용도.

**필요**: 이 필드가 "회사 공용 문서함" 폴더를 가리키도록 의미 확장.
기존 Billing 문서 관리 기능과의 하위 호환성 유지 필요.

**해결 방향**:
- DB 컬럼은 그대로 유지 (마이그레이션 부담 없음)
- DTO/응답 인터페이스에서 `companyRootFolderId` / `companyRootFolderName` 별칭 추가 또는
- 기존 `billingRootFolderId`를 "회사 공용 폴더"로 재해석 (UI 라벨만 변경)

#### 갭 3: 설정 저장 시 문서함 자동 등록

**현재**: 설정 페이지에서 폴더 지정 → `amb_drive_settings`에만 저장.
문서함(`DocumentsPage`)에서 해당 폴더를 보려면 별도로 `POST /drive/folders`로 등록해야 함.

**필요**: 설정에서 폴더 지정 시 `amb_drive_folders`에 자동 등록되어
문서함 페이지에서 바로 열람 가능.

**해결 방향**:
- `DriveSettingsService.updateSettings()` 에서 폴더 저장 시
  `DriveService.registerFolder()` 자동 호출
- 이미 등록된 경우 skip

## 5. 기술 제약사항

### 5.1 Google Drive API 인증 방식

현재 시스템은 **Service Account + Domain-Wide Delegation** 방식을 사용합니다.
이 방식은 다음 전제 조건이 필요합니다:

1. **Google Workspace 도메인 관리자**가 서비스 계정에 "도메인 전체 위임" 권한 부여
2. **위임 이메일**은 해당 Google Workspace 도메인의 사용자여야 함
3. 개인 Gmail 계정은 Domain-Wide Delegation 미지원

> **참고**: OAuth2 개인 인증 방식은 프론트엔드 redirect 필요, 토큰 갱신 관리 복잡성 등으로
> 현재 아키텍처에서는 채택하지 않습니다. Service Account 위임 방식이 서버 간 통신에 적합합니다.

### 5.2 "내 드라이브" 접근

Google Drive API에서 위임 사용자의 "내 드라이브"는 `folderId = 'root'`로 접근 가능합니다.
단, 해당 사용자의 Drive에 `ambCoWork` 같은 폴더가 존재해야 합니다.

## 6. 영향도 분석

### 6.1 영향받는 모듈

| 모듈 | 영향 수준 | 변경 내용 |
|------|-----------|-----------|
| Settings (Backend) | **중** | DriveSettingsService에 개인 Drive 탐색 + 자동 등록 로직 추가 |
| Settings (Frontend) | **중** | DriveSettingsPage 폴더 브라우저에 "내 드라이브" 추가 |
| Drive (Backend) | **저** | 자동 등록 시 DriveService 메서드 호출 (기존 로직 활용) |
| Documents (Frontend) | **없음** | 등록된 폴더 자동 표시 (변경 없음) |
| Billing (Backend) | **없음** | `getBillingRootFolderId()` 그대로 동작 |
| i18n | **저** | 설정 페이지 라벨 일부 업데이트 |
| 공유 타입 | **없음** | 기존 타입 그대로 활용 |

### 6.2 하위 호환성

- `drsBillingRootFolderId`는 Billing 문서 관리(`BillingDocumentService`)에서도 사용 중
- 동일 필드를 "회사 공용 폴더"로 재해석하더라도 Billing 기능에는 영향 없음
  (Billing은 이 폴더 하위에 자체 계층 구조를 생성)
- 기존 `amb_drive_folders` 등록 메커니즘과 충돌 가능성 검토 필요
  → `drfFolderId` unique 제약 → 중복 등록 방지 로직 필요

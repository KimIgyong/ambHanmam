# REQ-20260404-드라이브미등록가이드카드

## 1. 요구사항 요약 (Requirement Summary)
- KR: `/drive`에서 폴더가 등록되지 않은 사용자에게 `구글드라이브 연동 가이드` 카드뷰를 노출하고, 가이드 문서 링크를 제공한다.
- EN: Show a `Google Drive Integration Guide` card for users without registered folders on `/drive`, including a direct guide link.
- KR: `apps/portal-web/public/service/ama/ama-google-drive-guide.html` 파일을 기준으로 스테이징 → 프로덕션 순서로 배포한다.
- EN: Deploy the guide file in sequence: staging first, then production.

## 2. AS-IS 현황 분석 (Current State)
- KR: `/drive` 화면에서 미설정 상태(`isConfigured=false`)와 폴더 미등록 상태(`folders.length===0 && filteredSharedDrives.length===0`)는 안내 문구만 표시하고 외부 가이드 링크는 없다.
- EN: In `/drive`, both unconfigured and no-folder states show plain messages only, without a guide link.
- KR: 가이드 HTML은 `portal-web` 정적 리소스에 존재하며, 웹앱(`/drive`)에서 직접 연결하는 UI는 없다.
- EN: The guide HTML exists in portal-web static assets, but no direct UI entry exists from `/drive`.

## 3. TO-BE 요구사항 (Target State)
- KR: `/drive` 미등록 사용자 상태에서 카드뷰 형태의 가이드 안내를 표시한다.
- EN: Display a card-style guide prompt for unregistered-user states on `/drive`.
- KR: 카드에는 제목/설명/버튼을 제공하고, 버튼 클릭 시 가이드 문서를 새 탭으로 연다.
- EN: Card includes title/description/button; button opens guide in a new tab.
- KR: 환경별 링크를 구분한다.
- EN: Use environment-aware guide URLs.
: staging (`stg-ama`) -> `https://stg-www.amoeba.site/service/ama/ama-google-drive-guide.html`
: production (`ama`) -> `https://www.amoeba.site/service/ama/ama-google-drive-guide.html`

## 4. 갭 분석 (Gap Analysis)
- KR: 현재 `/drive`는 사용자 행동 유도(How-to) 링크가 없어 설정 실패/미등록 상황에서 이탈 가능성이 높다.
- EN: Current `/drive` lacks an actionable guide entry, increasing drop-off in setup-failure scenarios.
- KR: 가이드 문서는 존재하지만 진입점이 없어 발견성이 낮다.
- EN: Guide exists but discoverability is low due to missing entry point.

## 5. 사용자 플로우 (User Flow)
- KR:
1. 사용자가 `/drive` 진입
2. 시스템이 드라이브 설정/등록 폴더 상태 확인
3. 미설정 또는 미등록 상태이면 `구글드라이브 연동 가이드` 카드 노출
4. 사용자가 `가이드 열기` 클릭
5. 가이드 HTML 새 탭 오픈
- EN:
1. User enters `/drive`
2. System checks drive configured and folder registration status
3. If unconfigured or no-folder, show guide card
4. User clicks `Open Guide`
5. Guide HTML opens in a new tab

## 6. 기술 제약사항 (Technical Constraints)
- KR: `/drive`는 `apps/web` 애플리케이션, 가이드 정적 파일은 `apps/portal-web`에 있어 도메인 분기가 필요하다.
- EN: `/drive` is in `apps/web`, while guide static asset is in `apps/portal-web`; domain branching is required.
- KR: UI 텍스트는 i18n(`ko/en/vi`) 번역키를 사용해야 하며 하드코딩 금지.
- EN: UI strings must use i18n keys (`ko/en/vi`) without hardcoding.
- KR: 배포는 스테이징 검증 후 프로덕션 순차 진행.
- EN: Deployment must be staging-first, then production.

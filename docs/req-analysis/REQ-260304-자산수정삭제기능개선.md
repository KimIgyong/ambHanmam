# 요구사항 분석서 (REQ)

- 문서명: 자산 수정/삭제 기능 개선
- 작성일: 2026-03-04
- 대상 화면: `/assets`

## 1) AS-IS 현황
- 자산 목록에서 상세조회(Eye)만 가능하며 전체 필드 수정 UI가 없음.
- 상세 모달은 상태 변경만 지원(`PATCH /assets/:id/status`).
- 자산 삭제 API/버튼이 없음.

## 2) TO-BE 요구사항
- 등록된 자산의 입력 항목 전체 수정 가능.
- 삭제 버튼 동작 가능.
- 권한은 기존 자산 관리 정책(MANAGER/ADMIN) 유지.

## 3) 갭 분석
- 백엔드: 전체 수정/삭제 엔드포인트 부재.
- 프론트: 상세 모달 내 전체수정/삭제 UX 부재.
- i18n: 삭제 확인 문구 및 전체수정 라벨 부재.

## 4) 사용자 플로우
1. 자산 목록에서 상세 열기
2. `전체수정` 클릭 → 전체 필드 편집 후 저장
3. `삭제` 클릭 → 확인 후 삭제
4. 목록 자동 갱신

## 5) 기술 제약/주의사항
- 삭제는 soft delete(`ast_deleted_at`)로 처리.
- Request DTO는 snake_case 유지.
- 기존 상태변경 기능과 충돌 없이 공존.

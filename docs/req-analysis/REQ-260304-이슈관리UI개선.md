# REQ - 이슈관리 화면 UI 및 검색필터 기능 개선

- **작성일**: 2026-03-04
- **요청자**: 내부
- **영향 모듈**: Issues (이슈관리)

---

## 1. AS-IS 현황 분석

### 1.1 범위 필터 (Scope Filter)

| 항목 | 현재 상태 |
|------|----------|
| 필터 옵션 | 전체 / 내가 등록한 이슈 / 내 Cell (3개) |
| UI 형태 | 버튼 그룹 (bg-gray-100 배경, 선택 시 white + indigo) |
| 백엔드 scope 값 | `undefined`(전체), `my`, `my_cell` |
| `my` 필터 | `iss_reporter_id = userId` (작성자만) |
| `my_cell` 필터 | `UserCellEntity`로 사용자 Cell 조회 → `iss_cell_id IN (:cellIds)` |

**문제점:**
- "내가 관련된 이슈" (작성자 OR 담당자) 범위가 없음
- 기존 `GET /issues/my` 엔드포인트에서는 `reporterId OR assigneeId` 조회를 지원하지만, 목록 scope 필터에는 미적용

### 1.2 유형/상태 필터 (Type & Status Filter)

| 항목 | 현재 상태 |
|------|----------|
| UI 형태 | `<select>` 드롭다운 (단일 선택) |
| 유형 옵션 | BUG, FEATURE_REQUEST, OPINION, TASK, OTHER (5개) |
| 상태 옵션 | OPEN, APPROVED, IN_PROGRESS, RESOLVED, CLOSED, REJECTED (6개) |
| 백엔드 처리 | `iss_type = :type`, `iss_status = :status` (단일 값 비교) |

**문제점:**
- 드롭다운 방식으로 한 번에 하나의 유형/상태만 선택 가능
- 여러 상태를 동시에 보고 싶은 경우(예: OPEN + IN_PROGRESS) 불가능
- 필터 상태를 한눈에 파악하기 어려움

### 1.3 필터 프리셋 (Filter Preset)

- 현재 **필터 저장 기능 없음**
- 페이지 이동 시 필터 초기화
- 사용자마다 자주 사용하는 필터 조합이 다르나 매번 수동 설정 필요

### 1.4 담당자/상태 변경 권한

| 기능 | 현재 권한 체크 |
|------|--------------|
| `updateIssue` (PATCH /issues/:id) | **권한 체크 없음** - 누구나 담당자(`assignee_id`) 포함 모든 필드 수정 가능 |
| `updateIssueStatus` (PATCH /issues/:id/status) | reporter/assignee/MANAGER/ADMIN만 가능 |
| `approveIssue` / `rejectIssue` | MANAGER 이상만 (RolesGuard) |
| 프론트엔드 상태 변경 | `canChangeStatus = isManager \|\| isInvolved` 조건으로 워크플로우 버튼 표시 |
| 프론트엔드 수정 버튼(Edit) | IssueDetailModal에서 **항상 표시** (권한 체크 없음) |

**문제점:**
- `updateIssue` 서비스에 권한 검증이 없어 일반 사용자가 타인의 이슈 담당자를 변경 가능
- 프론트엔드 Edit 버튼이 모든 사용자에게 노출됨

## 2. TO-BE 요구사항

### 2.1 범위 필터 확장

- 기존 3개 범위에 **"내가 관련된 이슈"** 추가 (총 4개)
  - `전체`: 모든 이슈 (기존 유지)
  - `내가 등록한 이슈` (`my`): reporter가 본인인 이슈
  - **`내가 관련된 이슈` (`my_involved`)**: reporter OR assignee가 본인인 이슈 (신규)
  - `내 Cell` (`my_cell`): 본인 소속 Cell의 이슈 (기존 유지)

### 2.2 유형/상태 체크박스 필터

- Type과 Status를 **체크박스 멀티셀렉트** 방식으로 변경
- 여러 유형/상태를 동시에 선택 가능
- 선택된 항목은 칩(chip) 또는 체크 표시로 시각적 표시
- 백엔드: 쉼표 구분 문자열 또는 배열 형태로 다중 값 전달 → `IN (:...values)` 조건

### 2.3 필터 프리셋 저장

- 사용자별 필터 프리셋 저장/불러오기/삭제 기능
- 저장 항목: scope, type[], status[], severity, priority, project_id
- 프리셋 이름을 지정하여 최대 5개까지 저장
- 저장 위치: `UserEntity.usrTranslationPrefs`와 동일한 JSONB 패턴 → **새 컬럼 `usr_issue_filter_presets`** (JSONB)
- 형태: `[{ name: string, filters: { ... } }]`

### 2.4 담당자/상태 변경 권한 강화

- **`updateIssue`** 서비스에 권한 체크 추가:
  - 작성자 본인(reporter) → 모든 필드 수정 가능
  - 담당자(assignee) → 제한된 필드만 수정 가능 (resolution, done_ratio 등)
  - MANAGER/ADMIN → 모든 필드 수정 가능
  - 그 외 → 403 Forbidden
- 프론트엔드 Edit 버튼: reporter/assignee/MANAGER/ADMIN만 표시

## 4. 기술 제약사항

- `usr_issue_filter_presets` JSONB 컬럼 추가 시 스테이징/프로덕션은 수동 `ALTER TABLE` 필요 (synchronize: false)
- 멀티셀렉트 필터 파라미터: 쿼리스트링에서 `type=BUG,TASK` (쉼표 구분) 방식이 가장 간단
- 기존 단일 값 필터(`type`, `status`)를 다중 값으로 전환 시 하위 호환성 유지 필요 (단일 값도 계속 동작)
- 필터 프리셋 API는 기존 `/users/me/preferences` 패턴(없으면 신규) 또는 `PATCH /users/me` 확장 활용
- `UserCellEntity` 리포지토리는 이미 IssueService에 주입되어 있어 추가 모듈 설정 불필요

## 5. 사용자 플로우

### 5.1 필터 사용 플로우
1. 이슈 관리 페이지 진입
2. 범위 버튼 그룹에서 원하는 범위 선택 (전체/내가 등록/내가 관련/내 Cell)
3. 유형 체크박스에서 복수 유형 선택 (예: BUG + TASK)
4. 상태 체크박스에서 복수 상태 선택 (예: OPEN + IN_PROGRESS)
5. 실시간 목록 갱신

### 5.2 필터 프리셋 저장 플로우
1. 원하는 필터 조합 설정
2. "필터 저장" 버튼 클릭
3. 프리셋 이름 입력 (예: "내 진행중 버그")
4. 저장 완료 → 프리셋 목록에 추가
5. 다음 접속 시 프리셋 버튼 클릭으로 즉시 적용

### 5.3 권한 기반 수정 플로우
1. 이슈 상세 모달 열기
2. 작성자/담당자/MANAGER만 Edit 버튼 표시
3. Edit 클릭 → 수정 폼 (역할에 따라 수정 가능 필드 제한)
4. 저장 → 백엔드 권한 검증 → 성공/403 에러

## 1. 요구사항 요약

### 1.1 변경 요청 사항
1. **Todos 페이지 제목 변경**: "오늘의 할일 / Today's Tasks" → "나의 할일 / My Task"
2. **"My Cell" 탭 참조 대상 조사**: 현재 Cell 엔티티를 참조하는지, Unit 엔티티를 참조하는지 확인
3. **뷰 권한 탭 통합**: My Task, Notes, Calendar, Issues 페이지의 뷰 권한 탭을 아래 4개로 통일
   - 내것만보기 | 내가 속한 Unit 보기 | 내가 속한 Cell 보기 | 회사 전체 보기
4. **Today 페이지 탭 노출**: Today 페이지도 통합 탭 구조를 적용하되, 모든 사용자에게 노출

## 2. AS-IS 현황 분석

### 2.1 조직 구조 엔티티

| 엔티티 | 테이블 | 특징 | 계층 |
|--------|--------|------|------|
| **Cell** (`amb_cells`) | cel_id, cel_name, ent_id | 아메바 셀 (최소 조직 단위) | 단일 레벨 |
| **Unit** (`amb_units`) | unt_id, unt_name, unt_parent_id, unt_level | 부서 (계층 구조) | Level 1=Unit, 2=Team |
| **UserCell** (`amb_user_cells`) | usr_id ↔ cel_id | 사용자-Cell 매핑 | - |
| **UserUnitRole** (`amb_user_unit_roles`) | usr_id ↔ unt_id, uur_role | 사용자-Unit 매핑 + 역할 | MEMBER/TEAM_LEAD/UNIT_HEAD |

### 2.2 페이지별 현재 탭/필터 구조

#### (1) Todos 페이지 (`/todos`)
- **컴포넌트**: `apps/web/src/domain/todos/pages/TodoPage.tsx`
- **페이지 제목**: `todos.title` = "오늘의 할일" / "Today's Tasks" / "Việc cần làm hôm nay"
- **탭 구조**: `mine | cell | company` (3개 탭)
- **Cell 탭 구현**: `UserCellEntity`의 `celId`로 셀 멤버 조회 → **Cell 참조**
- **API**: `GET /todos` (mine), `GET /todos/cell` (cell), `GET /todos/company` (company)
- **권한 제한**: 없음 (모든 사용자 3개 탭 접근 가능)

#### (2) Meeting Notes 페이지 (`/meeting-notes`)
- **컴포넌트**: `apps/web/src/domain/meeting-notes/pages/MeetingNotesPage.tsx`
- **필터 구조**: `All | PRIVATE | CELL | ENTITY` (가시성 필터)
- **구현 방식**: 노트 생성 시 `mtnVisibility` 설정 → 조회 시 필터링
- **Cell/Unit 동시 사용**: `mtnUnit` (부서), `mtnCellId` (셀) 모두 저장
- **API**: `GET /meeting-notes?visibility=CELL|ENTITY`

#### (3) Calendar 페이지 (`/calendar`)
- **컴포넌트**: `apps/web/src/domain/calendar/pages/CalendarPage.tsx`
- **필터 구조**: `ALL | MY | UNIT | SHARED` (Zustand 스토어)
- **Unit 탭 구현**: `UserUnitRoleEntity`로 부서 계층 구조 조회 → **Unit 참조**
- **API**: `GET /calendars?filter_mode=MY_ONLY|UNIT|SHARED`
- **Cell 미사용**: Calendar에는 Cell 개념 없음

#### (4) Issues 페이지 (`/issues`)
- **컴포넌트**: `apps/web/src/domain/issues/pages/IssuesPage.tsx`
- **필터 구조**: `All | My | My Involved` (Scope 필터)
- **추가 필터**: `cell_id` 파라미터로 Cell 필터링 가능 (별도 드롭다운)
- **백엔드**: `my_cell` scope 지원 (코드 존재하나 프론트에서 미사용)
- **API**: `GET /issues?scope=my|my_involved|my_cell&cell_id=XX`

#### (5) Today 페이지 (`/today`)
- **컴포넌트**: `apps/web/src/domain/today/pages/TodayPage.tsx`
- **탭 구조**: `my | team | all` (역할 기반 가시성)
- **권한 제한**:
  - `my`: 모든 사용자
  - `team`: MANAGER / MASTER / ADMIN / SUPER_ADMIN만
  - `all`: MASTER / ADMIN / SUPER_ADMIN만
- **패널**: MyTodayPanel, TeamTodayPanel, AllTodayPanel

### 2.3 현황 비교 매트릭스

| 페이지 | 내것만 | Unit 범위 | Cell 범위 | 회사 전체 | 기타 |
|--------|--------|-----------|-----------|-----------|------|
| Todos | ✅ mine | ❌ | ✅ cell | ✅ company | - |
| Notes | ✅ PRIVATE | ❌ (백엔드만) | ✅ CELL | ✅ ENTITY | All 필터 |
| Calendar | ✅ MY | ✅ UNIT | ❌ | ✅ ALL | SHARED |
| Issues | ✅ my | ❌ | ⚠️ (cell_id만) | ✅ all (기본) | my_involved |
| Today | ✅ my | ❌ | ❌ | ⚠️ all (권한제한) | team (권한제한) |

## 3. TO-BE 요구사항

### 3.1 통합 뷰 권한 탭 구조

모든 대상 페이지에 아래 4개 탭을 동일하게 적용:

| 순번 | 탭 ID | 한국어 | English | Vietnamese | 설명 |
|------|-------|--------|---------|------------|------|
| 1 | `mine` | 내것만보기 | My Only | Chỉ của tôi | 현재 사용자의 데이터만 |
| 2 | `unit` | 내 Unit 보기 | My Unit | Unit của tôi | 사용자가 속한 부서(Unit)의 데이터 |
| 3 | `cell` | 내 Cell 보기 | My Cell | Cell của tôi | 사용자가 속한 셀(Cell)의 데이터 |
| 4 | `all` | 회사 전체 보기 | All | Tất cả | 회사(Entity) 전체 데이터 |

### 3.2 페이지별 적용 범위

| 페이지 | 내것만 | Unit | Cell | 전체 | 권한 제한 |
|--------|--------|------|------|------|-----------|
| **Todos** (/todos) | ✅ | ✅ 신규 | ✅ 유지 | ✅ 유지 | 없음 |
| **Notes** (/meeting-notes) | ✅ | ✅ 신규 | ✅ 유지 | ✅ 유지 | 없음 |
| **Calendar** (/calendar) | ✅ | ✅ 유지 | ✅ 신규 | ✅ 유지 | 없음 |
| **Issues** (/issues) | ✅ | ✅ 신규 | ✅ 활성화 | ✅ 유지 | 없음 |
| **Today** (/today) | ✅ | ✅ 신규 | ✅ 신규 | ✅ 기존→제한 해제 | **모든 사용자** |

### 3.3 Todos 페이지 제목 변경

| 항목 | AS-IS | TO-BE |
|------|-------|-------|
| 번역 키 | `todos.title` | `todos.title` (동일) |
| 한국어 | 오늘의 할일 | 나의 할일 |
| English | Today's Tasks | My Tasks |

## 4. 갭 분석

### 4.1 프론트엔드 갭

| 페이지 | 필요 작업 | 난이도 |
|--------|----------|--------|
| **Todos** | Unit 탭 추가 (mine→unit→cell→all), 제목 변경 | 중 |
| **Notes** | PRIVATE→Unit→Cell→Entity 탭 재구성, 필터 로직 변경 | 중 |
| **Calendar** | Cell 탭 추가, 필터 모드 확장 | 중 |
| **Issues** | Scope 필터에 unit/cell 탭 추가, my_involved 제거/이동 | 중 |
| **Today** | 4-탭 구조로 전환, team/all 역할 제한 해제 | 고 |

### 4.2 백엔드 갭

| 모듈 | 필요 API 변경 | 난이도 |
|------|-------------|--------|
| **Todos** | `GET /todos/unit` 엔드포인트 추가 (Unit 멤버 조회) | 중 |
| **Notes** | Unit 범위 필터링 로직 추가 | 중 |
| **Calendar** | Cell 범위 필터링 로직 추가 (`filter_mode=CELL`) | 중 |
| **Issues** | `scope=my_unit` 추가, `my_cell` 프론트 연동 | 저 |
| **Today** | Unit/Cell 범위 데이터 조회 API (AllTodayPanel 범위 축소) | 고 |

### 4.3 데이터 의존성

- **UserCellEntity**: 사용자가 Cell에 소속되지 않은 경우 → Cell 탭은 빈 결과
- **UserUnitRoleEntity**: 사용자가 Unit에 소속되지 않은 경우 → Unit 탭은 빈 결과
- **참고**: 모든 사용자가 반드시 Cell과 Unit 모두에 소속되어 있는지 확인 필요

## 5. 사용자 플로우

### 5.1 탭 전환 플로우

```
사용자 → [Todos 페이지 진입]
       → 기본 탭: "내것만보기" 활성
       → "내 Unit 보기" 클릭
       → API: GET /todos/unit
       → 사용자 Unit 멤버 전체 할일 표시
       → "내 Cell 보기" 클릭
       → API: GET /todos/cell
       → 사용자 Cell 멤버 전체 할일 표시
       → "회사 전체 보기" 클릭
       → API: GET /todos/company
       → 회사 전체 할일 표시
```

### 5.2 Today 페이지 플로우 (변경)

```
AS-IS:
  일반 사용자 → My 탭만 표시
  Manager → My + Team 탭 표시
  Admin → My + Team + All 탭 표시

TO-BE:
  모든 사용자 → 내것만 + Unit + Cell + 전체 (4탭 모두 표시)
```

## 6. 기술 제약사항

1. **Cell 미소속 사용자**: Cell에 소속되지 않은 사용자는 Cell 탭에서 빈 결과. 탭 비활성화 또는 안내 메시지 필요
2. **Unit 미소속 사용자**: 동일하게 Unit 탭 빈 결과 처리 필요
3. **Today 페이지 구조 변경**: 현재 MyTodayPanel/TeamTodayPanel/AllTodayPanel 3개 패널이 하드코딩 → 범위 파라미터를 받는 단일 패널로 리팩토링 필요
4. **Meeting Notes 가시성 모델**: 현재 노트 자체에 visibility 속성이 있어 "범위별 조회"와 "노트의 공개 범위"가 혼합됨 → 탭 필터와 생성 시 visibility는 분리 필요
5. **Calendar SHARED 모드**: 기존 SHARED 필터가 4-탭 구조에 포함되지 않음 → 별도 토글/체크박스로 분리 또는 제거 검토
6. **Issues my_involved 필터**: 기존 "내가 관여한 이슈" 필터가 4-탭에 포함되지 않음 → 추가 필터 옵션으로 유지 검토

## 7. 비기능 요구사항

1. **성능**: Unit/Cell 범위 조회 시 멤버 수가 많을 경우 쿼리 최적화 (인덱스 활용)
2. **일관성**: 5개 페이지 모두 동일한 탭 UI 컴포넌트 사용 (공통 컴포넌트 추출)
3. **i18n**: 3개 언어(ko/en/vi) 번역 키 통일
4. **반응형**: 모바일 화면에서 4개 탭 UI 처리 (스크롤 또는 드롭다운)

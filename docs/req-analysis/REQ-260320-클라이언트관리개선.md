# 요구사항 분석서: Entity별 고객사 초대 및 로그인 페이지 기능 개선

**문서번호**: REQ-클라이언트관리개선-20260320  
**작성일**: 2026-03-20  
**상태**: 변경 (v2 — Entity Code 필수 로그인)  
**변경일**: 2026-03-20

## 1. 요구사항 원문

1. Entity Settings에 Client Management 페이지 구현 (이메일, 이름, 회사, 클라이언트 등급)
2. Client Management 페이지에서 Invite Client 메일 보내기 기능 (초청 회사 Entity 정보 포함)
3. ~~로그인 페이지에 초청 회사 Entity Name 표기~~ → **변경**: 클라이언트 로그인 페이지도 USER_LEVEL 로그인 화면과 동일하게 초대받은 Entity Name을 알고, 선택한 회사 Entity Code가 있어야 로그인 가능
4. 로그인 후 클라이언트 포탈: 배정 프로젝트 + 초대된 로비 채팅 채널 메뉴만 제공

## 3. TO-BE 요구사항 정의

### REQ-01: Entity Settings - Client Management 페이지

**경로**: `/entity-settings/clients`

| 기능 | 설명 |
|------|------|
| 고객사 목록 | 현재 Entity(`cli_ent_id`)에 연결된 고객사 테이블 표시 |
| 표시 항목 | 이메일(대표연락처), 이름(담당자), 회사명, 클라이언트 등급(역할) |
| 고객사 등록 | 새 고객사 생성 → `cli_ent_id`에 현재 Entity 자동 설정 |
| 고객사 수정 | 회사명, 상태, 담당자 등 편집 |
| 고객 담당자 관리 | 해당 고객사의 CLIENT_LEVEL 사용자 목록 표시 |
| 초대 상태 확인 | PENDING 초대 목록 + 상태 관리 |

### REQ-02: Invite Client 메일 발송

| 기능 | 설명 |
|------|------|
| 초대 버튼 | Client Management 페이지 내 "Invite Client" 버튼 |
| 초대 폼 | 이메일(필수), 이름(선택), 역할(CLIENT_ADMIN/CLIENT_MEMBER) |
| Entity 정보 포함 | 초대 메일에 초청 회사(Entity) 이름, 로고 포함 |
| 초대 링크 | `/client/register?token=xxx` (기존 72시간 만료) |
| 이메일 발송 | MailService를 통해 실제 메일 발송 |
| 재발송 | 만료/미수락 초대 재발송 기능 |

### REQ-03: 클라이언트 로그인 페이지 — Entity Code 필수 (변경)

> **변경**: 단순 Entity Name 표시 → USER_LEVEL과 동일한 Entity Code 선택 필수 로그인

| 기능 | 설명 |
|------|------|
| **Entity Code 선택 (필수)** | USER_LEVEL `LoginPage.tsx`와 동일한 Entity Code 검색/선택 UI |
| Entity 검색 모달 | `EntitySearchModal` 재사용 또는 클라이언트 전용 Entity 검색 |
| 최근 Entity 빠른 선택 | 로컬스토리지 기반 최근 사용 Entity 버튼 |
| URL 파라미터 지원 | 초대 메일 링크에 `?entityCode=XX&entityName=YY` 포함 |
| Entity Code 없으면 로그인 불가 | selectedEntity 미선택 시 submit 차단 |
| Login 요청 확장 | `ClientLoginRequest`에 `entity_code` 필드 추가 (필수) |
| Backend 검증 | entity_code → Entity 조회 → Client의 `cliEntId` 일치 검증 |
| Login 응답 확장 | Entity 정보(이름, 코드, 로고) 포함 |
| 회원가입 페이지 | 초대 토큰 검증 시 Entity Code + Entity Name 함께 표시 |

### REQ-04: 클라이언트 포탈 메뉴 최적화

| 기능 | 설명 |
|------|------|
| 프로젝트 | 운영자가 멤버로 등록한 프로젝트만 접근 (기존 유지) |

## 5. 데이터 흐름

### 5.1 초대 → 가입 → 로그인 플로우 (개선 후)

```
[Entity Settings → Client Management]
  │
  ├─ 고객사 등록 (cli_ent_id = 현재 Entity)
  │
  ├─ "Invite Client" 클릭
  │   → 초대 폼 (이메일, 이름, 역할)
  │   → POST /api/v1/entity-settings/clients/:cliId/invite
  │   → 초대 토큰 생성 + 메일 발송
  │     메일 내용: Entity 이름 + 로고 + 초대 링크
  │
  ↓
[고객 담당자: 메일 수신]
  │
  ├─ 초대 링크 클릭 → /client/register?token=xxx
  │   → 토큰 검증 → Entity 이름 + Client 이름 표시
  │   → 가입 완료 → 자동 로그인
  │
  ↓
[클라이언트 로그인 페이지]
  │
  ├─ **Entity Code 선택 (필수)** — USER_LEVEL과 동일 패턴
  │   → EntitySearchModal 또는 클라이언트 전용 검색
  │   → 초대 메일 링크의 entityCode URL 파라미터 자동 세팅
  │   → 최근 사용 Entity 빠른 선택
  │
  ├─ 이메일 + 비밀번호 입력
  │
  ├─ 로그인 요청: POST /api/v1/client-auth/login
  │   { entity_code, email, password }
  │   → entity_code로 Entity 조회
  │   → Client의 cliEntId == Entity.entId 검증
  │   → 사용자 인증 → JWT 발급
  │
  ↓
[클라이언트 포탈]
  │
  ├─ Dashboard (프로젝트 통계 + 최근 이슈)
  ├─ Projects (배정된 프로젝트만)
  ├─ Issues (본인 reporter 이슈)
  ├─ Chat (초대된 Amoeba Talk 채널)  ← 신규
  └─ Profile
```

### 5.2 Entity-Client-User 관계도

```
HrEntity (회사/법인)
  │
  ├─ cli_ent_id ──→ SvcClient (고객사)
  │                    │
  │                    ├─ amb_client_invitations (초대)
  │                    │    └─ civToken → 가입 링크
  │                    │
  │                    ├─ amb_users (usr_cli_id)
  │                    │    └─ CLIENT_LEVEL 사용자들
  │                    │
  │                    └─ kms_project_clients (프로젝트 연결)
  │                         └─ 접근 가능 프로젝트
  │
  └─ amb_talk_channels (Amoeba Talk)
       └─ chn_type = 'CLIENT' + cli_id (신규)
```

## 6. 기술 제약사항

| 항목 | 제약 | 대응 |
|------|------|------|
| `cli_ent_id` nullable | 기존 레거시 데이터에 Entity 미연결 가능 | Entity Settings에서 생성 시 자동 설정, 기존 데이터 마이그레이션 불요 |
| Amoeba Talk 내부 전용 | `ent_id` 기반, CLIENT_LEVEL 미지원 | 채널에 `cli_id` 참조 추가 또는 CLIENT 멤버 허용 가드 확장 |
| MailService Entity 정보 | 기존 `sendInvitationEmail`은 내부 멤버용 | 클라이언트 전용 메일 메서드 신규 추가 |
| Client Login DTO | entity_code 없음 | entity_code 필수, Entity 검증 로직 추가 |
| Client Login UI | email+pw만 | EntitySearchModal + 최근 Entity + URL 파라미터 |
| Login 응답 구조 | Entity 정보 미포함 | `client.hrEntity` JOIN 추가 후 Entity 정보 응답 확장 |
| 클라이언트 Zustand Store | Entity 정보 미저장 | Store 스키마 확장 필요 |

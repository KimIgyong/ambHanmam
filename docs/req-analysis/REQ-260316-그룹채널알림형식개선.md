## 1. AS-IS 현황 분석

### 1.1 현재 푸시 알림 동작

그룹 채널(PUBLIC/PRIVATE)에서 메시지를 수신할 때, 알림이 다음과 같이 표시됨:

**Web Push (브라우저 알림)**
```
Title: Gray.Kim          ← 발신자 이름
Body:  Hello             ← 메시지 내용
```

**SSE 모달 (화면 상단 알림)**
```
Title: Gray.Kim 님의 메시지   ← getDisplayTitle()에서 변환
Body:  Hello                 ← 메시지 내용
```

### 1.2 문제점
- 알림에 **채널명(Lobby/그룹명)이 표시되지 않음**
- 여러 그룹 채널에 참여한 사용자는 메시지가 **어느 그룹에서 온 것인지 식별 불가**
- DM과 그룹 메시지의 알림 형식이 동일하여 구분이 안됨

### 1.3 현재 코드 구조

| 구분 | 파일 | 핵심 코드 |
|------|------|----------|
| **메시지 발송 → 푸시 호출** | `apps/api/.../message.service.ts` L205 | `this.sendMessagePush(channelId, userId, senderName, ...)` |
| **SSE 알림 생성** | `apps/api/.../message.service.ts` L255-268 | `ntfTitle: senderName`, `ntfMessage: preview` |
| **Web Push 발송** | `apps/api/.../message.service.ts` L271-276 | `title: senderName`, `body: preview` |
| **Push 수신/표시** | `apps/web/public/push-sw.js` L3-12 | `showNotification(data.title, { body: data.body })` |
| **SSE 모달 표시** | `apps/web/.../NotificationModal.tsx` L155 | `getDisplayTitle()` → `t('talkMessage', { sender })` |
| **푸시 서비스** | `apps/api/.../push.service.ts` | `sendPushToMany(recipientIds, payload)` |

### 1.4 채널 타입

| 타입 | 설명 | 알림 형식 변경 대상 |
|------|------|------------------|
| `DIRECT` | 1:1 DM | ✗ (현행 유지: 발신자명이 곧 채널 식별) |
| `PUBLIC` | 공개 그룹 | ✓ (채널명 필요) |
| `PRIVATE` | 비공개 그룹 | ✓ (채널명 필요) |

## 2. TO-BE 요구사항

### 2.1 그룹 채널 알림 형식 (PUBLIC / PRIVATE)

**Web Push (브라우저 알림)**
```
Title: Test Private 2           ← 채널명
Body:  Gray.Kim: Hello          ← 발신자: 메시지
```

**SSE 모달 (화면 상단 알림)**
```
Title: Test Private 2           ← 채널명
Body:  Gray.Kim: Hello          ← 발신자: 메시지
```

### 2.2 DM 알림 형식 (DIRECT) - 변경 없음

**Web Push**
```
Title: Gray.Kim                 ← 발신자명 (현행 유지)
Body:  Hello                    ← 메시지 내용
```

### 2.3 멘션 알림 형식

**그룹 채널 멘션**
```
Title: Test Private 2                    ← 채널명
Body:  Gray.Kim mentioned you: Hello     ← 발신자 멘션 + 메시지
```

**DM 멘션** (현행 유지)
```
Title: Gray.Kim mentioned you           ← 현행 유지
Body:  Hello                            ← 메시지 내용
```

## 4. 사용자 플로우

```
[그룹 채널 메시지 수신]
  ↓
[백엔드] sendMessagePush()
  ↓ channelId로 채널 정보 조회
  ↓ channel.chnType 확인
  ↓
  ├─ DIRECT → title: senderName, body: preview (현행)
  └─ PUBLIC/PRIVATE → title: channelName, body: "senderName: preview"
  ↓
[SSE] ntfTitle/ntfMessage 설정
[Push] title/body 설정
  ↓
[프론트엔드] 알림 표시
```

---

## 5. 기술 제약사항

- `sendMessagePush`에서 이미 `channelId`를 받으므로 채널 정보 조회 가능
- 채널 엔티티(`TalkChannelEntity`)에 `chnName`, `chnType` 컬럼 존재
- SSE `ntfTitle`은 프론트엔드 `NotificationModal`의 `getDisplayTitle()`에서 한번 더 가공됨
  → 그룹 채널인 경우 `getDisplayTitle()` 동작도 맞춰야 함
- Web Push는 `push-sw.js`에서 직접 `data.title`을 사용하므로 백엔드 변경만으로 적용됨

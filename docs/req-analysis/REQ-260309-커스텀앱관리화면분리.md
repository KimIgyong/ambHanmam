# 요구사항 분석서: 커스텀앱 관리 화면 분리 (USER_LEVEL / ADMIN_LEVEL)

- **문서번호**: REQ-커스텀앱관리화면분리-20260309
- **작성일**: 2026-03-09
- **작성자**: Claude Code

---

## 1. 요구사항 요약

USER_LEVEL 고객사가 자사 커스텀앱을 등록/관리하는 화면과, ADMIN_LEVEL 운영자가 전체 법인의 커스텀앱을 일괄 관리하는 화면을 각각의 설정 메뉴에서 접근할 수 있도록 한다.

---

## 2. AS-IS 현황 분석

### 2.1 현재 구현 상태

| 항목 | USER_LEVEL (MASTER) | ADMIN_LEVEL (SUPER_ADMIN/ADMIN) |
|------|---------------------|--------------------------------|
| **설정 메뉴** | `/entity-settings` (EntitySettingsPage) | `/admin` (SettingsPage) |
| **커스텀앱 메뉴 카드** | ✅ 있음 (`ENTITY_CUSTOM_APPS`) | ❌ 없음 |
| **커스텀앱 관리 페이지** | ✅ `/entity-settings/custom-apps` | ❌ 없음 |
| **라우트** | ✅ EntitySettingsGuard + MenuGuard | ❌ 없음 |

### 2.2 현재 접근 경로

```
USER_LEVEL(MASTER):
  /entity-settings → 카드 클릭 → /entity-settings/custom-apps → EntityCustomAppsPage
  → 자기 법인의 커스텀앱 CRUD

ADMIN_LEVEL:
  /admin → 카드 목록 (15개) → 커스텀앱 관련 메뉴 없음
  → 커스텀앱 관리 불가
```

### 2.3 백엔드 API 현황

| 엔드포인트 | 보호 | ADMIN 접근 |
|-----------|------|-----------|
| `GET /entity-settings/custom-apps` | @Auth + OwnEntityGuard | ✅ `entity_id` 쿼리로 접근 가능 |
| `POST /entity-settings/custom-apps` | @Auth + OwnEntityGuard | ✅ `entity_id` 쿼리로 접근 가능 |
| `PATCH /entity-settings/custom-apps/:id` | @Auth + OwnEntityGuard | ✅ `entity_id` 쿼리로 접근 가능 |
| `DELETE /entity-settings/custom-apps/:id` | @Auth + OwnEntityGuard | ✅ `entity_id` 쿼리로 접근 가능 |
| `POST /entity-settings/custom-apps/:id/health` | @Auth + OwnEntityGuard | ✅ `entity_id` 쿼리로 접근 가능 |

**핵심**: 백엔드 API는 이미 `resolveEntityId()` 유틸로 ADMIN_LEVEL이 `entity_id` 쿼리 파라미터를 통해 타 법인 데이터에 접근할 수 있도록 구현되어 있음. **백엔드 변경 불필요**.

### 2.4 기존 ADMIN 패턴 (참조)

ADMIN 설정 화면에서 법인별 데이터를 관리하는 기존 패턴:
- **법인 관리** (`/admin/entities`): 전체 법인 목록 → 법인 선택 → 상세 편집
- **사용자 관리** (`/admin/user-management`): 법인별 사용자 목록
- **메일 계정** (`/admin/mail-accounts`): 전체 메일 계정 관리

## 3. TO-BE 요구사항

### 3.1 USER_LEVEL 화면 (변경 없음)

현재 구현이 완전하므로 변경 불필요:
- `/entity-settings` 메뉴에 `ENTITY_CUSTOM_APPS` 카드 ✅
- `/entity-settings/custom-apps` 페이지에서 자기 법인 앱 CRUD ✅

### 3.2 ADMIN_LEVEL 화면 (신규)

| 항목 | 내용 |
|------|------|
| **경로** | `/admin/custom-apps` |
| **메뉴 카드** | SettingsPage에 커스텀앱 관리 카드 추가 |
| **기능** | 법인 선택 드롭다운 → 해당 법인의 커스텀앱 CRUD |
| **권한** | AdminGuard (SUPER_ADMIN, ADMIN만 접근) |

### 3.3 ADMIN 커스텀앱 관리 화면 기능 명세

#### 3.3.1 법인 선택
- 헤더 영역에 법인 선택 드롭다운 표시
- `GET /hr/entities` API로 전체 법인 목록 조회
- 법인 선택 시 해당 법인의 커스텀앱 목록 로드

#### 3.3.2 커스텀앱 CRUD
- 기존 `EntityCustomAppsPage.tsx`와 동일한 기능
- API 호출 시 `entity_id` 쿼리 파라미터에 선택된 법인 ID 전달
- 등록/수정/삭제/연결 테스트 모두 지원

#### 3.3.3 UI 구성

## 5. 기술 제약사항

1. **API 재사용**: 기존 `/entity-settings/custom-apps` API를 그대로 사용. `entity_id` 쿼리 파라미터만 추가하면 ADMIN이 타 법인 데이터 접근 가능.
2. **기존 훅 재사용**: `useCustomApps()` 등 기존 React Query 훅에 `entityId` 옵션 추가 필요.
3. **페이지 코드 재사용**: 기존 `EntityCustomAppsPage.tsx`의 로직을 최대한 재사용하되, 법인 선택 UI를 추가한 ADMIN 전용 래퍼 페이지 생성.

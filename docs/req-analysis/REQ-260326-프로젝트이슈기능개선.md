| Key | Emoji | 이름 |

| PRAY | 🙏 | Folded Hands |

**필요한 작업**:

### 3.9 D-4: URL 하이퍼링크 자동 변환

**대상**: 이슈 설명(description) + 코멘트(content)의 `dangerouslySetInnerHTML` 렌더링 영역

**동작**:
- 텍스트 내 URL 패턴(`http://`, `https://`)을 자동 감지
- `<a href="..." target="_blank" rel="noopener noreferrer">` 태그로 변환
- 긴 URL은 표시 텍스트를 80자 이내로 자르고 `...` 표시
- 이미 `<a>` 태그 안에 있는 URL은 중복 변환 방지

**구현**: `sanitizeHtml()` 후 처리하는 `autoLinkUrls()` 유틸리티 함수 생성

### 3.10 D-5: 번역 패널 위치 변경

**변경**: TranslationPanel을 AI Analysis 섹션 바로 아래로 이동

**Before 순서**: ... → AI Analysis → Workflow → Link Share → Comments → History → **Translation Panel**

**After 순서**: ... → AI Analysis → **Translation Panel** → Workflow → Link Share → [Comments | History] 탭

## 5. 사용자 플로우

### 5.1 이슈 리스트 정렬 플로우
1. 사용자가 프로젝트 상세 → Issues 탭 진입
2. 정렬 드롭다운에서 "Due Date 최신순" 선택
3. 테이블이 Due Date DESC로 재정렬 (Due Date null은 하단 배치)
4. 다른 정렬 옵션 선택 가능: Created 최신순, Priority 높은순

### 5.2 코멘트 이모티콘 플로우
1. 사용자가 코멘트에 마우스 hover (또는 모바일에서 탭)
2. 이모티콘 5개 버튼이 표시됨
3. 이모티콘 클릭 시 토글 (추가/제거)
4. 다른 사용자의 리액션은 코멘트 하단에 이모티콘 + 카운트로 표시
5. 자신이 이미 리액션한 이모티콘은 강조(highlighted) 표시

### 5.3 URL 하이퍼링크 플로우
1. 이슈 설명 또는 코멘트에 긴 URL 포함된 텍스트 표시
2. URL이 자동으로 클릭 가능한 파란색 링크로 변할
3. 긴 URL은 "https://example.com/very/long/pa..." 형태로 축약 표시
4. 클릭 시 새 탭에서 해당 URL 열림

## 6. 기술 제약사항

### 6.1 이모티콘 리액션
- Amoeba-Talk의 리액션 시스템(`amb_talk_reactions`, `TalkReactionEntity`)과 구조 유사하지만 별도 테이블 필요
- **DB 스키마 변경**: 스테이징/프로덕션은 `synchronize: false`이므로 수동 SQL 마이그레이션 필수
- 리액션 API는 Talk의 `message.service.ts` 패턴 참고

### 6.2 URL 자동 링크
- `sanitizeHtml()`이 이미 XSS 방지 처리 중 → 자동 링크 변환 시 sanitize를 통과한 후 적용 필요
- RichTextEditor(TipTap)로 작성된 코멘트는 이미 `<a>` 태그가 포함될 수 있으므로 중복 변환 방지 로직 필수
- 정규식으로 `<a>` 태그 바깥의 URL만 선택적으로 변환

### 6.3 정렬
- ProjectIssuesTab은 `size: 500`으로 전체 로드 → 클라이언트 정렬 가능
- IssuesPage의 리스트 뷰는 페이지네이션 사용 → 백엔드 정렬 필요 (추후 확장 시)

### 6.4 i18n
- 새로운 번역 키 필요: 정렬 옵션 라벨, 이모티콘 이름, 탭 라벨 등
- 3개 언어(ko/en/vi) 동기 업데이트 필수

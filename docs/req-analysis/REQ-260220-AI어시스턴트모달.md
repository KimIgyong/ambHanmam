# REQ-AI어시스턴트모달-20260220

## 요구사항 분석서: AI 어시스턴트 모달

### 1. AS-IS 현황

#### 1.1 Chat 시스템
- 9개 부서별 AI 에이전트 (`MANAGEMENT`, `ACCOUNTING`, `HR`, `LEGAL`, `SALES`, `IT`, `MARKETING`, `GENERAL_AFFAIRS`, `PLANNING`)
- `ChatPage` → URL 라우팅 기반 (`/chat/{department}/{conversationId}`)
- SSE 스트리밍 (`sendMessageSSE`)
- Zustand 전역 `useChatStore` (한 번에 하나의 대화만 관리)
- 대화/메시지는 `amb_conversations` + `amb_messages` 테이블에 영구 저장

#### 1.2 Agent 시스템
- `AgentsPage` → 부서별 에이전트 선택 그리드
- `useMyPermissions()` → `CHAT_{DEPARTMENT}` 메뉴 코드 기반 권한 체크
- `DEPARTMENTS` 상수 → 아이콘, 색상, i18n 키 포함

#### 1.3 TODO 시스템
- `useTodoList()` → status/date 필터 지원 React Query 훅
- `TodoResponse` → todoId, title, status, dueDate 등

#### 1.4 KMS 파이프라인
- `ModuleDataEvent` → `module.data.created` 이벤트 발행
- `WorkItemSyncListener` → WorkItem 생성 → AutoTagging → TagCloud 반영
- 현재 chat 모듈은 KMS 파이프라인에 연결되어 있지 않음

### 2. TO-BE 요구사항

1. 로그인 직후 모달창으로 AI 어시스턴트 자동 실행 (세션당 1회)
2. 모달에서 오늘의 할일(TODO) 요약 표시
3. 9개 부서별 에이전트 선택 UI 제공
4. 선택한 에이전트와 모달 내에서 실시간 채팅
5. 모달 대화도 서버에 저장 → KMS 지식 정제 연동
6. FAB 버튼으로 수동 재오픈 가능

### 3. 갭 분석

| 항목 | AS-IS | TO-BE | 갭 |
|------|-------|-------|-----|
| 채팅 진입 | ChatPage URL 라우팅 | 모달 오버레이 | 모달 전용 UX 필요 |
| 상태 관리 | useChatStore (전역 단일) | 모달 독립 스토어 | 별도 Zustand 스토어 필요 |
| TODO 연동 | 별도 TodoPage | 모달 내 요약 표시 | 기존 훅 재사용 가능 |

| MessageList | useChatStore 직접 참조 | props 기반 렌더링 | MessageListView 추출 필요 |

### 4. 사용자 플로우

```
[로그인] → [MainLayout 렌더]
    ↓
[sessionStorage 체크: amb_assistant_shown?]
    ↓ (없으면)
[모달 자동 오픈] → [WelcomeSection]
    ├── 오늘의 할일 요약 (최대 5개)
    ├── "전체 보기" → /todos 이동 + 모달 닫기
    └── 에이전트 선택 그리드 (3x3)
          ↓ (클릭)
        [ChatSection]
          ├── 에이전트 헤더 + 뒤로 버튼
          ├── 메시지 리스트 (SSE 스트리밍)
          └── 메시지 입력

[모달 닫기] → [FAB 버튼 표시]
    ↓ (클릭)
[모달 재오픈] → [이전 상태 유지 또는 WelcomeSection]
```

### 5. 기술 제약사항

- `useChatStore`와 모달 스토어 상태 충돌 방지 필수
- SSE 스트리밍은 기존 `sendMessageSSE()` 그대로 재사용
- 모달 z-index와 FAB 버튼 z-index 충돌 방지
- i18n 3개 언어(ko, en, vi) 모두 번역 필요
- 기존 `MessageList` 리팩토링 시 하위 호환성 유지

### 6. 업무효율 향상 제안

- 로그인 시 자동 인사 + TODO 표시로 업무 시작 가이드
- 에이전트 빠른 접근으로 채팅 진입 단계 축소 (3클릭 → 1클릭)
- 대화 자동 KMS 연동으로 지식 축적 파이프라인 완성

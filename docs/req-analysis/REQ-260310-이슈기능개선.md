# 요구사항 분석서: 이슈 기능 개선

**작성일**: 2026-03-10  
**분류**: UX 개선, 다국어 지원  
**관련 모듈**: 이슈 관리 (apps/web/src/domain/issues, apps/api/src/domain/issues)

## 1. 요구사항 원문

1. 이슈 등록 및 수정 시 "등록합니다" / "수정이 완료되었습니다" 안내 문구 보여주기
2. 이슈 등록 시 번역 기능 선택 가능하게 (폼 내 번역 버튼)
3. 이슈 [AI 검토]도 영어 기본 → 번역 버튼 추가

## 2. AS-IS 현황 분석

### 2.1 이슈 등록/수정 토스트 알림

**현재 코드** (`IssuesPage.tsx` `handleFormSubmit`):
```ts
// 등록 성공
createIssue.mutate({ ... }, {
  onSuccess: () => {
    setShowFormModal(false); // ← toast 없음
  },
});

// 수정 성공
updateIssue.mutate({ ... }, {
  onSuccess: () => {
    setShowFormModal(false);
    setEditingIssue(null);
    // ← toast 없음
  },
});
```

- `messages.created`, `messages.updated` i18n 키가 이미 정의되어 있으나 사용되지 않음
- 삭제(`deleteIssue`) / 상태변경(`updateIssueStatus`) 작업에도 toast 없음
- 비교: IssueFilters에서 프리셋 저장 시 `toast.success(t('issues:filter.presetSaved'))` 패턴을 이미 사용 중

### 2.2 이슈 등록 폼 번역 기능

**현재 코드** (`IssueFormModal.tsx`):
- 번역 기능 전혀 없음
- 사용자가 한국어/베트남어/영어로 제목·설명 작성 후, 다른 언어 참조자가 내용을 이해하기 어려움
- `IssueDetailModal`의 TranslationPanel은 **저장된 이슈**에 대한 번역(source_id 필수)
- 등록 폼에서는 아직 source_id가 없으므로 기존 TranslationPanel을 직접 사용 불가

**기존 번역 인프라**:
- 백엔드: `POST /translations/translate` → SSE 스트리밍 응답
- DTO: `{ source_type, source_id, source_fields, target_lang }` — source_id 필수
- `TranslationPanel.tsx`: 저장된 문서의 다국어 번역 관리 전용

### 2.3 AI 검토 기본 언어 및 번역

**현재 코드** (`IssueDetailModal.tsx` + `issue.service.ts`):

**프론트엔드** (`handleAiReview`):
```ts
const response = await fetch(`${API_BASE_URL}/issues/${issue.issueId}/ai-review`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', ... },
  // body 없음 — 언어 파라미터 전달 없음
});
```

**백엔드** (`issue.service.ts` `aiReview()`):
```ts
const systemPrompt = `...provide a structured review in the same language as the issue content.`;
// 제목이 한국어이면 한국어로 응답, 영어이면 영어로 응답
// 언어를 명시적으로 강제하지 않음
```

- 이슈 제목/설명이 한국어면 AI 검토 결과도 한국어
- 글로벌 팀에서 영어가 공통 언어임에도 AI 검토 기본이 영어 아님
- AI 검토 결과에 번역 버튼 없음

## 3. TO-BE 요구사항

### 3.1 이슈 등록/수정 토스트 알림

| 이벤트 | 메시지 (한국어) | 메시지 (영어) |
|--------|----------------|--------------|
| 등록 성공 | 이슈가 등록되었습니다 | Issue created successfully |
| 수정 성공 | 이슈가 수정되었습니다 | Issue updated successfully |
| 삭제 성공 | 이슈가 삭제되었습니다 | Issue deleted successfully |

- `toast.success()` 사용 (기존 sonner 라이브러리, patterns 이미 있음)
- 이미 있는 `issues:messages.created / updated / deleted` i18n 키 사용

### 3.2 이슈 등록 폼 번역 기능

- **위치**: `IssueFormModal.tsx` 제목·설명 입력 영역 하단
- **UX 흐름**:
  1. 사용자가 제목/설명 입력 완료
  2. "번역" 버튼 또는 언어 선택 드롭다운 클릭 (ko/en/vi)
  3. 현재 입력 내용을 선택 언어로 번역하여 팝업/인라인 패널에 표시
  4. 사용자가 "적용" 클릭 시 해당 필드를 번역된 내용으로 교체 (또는 취소)
- **구현 방식**:
  - 백엔드: `TranslateRequest`의 `source_id`를 optional(`@IsOptional()`)로 변경
  - source_id가 없을 경우 번역 결과를 DB에 저장하지 않고 SSE 스트리밍만 처리
  - 프론트: 인라인 번역 패널 (TranslationPanel 재사용 불가 → 경량 인라인 컴포넌트)

### 3.3 AI 검토 기본 언어 영어 + 번역 버튼

**백엔드**:
- `POST /issues/:id/ai-review` body에 `language?: string` 파라미터 추가 (default: `'en'`)
- `aiReview()` 서비스에 language 파라미터 전달
- 시스템 프롬프트에 `Respond only in English` 명시 (기본값)
- language가 `'ko'`이면 → 한국어로 응답

**프론트엔드** (`IssueDetailModal.tsx`):
- AI 검토 요청 시 `body: JSON.stringify({ language: 'en' })` 전달
- AI 검토 결과 박스 하단에 **번역 버튼** (🌐 ko / vi) 추가
- 버튼 클릭 시 기존 댓글 번역 패턴(`translateComment`) 동일하게 SSE 번역 처리
- 번역 결과는 기존 AI 텍스트 아래에 구분선과 함께 표시

## 5. 기술 제약사항

- **Source_id 없는 즉석 번역**: 기존 TranslationService는 source_id 기반 DB 저장을 전제로 설계. source_id를 optional로 변경하면 DB 저장 없이 번역만 처리 가능
- **AI 검토 언어 일관성**: systemPrompt에 언어 지시를 강제하면 이슈 제목이 한국어여도 영어로 응답 가능
- **번역 버튼 위치**: AI 검토 결과 박스 내부에 독립된 번역 상태 관리 필요 (기존 댓글 번역과 분리)

## 6. 사용자 플로우

### 이슈 등록
1. 이슈 등록 버튼 클릭 → 폼 모달 열림
2. **(신규)** "번역" 버튼 → 언어 선택 → 인라인 번역 결과 표시 → "적용" or "취소"
3. 등록 버튼 클릭
4. **(신규)** "이슈가 등록되었습니다" 토스트 표시

### AI 검토
1. 이슈 상세 → "AI 검토" 버튼 클릭
2. **(신규)** 영어로 AI 분석 결과 SSE 스트리밍
3. **(신규)** 결과 박스 우측: 🌐 KO / VI 번역 버튼
4. 클릭 시 번역 결과 인라인 표시

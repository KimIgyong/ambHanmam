## 1. 요구사항 원문

> https://stg-ama.amoeba.site/admin/total-users 여기 AI 일일한도/월간한도 설정된 값을
> https://stg-ama.amoeba.site/entity-settings/usage 페이지 Quota 영역에
> 월간 쿼터 / 일간 쿼터 노출하고,

> AI 사용시 "설정된 쿼터가 모두 소진되어있습니다. 크래딧을 충전하세요 - 관리자에게 보고하세요" 문구 안내

## 2. AS-IS 현황 분석

### 2.1 데이터 저장 구조 (이원화 문제)

현재 AI 쿼터 관련 데이터가 **2개의 테이블에 분산**되어 있으며, 이들이 연동되지 않는 상태:

| 구분 | 테이블명 | 용도 | 실데이터 |
|------|----------|------|----------|
| **A** | `amb_entity_ai_configs` | 관리자가 `/admin/total-users`에서 설정 | ✅ 3개 법인 데이터 있음 |
| **B** | `amb_entity_api_quotas` | 쿼터 체크 로직(`checkQuota`)이 조회 | ❌ 0건 (비어 있음) |

#### 테이블 A: `amb_entity_ai_configs` (실데이터 보유)
| 법인 | 일일 한도 | 월간 한도 | 활성 |
|------|-----------|-----------|------|
| KR01 (아메바컴퍼니) | 10,000 | 1,000,000 | ✅ |
| VN01 (AMOEBA CO.) | 1,000 | 10,000 | ✅ |
| ENT913 (아메바파트너스) | 500 | 10,000 | ✅ |

#### 테이블 B: `amb_entity_api_quotas` (비어 있음)
- 0건. 쿼터 체크 시 "할당량 미설정" → 항상 허용

### 2.2 관리자 설정 페이지 (`/admin/total-users > AI Configs 탭`)

- **컴포넌트**: `EntityAiConfigsTab.tsx`
- **기능**: 법인별 Provider, 공유키/자체키, 일일/월간 한도, 활성 여부 편집
- **저장 위치**: `amb_entity_ai_configs` 테이블 (Admin Service 경유)
- **문제**: 이 값이 `amb_entity_api_quotas`에 반영되지 않음

### 2.3 사용량 모니터링 페이지 (`/entity-settings/usage`)

- **컴포넌트**: `EntityUsagePage.tsx`
- **현재 상태**:
  - ✅ 일일/월간 사용량 (토큰 수, 요청 수) 표시 정상
  - ✅ 3단계 경고 배너 UI 구현됨 (CAUTION / WARNING / SUSPENDED)
  - ✅ 진행률 바 (QuotaBar) 구현됨
  - ❌ **Quota 영역**: `getEntityUsageSummary()`가 `amb_entity_api_quotas`에서 조회 → 항상 `null` → "할당량 미설정" 표시
  - ❌ 일간 쿼터 / 월간 쿼터 값이 표시되지 않음

### 2.4 쿼터 초과 제한 (미적용)

- **인터셉터**: `ApiQuotaInterceptor` 정의됨
  - `AiUsageService.checkQuota()` 호출
  - BLOCK 시 `E4010` (일간 초과) / `E4011` (월간 초과) ForbiddenException throw
- **문제점**:
  1. 인터셉터가 **어떤 컨트롤러에도 `@UseInterceptors()`로 적용되지 않음**
  2. `checkQuota()`가 `amb_entity_api_quotas` 조회 → 데이터 없음 → 항상 `allowed: true`
  3. 프론트엔드에 `E4010`/`E4011` 에러 처리 코드 없음
  4. AI 채팅 시 쿼터 초과 안내 메시지 없음

### 2.5 AI 사용량 기록 (정상 동작)

- `ClaudeService` → `AiUsageService.recordUsage()` → 건별 기록 + 일별 집계 UPSERT
- 테이블: `amb_ai_token_usage` (건별), `amb_ai_token_entity_summary` (일별 집계)
- 이 부분은 정상 동작 중

## 3. TO-BE 요구사항

### 3.1 기능 요구사항

#### FR-01: Quota 영역에 관리자 설정값 표시
- **대상 페이지**: `/entity-settings/usage`
- **표시 항목**:
  - 일간 쿼터: `amb_entity_ai_configs.eac_daily_token_limit` 값
  - 월간 쿼터: `amb_entity_ai_configs.eac_monthly_token_limit` 값
- **표시 형식**: `일간 쿼터: 10,000 토큰 / 월간 쿼터: 1,000,000 토큰`
- **미설정 시**: "할당량 미설정" 표시

#### FR-02: 쿼터 초과 시 AI 사용 차단
- **조건**: 일간 사용량 ≥ 일간 쿼터 OR 월간 사용량 ≥ 월간 쿼터
- **동작**: 해당 Entity의 모든 사용자 AI 기능 차단 (BLOCK)
- **적용 범위**: AI 채팅 (SSE 스트리밍), 번역, 보고서 생성 등 모든 AI API 호출

#### FR-03: 차단 시 안내 메시지
- **메시지**: "설정된 쿼터가 모두 소진되었습니다. 크래딧을 충전하세요 - 관리자에게 보고하세요"
- **표시 위치**:
  - AI 채팅 화면: 입력창 비활성화 + 안내 배너
  - `/entity-settings/usage` 페이지: SUSPENDED 배너 (기존)
- **다국어**: ko / en / vi 번역 키 추가

#### FR-04: 쿼터 0 (무제한) 처리
- 관리자가 일간/월간 한도를 0으로 설정한 경우 → 무제한 (쿼터 체크 건너뜀)
- 미설정 법인 → 무제한 (기본 동작)

## 6. 기술 제약사항 및 설계 방향

### 6.1 핵심 설계 결정: 데이터 소스 통합

**방안**: `getEntityUsageSummary()`와 `checkQuota()`가 `amb_entity_ai_configs` 테이블에서 한도를 읽도록 변경

**근거**:
- `amb_entity_api_quotas` 테이블은 비어 있고, 관리자가 이 테이블에 데이터를 넣는 UI가 없음
- `amb_entity_ai_configs`에 이미 `eac_daily_token_limit`, `eac_monthly_token_limit` 존재
- 두 테이블 동기화보다 소스를 하나로 통합하는 것이 안전

### 6.2 쿼터 제한 적용 위치

**방안**: `ClaudeService.ensureClient()` 또는 `sendMessage()`/`streamMessage()` 내부에서 쿼터 체크

**근거**:
- `ApiQuotaInterceptor`를 모든 AI 관련 컨트롤러에 적용하는 것보다 단일 진입점에서 체크하는 것이 누락 방지에 유리
- 모든 AI 호출이 `ClaudeService`를 경유하므로, 서비스 레벨에서 체크하면 완전성 보장

### 6.3 actionOnExceed 설정

- `amb_entity_ai_configs`에 `action_on_exceed` 컬럼 없음
- 요구사항이 "AI 사용 중지"이므로 **항상 BLOCK** 동작으로 구현
- 향후 WARN 모드 필요 시 컬럼 추가

### 6.4 영향 범위

| 영역 | 변경 여부 | 내용 |
|------|-----------|------|
| `AiUsageService` | ✅ 변경 | `getEntityUsageSummary()`, `checkQuota()`에서 `amb_entity_ai_configs` 조회 |
| `ClaudeService` | ✅ 변경 | AI 호출 전 쿼터 체크 추가 |
| `EntityUsagePage.tsx` | ✅ 변경 | SUSPENDED 배너 메시지 변경 |
| AI 채팅 프론트 | ✅ 변경 | E4010/E4011 에러 시 안내 메시지 표시 |
| i18n (ko/en/vi) | ✅ 변경 | 안내 메시지 번역 키 추가 |
| `EntityAiConfigsTab.tsx` | ❌ 변경 없음 | 관리자 설정 UI 그대로 유지 |
| `amb_entity_api_quotas` | ❌ 변경 없음 | 사용하지 않음 (향후 deprecation 고려) |
| DB 마이그레이션 | ❌ 불필요 | 기존 테이블/컬럼 활용 |

## 9. 수락 기준 (Acceptance Criteria)

- [ ] `/entity-settings/usage` 페이지 Quota 영역에 관리자가 설정한 일간/월간 쿼터 값이 표시된다
- [ ] 일간/월간 사용량 대비 진행률 바가 정확히 표시된다

- [ ] 쿼터가 0(무제한)이면 제한 없이 AI 사용 가능하다
- [ ] 미설정 법인은 제한 없이 AI 사용 가능하다
- [ ] ko/en/vi 3개 언어 안내 메시지가 정상 표시된다

# REQ-문서함파일업로드-20260219

## 요구사항 개요
- **요청일**: 2026-02-19
- **요청자**: 관리자
- **대상 페이지**: `/documents` (문서함)
- **요약**: 로그인한 사용자가 문서함에서 현재 탐색 중인 Google Drive 폴더에 파일을 업로드할 수 있어야 한다.

---

## 1. AS-IS 현황 분석

### 1.1 문서함 페이지 (`/documents`)
- Google Drive에 등록된 폴더 목록 표시
- 폴더 탐색 (하위 폴더 진입, 브레드크럼 네비게이션)
- 파일 검색 (전체 등록 폴더 대상)
- 파일 상세 정보 조회, 다운로드, Google Drive에서 열기
- **파일 업로드 기능 없음** (읽기 전용)

### 1.2 백엔드 Drive 모듈 (`/api/v1/drive`)
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/status` | Drive 설정 상태 |
| GET | `/folders` | 등록 폴더 목록 |
| POST | `/folders` | 폴더 등록 (Admin) |
| DELETE | `/folders/:id` | 폴더 해제 (Admin) |
| GET | `/files` | 파일 목록 |
| GET | `/files/:fileId` | 파일 상세 |
| GET | `/files/:fileId/download` | 파일 다운로드 |
| GET | `/search` | 파일 검색 |

- **업로드 엔드포인트 없음**

### 1.3 Google Drive 서비스 (인프라)
- `GoogleDriveService.uploadFile(folderId, buffer, filename, mimeType)` **이미 구현됨**
- `GoogleDriveService.createFolder(parentFolderId, folderName)` 구현됨
- 청구서(Billing) 모듈에서만 사용 중

### 1.4 기존 업로드 패턴 (Billing 모듈)
- `BillingDocumentController`: `@Post('upload')` + `FileInterceptor('file')`
- `FileService`: 로컬 파일 저장 (10MB 제한, 이미지/문서 MIME 타입)
- `DocumentManager.tsx`: 숨김 `<input type="file" ref>` + FormData 업로드

## 4. 사용자 플로우

```
1. 사용자가 /documents 페이지 접속
2. 등록된 폴더 카드 클릭 → 폴더 내부 진입
3. (선택) 하위 폴더로 이동
4. 상단 "업로드" 버튼 클릭
5. 파일 선택 대화상자 열림 (다중 파일 선택 가능)
6. 선택한 파일들이 업로드 큐에 추가
7. 각 파일별 업로드 진행률 표시
8. 업로드 완료 시 파일 목록 자동 갱신
9. 오류 발생 시 해당 파일 옆에 에러 표시
```

---

## 5. 기술 제약사항

- Google Drive API: Service Account 권한으로 업로드 (impersonated user 소유)
- Multer: NestJS `FileInterceptor` 사용 (메모리 스토리지)
- 파일 크기 제한: Multer 설정 + Nginx `client_max_body_size`
- Google Drive 할당량: 일일 업로드 제한 있음 (일반적으로 충분)

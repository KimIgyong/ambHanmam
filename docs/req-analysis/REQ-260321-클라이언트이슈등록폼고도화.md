# REQ-클라이언트이슈등록폼고도화-20260321

## 1. 요구사항 개요

클라이언트 포털의 이슈 등록 폼(`/client/issues/new`)을 내부 이슈 등록 폼(`/issues` → New Issue 모달)과 동등한 수준으로 고도화한다.

## 2. AS-IS 현황

### 2.1 클라이언트 포털 이슈 등록 폼 (현재)
- **파일**: `ClientIssueCreatePage.tsx` (120줄)
- **필드 4개**: 프로젝트(select), 유형(select), 제목(input), 설명(textarea)
- **제한사항**:
  - 긴급도(severity) 하드코딩 `'NORMAL'` (유효하지 않은 값)
  - 설명 필드가 단순 `<textarea>` (서식 지원 없음)
  - 담당자/관련자 지정 불가
  - 파일 첨부 불가
  - 시작일/종료예정일 불가
  - 관련 이슈 링크 불가
  - 번역 패널 없음

### 2.2 내부 이슈 등록 폼 (참조 모델)
- **파일**: `IssueFormModal.tsx` (980줄)
- **필드 15개+**: 프로젝트, 상위이슈, 유형, 긴급도, 제목, 설명(RichTextEditor), 우선순위, 담당자(AssigneeSelector), 관련자(ParticipantSelector), 에픽/컴포넌트, 영향 모듈, 시작일, 종료예정일, 진행률, 파일첨부, 해결내용
- **고급 기능**: 자동저장(Draft), 번역패널, Google Drive 연동, 멘션, 이미지 삽입

### 3.2 기능 요구사항

1. **RichTextEditor 공유**: 내부 폼에서 사용하는 동일한 `RichTextEditor` 컴포넌트 재사용
2. **번역 패널**: `TranslationPanel` 컴포넌트 재사용 (이슈 저장 후 상세 페이지에서 사용)
3. **파일 첨부**: 내부 폼과 동일한 파일 업로드 UI (드래그드롭 + URL 입력)
4. **관련 이슈 검색**: 프로젝트 내 이슈를 키워드로 검색하여 parent_issue로 연결
5. **i18n 완전 지원**: 모든 레이블, 플레이스홀더, 에러 메시지 다국어 (en/ko/vi)
6. **클라이언트 권한 범위**: 담당자/관련자는 해당 프로젝트에 배정된 내부 멤버만 표시

### 3.3 비기능 요구사항

- 클라이언트 포털 전용 페이지 (모달이 아닌 독립 페이지)
- 반응형 레이아웃 (모바일 대응)
- 기존 클라이언트 포털 디자인 시스템과 일관성 유지

## 5. 사용자 플로우

```
1. 고객이 /client/issues/new 접근 (또는 프로젝트 상세에서 New Issue 클릭)
2. 프로젝트 선택 (query param으로 사전 선택 가능)
3. 유형, 긴급도 선택
4. 제목 입력
5. RichTextEditor로 상세 설명 작성 (서식, 이미지, 파일 첨부 가능)
6. [선택] 관련 이슈 검색/연결
7. [선택] 담당자 지정
8. [선택] 관련자 추가
9. [선택] 파일 첨부 (드래그드롭 또는 URL)
10. [선택] 시작일/종료예정일 설정
11. 등록 버튼 클릭
12. 성공 시 이슈 상세 페이지로 이동
13. 상세 페이지에서 번역 패널 사용 가능
```

## 6. 기술 제약사항

- 클라이언트는 `ClientGuard` 인증을 사용 (내부 `JwtGuard`와 별도)
- 클라이언트가 접근 가능한 프로젝트는 `amb_project_clients` 테이블 기준
- AssigneeSelector/ParticipantSelector는 내부 멤버 API 사용 → 클라이언트용 프로젝트 멤버 API 필요
- RichTextEditor의 이미지 업로드는 기존 `/files/upload` 엔드포인트 재사용 (인증 토큰 필요)
- 번역 패널은 이슈 ID 필요 → 이슈 저장 후 상세 페이지에서만 동작

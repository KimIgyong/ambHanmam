### 1.2 현재 출퇴근 (Attendance) 시스템

`amb_attendances` 테이블:
- `att_type` 컬럼: `NORMAL` (정상출근), `REMOTE` (재택), `LEAVE` (휴가), `OFF` (휴일)
- 사용자가 직접 일정 등록 가능 (날짜, 타입, 시작/종료 시간)
- 법인(`ent_id`) 기반 격리
- **제약**: 주말 등록 불가, 재택 월 1회 제한, 다음주~1개월 범위만 등록 가능

- **연차 정보 없음**

### 1.4 현재 연차 계산 로직

베트남 노동법(2019) 기준:
- **기본 연차**: 12일 (근속 1년 이상)
- **추가 연차**: 근속 5년마다 +1일
- **이월**: 전년도 잔여 최대 5일까지
- **사용 집계**: Timesheet에서 attendance_code = 'AL' (Annual Leave) 건수
- **수식**: remaining = entitlement + carryForward - used

## 2. TO-BE 요구사항

### 2.1 기능 요구사항

#### FR-1: 휴가 신청 기능
- 직원이 직접 휴가(연차, 반차 등)를 신청할 수 있어야 한다
- **휴가 유형**: 연차(Annual Leave), 오전반차(AM Half), 오후반차(PM Half), 병가(Sick Leave), 특별휴가(Special Leave)
- 신청 시 필요 정보: 시작일, 종료일, 휴가 유형, 사유
- 신청 시 잔여 연차 자동 차감 검증 (잔여 부족 시 신청 불가)

#### FR-2: 휴가 승인 워크플로우
- 직원 신청 → 관리자(MANAGER/ADMIN) 승인/반려
- 승인 시: Timesheet에 AL 코드 자동 반영, leave_balance 자동 차감
- 반려 시: 사유 입력, 신청자에게 알림

#### FR-3: 연차 잔여 일수 셀프 확인
- My Page 또는 휴가 신청 화면에서 본인의 잔여 연차를 확인
- 표시 항목: 총 연차, 사용 일수, 잔여 일수, 이월 일수

#### FR-4: 휴가 내역 조회
- 본인의 휴가 신청 내역 조회 (상태별: 대기/승인/반려/취소)
- 관리자: 팀원의 휴가 신청 목록 조회 및 승인/반려 처리

### 2.2 Acceptance Criteria

#### AC-1: 휴가 신청
- [ ] 직원이 휴가 유형, 날짜, 사유를 입력하여 신청 가능
- [ ] 반차 선택 시 0.5일 차감, 연차 선택 시 1일 차감
- [ ] 잔여 연차 부족 시 신청 차단 + 안내 메시지
- [ ] 신청 완료 시 관리자에게 알림

#### AC-2: 승인/반려
- [ ] 관리자 화면에서 대기 중인 휴가 신청 목록 확인
- [ ] 승인 시 Timesheet 자동 반영 + leave_balance 차감
- [ ] 반려 시 사유 입력 필수

#### AC-3: 잔여 연차 확인
- [ ] My Page에서 연차 잔여 카드 표시
- [ ] 잔여 연차, 총 연차, 사용 일수, 이월 일수 표시
- [ ] 휴가 신청 화면에서도 잔여 일수 표시

### 2.3 사용자 플로우

```
1. 직원이 "내 휴가" 페이지 접속
   → 잔여 연차 요약 카드 표시 (총: 12일, 사용: 3일, 잔여: 9일)
   → 본인 휴가 신청 내역 테이블

2. "휴가 신청" 버튼 클릭 → 모달/폼 오픈
   → 휴가 유형 선택: 연차/반차(오전)/반차(오후)/병가/특별휴가
   → 시작일, 종료일 선택 (주말/공휴일 자동 제외)
   → 차감 일수 자동 계산 표시
   → 사유 입력
   → "신청" 버튼 → 서버 검증 (잔여 연차 체크) → 성공/실패

3. 관리자가 HR > Leave 페이지 또는 알림에서 대기 건 확인
   → 신청 상세 확인 (신청자, 기간, 유형, 사유)
   → "승인" 또는 "반려" (반려 시 사유 입력)
   → 승인 시: Timesheet + LeaveBalance 자동 업데이트

4. My Page 접속
   → 연차 잔여 요약 카드 표시
```

## 3. 기술 분석

### 3.1 신규 테이블 설계

#### `amb_hr_leave_requests` (휴가 신청)

| 컬럼명 | 타입 | Prefix | 설명 |
|--------|------|--------|------|
| `lrq_id` | UUID PK | lrq | PK |
| `emp_id` | UUID FK | emp | 직원 ID (→ amb_hr_employees) |
| `usr_id` | UUID FK | usr | 사용자 ID (→ amb_users) — 신청자 |
| `ent_id` | UUID FK | ent | 법인 ID |
| `lrq_type` | VARCHAR(20) | lrq | ANNUAL, AM_HALF, PM_HALF, SICK, SPECIAL |
| `lrq_start_date` | DATE | lrq | 시작일 |
| `lrq_end_date` | DATE | lrq | 종료일 |
| `lrq_days` | DECIMAL(4,1) | lrq | 차감 일수 (자동 계산) |
| `lrq_reason` | TEXT | lrq | 사유 |
| `lrq_status` | VARCHAR(20) | lrq | PENDING, APPROVED, REJECTED, CANCELLED |
| `lrq_approved_by` | UUID | lrq | 승인자 ID (nullable) |
| `lrq_rejected_reason` | TEXT | lrq | 반려 사유 (nullable) |
| `lrq_created_at` | TIMESTAMP | lrq | 생성일 |
| `lrq_updated_at` | TIMESTAMP | lrq | 수정일 |
| `lrq_deleted_at` | TIMESTAMP | lrq | 삭제일 (soft delete) |

### 3.2 API 설계

#### 직원용 (본인)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/hr/leave-requests/my` | 내 휴가 신청 내역 |
| GET | `/hr/leave-requests/my/balance` | 내 연차 잔여 확인 |
| POST | `/hr/leave-requests` | 휴가 신청 |
| PATCH | `/hr/leave-requests/:id/cancel` | 신청 취소 (PENDING만) |

#### 관리자용
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/hr/leave-requests` | 전체 휴가 신청 목록 (필터: 상태, 기간, 직원) |
| PATCH | `/hr/leave-requests/:id/approve` | 승인 |
| PATCH | `/hr/leave-requests/:id/reject` | 반려 (사유 필수) |

### 3.3 핵심 비즈니스 로직

1. **차감 일수 계산**
   - 연차(ANNUAL): 시작일~종료일 내 영업일 수 (주말/공휴일 제외)
   - 반차(AM_HALF, PM_HALF): 0.5일 (시작일 = 종료일)
   - 병가/특별휴가: 시작일~종료일 영업일 수 (별도 잔여 차감 없음)

2. **신청 시 검증**
   - 잔여 연차 >= 차감 일수 (ANNUAL, AM_HALF, PM_HALF만)
   - 중복 일자 신청 불가 (이미 PENDING/APPROVED 건이 겹치면 차단)
   - 과거 날짜 신청 불가

3. **승인 시 후처리**
   - `amb_hr_timesheets`에 AL 코드 자동 삽입 (날짜별)
   - `amb_hr_leave_balances.lvb_used` 증가, `lvb_remaining` 감소
   - 신청자에게 알림 (기존 notification 시스템 활용)

4. **취소 시 후처리** (PENDING만 가능)
   - 상태 CANCELLED로 변경 (차감 복구 불필요 — 아직 미차감)

## 5. 기술 제약사항

### 5.1 usr_id ↔ emp_id 매핑
- 현재 `amb_hr_employees`에는 `usr_id` 필드가 **없을 수 있음**
- `amb_users`와 `amb_hr_employees` 간 연결 방법 확인 필요
- 해결: emp_email = usr_email 매칭 또는 emp 테이블에 usr_id FK 추가

### 5.2 공휴일 참조
- `amb_hr_holidays` 테이블이 이미 존재 (연도별 공휴일)
- 영업일 계산 시 공휴일 + 주말 제외 필요

### 5.3 법인별 격리
- 연차/휴가 데이터는 법인(`ent_id`) 기준으로 격리
- 관리자는 자기 법인 소속 직원만 조회/승인 가능

### 5.4 반차 처리
- 반차 = 0.5일 차감
- Timesheet 반영 시 attendance_code 구분 필요 (AL vs AH_AM/AH_PM)

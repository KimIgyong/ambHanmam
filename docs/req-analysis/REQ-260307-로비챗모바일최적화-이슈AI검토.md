## 2. R1: 로비챗 모바일 최적화

### 2.1 AS-IS 현황

**현재 웹 레이아웃 (AmoebaTalkPage.tsx)**:
- 3단 고정 레이아웃: 채널 사이드바(`w-72`) + 채팅 영역(`flex-1`) + 멤버 패널(`w-64`)
- 높이: `h-[calc(100vh-4rem)]`
- **모바일 반응형 미적용** — 모바일 화면에서 채널 리스트가 화면을 과도하게 차지하여 채팅 영역이 짤림
- MainLayout 자체는 모바일 드로어 지원 (`hidden md:flex` 사이드바)

**주요 문제점**:
1. 채널 사이드바 `w-72`(288px)가 모바일에서 고정되어 채팅 영역이 거의 보이지 않음
2. 멤버 패널 `w-64`(256px)가 추가로 공간 차지
3. 모바일에서 채널 목록과 채팅 뷰를 동시에 표시하면 사용 불가
4. 뒤로 가기, 네비게이션 패턴 없음

**별도 모바일 앱 현황 (`apps/mobile/src/tabs/ChatTab/`)**:
- Capacitor 기반 모바일 앱에는 별도 채팅 탭 존재
- 채널리스트 ↔ 채팅뷰 상태 토글 방식 이미 구현
- 하지만 웹 모바일 브라우저 접속 시에는 적용되지 않음

### 2.2 TO-BE 요구사항 (Zalo 스타일 참조)

**핵심 패턴: 화면 전환 방식 (Single View Navigation)**

모바일(`md` 이하)에서는 채널 목록과 채팅 뷰를 **동시에 표시하지 않고**, 상태에 따라 전환한다.

#### 화면 구성

| 상태 | 표시 화면 | 헤더 | 비고 |
|------|----------|------|------|
| 채널 미선택 | 채널 리스트 (전체 화면) | "Amoeba Talk" + 검색 | 기본 화면 |
| 채널 선택됨 | 채팅 뷰 (전체 화면) | ← 뒤로 + 채널명 + 멤버 | 채팅 집중 |
| 멤버 패널 | 오버레이 또는 바텀시트 | 채널 멤버 목록 | 모바일 전용 |

#### 세부 스펙

**A. 모바일 채널 리스트 (md 이하)**
- 전체 화면 사용 (`w-full`)
- 상단: 제목 + 검색 아이콘 + 새 DM / 채널 생성 버튼
- 채널 아이템: 아바타 + 이름 + 마지막 메시지 미리보기 + 시간 + 읽지 않은 수
- 탭 클릭 → `setCurrentChannelId()` 후 채팅 뷰로 전환

**B. 모바일 채팅 뷰 (md 이하)**
- 전체 화면 사용
- 상단 헤더: `←` 뒤로 버튼 + 채널/상대방 이름 + 온라인 상태 + 멤버 버튼
- 뒤로 버튼: `setCurrentChannelId(null)` → 채널 리스트로 복귀
- 메시지 목록: `flex-1 overflow-y-auto`
- 하단 입력: 기존 `TalkMessageInput` 재사용 (패딩 조정)
- 멤버 패널: 바텀 시트 또는 모달로 표시

**C. 데스크톱 (md 이상)**
- 기존 3단 레이아웃 유지 (변경 없음)

### 2.3 사용자 플로우 (모바일)

```
[앱 진입] → [채널 리스트 (전체 화면)]
                 │
                 ├─ 채널 탭 → [채팅 뷰 (전체 화면)]
                 │                │
                 │                ├─ ← 뒤로 → [채널 리스트]
                 │                ├─ 멤버 버튼 → [멤버 바텀시트]
                 │                └─ 메시지 입력/전송
                 │
                 ├─ 새 DM 버튼 → [DM 모달]
                 └─ 채널 생성 → [생성 모달]
```

### 2.4 기술 구현 방향

1. **AmoebaTalkPage.tsx**: `md:` 브레이크포인트 기반 조건부 렌더링
   - `currentChannelId === null` → 채널 리스트만 표시 (모바일)
   - `currentChannelId !== null` → 채팅 뷰만 표시 (모바일)
   - `md:` 이상 → 기존 3단 레이아웃 유지

2. **ChannelList.tsx**: 모바일에서 `w-full` 적용, 아이템에 마지막 메시지 미리보기 추가

3. **ChannelHeader.tsx**: 모바일 전용 뒤로 가기 버튼 (`md:hidden`)

4. **MemberPanel.tsx**: 모바일에서 바텀시트/오버레이 형태로 변경

5. **TalkMessageInput.tsx**: 모바일 패딩 조정 (`px-2 py-2 md:px-4 md:py-3`)

### 2.5 영향 범위

| 파일 | 변경 내용 |
|------|----------|
| `AmoebaTalkPage.tsx` | 모바일/데스크톱 조건부 레이아웃 |
| `ChannelList.tsx` | `w-full` + 마지막 메시지 미리보기 |
| `ChannelHeader.tsx` | 모바일 뒤로 가기 버튼 |

## 3. R2: 이슈 AI 검토 버튼

### 3.1 AS-IS 현황

**현재 이슈 워크플로우**:
```
OPEN → [APPROVED | REJECTED]
APPROVED → [IN_PROGRESS | REJECTED]
IN_PROGRESS → [RESOLVED]
RESOLVED → [CLOSED | OPEN]
```

**기존 AI 관련 필드**:
- `iss_ai_analysis` (TEXT) — 엔티티에 존재하나 **자동 생성 로직 미구현**
- 코멘트의 `authorType: 'AI'` 지원 — AI가 코멘트를 쓸 수 있는 구조는 존재
- `IssueDetailModal.tsx`에 AI Analysis 표시 영역 있음 (indigo 박스)

**문제점**:
- 매니저가 이슈를 승인/반려할 때 판단 근거가 부족
- AI 분석 필드가 있지만 활용되지 않음
- 이슈 내용이 불명확하거나 중복인 경우 수동 확인 필요

### 3.2 TO-BE 요구사항

**AI 검토 버튼**: 이슈 상세 모달에 "AI 검토" 버튼을 추가하여, 클릭 시 Claude API로 이슈를 분석하고 결과를 표시한다.

#### AI 검토 내용

| 분석 항목 | 설명 |
|----------|------|
| 요약 | 이슈 핵심 내용 1-2문장 요약 |
| 심각도 평가 | AI가 판단한 심각도와 이유 |
| 영향 범위 | 영향받는 모듈/기능 분석 |
| 해결 방안 제안 | 가능한 해결 방향 제시 |
| 유사 이슈 | 기존 이슈 중 유사한 건이 있는지 확인 |
| 승인 권고 | APPROVE / REJECT / 추가 정보 필요 중 하나 |

#### 버튼 위치 및 동작

1. **이슈 상세 모달** (`IssueDetailModal.tsx`)
   - 상태 변경 버튼 영역 옆에 "🤖 AI 검토" 버튼 추가
   - 클릭 → SSE 스트리밍으로 분석 결과 실시간 표시
   - 완료 시 `iss_ai_analysis` 필드에 저장

2. **이슈 생성 후**
   - 이슈 생성 직후 AI 검토를 자동 실행하지 않음 (수동 트리거)
   - 매니저가 승인/반려 전 "AI 검토" 버튼으로 참고 자료 확인

3. **결과 표시**
   - 기존 AI Analysis 영역 (indigo 박스)에 렌더링
   - 마크다운 형식 지원
   - 로딩 중 스피너/스트리밍 텍스트 표시

### 3.3 사용자 플로우

```
[이슈 상세 모달]
    │
    ├─ "AI 검토" 버튼 클릭
    │       │
    │       ├─ 로딩 표시 (SSE 스트리밍)
    │       ├─ 분석 결과 실시간 렌더링
    │       └─ 완료 → iss_ai_analysis에 저장
    │
    ├─ 승인 버튼 (기존) — AI 분석 참고 후 판단
    └─ 반려 버튼 (기존) — AI 분석 참고 후 판단
```

### 3.4 기술 구현 방향

**백엔드**:
1. `POST /api/v1/issues/{id}/ai-review` — AI 검토 엔드포인트
2. Claude API 호출: 이슈 제목, 설명, 타입, 심각도, 관련 모듈 등을 프롬프트에 포함
3. SSE 스트리밍으로 응답 반환
4. 완료 시 `iss_ai_analysis` 필드 업데이트
5. (선택) 유사 이슈 검색: 기존 이슈 제목/설명과 유사도 비교

**프론트엔드**:
1. `IssueDetailModal.tsx`에 AI 검토 버튼 추가
2. SSE EventSource로 스트리밍 수신
3. AI Analysis 섹션에 실시간 렌더링 (마크다운)
4. 이미 분석이 있으면 "재검토" 버튼으로 표시

### 3.5 영향 범위

| 파일 | 변경 내용 |
|------|----------|
| `issue.controller.ts` | `POST /{id}/ai-review` 엔드포인트 추가 |
| `issue.service.ts` (API) | AI 검토 로직 + Claude API 호출 |
| `IssueDetailModal.tsx` | AI 검토 버튼 + 결과 표시 UI |
| `issue.service.ts` (Web) | `aiReview()` API 호출 메서드 |
| `useIssues.ts` | AI 검토 mutation 훅 |

## 5. 기술 제약사항

1. **반응형 브레이크포인트**: Tailwind `md:` (768px) 기준 — MainLayout과 동일
2. **SSE 스트리밍**: 기존 채팅/번역에 SSE 패턴 이미 구현되어 있어 재활용 가능
3. **Claude API**: `@anthropic-ai/sdk` 이미 설치, 기존 AI 어시스턴트 패턴 참조
4. **모바일 앱**: Capacitor 앱에는 별도 채팅 뷰 존재 — 웹 모바일만 대상

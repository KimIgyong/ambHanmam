### 2.4 index.html meta 태그 분석

**문제점:**
1. **title에 "AMA" 표기** — 리브랜딩 "Ạ/에이닷(A-dot)" 미반영 (7개소)
2. **모든 페이지 동일 meta** — `/pricing`에서도 랜딩 description 노출
3. **canonical이 정적** — 모든 페이지에서 `https://www.amoeba.site/` 고정
4. **og:image가 SVG** — Facebook/LinkedIn/Twitter에서 미리보기 렌더링 불가
5. **hreflang 없음** — 다국어 지원하나 검색엔진에 알리지 않음

## 5. 사용자 플로우

### 5.1 검색엔진 크롤러 플로우

```
Googlebot → robots.txt 확인 → sitemap.xml 로드
  → 각 URL 크롤링
    → index.html 로드 (JS 실행 대기)
    → CSR 렌더링 완료 후 콘텐츠 인덱싱
    → Helmet 동적 meta 읽기
    → canonical · hreflang 확인
    → JSON-LD 구조화 데이터 파싱
```

### 5.2 소셜 공유 플로우

```
사용자 → www.amoeba.site/pricing 공유
  → Facebook/LinkedIn 크롤러 → index.html 의 og 태그 읽기
    AS-IS: 랜딩 title/description 표시 (부정확)
    TO-BE: Pricing 전용 title/description 표시 (정확)
  → og:image 로드
    AS-IS: SVG → 렌더링 실패 → 미리보기 없음
    TO-BE: PNG → 정상 렌더링 → 미리보기 표시
```

## 6. 기술 제약사항

### 6.1 SPA 한계

| 제약 | 설명 | 대응 |
|------|------|------|
| **CSR 렌더링** | Googlebot은 JS 실행 후 크롤링하나 소셜 크롤러는 미지원 | Helmet은 소셜 크롤러에 무효 (정적 인덱스 필요) |
| **동적 meta 한계** | SPA에서 `<head>` 변경은 JS 실행 필요 | 주요 페이지는 prerender 또는 서버 미들웨어 고려 |
| **200 응답 코드** | 모든 경로가 200 반환 (nginx try_files) | 실제 404 HTTP 응답 불가 (SPA 특성) |

### 6.2 리브랜딩 의존성

- AMA→Ạ/에이닷(A-dot) 리브랜딩이 진행 중 (`index.html` 7개소 미완료)
- 이번 SEO 작업에서 index.html 리브랜딩도 함께 처리
- **브랜드명 표기 규칙**: EN 메타태그/title → "Ạ/A-dot", KO 메타태그/title → "Ạ/에이닷"

### 6.3 비용/시간 고려

| 접근법 | 복잡도 | 효과 |
|--------|--------|------|
| react-helmet-async (Googlebot용) | 낮음 | 높음 |
| prerender.io 연동 (소셜 크롤러용) | 중간 | 매우 높음 |
| Next.js 마이그레이션 (SSR) | 매우 높음 | 최대 |

**권장**: react-helmet-async로 Googlebot 대응, 소셜 크롤러는 index.html 정적 태그 최적화로 대응  
**이유**: 최소 변경으로 최대 효과, SSR 마이그레이션은 별도 프로젝트 검토

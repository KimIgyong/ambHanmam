# 요구사항 분석서 (REQ)

- 문서명: 웹메일 설정 관리 기능 추가
- 작성일: 2026-02-23
- 대상 경로: `/settings/mail-accounts`
- 관련 도메인: `apps/web/src/domain/settings`, `apps/api/src/domain/webmail`

## 1) 요구사항 배경

현재 웹메일 계정 관리 페이지는 "계정 목록" 중심으로 제공되어, 실제 운영 중인 메일 계정의 SMTP/IMAP 설정(호스트, 포트 등)을 관리자 UI에서 점검/수정하기 어렵다. 이로 인해 SMTP 연결 장애 발생 시 DB 직접 수정 또는 서버 환경변수 확인에 의존하게 되며, 장애 대응 시간이 증가한다.

## 2) AS-IS 현황 분석

### 프론트엔드
- `/settings/mail-accounts` 페이지는 계정 목록, 검색, 계정 추가/수정/삭제, 활성화 토글 제공.
- 계정 카드에서 사용자 매핑/활성상태/동기화 시각 조회 가능.
- SMTP/IMAP 상세 설정 조회/수정 UI 없음.

### 백엔드
- `mail/admin/accounts` 계정 CRUD API는 존재.
- `amb_mail_accounts`에 `mac_imap_host`, `mac_imap_port`, `mac_smtp_host`, `mac_smtp_port` 컬럼 존재.
- 암호는 암호화 저장(`mac_encrypted_password`, `mac_encrypted_iv`, `mac_encrypted_tag`).
- 계정 상세 설정 조회/수정 전용 API 부재.
- 연결 테스트 API 부재.

### 운영 이슈 관찰
- SMTP 설정 불일치 시 메일 발송 API에서 timeout 기반 장애가 발생.
- 최근 `POST /api/v1/mail/messages` 500 오류가 SMTP 연결 실패로 확인됨.

## 3) TO-BE 요구사항

### 기능 요구사항
1. `/settings/mail-accounts`에 **[웹메일 설정]** 탭을 추가한다.
2. 선택한 메일 계정의 현재 설정을 조회할 수 있어야 한다.
3. 아래 설정을 수정/저장할 수 있어야 한다.
   - 표시명
   - IMAP 호스트/포트
   - SMTP 호스트/포트
   - (선택) 비밀번호 변경
4. 저장 전/후 SMTP 연결 테스트를 수행할 수 있어야 한다.
5. 변경 이력 추적을 위해 수정 시각/수정 주체 정보를 응답에 포함할 수 있어야 한다(가능하면).

### 비기능 요구사항
- 권한: 관리자(ADMIN)만 접근 가능.
- 보안: 비밀번호는 평문 반환 금지, 저장 시 기존 암호화 로직 사용.
- 응답 형식: 표준 응답 `{ success, data, error?, timestamp }` 준수.
- i18n: 프론트 UI 텍스트 하드코딩 금지, `locales` 사용.
- 장애내성: 연결 테스트 timeout을 명시적으로 설정.

## 5) 사용자 플로우

1. 관리자 로그인 → `설정 > 메일 계정 관리` 진입
2. `[웹메일 설정]` 탭 선택
3. 계정 선택(드롭다운 또는 목록 선택)
4. 현재 설정 로드(호스트/포트/표시명)
5. 값 수정 후 `연결 테스트`
6. 테스트 성공 시 `저장`
7. 저장 성공 토스트/상태 메시지 확인

## 6) 기술 제약사항

1. **암호화 일관성**: 비밀번호 갱신 시 `MailAccountService`의 기존 암호화 유틸을 재사용해야 함.
2. **DB 스키마 호환성**: 기존 `amb_mail_accounts` 컬럼 범위 내 구현 우선. 신규 컬럼은 최소화.
3. **권한/보안**: `@Roles('ADMIN')` 및 Guard 체계 유지.
4. **API 규격**: 요청 snake_case, 응답 camelCase 준수.
5. **UI 규격**: 기존 Settings 디자인/컴포넌트 스타일 일관성 유지.

## 7) 수용 기준(AC)

- AC-1: `/settings/mail-accounts`에 `[웹메일 설정]` 탭이 표시된다.
- AC-2: 계정 선택 시 현재 SMTP/IMAP 설정이 화면에 로드된다.
- AC-3: 호스트/포트/표시명 수정 후 저장이 성공한다.
- AC-4: 비밀번호 입력 시에만 암호가 갱신된다(미입력 시 기존 유지).
- AC-5: 연결 테스트 실패 시 500 일반오류 대신 원인 메시지/상태를 확인할 수 있다.
- AC-6: i18n 키 기반으로 한/영/베 UI가 깨지지 않는다.

## 8) 제외 범위 (Out of Scope)

- 다중 SMTP 프로파일 관리
- 발송 이력 분석 리포트

  - 대응: 비밀번호 미입력 시 기존값 유지, 변경 시 명확한 안내 문구

# 요구사항 분석서: 이슈 관련자(Participant) 및 상태 이력 개선

**작성일**: 2026-03-10
**분류**: 기능 추가, 이슈 관리 개선

---

## 1. 요구사항 원문

1. 이슈 등록 시 담당자 이외에 **관련자(Participant)** 지정 필요 — 여러 명 가능
2. 이슈 **상태 변경 히스토리** 보기 필요
3. 담당자가 다른 사람으로 변경된 경우, **이전 담당자는 자동으로 관련자로 전환**

---

## 2. AS-IS 현황 분석

### 2.1 이슈 엔티티 (`issue.entity.ts`)
- **Reporter** (`iss_reporter_id`): 작성자 (필수)
- **Assignee** (`iss_assignee_id`): 담당자 (선택, 1명)
- **관련자 개념 없음**: Reporter + Assignee만 존재
- 권한: 작성자, 담당자, MANAGER/ADMIN만 수정/상태변경 가능

### 2.2 상태 변경 이력 (이미 존재)
- **테이블**: `amb_issue_status_logs` (`IssueStatusLogEntity`)
  - `isl_from_status`, `isl_to_status`, `isl_changed_by`, `isl_note`, `isl_created_at`
- **API**: `GET /issues/:id/status-logs` (이미 구현)
- **프론트엔드**: `IssueStatusHistory` 컴포넌트 — IssueDetailModal 하단에 토글로 표시
- **문제점**: 기본 접힘 상태(`showHistory=false`)로 사용자가 인지하기 어려움

### 2.3 담당자 변경 처리 (`issue.service.ts`)
- `updateIssue()` 메서드에서 `assignee_id` 변경 시 이벤트 발행 (`ASSIGNEE_EVENT`)
- 이전 담당자에 대한 처리 없음 — 변경 후 이전 담당자는 이슈와 무관해짐

### 2.4 이슈 폼 (`IssueFormModal.tsx`)
- `AssigneeSelector` 컴포넌트로 담당자 1명 선택
- 관련자 선택 UI 없음

### 2.5 이슈 상세 (`IssueDetailModal.tsx`)
- Reporter, Assignee 표시
- 관련자 표시 없음
- 상태 이력은 하단에 토글 형태 (기본 접힘)

## 3. TO-BE 요구사항

### 3.1 관련자(Participant) 기능
- 별도 연결 테이블 `amb_issue_participants` 신규
  - `isp_id` (PK, UUID), `iss_id` (FK), `usr_id` (FK), `isp_role` (varchar), `isp_created_at`
  - `isp_role`: 'PARTICIPANT' (수동 지정), 'FORMER_ASSIGNEE' (자동 전환)
- 이슈 생성/수정 시 관련자 복수 선택 가능
- 관련자도 이슈 상세 조회, 코멘트 작성 가능
- 관련자에게 상태 변경/코멘트 알림 발송

### 3.2 상태 변경 이력 UI 개선
- 이미 백엔드/API 구현 완료 → **프론트엔드 UI만 개선**
- IssueDetailModal에서 기본 펼침 상태로 변경 또는 더 눈에 띄는 위치로 이동

### 3.3 담당자 변경 시 자동 관련자 전환
- `updateIssue()`에서 `assignee_id` 변경 감지 시:
  - 이전 assignee가 있고, 새 assignee와 다르면
  - 이전 assignee를 `amb_issue_participants`에 자동 추가 (role: 'FORMER_ASSIGNEE')
  - 중복 방지: 이미 관련자로 등록되어 있으면 무시

## 5. 기술 제약사항

- `amb_issue_participants` 테이블 신규 → DB 마이그레이션 (스테이징 수동)
- 기존 `IssueResponse`에 `participants` 필드 추가 — 하위호환 (optional 배열)
- 필터 `scope: 'my_involved'`에 관련자도 포함되어야 함

## 6. 사용자 플로우

### 관련자 지정
1. 이슈 등록/수정 폼에서 담당자 아래 "관련자" 멀티 셀렉터
2. 멤버 검색 → 복수 선택 → 칩 형태로 표시
3. 저장 시 `participant_ids[]` 전송

### 담당자 변경 시 자동 전환
1. 이슈 상세에서 담당자를 A → B로 변경
2. A가 자동으로 관련자 목록에 추가 (FORMER_ASSIGNEE)
3. A에게 "관련자로 전환됨" 알림

### 상태 이력 확인
1. 이슈 상세 모달에서 상태 이력 섹션이 기본 펼쳐진 상태
2. 타임라인 형태로 변경 이력 확인

## 1. 요구사항 배경

AMB Management는 다수의 법인(HrEntity)으로 구성된 그룹사 관리 플랫폼이다.
현재 회원 초대 이메일은 모든 법인에 동일한 단일 템플릿으로 발송되며,
수신자가 어느 법인에 초대되었는지 이메일만으로는 식별하기 어렵다.

### 이슈 자료 (원문)
```
AMB Management - Invitation
받은편지함

AMB Management
PM 1:08 (0분 전)

나에게

AMB Management
You have been invited by kanguk to join AMB Management.

Role: MANAGER
Department: Holding

Click the link below to complete your registration:

모든 사용자에게 동일하게 전달됨.

사용자 초대한 법인명/조직명 포함되어야하며 법인별로 초대메일 템플릿 수정 가능해야 함
```

---

## 2. AS-IS 현황 분석

### 2.1 이메일 발송 흐름

```
InvitationService.create()
  └─ mailService.sendInvitationEmail(email, inviterName, role, department, token)
       └─ nodemailer로 단일 HTML 템플릿 발송

InvitationService.resend()
  └─ mailService.sendInvitationEmail(email, inviterName, role, department, token)
       └─ 동일 템플릿 재발송
```

### 2.2 현재 이메일 템플릿 (mail.service.ts)

```html
<h2 style="color: #4F46E5;">AMB Management</h2>
<p>You have been invited by <strong>${inviterName}</strong> to join AMB Management.</p>
<p><strong>Role:</strong> ${role}<br/><strong>Department:</strong> ${department}</p>
```

**문제점:**
- 제목: 항상 `"AMB Management - Invitation"` 고정
- 본문: "join AMB Management" 고정 — 법인명/조직명 미반영
- 디자인: 보라색(#4F46E5) 단일 브랜드 색상
- 로고: 텍스트 "AMB Management" 고정
- 법인 정보: `invCompanyId` DB 저장은 되지만 이메일 발송 시 미활용

### 2.3 관련 엔티티 현황

**`amb_invitations` (InvitationEntity)**
| 컬럼 | 현황 |
|------|------|
| `inv_company_id` | 초대 법인 ID 저장 ✅ |
| `inv_role` | 역할 저장 ✅ |
| `inv_unit` | 부서 저장 ✅ |

**`amb_hr_entities` (HrEntityEntity)**
| 컬럼 | 현황 |
|------|------|
| `ent_name` | 법인명(한국어) ✅ |
| `ent_name_en` | 법인명(영어) ✅ |
| `ent_country` | 국가 코드 ✅ |
| `ent_code` | 법인 코드 ✅ |
| 브랜드 색상 | **없음** ❌ |
| 이메일 로고 URL | **없음** ❌ |
| 이메일 표시명 | **없음** ❌ |

### 2.4 현재 `sendInvitationEmail` 시그니처

```typescript
async sendInvitationEmail(
  to: string,
  inviterName: string,
  role: string,
  department: string,
  token: string,
): Promise<boolean>
```

**법인 정보 파라미터 없음** — 메서드가 법인 컨텍스트를 인식하지 못함

## 3. TO-BE 요구사항

### 3.1 기능 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-01 | 초대 이메일 제목에 법인명 포함 | 필수 |
| F-02 | 초대 이메일 본문에 법인명/조직명 포함 | 필수 |
| F-03 | 법인별 브랜드 색상 적용 가능 | 선택 |
| F-04 | 법인별 로고 이미지 표시 가능 | 선택 |
| F-05 | 초대 이메일 재발송(resend) 시에도 법인 정보 반영 | 필수 |
| F-06 | 관리자가 법인별 이메일 메타 정보 설정 가능 | 선택 |

### 3.2 비기능 요구사항

- 법인 정보가 없는 경우(companyId null) 기본 "AMB Management" 템플릿 fallback
- 법인 이메일 컬럼 미설정 시 기본값으로 동작
- HTML 이메일 주요 클라이언트(Gmail, Outlook) 호환성 유지

### 3.3 기대 이메일 형태 (TO-BE)

**제목:** `[법인명] - AMB Management 초대`  
예: `[AMB Korea] - AMB Management Invitation`

**본문:**
```
[법인 로고 또는 법인명 헤더]

You have been invited by ${inviterName} to join ${companyName}.

Role: MANAGER
Department: Holding

Click the link below to complete your registration:
[Accept Invitation 버튼]
```

## 5. 사용자 플로우

### 5.1 초대 발송 (관리자)

```
관리자 → 멤버 초대 → 이메일/역할/부서/법인 입력
→ InvitationService.create()
→ [신규] HrEntityEntity 조회 (companyId 기반)
→ sendInvitationEmail(법인 정보 포함)
→ 법인별 커스텀 이메일 발송
```

### 5.2 초대 수신 (신규 사용자)

```
이메일 수신 (법인명/브랜드 표시)
→ Accept Invitation 클릭
→ /register?invitation_token=xxx
→ 회원가입 완료
```

## 6. 기술 제약사항

- NestJS `MailService` 수정 시 기존 `sendRawEmail`, `sendPasswordResetEmail` 영향 없음
- `HrEntityEntity` 스키마 변경 시 DB 마이그레이션 필요 (optional 컬럼이므로 무중단 가능)
- 이메일 로고: URL 기반 (base64 임베딩은 스팸 필터 위험으로 비권장)
- `peaceful-lovelace`, `goofy-dirac`, `epic-hoover` 환경들도 동기화 필요

## 7. 구현 범위 제안

### Phase 1 (필수) - DB 마이그레이션 없이 즉시 구현 가능
- `HrEntityEntity`의 **기존 필드** (`entName`, `entNameEn`, `entCountry`)만 활용
- `sendInvitationEmail()` 파라미터에 `companyName` 추가
- 이메일 제목·본문에 법인명 동적 반영
- 로고: 없으면 텍스트로 표시

### Phase 2 (선택) - DB 마이그레이션 필요
- `amb_hr_entities` 테이블에 이메일 메타 컬럼 추가
  - `ent_email_logo_url` (VARCHAR 500)
  - `ent_email_brand_color` (VARCHAR 10, 기본값 `#4F46E5`)
  - `ent_email_display_name` (VARCHAR 200)
- 관리자 UI에서 법인별 이메일 설정 가능
- 이메일 템플릿에 로고 이미지 + 브랜드 색상 동적 적용

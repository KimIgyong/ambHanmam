## 1. 요구사항 요약

사이드메뉴바의 3개 카테고리(Work Tools, Work Modules, Custom Apps)에 대하여:
1. **카테고리 제목 클릭 시** 해당 카테고리 하위 메뉴의 바로가기 카드뷰 페이지로 이동
2. **카테고리 제목 우측** 하위메뉴 닫기/펼치기 토글 버튼 추가
3. **3개의 카드뷰 페이지** 신규 구현 (카테고리별 독립 페이지)
4. **카드뷰 페이지의 메뉴 노출 기준**은 사이드바와 동일한 권한/유닛/셀 기준 적용

## 2. AS-IS 현황 분석

### 2-1. 현재 사이드바 구조

**파일**: `apps/web/src/layouts/MainLayout.tsx`

```
MainLayout
├── Header
├── Mobile Drawer (모바일용 오버레이 사이드바)
├── Desktop Sidebar
│   ├── 상단: "MENU" 레이블 + 사이드바 접기/펼치기 버튼
│   ├── nav (스크롤 가능)
│   │   ├── [WORK TOOLS] 카테고리 레이블 (p 태그 - 클릭 불가)
│   │   │   └── 하위 메뉴 아이템들 (button 태그)
│   │   ├── [WORK MODULES] 카테고리 레이블 (p 태그 - 클릭 불가)
│   │   │   └── 하위 메뉴 아이템들 (button 태그)
│   │   └── [CUSTOM APPS] 카테고리 레이블 (p 태그 - 클릭 불가)
│   │       └── 커스텀앱 아이템들 (button 태그)
│   └── 하단: My Page, Entity Settings, Settings, Logout
└── Main Content (Outlet)
```

### 2-2. 데이터 소스

| 카테고리 | 데이터 소스 | 필터링 방식 |
|---------|-----------|-----------|
| Work Tools | `useMyMenus()` | `menuCode`가 `WORK_TOOL_CODES` 배열에 포함된 항목 |
| Work Modules | `useMyMenus()` | `menuCode`가 `WORK_TOOL_CODES` 배열에 미포함된 항목 |
| Custom Apps | `useMyCustomApps()` | 별도 API (`/entity-settings/custom-apps/my`) |

**WORK_TOOL_CODES**: `TODO`, `MEETING_NOTES`, `DOCUMENTS`, `AMOEBA_TALK`, `ATTENDANCE`, `MY_LEAVE`, `NOTICES`, `ISSUES`, `PROJECT_MANAGEMENT`, `CALENDAR`

### 2-3. 권한/노출 기준

- `useMyMenus()`: 서버에서 현재 사용자의 역할(role) + 유닛(unit) + 셀(cell) 기반으로 접근 가능한 메뉴 목록 반환
- `useMyCustomApps()`: 현재 사용자 역할에 맞는 활성 커스텀 앱 목록 반환  
- 두 훅 모두 현재 선택된 법인(`currentEntity`) 기반으로 동작
- **별도 API를 추가할 필요 없음** — 기존 훅을 그대로 재사용

### 2-4. 현재 문제점 (AS-IS 한계)

| # | 문제 | 영향 |
|---|------|------|
| 1 | 카테고리 레이블(`p` 태그)이 클릭 불가 | 직관적인 네비게이션 불가 |
| 2 | 하위메뉴 닫기/펼치기 기능 없음 | 메뉴 수가 많을 때 사이드바 가독성 저하 |
| 3 | 카테고리별 메뉴 오버뷰 페이지 없음 | 전체 메뉴 일람을 한눈에 볼 수 없음 |

## 3. TO-BE 요구사항

### 3-1. 사이드바 UI 변경

#### 카테고리 헤더 개선
```
[Before]
<p class="...text-gray-400">Work Tools</p>

[After]
<div class="flex items-center justify-between">
  <button onClick={navigate('/menu/work-tools')}>Work Tools</button>
  <button onClick={toggleWorkTools}>
    {workToolsExpanded ? <ChevronUp /> : <ChevronDown />}
  </button>
</div>
```

- 카테고리 제목: 클릭 시 해당 카드뷰 페이지로 이동 (hover 스타일 적용)
- 우측 토글 버튼: 클릭 시 해당 카테고리 하위메뉴 펼침/닫힘 전환
- 각 카테고리별 독립적 상태 관리 (`useState`)
- 기본 상태: 모두 펼침(expanded)
- 사이드바가 collapsed(최소화) 상태일 때: 토글 버튼 미표시, 클릭이벤트 무효
- 모바일 드로어에도 동일 적용

### 3-2. 신규 카드뷰 페이지 (3개)

#### 페이지 경로

| 카테고리 | 경로 | 페이지 컴포넌트 |
|---------|------|--------------|
| Work Tools | `/menu/work-tools` | `MenuCategoryPage` (type: workTools) |
| Work Modules | `/menu/work-modules` | `MenuCategoryPage` (type: workModules) |
| Custom Apps | `/menu/custom-apps` | `MenuCategoryPage` (type: customApps) |

#### 카드뷰 페이지 요구사항

**레이아웃**:
- 페이지 제목: 카테고리명 (Work Tools / Work Modules / Custom Apps)
- 부제목: 해당 카테고리의 메뉴 개수 (예: "12 menus")
- 카드 그리드: 반응형 (모바일 2열, 태블릿 3열, 데스크탑 4~5열)

**카드 구성 요소**:
- 아이콘 (컬러 적용)
- 메뉴명 (다국어 번역 적용)
- 설명 (있는 경우)
- 클릭 시 해당 메뉴 경로로 이동

**카드 상호작용**:
- hover 효과: 카드 상승(shadow 강화), border 색상 변경
- active 메뉴: 강조 스타일(indigo 배경)
- 클릭: 해당 경로로 navigate

**권한 기반 노출**:
- `useMyMenus()` / `useMyCustomApps()` 재사용
- 사이드바와 동일한 메뉴가 표시됨 (별도 권한 체크 불필요)

**빈 상태 처리**:
- 표시할 메뉴가 없는 경우: "표시할 메뉴가 없습니다" 안내 메시지

### 3-3. 라우팅 추가

## 4. 사용자 플로우

### 4-1. 카테고리 제목 클릭

```
[사이드바 - Work Tools 제목 클릭]
    → navigate('/menu/work-tools')
    → Work Tools 카드뷰 페이지 렌더링
    → useMyMenus() 호출 (캐시 활용, 재요청 없음)
    → workTools 배열 기반 카드 렌더링
```

### 4-2. 카테고리 접기/펼치기

```
[사이드바 - Work Tools 우측 ChevronUp 버튼 클릭]
    → workToolsExpanded: true → false
    → Work Tools 하위 메뉴 아이템들 사라짐 (애니메이션)
    → ChevronUp → ChevronDown 아이콘 변경

[다시 ChevronDown 클릭]
    → workToolsExpanded: false → true
    → Work Tools 하위 메뉴 아이템들 펼쳐짐 (애니메이션)
```

### 4-3. 카드뷰에서 메뉴 진입

```
[Work Tools 카드뷰 페이지]
    → Todo 카드 클릭
    → navigate('/todos')
    → Todo 페이지 이동
```

## 6. 기술 제약사항 및 고려사항

### 6-1. 사이드바 collapsed 상태 처리
- 사이드바가 접힌(collapsed) 상태일 때 카테고리 레이블 자체가 숨겨짐
- collapsed 상태에서는 토글 버튼/클릭 이벤트 불필요 → 기존 `!isCollapsed` 조건 활용

### 6-2. 상태 지속성
- 카테고리 접기 상태는 **세션 내 메모리 상태**(`useState`)로 관리
- 페이지 새로고침(refresh) 시 모두 펼침 상태로 초기화 (localStorage 저장 불필요)

### 6-3. 모바일 드로어 동기화
- 데스크탑 사이드바와 모바일 드로어는 **동일한 상태**를 공유해야 함
- `renderMenuItems(inDrawer)` 함수 내에서 동일 상태 참조

### 6-4. 애니메이션
- TailwindCSS의 `transition` 클래스 활용
- 접기/펼치기 시 부드러운 높이 전환 (또는 단순 표시/숨김)

### 6-5. 카드뷰 페이지 단일 컴포넌트
- Work Tools / Work Modules / Custom Apps 카드뷰 페이지를 **하나의 컴포넌트**(`MenuCategoryPage`)로 구현
- `type` prop으로 구분하여 코드 중복 최소화

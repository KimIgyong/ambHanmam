# 요구사항 분석서: Today 페이지 우측 영역 UI 개선

**문서번호**: REQ-Today우측영역개선-20260324  
**작성일**: 2026-03-24  
**작성자**: AI Assistant  
**상태**: 분석 완료

---

## 1. 요구사항 요약

Today 페이지의 **우측 영역**을 기존 월간 History 캘린더에서 **조직 전체 소식을 통합 표시하는 사이드 패널**로 개선한다. 개인의 오늘 업무 파악 집중은 유지하면서, 별도 메뉴 이동 없이 회사 내 주요 소식을 한눈에 확인할 수 있도록 한다.

---

## 2. AS-IS 현황 분석

### 2.1 현재 Today 페이지 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│ Header: Today · 날짜/시간                                 │
│ Tab Bar: [My] [Unit] [Cell] [All] + [History][AI][Report] │
├────────────────────────────────┬─────────────────────────┤
│ Main Content (flex-1)          │ Right Panel (w-64)       │
│                                │                          │
│ • MissionSection (오늘의 미션)   │ TodayHistoryCalendar    │
│ • TodaySummaryBar (요약 카드)   │ ┌──────────────────┐   │
│ • 할일 목록 (Today/InProgress)  │ │ History (월간보기) │   │
│ • 이슈 목록 (3탭: 확인/작업/관계) │ │ 2026.03           │   │
│ • 회의록 (최근 5개)              │ │ [달력 그리드]      │   │
│ • 오늘 일정                     │ │ 통계: N일 기록     │   │
│                                │ └──────────────────┘   │
├────────────────────────────────┴─────────────────────────┤
│ Modals: AI Analysis, Reports                              │
└──────────────────────────────────────────────────────────┘
```

### 2.2 현재 우측 패널 특징

| 항목 | 상세 |
|------|------|
| **위치** | `w-64` (256px), `lg:sticky lg:top-8` |
| **표시 조건** | "My" 탭 + History 토글 ON일 때만 표시 |
| **내용** | `TodayHistoryCalendar` 단일 컴포넌트 |
| **기능** | 월간 캘린더, 날짜별 스냅샷 링크, 달성도 색상 점, 월간 통계 |

### 2.3 문제점

1. **정보 단절**: 회사 공지, 다른 팀 이슈 등을 보려면 별도 메뉴로 이동 필요
2. **활용도 저**: 우측 월간 캘린더가 차지하는 공간 대비 정보 밀도가 낮음
3. **조직 소식 부재**: 메인 랜딩페이지인데 조직 전체 소식 확인 경로 없음

### 2.4 관련 기존 시스템

| 시스템 | 엔티티 | 핵심 필드 | API | 프론트 훅 |
|--------|--------|----------|-----|----------|
| **공지사항** | `amb_notices` | `ntcIsPinned`, `ntcVisibility`, `ntcTitle` | `GET /api/v1/notices` | `useNoticeList()` |
| **회의록/노트** | `amb_meeting_notes` | `mtnVisibility` (ENTITY=회사공유), `mtnType` | `GET /api/v1/meeting-notes?visibility=ENTITY` | `useMeetingNoteList()` |
| **이슈** | `amb_issues` | `issStatus`, `issType` | `GET /api/v1/issues?status=IN_PROGRESS` | `useIssueList()` |

> **기존 API가 모두 존재** — 새 백엔드 엔드포인트 불필요

### 3.2 섹션별 상세 요구사항

#### ❶ 공지사항 (최근 2개)

| 항목 | 상세 |
|------|------|
| **데이터** | 기존 `GET /api/v1/notices` (size=2) |
| **정렬** | 고정(pinned) 우선 → 최신순 |
| **표시 정보** | 제목, 작성일 (상대 시간), 고정 아이콘(📌) |
| **인터랙션** | 클릭 시 `/notices/:id` 이동 |
| **빈 상태** | 공지사항 없음 표시 |
| **더보기** | 헤더에 "더보기" 링크 → `/notices` |

#### ❷ History (주간보기 기본)

| 항목 | 상세 |
|------|------|
| **기본 보기** | **주간 보기** — 해당 주의 7일 (월~일) |
| **월간 보기** | 토글 버튼으로 기존 월간 캘린더 표시 (옵션) |
| **표시 정보** | 주간: 요일별 날짜 + 달성도 색상 점 + 기록 여부 |
| **인터랙션** | 날짜 클릭 → `/today/history/:date` |
| **주 이동** | ◀ / ▶ 버튼으로 이전/다음 주 이동 |
| **통계** | 이번 주 기록 N일 / 평균 달성도 |

#### ❸ 공유 노트 (최근 5개)

| 항목 | 상세 |
|------|------|
| **데이터** | `GET /api/v1/meeting-notes?scope=all&size=5` (회사 전체 공유) |
| **필터** | visibility가 ENTITY 또는 PUBLIC인 노트 |
| **정렬** | 최신 작성순 |
| **표시 정보** | 제목, 작성자명, 작성일 (상대 시간), 타입 아이콘 |
| **인터랙션** | 클릭 시 `/meeting-notes/:id` 이동 |
| **빈 상태** | 공유된 노트 없음 표시 |
| **더보기** | 헤더에 "더보기" 링크 → `/meeting-notes` |

#### ❹ 진행중 이슈 (최근 5개)

| 항목 | 상세 |
|------|------|
| **데이터** | `GET /api/v1/issues?status=IN_PROGRESS,TEST,APPROVED&size=5` (회사 전체) |
| **필터** | 작업 진행중인 이슈 (scope 미지정 = entity 전체) |
| **정렬** | 우선순위 높은 순 → 최신순 |
| **표시 정보** | 심각도 컬러 뱃지, Ref 번호, 제목 (줄임), 담당자 아바타 |
| **인터랙션** | 클릭 시 이슈 상세 모달 또는 해당 프로젝트 이슈 페이지 이동 |
| **빈 상태** | 진행중 이슈 없음 표시 |
| **더보기** | 헤더에 "더보기" 링크 → 이슈 필터 페이지 |

### 3.3 반응형 동작

| 화면 크기 | 동작 |
|-----------|------|
| **Desktop (lg+)** | 좌우 2컬럼: Main + Right Panel (sticky) |
| **Tablet/Mobile** | 우측 패널 접힘, 토글 버튼으로 표시/숨김 |

### 3.4 표시 조건 변경

- **AS-IS**: "My" 탭 + History 토글 ON일 때만
- **TO-BE**: **"My" 탭에서 항상 표시** (History 토글 불필요, 제거)
  - 조직 소식은 개인 탭에서 항상 볼 수 있어야 함

## 5. 사용자 플로우

### 5.1 메인 랜딩 플로우

```
1. 로그인 → Today 페이지 랜딩 (My 탭)
2. 좌측: 개인 미션, 할일, 이슈, 일정 확인
3. 우측: 회사 소식 한눈에 확인
   ├── 공지사항 2개 → 클릭 시 공지 상세
   ├── 이번 주 History → 클릭 시 스냅샷 상세
   ├── 공유 노트 5개 → 클릭 시 노트 상세
   └── 진행중 이슈 5개 → 클릭 시 이슈 상세
4. 더보기 → 각 모듈 전체 목록 페이지로 이동
```

## 6. 기술 제약사항

1. **백엔드 변경 불필요**: 공지/노트/이슈 모두 기존 API로 충분
2. **프론트엔드만 변경**: 우측 패널 컴포넌트 리팩토링
3. **성능**: 4개 API 동시 호출 → React Query 캐싱으로 관리
4. **i18n**: 새 번역 키 추가 필요 (ko/en/vi)
5. **기존 컴포넌트 보존**: `TodayHistoryCalendar`를 리팩토링하여 주간/월간 모드 지원

## 2. TO-BE 요구사항

### 2.1 관리자 측 - 프로젝트 고객사 배정

> 프로젝트 상세 페이지의 사이드바에 "배정 고객사" 섹션을 추가하여, 관리자가 현재 Entity의 고객사 목록에서 선택하여 프로젝트에 고객사를 배정/해제할 수 있어야 한다.

**상세 요건**:
1. 프로젝트 상세 Overview 탭 사이드바에 `ProjectClientList` 컴포넌트 추가
2. "고객사 추가" 버튼 클릭 시 현재 Entity의 고객사(SvcClient) 목록 드롭다운 표시
3. 고객사 선택 → `POST /project/projects/:id/clients` 호출
4. 배정된 고객사 목록 표시 (회사명, 배정일, 상태)
5. X 버튼으로 배정 해제 → `DELETE /project/projects/:id/clients/:clientId` 호출

### 2.2 고객 포털 - 이슈 상태 변경 확장

> 현재 고객이 RESOLVED→CLOSED(해결 확인)만 수행 가능하다. 기존 확인 기능 외에 추가적으로 필요한 상태 변경은 없다. 단, 이슈 상세 페이지에서 현재 상태와 가능한 액션을 더 명확히 보여줘야 한다.

**검토 결과**: 기존 이슈 상태 전이 규칙이 적절함.
- 고객은 이슈를 등록(OPEN)하고, 내부에서 처리(APPROVED→IN_PROGRESS→RESOLVED)한 후 고객이 해결을 확인(CLOSED)하는 워크플로우
- 고객에게 추가 상태 변경 권한 부여 불필요

## 4. 사용자 플로우

### 4.1 관리자: 프로젝트에 고객사 배정

```
1. 관리자가 프로젝트 상세 페이지 접속 (/project/projects/:id)
2. Overview 탭 우측 사이드바에서 "배정 고객사" 섹션 확인
3. "고객사 추가" 버튼 클릭
4. 현재 Entity의 고객사 목록이 드롭다운으로 표시
5. 고객사 선택 → 배정 완료 (토스트 알림)
6. 배정된 고객사 목록에 즉시 반영
7. X 버튼으로 배정 해제 가능
```

### 4.2 고객사 직원: 프로젝트 이슈 관리

```
1. 고객사 직원이 /client/login에서 로그인
2. /client/projects에서 배정된 프로젝트 확인
3. 프로젝트 클릭 → 프로젝트 상세 페이지
4. "새 이슈 작성" 버튼으로 이슈 등록
5. /client/issues에서 내 이슈 목록 확인
6. 이슈 클릭 → 상세 페이지에서 진행 상황 확인
7. 코멘트 추가로 소통
8. 이슈가 RESOLVED되면 "해결 확인" 버튼으로 종료
```

## 5. 기술 제약사항

1. **인증 분리**: 고객 포털은 `ClientGuard` (CLIENT_LEVEL), 관리자는 `EntityGuard` (USER_LEVEL) 사용
2. **고객사 단위 배정**: 프로젝트 배정은 고객사(회사) 단위. 고객사 내 모든 직원이 동일 프로젝트 접근
3. **기존 API 활용**: 백엔드 API가 이미 완전 구현이므로 프론트엔드만 추가
4. **Entity 고객사 조회**: Entity Settings의 `entityClientApiService.getClients()`를 활용하여 배정 가능 고객사 목록 조회

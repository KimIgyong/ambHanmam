# REQ-260413-sidebar-user-admin: 사이드바 사용자/관리 기능 추가

**작성일:** 2026-04-13  
**Feature:** sidebar-user-admin  
**우선순위:** High

---

## 1. 요구사항 요약

좌측 사이드바 하단에 마이페이지/로그아웃 기능을 추가하고, 마이페이지 화면과 관리자 메뉴 화면을 신규 개발한다.

| # | 요구사항 | 분류 |
|---|---------|------|
| R1 | 사이드바 하단에 사용자 정보 + 마이페이지/로그아웃 링크 추가 | UI/네비게이션 |
| R2 | 마이페이지: 스탭이름, 비밀번호 변경, 메뉴 노출 설정 | 신규 화면 |
| R3 | 관리자 메뉴: 사이트명 설정, 기타 설정을 카드뷰로 노출 | 신규 화면 |

---

## 2. AS-IS 현황 분석

### 2.1 현재 사이드바 구조 (`AppSidebar.tsx`)

```
┌─────────────────────┐
│ 나의업무 (고정)       │  ← LNB_MY_TASKS
│  · 나의 업무          │
├─────────────────────┤
│ 메뉴 (GNB 연동)      │  ← activeSubMenus
│  · 회의관리           │
│  · 프로젝트관리       │
│  · ...               │
│                      │
│                      │
│  (하단 빈 공간)       │  ← 사용자/로그아웃 없음
└─────────────────────┘
```

**문제점:**
- 로그아웃 기능이 어디에도 없음 (쿠키 직접 삭제 외 방법 없음)
- 사용자 본인 정보 확인/수정 화면 없음
- 관리자 설정 화면 없음
- 미들웨어에 `ADMIN` 라우트(`/admin/*`) 분기 정의되어 있으나 관리자 화면 미구현

### 2.2 인증 현황

- 미들웨어: 쿠키(`hanmam_level`)로 `ADMIN` / `USER` / `CLIENT` 분기
- `auth.store.ts`: 빈 파일 (Zustand 스토어 미구현)
- 로그인: `/user/login` Mock 로그인 (쿠키 설정만)

### 2.3 관련 라우트 미들웨어

| 경로 패턴 | 필요 level | 상태 |
|-----------|-----------|------|
| `/admin/*` | ADMIN | 미들웨어 정의됨, 화면 없음 |
| `/dashboard`, `/management/*` 등 | USER | 동작 중 |
| `/mypage/*` | 미정의 | 신규 필요 |

## 3. TO-BE 요구사항

### 3.1 사이드바 하단 영역 (R1)

사이드바 하단에 고정(sticky) 사용자 영역을 추가한다.

```
┌─────────────────────┐
│ 나의업무 (고정)       │
│  · 나의 업무          │
├─────────────────────┤
│ 메뉴 (GNB 연동)      │
│  · 회의관리           │
│  · ...               │
│                      │
├─────────────────────┤  ← 하단 고정 (border-top)
│ 👤 홍길동             │  ← 사용자 이름 (Mock: 쿠키/스토어)
│ 마이페이지 | 로그아웃   │  ← 링크 2개
│ [관리자메뉴]          │  ← ADMIN level일 때만 노출
└─────────────────────┘
```

| 항목 | 동작 |
|------|------|
| 사용자 이름 | 현재 로그인 사용자 이름 표시 (Mock 단계: 고정값) |
| 마이페이지 링크 | `/mypage`로 이동 |
| 로그아웃 | `hanmam_level` 쿠키 삭제 → `/user/login`으로 리다이렉트 |
| 관리자메뉴 | `level === 'ADMIN'`일 때만 표시, `/admin/settings`로 이동 |

### 3.2 마이페이지 화면 (R2)

**경로:** `/mypage`  
**접근 권한:** USER, MANAGER, ADMIN 모두 접근 가능

| 섹션 | 내용 |
|------|------|
| **프로필 정보** | 이름, 사번, 부서, 직급 표시 (읽기 전용) |
| **비밀번호 변경** | 현재 비밀번호 + 신규 비밀번호 + 확인 입력 폼 |
| **메뉴 노출 설정** | GNB 탭별 ON/OFF 토글 → 사용자별 숨기고 싶은 메뉴 차단 |

```
┌──────────────────────────────────────────────┐
│ 마이페이지                                    │
├──────────────────────────────────────────────┤
│ 프로필 정보                                   │
│ ┌──────────────────────────────────────────┐ │
│ │ 이름: 홍길동      사번: EMP-2024-001     │ │
│ │ 부서: 개발1팀     직급: 대리             │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ 비밀번호 변경                                 │
│ ┌──────────────────────────────────────────┐ │
│ │ 현재 비밀번호: [________]                │ │
│ │ 새 비밀번호:   [________]                │ │
│ │ 비밀번호 확인: [________]                │ │
│ │                        [비밀번호 변경]    │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ 메뉴 노출 설정                                │
│ ┌──────────────────────────────────────────┐ │
│ │ ☑ 경영정보  ☑ 영업업무  ☐ 서비스업무     │ │
│ │ ☑ 업무지원  ☑ 나의업무  ☑ AMA업무도구    │ │
│ │                             [저장]       │ │
│ └──────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

### 3.3 관리자 메뉴 화면 (R3)

**경로:** `/admin/settings`  
**접근 권한:** ADMIN 전용

카드뷰 레이아웃으로 관리 설정 항목을 표시한다.

```
┌──────────────────────────────────────────────┐
│ 관리자 설정                                   │
├──────────────────────────────────────────────┤
│ ┌─────────────┐ ┌─────────────┐             │
│ │ 🏢 사이트명  │ │ 👥 사용자    │             │
│ │ 설정        │ │ 관리        │             │
│ │             │ │             │             │
│ └─────────────┘ └─────────────┘             │
│ ┌─────────────┐ ┌─────────────┐             │
│ │ 🔑 권한     │ │ 🏷️ 마스터   │             │
│ │ 관리        │ │ 코드        │             │
│ │             │ │             │             │
│ └─────────────┘ └─────────────┘             │
│ ┌─────────────┐ ┌─────────────┐             │
│ │ 🤖 AI 쿼터  │ │ 📋 시스템   │             │
│ │ 관리        │ │ 로그        │             │
│ │             │ │             │             │
│ └─────────────┘ └─────────────┘             │
└──────────────────────────────────────────────┘
```

| 카드 | 경로 | 설명 |
|------|------|------|
| 사이트명 설정 | `/admin/settings/site` | 사이트 타이틀, 로고, 회사명 변경 |
| 사용자 관리 | `/admin/settings/users` | 사용자 등록/수정/삭제, 역할 부여 |
| 권한 관리 | `/admin/settings/roles` | 역할별 메뉴/기능 접근 권한 설정 |
| 마스터 코드 | `/admin/settings/codes` | 공통 코드 관리 (부서, 직급, 분류 등) |
| AI 쿼터 관리 | `/admin/settings/ai-quota` | 사용자별 AI 토큰 일일/월간 한도 |
| 시스템 로그 | `/admin/settings/logs` | 시스템 접속/변경 이력 조회 |

## 5. 사용자 플로우

### 5.1 마이페이지 접근

```
사이드바 "마이페이지" 클릭
  → /mypage 이동
  → 프로필 정보 확인
  → 비밀번호 변경 또는 메뉴 설정 변경
  → [저장] → 성공 토스트
```

### 5.2 로그아웃

```
사이드바 "로그아웃" 클릭
  → hanmam_level 쿠키 삭제
  → /user/login 리다이렉트
```

### 5.3 관리자 설정 접근

```
ADMIN 사용자 사이드바 "관리자메뉴" 클릭
  → /admin/settings 이동
  → 카드 클릭 → 해당 설정 화면 진입
```

## 6. 기술 제약사항

| # | 제약 | 대응 |
|---|------|------|
| C1 | 현재 Mock 인증 (쿠키 기반) → 백엔드 API 미구현 | 프론트엔드 Mock 데이터로 먼저 UI 구현, API 연동은 auth-login 구현 후 |
| C2 | 비밀번호 변경 API 없음 | Mock 토스트만 표시, 실제 변경은 백엔드 구현 후 |
| C3 | 메뉴 노출 설정 저장소 없음 | localStorage 또는 Zustand 스토어에 임시 저장, 추후 DB 연동 |
| C4 | 관리자 설정 하위 화면들은 상세 구현 대상 아님 | 카드뷰 진입점만 먼저 구현, 각 설정 화면은 별도 태스크 |

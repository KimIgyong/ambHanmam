# REQ-프로젝트상세페이지-UX개선-20260328

## 1. 요구사항 요약

프로젝트 상세 페이지(`/project/projects/:id`)의 3가지 영역 개선:

| # | 요구사항 | 설명 |
|---|---------|------|
| R1 | **Notes 탭 추가** | Overview ↔ Issues 사이에 Notes 탭 추가. 해당 프로젝트에 연결된(Related) 회의록/메모 목록 표시 |
| R2 | **Overview 탭 UI 개선** | (a) 항목:값 레이아웃 컴팩트화, (b) Summary에 번역 패널 추가, (c) 프로젝트 코멘트 기능 추가 |
| R3 | **Import Issues 소스 선택 UX** | Import Issues 클릭 시 데이터를 즉시 가져오지 않고, 가져올 소스(Redmine/Asana) 선택 후 데이터 연동 |

## 2. AS-IS 현황 분석

### 2.1 프로젝트 상세 페이지 탭 구조

**파일**: `apps/web/src/domain/project/pages/ProjectDetailPage.tsx`

```typescript
type TabKey = 'overview' | 'issues' | 'wbs' | 'kanban' | 'gantt';
const TABS = [
  { key: 'overview', icon: FileText },
  { key: 'issues', icon: Bug },
  { key: 'kanban', icon: Columns3 },
  { key: 'wbs', icon: FolderTree },
  { key: 'gantt', icon: BarChart3 },
];
```

- Notes 탭 없음
- 탭 순서: Overview → Issues → Kanban → WBS → Gantt

### 2.2 Overview 탭 레이아웃

**파일**: `apps/web/src/domain/project/components/ProjectOverviewTab.tsx`

현재 구조:
- 3-column grid (`lg:grid-cols-3`): 좌측 2칸 + 우측 사이드바 1칸
- 좌측: 프로젝트 기본 정보 카드 → Summary → Note → Files
- 우측: ProjectMemberList + ProjectClientList
- 기본 정보는 `grid-cols-2`로 필드 배치 (label 위 + value 아래 세로 배치)

**문제점**:
1. 각 필드가 라벨(위) + 값(아래) 세로 배치 → 공간 낭비
2. Summary 섹션에 번역 패널 없음 (다른 모듈은 TranslationPanel 사용 중)
3. 프로젝트 수준 코멘트 기능 없음 (이슈/노트/할일에만 존재)

### 2.3 프로젝트 ↔ 회의록 관계

**Junction Table**: `amb_meeting_note_projects`
- `mnp_id` (PK, UUID)
- `mtn_id` (FK → amb_meeting_notes)
- `pjt_id` (FK → kms_projects)
- Unique: `(mtn_id, pjt_id)`

**엔티티**: `apps/api/src/domain/meeting-notes/entity/meeting-note-project.entity.ts`
- `MeetingNoteProjectEntity` with ManyToOne relations

**백엔드 현황**:
- `MeetingNoteService.getMeetingNotes()`: project ID 필터 미지원
- `ProjectService`: 회의록 역조회 메서드 없음
- 노트 생성 시 프로젝트 연결은 `syncProjects(noteId, projectIds)` 로 지원

### 2.4 Import Issues 모달

**파일**: `apps/web/src/domain/project/components/ProjectIssueImportModal.tsx`

현재 구조:
```
┌──────────────────────────────────┐
│ Import Issues                     │
│ [Redmine] [Asana]  ← 소스 탭     │
│──────────────────────────────────│
│ (즉시 Redmine 도구 목록 로드)     │
│ 도구 선택 → 프로젝트 → 이슈 목록  │
│ 체크박스 선택 → Import             │
└──────────────────────────────────┘
```

- 모달 오픈 시 `activeSource` 기본값 `'redmine'`
- `RedmineImportContent`가 즉시 렌더링되며 도구 목록 API 호출
- 소스 선택도 이미 탭으로 존재하나, 데이터 로딩이 선택 전에 시작됨

### 2.5 번역 패널

**파일**: `apps/web/src/domain/translations/components/TranslationPanel.tsx`

```typescript
interface TranslationPanelProps {
  sourceType: 'TODO' | 'MEETING_NOTE' | 'NOTICE' | 'ISSUE' | 'ISSUE_COMMENT' | 'PROJECT' | 'PARTNER' | 'CLIENT';
  sourceId: string;
  sourceFields: string[];
  originalLang: string;
  originalContent: Record<string, string>;
}
```

- `sourceType: 'PROJECT'` 이미 지원
- SSE 스트리밍 번역, ko/en/vi 3개 언어 지원
- Issues, Todos, Meeting Notes, Notices 등에서 활발히 사용 중

### 2.6 코멘트 시스템

프로젝트에는 코멘트 시스템 미구현. 관련 엔티티/API 없음.

기존 코멘트 패턴 (Issues 기준):
- **테이블**: `amb_issue_comments` (`isc_id`, `iss_id`, `usr_id`, `isc_content`, `isc_created_at`, `isc_deleted_at`)
- **리액션**: `amb_issue_comment_reactions` 별도 테이블
- **프론트**: `IssueDetailModal.tsx`에서 댓글 목록 + 입력 + 삭제 + 멘션 + 리액션 구현
- **번역**: 코멘트별 TranslationPanel 토글 지원

## 3. TO-BE 요구사항

### R1: Notes 탭 추가

**목표**: 프로젝트에 연결된 회의록/메모를 한눈에 볼 수 있는 전용 탭

| 항목 | 내용 |
|------|------|
| 탭 위치 | Overview와 Issues 사이 |
| 탭 이름 | Notes |
| 데이터 소스 | `amb_meeting_note_projects` JOIN → 해당 프로젝트의 노트 목록 |
| 표시 정보 | 제목, 유형(Meeting Note/Memo), 작성자, 작성일, Visibility |
| 클릭 동작 | 노트 상세 페이지로 이동 또는 모달 표시 |
| 빈 상태 | "연결된 노트가 없습니다" 메시지 |

**백엔드 변경**:
- `MeetingNoteService` 또는 `ProjectService`에 프로젝트별 노트 조회 메서드 추가
- API: `GET /api/v1/project/projects/:id/notes`
- 응답: 노트 목록 (id, title, type, visibility, authorName, createdAt)

### R2: Overview 탭 UI 개선

#### R2-a: 항목:값 레이아웃 컴팩트화

현재: 라벨(위) + 값(아래) 세로 배치
변경: `항목 : 값` 가로 배치 (항목명 좌측 고정, 값 우측)

```
Project Code : PJ-2020000    │  Status : Approved
Project Name : AMA - project
Category : SI                 │  Priority : High
Proposer : name               │  Manager : name
Start Date : YYYY-MM-DD       │  End Date : YYYY-MM-DD
Budget : 000                  │  Approved : YYYY-MM-DD
```

#### R2-b: Summary 번역 패널

- Summary 섹션 하단에 `TranslationPanel` 추가
- `sourceType="PROJECT"`, `sourceFields=['summary']`
- 기존 `TranslationPanel` 그대로 활용

#### R2-c: 프로젝트 코멘트

**DB 변경**:
- 신규 테이블: `amb_project_comments`
  - `pjc_id` (PK, UUID)
  - `pjt_id` (FK → kms_projects)
  - `usr_id` (FK → amb_users)
  - `pjc_content` (TEXT, HTML)
  - `pjc_created_at` (TIMESTAMP)
  - `pjc_deleted_at` (TIMESTAMP, nullable, soft delete)

**API**:
- `GET /api/v1/project/projects/:id/comments` — 코멘트 목록
- `POST /api/v1/project/projects/:id/comments` — 코멘트 추가 (body: `{ content }`)
- `DELETE /api/v1/project/projects/:id/comments/:commentId` — 코멘트 삭제

**프론트**:
- Overview 탭 하단(파일 섹션 아래)에 코멘트 섹션 추가
- RichTextEditor(80px, 멘션 지원) + 댓글 목록
- Issues/Todos 코멘트와 동일한 UX

### R3: Import Issues 소스 선택 UX

**현재**: 모달 오픈 시 소스 선택 탭 + 데이터가 동시에 표시
**변경**: 소스 미선택 시 안내 메시지만 표시, 소스 클릭 후 해당 소스의 데이터 로딩 시작

현재 이미 소스 탭(Redmine/Asana)이 존재하고 있으며, `activeSource` state로 전환.
다만 기본값이 `'redmine'`이어서 바로 데이터를 불러오는 것이 문제.

**변경 방안**: `activeSource` 초기값을 `null`로 설정하고, 미선택 시 소스 선택 유도 UI 표시

## 5. 사용자 플로우

### R1: Notes 탭

```
프로젝트 상세 → Notes 탭 클릭
 → API: GET /project/projects/:id/notes
 → 연결된 노트/회의록 목록 표시 (제목, 유형, 작성자, 일시)
 → 노트 클릭 → /meeting-notes/:noteId 이동
 → 연결된 노트 없음 → "연결된 노트가 없습니다" 빈 상태
```

### R2-b: Summary 번역

```
Overview 탭 → Summary 섹션 확인
 → 번역 패널(ko/en/vi) 표시
 → 언어 탭 클릭 → 번역 결과 표시 (또는 "Translate" 버튼)
 → "Translate All" → SSE 스트리밍 번역
```

### R2-c: 프로젝트 코멘트

```
Overview 탭 → 하단 코멘트 섹션
 → RichTextEditor에 코멘트 입력 → 전송 버튼 클릭
 → API: POST /project/:id/comments
 → 코멘트 목록 갱신
 → 코멘트 삭제: 본인 코멘트 삭제 버튼 → API: DELETE
```

### R3: Import Issues

```
Issues 탭 → Import Issues 버튼 클릭
 → 모달 오픈: 소스 선택 화면 (Redmine / Asana 카드)
 → 소스 미선택 상태: "가져올 소스를 선택하세요" 안내
 → Redmine 클릭 → Redmine 도구/프로젝트/이슈 목록 로딩
 → Asana 클릭 → Asana 그룹/태스크 로딩
```

## 6. 기술 제약사항

### 데이터 격리
- 모든 API에 `@Auth()` + `@UseGuards(OwnEntityGuard)` + `resolveEntityId()` 필수
- 프로젝트 코멘트 조회/생성/삭제 시 프로젝트의 `ent_id` 기반 접근 권한 검증
- 노트 조회 시에도 프로젝트 소속 법인 기반 필터링

### DB 마이그레이션
- 스테이징/프로덕션은 TypeORM synchronize 비활성화
- `amb_project_comments` 테이블은 수동 SQL로 생성 필요

### 기존 호환성
- TranslationPanel은 `sourceType='PROJECT'` 이미 지원 → 추가 백엔드 작업 불필요
- Import 모달은 기존 Redmine/Asana 로직 유지, 초기 state만 변경

### i18n
- 새 번역 키: Notes 탭 라벨, 코멘트 관련 텍스트, Import 소스 선택 안내문
- ko/en/vi 3개 언어 파일 업데이트 필요
- 네임스페이스: 기존 `project` 네임스페이스에 추가

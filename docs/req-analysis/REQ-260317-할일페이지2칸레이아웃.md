## 2. AS-IS 현황 분석

### 2.1 현재 레이아웃 구조
- **단일 컬럼 세로 리스트** 형태로 5개 섹션이 순차 배치
- 섹션 순서: `OVERDUE` → `DUE_TODAY` → `IN_PROGRESS` → `UPCOMING_SOON` → `UPCOMING`
- 각 섹션은 배경색/테두리가 다른 카드 형태의 `rounded-xl` 컨테이너
- **Completed 섹션**: 접힘/펼침(toggle) 방식으로 표시
  - `ChevronDown`/`ChevronRight` 아이콘으로 토글
  - 펼치면 완료된 모든 태스크가 `TodoItem` 컴포넌트로 렌더링

### 2.2 현재 컴포넌트 구조
```
TodoPage.tsx
├── ViewScopeTab (mine / unit / cell / company)
├── [mine 탭]
│   ├── OVERDUE 섹션 카드
│   ├── DUE_TODAY 섹션 카드
│   ├── IN_PROGRESS 섹션 카드
│   ├── UPCOMING_SOON 섹션 카드
│   ├── UPCOMING 섹션 카드
│   └── COMPLETED 섹션 카드 (접힘/펼침 토글)
├── [unit/cell/company 탭] → ScopedTodoList
├── TodoFormModal
├── TodoDeleteConfirmModal
├── TodoStatusHistory
├── TodoDetailModal
└── SaveTranslationDialog
```

### 2.3 데이터 분류 로직
- `computedStatus` 필드 기반으로 6개 그룹 분류 (`useMemo`)
- 백엔드에서 `computedStatus`를 계산하여 응답
- 상태: `OVERDUE`, `DUE_TODAY`, `IN_PROGRESS`, `UPCOMING_SOON`, `UPCOMING`, `COMPLETED`

### 2.4 라우팅
- `/todos` 단일 라우트 → `TodoPage` 컴포넌트
- 완료 태스크 전용 라우트 없음

## 3. TO-BE 요구사항

### 3.1 메인 목록 2칸 분할 뷰 (mine 탭)
기존 단일 컬럼 세로 리스트를 **2칸 분할 뷰**로 변경한다.

| 왼쪽 칸 | 오른쪽 칸 |
|---------|----------|
| OVERDUE (기한 초과) | UPCOMING_SOON (곧 시작) |
| DUE_TODAY (오늘 마감) | UPCOMING (예정) |
| IN_PROGRESS (진행 중) | |

> **참고**: OVERDUE는 긴급 항목이므로 왼쪽 칸 최상단에 표시

- 왼쪽 칸: **즉각적 주의 필요** — 기한 초과, 오늘 마감, 진행 중
- 오른쪽 칸: **향후 예정** — 곧 시작, 예정

### 3.2 Completed(완료) 섹션 변경
- **AS-IS**: 접힘/펼침 토글로 완료 목록 전체 표시
- **TO-BE**: 완료된 숫자만 표기 (목록 펼쳐보기 기능 제거)
- 제목("완료 (N)") 클릭 시 **별도 완료 페이지** (`/todos/completed`)로 이동

### 3.3 완료된 태스크 전용 페이지
- 새 라우트: `/todos/completed`
- 완료된 태스크 목록을 별도 페이지에서 열람
- 기존 `TodoItem` 컴포넌트 재사용
- 메인 Todo 페이지(`/todos`)로 돌아가기 링크 제공
- 태스크 재시작(Reopen) 기능 유지

## 4. 갭 분석 (AS-IS vs TO-BE)

| 섹션 배치 | 순차적 나열 (OVERDUE→DUE_TODAY→IN_PROGRESS→UPCOMING_SOON→UPCOMING) | 좌(OVERDUE+DUE_TODAY+IN_PROGRESS) / 우(UPCOMING_SOON+UPCOMING) | 중 |

| 라우팅 | 단일 라우트 | 2개 라우트 | 소 |

## 5. 사용자 플로우

### 5.1 메인 Todo 페이지 (`/todos`)
```
사용자 → /todos 진입
  ├── (mine 탭) 2칸 레이아웃 표시
  │   ├── 왼쪽 칸
  │   │   ├── OVERDUE (기한 초과)
  │   │   ├── DUE_TODAY (오늘 마감)
  │   │   └── IN_PROGRESS (진행 중)
  │   ├── 오른쪽 칸
  │   │   ├── UPCOMING_SOON (곧 시작)
  │   │   └── UPCOMING (예정)
  │   └── 하단 (2칸 전체 너비)
  │       └── COMPLETED 요약 바 (숫자만) → 클릭 시 /todos/completed
  ├── (unit/cell/company 탭) → 기존 ScopedTodoList (변경 없음)
  └── 기존 CRUD/모달 기능 동일
```

### 5.2 완료 태스크 페이지 (`/todos/completed`)
```
사용자 → "완료 (N)" 클릭 → /todos/completed 이동
  ├── 페이지 헤더: "완료된 할일" + /todos 복귀 링크
  ├── 완료된 태스크 목록 (TodoItem 컴포넌트 재사용)
  │   ├── 완료 체크 토글 → Reopen 가능
  │   ├── 수정/삭제 가능
  │   └── 상세보기 모달
  └── 빈 상태: "완료된 할일이 없습니다"
```

## 6. 기술 제약사항

1. **반응형**: 모바일(sm 이하)에서는 1칸으로 폴백 → `grid-cols-1 md:grid-cols-2`
2. **ScopedTodoList 미변경**: unit/cell/company 탭은 기존 레이아웃 유지
3. **기존 기능 보존**: 태스크 CRUD, 상세 모달, 번역 다이얼로그 등 기존 기능에 영향 없어야 함
4. **i18n**: 새로운 UI 텍스트는 ko/en/vi 번역 키 추가

# 요구사항 분석서: 프로젝트 멤버 추가 기능 버그 수정

**문서번호**: REQ-프로젝트멤버추가기능버그수정-20260304
**작성일**: 2026-03-04
**분류**: Bug Fix

## 1. AS-IS 현황 분석

### 1.1 현재 동작
- Project > Overview > Members 섹션에서 "Add Member" 클릭
- 사용자 선택 + 역할 지정 후 "Add" 버튼 클릭
- **결과**: 멤버 목록 변경 없음, 성공/에러 메시지 없음

### 1.2 코드 분석 - 근본 원인

#### 원인 1: Mapper 속성명 불일치 (Critical)
**파일**: `apps/api/src/domain/project/mapper/project.mapper.ts:54-66`

```typescript
// 현재 코드 (잘못됨)
userName: entity.user?.name || '',      // UserEntity에 'name' 속성 없음
userEmail: entity.user?.email || '',    // UserEntity에 'email' 속성 없음
```

`UserEntity`(`apps/api/src/domain/auth/entity/user.entity.ts`)의 실제 속성:
- `usrName` (line 25) — `name` 아님
- `usrEmail` (line 19) — `email` 아님

**영향**: `userName`과 `userEmail`이 항상 빈 문자열 `''`로 반환됨.
→ 멤버 추가 API는 성공하지만 멤버 목록에 이름/이메일 없이 표시되어 사용자가 추가되지 않은 것으로 인식.

#### 원인 2: 프론트엔드 피드백 부재 (Major)
**파일**: `apps/web/src/domain/project/components/ProjectMemberList.tsx:22-34`

- `addMember.mutate()`에 `onError` 콜백 없음
- 성공 시 토스트 메시지 없음
- 실패 시 에러 메시지 없음
- 로딩 상태 표시 불충분 (버튼 비활성화만 있음)

#### 영향 범위
동일한 Mapper 버그가 다음에도 영향:
- `ProjectMapper.toResponse()` — `proposerName`, `managerName` 빈 문자열
- 멤버 목록 조회 (`GET /project/projects/:id/members`) — 기존 멤버도 이름 없이 표시

### 1.3 데이터 흐름

```
[프론트엔드]
ProjectMemberList.handleAdd()
  → addMember.mutate({ projectId, data: { user_id, role } })
    → projectApiService.addMember()
      → POST /api/v1/project/projects/{id}/members

[백엔드]
ProjectController.addMember()
  → ProjectMemberService.addMember()
    → 프로젝트 존재 확인 ✅
    → 중복 체크 ✅
    → memberRepo.create() + save() ✅
    → memberRepo.findOne({ relations: ['user'] }) ✅
    → ProjectMapper.toMemberResponse()
      → entity.user?.name → undefined → '' ❌ (usrName이어야 함)
      → entity.user?.email → undefined → '' ❌ (usrEmail이어야 함)
```

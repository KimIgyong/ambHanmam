|------|------|

- 현재 이슈/할일 알림은 SSE → sonner toast로 우하단 표시
- 채팅 메시지는 Web Push만 → 브라우저 알림 (비포커스 시만 보임)
- **화면 중앙 모달은 어떤 알림에도 미구현**

## 3. TO-BE 요구사항

### 3.1 PC — 화면 중앙 모달 알림

- SSE 알림 수신 시 화면 중앙에 모달 팝업 표시
- 모달 내용: 알림 타입 아이콘 + 제목 + 메시지 요약
- 자동 닫힘 (5초) 또는 수동 닫힘 (X 버튼)
- "바로가기" 버튼 → 해당 리소스 페이지로 이동
- 여러 알림이 동시에 오면 스택/큐 처리

### 3.2 채팅 메시지 알림 보완

- 백엔드: 채팅 메시지도 SSE로 발행하여 PC에서 모달 알림 표시 가능하게
- 현재 Push만 발송 → SSE 채널 추가

### 3.3 모바일 Push Notification

- Capacitor Push Notification 플러그인 설치
- 앱 시작 시 Push 토큰 → 백엔드 등록
- 알림 수신 시 네이티브 푸시 표시
- 알림 클릭 시 해당 화면으로 이동

## 5. 기술 제약사항

### 5.1 PC 모달 vs Toast
- 현재 sonner toast는 우하단에 표시 — 사용자가 놓칠 수 있음
- 중앙 모달은 사용자 주의를 더 끌지만, 작업 중단 우려
- **방안**: 중앙 모달 + 자동 닫힘(5초) + 반투명 배경(클릭 안 막음)

### 5.2 채팅 알림 빈도
- 채팅은 메시지가 빈번 → 모든 메시지에 모달 표시 시 UX 저하
- **방안**: 현재 채팅 페이지에 있지 않을 때만 모달 표시
- 같은 채널 알림 통합 (최신 메시지만 표시)

### 5.3 모바일 Push 토큰 관리
- Web Push (VAPID) vs 모바일 Push (FCM/APNs)는 메커니즘이 다름
- 백엔드 `push-subscription` 테이블에 `device_type` 구분 필요
- Capacitor Push → FCM 토큰 사용 → firebase-admin SDK 또는 FCM HTTP API 필요

### 5.4 모바일 Push 백엔드 확장
- 현재 `web-push` 라이브러리는 VAPID(웹) 전용
- 모바일은 FCM (Android) / APNs (iOS) 필요
- **방안**: 단기적으로 Capacitor `@capacitor/push-notifications` + FCM 사용

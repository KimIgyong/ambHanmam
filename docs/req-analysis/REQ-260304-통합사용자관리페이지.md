# 요구사항 분석서: 통합 사용자 관리 페이지

- **작성일**: 2026-03-04
- **요청**: ADMIN_LEVEL 전용 통합 사용자 관리 페이지 구현
- **경로**: `/settings/total-users`

---

## 1. AS-IS 현황

### 1.1 현재 사용자 관련 관리 메뉴 (분산 상태)

| 메뉴 | 경로 | 관리 대상 | 한계 |
|------|------|-----------|------|
| Members | `/settings/members` | amb_users + cells | 법인 필터링된 멤버만 표시, 전체 목록 불가 |
| Portal Bridge | `/settings/portal-bridge` | portal_customers + mappings | 포탈 고객만 |
| Entities | `/settings/entities` | amb_hr_entities | 조직 관리만, 소속 사용자 연계 없음 |
| Units | `/settings/units` | amb_units | 부서 관리만 |
| Permissions | `/settings/permissions` | user_menu_permissions | 권한만 |

**문제점**: 사용자 관련 데이터가 여러 페이지에 분산되어 있어, 특정 사용자의 전체 현황을 파악하려면 여러 페이지를 오가야 한다.

### 1.2 시스템 내 사용자 관련 테이블 (10개)

| # | 테이블 | 설명 | 핵심 컬럼 |
|---|--------|------|-----------|
| 1 | `amb_users` | 메인 사용자 | email, name, role, level_code, status |
| 2 | `amb_svc_portal_customers` | 포탈 가입 고객 (읽기전용) | email, name, company, country |
| 3 | `amb_portal_user_mappings` | 포탈↔내부 매핑 | pct_id, usr_id, status |
| 4 | `amb_hr_entity_user_roles` | 법인별 사용자 역할 | ent_id, usr_id, eur_role |
| 5 | `amb_user_unit_roles` | 부서별 사용자 역할 | unt_id, usr_id, uur_role |
| 6 | `amb_user_cells` | 셀 소속 | usr_id, cel_id |
| 7 | `amb_hr_employees` | HR 직원 정보 | emp_code, usr_id, department |
| 8 | `amb_invitations` | 초대 이력 | email, status, role |
| 9 | `amb_user_menu_permissions` | 메뉴 접근 권한 | menu_code, usr_id |
| 10 | `amb_password_resets` | 비밀번호 재설정 | usr_id, token, expires |

## 2. TO-BE 요구사항

### 2.1 통합 사용자 관리 페이지

Settings에 **"Total User Management / 통합 사용자 관리"** 카드 추가.
ADMIN_LEVEL 사용자만 접근 가능.

### 2.2 탭 구조 (5개 탭)

#### Tab 1: Portal Customers (포탈 가입 고객)
- `amb_svc_portal_customers` 데이터 표시
- 매핑 상태(Mapped/Unmapped) 확인
- **기존 `/settings/portal-bridge` 페이지와 동일 기능** 연결 가능

#### Tab 2: Internal Users (내부 사용자)
- `amb_users` 전체 목록 (법인 필터 없이)
- **전체보기 / 법인별 보기** 필터
- 컬럼: 이름, 이메일, 역할, 레벨, 상태, 소속법인, 가입방법, 최종로그인, 가입일
- 행 액션: 상태 변경, 역할 변경, 삭제 (soft delete)
- 법인별 필터: 드롭다운으로 법인 선택 → 해당 법인 소속 사용자만 표시

#### Tab 3: Entity-User Roles (법인별 역할)
- `amb_hr_entity_user_roles` + `amb_hr_entities` + `amb_users` JOIN 데이터
- 어떤 사용자가 어떤 법인에 어떤 역할로 소속되어 있는지
- 역할 변경, 삭제 가능

#### Tab 4: Unit-User Roles (부서별 역할)
- `amb_user_unit_roles` + `amb_units` + `amb_users` JOIN 데이터
- 부서/팀별 사용자 역할 배정 현황
- 역할 변경, Primary 설정, 삭제 가능

#### Tab 5: Invitations (초대 이력)
- `amb_invitations` 전체 이력
- 상태별 필터 (PENDING/ACCEPTED/EXPIRED/CANCELLED)
- 재발송, 취소 가능

### 2.3 사용자 플로우

```
Settings 페이지
  └─ "Total User Management" 카드 클릭
      └─ /settings/total-users
          ├─ [Portal Customers] 탭 — 포탈 가입 고객 목록
          ├─ [Internal Users] 탭 — 내부 사용자 전체 목록 ★핵심
          ├─ [Entity Roles] 탭 — 법인별 사용자 역할
          ├─ [Unit Roles] 탭 — 부서별 사용자 역할
          └─ [Invitations] 탭 — 초대 이력
```

## 3. 갭 분석

### 3.1 백엔드 API 현황

| 필요 기능 | 기존 API | 상태 | 갭 |
|-----------|----------|------|-----|
| 포탈 고객 목록 | `GET /portal-bridge/customers` | ✅ 있음 | 없음 |
| 내부 사용자 전체 목록 | `GET /members` | ⚠️ 법인 필터 적용됨 | **전체 조회 API 필요** |
| 법인별 사용자 역할 | 없음 | ❌ | **새 API 필요** |
| 부서별 사용자 역할 | 없음 | ❌ | **새 API 필요** |
| 초대 이력 | `GET /invitations` | ✅ 있음 | 없음 |
| 사용자 상태 변경 | `PUT /members/:id/status` | ✅ 있음 | 없음 |
| 사용자 삭제 | `DELETE /members/:id` | ✅ 있음 | 없음 |
| 법인역할 삭제 | `DELETE /members/:id/entity-roles/:eurId` | ✅ 있음 | 없음 |

### 3.2 신규 백엔드 API 필요 목록

1. **`GET /admin/users`** — 전체 사용자 목록 (법인 필터 없이, 페이징/검색)
2. **`GET /admin/entity-user-roles`** — 전체 법인-사용자 역할 목록
3. **`GET /admin/unit-user-roles`** — 전체 부서-사용자 역할 목록
4. **`DELETE /admin/unit-user-roles/:uurId`** — 부서 역할 삭제

## 4. 기술 제약사항

### 4.1 권한
- ADMIN_LEVEL (SUPER_ADMIN, ADMIN) 사용자만 접근 가능
- `AdminGuard` + `MenuGuard` 이중 보호
- 기존 `isAdmin()` 체크: `user.level === 'ADMIN_LEVEL'`

### 4.2 데이터 규모
- 사용자 수가 많지 않은 내부 시스템이므로 페이징은 클라이언트 사이드도 가능
- 단, 확장성 위해 서버 사이드 페이징 권장

### 4.3 포탈 고객 테이블
- `amb_svc_portal_customers`는 **읽기 전용** (외부 DB 동기화 데이터)
- 수정/삭제 불가, 조회만 가능

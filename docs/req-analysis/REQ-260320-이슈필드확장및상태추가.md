# 요구사항 분석서: 이슈 필드 확장 및 상태 추가

> 작성일: 2026-03-20

## 1. 요구사항 요약

| 항목 | 내용 |
|------|------|
| **R1** | Redmine 가져오기 시 이슈넘버(`#id`)와 이슈 생성일(`created_on`)을 가져온다 |
| **R2** | AMA 이슈에 "참고용 이슈넘버" 필드를 추가하고, Redmine 이슈넘버를 입력한다 |
| **R3** | Redmine 가져오기 시 Redmine의 REQUEST 상태를 AMA `APPROVED`로 매핑하여 저장 |

## 2. AS-IS 현황

### 2.1 이슈 테이블 (`amb_issues`)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `iss_redmine_id` | INT, nullable | Redmine 이슈 ID (중복 방지용, 내부 추적용) |
| `iss_created_at` | TIMESTAMP | AMA 생성일 (TypeORM `@CreateDateColumn` 자동 생성) |

**문제점:**
- `iss_redmine_id`는 내부 중복 방지용이며 사용자에게 노출되지 않음
- "참고용 이슈넘버" 개념의 별도 필드 없음
- Redmine 원본 생성일(`created_on`)이 보존되지 않음 — AMA 생성일로 덮어씀

### 2.2 이슈 상태 체계

```
현재: OPEN → APPROVED → IN_PROGRESS → RESOLVED → CLOSED
                                                    ↗
      REJECTED ← (모든 상태)
```

| 상태 | 의미 | 전이 가능 |
|------|------|-----------|
| OPEN | 신규 등록 | → APPROVED, REJECTED |
| APPROVED | 승인됨 | → IN_PROGRESS, REJECTED |
| IN_PROGRESS | 진행중 | → RESOLVED |
| RESOLVED | 해결됨 | → CLOSED, OPEN |
| CLOSED | 완료 | (종료) |
| REJECTED | 반려 | → OPEN |

### 2.3 Redmine 가져오기 매핑

```typescript
// 현재 Redmine status → AMA status
STATUS_MAP = {
  1: 'OPEN',        // New
  2: 'IN_PROGRESS', // In Progress
  3: 'RESOLVED',    // Resolved
  4: 'OPEN',        // Feedback
  5: 'CLOSED',      // Closed
  6: 'REJECTED',    // Rejected
};
```

- Redmine `id`는 `iss_redmine_id`에 저장 (중복 방지용)
- Redmine `created_on`은 사용하지 않음 → AMA `iss_created_at`은 가져온 시점

### 2.4 IssueResponse (프론트 응답)

- `redmineId` 필드 없음
- 참고용 이슈넘버 필드 없음
- `createdAt`은 AMA 생성일 (Redmine 원본 아님)

## 3. TO-BE 요구사항

### 3.1 R1: Redmine 이슈넘버 + 생성일 가져오기

| 항목 | 변경 내용 |
|------|-----------|
| 이슈넘버 | Redmine `#id` → `iss_ref_number` 필드에 `"RM-{id}"` 형식으로 저장 |
| 생성일 | Redmine `created_on` → `iss_created_at`에 원본 일자로 저장 |

### 3.2 R2: 참고용 이슈넘버 필드 추가

- **새 컬럼**: `iss_ref_number` (VARCHAR(50), nullable)
- **용도**: 외부 시스템 참조번호 (Redmine, GitHub, Jira 등)
- **표시**: 이슈 목록/상세에서 참고용 이슈넘버 표시
- **입력**: Redmine 가져오기 시 `"RM-{id}"` 자동 입력, 수동 입력도 가능

### 3.3 R3: Redmine REQUEST → AMA APPROVED 매핑

- AMA 상태 체계는 **변경 없이 보존** (OPEN → APPROVED → IN_PROGRESS → RESOLVED → CLOSED)
- Redmine의 REQUEST 상태(status_id 등)를 가져올 때 AMA `APPROVED`로 매핑
- `STATUS_MAP`에 REQUEST 매핑 추가

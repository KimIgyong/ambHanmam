## 3. 해결 방안

### 3.1 백엔드: ClaudeService 레벨 쿼터 체크 (단일 진입점 100% 커버)

**선택 이유**: 모든 AI 서비스(13개)가 ClaudeService의 `sendMessage()` 또는 `streamMessage()`를 호출함. 여기서 1회 체크하면 누락 불가.

- **`sendMessage()`**: `usageContext?.entId` 존재 시 `checkQuota()` → 초과 시 BusinessException throw
- **`streamMessage()`**: `usageContext?.entId` 존재 시 `checkQuota()` → 초과 시 SSE error 이벤트 (에러 코드 포함)
- **ChatService 중복 제거**: 기존 ChatService의 `checkQuota()` 호출 제거

### 3.2 프론트엔드: 전역 쿼터 초과 배너

- **`useAiQuotaCheck()` 훅**: `GET /entity-settings/usage` API의 `quotaStage` 활용
- **`QuotaExceededBanner` 컴포넌트**: `quotaStage === 'SUSPENDED'` 시 경고 배너 표시
- **적용 대상**: AI 채팅, 번역, 리포트, Today 분석, 이슈 AI, 회계, KMS, Amoeba-Talk

## 4. 사이드 임팩트

- ClaudeService에 AiUsageService 의존 이미 있음 (`@Optional()` 주입) → 순환 의존 없음
- ClaudeModule이 AiUsageModule을 이미 import 중 → 모듈 변경 불필요
- Mock Mode에서는 쿼터 체크 스킵 (로컬 개발 영향 없음)

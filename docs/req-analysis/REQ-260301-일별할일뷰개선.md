## 1. 요구사항 요약

"오늘의 할 일" 페이지를 **특정 날짜 기준 일별 스냅샷 뷰**로 개선한다.  
현재는 모든 미완료 할 일 + 전체 완료 할 일이 혼재되어 "할 일 보드"처럼 동작하므로, 날짜 선택 기반의 진정한 일별 뷰로 전환한다.

## 2. AS-IS 현황 분석

### 2.1 프론트엔드 구조

| 파일 | 역할 |
|------|------|
| `apps/web/src/domain/todos/pages/TodoPage.tsx` (291줄) | 메인 페이지. 탭 3개(mine/group/company) |
| `apps/web/src/domain/todos/components/ScopedTodoList.tsx` (340줄) | group/company 탭 전용 리스트 (필터 바 + 3컬럼) |

### 2.2 현재 컬럼 분류 로직 (프론트엔드)

**TodoPage (mine 탭):**
```typescript
const today = new Date().toISOString().split('T')[0];

for (const todo of todos) {
  if (todo.status === 'COMPLETED') {
    completed.push(todo);           // 완료 → 완료 컬럼 (날짜 무관)
  } else if (todo.dueDate <= today) {
    todayList.push(todo);           // 예정일 ≤ 오늘 + 미완료 → 오늘 컬럼 (기한초과 포함)
  } else {
    scheduled.push(todo);           // 예정일 > 오늘 + 미완료 → 예정 컬럼
  }
}
```

**ScopedTodoList (group/company 탭):** 동일한 로직.

### 2.3 현재 3-컬럼 구성

| 컬럼 | 기준 | 문제점 |
|------|------|--------|
| **완료 (Completed)** | `status === 'COMPLETED'` (전체) | 과거 완료 건이 전부 표시됨 |
| **오늘 할일 (Today)** | `dueDate ≤ 오늘 && status ≠ COMPLETED` | 기한 초과 건이 혼재 |
| **예정 (Scheduled)** | `dueDate > 오늘 && status ≠ COMPLETED` | 미래 전체가 표시됨 |

### 2.4 백엔드 API 필터

| API | 파라미터 | 설명 |
|-----|---------|------|
| `GET /todos` | `status`, `date_from`, `date_to` | 내 할일 |
| `GET /todos/group` | `status`, `date_from`, `date_to`, `search` | 그룹 할일 |
| `GET /todos/company` | `status`, `date_from`, `date_to`, `search` | 회사 할일 |

- `date_from`/`date_to`는 `todo.tdoDueDate` 기준으로 필터링
- **완료일(completedAt) 기준 필터가 없음** → Done 컬럼에 "해당 날짜에 완료된 것"만 표시하려면 API 확장 필요

### 2.5 문제점 정리

1. **완료 컬럼**: 날짜 무관 전체 완료 건 표시 → 일별 스냅샷이 아님
2. **오늘 컬럼**: 기한 초과 건이 구분 없이 섞임 → Overdue 식별 불가
3. **날짜 선택 기능 없음**: mine 탭에 날짜 변경 UI 없음 (`const today = new Date()` 고정)
4. **백엔드**: `completedAt` 기준 필터 미지원 → "해당 날짜에 완료된 건" 조회 불가

## 3. TO-BE 요구사항

### 3.1 페이지 목적 재정의

**"오늘의 할 일"** = 특정 날짜(기본값: 오늘) 기준 **일별 스냅샷 보드**

### 3.2 4-컬럼 구성

| 컬럼 | 분류 기준 | 색상 |
|------|----------|------|
| **Todo** | `dueDate == 선택일 && status == 'SCHEDULED'` | 회색 (기존 Scheduled 색) |
| **In Progress** | `dueDate == 선택일 && status == 'IN_PROGRESS'` | 파란색 (기존 Today 색) |
| **Done** | `completedAt`의 날짜 == 선택일 | 초록색 (기존 Completed 색) |
| **Scheduled** | `dueDate > 선택일 && status ≠ 'COMPLETED'` | 연회색 |

### 3.3 기한 초과(Overdue) 규칙

- 조건: `dueDate < 선택일 && status ≠ 'COMPLETED'`
- 표시: **Todo 컬럼**에 "Overdue" 배지와 함께 표시
- 빨간 테두리/배경으로 시각적 구분

### 3.4 날짜 선택 기능

- mine/group/company 모든 탭에 **날짜 선택 UI** 추가
- 기본값: 오늘
- 날짜 변경 시 → 새 날짜 기준으로 할 일 재필터링/재그룹화
- 이전/다음 날짜 네비게이션 버튼 (◀ ▶)
- "오늘" 빠른 이동 버튼

### 3.5 Done 컬럼 조건 변경

- AS-IS: `status === 'COMPLETED'` (날짜 무관)
- TO-BE: `completedAt`의 날짜가 선택된 날짜와 동일한 건만 표시

## 4. 갭 분석

| 항목 | AS-IS | TO-BE | 변경 범위 |
|------|-------|-------|----------|
| 컬럼 수 | 3 (Completed, Today, Scheduled) | 4 (Todo, In Progress, Done, Scheduled) | 프론트엔드 |

| mine 탭 필터 | 없음 | 날짜 기반 필터 전달 | 프론트엔드 |

## 5. 사용자 플로우

```
1. 사용자가 "오늘의 할일" 페이지 진입
2. 기본 선택일 = 오늘 (2026-03-01)
3. 4컬럼 보드 표시:
   - Todo: 예정일 3/1 + SCHEDULED 상태인 건
   - In Progress: 예정일 3/1 + IN_PROGRESS 상태인 건
   - Done: 완료일(completedAt)이 3/1인 건
   - Scheduled: 예정일 > 3/1 + 미완료인 건
   + 기한초과: 예정일 < 3/1 + 미완료 → Todo 컬럼에 Overdue 배지
4. 사용자가 날짜를 3/2로 변경
5. 모든 컬럼이 3/2 기준으로 재필터링
6. group/company 탭도 동일하게 동작
```

## 6. 기술 제약사항

### 6.1 백엔드 변경 필요

- `GET /todos`, `GET /todos/group`, `GET /todos/company` API에 `completed_date` 파라미터 추가
- 또는: 프론트에서 모든 데이터를 받아 클라이언트 필터링 (데이터 볼륨이 작으면 이 방식이 간단)

### 6.2 클라이언트 필터링 vs 서버 필터링 검토

| 방식 | 장점 | 단점 |
|------|------|------|
| **클라이언트 필터링** | 백엔드 변경 없음, 날짜 전환 시 API 재호출 불필요 | 전체 데이터 로딩 필요 |
| **서버 필터링** | 네트워크 효율적, 대규모 데이터 대응 | API 변경, 날짜 변경마다 재호출 |

**권장**: 현재 할일 데이터 볼륨이 크지 않으므로 **클라이언트 필터링** 방식 채택.
- mine 탭: `useTodoList()` 기존 API 활용, 필터 없이 전체 조회 후 프론트 분류
- group/company 탭: 기존 필터 바 유지하되 날짜 선택으로 UI 통합

### 6.3 `completedAt` 데이터 유효성

- `TodoEntity.tdoCompletedAt`: 상태가 `COMPLETED`로 변경될 때 `new Date()`로 설정됨
- `TodoResponse.completedAt`: ISO 문자열로 매핑됨
- 기존 데이터에 completedAt이 null인 COMPLETED 건이 있을 수 있음 → Done 컬럼에 표시되지 않는 이슈 가능 (데이터 정합성 확인 필요)

## 9. 리스크 및 고려사항

1. **COMPLETED + completedAt = null**: 기존 데이터에 completedAt 없이 완료 처리된 건이 있으면 Done 컬럼에 누락됨
   - 대안: completedAt이 null이면 updatedAt을 폴백으로 사용
2. **Overdue 건이 많을 경우**: Todo 컬럼이 과도하게 길어질 수 있음
   - 대안: Overdue 섹션을 접을 수 있게 하거나 별도 영역에 표시
3. **그룹/회사 필터와 날짜 선택의 관계**: ScopedTodoList의 기존 quick range 필터(all/today/week/month)와 새 날짜 선택이 충돌할 수 있음
   - 대안: group/company 탭에서는 날짜 선택이 기존 quick range 필터를 대체

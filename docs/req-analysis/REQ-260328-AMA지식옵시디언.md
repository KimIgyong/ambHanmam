## 2. AS-IS 현황 분석

### 2.1 AMA 업무 라이프사이클 (미션 → 프로젝트)

```
┌─────────────────┐
│  오늘의 미션      │  amb_daily_missions
│  (Daily Mission) │  msn_content (HTML/텍스트)
└───────┬─────────┘
        │ 하루 경과 후 라인별 체크
        │  ┌── [완료] → Todo (COMPLETED) 자동 생성
        ├──┤── [미완료 + Task] → Todo (IN_PROGRESS) 자동 생성
        │  ├── [미완료 + 미션] → carry_over_text로 오늘 미션에 이월
        │  └── [해당없음] → 무시
        ▼
┌─────────────────┐
│  Task / ToDo     │  amb_todos
│                  │  pjt_id (FK→project), iss_id (FK→issue)
│                  │  참여자: amb_todo_participants
└───────┬─────────┘
        │ UI에서 "이슈로 변환" 버튼
        ▼
┌─────────────────┐
│  Issue (이슈)     │  amb_issues
│                  │  pjt_id (FK→project), epc_id (FK→epic)
│                  │  reporter, assignee, participants
│                  │  parent_id (서브태스크)
└───────┬─────────┘
        │ 프로젝트 할당
        ▼
┌─────────────────┐
│  Project         │  kms_projects
│                  │  epics, components, members
│                  │  parent_id (프로그램→프로젝트)
└─────────────────┘
```

## 3. TO-BE 요구사항

### 3.1 양방향 링크 노트 (`[[Wiki Link]]`)

| 항목 | 상세 |
|------|------|
| `[[노트 제목]]` 입력 | TipTap 커스텀 Extension으로 `[[` 트리거 → 자동완성 드롭다운 → 노트 선택 → WikiLink 노드 삽입 |
| 링크 렌더링 | 인라인 칩(chip) 형태, 클릭 시 해당 노트로 이동 |
| 미존재 노트 | 빨간색 표시, 클릭 시 자동 생성 |
| 백링크 테이블 | 저장 시 content에서 `[[...]]` 파싱 → `amb_note_links` UPSERT |
| 백링크 패널 | 노트 상세 우측/하단에 "이 노트를 참조하는 노트 N개" 리스트 |

### 3.2 업무 체인 양방향 연결

기존 Meeting Notes의 프로젝트/이슈 연결 테이블을 확장하여:

| 연결 대상 | 현황 | TO-BE |
|-----------|------|-------|
| 프로젝트 | ✅ `amb_meeting_note_projects` 존재 | 유지 |
| 이슈 | ✅ `amb_meeting_note_issues` 존재 | 유지 |
| 태스크/ToDo | ❌ 없음 | **신규** `amb_meeting_note_todos` |
| 미션 | ❌ 없음 | **신규** `amb_meeting_note_missions` |
| 노트↔노트 | ❌ 없음 | **신규** `amb_note_links` |

→ 노트에서 `[[미션:2026-03-28]]`, `[[태스크:서버배포]]`, `[[이슈:#142]]`, `[[프로젝트:AMA]]` 문법으로 크로스 레퍼런스 가능

### 3.3 그래프 뷰

| 항목 | 상세 |
|------|------|
| 노드 타입 | 노트(파란), 태스크(보라), 이슈(빨강), 프로젝트(초록), 미션(황색) |
| 엣지 | `amb_note_links` + `meeting_note_*` 연결 테이블 |
| 필터 | 타입별, 태그별, 날짜범위, 작성자 |
| 로컬 그래프 | 특정 노트 중심 2-depth |
| 전체 그래프 | 전체 노트 네트워크 (MY/TEAM/COMPANY 스코프) |
| 라이브러리 | `react-force-graph-2d` (D3 force + WebGL, 경량) |

### 3.4 폴더 트리 + 전역 검색

| 항목 | 상세 |
|------|------|
| 폴더 구조 | `amb_note_folders` (parent_id, 무한 depth) |

| 검색 범위 | 노트 제목 + 본문 + 태그 + 연결된 항목명 |

### 3.5 데일리 노트 (미션 연동)

| 항목 | 상세 |
|------|------|
| 자동 생성 | 오늘의 미션 작성 시 해당 날짜 데일리 노트 자동 생성 (없으면) |

## 5. 사용자 플로우

### 5.1 미션 → 노트 → 태스크 → 이슈 → 프로젝트 통합 플로우

```
[사용자 A - 아침]
  │
  ├─ 오늘의 미션 작성
  │   "1. API 인증 모듈 리팩토링\n2. 클라이언트 미팅 준비\n3. 보고서 작성"
  │
  ├─ 시스템: 데일리 노트 자동 생성 (2026-03-28)
  │   내용: "## 오늘의 미션\n- API 인증 모듈 리팩토링\n- 클라이언트 미팅 준비\n- 보고서 작성"
  │
[사용자 A - 업무 중]
  │
  ├─ 노트 작성: "API 인증 리팩토링 설계"
  │   본문에 [[JWT 토큰 구조]], [[보안 정책 문서]] 링크
  │   → 백링크 자동 등록
  │
  ├─ 그래프 뷰에서 직관적으로 노트 관계 탐색
  │
[사용자 A - 다음날 아침]
  │
  ├─ 어제 미션 체크:
  │   1. API 인증 모듈 리팩토링 → [미완료] → [Task 등록]
  │      → amb_todos 생성 (IN_PROGRESS)
  │      → 데일리 노트에 [[태스크:API 인증 모듈 리팩토링]] 자동 링크
  │   2. 클라이언트 미팅 준비 → [완료]
  │      → amb_todos 생성 (COMPLETED)
  │   3. 보고서 작성 → [미완료] → [오늘 미션으로]
  │      → carry_over_text로 이월
  │
  ├─ 태스크 "API 인증 모듈 리팩토링" 상세에서
  │   프로젝트 연결: [[프로젝트:AMA 플랫폼]]
  │   → "이슈로 변환" 클릭
  │   → 이슈 생성: assignee 지정, 관련자 등록
  │   → 노트 "API 인증 리팩토링 설계" ↔ 이슈 자동 연결 (같은 제목 매칭)
  │
  └─ 그래프 뷰:
     미션 ──→ 데일리노트 ──→ 설계노트 ──→ 태스크 ──→ 이슈 ──→ 프로젝트
                │                │
                └── JWT 토큰 구조  └── 보안 정책 문서
```

### 5.2 `[[Wiki Link]]` 입력 플로우

```
사용자 입력: [[
    ↓
TipTap WikiLink Extension 트리거
    ↓
자동완성 드롭다운 표시 (노트 제목 검색)
    ↓
  ┌── 기존 노트 선택 → WikiLink 노드 삽입 (파란 칩)
  │      저장 시 → amb_note_links UPSERT (source → target)
  │
  ├── 텍스트 입력 후 ]] → 미존재 노트 링크 (빨간 칩)
  │      클릭 시 → 새 노트 자동 생성 → 이동
  │
  └── 타입 프리픽스:
       [[미션:2026-03-28]] → 해당 날짜 미션으로 링크
       [[태스크:서버배포]] → 태스크로 링크
       [[이슈:#142]] → 이슈 번호로 링크
       [[프로젝트:AMA]] → 프로젝트로 링크
```

## 6. 기술 제약사항

| 항목 | 제약 | 대응 |
|------|------|------|
| **DB 마이그레이션** | 스테이징/프로덕션 synchronize OFF → 수동 SQL | 마이그레이션 SQL 작성, ALTER TABLE 포함 |
| **번들 크기** | react-force-graph-2d ~50KB, 기존 JS 3.9MB | 동적 import로 코드 스플릿 |
| **Full-text 검색** | PostgreSQL tsvector는 한/베트남어 사전 미약 | unaccent + 바이그램(bigram) 인덱스 병행 |
| **데이터 격리** | ent_id 기반 필수 | 모든 새 테이블에 ent_id 포함, OwnEntityGuard 적용 |
| **에디터 호환** | TipTap HTML 저장, `[[Link]]` 는 커스텀 노드 | Extension에서 HTML 시리얼라이즈 규칙 정의 |
| **성능** | 그래프 뷰 노드 수 제한 필요 | 최대 500노드 페이지네이션, 로컬 그래프 우선 |
| **보안** | WikiLink를 통한 타 법인 노트 접근 차단 | 링크 resolve 시 ent_id 필터 강제 |

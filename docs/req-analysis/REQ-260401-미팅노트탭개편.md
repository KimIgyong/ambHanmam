## 1. 요구사항 요약

Meeting Notes 페이지(`/meeting-notes`)의 탭 구조와 필터 레이아웃을 개편한다.

| 항목 | 내용 |
|------|------|
| **대상 페이지** | `/meeting-notes` (MeetingNotesPage.tsx) |
| **영향 범위** | Frontend (UI/UX) + Backend (API 쿼리 파라미터 확장) + i18n |
| **DB 변경** | `mtn_type` 값에 `DAILY_NOTE` 추가 (스키마 변경 없음, 데이터 마이그레이션 필요) |

## 2. AS-IS 현황 분석

### 2.1 현재 탭 구조

#### 최상단 탭 (NoteTab)
```
[ Notes (FileText 아이콘) ]  [ Notices (Megaphone 아이콘) ]
```
- `notes` 탭: Meeting Notes 목록
- `notices` 탭: `<NoticesPage embedded />` 렌더링

#### 필터 (2줄 구조)
```
줄 1 (ViewScopeTab): [ My Only ] [ My Unit ] [ My Cell ] [ All ]
줄 2 (TypeFilter):   [ 전체 ] [ 회의록 ] [ 메모 ]
```

#### 폴더 섹션 (NoteFolderSidebar)
```
FOLDER    [+]
[ 전체 N ] [ 미분류 N ] [ folder1 N ] ...
```
- `+` 버튼만 존재 (레이블 없음)

### 2.2 Daily Note 현황

**생성 위치**: `apps/api/src/domain/today/service/daily-note.service.ts`

- `today` 도메인에서 미션 완료 시 자동 생성
- **타입**: `MEMO` (Meeting Note 서비스의 일반 MEMO와 동일 타입)
- **제목 패턴**: `📅 Daily Note YYYY-MM-DD` (예: `📅 Daily Note 2026-04-01`)
- **가시성**: `PRIVATE` (본인만 볼 수 있음)
- **구분 방법**: 현재 제목 패턴만으로 식별 (타입 구분 없음)

```typescript
// daily-note.service.ts
const dto: CreateMeetingNoteRequest = {
  type: 'MEMO',                          // ← MEMO로 저장
  title: `📅 Daily Note ${date}`,
  content: `<h2>📅 ${date} Daily Note</h2><p></p>`,
  visibility: 'PRIVATE',
};
```

### 2.3 현재 API 쿼리 파라미터 (`GET /api/v1/meeting-notes`)

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `type` | `string` | `MEETING_NOTE` \| `MEMO` (단일 타입 필터) |
| `scope` | `string` | `mine` \| `unit` \| `cell` \| `all` |
| `search` | `string` | 텍스트 검색 |
| `folder_id` | `string` | 폴더 ID |
| `page`, `size` | `number` | 페이징 |

**현재 Daily Note 제외 처리 없음**: type 파라미터가 없으면 MEETING_NOTE + MEMO + DAILY_NOTE(=현재 MEMO) 전부 조회됨.

## 3. TO-BE 요구사항

### 3.1 상단 탭 (Notes / Notices) 삭제

- `Notes | Notices` 최상단 탭을 **완전 제거**
- Notices 임베드는 제거 (Notices는 별도 메뉴로 접근)

### 3.2 Note Type 탭 + ViewScope 탭을 1줄로 합치기

**변경 전 (2줄)**:
```
[My Only] [My Unit] [My Cell] [All]          ← 줄 1 (ViewScopeTab)
[All] [Meeting Note] [Memo]                  ← 줄 2 (TypeFilter)
```

**변경 후 (1줄)**:
```
[All] [Meeting Note] [Memo] [Daily Note]     [My Only] [My Unit] [My Cell] [All]
└──────────── 좌측 정렬 ─────────────────    └──────────── 우측 정렬 ──────────┘
```

- **좌측 (Note Type 탭)**: `All (Meeting Note + Memo)` | `Meeting Note` | `Memo` | `Daily Note`
  - **All**: Meeting Note + Memo만 표시 (**Daily Note 제외**)
  - **Daily Note**: Daily Note만 표시
- **우측 (ViewScope 탭)**: `My Only` | `My Unit` | `My Cell` | `All`
  - 기존 ViewScopeTab 컴포넌트 그대로 활용, 오른쪽 정렬로 변경

### 3.3 Daily Note 타입 구분

- `mtn_type` 컬럼에 `DAILY_NOTE` 값 추가 (VARCHAR, 스키마 변경 없음)
- 기존 Daily Note (`📅 Daily Note` 제목) → `DAILY_NOTE` 타입으로 마이그레이션
- `daily-note.service.ts`에서 생성 시 `type: 'DAILY_NOTE'`로 변경
- API 조회 시 `type` 파라미터로 `DAILY_NOTE` 지원

### 3.4 기본 조회 동작 변경

| 탭 | API 요청 | 설명 |
|---|---|---|
| All | `type` 없음 + `exclude_daily=true` | Meeting Note + Memo 조회 (Daily Note 제외) |
| Meeting Note | `type=MEETING_NOTE` | 회의록만 |
| Memo | `type=MEMO` | 메모만 |
| Daily Note | `type=DAILY_NOTE` | 데일리 노트만 |

**백엔드 변경**: `exclude_daily=true` 파라미터 추가 시 `mtn_type != 'DAILY_NOTE'` 필터 적용

### 3.5 폴더 "Add Folder" 텍스트 추가

```
FOLDER    [Add Folder] [+]
```
- `+` 버튼 앞에 "Add Folder" 텍스트(버튼) 추가
- 클릭 시 기존 `+` 버튼과 동일한 새 폴더 입력 인라인 폼 열기

## 5. 사용자 플로우

### 5.1 기본 화면 진입
```
/meeting-notes 접속
→ 상단 탭 없음
→ 폴더 섹션 (폴더 Add Folder [+])
→ 1줄 필터: [All*] [Meeting Note] [Memo] [Daily Note]  [My Only*] [My Unit] [My Cell] [All]
→ Meeting Note + Memo 목록 (Daily Note 제외)
```

### 5.2 Daily Note 탭 클릭
```
[Daily Note] 탭 클릭
→ type=DAILY_NOTE 로 API 호출
→ 📅 형식 Daily Note 목록 표시
→ ViewScope 탭은 여전히 우측에서 동작 (My Only 기본)
```

### 5.3 폴더 추가
```
"Add Folder" 텍스트 클릭 OR [+] 클릭
→ 인라인 폴더명 입력 폼 오픈
→ Enter: 폴더 생성 / Esc: 취소
```

### 5.4 필터 조합 예시
```
Memo 탭 + My Unit → scope=unit&type=MEMO
Daily Note 탭 + All → type=DAILY_NOTE (scope 파라미터 없음)
All 탭 + My Cell → exclude_daily=true&scope=cell
```

## 6. 기술 제약사항

### 6.1 DB
- `mtn_type VARCHAR(20)` 컬럼: 현재 값 `MEETING_NOTE` | `MEMO`
- `DAILY_NOTE` 신규 값 추가 가능 (VARCHAR이므로 스키마 변경 없음)
- **데이터 마이그레이션 필요**: 기존 Daily Note(MEMO 타입, 제목 `📅 Daily Note%`) → `DAILY_NOTE` 타입으로 UPDATE
- 스테이징/프로덕션 `synchronize: false` → 마이그레이션 SQL 수동 적용 필수 (스키마 변경 없으므로 데이터 UPDATE만)

### 6.2 백엔드
- `getMeetingNotes()` 서비스 메서드: `exclude_daily` 파라미터 추가 필요
- `daily-note.service.ts`: `type: 'MEMO'` → `type: 'DAILY_NOTE'` 변경
- 기존 `type` 파라미터 방식 그대로 활용 (신규 `DAILY_NOTE` 값 추가)

### 6.3 프론트엔드
- `ViewScopeTab` 컴포넌트: `className` prop 지원 → 우측 정렬은 부모 컨테이너 flex layout으로 제어
- Notes/Notices 탭 제거 → `NoteTab` 타입 + 탭 관련 state 모두 제거
- `typeFilter` state: `'' | 'MEETING_NOTE' | 'MEMO' → '' | 'MEETING_NOTE' | 'MEMO' | 'DAILY_NOTE'`
- `useMeetingNoteList` hook에 `exclude_daily` 파라미터 전달 지원 필요

### 6.4 i18n
- 3개 언어(ko/en/vi) 번역 파일 업데이트
- 신규 키: `type.DAILY_NOTE`, `folder.addFolder`
- 기존 탭 관련 키 정리 (notices 관련 import 제거)

## 2. AS-IS 현황 분석

### 2.1 현재 화면 구조

**URL**: `/entity-settings/permissions`

```
[탭 구조]
Unit별 | Cell별 | 개인별

[공통 기능 — Unit별/Cell별]
- 행: 메뉴 목록 (getAvailableMenus() 반환)
- 열: Unit/Cell 목록
- 셀: 접근 허용/차단 체크박스

[개인별]
- 특정 멤버의 권한 오버라이드
```

**파일**: `apps/web/src/domain/entity-settings/pages/EntityPermissionPage.tsx`

### 2.2 메뉴 관련 DB 구조

#### `amb_menu_config` — 시스템 전역 메뉴 설정

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `mcf_id` | UUID PK | |
| `mcf_menu_code` | VARCHAR(50) UNIQUE | 메뉴 코드 |
| `mcf_label_key` | VARCHAR(100) | i18n 키 |
| `mcf_icon` | VARCHAR(50) | Lucide 아이콘명 |
| `mcf_path` | VARCHAR(200) | 라우트 경로 |
| `mcf_category` | VARCHAR(30) | 카테고리 (예: `WORK_TOOL`) |
| `mcf_enabled` | BOOLEAN | 활성 여부 |
| `mcf_sort_order` | INT | 정렬 순서 |

> 현재 법인별 메뉴 설정 테이블 없음 — **모든 법인이 동일한 기본 메뉴 순서 사용**.

### 2.3 사이드메뉴 구분 로직

```typescript
// apps/web/src/domain/menu/constants/menuConfig.ts
export const WORK_TOOL_CODES = [
  'TODAY', 'TODO', 'MEETING_NOTES', 'DRIVE',
  'AMOEBA_TALK', 'ISSUES', 'PROJECT_MANAGEMENT', 'CALENDAR',
];

// MainLayout.tsx
const workTools = sortedMenus.filter((m) => WORK_TOOL_CODES.includes(m.menuCode));
const workModules = sortedMenus.filter((m) => !WORK_TOOL_CODES.includes(m.menuCode));
```

- **업무도구**: `WORK_TOOL_CODES`에 속하는 메뉴
- **업무모듈**: 그 외 (`ACCOUNTING`, `HR`, `BILLING`, `KMS` 등)
- `sortOrder`는 `amb_menu_config.mcf_sort_order` 기반으로 정렬

### 2.4 현재 getAvailableMenus() 필터 규칙

```typescript
// entity-permission.service.ts
const RESTRICTED_MENU_PREFIXES = ['SETTINGS_', 'ENTITY_'];
const ADMIN_MODULE_CODES = ['AGENTS', 'SERVICE_MANAGEMENT', 'SITE_MANAGEMENT'];
// → 위 메뉴 제외, mcf_enabled=true인 나머지 메뉴만 반환
```

### 2.5 i18n 현황

- 네임스페이스: `entitySettings`
- 현재 번역 키:
  ```json
  "permissions.title": "권한 설정"
  "permissions.tabs.unit": "Unit별"
  "permissions.tabs.cell": "Cell별"
  "permissions.tabs.individual": "개인별"
  ```

### 2.6 사이드메뉴 렌더링 — MainLayout.tsx

```typescript
const sortedMenus = (() => {
  const menus = (myMenus || []).filter(...);
  // ATTENDANCE, MY_LEAVE는 권한 관계없이 항상 추가
  return menus.sort((a, b) => a.sortOrder - b.sortOrder);
})();

const workTools = sortedMenus.filter((m) => WORK_TOOL_CODES.includes(m.menuCode));
const workModules = sortedMenus.filter((m) => !WORK_TOOL_CODES.includes(m.menuCode));
```

`WORK_TOOL_CODES` 배열에 포함되면 업무도구 섹션, 아니면 업무모듈 섹션에 렌더링되며, **현재 법인별 override는 없음**.

#### 3.3.2 기능 명세

| 기능 | 설명 |
|------|------|
| 메뉴 목록 표시 | `getAvailableMenus()` 반환 메뉴 전부 (권한 필터링 없이 목록만) |
| 카테고리 변경 | 드롭다운으로 `업무도구` / `업무모듈` 선택 가능 |
| 순서 변경 (↑/↓) | 한 행씩 위/아래 이동 |
| 저장 | Entity별 설정으로 DB 저장 |
| 초기화 | 시스템 기본값으로 복원 (entity 설정 삭제) |
| 미설정 시 기본값 | entity 설정 없으면 `amb_menu_config` 기본값 사용 |

#### 3.3.3 카테고리 선택지

| 선택 값 | DB 저장값 | 사이드메뉴 섹션 |
|---------|-----------|---------------|
| 업무도구 | `WORK_TOOL` | 업무도구 섹션 |
| 업무모듈 | `WORK_MODULE` | 업무모듈 섹션 |

### 3.4 메뉴 순서/카테고리 적용 방식

```
사이드메뉴 렌더링 시:
1. entity용 menu config 조회 (GET /api/v1/menus?entity_id=xxx)
2. 존재하면 → entity별 sort_order + category 사용
3. 없으면 → amb_menu_config 기본값 사용
```

## 5. 사용자 플로우

### 5.1 관리자가 메뉴 순서를 변경하는 플로우

```
1. /entity-settings/permissions 접속
2. "메뉴 및 권한 설정" 페이지 표시 (기본 탭: 메뉴관리)
3. 현재 법인의 메뉴 목록 표시 (entity 설정 없으면 기본 순서)
4. 원하는 메뉴의 ↑ 또는 ↓ 버튼 클릭 → 행 위치 변경
5. 카테고리 드롭다운 변경 → 업무도구/업무모듈 전환
6. [저장] 클릭 → PUT /entity-settings/permissions/menu-config 호출
7. 저장 성공 토스트 표시
8. 이후 해당 법인 구성원 사이드메뉴에 변경된 순서/카테고리 반영

[초기화 플로우]
4-1. [초기화] 클릭 → 확인 다이얼로그
4-2. 확인 → DELETE /entity-settings/permissions/menu-config 호출
4-3. entity 설정 삭제 → 시스템 기본값으로 복원
```

### 5.2 구성원이 사이드메뉴를 보는 플로우

```
1. 로그인 후 MainLayout 렌더링
2. useMyMenus() → GET /api/v1/menus 호출
3. 해당 법인의 entity별 메뉴 설정 있으면 → 설정된 순서/카테고리 반환
4. 없으면 → 시스템 기본 순서/카테고리 반환
5. MainLayout에서 카테고리별 섹션 분리하여 렌더링
```

## 6. 기술 제약사항

| 항목 | 내용 |
|------|------|
| DB | Staging/Production에서 TypeORM `synchronize: false` → 수동 SQL 마이그레이션 필수 |
| 데이터 격리 | `ent_id` 기반 격리 — 모든 User Level API에 `OwnEntityGuard` + `resolveEntityId()` 필수 |
| i18n | 3개 언어 (ko/en/vi) 모두 번역 키 추가 필수 |
| 기존 기능 | Unit별/Cell별/개인별 탭 기존 동작 변경 없이 유지 |
| 사이드메뉴 정렬 | `useMyMenus()`/`/api/v1/menus` 반환 데이터에 entity별 override 반영 필요 |
| 카테고리 구분 | 현재 `WORK_TOOL_CODES` 상수 → API 반환 카테고리 필드로 대체 |
| 권한 | MASTER 이상만 메뉴관리 탭 수정 가능 (기존 OwnEntityGuard 로직 재사용) |

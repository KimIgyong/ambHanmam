## 1. 요구사항 개요

회계(Accounting) 모듈에 대시보드 기능을 추가하여 계좌별/통합 통계, 월별 입출금 현황, 정기 지출 관리 및 예상 문서 생성 기능을 제공한다.

### 1.1 요구 기능 목록

| # | 기능 | 설명 |
|---|------|------|
| F1 | 계좌별 통계 페이지 | 개별 계좌의 월별 입금/출금 통계 및 잔액 추이 |
| F2 | 월별 입금/출금 통계 | 월 단위 입출금 집계, 차트 시각화 |
| F3 | 계좌 통합 통계 페이지 | 선택한 계좌들의 통합 통계 |
| F4 | 정기 지출 항목 관리 | 매월 반복되는 출금 항목 등록/수정/삭제 |
| F5 | 정기 지출 예상 문서 | 월별 정기 지출 예상 보고서 생성 |

## 2. AS-IS 현황 분석

### 2.1 백엔드 현황

**모듈**: `apps/api/src/domain/accounting/`

| 항목 | 현황 |
|------|------|
| 엔티티 | `BankAccountEntity` (amb_bank_accounts), `TransactionEntity` (amb_transactions) |
| 컨트롤러 | `AccountController` (6 endpoints), `TransactionController` (5 endpoints) |
| 서비스 | `AccountingService` (CRUD + 잔액 계산), `ExcelImportService` (엑셀 임포트) |
| 통계 API | `GET /accounts/summary` — 통화별 총 잔액만 제공 (월별 통계 없음) |

**기존 데이터 모델 핵심 필드**:
- `BankAccountEntity`: bac_id, ent_id, bac_bank_name, bac_account_number, bac_currency, bac_opening_balance, bac_is_active
- `TransactionEntity`: txn_id, bac_id, txn_date, txn_net_value, txn_vat, txn_bank_charge, txn_total_value, txn_balance, txn_cumulative_balance, txn_vendor

**입출금 구분**: 별도 컬럼 없이 `txn_balance` 부호로 판단 (양수=입금, 음수=출금)

### 2.2 프론트엔드 현황

**모듈**: `apps/web/src/domain/accounting/`

| 항목 | 현황 |
|------|------|
| 페이지 | `AccountingPage` (/accounting), `AccountTransactionsPage` (/accounting/:accountId) |
| 컴포넌트 | AccountCard, AccountFormModal, BalanceSummaryCard, TransactionTable, TransactionFormModal, ExcelImportModal |

### 2.3 갭 분석

| 요구 기능 | AS-IS | GAP |
|----------|-------|-----|
| 계좌별 월별 통계 | 거래 목록만 조회 가능 | 월별 집계 API + 차트 UI 필요 |

## 3. TO-BE 요구사항

### 3.1 F1: 계좌별 통계 페이지

**경로**: `/accounting/:accountId/dashboard` (기존 거래 내역 페이지에 탭 추가 가능)

**표시 항목**:
- 월별 입금/출금 금액 바 차트 (최근 12개월)
- 월별 잔액 추이 라인 차트
- 기간별 통계 요약 (선택 기간의 총 입금, 총 출금, 순이익)
- 주요 거래처 Top 5 (출금 기준)

**필터**: 기간 선택 (시작월 ~ 종료월)

### 3.2 F2: 월별 입금/출금 통계

계좌별 통계의 핵심 데이터로, F1 내부에 포함.

**백엔드 집계 로직**:
```
SELECT
  DATE_TRUNC('month', txn_date) AS month,
  SUM(CASE WHEN txn_balance > 0 THEN txn_balance ELSE 0 END) AS deposit,
  SUM(CASE WHEN txn_balance < 0 THEN ABS(txn_balance) ELSE 0 END) AS withdrawal
FROM amb_transactions
WHERE bac_id = :accountId AND txn_deleted_at IS NULL
GROUP BY month
ORDER BY month
```

### 3.3 F3: 계좌 통합 통계 페이지

**경로**: `/accounting/dashboard`

**기능**:
- 체크박스로 통합할 계좌 선택 (같은 통화끼리만 통합 가능)
- 선택된 계좌들의 합산 월별 입금/출금 차트
- 통화별 총 잔액 파이 차트
- 전체 계좌 잔액 히트맵 또는 테이블

**제약**: 통화가 다른 계좌는 별도 그룹으로 표시 (환율 변환 없음)

### 3.4 F4: 정기 지출 항목 관리

**새 엔티티**: `RecurringExpenseEntity` (amb_recurring_expenses)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| rex_id | UUID PK | 정기지출 ID |
| ent_id | UUID FK | 법인 ID |
| bac_id | UUID FK | 출금 계좌 ID |
| rex_name | varchar(200) | 지출 항목명 (예: 사무실 임대료) |
| rex_vendor | varchar(300) | 거래처명 |
| rex_amount | decimal(18,2) | 예상 금액 |
| rex_currency | varchar(10) | 통화 |
| rex_day_of_month | int | 매월 지출일 (1~31) |
| rex_category | varchar(100) | 분류 (임대료, 급여, 보험, 통신비 등) |
| rex_description | text | 비고 |
| rex_is_active | boolean | 활성 여부 |
| rex_start_date | date | 시작일 |
| rex_end_date | date | 종료일 (NULL=무기한) |
| rex_created_at | timestamp | 생성일 |
| rex_updated_at | timestamp | 수정일 |
| rex_deleted_at | timestamp | 삭제일 (Soft Delete) |

**CRUD API**:
- `GET /accounts/recurring-expenses` — 정기 지출 목록
- `POST /accounts/recurring-expenses` — 정기 지출 등록
- `PATCH /accounts/recurring-expenses/:id` — 수정
- `DELETE /accounts/recurring-expenses/:id` — 삭제

### 3.5 F5: 정기 지출 예상 문서

**기능**: 특정 월의 예상 지출 문서를 생성

**문서 내용**:
- 대상 월 (예: 2026년 4월)
- 정기 지출 항목 목록 (항목명, 거래처, 예상금액, 출금계좌, 예정일)
- 통화별 합계
- 전월 실제 지출 대비 비교 (있는 경우)

**API**: `GET /accounts/recurring-expenses/forecast?month=2026-04`

**출력 형태**:
- 화면 내 테이블 (기본)
- PDF/엑셀 다운로드 (선택, 추후 확장 가능)

## 4. 사용자 플로우

### 4.1 계좌별 통계 조회
1. `/accounting` 에서 계좌 카드 클릭
2. 거래 내역 페이지에서 "통계" 탭 선택
3. 월별 입출금 차트, 잔액 추이, 주요 거래처 확인
4. 기간 필터로 조회 범위 변경

### 4.2 통합 통계 조회
1. `/accounting` 상단에 "대시보드" 버튼/탭 클릭
2. `/accounting/dashboard` 이동
3. 계좌 체크박스로 통합 대상 선택
4. 통화별 그룹화된 통합 차트 확인

### 4.3 정기 지출 관리
1. `/accounting/dashboard` 내 "정기 지출" 섹션
2. "정기 지출 추가" 버튼 → 모달 폼
3. 항목명, 거래처, 금액, 출금계좌, 매월 지출일, 분류 입력
4. 등록된 항목 목록에서 수정/삭제/활성 토글

### 4.4 정기 지출 예상 문서
1. "정기 지출" 섹션에서 "예상 문서 생성" 버튼 클릭
2. 대상 월 선택 (기본: 다음 달)
3. 예상 지출 테이블 표시 (항목별 금액, 통화별 합계)
4. 필요 시 PDF/엑셀 다운로드

## 5. 기술 제약사항 및 고려사항

### 5.1 차트 라이브러리 선택

**recharts 권장** (React 네이티브, TailwindCSS 호환)
- 바 차트: 월별 입출금
- 라인 차트: 잔액 추이
- 파이 차트: 통화별 비중

### 5.2 성능 고려

- 월별 집계 쿼리는 대량 데이터 시 느릴 수 있음 → DB 인덱스 활용 (`IDX_txn_bac_date` 기존 인덱스 활용 가능)
- 통합 통계는 여러 계좌 조회 필요 → `WHERE bac_id IN (...)` 단일 쿼리로 처리
- 대시보드 데이터는 `staleTime: 5분` 설정으로 불필요한 재요청 방지

### 5.3 다중 통화 처리

- VND, USD, KRW 통화별로 분리 표시 (환율 변환 없음)
- 통합 차트에서 같은 통화끼리만 합산 가능
- 금액 포맷: 기존 `BalanceSummaryCard`의 `formatCurrency()` 재사용

### 5.4 스테이징 환경 주의

- `NODE_ENV=production`이므로 새 테이블(`amb_recurring_expenses`)은 수동 DDL 필요
- 기존 `amb_transactions` 테이블에는 변경 없음

### 5.5 i18n

- `accounting.json`에 대시보드 관련 키 추가 (ko/en/vi)
- 차트 레이블, 필터 옵션, 정기 지출 폼 필드 등

## 7. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 대량 거래 데이터 집계 성능 | 느린 차트 로딩 | 기존 인덱스 활용 + 필요 시 materialized view |
| 차트 라이브러리 번들 크기 | 초기 로드 시간 증가 | recharts lazy import (동적 로딩) |
| 다중 통화 통합 시 혼동 | 사용자 오해 | 통화별 그룹 분리 표시, 합산 불가 안내 |
| 정기 지출 vs 실제 지출 괴리 | 예상 문서 정확도 | 전월 실제 지출 대비 표시 |

## 1. 요구사항 요약

Entity Settings에 **사이트 설정(Site Configure)** 카드뷰 메뉴를 추가하고, 2가지 하위 기능을 구현한다:

| # | 기능 | 설명 |
|---|------|------|
| A | 로그인 모달 안내창 | 사용자 로그인 시 모달 안내창 노출/미노출 설정 + 안내 콘텐츠 관리 |
| B | 사용자 팁 관리 | 메뉴별 팁/설명 입력 및 관리 (각 페이지에서 툴팁 또는 배너로 표시) |

---

## 2. AS-IS 현황 분석

### 2.1 Entity Settings 페이지 현황
- `EntitySettingsPage.tsx`: 11개 카드 그리드 (Organization, Members, Clients, Permissions, Work Statistics, Activity Index, Attendance, External Connect, Usage, Custom Apps, Email Templates)
- 카드는 `ENTITY_MENU_ITEMS` 배열 기반, `useMyMenus()` hook으로 접근 가능 항목 필터링
- **사이트 설정 카드 없음**

### 2.2 로그인 후 플로우
- `LoginPage.tsx` → `authService.login()` → `auth.store.login(user)` → 라우트 리다이렉트
- `MainLayout.tsx`: `<AIAssistantModal />`, `<NotificationModal />`이 고정 레이어에 있음
- **로그인 직후 안내 모달 없음**

### 2.3 Entity 설정 저장
- `amb_hr_entities` 테이블: 기본 정보/급여/근무 설정 컬럼 보유
- **사이트 구성(모달 설정, 팁 데이터) 저장 테이블 없음**

### 2.4 팁/가이드 시스템
- i18n 힌트텍스트만 존재 (`t('*.Hint')`)
- **Entity별 커스텀 팁/설명 저장 기능 없음**

## 3. TO-BE 요구사항

### 3.1 Entity Settings 카드 추가
- **Site Configure** 카드를 Entity Settings 그리드에 추가
- 메뉴코드: `ENTITY_SITE_CONFIG`
- 아이콘: `Settings2` (lucide)

### 3.2 사이트 설정 페이지 — 2개 섹션

#### A. 로그인 모달 안내창 설정
| 항목 | 설명 |
|------|------|
| 토글 | 모달 노출 ON/OFF |
| 제목 | 모달 제목 텍스트 |
| 내용 | 리치텍스트 또는 마크다운 안내 내용 |
| 미리보기 | 설정 저장 전 미리보기 |

#### B. 메뉴별 사용자 팁
| 항목 | 설명 |
|------|------|
| 메뉴 목록 | Entity가 접근 가능한 메뉴 리스트 |
| 팁 제목 | 각 메뉴에 대한 팁 제목 |
| 팁 내용 | 각 메뉴에 대한 설명/팁 텍스트 |
| 노출 여부 | 각 팁의 표시/숨김 토글 |

### 3.3 로그인 모달 노출 동작
- `MainLayout.tsx`에서 인증 후 최초 마운트 시 모달 표시
- **하루 한 번만** 노출 (localStorage 기반 날짜 체크)
- 사용자가 "오늘 다시 보지 않기" 가능

### 3.4 메뉴별 팁 노출 위치
- 각 페이지 상단에 접을 수 있는 배너(collapsible) 형태
- 팁이 설정된 메뉴에서만 표시

## 5. 사용자 플로우

### 5.1 관리자 — 사이트 설정
```
Entity Settings → Site Configure 카드 클릭
  → 사이트 설정 페이지
    ├── [로그인 모달 설정] 섹션
    │   ├── 모달 노출 ON/OFF 토글
    │   ├── 제목 입력
    │   ├── 내용 입력 (텍스트에어리어)
    │   ├── [미리보기] 버튼
    │   └── [저장] 버튼
    └── [메뉴별 팁 관리] 섹션
        ├── 메뉴 목록 (테이블)
        │   ├── 메뉴명 | 팁 제목 | 노출 | 편집
        │   └── ...
        └── 팁 편집 모달
            ├── 팁 제목 입력
            ├── 팁 내용 입력
            ├── 노출 토글
            └── [저장] 버튼
```

### 5.2 사용자 — 로그인 모달 경험
```
로그인 → MainLayout 마운트
  → API에서 entity site config 조회
  → 모달 활성화 && 오늘 미표시 아님
    → 모달 표시 (제목 + 내용)
    → [확인] / [오늘 다시 보지 않기] 버튼
```

### 5.3 사용자 — 메뉴별 팁 경험
```
메뉴 페이지 진입
  → 해당 메뉴에 팁 설정값이 있고 노출=ON
    → 페이지 상단에 접을 수 있는 안내 배너 표시
    → [접기/펼치기] 토글
```

## 6. 기술 제약사항

| 항목 | 제약 |
|------|------|
| DB 마이그레이션 | 스테이징/프로덕션은 수동 SQL 필수 (TypeORM synchronize OFF) |
| 데이터 격리 | `ent_id` 기반 필수 — `OwnEntityGuard` + `resolveEntityId()` |
| i18n | 고정 UI 텍스트는 ko/en/vi 번역 필수, 사용자 입력 데이터는 원문 저장 |
| 권한 | `ENTITY_SITE_CONFIG` 메뉴코드 등록 필요 |
| 모달 빈도 | localStorage 기반 하루 1회 제한 — 서버 부하 최소화 |

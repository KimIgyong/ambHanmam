# REQ-커스텀앱-앱스토어-메뉴분리-20260330

## 1. 요구사항 요약 (Requirement Summary)

**제목**: 커스텀앱과 앱스토어 메뉴 완전 분리  
**작성일**: 2026-03-30  
**참조 문서**: `docs/report/RPT-커스텀앱-앱스토어-현황분석-20260330.md`

**배경**:
- **커스텀앱**: Entity(법인) 사용자가 직접 등록하는 외부 서비스 관리 도구
- **앱스토어**: 아메바 회사와 파트너사가 만든 공식 앱 플랫폼
- 두 개념이 하나의 탭 페이지(`EntityCustomAppsTabPage`)에 혼재 → 사용자 혼란, 데이터 경계 모호
- "앱스토어에서 가져오기(Import)" 기능이 앱스토어앱을 커스텀앱 테이블에 복제하여 개념적 분리 위반

**목표**:
1. 커스텀앱 페이지를 순수한 사용자 등록 외부 서비스 관리 전용으로 독립
2. 앱스토어 구독 관리 전용 페이지를 신설하여 Subscription API 연동
3. 라우팅/메뉴 체계 정리 및 중복 제거

## 2. AS-IS 현황 분석

### 2.1 페이지/라우팅 현황

| 라우트 | 컴포넌트 | 가드 | 용도 | 문제 |
|--------|---------|------|------|------|
| `/entity-settings/custom-apps` | `EntityCustomAppsTabPage` | EntitySettingsGuard + MenuGuard | 3탭 통합 (커스텀/공인앱스토어/테스트앱스토어) | ⚠️ 커스텀앱과 앱스토어가 하나의 페이지에 혼재 |
| `/entity-settings/apps` | `EntityAppsPage` | EntitySettingsGuard | OAuth 기반 파트너앱 설치/스코프 관리 | 역할 불명확 (앱스토어와 중복?) |
| `/app-store` | `AppStorePage` | 없음 | 단독 앱스토어 iframe (프로덕션) | 라우트 접근 경로 부재 (사이드바 직링크 없음) |

| `/menu/custom-apps` | `MenuCategoryPage` | 없음 | 사이드바 "Custom Apps" 클릭 → 카드뷰 | 정상 |
| `/apps/:appCode` | `CustomAppHostPage` | 없음 | 개별 앱 iframe 호스트 실행 | 정상 |

### 2.2 EntityCustomAppsTabPage 구조 (현행)

```
EntityCustomAppsTabPage (3탭)
├── [탭: 커스텀앱]        → EntityCustomAppsPage (CRUD)
│   ├── "앱 추가" 버튼
│   ├── "앱스토어에서 가져오기" 버튼 ← ⚠️ 도메인 혼재
│   ├── ImportFromStoreModal      ← ⚠️ 파트너앱을 커스텀앱 테이블에 복제
│   └── 앱 카드 그리드 (편집/삭제/연결테스트)
├── [탭: 공인앱스토어]    → iframe (apps.amoeba.site)
└── [탭: 테스트앱스토어]  → iframe (stg-apps.amoeba.site)
```

### 2.3 백엔드 API 현황

| 엔드포인트 | 용도 | 데이터 소스 | 문제 |
|-----------|------|-----------|------|
| `GET /entity-settings/custom-apps` | 관리자 앱 목록 | `amb_entity_custom_apps` | ✅ |
| `GET /entity-settings/custom-apps/my` | 사이드바용 앱 목록 | `amb_entity_custom_apps` | ✅ |
| `POST /entity-settings/custom-apps` | 앱 등록 | `amb_entity_custom_apps` | ✅ |
| `PATCH /entity-settings/custom-apps/:id` | 앱 수정 | `amb_entity_custom_apps` | ✅ |
| `DELETE /entity-settings/custom-apps/:id` | 앱 삭제 | `amb_entity_custom_apps` | ✅ |
| `POST /entity-settings/custom-apps/:id/token` | 토큰 발급 | `amb_entity_custom_apps` | ✅ |
| `POST /entity-settings/custom-apps/:id/health` | 연결 테스트 | HTTP fetch | ✅ |
| `GET /entity-settings/custom-apps/importable` | 구독앱 목록 | `amb_partner_app_installs` (내부DB) | ⚠️ 제거 대상 |
| `POST /entity-settings/custom-apps/import/:papId` | 앱스토어앱 → 커스텀앱 복제 | 내부DB → `amb_entity_custom_apps` | ⚠️ 제거 대상 |

### 2.4 데이터베이스 현황

| 테이블 | 용도 | Entity 격리 |
|--------|------|------------|
| `amb_entity_custom_apps` | 커스텀앱 + import된 앱스토어앱 | ✅ `ent_id` |
| `amb_partner_apps` | 파트너앱 마스터 | ❌ 전체 공유 |
| `amb_partner_app_installs` | 법인별 앱 구독/설치 | ✅ `pai_entity_id` |

`amb_entity_custom_apps` 핵심 컬럼:
- `eca_source_pap_id` (UUID, nullable): 앱스토어 import 추적 → 분리 후 불필요

### 2.5 사이드바 현황

```
MainLayout.tsx
├── [일반 메뉴 항목들]
├── [Custom Apps 섹션] ← useMyCustomApps() 훅으로 동적 렌더링
│   ├── 섹션 헤더 클릭 → /menu/custom-apps (카드뷰)
│   └── 앱 항목 클릭 → /apps/{code} (앱 실행)
└── [App Store 섹션] ← ❌ 없음 (앱스토어 앱은 사이드바에 미표시)
```

### 2.6 외부 API 현황

**AppStore Subscription API** (`docs/AppStore-Subscription-API-for-AMA.md`):
- `GET /api/v1/platform/subscriptions/entity/{ent_id}`
- 인증 불필요 (Public), UUID 기반
- 응답: Entity의 전체 앱 목록 + 구독 상태 (`PENDING`/`ACTIVE`/`SUSPENDED`)
- 현재 **미활용** 상태

### 3.4 TO-BE 백엔드 API

**유지**:
- `GET /entity-settings/custom-apps` — 커스텀앱 관리 목록

## 6. 기술 제약사항

### 6.1 외부 Subscription API 의존성

| 제약사항 | 영향 | 대응 방안 |
|---------|------|----------|
| API 다운타임 | 앱스토어 관리 페이지 조회 실패 | try-catch + 에러 메시지 표시, 재시도 버튼 |
| CORS 제한 | 프론트엔드 직접 호출 불가 | **백엔드 프록시** 방식으로 구현 |
| 네트워크 지연 | 느린 응답 UX | 10초 timeout, 로딩 UI, 캐싱(staleTime 1분) |
| API 응답 구조 변경 | 매핑 오류 | 방어적 파싱 + 타입 검증 |

### 6.2 기존 Import 데이터 처리

| 제약사항 | 설명 | 대응 방안 |
|---------|------|----------|
| `eca_source_pap_id` 값이 있는 기존 앱 | Import로 생성된 커스텀앱이 DB에 존재 | 기존 데이터 유지, 커스텀앱으로 계속 취급 (삭제 안 함) |
| `PartnerAppEntity`/`PartnerAppInstallEntity` | 다른 서비스에서 참조 가능 | import 제거만 하고 Entity는 유지 |
| `eca_source_pap_id` 컬럼 | 테이블에 남아있음 | 삭제하지 않음 (하위 호환) |

### 6.3 배포 제약

| 제약사항 | 설명 |
|---------|------|
| 프론트/백 동시 배포 | API 제거 + UI 변경 동시 반영 필요 |
| 환경변수 추가 | `APP_STORE_API_URL` 스테이징/프로덕션에 추가 필요 |
| DB 마이그레이션 | 없음 (테이블 구조 변경 없음, `eca_source_app_slug` 추가는 보류) |

### 6.4 보안

| 항목 | 설명 |
|------|------|
| Subscription API (Public) | `ent_id`(UUID)로만 조회, 민감 데이터 미포함 → 리스크 낮음 |
| 백엔드 프록시 | AMA 인증 사용자만 호출 가능 (`@Auth()` + `OwnEntityGuard`) |
| Entity 격리 | `resolveEntityId()` 기반, 타 법인 구독 정보 열람 불가 |

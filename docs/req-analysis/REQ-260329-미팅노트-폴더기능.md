## 요구사항 분석서

- **프로젝트**: AMA Management
- **작성일**: 2026-03-29
- **대상 URL**: https://stg-ama.amoeba.site/meeting-notes

---

## 1. AS-IS 현황 분석

### 1.1 데이터 구조

**테이블**: `amb_meeting_notes`
- PK: `mtn_id` (UUID)
- 격리: `ent_id` (법인), `usr_id` (작성자)
- 타입: `mtn_type` (MEETING_NOTE | MEMO)
- 공개: `mtn_visibility` (PRIVATE | UNIT | ENTITY | CELL | PUBLIC)
- **폴더 개념 없음** — `folder_id`, `category_id`, `parent_id` 등 미존재

**연관 테이블**: participants, comments, projects, issues, ratings (5개)

### 1.2 현재 목록 페이지 UI

- **완전한 flat list** — 폴더/카테고리 없이 전체 노트를 날짜순 나열
- 뷰 모드: 카드 뷰 / 리스트 뷰 (토글)
- 필터: 타입(전체/회의록/메모), 범위(mine/unit/cell/all), 텍스트 검색
- 페이지네이션: 20건/페이지

### 1.3 기존 API

| 엔드포인트 | 설명 |
|-----------|------|
| `GET /meeting-notes` | 목록 (visibility, type, scope, search, page, size) |
| `POST /meeting-notes` | 생성 |
| `GET /meeting-notes/:id` | 상세 |
| `PATCH /meeting-notes/:id` | 수정 |
| `DELETE /meeting-notes/:id` | 삭제 (soft) |

### 1.4 기존 CreateMeetingNoteRequest DTO

```
type, title, content, meeting_date, visibility, unit, cell_id,
assignee_id, participant_ids, project_ids, issue_ids
```
— `folder_id` 필드 없음

## 2. TO-BE 요구사항

### 2.1 폴더 기능 명세

법인(`ent_id`) 단위로 미팅노트 폴더를 관리하고, 노트를 폴더에 분류할 수 있다.

### 2.2 폴더 데이터 모델

**새 테이블**: `amb_meeting_note_folders`

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `mnf_id` | UUID PK | 폴더 ID |
| `ent_id` | UUID NOT NULL | 법인 격리 |
| `usr_id` | UUID NOT NULL | 생성자 |
| `mnf_name` | VARCHAR(100) | 폴더 이름 |
| `mnf_color` | VARCHAR(7) NULL | 아이콘 색상 (#hex) |
| `mnf_sort_order` | INT DEFAULT 0 | 정렬 순서 |
| `mnf_visibility` | VARCHAR(20) DEFAULT 'PRIVATE' | 마스터/팀 공유 폴더 |
| `mnf_created_at` | TIMESTAMPTZ | 생성일 |
| `mnf_updated_at` | TIMESTAMPTZ | 수정일 |
| `mnf_deleted_at` | TIMESTAMPTZ NULL | Soft Delete |

**기존 테이블 변경**: `amb_meeting_notes`에 `mtn_folder_id UUID NULL` 컬럼 추가

### 2.3 폴더 API

| 엔드포인트 | Method | 설명 |
|-----------|--------|------|
| `GET /meeting-notes/folders` | GET | 폴더 목록 (법인별) |
| `POST /meeting-notes/folders` | POST | 폴더 생성 |
| `PATCH /meeting-notes/folders/:id` | PATCH | 폴더 수정 (이름, 색상, 순서) |
| `DELETE /meeting-notes/folders/:id` | DELETE | 폴더 삭제 (노트는 미분류로 이동) |
| `PATCH /meeting-notes/:id` | PATCH | 노트 수정 시 `folder_id` 변경 가능 |

### 2.4 UI 변경

```
┌───────────────────────────────────────────────────────┐
│  📂 노트                                      [+ 새 노트]  │
│─────────────────────────────────────────────────────────│
│  [Scope: mine | unit | cell | all]                   │
│  [전체] [회의록] [메모]     🔍 검색...                │
│─────────────────────────────────────────────────────────│
│                                                       │
│  📁 전체 (미분류)                              (접기 ▼) │
│  ├── 회의록1 / 홍길동 / 2026-03-28                     │
│  ├── 메모2 / 김영희 / 2026-03-27                       │
│  └── ...                                             │
│                                                       │
│  📁 마케팅 전략 (3)                       ⋯  (접기 ▼) │
│  ├── 마케팅 회의 / 홍길동 / 2026-03-25                 │
│  └── ...                                             │
│                                                       │
│  📁 프로젝트 A (5)                        ⋯  (접기 ▼) │
│  ├── ...                                             │
│                                                       │
│  [+ 새 폴더]                                         │
│                                                       │
│  [<<] 1/3 [>>]                                       │
└───────────────────────────────────────────────────────┘
```

**주요 UI 동작**:
1. 사이드바 또는 인라인 방식으로 폴더 목록 표시
2. 각 폴더는 **접기/펼치기** 가능 (localStorage 상태 기억)
3. "전체(미분류)" 섹션 = `folder_id IS NULL`인 노트
4. 폴더명 옆 **⋯ 메뉴**: 이름 변경, 색상 변경, 삭제
5. 폴더 클릭 시 해당 폴더 노트만 필터링
6. 노트 생성/수정 폼에 **폴더 선택** 드롭다운 추가
7. 드래그앤드롭 없이, 기존 필터 기반으로 **폴더 필터** 추가

### 2.5 사용자 액션

| 액션 | 동작 |
|------|------|
| 폴더 생성 | "새 폴더" 클릭 → 인라인 이름 입력 → 저장 |
| 폴더 수정 | ⋯ 메뉴 → 이름/색상 변경 |
| 폴더 삭제 | ⋯ 메뉴 → 삭제 확인 → 소속 노트는 미분류로 이동 |
| 노트를 폴더에 배치 | 노트 생성/수정 폼에서 폴더 선택 |
| 폴더 필터링 | 폴더 클릭 → 해당 폴더 노트만 표시 |

## 4. 기술 제약사항

- **DB**: 스테이징/프로덕션 수동 SQL 마이그레이션 필수
- **보안**: 폴더도 `ent_id` 기반 격리 필수. EntityGuard 적용
- **호환성**: 기존 노트(`folder_id NULL`)는 "미분류"로 자연 표시
- **성능**: 폴더는 법인당 수십 개 수준 → 페이지네이션 불필요
- **i18n**: 폴더 관련 번역 키 추가 (ko/en/vi)

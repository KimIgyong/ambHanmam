# REQ-구성원리스트정렬필터-20260329
# Member List Sorting & Filter Enhancement

## 1. 요구사항 요약 (Requirement Summary)

구성원 관리 페이지의 구성원/클라이언트 리스트에 정렬 및 필터 기능을 추가하고, 테이블 레이아웃을 개선한다.

| 항목 | 내용 |
|------|------|
| 기본 정렬 | 최신 가입일순 (desc), 정렬 순서 선택 가능 |
| 기본 상태 필터 | ACTIVE만 표시 (INACTIVE 선택 가능) |
| 역할 필터 | 다중 선택 가능 (checkbox) |
| 유닛 필터 | 단일 선택 (dropdown) |
| 상태 필터 | 다중 선택 가능 (checkbox) |
| 레이아웃 변경 | 이름+이메일을 하나의 셀로 병합 (이름 상단, 이메일 하단) |

## 2. AS-IS 현황 분석 (Current State)

### 2.1 MemberList 컴포넌트 (`EntityMemberPage.tsx`)

**현재 테이블 컬럼 구조:**
| # | 컬럼 | 내용 |
|---|------|------|
| 1 | Email | `m.email` |
| 2 | Name | `m.name` |
| 3 | Role | 역할 배지 + OWNER 표시 |
| 4 | Connected Client | (isClientView일 때만) `m.clientInfo.clientName` |
| 5 | Department/Unit | `unitRoles?.find(isPrimary)?.unitName \|\| m.unit` |
| 6 | Status | 상태 배지 |
| 7 | Joined At | `m.createdAt` (LocalDateTime) |

**현재 정렬:** 없음 (서버 응답 순서 그대로)
**현재 필터:** 없음 (`members` prop을 그대로 렌더링)
**현재 기본 상태:** 모든 상태(ACTIVE, INACTIVE, SUSPENDED)가 표시됨

### 2.2 데이터 구조

```typescript
// EntityMember (entity-settings.service.ts)
interface EntityMember {
  userId: string;
  email: string;
  name: string;
  role: string;       // MASTER | MANAGER | MEMBER | VIEWER | CLIENT_MANAGER | CLIENT_MEMBER
  unit: string;
  levelCode: string;  // USER_LEVEL | CLIENT_LEVEL
  status: string;     // ACTIVE | INACTIVE | SUSPENDED
  joinMethod: string;
  createdAt: string;
  unitRoles: MemberUnitRole[];
  cells: MemberCellInfo[];
  hrEmployee: MemberHrEmployee | null;
  clientInfo: MemberClientInfo | null;
}
```

### 2.3 유닛 데이터
- `useOrgUnits()` 훅으로 조직 유닛 목록 호출 가능
- `OrgUnit { unitId, name, nameLocal, parentId, level, isActive }`
- 이미 MemberDetailModal에서 사용 중

### 2.4 역할/상태 상수
```typescript
const USER_ROLES = ['MASTER', 'MANAGER', 'MEMBER', 'VIEWER'];
const CLIENT_ROLES = ['CLIENT_MANAGER', 'CLIENT_MEMBER'];
const MEMBER_STATUSES = ['ACTIVE', 'INACTIVE', 'SUSPENDED'];
```

## 3. TO-BE 요구사항 (Target State)

### 3.1 필터 바 (Filter Bar)
MemberList 테이블 상단에 필터 바 추가:

```
[정렬: 최신가입일순 ▼] [역할: 전체 ▼] [유닛: 전체 ▼] [상태: ● ACTIVE ▼]
```

| 필터 | 타입 | 기본값 | 동작 |
|------|------|--------|------|
| 정렬 | 단일 선택 dropdown | `joinedAt_desc` (최신가입일순) | `joinedAt_asc`, `joinedAt_desc`, `name_asc`, `name_desc` |
| 역할 | 다중 선택 checkbox dropdown | 전체 (빈 배열 = 필터 없음) | USER_ROLES 또는 CLIENT_ROLES (탭에 따라) |
| 유닛 | 단일 선택 dropdown | 전체 (`''`) | `useOrgUnits()` 목록에서 선택 |
| 상태 | 다중 선택 checkbox dropdown | `['ACTIVE']` | MEMBER_STATUSES에서 다중 선택 |

### 3.2 테이블 레이아웃 변경
**변경 후 컬럼 구조:**
| # | 컬럼 | 내용 |
|---|------|------|
| 1 | Name / Email | 이름 (굵게) + 이메일 (회색 작은 글씨) 세로 배치 |
| 2 | Role | 역할 배지 + OWNER 표시 |
| 3 | Connected Client | (isClientView일 때만) |
| 4 | Department/Unit | 유닛명 |
| 5 | Status | 상태 배지 |
| 6 | Joined At | 가입일 |

### 3.3 기본 동작
- 페이지 진입 시 `status=['ACTIVE']` 필터 + `joinedAt_desc` 정렬 적용
- 필터/정렬 변경 시 클라이언트 사이드에서 즉시 반영 (서버 재호출 불필요)
- `members` 탭과 `clients` 탭 각각 독립적 필터 상태 비유지 (탭 전환 시 초기화)

## 5. 사용자 플로우 (User Flow)

1. 구성원 관리 페이지 진입
2. `구성원` 탭 활성 → MemberList에 ACTIVE 구성원만 최신가입일순으로 표시
3. 필터 바에서:
   - 정렬 드롭다운으로 정렬 순서 변경 가능
   - 역할 드롭다운 클릭 → 체크박스로 복수 역할 선택/해제
   - 유닛 드롭다운에서 특정 유닛 선택
   - 상태 드롭다운 클릭 → 체크박스로 복수 상태 선택/해제 (INACTIVE 추가 등)
4. 필터 조합은 AND 조건 (역할 AND 유닛 AND 상태)
5. 필터된 결과에서 구성원 행 클릭 → 기존 MemberDetailModal 표시

## 6. 기술 제약사항 (Technical Constraints)

- **클라이언트 사이드 처리**: 현재 멤버 수가 수백 명 수준이므로 서버 사이드 pagination이 불필요. `useMemo`로 필터/정렬 처리.
- **유닛 데이터**: `useOrgUnits()` 이미 사용 가능하나 MemberList에서 호출 필요.
- **i18n**: 필터 레이블, 정렬 옵션 등 새 번역 키 필요 (ko/en/vi).
- **탭별 역할**: members 탭은 `USER_ROLES`, clients 탭은 `CLIENT_ROLES` 사용.
- **성능**: `useMemo`로 필터/정렬 결과를 메모이제이션하여 불필요한 재계산 방지.

## 1. AS-IS 현황 분석

### 1.1 로그인 플로우

```
LoginPage → POST /auth/login → JWT 발급
    ↓
EntitySelectPage → 법인 선택 (1개면 자동)
    ↓
navigate('/') → TodayPage (일반) 또는 navigate('/admin') (관리자)
```

- **현재 로그인 후 모달/팝업 없음** — 곧바로 TodayPage로 진입
- Entity 선택 후 `setCurrentEntity(entity) → navigate(fromPath)` 흐름

### 1.2 사용자 역할 체계

| 속성 | 값 | 설명 |
|------|------|------|
| `user.role` | `MASTER`, `ADMIN`, `MANAGER`, `MEMBER` | 법인 내 역할 |
| `user.level` | `ADMIN_LEVEL`, `USER_LEVEL`, `PARTNER_LEVEL`, `CLIENT_LEVEL` | 시스템 레벨 |
| Auth Store 헬퍼 | `isMaster()`, `isAdmin()`, `isMasterOrAdmin()` | 역할 확인 메서드 |

**이 모달의 대상**: `user.role === 'MASTER'` (Entity MASTER 사용자)

### 1.3 기존 AI 사용량 시스템

#### DB 테이블

| 테이블 | 용도 |
|--------|------|
| `amb_ai_token_usage` | 개별 AI 요청 기록 (usr_id, ent_id, source_type, tokens) |
| `amb_ai_token_entity_summary` | 법인별 일일 집계 (ent_id, date, total_tokens) |
| `amb_entity_api_quotas` | 법인별 쿼터 설정 (daily_limit, monthly_limit, action_on_exceed) |
| `amb_login_histories` | 로그인 이력 (usr_id, ent_id, ip, user_agent) |
| `amb_users` | 사용자 (usr_last_login_at 필드) |

#### 기존 API 엔드포인트

| 엔드포인트 | 설명 | 응답 |
|-----------|------|------|
| `GET /entity-settings/usage` | 일일+월간 사용량 종합 | daily, monthly, quota, warnings (quotaStage) |
| `GET /entity-settings/usage/monthly` | 월간 사용량 | yearMonth, totalTokens |
| `GET /entity-settings/work-statistics/member-activities` | 멤버별 활동 | userId별 loginCount, aiRequests, aiTokens 등 |
| `GET /entity-settings/work-statistics/login-history` | 로그인 이력 | userId, name, ip, createdAt |
| `GET /entity-settings/quota` | 쿼터 설정 조회 | dailyLimit, monthlyLimit |
| `GET /entity-settings/members` | 멤버 목록 | EntityMember[] (status 포함) |

#### 쿼터 경고 단계

| 단계 | 조건 | 설명 |
|------|------|------|
| `NORMAL` | 잔여 > 10% | 정상 |
| `CAUTION` | 잔여 ≤ 10% | 주의 |
| `WARNING` | 잔여 ≤ 5% | 경고 |
| `SUSPENDED` | 잔여 0% | 초과 차단 |

### 1.4 기존 UI 패턴

- **EntityUsagePage**: 일일/월간 사용량 바 차트, 쿼터 상태 표시, 토큰 충전 모달 (1M/5M/10M/50M 패키지)
- **TodayPage**: AnalysisModal, ReportsModal (버튼 클릭 시 열림)
- **모달 패턴**: `useState` + 조건부 렌더링, TailwindCSS 오버레이 (`bg-black/50`)

## 4. 사용자 플로우

### 4.1 시퀀스

```
사용자 로그인
    ↓
EntitySelectPage → 법인 선택
    ↓
navigate('/') → TodayPage 마운트
    ↓
[조건 체크]
├─ user.role !== 'MASTER' → 모달 안 보임
├─ localStorage에 오늘 날짜 키가 있음 → 모달 안 보임
└─ MASTER + 오늘 안 봄 → 모달 표시
    ↓
[API 호출 (병렬)]
├─ GET /entity-settings/members → 총 직원 수
├─ GET /entity-settings/usage → AI 사용량 + 쿼터
└─ GET /entity-settings/work-statistics/member-activities?start_date=7일전 → 활성 사용자
    ↓
모달 렌더링
    ↓
[사용자 액션]
├─ × 닫기 → 모달 닫힘
├─ 오늘 안 보기 → localStorage 저장 + 모달 닫힘
├─ 토큰 충전하기 → /entity-settings/usage 이동
└─ 상세 보기 → /entity-settings/usage 이동
```

### 4.2 모달 표시 빈도 로직

```typescript
const STORAGE_KEY = 'amb-usage-modal-dismissed';

// 체크: 오늘 이미 닫았는지
const dismissed = localStorage.getItem(STORAGE_KEY);
const today = new Date().toISOString().slice(0, 10); // 'YYYY-MM-DD'
const shouldShow = !dismissed || dismissed !== today;

// 닫기: "오늘 안 보기" 클릭 시
localStorage.setItem(STORAGE_KEY, today);
```

## 5. 기술 제약사항

### 5.1 성능

- 모달은 **TodayPage 마운트 후** 비동기로 데이터를 로드 → TodayPage 자체 로딩 차단 안 함
- 3개 API 병렬 호출 → 가장 느린 것 기준 ~200-500ms
- 모달 로딩 중 스피너 표시

### 5.2 호환성

- **기존 TodayPage 로직 영향 없음** — 모달은 독립 컴포넌트로 조건부 렌더링
- **기존 EntityUsagePage와 데이터 소스 동일** — 동일 API 호출, 캐시 공유 가능

### 5.3 보안

- MASTER 역할 검증은 클라이언트 사이드 (`useAuthStore.isMaster()`)
- API는 기존 `@Auth()` + `OwnEntityGuard`로 보호됨 → entity 격리 자동 적용
- 타 법인 데이터 노출 위험 없음 (API 자체가 entityId 기반 필터)

### 5.4 i18n

- 모달 텍스트는 번역 키 사용 필수 (ko/en/vi)
- 기존 `entitySettings` 네임스페이스에 키 추가 또는 별도 네임스페이스

## 1. AS-IS 현황 분석

### 1.1 현재 시스템 구조

현재 AMB Management에는 **문서 자동 생성 모듈과 데이터 기반 의사결정 대시보드가 모두 존재하지 않으며**, 회사소개서·서비스소개서·제품소개서·IR자료는 Google Drive에 산재되어 수동 관리 중이고, 경영 KPI는 체계적으로 수집/분석되지 않고 있다.


### 1.3 5A Matrix 원본 분석 (Cafe24 VN 2021/3Q)

5A Matrix는 **Philip Kotler의 5A 마케팅 프레임워크**를 SaaS 커머스 운영에 맞게 재해석한 아메바 고유의 의사결정 체계이다. 원본은 김익용 대표의 Cafe24 Vietnam 법인장 재임 시절(2021)에 작성되었다.

| 원본 구성요소 | 원본 적용 대상 | amoeba 2.0 일반화 |
|-------------|-------------|-------------------|
| A1: Advertise (광고/노출) | 광고비, CPV, 미디어 채널 | **Lead Generation** — 리드 유입 비용, 채널별 효과 |
| A2: Acquisition (가입/전환) | 회원가입 전환율 | **Client Acquisition** — 문의→견적→계약 전환율 |
| A3: Activation (활성화) | 첫 상품등록, 첫 주문 | **Service Activation** — 킥오프, 첫 딜리버리, SLA 충족 |
| A4: Accelerate (성장/가속) | 유효몰 수, GMV | **Revenue Growth** — 매출 성장률, 계약 확대 |
| A5: Advocate (옹호/추천) | SNS 반응, CS 만족도 | **Client Advocacy** — NPS, 재계약률, 레퍼럴 |
| Localization Gauge | 번역/기능/UX 현지화 | **Operational Readiness** — 프로세스/역량/품질 성숙도 |
| Strategy Step | Acquisition→Extend | **Growth Phase** — Build→Launch→Scale→Diversify |

### 1.5 문서 관리 현재 실태 (Google Drive)

기초데이터 문서(v1.2.1): 10개 소스 문서가 `ambCoWork/company` 폴더에 산재.

**주요 문제점**:
1. 문서 간 **13개 항목 데이터 불일치** (연락처, 주소, 서비스 일정, 명칭 등)
2. **9개 데이터 공백** (인증, SLA, 고객추천, 스크린샷, 실적 등)
3. 조직 규모 **계획 vs 현실 괴리** (2025 목표 35명 → 2026 실제 16명)
4. 수동 갱신으로 **버전 혼재** (같은 정보가 문서마다 다른 값)
5. **경영 KPI 데이터 미수집** — Type 4 IR 문서에 들어갈 실적 데이터가 체계적으로 관리되지 않음

## 2. TO-BE 요구사항 정의

### 2.1 통합 시스템 개요

DocBuilder와 5A Matrix는 KMS 내 별도 서브모듈이지만, **데이터 흐름에서 밀접하게 연동**된다. 5A Matrix의 KPI 스냅샷과 AI 인사이트는 DocBuilder의 기초데이터로 자동 공급되며, 특히 Type 4(IR/전략소개서)의 핵심 데이터 소스가 된다.

```
┌──────────────────────────────────────────────────────────────────────┐
│                  KMS Module — 2개 서브모듈 통합                        │
│                                                                       │
│  ┌─── Document Builder ────────┐    ┌─── 5A Matrix (DDD) ─────────┐  │
│  │                              │    │                              │  │
│  │  Base Data ──▶ AI Engine     │◀───│  5A Funnel KPI Dashboard    │  │
│  │       │        ──▶ PPTX/DOCX│    │  Operational Readiness      │  │
│  │       │                      │    │  Strategy Position Map      │  │
│  │  Feedback Loop               │    │  AI Insights / Forecasting  │  │
│  │  Google Drive Sync           │    │  Cross-Module Data Collect  │  │
│  │                              │    │                              │  │
│  └──────────────────────────────┘    └──────────────────────────────┘  │
│                    │                              │                     │
│                    ▼                              ▼                     │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │     Shared Infrastructure: Tags / WorkItem / Module Events       │  │
│  └──────────────────────────────────────────────────────────────────┘  │
│                    │                                                    │
│  ┌────────┬────────┼──────────┬────────────┬──────────┐               │
│  │Billing │Service │   HR     │  Project   │ Webmail  │               │
│  │Module  │ Mgmt   │ Module   │  Module    │ Module   │               │
│  └────────┴────────┴──────────┴────────────┴──────────┘               │
└──────────────────────────────────────────────────────────────────────┘
```

**핵심 연동 포인트**:
- 5A Snapshots (`amb_kms_ddd_snapshots`) → DocBuilder 기초데이터 **KPI_DASHBOARD** 카테고리에 자동 반영
- 5A Gauge Scores → DocBuilder **OPERATIONAL_READINESS** 카테고리에 자동 반영
- 5A AI Insights → Type 4 IR 문서의 Traction/Metrics 섹션 데이터 소스
- DocBuilder 문서 생성 시 → DDD 관련 기초데이터는 최신 스냅샷에서 자동 풀링

### 2.2 4가지 문서 타입 + 5A 데이터 매핑

| Type | 문서명 | 대상 | 5A Matrix 데이터 활용 |
|------|--------|------|---------------------|
| 1 | 회사소개서 | 잠재 고객, 파트너, 정부 | Operational Readiness(프로세스 성숙도) → Core Competencies 섹션 |
| 2 | 서비스소개서 | B2B 클라이언트 | A3 SLA 충족률 + A5 재계약률 → Quality/SLA 섹션 |
| 3 | 제품소개서 | SaaS 구매 담당자 | A1 유입 데이터 + A4 GMV/유효몰 → Traction 섹션 |
| 4 | 전략소개서(IR) | 투자자, 전략 파트너 | **전 5A Stage + Gauge + Strategy** → KPI/Traction/Growth 핵심 데이터 |

**Type 4(IR)에서의 5A 데이터 활용 상세**:

| IR 섹션 (Sequoia 10대 목차) | 5A 데이터 소스 | 활용 |
|---------------------------|--------------|------|
| 7. Business Model | A2 전환율, A4 매출 성장 | 실제 지표(LTV/CAC/Churn) 검증 근거 |
| 8. Team | Gauge — Capability | 팀 역량 성숙도 정량화 |
| 9. Financial | A4 매출 스냅샷 QoQ/YoY | 3년 P&L 실적 vs 계획 비교 |
| 10. Traction & Metrics | 전체 5A Stage Primary KPI | 핵심 지표 하이라이트 |
| 11. Growth Strategy | Strategy Position Map | 현재 단계 + 다음 목표 시각화 |
| 5A Summary | 전체 | 아메바 고유 경영 방법론 소개 |

| ★ 지표 과다 정의 | 대시보드 복잡도 증가 | 중간 | Stage당 Primary KPI 1개 + Secondary 2~3개로 제한 |

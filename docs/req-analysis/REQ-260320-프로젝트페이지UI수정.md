## 1. AS-IS 현황 분석

### 1.1 Issues 탭 (`ProjectIssuesTab.tsx`)

| 항목 | 현재 상태 |
|------|----------|
| **데이터 로딩** | `size: 500` 한 번에 전체 로드 (페이지네이션 없음) |
| **상태 필터** | 단일 선택 `<select>` — 하나만 선택 가능 |
| **타입 필터** | 단일 선택 `<select>` — 하나만 선택 가능 |
| **기본 필터** | CLOSED만 기본 제외 (REJECTED는 표시됨) |
| **테이블** | 스크롤 없이 전체 렌더링, 모바일 대응 부족 |

**문제점:**
- 이슈 수가 많을 경우 모든 이슈를 한꺼번에 렌더링하여 성능 저하
- `REJECTED`도 기본 뷰에서 제외해야 하지만 현재 노출됨
- 멀티 필터 불가 — 예: OPEN + IN_PROGRESS 동시 선택 불가

### 1.2 Kanban 탭 (`ProjectKanbanTab.tsx` → `IssueKanbanView.tsx`)

| 항목 | 현재 상태 |
|------|----------|
| **컬럼** | 6개 고정: OPEN, APPROVED, IN_PROGRESS, RESOLVED, CLOSED, REJECTED |
| **필터** | 없음 — 모든 상태 컬럼이 항상 표시 |
| **데이터** | `size: 200` 한 번에 로드 |

**문제점:**
- CLOSED/REJECTED 컬럼이 기본적으로 나타남 — 작업 중인 이슈에 집중 어려움
- 상태 필터가 없어 특정 상태만 볼 수 없음
- 컬럼이 6개 고정이라 가로 스크롤 필요

### 1.3 Gantt 탭 (`ProjectGanttTab.tsx` → `IssueGanttView.tsx`)

| 항목 | 현재 상태 |
|------|----------|
| **좌측 라벨 (그룹모드)** | Group(120px) + Title(200px) + Author(80px) + Assignee(80px) + Progress(60px) = 600px |
| **좌측 라벨 (플랫모드)** | Title만 240px |
| **모바일** | 600px 고정폭 → 모바일 화면에서 가로 스크롤 필수 |
| **그룹 행** | Epic/Component 이름 + 이슈 수 + 진행률 표시 |
| **이슈 행 그룹 컬럼** | Epic/Component 소속 이슈 앞에 그룹명 중복 표시 |
| **Author/Assignee** | 항상 표시, 숨기기 옵션 없음 |

**문제점:**
- 모바일에서 600px 라벨 영역이 차지하여 간트 바가 거의 안 보임
- Author/Assignee 컬럼을 숨길 수 없어 좁은 화면에서 불편
- Epic/Component 소속 이슈 행의 그룹 컬럼에 이미 그룹 헤더 행에서 표시된 이름이 다시 나옴 (중복)

### 1.4 WBS 탭 (`ProjectWbsTab.tsx`)

| 항목 | 현재 상태 |
|------|----------|
| **CLOSED 토글** | 글로벌 `showClosed` 상태 → API 파라미터 `include_closed` |
| **토글 위치** | 오른쪽 상단에 단일 버튼 `[Closed 보기]` |
| **토글 동작** | ON/OFF 시 전체 WBS 트리 재로딩 (include_closed: true/false) |
| **기본 상태** | `showClosed: false` → CLOSED 이슈 기본 미표시 |

**문제점:**
- 현재 글로벌 토글이므로 모든 그룹에 일괄 적용됨
- 요구사항: Epic/Component/Unassigned **각 그룹 제목 옆**에 독립적으로 `[closed 포함 보기]` 버튼 배치
- REJECTED도 CLOSED와 동일하게 기본 숨김 처리 필요 여부 확인 필요 (요구사항에 "closed 된 이슈"만 명시)

## 2. TO-BE 요구사항

### 2.1 Issues 탭: 페이징 + 다중 필터

**FR-ISS-01. 페이징 처리**
- 클라이언트 사이드 페이징 (전체 데이터 로드 후 프론트에서 페이지 분할)
- 한 페이지당 기본 20건
- `이전/다음` + 페이지 번호 네비게이션
- 현재 `N/M 건` 표시

**FR-ISS-02. 상태 다중선택 필터**
- `<select>` → 다중 선택 가능한 체크박스 드롭다운
- 기본 선택: CLOSED, REJECTED **제외** (OPEN, APPROVED, IN_PROGRESS, RESOLVED 선택됨)
- 사용자가 명시적으로 CLOSED, REJECTED를 포함 가능

**FR-ISS-03. 타입 다중선택 필터**
- `<select>` → 다중 선택 가능한 체크박스 드롭다운
- 기본: 전체 타입 선택

### 2.2 Kanban 탭: 상태 다중선택 필터

**FR-KAN-01. 상태 다중선택 필터**
- 칸반 보드 상단에 상태 필터 체크박스 바
- 기본: CLOSED, REJECTED **제외** (4개 컬럼만 표시)
- 각 상태를 개별적으로 ON/OFF → 해당 컬럼 표시/숨김
- 최소 1개 상태는 항상 선택 상태

### 2.3 Gantt 탭: Author/Assignee 가리기 + 모바일 최적화 + 그룹명 제거

**FR-GNT-01. Author/Assignee 가리기 옵션**
- 간트 차트 상단에 토글 컨트롤:
  - `[ ] 작성자 표시` — 기본 OFF (모바일) / ON (데스크톱)  
  - `[ ] 담당자 표시` — 기본 ON
- 숨길 경우 해당 컬럼 폭 0px → 전체 라벨 영역 축소

**FR-GNT-02. 모바일 화면 최적화**
- 모바일(< 768px):
  - Author 컬럼 기본 숨김
  - Group 컬럼 폭 축소 (120px → 80px)
  - Title 컬럼 폭 축소 (200px → 120px)
  - 전체 라벨 영역 자동 축소
- 화면 크기별 자동 적용 (CSS media query 또는 JS 감지)

**FR-GNT-03. Epic/Component 소속 이슈의 그룹명 제거**
- Epic/Component 그룹 헤더 행에 이미 그룹 이름이 표시되므로
- 해당 그룹 하위 이슈 행의 그룹 컬럼에는 그룹명을 표시하지 않음
- 그룹 컬럼 자체는 유지 (들여쓰기 효과), 텍스트만 제거

### 2.4 WBS 탭: 그룹별 Closed 토글

**FR-WBS-01. 그룹별 Closed 토글**
- 기본 뷰에서 CLOSED 이슈 미표시 (현재와 동일)
- 기존 글로벌 `[Closed 보기]` 버튼 유지 (전체 그룹 일괄 토글)
- **추가**: 각 Epic/Component/Unassigned 그룹의 제목 옆에 `[closed N건 보기]` 버튼
  - 클릭 시 해당 그룹만 CLOSED 이슈 포함하여 표시
  - N = 해당 그룹의 CLOSED 이슈 수 (= 서버 응답의 `issueCount - 표시된 이슈 수` 또는 별도 카운트)
  - 다시 클릭 시 숨김
- **API 변경**: 항상 `include_closed: true`로 전체 데이터 로드, 프론트에서 그룹별 필터링
  - 또는 기존처럼 `include_closed` 파라미터 유지하되, 그룹별 토글 시 전체 데이터 사용

## 4. 사용자 플로우

### 4.1 Issues 탭 필터 + 페이징 플로우

```
사용자가 프로젝트 상세 → Issues 탭 진입
→ 기본 뷰: CLOSED/REJECTED 제외, 전체 타입, 20건/페이지
→ 상태 필터 드롭다운 클릭 → 체크박스 목록 표시
  → CLOSED 체크  → 목록에 CLOSED 이슈 추가, 페이지 1로 리셋
→ 타입 필터 드롭다운 클릭 → BUG만 체크
  → BUG 이슈만 표시
→ 페이지 네비게이션: [1] 2 3 ... 다음
  → 페이지 2 클릭 → 21~40번 이슈 표시
```

### 4.2 Kanban 상태 필터 플로우

```
사용자가 Kanban 탭 진입
→ 기본 뷰: OPEN, APPROVED, IN_PROGRESS, RESOLVED 4개 컬럼
→ 상태 바에서 CLOSED 체크 → CLOSED 컬럼 추가
→ OPEN 체크 해제 → OPEN 컬럼 숨김
→ 드래그앤드롭: 표시된 컬럼 간에만 이동 가능
```

### 4.3 Gantt Author/Assignee 토글 플로우

```
데스크톱: Author/Assignee 기본 표시
→ [작성자 숨기기] 클릭 → Author 컬럼 숨김, 라벨 영역 520px으로 축소
→ [담당자 숨기기] 클릭 → Assignee 컬럼도 숨김, 440px으로 축소

모바일 진입:
→ Author 자동 숨김, 라벨 영역 컴팩트 모드
→ [담당자 표시] 토글로 Assignee ON/OFF
```

### 4.4 WBS 그룹별 Closed 토글 플로우

```
WBS 탭 진입 → 기본: CLOSED 이슈 숨김
→ Epic "인증 시스템" 제목 옆에 [closed 3건 보기] 표시
→ 클릭 → 해당 Epic 하위에 CLOSED 이슈 3건 추가 표시
→ 다시 클릭 → CLOSED 이슈 숨김
→ 다른 Epic/Component는 영향 없음
→ 글로벌 [Closed 보기] 클릭 → 모든 그룹 CLOSED 포함
```

## 5. 기술 제약사항

### 5.1 데이터 로딩 전략
- **Issues/Kanban/Gantt**: 현재 `size: 200~500`으로 전체 로드 → 클라이언트 사이드 필터링/페이징 유지
- 서버 사이드 페이징 전환은 이번 스코프에서 제외 (향후 대량 데이터 시 검토)

### 5.2 WBS 데이터 전략
- WBS API는 `include_closed` 파라미터 존재
- 그룹별 토글 구현을 위해 항상 `include_closed: true`로 요청하고 프론트에서 필터링
- 또는 `include_closed: false`(기본) + 개별 그룹 토글 시 `include_closed: true`로 재요청
- **권장**: 항상 전체 데이터 로드 후 프론트에서 필터링 → 즉시 토글 반응

### 5.3 IssueKanbanView 공유 컴포넌트
- `IssueKanbanView`는 Issues 페이지(`/issues`)에서도 사용될 수 있음
- 필터 제어를 외부에서 주입할 수 있도록 props 설계 필요

### 5.4 IssueGanttView 공유 컴포넌트
- `IssueGanttView`도 공유 가능한 컴포넌트
- visibility 옵션을 props로 전달

# 요구사항 분석서: 로그인 법인 기억하기

- **문서번호**: REQ-로그인법인기억하기-20260309
- **작성일**: 2026-03-09
- **요청자**: 김이경 대표

---

## 1. 요구사항 요약

로그인 페이지의 "Find your organization" 법인 검색에서 한 번 선택한 법인을 로컬(쿠키/localStorage)에 저장하여, 다음 로그인 시 검색 없이 바로 선택할 수 있게 한다.

---

## 2. AS-IS 현황

### 로그인 플로우
1. 사용자가 "Find your organization" 버튼 클릭
2. `EntitySearchModal` 오픈 → 법인명/코드 2자 이상 입력 → API 검색 (`GET /auth/entities/search?q=`)
3. 검색 결과에서 법인 선택 → `selectedEntity` 상태 저장
4. 이메일/비밀번호 입력 후 로그인
5. 로그인 성공 시 `/select-entity` 이동

### 문제점
- 매 로그인마다 법인을 검색해야 함 (회사명을 기억하고 타이핑해야 함)
- 다법인 소속 사용자가 반복적으로 같은 회사를 검색하는 불편
- 모바일에서 타이핑 부담 큼

### 관련 파일
| 파일 | 역할 |
|------|------|
| `apps/web/src/domain/auth/pages/LoginPage.tsx` | 로그인 페이지 |
| `apps/web/src/domain/auth/components/EntitySearchModal.tsx` | 법인 검색 모달 |
| `apps/web/src/domain/auth/service/auth.service.ts` | 인증 서비스 (searchEntities) |

## 3. TO-BE 요구사항

### 핵심 기능
- **최근 로그인 법인 저장**: 법인 선택 후 로그인 성공 시, 해당 법인 정보를 localStorage에 저장
- **최근 법인 목록 표시**: EntitySearchModal 열 때, 검색 입력 전에 "최근 사용" 법인 목록을 표시
- **바로 선택**: 최근 법인 클릭 시 검색 없이 바로 선택 가능
- **로그인 페이지 자동 설정**: 저장된 법인이 1개 이상이면, 가장 최근 법인을 자동으로 selectedEntity에 세팅

### 저장 방식
- **저장소**: localStorage (`amb_recent_entities` 키)
  - 쿠키 대신 localStorage 사용 이유: 쿠키는 서버 요청마다 전송되어 불필요한 트래픽 발생, HttpOnly 쿠키와 혼동 방지, 저장 용량 제한(4KB) 등
- **저장 데이터**: `EntitySearchItem[]` (entityId, code, name, nameEn, country)
- **최대 개수**: 5개 (FIFO, 중복 시 최근으로 이동)
- **만료**: 없음 (브라우저 데이터 삭제 시 자동 제거)

### 사용자 플로우 (변경 후)

```
[첫 로그인]
1. "Find your organization" 클릭
2. EntitySearchModal: "최근 사용" 섹션 없음 → 검색으로 법인 선택
3. 로그인 성공 → 법인 정보 localStorage에 저장

[재 로그인]
1. 로그인 페이지 진입 → 가장 최근 법인이 자동 선택됨
2. 바로 이메일/비밀번호 입력 후 로그인 가능
3. 다른 법인 원하면 "Find your organization" 클릭
4. EntitySearchModal: 상단에 "최근 사용" 법인 목록 표시
5. 최근 법인 클릭으로 즉시 선택 OR 검색으로 새 법인 선택
```

## 5. 기술 제약사항

- **보안**: 저장하는 데이터는 법인 공개 정보(이름, 코드, 국가)뿐이며 민감 정보 아님
- **백엔드 변경 없음**: 순수 프론트엔드 기능
- **다중 브라우저**: localStorage는 브라우저별 독립 → 다른 브라우저에서는 다시 검색 필요
- **시크릿 모드**: localStorage 사용 불가 → graceful fallback (기존 검색 방식)

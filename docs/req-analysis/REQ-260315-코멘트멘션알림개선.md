## 1. 요구사항 요약

사용자가 코멘트에서 `@username`으로 멘션되었을 때 알림을 받아, 중요한 메시지를 놓치지 않도록 한다.
- **적용 범위**: 태스크(TODO), 이슈(Issue), 회의록(Meeting Note) 코멘트 전체

### Acceptance Criteria
1. 코멘트에 `@username` 포함 시 해당 사용자에게 알림 생성
2. 알림 목록에 표시: 작성자 이름, "mentioned you in a comment", 리소스 제목, 타임스탬프
3. 알림 클릭 시 해당 리소스 상세 → 멘션된 코멘트로 스크롤
4. 클릭 후 "읽음" 처리

### 3.2 개선 필요 항목 상세

#### G4: 이슈 코멘트 RichTextEditor @멘션 자동완성

**현재 상태:**
- 이슈 코멘트는 `RichTextEditor` (Tiptap 기반) 사용
- `@username` 수동 입력 시 백엔드에서 멘션 감지는 되지만, 사용자가 정확한 username을 알고 있어야 함
- 자동완성이 없어 UX 불편

**개선 방안 (2가지 옵션):**

| 옵션 | 방법 | 장점 | 단점 |
|------|------|------|------|
| **A** | `RichTextEditor`에 Tiptap Mention 확장 추가 | 리치 텍스트 + 멘션 모두 지원 | 구현 복잡도 높음, Tiptap Mention 플러그인 설정 필요 |
| **B** | 이슈 코멘트 입력을 `MentionInput`으로 교체 | 빠른 구현, 기존 검증된 컴포넌트 | 이슈 코멘트에서 리치 텍스트 포맷팅 상실 |

**권장**: **옵션 A** — Tiptap `@tiptap/extension-mention` 확장으로 RichTextEditor에 @멘션 자동완성 추가. 이슈 코멘트는 긴 설명이 많아 리치 텍스트 유지가 중요.

## 4. 사용자 플로우

### 4.1 코멘트 @멘션 알림 플로우

```
[사용자 A] 코멘트 입력 (@사용자B 입력)
  ↓
[프론트엔드] @멘션 자동완성 드롭다운 표시
  → 멤버 선택 → @username 삽입
  ↓
[API 전송] POST /todos/:id/comments  (또는 /issues, /meeting-notes)
  → content: "이건 @사용자B 확인 부탁"
  ↓
[백엔드] addComment()
  → extractMentionedUserIds(content, entityUsers, authorId)
  → mentionedIds = ['사용자B의 UUID']
  ↓
[이벤트 발행] MENTION_EVENT
  → resourceType, resourceId, commentId, senderId, recipientIds
  ↓
[NotificationListener]
  → COMMENT_MENTION 알림 생성 (DB 저장)
  → SSE emit → 사용자 B 브라우저에 실시간 전달
  → Web Push → 사용자 B 디바이스에 푸시
  ↓
[사용자 B 프론트엔드]
  → NotificationModal 슬라이드 (5초)
  → NotificationBell 배지 카운트 증가
  ↓
[사용자 B] 알림 클릭
  → markAsRead API 호출
  → 해당 리소스 상세 페이지로 이동 (?commentId=...)
  → 멘션된 코멘트로 스크롤 + 하이라이트 (3초)
```

### 4.2 알림 표시 내용

```
┌──────────────────────────────────────┐
│ 🔔 [사용자A 이름]                     │
│ mentioned you in a comment           │
│ [태스크/이슈/회의록 제목]             │
│ 2분 전                                │
└──────────────────────────────────────┘
```

## 5. 기술 제약사항

### 5.1 RichTextEditor Mention 확장 관련
- Tiptap `@tiptap/extension-mention` 패키지 추가 필요
- `tippy.js` (드롭다운 포지셔닝) 의존성 필요
- 기존 RichTextEditor는 Meeting Notes 본문 편집기로도 사용 → 멘션 기능을 옵션으로 제공해야 함 (prop으로 활성화/비활성화)
- 멘션 렌더링: 에디터 내 `@username`을 특별한 노드로 표시 (배경색, 클릭 불가 등)
- HTML 출력에서 멘션 추출: 백엔드 `extractMentionedUserIds`는 plain text 기반 → RichTextEditor의 HTML/JSON 출력에서 멘션 추출 방식 검토 필요

### 5.2 멤버 목록 조회
- 현재 `MentionInput`은 로컬 멤버 목록 사용 (전역 상태 또는 API)
- RichTextEditor의 멘션 자동완성도 동일한 멤버 목록 API 활용 필요
- 법인(entity) 범위 내 사용자만 대상

### 5.3 기존 기능과의 호환성
- 백엔드 멘션 추출 유틸 (`extractMentionedUserIds`)은 plain text `@username` 패턴 기반
- RichTextEditor가 HTML을 전송하는 경우, HTML 태그 제거 후 멘션 추출 필요 또는 Tiptap mention 노드에서 직접 추출

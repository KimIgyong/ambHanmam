# My Page UI Restructure

## 1. 요구사항 요약

마이페이지(`/my-page`) 화면의 레이아웃을 전면 재구성한다.

### 핵심 변경
1. **기본정보 섹션** 레이아웃 재설계: 프로필 사진 + 정보 항목을 2컬럼 그리드로 배치
2. **편집 가능/불가능 항목** 구분 명확화
3. **카드뷰 퀵링크** 추가: 출퇴근 / 내 휴가 / 지출결의
4. **섹션 순서 변경**: 기본정보 → 카드링크 → 연차현황 → 지출결의 → 소속법인 → Unit → Cell
5. 비밀번호 변경 / 타임존 섹션은 페이지 하단 유지

### 기본정보 레이아웃 (TO-BE)
```
┌─────────────────────────────────────────────────────┐
│ 기본 정보                                            │
│ ┌──────────┐  이름: {name} ✏️ (변경 가능)            │
│ │ 프로필    │  이메일: {email} (변경 불가)             │
│ │  사진     │                                        │
│ │          │  Unit/부서: {unit} (마스터만 변경)        │
│ │ 변경/삭제 │  역할: {role} (마스터만 변경)             │
│ └──────────┘                                        │
│ 회사이메일: {companyEmail} ✏️ (변경 가능)  │ 가입일: {date} │
│ 비상연락처: {phone} ✏️ (변경 가능)                    │
└─────────────────────────────────────────────────────┘
```

## 2. AS-IS 현황 분석

### 2.1 현재 MyPage 레이아웃 순서
1. ProfileSection (기본정보 + 프로필 이미지)
2. Entities 테이블 (소속 법인)
3. Units 테이블 (Unit 배정)
4. Cells 테이블 (Cell 배정)
5. Leave Balance (연차 현황)
6. Expense Requests (지출결의서)
7. PasswordChangeSection
8. TimezoneSection

### 2.2 현재 ProfileSection 구조
- 프로필 이미지: 좌측 아바타 + 사진변경/삭제 버튼
- 기본정보: `2-column grid`로 표시
  - 이름 (편집 가능 - Pencil 아이콘 인라인 편집)
  - 이메일 (읽기 전용)
  - Unit/부서 (읽기 전용)
  - 역할 (읽기 전용)
  - 회사이메일 (읽기 전용 - **현재 편집 불가!**)
  - 가입일 (읽기 전용)
  - 비상연락처 (편집 가능 - Pencil 아이콘 인라인 편집)

### 2.3 현재 편집 가능 항목
| 항목 | 현재 | TO-BE |
|------|------|-------|
| 이름 | ✅ 편집 가능 | ✅ 유지 |
| 이메일 | ❌ 읽기 전용 | ❌ 유지 (변경 불가) |
| Unit/부서 | ❌ 읽기 전용 | ❌ 유지 (마스터만 변경 - 구성원관리에서) |
| 역할 | ❌ 읽기 전용 | ❌ 유지 (마스터만 변경) |
| 회사이메일 | ❌ 읽기 전용 | ✅ **편집 가능으로 변경** |
| 가입일 | ❌ 읽기 전용 | ❌ 유지 |

## 3. TO-BE 요구사항

### 3.1 기본정보 섹션 레이아웃
프로필 이미지를 좌측에 배치하고, 우측에 정보 항목을 그리드로 배치:

**Row 1**: [프로필 사진 (좌)] [이름 ✏️ / 이메일 (우)]
**Row 2**: [사진변경/삭제 (좌)] [Unit·부서 / 역할 (우)]
**Row 3**: [회사이메일 ✏️ / 가입일]
**Row 4**: [비상연락처 ✏️]

### 3.2 카드뷰 퀵링크 (기본정보 아래)
3개 카드를 가로로 나열, 클릭 시 해당 페이지로 네비게이션:
- 출퇴근 (Clock 아이콘, 파란색)
- 내 휴가 (CalendarDays 아이콘, 초록색)
- 지출결의 (FileText 아이콘, 보라색)

### 3.3 섹션 순서 (TO-BE)
1. **기본정보** (ProfileSection 리디자인)
2. **카드뷰 퀵링크** (출퇴근/내휴가/지출결의)
3. **연차현황** (Leave Balance)
4. **지출결의서** (Expense Requests)
5. **소속법인** (Entities)
6. **부서/Unit 현황** (Units)
7. **Cell 배정현황** (Cells)
8. 비밀번호 변경 (PasswordChangeSection)
9. 타임존 & 언어 (TimezoneSection)

### 3.4 회사이메일 편집 기능
- 백엔드: `UpdateProfileDto`에 `company_email` 필드 추가
- 프론트엔드: 비상연락처와 동일한 인라인 편집 패턴 적용

## 5. 사용자 플로우

1. `/my-page` 진입
2. 기본정보 섹션에서 프로필 사진 좌측, 이름/이메일/Unit/역할/회사이메일/가입일/비상연락처 우측 표시
3. 이름/회사이메일/비상연락처 옆 ✏️ 클릭 → 인라인 편집 → Enter 저장 / Esc 취소
4. 카드뷰 "출퇴근" 클릭 → `/attendance` 이동
5. 카드뷰 "내 휴가" 클릭 → `/leave-requests` 이동
6. 카드뷰 "지출결의" 클릭 → `/expense-requests` 이동
7. 아래로 스크롤하여 연차현황/지출결의/소속법인/Unit/Cell 확인

## 6. 기술 제약사항

- **백엔드 변경**: `users/me/profile` PATCH DTO에 `company_email` 추가 필요
- **DB**: `amb_users.usr_company_email` 컬럼이 이미 존재하는지 확인 필요
- **라우트**: `/attendance`, `/leave-requests` 경로가 존재하는지 확인 필요 (없으면 카드 비활성화 또는 숨김)
- **i18n**: 카드뷰 라벨, 편집 힌트 등 새 번역 키 필요 (ko/en/vi)
- **프론트엔드 전용 가능**: 섹션 순서 변경, 레이아웃 변경은 프론트엔드만으로 처리

## 1. 태스크/할일 칸반뷰 UI 개선

### 1.1 AS-IS 현황

#### 뷰 모드 및 Scope 탭
- 3가지 뷰 모드 지원: **List** | **Card** | **Kanban** (`localStorage`에 저장)
- 상단 `ViewScopeTab` 컴포넌트로 scope 선택: **Mine** | **Unit** | **Cell** | **All**
- scope='mine'일 때: 직접 `TodoKanbanView` 렌더링
- scope='unit'|'cell'|'all'일 때: `ScopedTodoList` 경유하여 `TodoKanbanView` 렌더링

#### 칸반뷰 컬럼 구조
- 3컬럼: **DUE_TODAY** (오렌지) | **IN_PROGRESS** (블루) | **COMPLETED** (그린)
- `computedStatus` 기반 분류:
  - `COMPLETED` → COMPLETED 컬럼
  - `DUE_TODAY`, `OVERDUE` → DUE_TODAY 컬럼
  - 나머지(`IN_PROGRESS`, `UPCOMING_SOON`, `UPCOMING`) → IN_PROGRESS 컬럼

#### 카드 렌더링 정보 (현재)
- 제목 (상태별 스타일 — 완료시 취소선, 마감초과시 빨간색)
- 마감일 + 초과일수
- 태그 (최대 2개)
- 액션 버튼 (소유자 한정): 완료 체크, Issue 전환, 이력, 삭제
- **❌ 작성자 이름 미표시**

#### 백엔드 데이터
- `TodoResponse`에 `userName` 필드 이미 존재 (`user.usrName` 매핑)
- `TodoEntity`에 `@ManyToOne` 관계로 `user` 엔티티 조인됨
- Unit/Cell/Company 쿼리: `leftJoinAndSelect('todo.user', 'user')` — 이미 user 데이터 포함

## 2. 로비챗/알림 기능 개선

### 2.1 AS-IS 현황

#### 알림 UI 구조
- **NotificationBell**: 헤더 우상단 종 아이콘 + 읽지 않은 건수 뱃지
- 클릭 시 드롭다운 패널: 알림 목록 (아이콘 + 제목 + 메시지 프리뷰 + 상대시간)
- 알림 항목 클릭 → `handleNotificationClick()`으로 `window.location.href` 직접 이동
- **NotificationModal**: SSE로 실시간 수신 → 상단 중앙 5초 자동 해제 토스트

#### 라우팅 로직
```
ntfResourceType → 이동 경로
TODO           → /todos?id={resourceId}
ISSUE          → /issues?id={resourceId}
MEETING_NOTE   → /meeting-notes/{resourceId}
CALENDAR       → /calendar?event={resourceId}
TALK           → /amoeba-talk?channel={resourceId}
```

#### 알림 데이터 구조
- `ntfResourceType`: 리소스 유형 (TODO, ISSUE, MEETING_NOTE, CALENDAR)
- `ntfResourceId`: 해당 리소스의 UUID
- `ntfType`: NOTIFICATION 유형 (TODO_ASSIGNED, ISSUE_ASSIGNED, COMMENT_MENTION 등)
- `ntfMessage`: 알림 메시지 (COMMENT_MENTION의 경우 JSON에 `commentId` 포함)

### 2.4 상세 설계 방향

#### 2.4.0 전체 알림보기 버튼
- 알림 드롭다운 목록 하단(footer)에 "전체 알림보기" 링크 버튼 배치
- 클릭 시 `/amoeba-talk` 페이지로 이동하면서 `NOTIFICATION_CHANNEL_ID` (`__NOTIFICATION_CENTER__`) 채널 자동 선택
- 이동 방식: `window.location.href = '/amoeba-talk?channel=__NOTIFICATION_CENTER__'`
- `NotificationChatView` 컴포넌트가 전체 알림 리스트를 렌더링

#### 2.4.1 바로가기 버튼
- 기존 알림 항목 클릭 → 페이지 이동 로직을 **"바로가기" 버튼**에 분리
- 알림 항목 본체 클릭은 읽음 처리만 수행 (또는 미리보기 트리거)
- 라우팅 로직: 기존 `handleNotificationClick()`의 switch 문 재활용

#### 2.4.2 미리보기 사이드 패널
- 알림 항목의 "미리보기" 버튼 클릭 → 오른쪽에서 슬라이드인 패널으로 상세 내용 표시
- 리소스 유형별 미리보기 컨텐츠:
  - **TODO**: 할일 상세 (제목, 설명, 마감일, 상태, 태그, 댓글)
  - **ISSUE**: 이슈 상세 (제목, 설명, 상태, 우선순위, 담당자, 댓글)
  - **MEETING_NOTE**: 미팅노트 상세
  - **CALENDAR**: 캘린더 이벤트 상세
- 패널 닫기: X 버튼 또는 바깥 영역 클릭

### 2.5 기술 제약사항
- 미리보기 패널은 신규 컴포넌트 필요
- 각 리소스 유형별 상세 조회 API는 이미 존재 (useTodoDetail, useIssueDetail 등)
- 사이드 패널 내에서 리소스별 요약 뷰 컴포넌트 필요

## 3. 로비챗 메시지 삭제 기능 개선

### 3.1 AS-IS 현황

#### 삭제 규칙
- **모두에게 삭제** (`deleteMessage`): 본인 메시지 + 1시간 이내만 가능
  - 백엔드: `message.service.ts` — 1시간 초과 시 `TALK_DELETE_TIME_EXPIRED` (E6008) 에러
  - 프론트: `canDeleteForAll()` — 1시간 체크 후 버튼 표시/숨김
  - DB: `msg_deleted_at` soft delete (전체 사라짐)
- **나만 삭제** (`hideMessage`): 시간 제한 없이 항상 가능
  - DB: `amb_talk_message_hides` 테이블에 레코드 삽입
  - 해당 유저의 메시지 목록 조회 시 `NOT IN` 서브쿼리로 필터

#### UI 구현
- 메시지 호버 시 액션 버튼 표시:
  - 🗑️ (Trash2): 본인 메시지 → `handleDelete()` (모두에게 삭제, confirm dialog)
  - 👁️‍🗨️ (EyeOff): 드롭다운 토글 →
    - "나만 삭제" (항상 노출)
    - "모두에게 삭제" (빨간색, `canDeleteForAll()` 통과 시만 노출)

#### 문제점
1. 🗑️ 아이콘과 👁️‍🗨️ 아이콘이 분리되어 있어 UX가 혼란스러움
2. 1시간 이후 "모두에게 삭제" 버튼이 단순히 사라지는 것 — 왜 사라졌는지 사용자가 인지 못함
3. 삭제 옵션이 두 곳에 분산 (Trash2 직접 삭제 + EyeOff 드롭다운 내 삭제)

### 3.2 TO-BE 요구사항

| # | 요구사항 | 상세 |
|---|---------|------|
| 3-1 | 1시간 이내 삭제 | "내 대화창만 삭제" + "모두에게 삭제" 둘 다 선택 가능 |
| 3-2 | 1시간 이후 삭제 | "내 대화창에서만 삭제 가능합니다" 안내 후 "내 대화창만 삭제"만 가능 |
| 3-3 | UI 통합 | 기존 분산된 🗑️ + 👁️‍🗨️ 아이콘을 하나의 삭제 드롭다운으로 통합 |

### 3.3 갭 분석

| 항목 | AS-IS | TO-BE | 갭 |
|------|-------|-------|----|
| 1시간 이내 (본인) | 모두에게 삭제 + 나만 삭제 | 동일 (변경 없음) | 없음 |
| 1시간 이후 (본인) | "모두에게 삭제" 버튼 사라짐 (설명 없음) | "내 대화창에서만 삭제 가능합니다" 안내 표시 | **프론트 UI 변경** |
| 타인 메시지 | 나만 삭제만 가능 | 동일 | 없음 |
| 삭제 UI | 🗑️ + 👁️‍🗨️ 분리 | 단일 삭제 드롭다운으로 통합 | **프론트 UI 리팩터** |

### 3.4 상세 설계 방향

#### 통합 삭제 드롭다운

```
│ 🗑️ 모두에게 삭제 (빨간색)    │
└─────────────────────────────┘

[본인 메시지, 1시간 이후]
┌─────────────────────────────┐
│ ⓘ 1시간이 지나 모두에게      │
│   삭제할 수 없습니다         │
│ ───────────────────         │
│ 🙈 내 대화창에서 삭제        │
└─────────────────────────────┘

[타인 메시지]
┌─────────────────────────────┐
│ 🙈 내 대화창에서 삭제        │
└─────────────────────────────┘
```

### 3.5 기술 제약사항
- 백엔드 변경 없음 — 기존 `deleteMessage`/`hideMessage` API 그대로 사용
- 프론트엔드 `TalkMessageList.tsx` UI 수정만 필요
- 1시간 체크 로직 (`canDeleteForAll`)은 기존 코드 재활용
- i18n 번역 키 추가 필요 (안내 메시지)

## 4. 사용자 플로우

### 4.1 칸반뷰 작성자 이름 확인 플로우
```
1. /todos 페이지 진입
2. Kanban 뷰 모드 선택
3. Unit/Cell/All 스코프 탭 선택
4. 칸반 카드에서 작성자 이름 확인
5. 카드 클릭 → 상세 보기
```

### 4.2 알림 전체보기/바로가기/미리보기 플로우
```
1. 알림 수신 (SSE)
2. 🔔 종 아이콘 클릭 → 알림 드롭다운 열림
3-A. "전체 알림보기" 클릭 → 로비챗 알림센터(/amoeba-talk?channel=__NOTIFICATION_CENTER__)로 이동
3-B. 개별 알림 "바로가기" 클릭 → 해당 리소스 페이지로 이동
3-C. "미리보기" 클릭 → 오른쪽 사이드 패널에 상세 표시
4. 사이드 패널에서 내용 확인 후 X 닫기 또는 "바로가기"로 전체 페이지 이동
```

### 4.3 메시지 삭제 플로우
```
[1시간 이내]
1. 본인 메시지 호버 → 삭제 아이콘 클릭
2. 드롭다운 표시: "내 대화창에서 삭제" / "모두에게 삭제"
3-A. "내 대화창에서 삭제" 선택 → 확인 팝업 → hideMessage API
3-B. "모두에게 삭제" 선택 → 확인 팝업 → deleteMessage API

[1시간 이후]
1. 본인 메시지 호버 → 삭제 아이콘 클릭
2. 드롭다운 표시: 안내 메시지 + "내 대화창에서 삭제"
3. "내 대화창에서 삭제" 선택 → 확인 팝업 → hideMessage API
```

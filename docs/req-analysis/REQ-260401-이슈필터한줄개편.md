# REQ-이슈필터한줄개편-20260401

## 1. 요구사항 요약
- 대상 화면: /issues
- 목적: 이슈 화면의 좌측 이슈 범위 탭과 우측 ViewScope 탭을 한 줄에 배치
- 목표 레이아웃:
  - 좌측 정렬: [My Involved] [Assigned Me] [My Issues] [All]
  - 우측 정렬: [My Only] [My Unit] [My Cell] [All]
- 진행 방식: 요구사항 정리 → 작업계획서(화면 구성안 포함) → 구현

## 2. AS-IS 현황 분석
- 파일: apps/web/src/domain/issues/components/IssueFilters.tsx
- 현재 구조:
  - 1줄: ViewScopeTab 단독
  - 2줄: Preset 바
  - 3줄: 좌측 이슈 범위 탭(all/my/my_assigned/my_involved)
- 현재 좌측 탭 순서:
  - All → My Issues → Assigned to Me → My Involved
- 현재 우측 ViewScope 매핑:
  - mine -> my
  - unit -> my_unit
  - cell -> my_cell
  - all -> undefined
- API 파라미터:
  - scope 값으로 my, my_assigned, my_involved, my_unit, my_cell, undefined 사용
- 데이터/DB 영향:
  - UI 레이아웃 변경이며 DB 스키마 변경 없음

## 3. TO-BE 요구사항
- 좌측/우측 탭을 동일 행으로 통합
- 좌측 탭 순서를 다음으로 변경
  - My Involved -> my_involved
  - Assigned Me -> my_assigned
  - My Issues -> my
  - All -> undefined
- 우측 ViewScope는 기존 로직 유지하되 같은 행 우측에 배치
- 기존 필터/프리셋/검색/타입칩/드롭다운 동작은 유지

## 4. 갭 분석
- 레이아웃 갭:
  - AS-IS: 좌측/우측 탭이 분리된 줄
  - TO-BE: 좌측/우측 탭 단일 줄
- 순서 갭:
  - AS-IS: All 우선
  - TO-BE: My Involved 우선
- 라벨 갭:
  - AS-IS: Assigned to Me
  - TO-BE: Assigned Me

## 5. 사용자 플로우
1. /issues 진입
2. 상단 필터 첫 줄에서 좌측 범위 탭/우측 ViewScope 탭을 동시에 확인
3. 좌측 탭 클릭 시 scope(my_involved/my_assigned/my/undefined) 즉시 변경
4. 우측 탭 클릭 시 scope(my/my_unit/my_cell/undefined)로 즉시 변경
5. 리스트/칸반/간트 모두 동일 필터 결과를 사용

## 6. 기술 제약사항
- i18n 하드코딩 금지: issues.json 번역키 유지/보완
- 기존 API 파라미터 계약 유지: scope 문자열 변경 없이 순서/레이아웃만 변경
- React Query 캐시 키는 filters 객체 기반이므로 추가 조정 불필요
- 반응형 환경에서 탭이 줄바꿈되어도 좌측/우측 그룹 구조가 유지되어야 함

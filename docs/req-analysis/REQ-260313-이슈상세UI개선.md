## 2. AS-IS 현황 분석

### 2.1 상태 변경 메모란 (R1)

**현재 구현**: `IssueDetailModal.tsx` L572-579
- `<input>` 태그, placeholder: `t('issues:workflow.notePlaceholder')` → *"메모를 입력하세요 (선택)"*
- 워크플로우 버튼과 함께 1줄 텍스트 입력

### 2.2 상태 이력 표시 (R2)

**현재 구현**: `IssueStatusHistory.tsx`
- 형식: `[이전상태] → [변경상태] 변경자이름 - 메모 날짜시간`
- 예: `승인 → 진행 중 Gray.Kim - 상태변경시 메모 2026-03-13 23:08`
- 상태 뱃지 2개(from → to) + 사용자명 + 메모 + 날짜 가로 배치

### 2.3 코멘트 상태 표기 (R3)

**현재 구현**: `IssueDetailModal.tsx` 코멘트 섹션
- `IssueCommentResponse` 타입: `commentId, issueId, authorId, authorName, authorType, content, createdAt`
- **작성 시점의 이슈 상태 미저장** → DB 컬럼 없음
- `IssueCommentEntity`: `isc_id, iss_id, isc_author_id, isc_author_type, isc_content, isc_created_at`

### 2.4 코멘트 첨부 기능 (R4)

**현재 구현**: `MentionInput` 컴포넌트 사용 (일반 textarea 기반 멘션 입력)
- 이미지/파일 첨부 기능 없음
- `RichTextEditor` 컴포넌트는 이슈 설명(description) 필드에만 사용 중
- `RichTextEditor`에는 이미 이미지 드래그앤드롭/붙여넣기 + 파일 업로드 기능 내장

## 3. TO-BE 요구사항

### 3.1 R1: 상태 변경 메모란 플레이스홀더 변경

- **변경 내용**: 플레이스홀더를 `"상태변경시 남기실 메모를 적어주세요"` 로 변경
- **영향 범위**: 번역 파일 3개 (ko, en, vi)
- **변경 최소화**: 번역 키 값만 변경

### 3.2 R2: 상태 이력 표시 방식 변경

**현재**:
```
[승인] → [진행 중]  Gray.Kim - 상태변경시 메모  2026-03-13 23:08
[열림] → [승인]     Gray.Kim                    2026-03-13 23:08
```

**개선**:
```
Gray.Kim 이 상태를 [진행 중]으로 변경  2026-03-13 23:08
 - 상태변경시 메모
Gray.Kim 이 상태를 [열림] 에서 [승인]으로 변경  2026-03-13 23:08
```

- 사용자명을 앞에 배치 (행위 주체 중심)
- 이전 상태+새 상태 모두 표시하되, 자연스러운 문장 형태
- 메모가 있으면 다음 줄에 들여쓰기로 표시
- 번역 키: `"statusChangedTo": "{{user}} 이(가) 상태를 [{{toStatus}}]으로 변경"` 등

### 3.3 R3: 코멘트 저장 시 이슈 상태 표기

- **DB 변경**: `amb_issue_comments` 테이블에 `isc_issue_status` 컬럼 추가 (VARCHAR(20), nullable)
- **엔티티 변경**: `IssueCommentEntity`에 `iscIssueStatus` 필드 추가
- **서비스 변경**: `addIssueComment()` 시 현재 이슈 상태를 함께 저장
- **타입 변경**: `IssueCommentResponse`에 `issueStatus?: string` 필드 추가
- **프론트엔드**: 코멘트 표시 시 상태 뱃지 표시

### 3.4 R4: 코멘트 작성 시 이미지/파일 첨부

- MentionInput → RichTextEditor로 교체하되, 멘션 기능은 유지 필요
- **대안**: MentionInput 유지 + 파일 첨부 버튼/드래그앤드롭 영역 추가
  - 이미지 드래그앤드롭 → 업로드 후 `![alt](url)` 또는 HTML img 태그로 content에 삽입
  - 파일 첨부 → Google Drive 업로드 후 링크 삽입
- RichTextEditor를 코멘트 입력에도 사용하는 것이 가장 일관성 있는 방식

## 5. 기술 제약사항

1. **R3 DB 마이그레이션**: 기존 코멘트의 `isc_issue_status`는 null → 별도 마이그레이션 불필요 (nullable)
2. **R4 멘션 호환**: RichTextEditor로 교체 시 `@멘션` 기능 유지 필요; TipTap에 Mention extension 추가로 해결 가능
3. **R4 코멘트 content 형식 변경**: 기존 plain text → HTML로 변경 시 기존 코멘트 표시 호환성 유지 필요 (dangerouslySetInnerHTML + sanitize)

## 6. 사용자 플로우

### 상태 변경 플로우 (R1, R2)
1. 사용자가 이슈 상세 모달 열기
2. 워크플로우 버튼 영역에서 메모 입력란 확인 → 새 플레이스홀더 표시
3. 메모 입력 후 상태 변경 버튼 클릭
4. 상태 이력 섹션에 새로운 형식으로 이력 추가

### 코멘트 작성 플로우 (R3, R4)
1. 코멘트 입력 영역에서 RichTextEditor 사용
2. 이미지 드래그앤드롭 또는 파일 업로드 버튼으로 파일 첨부
3. 코멘트 등록 시 현재 이슈 상태가 함께 저장
4. 코멘트 목록에서 작성 시점의 상태 뱃지 표시

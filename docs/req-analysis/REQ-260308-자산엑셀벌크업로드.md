**작성일**: 2026-03-08
**요청**: 자산관리 엑셀 벌크 업로드 기능 구현

---

## 1. 요구사항 요약

자산관리(`/assets`) 페이지에 엑셀 파일을 통한 대량 자산 등록 기능을 추가한다.

### 핵심 기능
1. **엑셀 템플릿 다운로드** — 사전 정의된 양식의 엑셀 파일 다운로드
2. **엑셀 벌크 업로드** — 작성된 엑셀 파일을 업로드하여 자산 일괄 등록

## 3. TO-BE 요구사항

### 3.1 엑셀 템플릿 다운로드

| 항목 | 내용 |
|------|------|
| **엔드포인트** | `GET /assets/import/template` |
| **응답** | `.xlsx` 파일 다운로드 |
| **파일명** | `asset_import_template.xlsx` |

**템플릿 구성**:
- **Sheet 1: "자산 데이터"** — 데이터 입력 시트
- **Sheet 2: "입력 가이드"** — 컬럼별 설명 및 허용값

**데이터 시트 컬럼**:

| # | 컬럼명 | 필수 | 설명 | 허용값/형식 |
|---|--------|------|------|-------------|
| A | asset_name | O | 자산명 | 텍스트 (최대 200자) |
| B | asset_category | O | 카테고리 | IT_EQUIPMENT, SUPPLY, FACILITY, MEETING_ROOM |
| C | ownership_type | O | 보유형태 | PURCHASE, LEASE, OTHER |
| D | status | - | 초기 상태 | STORED(기본), IN_USE, RESERVED |
| E | quantity | - | 수량 | 정수 (기본: 1) |
| F | unit | - | 부서/유닛 | 텍스트 |
| G | location | - | 위치 | 텍스트 |
| H | manufacturer | - | 제조사 | 텍스트 |
| I | model_name | - | 모델명 | 텍스트 |
| J | serial_no | - | 시리얼번호 | 텍스트 |
| K | barcode | - | 바코드 | 텍스트 |
| L | purchase_date | - | 구매일 | YYYY-MM-DD |
| M | vendor | - | 공급업체 | 텍스트 |
| N | currency | - | 통화 | USD, KRW, VND |
| O | purchase_amount | - | 구매금액 | 숫자 |
| P | depreciation_years | - | 감가상각년수 | 정수 |
| Q | residual_value | - | 잔존가치 | 숫자 |

> 회의실 관련 필드(room_*)는 단건 등록에서만 지원, 벌크 업로드 대상 제외

### 3.2 엑셀 벌크 업로드

| 항목 | 내용 |
|------|------|
| **엔드포인트** | `POST /assets/import/excel` |
| **요청** | `multipart/form-data` (file 필드) |
| **파일 제한** | `.xlsx` 만 허용, 최대 10MB |
| **최대 행수** | 500건 |

**처리 프로세스**:
1. 파일 수신 및 확장자/크기 검증
2. ExcelJS로 파싱 → 1번째 시트 읽기
3. 헤더 행(1행) 검증 — 필수 컬럼 존재 확인
4. 데이터 행 순회 (2행부터):
   - 필수값 검증 (asset_name, asset_category, ownership_type)
   - 코드값 검증 (category, ownership_type, status, currency)
   - 숫자/날짜 형식 검증
5. 유효한 행 → 자산 코드 자동 생성 → DB 배치 인서트
6. 각 행별 변경이력(CREATE) 자동 기록
7. 결과 리포트 반환

**응답 형식**:
```json
{
  "success": true,
  "data": {
    "totalRows": 50,
    "successCount": 48,
    "failCount": 2,
    "errors": [
      { "row": 5, "field": "asset_category", "message": "Invalid category: 'UNKNOWN'" },
      { "row": 23, "field": "asset_name", "message": "Required field is empty" }
    ]
  }
}
```

### 3.3 프론트엔드 UI

**자산 목록 탭 상단에 버튼 2개 추가**:
1. **템플릿 다운로드** 버튼 (Download 아이콘)
2. **엑셀 업로드** 버튼 (Upload 아이콘) → 모달 오픈

**ExcelImportModal**:
- 파일 드래그&드롭 또는 파일 선택
- `.xlsx` 파일만 수용
- 업로드 진행 상태 표시 (로딩 스피너)
- 결과 표시: 성공/실패 건수 + 에러 상세 목록
- 완료 후 자산 목록 자동 새로고침

## 5. 사용자 플로우

```
1. 자산 목록 탭 → [템플릿 다운로드] 버튼 클릭
   → asset_import_template.xlsx 다운로드

2. 엑셀 파일에 자산 데이터 입력 (최대 500건)

3. 자산 목록 탭 → [엑셀 업로드] 버튼 클릭
   → ExcelImportModal 오픈

4. 파일 선택 또는 드래그&드롭
   → 업로드 시작 (로딩 표시)

5. 결과 확인
   → 성공: "48건 등록 완료"
   → 실패: "2건 실패" + 행번호/필드/사유 표시

6. 모달 닫기 → 자산 목록 자동 새로고침
```

## 6. 기술 제약사항

1. **ExcelJS**: 이미 설치됨, 서버사이드 파싱/생성 모두 지원
2. **파일 크기**: 10MB 제한 (Multer `FileInterceptor`)
3. **행 수 제한**: 500건 (대량 데이터 시 메모리/성능 고려)
4. **자산 코드**: `AST-YYYYMMDD-XXXXX` 자동 생성 (기존 `generateAssetCode()` 메서드 활용)
5. **권한**: ADMIN/SYSTEM_ADMIN/MANAGER만 벌크 업로드 가능
6. **변경이력**: 각 건별 `CREATE` 로그 자동 기록 (기존 패턴 준수)
7. **트랜잭션**: 전체 실패가 아닌 행별 처리 (성공한 행은 저장, 실패한 행은 에러 리포트)
8. **i18n**: 3개 언어 번역 키 추가 필요 (ko, en, vi)

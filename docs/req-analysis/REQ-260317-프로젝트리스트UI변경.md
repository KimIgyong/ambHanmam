# 요구사항 분석서: 프로젝트 리스트 UI 변경

- **작성일**: 2026-03-17
- **요청 URL**: https://stg-ama.amoeba.site/project/projects

---

## 1. AS-IS 현황 분석

### 1.1 레이아웃 구조
- **ProjectLayout.tsx**: 프로젝트/제안 서브메뉴를 두 가지 형태로 제공
  - **모바일**: 상단 가로 스크롤 탭 (border-b)
  - **데스크톱**: 좌측 사이드바 (w-48, border-r) → 콘텐츠 영역이 사이드바만큼 줄어듦
- 서브메뉴 항목: `프로젝트`, `제안서` 2개

### 1.2 프로젝트 리스트 (ProjectListPage.tsx)
- **모바일**: 카드뷰 (space-y-3)
  - 표시 항목: 프로젝트코드, 상태, 프로젝트명, 법인, 카테고리, 매니저, 이슈수, **예산**
- **데스크톱**: 테이블뷰 (table)
  - 컬럼: 코드, 프로젝트명, 법인, 카테고리, 상태, PM, 이슈수, **예산**
- 필터: 검색(텍스트) + 상태(select)
- **+New Proposal 버튼 없음** (제안서 리스트에만 존재)

### 1.3 제안서 리스트 (ProposalListPage.tsx)
- 상단에 `+ 새 제안서` 버튼 있음
- 테이블뷰만 사용 (모바일 카드뷰 미분리)
- 필터: 검색(텍스트) + 상태(select)

### 1.4 라우터 구조
```
/project                    → /project/projects로 리다이렉트
/project/projects           → ProjectListPage (ProjectLayout 내부)
/project/projects/:id       → ProjectDetailPage
/project/proposals          → ProposalListPage (ProjectLayout 내부)
/project/proposals/new      → ProposalFormPage
/project/proposals/:id      → ProposalDetailPage
```

### 1.5 데이터 (ProjectResponse 타입)
- **있는 필드**: projectId, code, name, category, managerName, issueCount, openIssueCount, budget, members?(배열)
- **없는 필드**: `memberCount` (리스트 API에서 members 관계를 로드하지 않음)

## 2. TO-BE 요구사항

### 2.1 서브메뉴 → 탭메뉴 전환
- ProjectLayout의 **좌측 사이드바(데스크톱)** 제거
- 프로젝트/제안서 전환을 **리스트 화면 상단 탭 메뉴**로 이동
- 탭 메뉴 위치: 페이지 헤더 영역 내 (검색/필터 위)
- 콘텐츠 영역이 전체 너비 사용

### 2.2 +New Proposal 버튼 공통 배치
- 현재: 제안서 리스트에만 존재
- 변경: **프로젝트 리스트에도 동일한 위치**에 `+New Proposal` 버튼 배치
- 위치: 탭 메뉴 우측 또는 헤더 우측 (제안 리스트와 동일)

### 2.3 프로젝트 리스트 카드뷰 변경
- **데스크톱 테이블뷰 제거** → 카드뷰로 통일 (데스크톱은 그리드 레이아웃)
- 카드뷰 표시 항목 (변경):
  | 항목 | 포함 여부 |
  |------|----------|
  | 프로젝트명 | ✅ |
  | 프로젝트코드 | ✅ |
  | 카테고리 | ✅ |
  | 매니저(PM) | ✅ |
  | 멤버수 | ✅ (신규) |
  | 이슈건수 | ✅ |
  | **예산** | ❌ 제거 |
  | 법인 | ❌ 제거 |
  | 상태 | 유지 (뱃지) |

### 2.4 멤버수 표시 (신규)
- ProjectResponse에 `memberCount` 필드가 없음
- 백엔드: findAll 쿼리에 memberCount 서브쿼리 추가 필요
- 프론트엔드: 카드에 멤버수 표시

## 3. 갭 분석

| 영역 | AS-IS | TO-BE | 갭 |
|------|-------|-------|-----|
| 네비게이션 | 사이드바 + 모바일 탭 | 상단 탭 (통합) | ProjectLayout 개편 |

| 카드 데이터 | 예산 포함 | 예산 제거, 멤버수 추가 | 카드 항목 변경 |
| 멤버수 | 미제공 | 카드에 표시 | 백엔드 서브쿼리 + 타입 추가 |

## 4. 사용자 플로우

### 변경 후 플로우
1. 사용자가 `/project`에 접근 → `/project/projects`로 리다이렉트
2. **상단에 [프로젝트] [제안서] 탭** + 우측에 [+New Proposal] 버튼 표시
3. 프로젝트 탭 선택: 카드뷰 그리드로 프로젝트 목록 표시
4. 제안서 탭 선택: 기존 제안서 리스트 표시
5. 카드 클릭 시 프로젝트 상세 페이지로 이동

---

## 5. 기술 제약사항

### 5.1 백엔드 변경 필요
- `memberCount` 서브쿼리를 `findAll`에 추가해야 함
- `ProjectResponse` 타입에 `memberCount` 필드 추가 필요

### 5.2 프론트엔드 구조 변경
- ProjectLayout.tsx: 사이드바 제거 → 단순 Outlet 랩퍼로 변경
- ProjectListPage.tsx: 탭 메뉴 포함, 카드뷰 전면 재구성
- 또는: 탭 메뉴를 ProjectLayout에 유지하되 사이드바→상단탭으로 변환

### 5.3 라우팅
- 라우터 경로 변경 불필요 (동일한 구조 유지)
- ProjectLayout이 여전히 부모 레이아웃 역할

### 5.4 i18n
- 추가 번역 키: `project.memberCount` (ko/en/vi)

## 2. AS-IS 현황 분석

### 2.1 현재 알림 시스템

| 구성 요소 | 현재 상태 |
|----------|----------|
| **NotificationBell** | 헤더 영역 벨 아이콘, 클릭 시 드롭다운으로 최근 10건 표시 |
| **NotificationModal** | SSE로 수신 시 화면 상단 토스트 팝업 (5초 후 자동 닫힘) |
| **Notification API** | `GET /notifications` (페이지네이션), `GET /notifications/unread-count` |
| **Notification SSE** | `/api/v1/notifications/stream` - 실시간 스트림 |
| **알림 유형** | TODO_ASSIGNED, ISSUE_ASSIGNED, ISSUE_STATUS_CHANGED, NOTE_ASSIGNED, CALENDAR_INVITED, COMMENT_MENTION, TALK_MESSAGE |

**현재 문제점:**
- 알림은 헤더 벨 드롭다운(최대 10건)으로만 확인 가능
- 과거 알림을 스크롤하여 전체 조회할 수 있는 UI 없음
- 알림 → 해당 리소스 이동만 가능, 알림 자체를 대화형으로 확인 불가

### 2.2 현재 Amoeba Talk 채널 구조

| 섹션 | 내용 | 순서 |
|------|------|------|
| **채널 (Group)** | PUBLIC/PRIVATE 채널, 접기/펼치기 | 1 |
| **다이렉트 메시지** | 1:1 DM 채널, 접기/펼치기 | 2 |
| **보관됨** | 아카이브된 채널 | 3 |

- ChannelList.tsx에서 채널을 `groupChannels`, `dmChannels`, `archivedChannels`로 분류
- 채널 선택 시 `setCurrentChannelId(channelId)` → AmoebaTalkPage에서 메시지 영역 렌더링

## 3. TO-BE 요구사항

### 3.1 핵심 기능

**"알림 센터" 가상 채널을 ChannelList 최상단에 배치하여, 사용자가 받은 모든 알림을 대화 형식으로 확인할 수 있는 뷰 제공**

### 3.2 상세 요구사항

| 항목 | 설명 |
|------|------|
| **위치** | ChannelList 최상단 (채널 섹션, DM 섹션보다 위) |
| **아이콘** | 🔔 벨 아이콘 (Bell) |
| **이름** | "알림 센터" (ko) / "Notifications" (en) / "Thông báo" (vi) |
| **미읽음 배지** | 읽지 않은 알림 수 표시 (기존 NotificationBell과 동일한 unreadCount) |
| **채널 아이템 미리보기** | 최신 알림의 title을 lastMessage처럼 미리보기 표시 |
| **선택 시 표시** | 채팅 메시지 형식으로 알림 목록 표시 (무한 스크롤) |
| **알림 아이템** | 타입 아이콘 + 제목 + 상세 메시지 + 시간 + 읽음 상태 |
| **클릭 동작** | 알림 아이템 클릭 시 해당 리소스 페이지로 이동 |
| **읽음 처리** | 알림 센터 진입 시 보이는 항목 자동 읽음 처리 (또는 클릭 시) |
| **전체 읽음** | "모두 읽음" 버튼 제공 |
| **실시간 업데이트** | 기존 Notification SSE를 통해 새 알림 실시간 반영 |

## 5. 기술적 접근 방식

### 5.1 가상 채널 방식 (프론트엔드 전용)

- DB에 실제 채널 생성 **안 함** (NOTIFICATION 타입 채널 불필요)
- 프론트엔드에서 예약된 가상 ID (`__NOTIFICATION_CENTER__`) 사용
- `currentChannelId === '__NOTIFICATION_CENTER__'`일 때 NotificationChatView 렌더링
- 기존 Talk 메시지/채널 API와 완전 분리, Notification API만 사용

### 5.2 구현 범위

| 구분 | 작업 | 변경 유형 |
|------|------|----------|
| **프론트 컴포넌트** | `NotificationChannelItem` - 채널 리스트 아이템 | 신규 |
| **프론트 컴포넌트** | `NotificationChatView` - 알림 대화 뷰 | 신규 |
| **프론트 수정** | `ChannelList.tsx` - 최상단에 알림 채널 추가 | 수정 |
| **프론트 수정** | `AmoebaTalkPage.tsx` - 가상 채널 조건부 렌더링 | 수정 |
| **프론트 수정** | `talk.store.ts` - 가상 채널 ID 상수 추가 | 수정 |
| **프론트 훅** | `useNotifications` - notification API 페이지네이션 훅 | 신규 |
| **i18n** | talk.json에 알림 센터 키 추가 (ko/en/vi) | 수정 |
| **백엔드** | 변경 없음 (기존 Notification API로 충분) | - |

### 5.3 사이드 임팩트

- **ChannelList**: 기존 채널 rendering 로직 변경 없음, 새 섹션 추가만
- **AmoebaTalkPage**: `currentChannelId` 분기 처리 추가 (가상 채널 여부 판단)
- **MemberPanel**: 알림 센터에서는 숨김 처리
- **SSE**: 기존 Notification SSE 그대로 사용, Talk SSE와 독립
- **NotificationBell**: 기존 동작 유지, unreadCount 공유

## 6. 사용자 플로우

1. 사용자가 로비 채팅(Amoeba Talk) 페이지 진입
2. ChannelList 최상단에 "🔔 알림 센터" 표시 (미읽음 배지 포함)
3. 알림 센터 클릭 → 채팅 영역에 알림 목록이 대화 형식으로 표시
4. 각 알림 아이템 클릭 → 해당 리소스 페이지로 이동 (할일, 이슈 등)
5. "모두 읽음" 클릭 → 모든 알림 읽음 처리
6. 실시간으로 새 알림 수신 시 목록 상단에 추가

---

## 7. 기술 제약사항

- Talk SSE는 채널별 구독이므로, 알림 센터에서는 Notification SSE만 사용
- 알림 센터는 읽기 전용 (메시지 입력 불가)
- 기존 ChannelHeader 대신 专用 헤더 필요 (알림 센터 전용)

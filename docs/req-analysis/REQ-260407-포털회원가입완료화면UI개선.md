> **연관 문서**: `REQ-260406-포털회원가입플로우개선.md`, `PLN-260406-포털회원가입플로우개선.md`

## 1. 요구사항 요약 (Requirement Summary)

포털 회원가입 완료 화면에서 **생성된 회사명/회사코드를 표시**하고, AMA 로그인 버튼 클릭 시 **첫 로그인 비밀번호 변경 화면으로 자연스럽게 이동**되도록 개선한다.

### 핵심 변경 사항
| # | 항목 | 설명 |
|---|------|------|
| 1 | 완료 화면에 회사 정보 표시 | 생성된 회사명(Entity Name) + 회사코드(Entity Code) 확인 가능 |
| 2 | AMA 로그인 URL에 Entity Code 포함 | `ama.amoeba.site/{entityCode}/login`으로 entity 자동 선택 |
| 3 | 이메일 템플릿에 Entity Code 추가 | ACCOUNT_CREATED 이메일에 회사코드 포함 + 로그인 URL 개선 |

### 비밀번호 변경 플로우 (기존 구현 확인)
- AMA 웹앱의 `AuthGuard`가 `mustChangePw === true`인 사용자를 자동으로 `/force-change-password`로 리다이렉트
- `ForceChangePasswordPage`에서 임시 비밀번호 → 새 비밀번호 변경 후 `/today`로 이동
- **추가 구현 불필요** — entity code만 로그인 URL에 포함하면 기존 메커니즘 작동

## 2. AS-IS 현황 분석 (Current State Analysis)

### 2.1 포털 등록 API 응답

**파일**: `apps/portal-api/src/domain/auth/service/portal-auth.service.ts`

현재 register() 응답:
```json
{
  "success": true,
  "data": {
    "customerId": "UUID",
    "email": "user@company.com",
    "name": "John Doe",
    "emailVerified": true
  }
}
```

- ❌ `entityCode` (회사코드) 미포함
- ❌ `entityName` (회사명) 미포함
- ❌ `companyName` (입력한 회사명) 미포함

### 2.2 Auto-Provision 흐름

**파일**: `apps/portal-api/src/domain/auth/service/portal-auth.service.ts` → `triggerAutoProvision()`

```
포털 register → triggerAutoProvision(pctId) → POST ama-api/portal-bridge/customers/{pctId}/auto-provision
```

- 동기 호출 (8초 타임아웃)
- AMA API에서 Entity 생성 → User 생성 → Mapping 생성
- **현재**: triggerAutoProvision()의 응답을 무시하고 있음 (response.ok만 체크)
- **AMA API 응답에 포함되는 데이터**: `{ userId, email, name, role, entityId, entityName, mappingId }`
- ❌ AMA API 응답에도 `entityCode` 미포함

### 2.3 포털 완료 화면 (SignupCompleteScreen)

**파일**: `apps/portal-web/src/components/signup/SignupCompleteScreen.tsx`

- 일반적인 성공 메시지 + 서비스 카드 표시
- ❌ 회사명/회사코드 표시 없음
- AMA CTA 버튼: `ama.amoeba.site/user/login` (일반 로그인 페이지)
  - entity code 없이 이동 → 사용자가 entity 검색/선택 필요

### 2.4 Signup Store

**파일**: `apps/portal-web/src/stores/signup.store.ts`

- 사용자 입력 데이터만 저장 (fullName, email, password, companyName 등)
- ❌ API 응답 데이터 (entityCode, entityName) 저장 안 함

### 2.5 AMA 로그인 페이지 (이미 구현됨)

| 경로 | 기능 | 상태 |
|------|------|------|
| `/user/login` | 일반 로그인 (entity 선택 필요) | ✅ query params `entityCode`, `entityName` 지원 |
| `/{entCode}/login` | Entity 코드 기반 직접 로그인 | ✅ URL path param으로 entity 자동 조회 |
| `/force-change-password` | 비밀번호 강제 변경 | ✅ AuthGuard에서 mustChangePw=true 시 자동 리다이렉트 |

### 2.6 이메일 템플릿 (ACCOUNT_CREATED)

**파일**: `apps/api/src/domain/settings/service/email-template.service.ts`

현재 변수:
```
userName, userEmail, tempPassword, loginUrl, entityName, role
```

- ❌ `entityCode` 변수 없음
- `loginUrl` = `${frontendUrl}/login` (entity code 미포함)

### 2.7 Auto-Provision 임시 비밀번호

**파일**: `apps/api/src/domain/portal-bridge/service/portal-bridge.service.ts`

- 임시 비밀번호 12자 생성 (`uuidv4().slice(0, 12)`)
- `usrMustChangePw: true` 설정
- 이메일로 임시 비밀번호 발송
- ❌ entityCode를 이메일 변수에 미포함

## 5. 사용자 플로우 (User Flow)

### 5.1 TO-BE 전체 플로우

```
1. 포털 회원가입 (Step 1 계정 → Step 2 회사정보)
   ↓
2. 포털 API 등록
   ├─ PortalCustomer + SvcClient 생성
   └─ triggerAutoProvision → AMA API 호출
       ├─ Entity 생성 (entCode: VN1234)
       ├─ User 생성 (usrMustChangePw: true)
       ├─ 임시 비밀번호 이메일 발송 (entityCode 포함)
       └─ 응답: { entityCode: 'VN1234', entityName: 'ABC Trading', ... }
   ↓
3. 포털 API 응답
   { customerId, email, name, emailVerified, entityCode: 'VN1234', entityName: 'ABC Trading' }
   ↓
4. 완료 화면 표시
   ├─ 회사명: ABC Trading
   ├─ 회사코드: VN1234
   ├─ "이 코드를 저장해 두세요" 안내
   └─ [AMA 로그인하기 →] 버튼
   ↓
5. AMA 로그인 (ama.amoeba.site/VN1234/login)
   ├─ Entity 자동 선택됨 (코드로 조회)
   ├─ 이메일 + 임시 비밀번호 입력
   └─ 로그인 성공 → JWT에 mustChangePw: true
   ↓
6. 비밀번호 변경 (자동 리다이렉트)
   ├─ AuthGuard → /force-change-password
   ├─ 현재 비밀번호(임시) + 새 비밀번호 입력
   ├─ usrMustChangePw: false 업데이트
   └─ /today (대시보드)로 이동
```

## 6. 기술 제약사항

| 항목 | 제약 | 대응 |
|------|------|------|
| Auto-Provision 타임아웃 | 8초 → entity 생성 실패 시 entityCode 없음 | 실패 시 entityCode 없이 기존 화면 표시 (graceful degradation) |
| Entity Code 생성 규칙 | `{KR|VN}{4자리}` (최대 9000개/국가) | 현행 유지 |
| 이메일 발송 실패 | 임시 비밀번호 이메일 미수신 가능 | 완료 화면에 "이메일을 확인하세요" 안내 강화 |
| 기존 가입자 | 이미 가입 완료된 사용자는 이 플로우 미적용 | 신규 가입만 대상 |

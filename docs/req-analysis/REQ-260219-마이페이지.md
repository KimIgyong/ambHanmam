# REQ-마이페이지 요구사항 분석서

**작성일**: 2026-02-19
**요청자**: 관리자
**우선순위**: 높음

---

## 1. 배경 및 문제

현재 시스템에서 일반 사용자(USER 역할)가 로그인하면 본인 정보를 확인하거나 관리할 수 있는 페이지가 없음.
- Settings 페이지는 ADMIN/MANAGER 전용 메뉴로 구성됨
- 사용자가 본인 비밀번호를 변경할 수 있는 UI가 없음
- 소속 부서, 법인 정보를 확인할 수 있는 화면이 없음

## 2. 요구사항 목록

### 2.1 마이페이지 신설 (필수)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| R-01 | 모든 로그인 사용자가 접근 가능한 마이페이지 | 필수 |
| R-02 | 본인 프로필 정보 조회 (이름, 이메일, 역할, 가입일) | 필수 |
| R-03 | 소속 부서 정보 조회 (부서명, 부서 내 역할, 주요 부서 여부) | 필수 |
| R-04 | 소속 법인 정보 조회 (법인명, 법인코드, 국가, 법인 내 역할) | 필수 |
| R-05 | 비밀번호 변경 기능 (현재 비밀번호 확인 → 새 비밀번호 설정) | 필수 |
| R-06 | 사이드바 또는 헤더에서 마이페이지 접근 가능 | 필수 |

### 2.2 추가 고려사항 (선택)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| R-07 | 서명 이미지 관리 (기존 Settings에서 이관) | 선택 |
| R-08 | 회사 이메일 정보 표시 | 선택 |

## 3. 기존 시스템 분석

### 3.1 이미 구현된 백엔드 기능

| 기능 | 엔드포인트 | 상태 |
|------|-----------|------|
| 비밀번호 변경 | `POST /auth/change-password` | 구현 완료 |
| 서명 업로드 | `POST /users/me/signature` | 구현 완료 |
| 서명 조회 | `GET /users/me/signature` | 구현 완료 |
| 서명 삭제 | `DELETE /users/me/signature` | 구현 완료 |

### 3.2 추가 개발 필요한 백엔드 기능

| 기능 | 엔드포인트 | 설명 |
|------|-----------|------|
| 내 프로필 조회 | `GET /users/me/profile` | 부서, 법인 관계 포함 상세 정보 |

### 3.3 관련 데이터 구조

**UserEntity** (`amb_users`):
- `usr_id`, `usr_email`, `usr_name`, `usr_department`, `usr_role`
- `usr_company_email`, `usr_created_at`

**UserDeptRoleEntity** (`amb_user_dept_roles`):
- `usr_id` → `dep_id` (부서 연결)
- `udr_role` (MEMBER, TEAM_LEAD, DEPARTMENT_HEAD)
- `udr_is_primary` (주요 부서 여부)

**EntityUserRoleEntity** (`amb_hr_entity_user_roles`):
- `usr_id` → `ent_id` (법인 연결)
- `eur_role`, `eur_status`

**DepartmentEntity** (`amb_departments`):
- `dep_name`, `ent_id` (소속 법인)

**HrEntityEntity** (`amb_hr_entities`):
- `ent_code`, `ent_name`, `ent_country`, `ent_currency`

### 3.4 JWT Payload (현재)

```typescript
interface UserPayload {
  userId: string;
  email: string;
  role: string;
  departmentId?: string;
  departmentRole?: string;
  entityId?: string;
}
```

### 3.5 기존 ChangePasswordRequest DTO

```typescript
{
  current_password: string;  // 현재 비밀번호 (8자 이상)
  new_password: string;      // 새 비밀번호 (8자 이상)
}
```

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 접근성 | 모든 로그인 사용자 접근 가능 (MenuGuard 제외) |
| i18n | ko, en, vi 3개 언어 지원 |
| 보안 | 비밀번호 변경 시 현재 비밀번호 검증 필수 |
| UX | 사이드바가 아닌 헤더 영역에서 접근 (사용자 이름 클릭) |

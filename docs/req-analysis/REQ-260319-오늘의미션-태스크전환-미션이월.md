## 1. 요구사항 원문

> 기능개선 - 오늘의미션에서 하루 지난후 어제의 미션을 태스크로 지정하기와 오늘의 미션으로 옮기기  
> 오늘의 미션으로 옮기기 하면 바로 하단에 오늘의 미션 입력칸으로 이동해야함

## 2. AS-IS (현재 구현 상태)

### 2.1 미션 체크 플로우

| 단계 | 동작 | 구현 상태 |
|------|------|-----------|
| Step 1 | 어제 미션 존재 & 미체크 시, 라인별 체크 UI 표시 | ✅ 구현됨 |
| Step 2 | 각 라인: [완료] / [미완료] / [해당없음] 선택 | ✅ 구현됨 |
| Step 3 | 미완료 선택 시: [오늘 미션으로] / [Task 등록] 서브 선택 | ✅ 구현됨 |
| Step 4 | 저장 시 → Done=COMPLETED Todo, Task=IN_PROGRESS Todo, Mission=carry-over text | ✅ 구현됨 |
| Step 5 | carry-over 항목 → 오늘 미션 에디터에 pre-fill | ✅ 구현됨 |

### 2.2 현재 제약사항

1. **어제 미션만 체크 가능**: `getYesterdayUncheckedMission()`이 정확히 "어제"만 조회
   - 금요일에 미션 작성 후 월요일에 출근하면 → 해당 미션 미표시 (하루 넘김)
   - 공휴일/휴가 후 복귀 시 → 미작성 미션 누락

2. **"오늘 미션으로 옮기기" 후 UX 미흡**:
   - 미션 체크에서 [오늘 미션으로] 선택 → 저장 후 carry-over text만 DB에 저장됨
   - 오늘 미션 에디터로 **자동 스크롤 미구현** → 사용자가 수동으로 아래로 이동해야 함
   - carry-over 항목이 오늘 미션 에디터에 자동 반영되는 흐름이 불분명함

3. **미완료 미션의 추적 불가**:
   - 하루 이상 지난 미체크 미션 → 보이지 않음 → 방치됨
   - 미션 항목별 처리 상태 추적 (어떤 것이 task가 되었는지, 어떤 것이 이월되었는지) 불명확

### 2.3 관련 코드 구조

```
── 백엔드 ──
apps/api/src/domain/today/service/mission.service.ts
  ├─ saveMission()           : 미션 UPSERT + 스냅샷
  ├─ saveCheck()             : 어제 미션 라인별 체크 + Todo 자동 생성
  ├─ getTodayMission()       : 오늘 미션 조회
  └─ getYesterdayUncheckedMission()  : ★ 어제만 조회 (제약!)

apps/api/src/domain/today/controller/today.controller.ts
  └─ GET /today/me : getMyToday + getTodayMission + getYesterdayUncheckedMission

── 프론트엔드 ──
apps/web/src/domain/today/components/mission/MissionSection.tsx
  ├─ Step 1: 어제 미션 라인별 체크 (상단)
  ├─ Step 2: 오늘 미션 작성 에디터 (하단)
  └─ carry-over pre-fill (useEffect)

apps/web/src/domain/today/components/MyTodayPanel.tsx
  └─ <MissionSection> 렌더링
```

## 3. TO-BE (개선 요구사항)

### 3.1 기능 요구사항

#### FR-01: 미체크 미션의 기간 확장 ("하루 지난후")
- **현재**: 정확히 어제(1일 전) 미션만 체크 가능
- **개선**: 가장 최근의 미체크 미션을 자동 감지하여 표시
  - 금요일 미션 → 월요일에도 체크 가능
  - 2일 이상 미체크 미션이 있으면 가장 최근 것 표시
  - 범위 제한: 최대 7일 이내의 미체크 미션만 대상

#### FR-02: "오늘의 미션으로 옮기기" 후 자동 스크롤
- **현재**: 미션 체크 저장 후 오늘 미션 에디터로 수동 이동
- **개선**: 
  1. 미션 체크에서 "오늘 미션으로" 선택 항목이 있는 상태에서 저장 완료 시
  2. 어제 미션 체크 섹션이 접히고(또는 완료 표시되고)
  3. 오늘 미션 에디터 영역으로 **자동 스크롤** (smooth scroll)
  4. carry-over 항목이 에디터에 자동 pre-fill된 상태로 표시

#### FR-03: 미션 체크 버튼 라벨의 접근성 개선 (보조)
- "태스크로 지정하기" / "오늘의 미션으로 옮기기" 문구를 더 직관적으로
  - 현재: "오늘 미션으로" / "Task 등록"
  - 개선: 요구사항 원문의 표현과 일치하도록 검토 (현재 이미 적절)

### 3.2 비기능 요구사항

| ID | 요구사항 | 설명 |
|----|----------|------|
| NFR-01 | 하위 호환 | 기존 체크 완료된 미션 데이터에 영향 없어야 함 |
| NFR-02 | 성능 | 미체크 미션 조회 시 7일 이내만 탐색 (무제한 조회 방지) |
| NFR-03 | UX 일관성 | 자동 스크롤은 smooth scroll, 500ms 이내 |

## 5. 사용자 플로우 (TO-BE)

### 5.1 시나리오: 금요일 미션 → 월요일 체크

```
1. 월요일 출근 → /today 접속
2. 상단 "미체크 미션" 섹션 표시 (금요일 날짜 표시: "2026-03-13")
3. 라인별 체크:
   - 항목 A: [완료] → Done 배지
   - 항목 B: [미완료] → [Task로 지정하기] → Task 배지
   - 항목 C: [미완료] → [오늘 미션으로 옮기기] → 이월 배지
4. [저장] 클릭
5. 미션 체크 섹션 → 완료 표시 (접힘)
6. ★ 화면이 자동으로 하단 "오늘의 미션" 에디터로 스크롤
7. 에디터에 항목 C가 pre-fill 된 상태
8. 사용자가 추가 미션을 작성 후 [저장]
```

### 5.2 시나리오: 어제 미션 → 오늘 태스크 전환

```
1. /today 접속 → 어제 미체크 미션 표시
2. 항목 X: [미완료] → [Task로 지정하기]
3. [저장] → 항목 X가 `amb_todos` 테이블에 IN_PROGRESS 상태로 생성
4. 하단 Tasks 섹션에 즉시 반영
```

## 6. 기술 제약사항

1. **`getYesterdayUncheckedMission` → `getLatestUncheckedMission` 변경 시**:
   - 기존 API 응답 구조 (`yesterdayMission` 필드) 유지 필요 (프론트 호환)
   - 쿼리: `msnDate < today AND msnContent IS NOT NULL AND msnCheckResult IS NULL ORDER BY msnDate DESC LIMIT 1`
   - 최대 7일 제한 필요

2. **자동 스크롤 구현 시**:
   - `ref`로 오늘 미션 에디터 영역 참조 필요
   - `scrollIntoView({ behavior: 'smooth', block: 'start' })` 사용
   - mutation `onSuccess` 콜백에서 invalidateQueries 후 스크롤 실행
   - 쿼리 무효화 → 리렌더 → 에디터 마운트 후 스크롤 (타이밍 주의)

3. **carry-over pre-fill 타이밍**:
   - `saveMissionCheck` 응답에 `carryOverText` 포함됨
   - mutation 성공 후 `setContent(carryOverText)` 직접 설정 가능
   - 또는 쿼리 무효화 후 새 데이터의 `carryOverText`로 자동 반영 (현재 방식)

# REQ-노트기능개선-20260403

## 1. 요구사항 요약

노트(회의록/메모) 모듈에 대한 3가지 기능 개선:

| # | 요구사항 | 구현 현황 |
|---|---------|----------|
| 1 | 노트 작성/수정 시 "이슈로 등록" 체크박스 + 프로젝트 선택 → 이슈 자동 생성 | ❌ 신규 구현 필요 |
| 2 | 담당자/관련자 지정 시 알림 발송 | ✅ **이미 구현됨** |
| 3 | 노트 코멘트 멘션 기능 | ⚠️ 메인 댓글은 구현됨, **답글에 멘션 누락** |

## 2. AS-IS 현황 분석

### 2.1 이슈로 등록 관련 현황
- 노트 폼에 **관련 이슈(Related Issues)** 멀티셀렉트 존재 → **기존 이슈**를 검색/연결하는 용도
- 조인 테이블: `amb_meeting_note_issues` (mtn_id ↔ iss_id)
- `syncIssues()` 메서드로 연결 관계 동기화
- **이슈 신규 생성 기능은 없음** — 기존 이슈 링크만 가능

### 2.2 유사 패턴: Todo → Issue 전환
- `CreateIssueRequest`에 `source_todo_id`, `source_todo_title` 필드 존재
- Issue 생성 시 시스템 코멘트로 출처 기록: `"Converted from Task: {title}"`
- 패턴: DTO 전용 필드 + 시스템 코멘트 (엔티티 컬럼 미추가)

### 2.3 담당자/관련자 알림 현황 (✅ 이미 구현)
| 시점 | 구현 위치 | 동작 |
|------|----------|------|
| 노트 **생성** 시 | `meeting-note.service.ts` L380-402 | 담당자 + 참여자에게 `ASSIGNEE_EVENT` 발행 |
| 노트 **수정** 시 | `meeting-note.service.ts` L471-500 | **새로 추가된** 담당자/참여자에게만 `ASSIGNEE_EVENT` 발행 |
| 이벤트 타입 | `NOTIFICATION_TYPE.NOTE_ASSIGNED` | 알림 시스템에서 처리 |

### 2.4 코멘트 멘션 현황 (⚠️ 부분 구현)
| 구분 | 백엔드 | 프론트엔드 |

## 3. TO-BE 요구사항

### 3.1 [신규] 이슈로 등록 기능

**UI (MeetingNoteFormModal)**:
- 기존 "관련 이슈" 셀렉트 위에 "이슈로 등록" 체크박스 섹션 추가
- 체크 시: 프로젝트 단일 선택(필수) + 이슈 타입(TASK 기본) + 심각도(MINOR 기본)

**백엔드**:
- DTO: `create_issue`, `create_issue_project_id`, `create_issue_type`, `create_issue_severity` 추가
- 서비스: `create_issue=true` → `IssueService.createIssue()` → 노트에 자동 연결 + 시스템 코멘트

### 3.2 [확인완료] 담당자/관련자 알림
- ✅ 이미 구현 — 추가 작업 불필요
- 생성 시: 담당자 + 참여자 전체에게 알림
- 수정 시: 새로 추가된 담당자/참여자에게만 알림

### 3.3 [수정] 답글 멘션 기능
- `NoteCommentSection.tsx` 답글 `RichTextEditor`에 `enableMention` prop 추가
- 백엔드는 이미 처리됨 (동일 `addComment` 메서드)

## 5. 사용자 플로우

### 5.1 이슈로 등록 (작성 시)
1. 노트 작성 폼 열기 → 제목/내용 입력
2. ☑ "이슈로 등록" 체크 → 프로젝트 선택
3. (선택) 이슈 타입/심각도 변경
4. 저장 → 노트 + 이슈 동시 생성 + 자동 연결

### 5.2 이슈로 등록 (수정 시)
1. 기존 노트 수정 → ☑ "이슈로 등록" 체크
2. 프로젝트 선택 → 저장
3. 노트 수정 + 새 이슈 생성 + 연결

### 5.3 답글 멘션
1. 댓글에 답글 입력 시 `@` 입력
2. 멤버 목록 드롭다운 표시 → 선택
3. 답글 저장 → 멘션된 사용자에게 알림 발송

1. **모듈 의존성**: `MeetingNotesModule`에서 `IssueService` 사용 → `IssuesModule` import 필요 (순환 의존성 없음 확인)

3. **트랜잭션**: 노트 저장 + 이슈 생성 + 연결을 하나의 흐름으로 처리
4. **권한**: IssueService.createIssue() 호출 시 동일 사용자/법인 ID 전달

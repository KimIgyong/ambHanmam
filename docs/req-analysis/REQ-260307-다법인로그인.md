# 요구사항 분석서: 다법인 소속 사용자 로그인 시 법인 선택

- **문서번호**: REQ-다법인로그인-20260307
- **작성일**: 2026-03-07

---

## 1. 배경

동일 이메일(`fremd@naver.com`)이 포털 브릿지를 통해 AAA 법인의 MASTER로 등록되고, 이후 BBB 법인에서 직원으로 초대받는 경우, 로그인 시 어떤 법인 컨텍스트로 진입할지 선택이 필요하다.

---

## 2. AS-IS 현황

### 2.1 현재 로그인 플로우

```
LoginPage → POST /auth/login → navigate('/') → MainLayout
  └─ MainLayout 마운트 시 GET /hr/entities → 엔티티 목록 로드
     └─ setEntities() → 첫 번째 엔티티 자동 선택
        └─ EntitySelector 드롭다운 (헤더, 2개 이상일 때만 표시)
```

### 2.2 문제점

| # | 문제 | 설명 |
|---|------|------|
| 1 | 암묵적 법인 선택 | 첫 번째 법인이 자동 선택되어 사용자가 인지 못함 |
| 2 | register 링크 불필요 | 매니지먼트 사이트는 초대/포털 브릿지로만 가입해야 함 |
| 3 | 법인 전환 인지 어려움 | 헤더 드롭다운만으로는 현재 법인 컨텍스트 인식이 약함 |

---

## 3. TO-BE 요구사항

### 3.1 로그인 페이지

- register 링크 주석처리 (초대 기반 가입만 허용)

### 3.2 로그인 후 법인 선택 플로우

```
LoginPage → POST /auth/login
  ├─ 소속 법인 1개 → 바로 대시보드 진입 (기존과 동일)
  └─ 소속 법인 2개 이상 → EntitySelectPage 표시
     └─ 법인 카드 선택 → 대시보드 진입
```

### 3.3 EntitySelectPage 화면 구성

- 법인 카드 리스트 (법인명, 국가 플래그, 법인코드, 역할 표시)
- 카드 클릭 시 해당 법인 선택 후 대시보드 이동
- 로그아웃 버튼 제공

---

## 4. 기술 구현 방향

| 항목 | 방안 |
|------|------|
| 법인 선택 시점 | 로그인 성공 후, 대시보드 진입 전 |
| 라우팅 | `/select-entity` 페이지 추가 (인증 필수, AuthGuard 내부) |
| 선택 판단 | MainLayout 또는 AuthGuard에서 엔티티 미선택 상태 감지 |
| 기존 호환 | 단일 법인 사용자는 기존과 동일하게 동작 |

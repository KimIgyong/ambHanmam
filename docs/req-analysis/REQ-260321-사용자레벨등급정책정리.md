# AMA 사용자 레벨 및 등급 정책 분석서

**작성일:** 2026-03-21
**문서 유형:** 정책 분석 / AS-IS vs TO-BE 비교

---

## 1. 개요

AMA 시스템의 사용자 레벨 및 등급 체계를 정책 기준으로 정의하고, 현재 구현 상태와 비교 분석한다.

---

## 2. 정책 정의 — 레벨 및 등급 체계

### 2.1 레벨 구조 (4개 레벨)

```
┌─────────────────────────────────────────────────────────────────┐
│                    기본 레벨 (2개)                                │
├──────────────────────────────┬──────────────────────────────────┤
│       ADMIN_LEVEL            │          USER_LEVEL              │
│  (서비스 개발/운영/관리)       │   (서비스 이용자)                  │
├──────────────────────────────┼──────────────────────────────────┤
│     서브 레벨 (2개)           │         서브 레벨 (2개)            │
├──────────────────────────────┼──────────────────────────────────┤
│      PARTNER_LEVEL           │        CLIENT_LEVEL              │
│  (개발/운영/마케팅 협력사)      │   (법인 고객사 직원)              │
└──────────────────────────────┴──────────────────────────────────┘
```

### 2.2 레벨별 정의

| 레벨 | 구분 | 정의 | 소속 |
|------|------|------|------|
| **ADMIN_LEVEL** | 기본 | AMA 서비스 전체를 개발, 관리, 운영, 설정하는 HQ 스태프 | 본사 (ROOT) |
| **PARTNER_LEVEL** | 서브 (ADMIN 하위) | AMA 서비스 운영을 지원하는 개발/운영/마케팅 협력사 | 파트너사 |
| **USER_LEVEL** | 기본 | 포털사이트(www.amoeba.site)를 통해 가입한 아메바 서비스 이용자 | 하위 법인 (SUBSIDIARY) |
| **CLIENT_LEVEL** | 서브 (USER 하위) | USER_LEVEL Entity에 속한 고객사 직원 | 고객사 |

### 2.3 레벨별 등급 (총 8개 + 2개 예정)

| 레벨 | 등급 | 설명 | 데이터 범위 |
|------|------|------|------------|
| **ADMIN_LEVEL** | SUPER_ADMIN | 시스템 최고 관리자, 전체 권한 | ALL |
| | ADMIN | 운영 관리자, 일부 설정 제외 | ALL |
| **PARTNER_LEVEL** | PARTNER_ADMIN | 파트너사 관리자 (미개발) | PARTNER_ORG |
| | PARTNER_MEMBER | 파트너사 일반 직원 (미개발) | PARTNER_ORG |
| **USER_LEVEL** | MASTER | 법인 관리자, 법인 내 전체 관리 | OWN_ORG |
| | MANAGER | 부서/팀 관리자 | OWN_ORG |
| | MEMBER | 일반 직원 | OWN_ORG |
| | VIEWER | 읽기 전용 | OWN_ORG |
| **CLIENT_LEVEL** | CLIENT_ADMIN | 고객사 관리자, 멤버 초대 가능 | OWN_ONLY |
| | CLIENT_MEMBER | 고객사 일반 직원, 배정 프로젝트만 | OWN_ONLY |

### 2.4 레벨별 제공 기능

#### ADMIN_LEVEL
| 기능 | 설명 |
|------|------|
| 서비스 개발/관리 | AMA 전체 시스템 관리 |
| 관리자 대시보드 | `/admin` — 사용자, 법인, 서비스, AI 사용량 관리 |
| 서비스 관리 | 서비스 카탈로그, 플랜, 고객, 구독 관리 |
| CMS 관리 | 사이트 메뉴, 페이지, 포스트 관리 |
| 포털 고객 관리 | 포털 등록사용자 → AMB 계정 생성 |
| 전체 데이터 접근 | 모든 법인의 데이터 조회/수정 |

#### PARTNER_LEVEL (개발 예정)
| 기능 | 설명 |
|------|------|
| 파트너 로그인 | `/partner/login` — 파트너 전용 로그인 |
| 파트너 마이페이지 | 파트너별 대시보드 |
| 플러그인 앱 개발 | AMA 용 서드파티 앱 개발 |
| 앱 프로젝트 신청 | 앱스토어 등록 신청/관리 |
| 앱 프로젝트 관리 | 신청/승인/등록 상태 관리 |

#### USER_LEVEL
| 기능 | 설명 |
|------|------|
| 메인 로그인 | `/login` — 법인코드 + 이메일 + 비밀번호 |
| 전체 업무 기능 | 대시보드, AI채팅, 프로젝트, 이슈, 캘린더, HR, Billing 등 |
| 법인 내 데이터 | 소속 법인 데이터만 접근 |
| 멤버 초대 | MASTER: 법인 내 멤버 초대 가능 |
| 고객사 관리 | 고객사 등록, 고객 직원 초대 (CLIENT_LEVEL) |

#### CLIENT_LEVEL
| 기능 | 설명 |
|------|------|
| 고객 포털 로그인 | `/client/login` — 법인코드 + 이메일 + 비밀번호 |
| 프로젝트 조회 | 배정된 프로젝트 내용 조회 |
| 이슈 등록 | 프로젝트에 이슈 등록 가능 |
| 이슈 추적 | 등록한 이슈의 처리 상태 추적 |
| 프로필 관리 | 자신의 프로필 수정 |

## 3. AS-IS (현재 구현 상태)

### 3.1 구현된 레벨 체계

| 항목 | 상태 | 구현 내용 |
|------|------|----------|
| **ADMIN_LEVEL** | ✅ 완료 | SUPER_ADMIN, ADMIN 등급 구현 |
| **USER_LEVEL** | ✅ 완료 | MASTER, MANAGER, MEMBER, VIEWER 등급 구현 |
| **CLIENT_LEVEL** | ✅ 완료 | CLIENT_ADMIN, CLIENT_MEMBER 등급 구현 |
| **PARTNER_LEVEL** | ❌ 미구현 | 코드에 존재하지 않음 |

### 3.2 구현된 코드 위치

| 구성요소 | 파일 위치 | 설명 |
|---------|----------|------|
| 레벨/등급 타입 | `packages/types/src/user-level.types.ts` | USER_LEVEL_CODE, USER_ROLE, VALID_LEVEL_ROLES |
| 권한 타입 | `packages/types/src/permission.types.ts` | DATA_SCOPE, ROLE_DATA_SCOPE, MENU_PERMISSIONS |
| 사용자 엔티티 | `apps/api/src/domain/auth/entity/user.entity.ts` | usrLevelCode, usrRole 필드 |
| 역할 가드 | `apps/api/src/global/guard/roles.guard.ts` | ROLE_HIERARCHY (0~6 레벨) |
| 관리자 가드 | `apps/api/src/global/guard/admin-level.guard.ts` | ADMIN_LEVEL 전용 |
| 레벨역할 가드 | `apps/api/src/domain/auth/guard/level-role.guard.ts` | 상태 + 레벨 + 역할 검증 |
| 데이터 스코프 | `apps/api/src/domain/auth/interceptor/data-scope.interceptor.ts` | ALL/OWN_ORG/CLIENT 자동 필터 |
| JWT 페이로드 | `apps/api/src/domain/auth/interface/jwt-payload.interface.ts` | level, role, companyId, cliId |

### 3.3 현재 역할 계층 (ROLE_HIERARCHY)

```
Level 6: SUPER_ADMIN   ─── ADMIN_LEVEL
Level 5: ADMIN         ─── ADMIN_LEVEL
Level 4: MASTER        ─── USER_LEVEL
Level 3: MANAGER       ─── USER_LEVEL
Level 2: MEMBER        ─── USER_LEVEL
Level 1: VIEWER        ─── USER_LEVEL
Level 0: CLIENT_ADMIN  ─── CLIENT_LEVEL  (격리됨)
Level 0: CLIENT_MEMBER ─── CLIENT_LEVEL  (격리됨)
```

### 3.4 현재 로그인 페이지

| URL | 대상 | 앱 | 상태 |
|-----|------|-----|------|
| `/login` | ADMIN_LEVEL + USER_LEVEL | apps/web | ✅ 구현 |
| `/client/login` | CLIENT_LEVEL | apps/web | ✅ 구현 |
| 포털 `/login` | 포털 고객 | apps/portal-web | ✅ 구현 |
| `/partner/login` | PARTNER_LEVEL | - | ❌ 미구현 |

### 3.5 현재 가입/계정 생성 경로

| 경로 | 대상 | 설명 | 상태 |
|------|------|------|------|
| 포털 회원가입 | 포털 고객 | www.amoeba.site `/register` → PortalCustomerEntity | ✅ |
| 포털→AMB 전환 | USER_LEVEL | Admin이 Portal-bridge에서 법인+역할 지정하여 생성 | ✅ |
| 법인 내 초대 | USER_LEVEL | Entity Settings → Members → Invite | ✅ |
| 고객사 초대 | CLIENT_LEVEL | Entity Settings → Clients → Invite | ✅ |
| 직접 회원가입 | USER_LEVEL | `/register` → 법인 생성 또는 가입 | ✅ |
| 파트너 계정 생성 | PARTNER_LEVEL | - | ❌ 미구현 |

## 4. 갭 분석 (정책 vs 구현)

### 4.1 일치 항목 (구현 완료)

| 정책 항목 | 구현 상태 | 비고 |
|----------|----------|------|
| ADMIN_LEVEL: SUPER_ADMIN, ADMIN | ✅ 완전 일치 | 전체 시스템 관리 권한 구현 |
| USER_LEVEL: MASTER, MANAGER, MEMBER, VIEWER | ✅ 완전 일치 | 법인별 역할 분리 구현 |
| CLIENT_LEVEL: CLIENT_ADMIN, CLIENT_MEMBER | ✅ 완전 일치 | 고객사 직원 이슈 등록, 프로젝트 조회 구현 |
| ADMIN → USER: Portal-bridge 계정 전환 | ✅ 완료 | USER_LEVEL 계정만 생성 가능 |
| USER → CLIENT: 고객사 초대 | ✅ 완료 | 이메일 초대 → 토큰 인증 → 계정 생성 |
| 데이터 스코프 (ALL/OWN_ORG/OWN_ONLY) | ✅ 완료 | 인터셉터로 자동 필터링 |
| 역할 계층 관리 (RolesGuard) | ✅ 완료 | 6단계 계층 + CLIENT 격리 |

### 4.2 불일치 항목 (갭)

| 정책 항목 | 현재 상태 | 갭 설명 | 우선순위 |
|----------|----------|--------|---------|
| **PARTNER_LEVEL 레벨 자체** | ❌ 미구현 | USER_LEVEL_CODE에 PARTNER_LEVEL 없음 | 🔴 높음 |
| **PARTNER_LEVEL 등급** | ❌ 미구현 | PARTNER_ADMIN, PARTNER_MEMBER 역할 없음 | 🔴 높음 |
| **/partner/login 페이지** | ❌ 미구현 | 파트너 전용 로그인 페이지 없음 | 🔴 높음 |
| **파트너 마이페이지** | ❌ 미구현 | 파트너 대시보드/메뉴 없음 | 🔴 높음 |
| **앱 프로젝트 신청** | ❌ 미구현 | 서드파티 앱스토어 등록 시스템 없음 | 🟡 중간 |
| **앱 프로젝트 관리** | ❌ 미구현 | 신청/승인/등록 상태 관리 없음 | 🟡 중간 |
| **Portal-bridge에서 PARTNER 계정 생성** | ❌ 미구현 | 현재 USER_LEVEL만 생성 가능 | 🔴 높음 |
| **ADMIN Setting: 파트너 앱 관리** | ❌ 미구현 | 파트너가 신청한 플러그인앱 관리 페이지 없음 | 🟡 중간 |

### 4.3 현재 정책과 다른 구현 (확인 필요)

| 항목 | 정책 | 현재 구현 | 검토 사항 |
|------|------|----------|----------|
| 레벨 구조 | 4개 (ADMIN/PARTNER/USER/CLIENT) | 3개 (ADMIN/USER/CLIENT) | PARTNER 추가 필요 |
| PARTNER 소속 | ADMIN_LEVEL의 서브 레벨 | 없음 | 새로운 레벨 또는 ADMIN_LEVEL 하위 서브 타입 |
| 포털→계정 전환 | USER_LEVEL 또는 PARTNER_LEVEL 선택 | USER_LEVEL만 가능 | PARTNER_LEVEL 옵션 추가 필요 |

## 5. 전체 레벨/등급/접근 체계 정리표

### 5.1 레벨-등급 매트릭스

```
 ADMIN_LEVEL (기본 레벨)
 ├── SUPER_ADMIN (등급 6) — 시스템 최고 관리자
 ├── ADMIN (등급 5)       — 운영 관리자
 │
 └── [서브] PARTNER_LEVEL ← 미개발
     ├── PARTNER_ADMIN    — 파트너사 관리자
     └── PARTNER_MEMBER   — 파트너사 직원

 USER_LEVEL (기본 레벨)
 ├── MASTER (등급 4)  — 법인 관리자
 ├── MANAGER (등급 3) — 부서/팀 관리자
 ├── MEMBER (등급 2)  — 일반 직원
 ├── VIEWER (등급 1)  — 읽기 전용
 │
 └── [서브] CLIENT_LEVEL ← 구현 완료
     ├── CLIENT_ADMIN (등급 0)  — 고객사 관리자
     └── CLIENT_MEMBER (등급 0) — 고객사 직원
```

### 5.2 레벨별 접근 페이지/기능

| 레벨 | 로그인 URL | 메인 레이아웃 | 핵심 기능 |
|------|-----------|-------------|----------|
| ADMIN_LEVEL | `/login` | MainLayout + AdminLayout | 전체 시스템 관리, 서비스 관리, CMS, 포털 브릿지 |
| PARTNER_LEVEL | `/partner/login` (예정) | PartnerLayout (예정) | 플러그인 앱 개발, 앱 프로젝트 관리 |
| USER_LEVEL | `/login` | MainLayout | AI채팅, 프로젝트, 이슈, 캘린더, HR, Billing, 회계, 메일 등 |
| CLIENT_LEVEL | `/client/login` | ClientPortalLayout | 프로젝트 조회, 이슈 등록/추적, 프로필 관리 |

### 5.3 데이터 접근 범위

| 레벨 | 등급 | 데이터 범위 | 설명 |
|------|------|-----------|------|
| ADMIN_LEVEL | SUPER_ADMIN | ALL | 모든 법인/고객 데이터 |
| ADMIN_LEVEL | ADMIN | ALL | 모든 법인/고객 데이터 |
| PARTNER_LEVEL | PARTNER_ADMIN | PARTNER_ORG (예정) | 자사 파트너 데이터 |
| PARTNER_LEVEL | PARTNER_MEMBER | PARTNER_ORG (예정) | 자사 파트너 데이터 |
| USER_LEVEL | MASTER | OWN_ORG | 소속 법인 데이터 |
| USER_LEVEL | MANAGER | OWN_ORG | 소속 법인 데이터 |
| USER_LEVEL | MEMBER | OWN_ORG | 소속 법인 데이터 |
| USER_LEVEL | VIEWER | OWN_ORG | 소속 법인 데이터 (읽기 전용) |
| CLIENT_LEVEL | CLIENT_ADMIN | OWN_ONLY | 배정 프로젝트 + 고객사 멤버 관리 |
| CLIENT_LEVEL | CLIENT_MEMBER | OWN_ONLY | 배정 프로젝트만 |

### 5.4 계정 생성 경로

| 생성 경로 | 생성되는 레벨 | 생성 주체 | 상태 |
|----------|-------------|---------|------|
| 포털 회원가입 (www.amoeba.site) | PortalCustomer (미매핑) | 본인 | ✅ |
| Portal-bridge → AMB 계정 전환 | USER_LEVEL | ADMIN_LEVEL | ✅ |
| Portal-bridge → PARTNER 계정 전환 | PARTNER_LEVEL | ADMIN_LEVEL | ❌ 예정 |
| Entity Settings → 멤버 초대 | USER_LEVEL | MASTER / ADMIN_LEVEL | ✅ |
| Entity Settings → 고객사 초대 | CLIENT_LEVEL | USER_LEVEL (MASTER+) | ✅ |
| `/register` 직접 가입 | USER_LEVEL | 본인 | ✅ |
| `/partner/login` 파트너 가입 | PARTNER_LEVEL | - | ❌ 예정 |

## 6. PARTNER_LEVEL 개발 시 필요한 작업 목록

### 6.1 타입/상수 변경

| 파일 | 변경 내용 |
|------|----------|
| `packages/types/src/user-level.types.ts` | USER_LEVEL_CODE에 `PARTNER_LEVEL` 추가 |
| | USER_ROLE에 `PARTNER_ADMIN`, `PARTNER_MEMBER` 추가 |
| | VALID_LEVEL_ROLES에 `PARTNER_LEVEL: [PARTNER_ADMIN, PARTNER_MEMBER]` 추가 |
| | VALID_LEVEL_ORG_LEVEL에 `PARTNER_LEVEL` 매핑 추가 |
| `packages/types/src/permission.types.ts` | ROLE_DATA_SCOPE에 PARTNER 역할 추가 |

| | PartnerLevelGuard 신규 작성 |

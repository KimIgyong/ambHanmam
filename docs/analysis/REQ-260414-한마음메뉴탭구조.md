# REQ-260414 한마음 시스템 메뉴 탭 구조 요구사항 분석서

## 1. 요구사항 요약

한마음 시스템(hm.hanmam.kr) 메뉴 구조를 AMA 웹 프론트엔드에 구현한다.

| 항목 | 내용 |
|------|------|
| **GNB 탭 메뉴** | 경영정보 · 영업업무 · 서비스업무 · 업무지원 · 알림게시판 · 자금업무 6개 1단계 메뉴를 헤더 로그인 회사명 우측에 탭 형태로 배치 |
| **LNB 서브 메뉴** | 1단계 탭 클릭 시 해당 카테고리의 서브메뉴를 좌측 사이드바에 표시 |
| **목업 페이지** | `docs/specifications/00_*.md ~ 40_*.md` 총 41개 화면 명세 기반 목업 구현 |
| **LNB 고정 메뉴** | 나의업무 · 업무연락 · 전자결재 · 쪽지 · 나의메뉴 (모든 탭에서 공통 노출) |
| **산출물** | 요구사항 분석서 + 작업 계획서 (구현은 별도 진행) |

## 2. AS-IS 현황 분석

### 2.1 현재 레이아웃 구조 (`MainLayout.tsx`)

```
┌─────────────────────────────────────────────────────┐
│ Header (GNB)                                        │
│ [≡] [Logo] [회사명]     [검색]    [Entity][알림][언어]│
├──────────┬──────────────────────────────────────────┤
│ Sidebar  │ Main Content                             │
│ (LNB)    │                                          │
│          │ <Outlet />                               │
│ Work     │                                          │
│ Tools    │                                          │
│ --------│                                          │
│ Work     │                                          │
│ Modules  │                                          │
│ --------│                                          │
│ Custom   │                                          │
│ Apps     │                                          │
│ --------│                                          │
│ My Page  │                                          │
│ Settings │                                          │
└──────────┴──────────────────────────────────────────┘
```

**GNB (Header)**: 높이 `h-14 md:h-16`
- 좌: 햄버거(모바일) + 로고 SVG + 회사명(Entity Name)
- 중: GlobalSearchBar (md 이상)
- 우: EntitySelector + NotificationBell + LanguageSelector

**LNB (Sidebar)**: 폭 `w-52` / 접힌상태 `w-16`
- Work Tools: TODAY, TODO, CHAT 등 (`WORK_TOOL_CODES`)
- Work Modules: 동적 메뉴 (`useMyMenus()`)
- Custom Apps: 엔티티별 앱 (`useMyCustomApps()`)
- 하단: My Page, Entity Settings, Admin Settings

**라우팅**: `createBrowserRouter` → `MainLayout` 하위에 평탄한 route 구조
- `/today`, `/todos`, `/issues`, `/hr/*`, `/billing/*`, `/project/*`, `/kms/*` 등
- 접근제어: `AuthGuard` → `MenuGuard(menuCode)`

### 2.2 현재 메뉴 시스템
- `useMyMenus()`: API에서 사용자 메뉴 권한 조회
- `menuConfig.ts`: `WORK_TOOL_CODES`, `ICON_MAP`, `COLOR_MAP`
- 메뉴 카테고리: `WORK_TOOL` / `WORK_MODULE` 2분류
- 1단계 탭 메뉴 개념 **없음** — 모든 메뉴가 사이드바에 플랫하게 나열

### 2.3 현재 레이아웃 파일
| 파일 | 용도 |
|------|------|
| `MainLayout.tsx` | GNB + LNB + Content (메인 레이아웃) |
| `AdminLayout.tsx` | 관리자 전용 레이아웃 |
| `SubMenuLayout.tsx` | 탭형 서브메뉴 레이아웃 |
| `AuthLayout.tsx` | 인증 페이지 레이아웃 |
| `HrLayout.tsx`, `BillingLayout.tsx`, `ProjectLayout.tsx`, `KmsLayout.tsx` 등 | 도메인별 서브레이아웃 |

## 5. 사용자 플로우

### 5.1 GNB 탭 → 서브메뉴 → 페이지 이동

```
1. 사용자 로그인 후 기본 화면 진입
2. 헤더의 GNB 탭 메뉴 중 "경영정보" 클릭
   → 사이드바가 "경영정보" 서브메뉴로 전환
   → 첫번째 서브메뉴(메인 대시보드) 페이지 자동 이동
3. 사이드바에서 "회의관리" 클릭
   → 메인 영역에 회의관리 목업 페이지 렌더링
4. GNB에서 "영업업무" 탭 클릭
   → 사이드바가 "영업업무" 서브메뉴로 전환
   → 첫번째 서브메뉴(고객사 관리) 페이지 자동 이동
```

### 5.2 LNB 고정 메뉴 접근

```
1. 사이드바 하단의 "나의 업무" 클릭
   → GNB 탭은 현재 선택 유지 (또는 해제)
   → 메인 영역에 나의 업무 페이지 렌더링
2. "업무연락" 클릭 → 업무연락 페이지 (내부 탭: 처리중/담당/관심)
```

### 5.3 모바일 반응형

```
1. 모바일에서 GNB 탭은 가로 스크롤 또는 햄버거 메뉴 내 표시
2. 사이드바는 기존 드로어 방식 유지
3. 드로어 내에 현재 활성 탭의 서브메뉴 + 고정 메뉴 표시
```

## 6. 기술 제약사항

### 6.1 기존 AMA 기능과 병존
- 한마음 메뉴는 `/hm/*` prefix로 별도 라우트 그룹 구성
- 기존 AMA 메뉴(Today, Todos, Chat 등)는 그대로 유지
- 선택 방안: **별도 HanmamLayout** 사용 또는 MainLayout에 모드 전환 로직 추가

### 6.2 목업 수준 정의
- 백엔드 API 연동 없이 프론트엔드만 구현
- 데이터는 하드코딩된 더미 데이터 또는 빈 상태로 표시
- 검색·필터 UI는 렌더링만 (실제 동작 불필요)
- 등록/수정 모달은 UI 렌더링만 (실제 저장 불필요)

### 6.3 i18n
- 한마음 전용 네임스페이스 `hanmam` 추가
- 현재는 한국어(ko) 우선, 추후 en/vi 확장
- 메뉴명, 컬럼 헤더, 버튼 라벨 등 i18n 키 사용

### 6.4 반응형
- 기존 MainLayout의 반응형 패턴(모바일 드로어, 사이드바 접기) 동일 적용
- GNB 탭 메뉴: 모바일에서는 가로 스크롤(overflow-x-auto) 처리

### 6.5 라우트 구조
- 모든 한마음 페이지는 `/hm/` prefix 사용
- `AuthGuard` 적용 (인증 필수)
- 메뉴 권한 제어는 목업 단계에서는 생략 가능

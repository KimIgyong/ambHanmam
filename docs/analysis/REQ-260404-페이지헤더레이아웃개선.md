## 1. AS-IS 현황 분석

### 1-1. 페이지 헤더 구조 (2줄 문제)

`MainLayout.tsx`에서 컨텐츠 영역(`<main>`)을 렌더링할 때 두 줄이 중첩된다.

```
┌──────────────────────────────────────────────┐
│ [줄 1] CurrentMenuGuideBar                   │  ← MainLayout.tsx:546
│   "My Task"  📖 Manual / Tip                │
├──────────────────────────────────────────────┤
│ [줄 2] 페이지 h1 타이틀                      │  ← 각 페이지 컴포넌트 내부
│   My Tasks                  [뷰모드] [+Add] │
└──────────────────────────────────────────────┘
```

### 1-2. 메뉴 아이콘 불일치

사이드바에서는 모든 메뉴에 아이콘을 노출하지만, 컨텐츠 영역 h1에는 아이콘 적용이 불일치함.

| 페이지 | URL | h1 아이콘 | 비고 |
|--------|-----|---------|------|
| Today | `/today` | ✅ CalendarCheck | TodayPage 자체적으로 추가 |
| My Tasks | `/todos` | ❌ 없음 | 아이콘 미적용 |
| 기타 대부분 | 각 URL | ❌ 없음 | 통일 미적용 |

**사이드바 아이콘 정의**: `domain/menu/constants/menuConfig.ts`의 `ICON_MAP` + `COLOR_MAP`  
**현재 Today 방식**: 페이지 자체에서 `CalendarCheck`를 직접 import하여 h1 앞에 배치 (일관성 없음)

### 1-3. 매뉴얼/팁 버튼 동작 제한

현재 `CurrentMenuGuideBar.tsx`에서 팁 콘텐츠가 없으면 버튼을 `disabled` 처리함:

```tsx
const canOpen = !!tipContent;
<button disabled={!canOpen} ...>
```

→ 관리자가 아직 팁 콘텐츠를 입력하지 않은 메뉴에서는 버튼 자체가 비활성화되어 사용자가 기능의 존재를 모름.

### 1-4. 컨텐츠 영역 정렬 불일치

페이지마다 컨테이너 클래스가 제각각:

| 페이지 | 최상위 컨테이너 | 내부 컨테이너 |
|--------|--------------|------------|
| TodayPage | `h-full overflow-y-auto bg-gray-50` | `mx-auto max-w-6xl px-4 py-4 md:py-8` |
| TodoPage | `h-full overflow-y-auto` | `mx-auto max-w-7xl px-4 py-8` |
| HR/Billing 페이지 | `flex h-full flex-col overflow-hidden` | Header: `px-6 py-4` |

→ max-width 불일치(6xl vs 7xl), padding 불일치(px-4 vs px-6), 배경색 불일치(gray-50 vs 없음)

### 1-5. HR / 파트너(Billing) 서브메뉴 구조

현재 HR, Billing(파트너 포함) 모두 **별도 레이아웃 컴포넌트**로 서브메뉴를 처리:

```
HrLayout.tsx
├── Mobile: 가로 스크롤 탭 (border-b, bg-gray-50)
├── Desktop: 좌측 사이드바 w-48 (border-r, bg-gray-50)
└── <Outlet /> → 각 서브 페이지

BillingLayout.tsx  (동일 패턴, 색상만 orange)
```

**문제점**:
- 사이드바가 두 개 중첩되어 보이는 효과 (글로벌 사이드바 + HR/Billing 서브 사이드바)
- 모바일에서 가로 스크롤 탭이 별도 줄 추가
- 컨텐츠 영역이 더 좁아짐

## 2. TO-BE 요구사항 (v2)

### REQ-01. 페이지 헤더 1줄 통합

- `CurrentMenuGuideBar` 제거 → 첫 번째 줄 삭제
- 페이지 h1 타이틀 우측에 매뉴얼/팁 아이콘 버튼 배치
- 한 줄로 통합: `[아이콘] 페이지 제목  [📖]  ···  [액션버튼]`

### REQ-02. 매뉴얼/팁 항상 클릭 가능 (준비중 표시)

- 팁 콘텐츠 유무와 관계없이 📖 아이콘 버튼 항상 활성화
- 팁 콘텐츠가 없는 경우: 우측 슬라이드 패널을 열고 **"준비중"** 메시지 노출
- 팁 콘텐츠가 있는 경우: 기존과 동일하게 콘텐츠 표시

### REQ-03. 모든 메뉴 컨텐츠 h1 좌측에 메뉴별 아이콘 노출

- 사이드바의 `ICON_MAP` 동일 아이콘 사용
- 색상은 **단색 (gray-600)** 으로 통일 (사이드바의 컬러맵과 별개)
- 아이콘 크기: `h-6 w-6`
- `PageTitle` 컴포넌트에서 Context의 `icon` 정보를 받아 자동 렌더링

### REQ-04. 컨텐츠 영역 좌측 정렬 통일

- 모든 페이지 컨테이너를 `px-6 py-6` 패딩으로 통일
- `max-w` 제한 제거 또는 `max-w-screen-2xl`로 통일 (전체 너비 활용)
- 배경색: `bg-white` 또는 없음으로 통일 (gray-50 혼용 제거)
- HR/Billing 등 서브레이아웃 페이지도 동일 기준 적용

### REQ-05. HR / 파트너(Billing) 서브메뉴 → 컨텐츠 탭메뉴로 전환

- `HrLayout.tsx`, `BillingLayout.tsx`의 사이드바/모바일탭 방식 제거
- 컨텐츠 영역 상단에 **수평 탭메뉴** 형태로 서브메뉴 배치
- 탭 클릭 시 URL 이동 (현재 라우팅 구조 유지)
- 데스크톱·모바일 동일한 탭 UI 사용

## 4. 사용자 플로우

### REQ-01,02,03 헤더 통합 + 아이콘 + 팁

```
페이지 이동
  → PageTitle 렌더: [☑ 아이콘] My Tasks [📖]
  → 사용자: 메뉴 아이콘으로 컨텍스트 인식, 팁 버튼 즉시 인식
  → 팁 클릭:
      ├─ 콘텐츠 있음 → 슬라이드 패널에 팁 콘텐츠 표시
      └─ 콘텐츠 없음 → 슬라이드 패널에 "준비중" 표시
```

### REQ-05 HR/파트너 탭 전환

```
[현재]
HR 클릭 → HrLayout 사이드바 + 직원 페이지
  → 사이드바: 직원 | 근태 | 휴가 | 급여 | ...
  → 우측 컨텐츠: EmployeeListPage

[개선 후]
HR 클릭 → HR 페이지 (단일 컴포넌트)
  → 상단 탭: [직원] [근태] [휴가] [급여] [퇴직금] [보고서] ...
  → 탭 아래 컨텐츠: 각 탭의 페이지 컨텐츠
```

## 5. 기술 제약사항

- **라우팅 구조 유지**: HR, Billing의 URL 구조(`/hr/employees`, `/billing/partners` 등)는 그대로 유지하면서 레이아웃만 탭으로 전환
- **권한 조건 탭 필터링**: `adminOnly`, `krOnly` 옵션에 따른 탭 노출 조건 유지 필요
- **ICON_MAP 연동**: `menuConfig.ts`의 기존 아이콘 매핑을 PageTitle에서 재사용
- **다수 페이지 교체**: h1 → PageTitle 교체 대상 약 30개 이상 (우선 주요 업무 페이지 13개부터)

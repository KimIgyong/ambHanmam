### 2단계: R1 — 검색 방식을 버튼 클릭으로 변경 (FE only)

**변경 파일**: `IssueFilters.tsx`

**구현**:
1. `useDebounce` 제거 (import + 사용부 삭제)
2. `searchInput` 상태는 유지
3. `handleSearch()` 함수 추가: `onChange({ ...filters, search: searchInput || undefined })`
4. 검색 입력란 우측에 검색 버튼 추가 (Search 아이콘 사용)
5. `onKeyDown` 핸들러에서 Enter 키 시 `handleSearch()` 호출
6. 검색 입력란 비우고 Enter/버튼 시 검색 초기화

**현재 코드 (삭제)**:
```typescript
// IssueFilters.tsx:37-42 삭제
const debouncedSearch = useDebounce(searchInput, 500);
useEffect(() => {
  if (debouncedSearch !== (filters.search || '')) {
    onChange({ ...filters, search: debouncedSearch || undefined });
  }
}, [debouncedSearch]);
```

**변경 코드**:
```typescript
const handleSearch = () => {
  onChange({ ...filters, search: searchInput.trim() || undefined });
};
```

**사이드 임팩트**: 없음 (검색 동작 타이밍만 변경)

### 4단계: R4 — 간트차트 계층(쓰레드) 표현 (FE only)

**변경 파일**: `IssueGanttView.tsx`

#### 4-1: 이슈 계층 정렬 로직

**현재 정렬**: `priority ASC, status ASC` (플랫)

**변경 정렬**:
```typescript
const buildHierarchicalList = (issues: IssueResponse[]): IssueResponse[] => {
  // 1. 부모가 없는 이슈 (최상위) 추출
  const roots = issues.filter(i => !i.parentIssueId);
  // 2. parentIssueId별 자식 그룹핑
  const childrenMap = new Map<string, IssueResponse[]>();
  issues.forEach(i => {
    if (i.parentIssueId) {
      const children = childrenMap.get(i.parentIssueId) || [];
      children.push(i);
      childrenMap.set(i.parentIssueId, children);
    }
  });
  // 3. 최상위 이슈 뒤에 자식 삽입
  const result: IssueResponse[] = [];
  roots.sort((a, b) => a.priority - b.priority);
  for (const root of roots) {
    result.push(root);
    const children = childrenMap.get(root.issueId) || [];
    children.sort((a, b) => a.priority - b.priority);
    result.push(...children);
  }
  return result;
};
```

#### 4-2: 좌측 라벨 들여쓰기

```tsx

// 좌측 라벨 렌더링

    <span className="text-gray-300">└</span>
  )}
  <span className="truncate text-xs text-gray-700">{issue.title}</span>
</div>
```

#### 4-3: 간트바 연결선 (선택적)

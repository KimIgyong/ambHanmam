### P2. SSE 채널 접근 권한 미검증 ⭐
> 위험2 | 위험도: 5 | 공격용이성: 4 | 점수: **100**

**현황**: 인증된 사용자라면 아무 채널ID로 SSE 구독 가능. 멤버십 검증 없음
**위험**: 비공개 채널의 실시간 메시지를 외부인이 수신 가능

**작업내용**:
| 단계 | 내용 | 파일 |
|------|------|------|
| 1 | SSE 컨트롤러에서 userId 추출 | `message.controller.ts` |
| 2 | 채널 멤버십 검증 로직 추가 | `talk-sse.service.ts` 또는 컨트롤러 |
| 3 | 비멤버 요청 시 ForbiddenException | - |

**코드 변경**:
```typescript
// message.controller.ts SSE 엔드포인트
@Sse('events')
sse(@Param('channelId') channelId: string, @CurrentUser() user): Observable<MessageEvent> {
  const isMember = await this.channelService.isMember(channelId, user.userId);
  if (!isMember) throw new ForbiddenException('Not a channel member');
  return this.sseService.subscribe(channelId);
}
```

### P5. Docker 컨테이너 root 실행
> 위험5 | 위험도: 4 | 공격용이성: 3 | 점수: **60**

**현황**: 모든 Dockerfile production stage에서 `USER` 지시어 없음 → root 실행
**위험**: 컨테이너 탈출 공격 시 호스트 시스템 root 권한 획득

**작업내용**:
| 단계 | 내용 | 파일 |
|------|------|------|
| 1 | staging Dockerfile 4개에 non-root user 추가 | `docker/staging/Dockerfile.*` |
| 2 | production Dockerfile 4개에 동일 적용 | `docker/production/Dockerfile.*` |
| 3 | 파일 권한 조정 (uploads 디렉토리 등) | Docker volumes |

**코드 변경** (각 Dockerfile production stage):
```dockerfile
RUN addgroup -g 1001 -S nodejs && adduser -S nodejs -u 1001
# ... COPY 이후
USER nodejs
```

### P6. HSTS + Nginx 보안 헤더 추가
> 주의1,2 통합 | 위험도: 3 | 공격용이성: 3 | 점수: **45**

**현황**: Helmet 기본 설정만, Nginx에 보안 헤더 없음. HSTS 미설정
**위험**: 다운그레이드 공격 (HTTPS→HTTP), 클릭재킹, MIME 스니핑

**작업내용**:
| 단계 | 내용 | 파일 |
|------|------|------|
| 1 | Nginx 보안 헤더 추가 | `docker/staging/nginx.conf`, `docker/production/nginx.conf` |
| 2 | Helmet HSTS 명시 설정 | `apps/api/src/main.ts` |

**Nginx 추가 내용**:
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-Frame-Options "DENY" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;
```

### P13. 파일 매직 바이트 검증

### P14. 계정 잠금 로직
> 주의5

**현황**: 로그인 실패 횟수 제한 없음 (Rate Limit 5회/분은 있으나 계정 잠금은 없음)
**작업**: 5회 연속 실패 시 15분 계정 잠금, DB에 실패 카운터 추가

---

### P15. DB 자동 백업 스크립트
> 위험8

**현황**: PostgreSQL 데이터 백업 없음 (Docker 이미지 백업만 있음)
**작업**: pg_dump 자동 백업 스크립트 + crontab (매일 02:00)

---

### P16. Access Token에 tokenVersion 검증
> 주의7

**현황**: Refresh Token만 tokenVersion 검증. Access Token은 4시간 내 무효화 불가
**작업**: JWT Strategy에서 access token의 tokenVersion 검증 추가

---

### P17. 데이터 수정/삭제 감사 로그
> 위험10

**현황**: 접근 감사만 있고, 데이터 변경(수정/삭제) 감사 로그 없음
**작업**: TypeORM Subscriber 또는 서비스 레벨에서 중요 엔티티 변경 시 감사 로그 기록

## Phase 3: 🟢 개선 — 중기 보안 강화 (7건)

### P18. CSRF 토큰 미들웨어

### P21. DB SSL 연결

### P22. AI 프롬프트 입력 이스케이프

### P23. 체계적 로깅 도입

### P24. git 히스토리 민감 데이터 정리

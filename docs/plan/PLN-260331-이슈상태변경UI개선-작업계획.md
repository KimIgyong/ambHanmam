# PLAN-이슈상태변경UI개선-작업계획-20260331

## 1. 시스템 개발 현황 분석

### 현재 구현 상태
- `IssueDetailModal.tsx` (#664-703): Workflow 섹션 내 `<select>` 드롭다운으로 상태 변경
- 드롭다운은 8개 상태 전체를 나열하고 현재 상태만 제외, 권한 필터 적용
- `ISSUE_VALID_TRANSITIONS`가 `@amb/types`에 정의되어 있으나 프론트에서 미사용
- `handleWorkflowAction()`이 APPROVED/REJECTED는 별도 API, 나머지는 공통 API 호출
- i18n: `issues:status.{STATUS}`, `issues:workflow.changeStatus` 등 이미 정의

### 변경 대상
- `IssueDetailModal.tsx`의 Workflow 섹션 UI만 수정 (1개 파일)
- i18n JSON 3개 파일에 신규 키 추가

## 2. 단계별 구현 계획

### Phase 1: 프론트엔드 UI 수정 (IssueDetailModal.tsx)

**목표**: Workflow 섹션을 다음 단계 버튼 + 모든 상태 드롭다운으로 분리

**구현 내용**:

1. `ISSUE_VALID_TRANSITIONS`를 `@amb/types`에서 import
2. Workflow 섹션 레이아웃 변경:
   - `flex justify-between` 컨테이너로 좌우 분리
   - **좌측**: 유효 전이 상태를 버튼으로 렌더링
     - `ISSUE_VALID_TRANSITIONS[currentIssue.status]`에서 유효 상태 추출
     - 권한 필터링 적용 (기존 로직 동일)
     - 각 버튼에 STATUS_COLORS 적용
   - **우측**: 기존 `<select>` 드롭다운 유지
     - 현재 상태 제외, 권한 필터 적용 (기존 로직 동일)
3. 워크플로우 노트 입력은 하단 유지

**사이드 임팩트**: 없음 (동일 `handleWorkflowAction` 사용)

### Phase 2: i18n 키 추가

**목표**: 3개 언어(ko/en/vi) 번역 파일에 신규 키 추가

**추가 키**:
- `issues:workflow.allStatuses` — 드롭다운 라벨 "모든 상태"

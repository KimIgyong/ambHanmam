## 2. 단계별 구현 계획

### Phase 1: RichTextEditor 컴포넌트 개선 (단일 파일 수정)

**변경 내용**:

1. **Props 추가**: `maxHeight?: string` (기본값 없음 — 미지정 시 기존 동작 유지)
2. **레이아웃 구조 변경**:
   - wrapper: `flex flex-col` 추가
   - toolbar: `sticky top-0 z-10 bg-white` 추가 → 에디터 내부 스크롤 시 툴바 고정
   - editor body: `overflow-y-auto` + `overscroll-behavior: contain` + `style={{ maxHeight }}`
3. **Tailwind 동적 클래스 수정**:
   - `[&_.ProseMirror]:min-h-[${minHeight}]` → `style={{ minHeight }}` 사용
4. **resize-y 제거**: maxHeight 기반 자동 스크롤로 대체
5. **source 모드도 동일하게 maxHeight 적용**

**하위 호환성**:
- `maxHeight` 미지정 시 기존과 동일하게 동작 (height 제한 없음, 콘텐츠따라 확장)
- 코멘트 사용처(80px)는 `maxHeight` 미지정이므로 영향 없음

### Phase 2: 사용처에 maxHeight 적용

본문 편집용 사용처에 `maxHeight` prop 추가:

| 파일 | 현재 | 변경 |
|------|------|------|
| `IssueFormModal.tsx` | `minHeight="120px"` | + `maxHeight="300px"` |
| `TodoFormModal.tsx` | `minHeight="100px"` | + `maxHeight="300px"` |
| `MeetingNoteFormModal.tsx` | (기본 150px) | + `maxHeight="400px"` |
| `NoticeFormModal.tsx` | (기본 150px) | + `maxHeight="400px"` |
| `ClientIssueCreatePage.tsx` | `minHeight="250px"` | + `maxHeight="400px"` |
| `MissionSection.tsx` (×2) | `minHeight="100px"` | + `maxHeight="250px"` |

코멘트 사용처(4곳): **변경 없음** (80px 유지, maxHeight 미지정)

## 1) 시스템 개발 현황 기반 요약

- 현재 코드베이스에 자산관리 전용 도메인(`asset`, `asset-request`, `meeting-room`)은 없음.
- 재사용 가능한 공통 기반은 존재함:
  - 메뉴/권한: `settings` 역할/그룹/사용자 권한 체계
  - 승인 패턴: `billing`, `project` 승인 상태 전이
  - 이력 패턴: `issues/todo/project` 상태 로그 구조
  - 검색: `search` + `work_items`
  - 알림/스케줄러: `mail`, `@nestjs/schedule`
  - 파일 업로드: `infrastructure/file`
- 결론: 신규 도메인 구축이 필수이며, 공통 인프라를 최대 재사용해야 함.

## 2) 구현 목표

1. 자산 마스터/신청/승인/반납/회의실 예약의 핵심 업무 플로우 구축
2. 역할별 접근제어(일반/부서장/자산관리자) 및 데이터 스코프 강제
3. 자산·신청 변경 이력 및 감사로그 체계 구축
4. 회의실 시간중복 검증 및 캘린더 뷰 제공
5. 통계·리포트 및 엑셀 입출력 고도화

## 4) 사이드 임팩트 분석

- 권한 체계: 신규 메뉴코드 다수 추가로 기본 권한 매트릭스 정합성 검증 필요
- 검색 연계: WorkItem 동기화 시 공개범위 필터 누락 시 보안 리스크
- 알림 시스템: 스케줄러 과부하/중복 발송 방지 설계 필요
- DB 부하: 회의실 시간중복 검증 쿼리 및 통계 집계로 인덱스 전략 중요

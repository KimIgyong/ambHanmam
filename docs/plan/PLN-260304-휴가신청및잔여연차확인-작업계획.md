### Phase 2: 백엔드 — LeaveRequest 서비스 + 컨트롤러

#### Step 2.1: LeaveRequestService 생성
**새 파일**: `apps/api/src/domain/hr/service/leave-request.service.ts`

**메서드:**
- `getMyLeaveRequests(userId, year)` — 본인 신청 내역 (usr_id → emp_id 매핑)
- `getMyLeaveBalance(userId, year)` — 본인 연차 잔여 조회 (usr_id → emp_id 매핑)
- `createLeaveRequest(userId, entId, dto)` — 휴가 신청
  - usr_id → emp_id 변환
  - 차감 일수 계산 (영업일, 공휴일 제외)
  - 잔여 연차 검증 (ANNUAL, AM_HALF, PM_HALF만)
  - 중복 일자 검증 (PENDING/APPROVED 건 겹침 체크)
- `cancelLeaveRequest(userId, requestId)` — 신청 취소 (PENDING만)
- `getLeaveRequests(entId, params)` — 관리자: 전체 목록 (상태/기간/직원 필터)
- `approveLeaveRequest(requestId, approvedByUserId)` — 승인
  - Timesheet 자동 삽입 (ANNUAL → 'AL', HALF → 'AH')
  - LeaveBalance 차감 (lvb_used ↑, lvb_remaining ↓)
- `rejectLeaveRequest(requestId, approvedByUserId, reason)` — 반려

**헬퍼:**
- `calcBusinessDays(startDate, endDate, holidays)` — 영업일 수 계산 (주말+공휴일 제외)
- `findEmployeeByUserId(userId)` — usr_id → emp_id 매핑

## 3. 설계 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| usr_id → emp_id 매핑 | EmployeeEntity.usrId 활용 | 이미 FK 존재 |
| 반차 Timesheet 코드 | 'AH' (Annual Half) | 기존 'AL'과 구분, 0.5일 차감 |
| 병가/특별휴가 차감 | 잔여 연차 차감 안 함 | 별도 카운트, 사유 기록 목적 |
| 승인자 | AdminGuard (ADMIN/SUPER_ADMIN) | 기존 HR 권한 패턴 유지 |
| 테이블 생성 | TypeORM synchronize 활용 | 개발/스테이징 자동 생성 |

## 4. 사이드 임팩트 분석

### 4.1 안전
- 기존 LeaveController/LeaveService 변경 없음 (별도 leave-request 경로)
- 기존 LeavePage.tsx는 탭 추가만 (기존 UI 영향 없음)
- 기존 Timesheet에 INSERT만 (기존 데이터 변경 없음)

### 4.2 주의사항
- **승인 시 Timesheet 삽입**: 이미 해당 날짜에 Timesheet 데이터 있으면 UPDATE 필요
- **LeaveBalance 차감**: 동시 신청 시 race condition 가능 — DB 트랜잭션으로 보호
- **MyPage 비로그인 직원**: usr_id → emp_id 매핑 없는 사용자는 "직원 정보 없음" 표시

## 5. 검증 방법

1. `npm run build` — 빌드 성공 확인
2. 일반 사용자로 로그인 → My Page에서 연차 잔여 카드 확인
3. `/hr/my-leave` 접속 → 잔여 연차 + 빈 신청 목록
4. "휴가 신청" → 연차 3일 신청 → PENDING 상태
5. 관리자로 로그인 → `/hr/leave` → 신청 관리 탭 → 승인
6. 승인 후 Timesheet에 AL 코드 자동 삽입 확인
7. 승인 후 잔여 연차 감소 확인
8. PENDING 상태 신청 취소 테스트
9. 잔여 연차 부족 시 신청 차단 테스트

## 2. TO-BE 설계

### 2.1 번호 형식

```
ISN-{일련번호}
```
- 일련번호: 법인(Entity)별 독립 증가 (1부터 시작)
- 예시: ISN-1, ISN-2, ... ISN-1000
- Redmine 가져오기 이슈도 동일 형식 (RM-xxx → ISN-xxx)

### 2.2 시퀀스 테이블 설계

```sql
CREATE TABLE amb_issue_sequences (
    isq_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    ent_id UUID NOT NULL UNIQUE REFERENCES amb_entities(ent_id),
    isq_last_number INTEGER NOT NULL DEFAULT 0,
    isq_updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

- 법인당 1행, `isq_last_number`가 마지막 발급 번호
- `UNIQUE(ent_id)` — 법인당 시퀀스 유일성 보장

### 2.3 채번 로직 (동시성 안전)

```typescript
async generateRefNumber(entityId: string, manager?: EntityManager): Promise<string> {
  const repo = manager 
    ? manager.getRepository(IssueSequenceEntity)
    : this.issueSequenceRepo;
  
  // 1. SELECT ... FOR UPDATE (행 잠금)
  let seq = await repo
    .createQueryBuilder('seq')
    .setLock('pessimistic_write')
    .where('seq.entId = :entityId', { entityId })
    .getOne();
  
  // 2. 없으면 생성 (첫 이슈)
  if (!seq) {
    seq = repo.create({ entId: entityId, isqLastNumber: 0 });
  }
  
  // 3. 증가 + 저장
  seq.isqLastNumber += 1;
  await repo.save(seq);
  
  return `ISN-${seq.isqLastNumber}`;
}
```

### 2.4 프론트엔드 표시

현재 이미 `IssueItem.tsx`, `IssueDetailModal.tsx`에서 `refNumber` 배지를 표시 중.
- **변경 없음**: 기존 UI 그대로 사용 (ISN-1 형태가 자동 표시됨)
- Redmine 가져온 이슈 목록 페이지에서도 `refNumber` 컬럼이 이미 노출됨

### Step 8: 기존 이슈 마이그레이션 (백필)

기존 이슈(`iss_ref_number IS NULL` 또는 `RM-xxx`)에 ISN-번호 부여.

**SQL 마이그레이션**:
```sql
-- 법인별 시퀀스 초기화
INSERT INTO amb_issue_sequences (isq_id, ent_id, isq_last_number)
SELECT gen_random_uuid(), ent_id, COUNT(*)
FROM amb_issues
WHERE iss_deleted_at IS NULL
GROUP BY ent_id
ON CONFLICT (ent_id) DO UPDATE SET isq_last_number = EXCLUDED.isq_last_number;

-- 기존 이슈에 ISN-번호 부여 (법인별 생성일 순)
WITH numbered AS (
  SELECT iss_id, ent_id,
         ROW_NUMBER() OVER (PARTITION BY ent_id ORDER BY iss_created_at ASC) AS rn
  FROM amb_issues
  WHERE iss_deleted_at IS NULL
)
UPDATE amb_issues i
SET iss_ref_number = 'ISN-' || n.rn
FROM numbered n
WHERE i.iss_id = n.iss_id;
```

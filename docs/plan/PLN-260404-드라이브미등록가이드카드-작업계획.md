# PLAN-20260404-드라이브미등록가이드카드-작업계획

## 1. 시스템 개발 현황 분석
- KR: `/drive`의 empty state는 존재하나, 설정 안내 링크/외부 가이드 연결이 없다.
- EN: `/drive` has empty states but no direct external guide linkage.
- KR: 가이드 문서(`ama-google-drive-guide.html`)는 정적 파일로 이미 존재한다.
- EN: The guide HTML already exists as a static file.

## 2. 단계별 구현 계획
### Phase 1. `/drive` 안내 카드 UI 추가
- 목적: 미등록 사용자에게 즉시 실행 가능한 가이드를 제공
- 구현:
1. `apps/web/src/domain/drive/pages/DocumentsPage.tsx`에 가이드 URL 계산 로직 추가
2. 아래 2개 상태에 카드뷰 삽입
: 드라이브 미설정 상태
: 폴더 미등록 상태
3. 카드 버튼은 새 탭(`target=_blank`)으로 가이드 문서 이동

### Phase 2. 다국어(i18n) 문구 추가
- 목적: 다국어 환경에서 동일 UX 제공
- 구현:
1. `apps/web/src/locales/ko/drive.json`에 guideCard 키 추가
2. `apps/web/src/locales/en/drive.json`에 guideCard 키 추가
3. `apps/web/src/locales/vi/drive.json`에 guideCard 키 추가

### Phase 3. 가이드 문서 메타 갱신
- 목적: 문서 최신성/연결 위치 명시
- 구현:
1. `apps/portal-web/public/service/ama/ama-google-drive-guide.html` 헤더 메타에 연결 위치/업데이트일 반영

### Phase 4. 배포 (Staging -> Production)
- 목적: 요청된 순차 배포 이행
- 구현:
1. 스테이징 서버에 변경 파일 반영
2. `bash docker/staging/deploy-staging.sh` 실행
3. 스테이징 확인(`/drive` 카드, 가이드 링크)
4. 프로덕션 서버에 동일 변경 반영
5. `bash docker/production/deploy-production.sh` 실행
6. 프로덕션 확인

## 4. 화면 구성안 (UI Composition)
- 화면: `/drive`
- 조건 A: `isConfigured=false`
: 기존 경고 메시지 하단에 `구글드라이브 연동 가이드` 카드 추가
: 카드 구성 = 제목 + 설명 + `가이드 열기` 버튼(외부링크 아이콘)
- 조건 B: `folders.length===0 && filteredSharedDrives.length===0`
: Empty state 하단에 동일 가이드 카드 추가
- 스타일 방향:
: 기존 Tailwind 톤 유지 (blue tint card / bordered surface)
: 본문 CTA와 시각적 구분이 되는 보조 행동 카드

## 5. 사이드 임팩트 분석
- 긍정 영향:
1. 설정 실패/미등록 사용자의 자가 해결률 증가
2. 운영팀 문의량 감소 기대
- 잠재 리스크:
1. 도메인 분기 로직 오판 시 링크 오작동 가능
2. portal-web 정적 파일 경로 변경 시 링크 깨짐 가능
- 완화 방안:
1. stg/prod 각각 실링크 클릭 검증
2. 링크 경로 변경 시 단일 상수화 검토

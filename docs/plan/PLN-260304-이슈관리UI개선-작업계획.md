## 단계 2: 유형/상태 멀티셀렉트 체크박스 전환

### 2-1. Backend - 다중 값 필터 처리
**파일**: `apps/api/src/domain/issues/service/issue.service.ts`

`getIssues()` 필터 시그니처 변경:
```typescript
// 기존: type?: string; status?: string;
// 변경: type?: string; status?: string; (쉼표 구분 다중값 지원)

if (filters?.type) {
  const types = filters.type.split(',');
  if (types.length === 1) {
    qb.andWhere('iss.issType = :type', { type: types[0] });
  } else {
    qb.andWhere('iss.issType IN (:...types)', { types });
  }
}
// status도 동일 패턴
```

하위 호환성: 단일 값(`BUG`)도 `split(',')` 시 배열 길이 1로 기존과 동일 동작.

### 3-3. Backend - 필터 프리셋 API
**파일**: `apps/api/src/domain/issues/controller/issue.controller.ts`

신규 엔드포인트 2개:

```
GET    /issues/filter-presets          → 본인 프리셋 목록 조회
PUT    /issues/filter-presets          → 본인 프리셋 전체 저장 (배열 교체)
```

PUT 방식으로 전체 프리셋 배열을 교체하는 단순한 방식 채택 (추가/삭제를 클라이언트에서 처리).

**파일**: `apps/api/src/domain/issues/service/issue.service.ts`

```typescript
async getFilterPresets(userId: string): Promise<any[]> {
  const user = await this.userRepository.findOne({ where: { usrId: userId } });
  return user?.usrIssueFilterPresets || [];
}

async saveFilterPresets(userId: string, presets: any[]): Promise<void> {
  await this.userRepository.update(userId, { usrIssueFilterPresets: presets });
}
```

  // 권한 체크
  const isReporter = entity.issReporterId === userId;

## 구현 순서

1. **단계 1** (범위 필터 확장) → 단독 완결, 가장 단순
2. **단계 2** (체크박스 필터) → 백엔드 다중 값 처리 + 프론트 UI 변경
3. **단계 4** (권한 강화) → 단계 2와 독립적, 병행 가능
4. **단계 3** (필터 프리셋) → 단계 2 완료 후 진행 (체크박스 필터 데이터를 저장하므로)

## 검증 방법

1. 범위 필터: "내가 관련된 이슈" 클릭 시 reporter 또는 assignee가 본인인 이슈만 표시
2. 체크박스: BUG + TASK 동시 선택 시 두 유형 모두 표시, OPEN + IN_PROGRESS 동시 선택 시 두 상태 모두 표시
3. 프리셋: 필터 저장 → 다른 페이지 이동 → 복귀 후 프리셋 클릭 → 저장된 필터 복원 확인
4. 권한: 관련 없는 사용자로 이슈 수정 시도 → 403 에러 반환 확인, Edit 버튼 미노출 확인
5. 하위 호환: 단일 값 type/status 쿼리도 정상 동작 확인

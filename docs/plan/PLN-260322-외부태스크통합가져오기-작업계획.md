**공개 메서드**:

| 메서드 | 설명 | 반환 |
|--------|------|------|
| `getAvailableProviders(entityId)` | Entity에 등록+활성 상태인 외부 태스크 도구 조회 | `ExternalProviderMeta[]` |
| `testConnection(provider, appId, entityId)` | 연결 테스트 | `{ success, info?, error? }` |
| `fetchProjects(provider, appId, entityId)` | 프로젝트 목록 | `ExternalProject[]` |
| `fetchGroups(provider, appId, entityId, projectId)` | 그룹 목록 | `ExternalGroup[]` |
| `fetchTasks(provider, appId, entityId, groupId, options?)` | 태스크 목록 + 중복 표시 | `PaginatedResult<ExternalTask>` |
| `importTasks(provider, appId, entityId, userId, request)` | 선택 태스크 가져오기 | `ExternalTaskImportResult` |
| `getImportedTasks(entityId, provider?, page?, size?)` | 가져온 태스크 조회 | `PaginatedResult<...>` |
| `getImportLogs(entityId, page?, size?)` | 가져오기 이력 | `PaginatedResult<...>` |

**내부 흐름 (importTasks)**:
```
1. ProviderRegistry.getProvider(provider)
2. EntityCustomAppService에서 appId로 Custom App 조회 → PAT 복호화
3. 각 task_id에 대해:
   3-1. mappingRepo에서 UNIQUE(ent_id, provider, external_id) 중복 체크
   3-2. 기존 iss_redmine_id도 체크 (Redmine인 경우)
   3-3. provider.convertToIssueData(task, defaults) → IssueEntity 생성
   3-4. IssueSequenceService로 ref_number 생성
   3-5. issueRepo.save(issue)
   3-6. mappingRepo.save(mapping)
4. logRepo.save(importLog)
5. 결과 반환
```

**사이드 임팩트**:
- `IssuesModule` export 필요: `IssueSequenceService`
- `EntitySettingsModule` export 필요: `EntityCustomAppService` (이미 export 중인지 확인 필요)

## 7. 테스트 전략

### 7.1 백엔드 검증

| 항목 | 방법 |
|------|------|
| Provider 연결 테스트 | Asana: VN01 Entity PAT으로 실제 API 호출 |
| Redmine Provider | 기존 Redmine 서버에 Entity Custom App으로 연결 후 테스트 |
| 중복 방지 | 동일 태스크 2회 가져오기 시도 → skipped 확인 |
| Entity 격리 | 다른 Entity에서 접근 시 403 확인 |
| 기존 Migration 호환 | `iss_redmine_id` 있는 이슈 → 신규 가져오기 시 중복 감지 확인 |

### 7.2 프론트엔드 검증

| 항목 | 방법 |
|------|------|
| 설정 페이지 | Entity Settings → 외부 태스크 도구 → Asana 연결 등록 → 연결 테스트 |
| 가져오기 페이지 | Issues → 외부 가져오기 → Asana 선택 → 프로젝트 → 섹션 → 태스크 선택 → 등록 |
| 미설정 안내 | 연결 없을 때 "Entity Settings에서 먼저 설정하세요" 안내 확인 |
| 중복 표시 | 이미 등록된 태스크에 "등록됨" 뱃지 확인 |
| 빌드 | `npm run -w @amb/web build` — 프론트엔드 빌드 성공 확인 |

#### 1-1. StatusBadge 공통 컴포넌트 추출

현재 `MemberListTab.tsx` 내부에 로컬 함수로 정의된 `StatusBadge`를 공통 컴포넌트로 분리합니다.

- **파일**: `apps/web/src/domain/members/components/StatusBadge.tsx` (신규)
- 기존 `MemberListTab.tsx`의 StatusBadge 로직 이동
- Props: `{ status: string }`
- 5가지 상태 컬러 매핑: ACTIVE(green), PENDING(yellow), INACTIVE(gray), SUSPENDED(orange), WITHDRAWN(red)
- 번역: `useTranslation('members')` → `t('userStatus.${status}')`

```typescript
// StatusBadge.tsx
export default function StatusBadge({ status }: { status: string }) {
  const colorMap: Record<string, string> = {
    ACTIVE: 'bg-green-100 text-green-700',
    PENDING: 'bg-yellow-100 text-yellow-700',
    INACTIVE: 'bg-gray-100 text-gray-500',
    SUSPENDED: 'bg-orange-100 text-orange-700',
    WITHDRAWN: 'bg-red-100 text-red-700',
  };
  const { t } = useTranslation('members');
  return (
    <span className={`inline-flex rounded-full px-2 py-0.5 text-xs font-medium ${colorMap[status] || colorMap.ACTIVE}`}>
      {t(`userStatus.${status}`)}
    </span>
  );
}
```

## 캐시 무효화 검증

### `memberKeys` 구조:
```typescript
const memberKeys = {
  all: ['members'] as const,
  list: (entityId) => ['members', 'list', entityId] as const,
  detail: (id) => ['members', 'detail', id] as const,
};
```

### 승인/거절 훅의 onSuccess:
```typescript
queryClient.invalidateQueries({ queryKey: memberKeys.all });
// memberKeys.all = ['members'] → prefix 매칭으로 ['members', 'detail', id]도 무효화됨
```

**결론**: React Query의 `invalidateQueries`는 prefix 매칭이므로 `['members']`로 무효화하면 `['members', 'detail', id]`도 함께 무효화됩니다. **추가 수정 불필요.**

## 2. 단계별 구현 계획

### Step 1: 최근 법인 저장/조회 유틸리티

**파일**: `apps/web/src/domain/auth/utils/recent-entities.ts` (신규)

```typescript
const STORAGE_KEY = 'amb_recent_entities';
const MAX_ITEMS = 5;

interface RecentEntity {
  entityId: string;
  code: string;
  name: string;
  nameEn: string | null;
  country: string;
}

// 최근 법인 목록 조회
function getRecentEntities(): RecentEntity[]

// 법인 추가 (중복 시 최상단으로 이동, 최대 5개)
function addRecentEntity(entity: RecentEntity): void

// 전체 삭제 (선택적)
function clearRecentEntities(): void
```

- try-catch로 localStorage 접근 실패 시 graceful fallback
- JSON.parse 실패 시 빈 배열 반환

### Step 2: LoginPage 수정

**파일**: `apps/web/src/domain/auth/pages/LoginPage.tsx` (수정)

변경 내용:
1. 페이지 진입 시 `getRecentEntities()`로 최근 법인 로드
2. 최근 법인이 있으면 첫 번째 항목을 `selectedEntity` 초기값으로 설정
3. 로그인 성공 시(`onSubmit` 성공 후) `addRecentEntity(selectedEntity)` 호출

```typescript
// 초기화
const recentEntities = getRecentEntities();
const [selectedEntity, setSelectedEntity] = useState<EntitySearchItem | null>(
  recentEntities[0] || null
);

// 로그인 성공 시
const result = await authService.login(...);
if (selectedEntity) {
  addRecentEntity(selectedEntity);
}
```

### Step 3: EntitySearchModal 수정

**파일**: `apps/web/src/domain/auth/components/EntitySearchModal.tsx` (수정)

변경 내용:
1. 검색 입력 전 (query가 비어있을 때) "최근 사용" 섹션 표시
2. 최근 법인이 없으면 기존처럼 빈 상태
3. 최근 법인 클릭 시 `onSelect` 호출

```
[모달 레이아웃]
┌─ Find your organization ──────── X ┐
│                                     │
│  🔍 Search by company name or code  │
│                                     │
│  ── Recent ──────────────────────   │
│  🇻🇳 ABC Vietnam (ABCVN)          │
│  🇰🇷 XYZ Korea (XYZKR)            │
│                                     │
└─────────────────────────────────────┘
```

- 검색 입력 시 기존 검색 결과로 전환 (최근 목록 숨김)
- 검색 지우면 다시 최근 목록 표시

## 6. 검증 방법

1. 법인 검색 → 로그인 성공 → 로그아웃 → 로그인 페이지 재진입 → 법인 자동 선택 확인
2. EntitySearchModal 열기 → 검색 입력 전 "최근 사용" 목록 표시 확인
3. 다른 법인으로 로그인 → 최근 목록에 2개 표시 확인 (최신 순)
4. 시크릿 모드에서 최근 목록 없이 정상 동작 확인
5. localStorage 직접 확인: `localStorage.getItem('amb_recent_entities')`

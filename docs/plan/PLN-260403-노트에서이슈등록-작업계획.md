# PLAN-노트기능개선-작업계획-20260403

## 1. 시스템 개발 현황 분석

### 현재 상태
- 노트 폼에 **기존 이슈 연결** 기능 존재 (issue_ids 멀티셀렉트)
- 이슈 생성은 별도 이슈 모듈에서만 가능
- Todo → Issue 전환 패턴 존재 (source_todo_id + 시스템 코멘트)
- MeetingNotesModule ↔ IssuesModule 간 순환 의존성 없음
- **담당자/관련자 알림**: ✅ 이미 구현 (생성 시 + 수정 시 새로 추가된 대상만)
- **메인 댓글 멘션**: ✅ 이미 구현 (extractMentionedUserIds + MENTION_EVENT + enableMention)
- **답글 멘션**: ❌ 프론트엔드 RichTextEditor에 `enableMention` prop 누락

### 변경 방향
- "이슈로 등록" 체크박스 + 프로젝트 선택 UI 추가 (신규 기능)
- 답글 RichTextEditor에 `enableMention` 추가 (1줄 수정)
- 기존 "관련 이슈" 연결, 알림 기능은 그대로 유지

## 2. 단계별 구현 계획

### Phase 1: 백엔드 — DTO + 모듈 + 서비스 확장 (이슈로 등록)

#### 1-1. DTO 확장
**수정**: `apps/api/src/domain/meeting-notes/dto/request/create-meeting-note.request.ts`
- `create_issue?: boolean` 추가
- `create_issue_project_id?: string` (UUID) 추가
- `create_issue_type?: string` 추가 (기본값: TASK)
- `create_issue_severity?: string` 추가 (기본값: MINOR)

**수정**: `apps/api/src/domain/meeting-notes/dto/request/update-meeting-note.request.ts`
- 동일 필드 추가

**사이드 임팩트**: 기존 DTO에 optional 필드 추가 — 미전송 시 무시됨

#### 1-2. 모듈 의존성
**수정**: `apps/api/src/domain/meeting-notes/meeting-notes.module.ts`
- `IssuesModule` import 추가

**사이드 임팩트**: 순환 의존성 없음 확인 완료

#### 1-3. 서비스 로직
**수정**: `apps/api/src/domain/meeting-notes/service/meeting-note.service.ts`
- `IssueService` DI 주입
- `createMeetingNote()`: create_issue=true 시 → IssueService.createIssue() 호출 → syncIssues에 새 이슈 ID 추가
- `updateMeetingNote()`: create_issue=true 시 → IssueService.createIssue() 호출 → syncIssues에 새 이슈 ID 추가

**사이드 임팩트**: create_issue=false(기본값)이면 기존 동작과 동일

### Phase 2: 프론트엔드 — 이슈로 등록 UI + 답글 멘션 수정

#### 2-1. i18n 키 추가
**수정**: `apps/web/src/locales/ko/meetingNotes.json`, `en/meetingNotes.json`, `vi/meetingNotes.json`
- `form.createIssue`: "이슈로 등록" / "Register as Issue" / "Đăng ký Issue"
- `form.createIssueProject`: "이슈 프로젝트" / "Issue Project" / "Dự án Issue"
- `form.createIssueType`: "이슈 타입" / "Issue Type" / "Loại Issue"
- `form.createIssueSeverity`: "심각도" / "Severity" / "Mức độ"
- `form.createIssueProjectRequired`: "프로젝트를 선택하세요" / "Please select a project" / "Vui lòng chọn dự án"

#### 2-2. 폼 모달 UI (이슈로 등록)
**수정**: `apps/web/src/domain/meeting-notes/components/MeetingNoteFormModal.tsx`
- 새 state: `createIssue`, `createIssueProjectId`, `createIssueType`, `createIssueSeverity`
- 기존 "관련 이슈" 셀렉트 위에 "이슈로 등록" 체크박스 섹션 추가
- 체크 시: 프로젝트 드롭다운(필수) + 이슈 타입 + 심각도 표시
- handleSubmit에 create_issue 관련 필드 포함

**사이드 임팩트**: 체크박스 미선택 시 기존과 동일

#### 2-3. 서비스 타입
**수정**: `apps/web/src/domain/meeting-notes/service/meeting-note.service.ts`
- `MeetingNoteFormData` 타입에 create_issue 관련 필드 추가

#### 2-4. 답글 멘션 수정 (1줄)
**수정**: `apps/web/src/domain/meeting-notes/components/NoteCommentSection.tsx`
- 답글 `RichTextEditor`에 `enableMention` prop 추가

**사이드 임팩트**: 없음 — 백엔드는 이미 답글 멘션 처리 지원

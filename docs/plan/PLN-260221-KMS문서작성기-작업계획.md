## 2. 구현 계획

### 2.1 Phase 구성 개요

DocBuilder와 5A Matrix를 **병렬 진행 가능한 Phase로 분리**하되, 통합 포인트(DDD→DocBuilder 브릿지)를 후반 Phase에 배치하여 양쪽 기반이 모두 완성된 후 연결한다.

```
Phase 0: 기반 구축 — DB + 시드 데이터 (2주)
  ├── DocBuilder 8개 테이블 + TypeORM 엔티티
  ├── 5A Matrix 6개 테이블 + TypeORM 엔티티
  ├── 기초데이터 카테고리/문서타입 시드
  ├── 5A Framework/Metrics 시드 (amoeba 5A 템플릿)
  └── 기초데이터 CRUD API + UI

Phase 1: DocBuilder — AI 추출 + 문서 생성 엔진 (4주)
  ├── Google Drive 문서 스캔 + AI 추출 파이프라인
  ├── 데이터 버전 관리
  ├── PPTX/DOCX 생성 엔진
  ├── 문서 타입 레지스트리 + 템플릿
  ├── 대상별 AI 커스터마이징 + 다국어
  ├── 실시간 미리보기
  └── 데이터 스냅샷

Phase 2: 5A Matrix — 대시보드 + 수동 입력 (3주)
  ├── Framework/Dashboard CRUD API
  ├── 5A 퍼널 Overview UI
  ├── Stage Detail 차트 페이지
  ├── 수동 스냅샷 입력 + QoQ/MoM 비교
  ├── Operational Readiness Gauge UI
  └── Strategy Position Map

Phase 3: DocBuilder — 피드백 루프 + Google Drive (2주)
  ├── 문서 다운로드/재업로드
  ├── AI Diff 분석 엔진
  ├── 기초데이터 업데이트 제안
  ├── Google Drive 양방향 동기화
  ├── 문서 수명주기 관리
  └── KMS 폴크소노미 연동

Phase 4: 5A Matrix — AI 엔진 + 자동 수집 (3주)
  ├── Billing/Service/HR/Project 자동 데이터 수집
  ├── AI 트렌드 분석 + 이상 탐지
  ├── AI 인사이트 자동 생성 (SSE)
  ├── 분기 종료 자동 리포트
  ├── 대화형 드릴다운 (경영 에이전트 연동)
  └── KMS 태그 도메인 바인딩

Phase 5: 통합 + 크로스 모듈 + 고도화 (2주)
  ├── ★ DDD→DocBuilder 브릿지 (5A→기초데이터 자동 동기화)
  ├── Billing/HR → DocBuilder 데이터 동기화
  ├── Stale data 감지 + 알림
  ├── 브랜드 엔진 + 비주얼 자산 관리
  ├── 일괄 생성
  └── 통합 테스트 + 최적화 + 배포
```

**총 기간: 16주 (약 4개월)**
- Phase 0: Week 1-2
- Phase 1: Week 3-6
- Phase 2: Week 7-9 (Phase 1 완료 후 또는 별도 인력으로 병렬 진행 가능)
- Phase 3: Week 10-11
- Phase 4: Week 10-12 (Phase 3과 병렬 진행 가능)
- Phase 5: Week 13-14 → 버퍼 Week 15-16

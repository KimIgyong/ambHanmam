## 2. 구현 계획

### 2.1 구현 전략

**RichTextEditor에 자체 @멘션 자동완성 오버레이 추가**

- Tiptap `@tiptap/extension-mention` 패키지 **사용하지 않음** (추가 의존성 최소화)
- Tiptap 에디터의 `onUpdate` 이벤트 + 커서 위치 감지로 `@` 입력 감지
- 기존 `useMemberList()` 훅 재활용하여 멤버 드롭다운 표시
- 멤버 선택 시 에디터에 `@username` 텍스트 삽입

### 2.2 단계별 구현

#### Step 1: RichTextEditor에 멘션 기능 추가

**파일**: `apps/web/src/domain/meeting-notes/components/RichTextEditor.tsx`

변경사항:
1. `enableMention?: boolean` prop 추가 (기본값 `false` → 기존 사용처 영향 없음)
2. `useMemberList()` 훅으로 멤버 목록 조회 (enableMention일 때만)
3. `onUpdate` 콜백에서 커서 앞 텍스트 분석 → `@query` 감지
4. 에디터 아래에 멤버 드롭다운 렌더링
5. 멤버 선택 시 `@` 부터 현재 커서까지 삭제 → `@username ` 삽입

#### Step 2: IssueDetailModal에서 enableMention 활성화

**파일**: `apps/web/src/domain/issues/components/IssueDetailModal.tsx`

변경사항:
- `<RichTextEditor enableMention />` prop 추가

### 2.3 사이드 임팩트 분석

| 영향 범위 | 분석 |
|-----------|------|
| RichTextEditor (Meeting Note 본문) | `enableMention` 기본값 `false` → 영향 없음 |
| RichTextEditor (Issue 본문 편집) | `enableMention` 전달하지 않으면 기존 동작 유지 |
| 백엔드 issue.service.ts | 이미 멘션 감지 구현됨 → 변경 불필요 |
| 알림 시스템 전체 | 변경 불필요 |

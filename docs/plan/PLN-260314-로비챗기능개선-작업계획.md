## 단계 1: 메시지 입력창 버튼 레이아웃 변경 (R3)

### 변경 내용
파일첨부(📎), 이미지첨부(🖼️) 버튼을 textarea 좌측에서 번역 토글과 같은 라인으로 이동.
번역은 좌측 유지, 파일/이미지 버튼은 우측 정렬.

### 변경 파일
- `apps/web/src/domain/amoeba-talk/components/TalkMessageInput.tsx`

### 현재 구조(AS-IS)
```tsx
{/* 번역 라인 */}
<div className="mb-2 flex items-center gap-2">
  <button onClick={toggleTranslation}>🌐 번역</button>
  {selectedLangLabel && <button>→ {selectedLangLabel}</button>}
</div>

{/* 입력 라인 */}
<div className="flex items-end gap-2">
  <button>[📎]</button>     {/* 파일첨부 */}
  <button>[🖼️]</button>     {/* 이미지첨부 */}
  <textarea />              {/* 메시지 입력 */}
  <button>[▶전송]</button>
</div>
```

### 변경 후 구조(TO-BE)
```tsx
{/* 번역 + 파일/이미지 라인 */}
<div className="mb-2 flex items-center justify-between">
  {/* 좌측: 번역 */}
  <div className="flex items-center gap-2">
    <button>🌐 번역</button>
    {selectedLangLabel && <button>→ {selectedLangLabel}</button>}
  </div>
  {/* 우측: 파일/이미지 */}
  <div className="flex items-center gap-1">
    <button>[📎]</button>
    <button>[🖼️]</button>
  </div>
</div>

{/* 입력 라인 */}
<div className="flex items-end gap-2">
  <textarea />
  <button>[▶전송]</button>
</div>
```

### 사이드 임팩트
- hidden `<input type="file">` 요소 2개도 함께 이동해야 함
- 클립보드 붙여넣기 기능(이전 구현)은 textarea에 바인딩되어 있어 영향 없음

## 단계 2: 동시번역 메시지 포맷 변경 — 백엔드 (R4, R5)

### 변경 내용
메시지 저장 시 번역 포맷에서:
1. `\n\n---\n` → `\n---\n` (빈 줄 제거)
2. `🌐` 아이콘 삭제
3. 언어명을 2자리 대문자 국가코드로 변경

### 변경 파일
- `apps/api/src/domain/amoeba-talk/service/message.service.ts`

### 현재 코드(AS-IS)
```typescript
const LANG_LABELS: Record<string, string> = {
  en: 'English', ko: '한국어', vi: 'Tiếng Việt', ja: '日本語', zh: '中文',
};

// 포맷:
finalContent = `${finalContent}\n\n---\n🌐 [${langLabel}] ${result.translatedContent}`;
```

### 변경 후 코드(TO-BE)
```typescript
const LANG_LABELS: Record<string, string> = {
  en: 'EN', ko: 'KO', vi: 'VI', ja: 'JA', zh: 'ZH',
};

// 포맷:  
finalContent = `${finalContent}\n---\n[${langLabel}] ${result.translatedContent}`;
```

### 사이드 임팩트
- 기존 DB에 저장된 메시지는 이전 포맷(`\n\n---\n🌐 [English]`)으로 유지됨
- 프론트엔드 렌더링이 두 가지 포맷 모두 처리할 수 있어야 함 → `whitespace-pre-wrap`이 적용되어 있으므로 그대로 표시됨 (문제 없음)

## 단계 5: 모바일 PWA 노치 safe-area 적용 (R1)

### 변경 내용
채팅 내 모달/드롭다운에 아이폰 노치(safe-area-inset-top) 여백 적용.

### 변경 파일
- `apps/web/src/domain/amoeba-talk/components/CreateChannelModal.tsx`
- `apps/web/src/domain/amoeba-talk/components/NewDmModal.tsx`
- `apps/web/src/domain/amoeba-talk/components/InviteMemberModal.tsx`
- `apps/web/src/domain/amoeba-talk/components/MemberPanel.tsx`
- (신규) `apps/web/src/domain/amoeba-talk/components/EditChannelModal.tsx`

### 변경 방법
모달 컨테이너의 safe-area 패딩 적용:
```tsx
// AS-IS
<div className="fixed inset-0 z-50 flex items-center justify-center bg-black/40">
  <div className="w-full max-w-md rounded-lg bg-white shadow-xl">

// TO-BE  
<div className="fixed inset-0 z-50 flex items-center justify-center bg-black/40 pt-[env(safe-area-inset-top)]">
  <div className="w-full max-w-md rounded-lg bg-white shadow-xl max-h-[calc(100vh-env(safe-area-inset-top)-env(safe-area-inset-bottom))] overflow-y-auto">
```

MemberPanel 오버레이:
```tsx
// AS-IS
<div className="fixed inset-y-0 right-0 z-50 flex w-72 flex-col bg-white shadow-xl ...">

// TO-BE
<div className="fixed inset-y-0 right-0 z-50 flex w-72 flex-col bg-white shadow-xl pt-[env(safe-area-inset-top)] pb-[env(safe-area-inset-bottom)] ...">
```

## 단계 6: 모바일 입력 포커스 줌 방지 (R2)

### 변경 내용
iOS Safari/PWA에서 textarea 포커스 시 자동 줌을 방지.

### 변경 파일
- `apps/web/src/domain/amoeba-talk/components/TalkMessageInput.tsx`

### 변경 방법
textarea에 `text-base` (16px) 적용 (iOS는 font-size < 16px일 때 자동 줌 발생):
```tsx
// AS-IS
className="w-full resize-none rounded-lg border ... text-sm text-gray-700 ..."

// TO-BE (모바일에서만 16px)
className="w-full resize-none rounded-lg border ... text-base md:text-sm text-gray-700 ..."
```

### 사이드 임팩트
- `text-base`는 모바일에서 입력 텍스트가 약간 커짐 (14px → 16px) — UX 관점에서 오히려 가독성 향상
- `maximum-scale=1`은 접근성 이슈가 있으므로 사용하지 않음

## 단계 8: 채널 목록 DM 온라인 상태 표시 (R8)

### 변경 내용
채널 목록에서 DM 채널의 상대방 아바타에 온라인 상태 dot 표시.

### 변경 파일
- `apps/web/src/domain/amoeba-talk/components/ChannelList.tsx`
- `packages/types/src/domain.types.ts` — `TalkChannelResponse`에 `dmPartnerUserId` 필드 추가 검토

### 제약사항
- `TalkChannelResponse`에는 `members` 배열이 없음 (성능상 채널 목록에는 요약 정보만 제공)
- DM 상대방의 userId를 알기 위한 방법:
  - **방법 A**: 백엔드에서 DM 채널 응답 시 `dmPartnerUserId` 필드 추가 (권장)
  - **방법 B**: 채널명 "user1, user2" 형식에서 이름으로 매칭 후 presenceMap 조회 (부정확)

### 백엔드 변경 (방법 A 채택)
`TalkChannelResponse`에 `dmPartnerUserId` 옵셔널 필드 추가:
- `packages/types/src/domain.types.ts` — `dmPartnerUserId?: string | null`
- `apps/api/src/domain/amoeba-talk/service/channel.service.ts` — DM 채널 조회 시 상대방 userId 포함
- `apps/api/src/domain/amoeba-talk/mapper/channel.mapper.ts` — 매퍼에 필드 추가

### 프론트엔드 변경
```tsx
// ChannelList.tsx renderChannel DM 항목
{isDm && (
  <div className="relative">
    <div className="flex h-7 w-7 ...">
      {displayName.charAt(0).toUpperCase()}
    </div>
    {/* 온라인 상태 dot */}
    <span className={`absolute -bottom-0.5 -right-0.5 h-2.5 w-2.5 rounded-full border-2 border-gray-50 ${
      presenceMap[channel.dmPartnerUserId || ''] === 'online' ? 'bg-green-500' : 'bg-gray-300'
    }`} />
  </div>
)}
```

## 구현 순서 및 의존관계

```
단계 4 (플레이스홀더)     ──┐
단계 2 (백엔드 포맷)      ──┤
단계 3 (프론트 번역표시)   ──┼── 독립적, 병렬 가능
단계 6 (줌 방지)          ──┤
                          ──┘
단계 1 (버튼 레이아웃)     ──── 단독
단계 5 (노치 safe-area)   ──── 단독
단계 7 (채널 수정)        ──── 단독 (store 변경 포함)
단계 8 (온라인 상태)      ──── 백엔드 선행 필요
```

### 권장 실행 순서
1. **단계 2 + 4 + 6** — 간단한 변경들 일괄 처리
2. **단계 1 + 3** — 입력창 레이아웃 + 번역표시 변경
3. **단계 5** — 모달 safe-area
4. **단계 7** — 채널 수정 모달 (신규 컴포넌트)
5. **단계 8** — DM 온라인 상태 (백엔드 + 프론트 + 타입)

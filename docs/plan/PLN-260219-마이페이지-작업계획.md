## 1. 작업 범위

### 백엔드 (NestJS)
- 내 프로필 상세 조회 API 신규 개발 (`GET /users/me/profile`)
- 기존 `POST /auth/change-password`는 재사용 (변경 없음)

### 프론트엔드 (React)
- 마이페이지 컴포넌트 신규 개발
- 라우터 등록
- 헤더에서 마이페이지 접근 링크 추가
- i18n 번역 파일 추가 (ko, en, vi)

### STEP 1: 백엔드 - 프로필 조회 API

**목적**: 사용자 본인의 상세 정보(부서, 법인 포함)를 반환하는 API 생성

**신규 파일**:
| 파일 | 설명 |
|------|------|
| `apps/api/src/domain/auth/dto/response/my-profile.response.ts` | 응답 DTO |

**수정 파일**:
| 파일 | 변경 내용 |
|------|----------|
| `apps/api/src/domain/auth/controller/user.controller.ts` | `GET /users/me/profile` 엔드포인트 추가 |
| `apps/api/src/domain/auth/service/user.service.ts` | `getMyProfile()` 메서드 추가 |
| `apps/api/src/domain/auth/auth.module.ts` | 필요 시 엔티티 import 추가 (UserDeptRole, EntityUserRole, Department, HrEntity) |

**구현 내용**:
```
1. UserService.getMyProfile(userId):
   - UserEntity 조회
   - UserDeptRoleEntity JOIN DepartmentEntity 조회 → 부서 목록
   - EntityUserRoleEntity JOIN HrEntityEntity 조회 → 법인 목록
   - 응답 DTO로 매핑하여 반환
```

## 3. 사이드 임팩트 분석

### 3.1 영향 없음 (안전)

| 영역 | 이유 |
|------|------|
| 기존 라우트 | 새 경로 `/my-page` 추가만 하므로 기존 라우트에 영향 없음 |
| 기존 API | 새 엔드포인트 추가만 하며, 기존 API 변경 없음 |
| 비밀번호 변경 | 기존 `POST /auth/change-password` 재사용, 변경 없음 |
| 사이드바 메뉴 | 사이드바에 메뉴 추가하지 않으므로 기존 메뉴 시스템 영향 없음 |
| DB 스키마 | 신규 테이블/컬럼 없음, 기존 테이블 조회만 수행 |

### 3.2 주의 필요 (낮은 위험)

| 영역 | 리스크 | 대응 |
|------|--------|------|
| `auth.module.ts` | 엔티티 import 추가 시 모듈 의존성 변경 | TypeOrmModule.forFeature에 추가할 엔티티가 이미 다른 모듈에서 사용 중이므로 안전 |
| 헤더 레이아웃 | 사용자 이름 클릭 동작 추가 | 기존에는 텍스트만 표시 → 클릭 가능 링크로 변경. 시각적 변화 최소화 (밑줄/커서만) |
| `domain.types.ts` | 타입 추가 | export만 추가하므로 기존 타입에 영향 없음 |
| i18n.ts | 네임스페이스 추가 | 기존 네임스페이스에 영향 없음, 추가만 수행 |

### 3.3 확인 필요

| 영역 | 확인 사항 |
|------|----------|
| UserDeptRoleEntity 데이터 | 실제 스테이징 DB에 `amb_user_dept_roles` 데이터가 있는지 확인 필요. 없으면 부서 섹션이 빈 상태로 표시됨 |
| EntityUserRoleEntity 데이터 | `amb_hr_entity_user_roles` 데이터 존재 여부 확인 |
| 새로 가입한 USER 역할 | 부서/법인 미배정 사용자의 경우 해당 섹션을 "미배정" 상태로 안내 |

## 5. 테스트 항목

| 항목 | 검증 내용 |
|------|----------|
| 프로필 조회 | 로그인 후 `/my-page` 접근 시 본인 정보 정상 표시 |
| 부서 정보 | 부서 배정된 사용자 → 부서 목록 표시 |
| 부서 미배정 | 부서 없는 사용자 → "배정된 부서가 없습니다" 메시지 |
| 법인 정보 | 법인 배정된 사용자 → 법인 목록 표시 |
| 비밀번호 변경 성공 | 현재 비밀번호 정확 + 새 비밀번호 8자 이상 → 변경 성공 |
| 비밀번호 변경 실패 | 현재 비밀번호 틀림 → 에러 메시지 표시 |
| 비밀번호 확인 불일치 | 새 비밀번호 ≠ 확인 → 클라이언트 검증 에러 |
| 권한 | USER, MANAGER, ADMIN 모두 접근 가능 |
| i18n | 언어 변경 시 마이페이지 텍스트 정상 변환 |
| 헤더 링크 | 사용자 이름 클릭 → `/my-page` 이동 |
| 반응형 | 모바일 뷰에서 레이아웃 정상 |

## 1. 시스템 현황 분석

### 1.1 기존 Admin 모듈 구조

- `AdminModule`: `admin.controller.ts` + `admin.service.ts` (단일 컨트롤러/서비스)
- `AdminLevelGuard` + `SuperAdminGuard`로 접근 제어
- TypeORM Entity 패턴: UUID PK, 3자 prefix snake_case, explicit column types
- 프론트엔드: React Query + Zustand + axios interceptor + sonner toast

### 1.2 에러 처리 현황

- `HttpExceptionFilter`: 응답 포맷팅만 (DB 기록 없음)
- `BusinessExceptionFilter`: 응답 포맷팅만 (DB 기록 없음)
- axios interceptor: 401 토큰 갱신만 처리
- ErrorBoundary: 없음
- 전역 에러 핸들러(`window.onerror`): 없음

## 4. 핵심 설계 결정

1. **SiteErrorController를 별도 컨트롤러로 분리**: AdminController가 이미 비대하므로 독립 컨트롤러로 분리
2. **POST /api/v1/site-errors는 Public**: 비인증 사용자 에러도 수집 (Optional Auth)
3. **GET/PATCH는 AdminLevelGuard**: 조회/수정은 관리자만
4. **ExceptionFilter에서 비동기 기록**: `ModuleRef`를 통해 SiteErrorService 접근 (NestJS 추천 패턴)
5. **ErrorReporter는 독립 유틸**: 스토어 의존 최소화, 안정성 확보

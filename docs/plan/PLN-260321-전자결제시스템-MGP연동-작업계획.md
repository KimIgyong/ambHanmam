**megapay.service.ts 주요 메서드:**

```typescript
class MegapayService {
  // 1. Payment Link 생성
  async createPaymentLink(dto: CreatePaymentLinkDto): Promise<PaymentLinkResult>
  //   - invoiceNo 생성 (유일), merTrxId 생성 (merId + timestamp)
  //   - merchantToken = SHA256(timeStamp + invoiceNo + merId + amount + linkExptime + encodeKey)
  //   - POST {baseUrl}/pg_was/createlink.do
  //   - 응답: paymentLink URL + qrCode(base64)
  //   - DB에 PENDING 트랜잭션 저장

  // 2. Callback 처리 (프론트 리다이렉트용)
  async handleCallback(params: CallbackParams): Promise<CallbackResult>
  //   - merchantToken 검증
  //   - 금액 검증 (요청 = 응답)
  //   - 트랜잭션 상태 업데이트
  //   - 거래 조회 API로 최종 확인

  // 3. IPN 처리 (서버 알림)
  async handleIpn(body: IpnBody): Promise<void>
  //   - merchantToken 검증
  //   - 멱등성: 이미 SUCCESS면 스킵
  //   - 금액 검증
  //   - 트랜잭션 상태 업데이트
  //   - 성공 시 → 쿼터 충전 이벤트 발행

  // 4. 거래 조회 (invoiceNo 기반)
  async queryTransaction(invoiceNo: string): Promise<TransactionStatus>
  //   - POST {baseUrl}/pg_was/order/trxStatusInvoiceNo.do
  //   - merchantToken = SHA256(merId + invoiceNo + timeStamp + encodeKey)

  // 5. 환불
  async refund(transactionId: string, reason: string): Promise<RefundResult>
  //   - POST {baseUrl}/pg_was/cancel/paymentCancel.do
  //   - hash = SHA256(trxId + cancelPw + timeStamp + hashKeyRefund)
}
```

**MegaPay Hash 유틸리티:**
```typescript
// megapay-hash.util.ts
import { createHash } from 'crypto';

export function megapayHash(...parts: string[]): string {
  return createHash('sha256').update(parts.join('')).digest('hex');
}

export function generateInvoiceNo(merId: string): string {
  // MegaPay 시스템에서 유일해야 함
  // 형식: {merId}{YYYYMMDD}{HHmmss}{random4}
  const now = new Date();
  const dateStr = format(now, 'yyyyMMddHHmmss');
  const random = Math.floor(Math.random() * 10000).toString().padStart(4, '0');
  return `${merId}${dateStr}${random}`;
}

export function generateMerTrxId(merId: string): string {
  return `${merId}${Date.now()}`;
}
```

### 3.2 보안 고려사항

| 항목 | 대응 |
|------|------|
| PG 인증 키 저장 | 기존 `CryptoService` AES-256-GCM 재사용. 키 3개 각각 개별 IV/Tag |
| merchantToken 위변조 | SHA256 검증 필수. 불일치 시 거래 거부 + 경고 로그 |
| IPN 엔드포인트 | `@Public()` + merchantToken 검증. IP 화이트리스트 선택 적용 |
| 결제 금액 위변조 | DB 요청 금액 ↔ IPN 응답 금액 비교 검증 |
| 중복 처리 | `pgt_invoice_no` UNIQUE + 상태 체크로 멱등성 보장 |
| 키 노출 방지 | 프론트엔드에 키 원문 전달하지 않음. last4만 표시 |
| SSRF 방지 | MegaPay API URL은 whitelist 된 도메인만 허용 (`sandbox.megapay.vn`, `pg.megapay.vn`) |

## 7. 위험 요소 및 대응

| 위험 | 영향 | 대응 |
|------|------|------|
| MegaPay Sandbox 접근 불가 | Phase 2 블로킹 | Phase 1 (설정 관리)은 Sandbox 없이 진행 가능. PG사에 Sandbox 계정 요청 |
| IPN 수신 실패 | 충전 누락 | 거래 조회 API 폴백 + 수동 확인 도구 제공 |
| 결제 중 세션 만료 | 결제 결과 미수신 | Callback URL에 트랜잭션 ID 포함, 재접속 시 상태 조회 |
| 동시 IPN + Callback | 중복 충전 | invoiceNo 기반 DB UNIQUE + 상태 비교 (이미 SUCCESS면 스킵) |
| MegaPay API 변경 | 연동 오류 | 요청/응답 전체를 JSONB 저장하여 디버깅 용이 |
| VND 금액 정밀도 | 계산 오류 | BIGINT 사용 (소수점 없음), 최소 10,000 VND 검증 |

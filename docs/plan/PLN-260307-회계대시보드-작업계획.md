```

- rex_is_active (boolean, default true)

- rex_created_at, rex_updated_at, rex_deleted_at
```

#### 2-3. 매퍼
**파일**: `apps/api/src/domain/accounting/mapper/recurring-expense.mapper.ts` (신규)

#### 2-4. 서비스
**파일**: `apps/api/src/domain/accounting/service/recurring-expense.service.ts` (신규)

- `getRecurringExpenses(entityId)` — 목록 조회
- `createRecurringExpense(dto, userId, entityId)` — 생성
- `updateRecurringExpense(id, dto, entityId)` — 수정
- `deleteRecurringExpense(id, entityId)` — 삭제 (Soft)
- `getForecast(entityId, month)` — 특정 월 예상 지출 생성
  - 활성 항목 중 해당 월에 유효한 항목 필터 (start_date <= month, end_date >= month OR null)
  - 통화별 합계 계산
  - 전월 실제 지출 비교 (거래처명 매칭)

## 사이드 임팩트

- recharts 번들 크기 증가 (~200KB gzipped) → lazy import로 완화
- 기존 계좌/거래 CRUD에는 영향 없음 (read-only 통계 API 추가)
- `app.module.ts` entities 배열에 1개 엔티티 추가
- 스테이징 DB에 `amb_recurring_expenses` 테이블 수동 DDL 필요

## 검증

1. `npm run build` 빌드 확인
2. `/accounting/dashboard` — 통합 대시보드 표시, 계좌 선택 후 통합 차트 확인
3. `/accounting/:accountId` — 통계 탭에서 월별 차트, 거래처 Top5 확인
4. 정기 지출 CRUD — 생성/수정/삭제/활성 토글
5. 예상 문서 — 월 선택 후 예상 지출 테이블 표시
6. 다국어 — ko/en/vi 전환 시 모든 레이블 정상 표시

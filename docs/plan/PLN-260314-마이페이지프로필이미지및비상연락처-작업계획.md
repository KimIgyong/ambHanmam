## 2. 현황 분석 기반 영향도

### 사이드 임팩트
- `MyProfileResponse` 타입 변경 → 이 타입을 사용하는 다른 영역 없음 (마이페이지 전용)
- `UserEntity` 컬럼 추가 → `synchronize: false` 이므로 자동 적용 안됨, SQL 마이그레이션 필수
- 기존 `GET /users/me/profile` 응답에 필드 추가 → 하위 호환 (기존 클라이언트 영향 없음)

## 3. 단계별 구현 계획

### Phase 1: DB 마이그레이션

**파일**: `apps/api/src/database/migrations/20260314-user-profile-image-phone.sql`

```sql
ALTER TABLE amb_users
  ADD COLUMN IF NOT EXISTS usr_phone       VARCHAR(30)  DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS usr_profile_image BYTEA      DEFAULT NULL;
```

### Phase 2: 백엔드

#### 2-1. `UserEntity` 컬럼 추가
파일: `apps/api/src/domain/auth/entity/user.entity.ts`
- `usrPhone: string | null`
- `usrProfileImage: Buffer | null`

#### 2-2. `UserController` API 추가
파일: `apps/api/src/domain/auth/controller/user.controller.ts`

추가 엔드포인트:
| Method | Path | 설명 |
|--------|------|------|
| `PATCH` | `/users/me/profile` | `{ phone }` 수정 |
| `POST` | `/users/me/profile-image` | multipart/form-data 이미지 업로드 |
| `GET` | `/users/me/profile-image` | 이미지 binary 응답 |
| `DELETE` | `/users/me/profile-image` | 이미지 삭제 |

- `GET /users/me/profile` 응답에 `phone`, `profileImageUrl` 필드 추가

### Phase 3: 공유 타입

파일: `packages/types/src/domain.types.ts`

`MyProfileResponse`에 추가:
```typescript
phone: string | null;
profileImageUrl: string | null; // "/api/v1/users/me/profile-image" 또는 null
```

### Phase 4: 프론트엔드

#### 4-1. `my-page.service.ts` API 메서드 추가
- `updateProfile({ phone })`  
- `uploadProfileImage(file)`  
- `deleteProfileImage()`  
- `getProfileImageUrl()` → `/users/me/profile-image`

#### 4-2. `useMyPage.ts` hooks 추가
- `useUpdateProfile()`
- `useUploadProfileImage()`
- `useDeleteProfileImage()`

#### 4-3. `ProfileSection.tsx` UI 개선
- 프로필 이미지 아바타 영역 (상단)
  - 이미지 있으면 `<img>` 표시, 없으면 이름 이니셜 컬러 아바타
  - 호버 시 카메라 아이콘 오버레이
  - 클릭 → `<input type="file" accept="image/png,image/jpeg">` 트리거
  - 이미지 선택 후 미리보기 → 저장 버튼 노출
  - 삭제 버튼 (이미지 있을 때만)
- 전화번호 필드
  - 표시 모드: 아이콘 + 값 또는 "미설정"
  - 편집 버튼 클릭 → input 활성화 → 저장/취소

#### 4-4. i18n 번역 추가 (3개 언어)
- `myPage:profile.phone` → 비상연락처
- `myPage:profile.phoneEdit` → 수정 / 저장 / 취소
- `myPage:profile.profileImage` → 관련 문구
- `myPage:profile.updateSuccess` → 수정 완료 메시지

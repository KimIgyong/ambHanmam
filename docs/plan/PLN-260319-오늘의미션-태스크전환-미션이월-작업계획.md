## 2. 변경 계획

### Phase 1: 백엔드 - 미체크 미션 조회 범위 확장

#### Task 1.1: `getYesterdayUncheckedMission` → `getLatestUncheckedMission` 변경

**파일**: `apps/api/src/domain/today/service/mission.service.ts`

**현재 코드** (L228~L241):
```typescript
async getYesterdayUncheckedMission(userId: string, timezone?: string) {
  const yesterdayStr = getLocalYesterday(timezone);
  const mission = await this.missionRepo.findOne({
    where: { usrId: userId, msnDate: yesterdayStr },
  });
  if (!mission || !mission.msnContent || mission.msnCheckResult) {
    return null;
  }
  return mission;
}
```

**변경 후**:
```typescript
async getLatestUncheckedMission(userId: string, timezone?: string) {
  const today = getLocalToday(timezone);
  // 7일 이내의 미체크 미션 중 가장 최근 것
  const mission = await this.missionRepo
    .createQueryBuilder('m')
    .where('m.usr_id = :userId', { userId })
    .andWhere('m.msn_date < :today', { today })
    .andWhere("m.msn_date >= :weekAgo", { weekAgo: this.getDateNDaysAgo(today, 7) })
    .andWhere('m.msn_content IS NOT NULL')
    .andWhere('m.msn_check_result IS NULL')
    .orderBy('m.msn_date', 'DESC')
    .getOne();
  return mission ?? null;
}

private getDateNDaysAgo(dateStr: string, days: number): string {
  const d = new Date(dateStr + 'T00:00:00');
  d.setDate(d.getDate() - days);
  return d.toISOString().split('T')[0];
}
```

**변경 이유**:
- `msnDate = yesterday` 고정 조건 → `msnDate < today AND msnDate >= 7일전` 범위로 확장
- 주말/공휴일/휴가 후 복귀 시에도 미체크 미션이 표시됨

### Phase 2: 프론트엔드 - 자동 스크롤 + UX 개선

#### Task 2.1: MissionSection에 ref 기반 자동 스크롤 구현

**파일**: `apps/web/src/domain/today/components/mission/MissionSection.tsx`

**변경 내용**:

1. **오늘 미션 에디터 영역에 `ref` 추가**:
```tsx
const missionEditorRef = useRef<HTMLDivElement>(null);
```

2. **`handleCheckSave` → `onSuccess`에서 스크롤 로직 추가**:
```tsx
const handleCheckSave = () => {
  if (!yesterdayMission || !canSave) return;
  const lines: MissionCheckLine[] = lineStates.map((l) => ({
    lineIndex: l.lineIndex,
    text: l.text,
    state: l.state as 'done' | 'incomplete' | 'na',
    subChoice: l.subChoice,
  }));
  
  // carry-over 항목이 있는지 미리 확인
  const hasCarryOver = lineStates.some(l => l.state === 'incomplete' && l.subChoice === 'mission');
  
  saveMissionCheck.mutate(
    { date: yesterdayMission.msnDate, lines },
    {
      onSuccess: (data) => {
        // carry-over 항목이 있으면 에디터에 즉시 반영 + 자동 스크롤
        if (hasCarryOver && data?.carryOverText) {
          setContent(data.carryOverText);
        }
        if (hasCarryOver) {
          // 쿼리 무효화 후 리렌더 대기 → 스크롤
          setTimeout(() => {
            missionEditorRef.current?.scrollIntoView({ behavior: 'smooth', block: 'start' });
          }, 300);
        }
      },
    },
  );
};
```

3. **오늘 미션 에디터 영역을 `ref`로 감싸기**:
```tsx
{!mission && (
  <div ref={missionEditorRef}>
    {/* 오늘 미션 에디터 기존 내용 */}
  </div>
)}
```

**주의**: `saveMissionCheck` 성공 후 `invalidateQueries`가 실행되어 `data`가 리페치됨. 리페치 완료 후 `yesterdayMission`이 `null`이 되고 오늘 미션 에디터가 표시됨. `setTimeout`으로 리렌더 후 스크롤 보장.

## 4. 사이드 임팩트 분석

| 영역 | 리스크 | 대응 |
|------|--------|------|
| 미션 체크 API | `saveCheck()`의 date 파라미터가 어제가 아닌 날짜로 올 수 있음 | `saveCheck` 이미 date 기반으로 동작 (제약 없음) |
| 기존 체크 데이터 | 기존 완료된 미션은 `msnCheckResult`가 있으므로 조회 대상 아님 | 영향 없음 |
| 스냅샷 시스템 | 미체크 미션 자체는 스냅샷과 무관 | 영향 없음 |
| 프론트 리렌더 | 체크 저장 후 query invalidation → 리렌더 중 스크롤 타이밍 | `setTimeout(300ms)` 사용 |
| 프론트 `carryOverText` | 현재 `useEffect`로 `carryOverText` pre-fill | mutation `onSuccess`에서 직접 set으로 보완 |

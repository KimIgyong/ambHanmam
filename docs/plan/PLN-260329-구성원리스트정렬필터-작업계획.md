### Phase 2: MemberList 컴포넌트 리팩토링
1. **필터 상태 추가** (`useState`)
   - `sortBy`: `'joinedAt_desc'` (기본)
   - `selectedRoles`: `string[]` (빈 배열 = 전체)
   - `selectedUnit`: `string` (빈 문자열 = 전체)
   - `selectedStatuses`: `string[]` (기본 `['ACTIVE']`)

2. **useOrgUnits 호출 추가**
   - MemberList 내에서 유닛 목록 조회

3. **useMemo로 필터/정렬 처리**
   ```typescript
   const filteredMembers = useMemo(() => {
     let result = members || [];
     // 상태 필터
     if (selectedStatuses.length > 0)
       result = result.filter(m => selectedStatuses.includes(m.status));
     // 역할 필터
     if (selectedRoles.length > 0)
       result = result.filter(m => selectedRoles.includes(m.role));
     // 유닛 필터
     if (selectedUnit)
       result = result.filter(m => 
         m.unitRoles?.some(ur => ur.unitId === selectedUnit) || false);
     // 정렬
     result = [...result].sort(sortFn);
     return result;
   }, [members, selectedStatuses, selectedRoles, selectedUnit, sortBy]);
   ```

4. **필터 바 UI** (테이블 상단)
   - 정렬 드롭다운 (단일 선택)
   - 역할 멀티셀렉트 드롭다운 (체크박스)
   - 유닛 드롭다운 (단일 선택)
   - 상태 멀티셀렉트 드롭다운 (체크박스)

5. **테이블 레이아웃 변경**
   - Email + Name 2개 컬럼 → 1개 "이름/이메일" 컬럼으로 병합
   - 상단: 이름 (font-medium), 하단: 이메일 (text-xs text-gray-500)

- 사이드 임팩트: MemberList UI 변경. MemberDetailModal 영향 없음 (onSelect로 member 객체 전달은 동일).

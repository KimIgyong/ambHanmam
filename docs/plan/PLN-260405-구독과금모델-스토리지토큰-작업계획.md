### Phase 3. 토큰 시스템 통합 (P0)

**목표**: 기존 AI 쿼터 시스템과 신규 토큰 지갑 시스템 연동

#### 2.3.1 TokenBalanceGuard 적용

```typescript
// guard/token-balance.guard.ts
@Injectable()
export class TokenBalanceGuard implements CanActivate {
  async canActivate(context: ExecutionContext): Promise<boolean> {
    const entId = user.entityId;
    const balance = await this.tokenWalletService.getTotalBalance(entId);
    if (balance <= 0) {
      throw new ForbiddenException({ code: 'E7001', message: 'Insufficient token balance.' });
    }
    request['tokenBalance'] = balance;
    return true;
  }
}
```

적용 대상: AI 채팅/에이전트 요청 컨트롤러 (`@UseGuards(TokenBalanceGuard)`)

#### 2.3.2 기존 ApiQuotaInterceptor와의 공존

이행 기간 전략:
1. `TokenBalanceGuard` 우선 적용 (지갑 잔액 검사)
2. `ApiQuotaInterceptor` 유지 (일일 한도 안전장치)
3. 토큰 지갑으로 완전 이행 후 ApiQuotaInterceptor 제거

#### 2.3.3 AI 사용 후 차감 연동

## 4. Side Impact Analysis (사이드 임팩트 분석)

### 4.1 기존 service-management 도메인

| 영향 | 심각도 | 대응 |
|---|---|---|
| `amb_svc_plans`/`amb_svc_subscriptions` | **없음** | 기존 테이블 변경하지 않음. 신규 `amb_sub_*` prefix로 완전 분리 |
| ServiceManagement 서비스 쿼리 | **없음** | 수정 불필요 |

### 4.2 기존 AI 쿼터 시스템

| 영향 | 심각도 | 대응 |
|---|---|---|
| ApiQuotaInterceptor 공존 | 중간 | 이행 기간 동안 양쪽 모두 활성화, 완전 전환 후 Interceptor 제거 |
| EntityAiConfigEntity 데이터 | 낮음 | 기존 daily/monthly limit은 안전장치로 유지 |
| AiQuotaTopupService (MegaPay) | 낮음 | MegaPay 경로도 당분간 유지 |

### 4.3 기존 PolarModule 폐기

| 영향 | 심각도 | 대응 |
|---|---|---|
| 기존 polar 엔드포인트 경로 변경 | 중간 | `/api/v1/polar/*` → `/api/v1/subscriptions/*` + `/api/v1/webhooks/polar` |
| 기존 `amb_pol_webhook_events` | 낮음 | 이행 후 테이블 폐기 가능, 데이터 보존 |

### 4.4 FileService / DriveController

| 영향 | 심각도 | 대응 |
|---|---|---|
| 업로드 시 추가 검사 오버헤드 | 낮음 | 쿼터 확인은 단일 SELECT (DB 부하 미미) |
| 기존 업로드 에러 처리와 충돌 | 낮음 | E7002 에러를 기존 에러보다 먼저 검사 |
| 순환 의존성 | 중간 | SubscriptionModule을 export하고 `forwardRef` 사용 |

### 4.5 보안

| 항목 | 대응 |
|---|---|
| Polar webhook 서명 검증 | Svix HMAC-SHA256, 5분 타임윈도우, rawBody 필수 |
| POLAR_ACCESS_TOKEN 노출 방지 | env 파일에만 저장, 로그 출력 금지 |
| `ent_id` 데이터 격리 | 모든 구독/토큰/스토리지 쿼리에 ent_id 필터 필수 |
| 멱등성 보장 | webhook-id 기반 중복 처리 방지 |

| 10 | Controller: `@Auth()` + `@UseGuards(OwnEntityGuard)` + `resolveEntityId()` | ☑ |

| 14 | 인덱스: `idx_{table}_{column}` | ☑ |

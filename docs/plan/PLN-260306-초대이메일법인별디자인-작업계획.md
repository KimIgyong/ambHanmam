## 1. 현황 분석 요약

### 핵심 문제
- `MailService.sendInvitationEmail()`이 법인 정보 없이 단일 고정 템플릿 사용
- 초대 이메일 제목/본문에 법인명 미포함
- 법인별 브랜드 아이덴티티(색상, 로고) 적용 불가

### 현재 호출 시그니처
```typescript
// mail.service.ts - AS-IS
async sendInvitationEmail(
  to: string,
  inviterName: string,
  role: string,
  department: string,
  token: string,
): Promise<boolean>

// invitation.service.ts - AS-IS (create/resend 공통)
await this.mailService.sendInvitationEmail(
  dto.email,
  inviter?.usrName || 'Admin',
  dto.role,
  dto.department,
  token,
);
```

### 2.1 Phase 1 — 법인명 이메일 반영 (즉시 구현, DB 마이그레이션 없음)

#### 2.1.1 `mail.service.ts` 수정

**AS-IS:**
```typescript
async sendInvitationEmail(
  to: string,
  inviterName: string,
  role: string,
  department: string,

): Promise<boolean>
```

**TO-BE:**
```typescript
interface InvitationEmailCompanyInfo {

  color?: string;     // Phase 2: ent_email_brand_color

async sendInvitationEmail(
  to: string,
  inviterName: string,
  role: string,
  department: string,
  token: string,
  companyInfo?: InvitationEmailCompanyInfo,  // 추가 (optional)

```

**템플릿 변경:**
```typescript
const companyName = companyInfo?.name || 'AMB Management';
const brandColor = companyInfo?.color || '#4F46E5';
```

### 2.2 Phase 2 — 법인별 브랜드 이메일 커스터마이징 (DB 마이그레이션 포함)

#### 작업 목록

| 순서 | 파일/작업 | 변경 내용 |
|------|-----------|-----------|
| 1 | DB 마이그레이션 | `amb_hr_entities` 테이블 컬럼 3개 추가 |
| 2 | `hr-entity.entity.ts` | 신규 컬럼 매핑 추가 |
| 3 | `mail.service.ts` | `companyInfo` 인터페이스에 `color`, `logoUrl` 활용 |
| 4 | `invitation.service.ts` | `companyInfo`에 색상/로고 포함 |
| 5 | 관리자 API | 법인 이메일 메타 설정 조회/수정 엔드포인트 |
| 6 | 관리자 UI | 법인 설정 페이지에 이메일 브랜딩 섹션 추가 |

#### 2.2.1 DB 마이그레이션

```sql
ALTER TABLE amb_hr_entities
  ADD COLUMN IF NOT EXISTS ent_email_display_name VARCHAR(200) NULL,
  ADD COLUMN IF NOT EXISTS ent_email_brand_color  VARCHAR(10)  NULL DEFAULT '#4F46E5',
  ADD COLUMN IF NOT EXISTS ent_email_logo_url     VARCHAR(500) NULL;

COMMENT ON COLUMN amb_hr_entities.ent_email_display_name IS '초대 이메일 표시 법인명 (미설정 시 ent_name_en 또는 ent_name 사용)';
COMMENT ON COLUMN amb_hr_entities.ent_email_brand_color  IS '초대 이메일 브랜드 색상 (hex #RRGGBB)';
COMMENT ON COLUMN amb_hr_entities.ent_email_logo_url     IS '초대 이메일 로고 이미지 URL';
```

#### 2.2.2 `hr-entity.entity.ts` 컬럼 추가

```typescript
@Column({ name: 'ent_email_display_name', length: 200, nullable: true })
```

## 4. 단계별 구현 순서

### Phase 1 (데이터베이스 변경 없음, 약 2시간)

```
Step 1. invitation.module.ts
  → TypeOrmModule에 HrEntityEntity 추가
  → InvitationService 생성자에 hrEntityRepository 주입

Step 2. mail.service.ts
  → InvitationEmailCompanyInfo 인터페이스 정의
  → sendInvitationEmail() 파라미터 추가
  → 이메일 템플릿 수정 (제목 + 본문 법인명 동적 반영)

Step 3. invitation.service.ts
  → create(): company 조회 + companyInfo 구성 + sendInvitationEmail에 전달
  → resend(): company 조회 + companyInfo 구성 + sendInvitationEmail에 전달

Step 4. 테스트
  → 법인 있는 초대: 법인명 표시 확인
  → 법인 없는 초대(companyId null): "AMB Management" fallback 확인
  → 재발송: 동일 법인명 표시 확인
```

### Phase 2 (데이터베이스 변경 포함, 약 4시간 추가)

```
Step 5. SQL 마이그레이션 실행
  → 개발 DB에 컬럼 추가
  → hr-entity.entity.ts 컬럼 매핑 추가

Step 6. invitation.service.ts 확장
  → companyInfo에 color, logoUrl 포함

Step 7. mail.service.ts 확장
  → 브랜드 색상 + 로고 URL HTML 템플릿 반영

Step 8. 관리자 UI (HrEntitySettings)
  → 법인 설정 페이지에 '이메일 브랜딩' 섹션 추가
  → 표시명 / 브랜드 색상 (컬러피커) / 로고 URL 입력 폼

Step 9. 스테이징 배포 및 테스트
```

## 5. 테스트 시나리오

| 시나리오 | 기대 결과 |
|----------|----------|
| `companyId = null`인 초대 발송 | 제목: "AMB Management - Invitation", 본문: "join AMB Management" |
| `companyId = AMB_KR (entName="AMB Korea")`인 초대 발송 | 제목: "[AMB Korea] AMB Management - Invitation", 본문: "join AMB Korea" |
| `entNameEn = "AMB Korea"`, `entName = "에이엠비코리아"` | 영문명 우선 사용: "AMB Korea" |

| 재발송(resend) | 동일 법인명 적용 확인 |
| Phase 2: 브랜드 색상 `#FF6B35` 설정 | 버튼/헤더 색상 오렌지 적용 |
| Phase 2: 로고 URL 설정 | 이미지 표시, 텍스트 헤더 미표시 |

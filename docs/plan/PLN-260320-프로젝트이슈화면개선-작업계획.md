| 항목 | AS-IS | TO-BE |
|------|-------|-------|

## 3. 구현 단계

### Step 1: `IssueResponse`에 `redmineId` 필드 추가
- `packages/types/src/domain.types.ts` — `refNumber` 뒤에 `redmineId: number | null` 추가
- `apps/api/src/domain/issues/mapper/issue.mapper.ts` — `redmineId: entity.issRedmineId || null` 추가

### Step 2: 상태 필터 동적 생성 (ProjectIssuesTab)
- 전체 이슈에서 상태 목록 추출 → `STATUS_ORDER` 순서로 정렬 → 필터 드롭다운 렌더링
- 테이블에 참조번호 컬럼 추가 (# 열에 ISN-xxx)

### Step 3: 이슈 상세 참조번호 + Redmine# 표시
- 프로젝트 행에 참조번호 배지 + Redmine# 배지 (있을 때만)
- 기존 헤더의 참조번호 배지는 유지 또는 제거 (프로젝트 옆으로 이동)

### Step 4: Assignee 없으면 Reporter 지정
- `IssueService.createIssue` — `issAssigneeId: dto.assignee_id || userId`
- `ClientIssueService.createIssue` — `issAssigneeId: userId`
- `RedmineMigrationService` — `issAssigneeId: assigneeId || reporterId`

### Step 5: MASTER 수정 권한 추가
- **Backend** `issue.service.ts updateIssue()`: `MASTER`, `SUPER_ADMIN` 추가
- **Frontend** `IssueDetailModal.tsx`: `isManager` 정의에 `MASTER`, `SUPER_ADMIN` 포함

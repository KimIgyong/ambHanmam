## 2. 미조치 항목 긴급도 순위 (총 7건)

### 순위 산정 기준
- **보안 위험도**: 공격 가능성 × 피해 범위
- **프로덕션 영향**: 배포 전 필수 vs 운영 개선
- **구현 복잡도**: 사이드 임팩트 및 작업 규모

---

### 🔴 Priority 1: 비밀번호 복잡성 규칙 추가 (보안)
> 원본: MEDIUM 4.1 (부분조치)

**현황**: MinLength(8) 통일은 완료되었으나, 대/소문자/숫자/특수문자 복잡성 검증이 없음
**위험**: 무차별 대입(brute-force) 공격에 취약. `12345678` 같은 약한 비밀번호 허용

**작업내용**:
| 단계 | 내용 | 파일 |
|------|------|------|
| 1 | 비밀번호 복잡성 Regex Validator 추가 | `apps/api/src/domain/auth/dto/request/*.ts` |
| 2 | 최소 8자 + 대문자1 + 소문자1 + 숫자1 + 특수문자1 | 4개 DTO (login, register, reset-password, change-password) |
| 3 | 프론트엔드 비밀번호 유효성 메시지 | `apps/web/src/domain/auth/` 관련 폼 |
| 4 | i18n 에러 메시지 추가 | `locales/ko/en/vi` |

**사이드 임팩트**: 기존 사용자 로그인에는 영향 없음 (비밀번호 변경/등록 시에만 적용)

**사이드 임팩트**: 메일 서식(스타일, 이미지) 깨짐 가능성 → allowedTags/Attributes 충분 확인 필요

**위험**: DB 침해 시 Postal 메일 계정 접근 가능

**작업내용**:

# 에디터 체크리스트(TaskList) 기능 추가 작업계획

## 1. AS-IS 현황

### 1.1 현재 에디터 구조
- **컴포넌트**: `apps/web/src/domain/meeting-notes/components/RichTextEditor.tsx`
- **에디터 엔진**: TipTap v3 (ProseMirror 기반)
- **현재 확장(Extensions)**:
  - `StarterKit` (bold, italic, heading, bulletList, orderedList 등 기본 포함)
  - `Underline`
  - `Image` (드래그&드롭, 붙여넣기, Google Drive 연동)
  - `Link`
  - `Placeholder`
- **사용처**: Meeting Notes, Issues, Todos, Notices, Mission (총 7개 컴포넌트)
- **데이터 포맷**: HTML string (DB 저장/조회)

### 1.2 현재 툴바 버튼
`Bold | Italic | Underline | Heading2 | Image | Attach | Link | Undo | Redo | Source`

## 2. TO-BE 요구사항

### 2.1 기능
- 에디터 내에서 **체크박스가 있는 리스트(체크리스트)** 삽입
- 각 항목을 클릭하여 완료/미완료 토글
- 회의록/메모 작성 시 **Action Items, 할일 목록** 등에 활용
- 저장된 노트 열람 시에도 체크 상태 확인 가능

### 2.2 UX
- 툴바에 **체크리스트 버튼(☑)** 추가
- 클릭 시 현재 커서 위치에 체크리스트 삽입
- 이미 체크리스트일 때 다시 클릭하면 일반 텍스트로 전환 (토글)
- Enter 키로 다음 체크항목 자동 생성 (TipTap 기본 동작)

## 3. 기술 분석

### 3.1 TipTap TaskList 확장
TipTap에서 공식 제공하는 확장 2개로 구현:
- `@tiptap/extension-task-list` — `<ul data-type="taskList">` 컨테이너
- `@tiptap/extension-task-item` — `<li data-type="taskItem" data-checked="true/false">` 개별 항목

### 3.2 생성되는 HTML (DB 저장 포맷)
```html
<ul data-type="taskList">
  <li data-type="taskItem" data-checked="true"><p>완료된 항목</p></li>
  <li data-type="taskItem" data-checked="false"><p>미완료 항목</p></li>
</ul>
```
- 기존 HTML 기반 저장 방식과 **100% 호환** (추가 직렬화 불필요)
- 기존 데이터에 영향 없음 (새로운 data-type 속성만 추가됨)

### 3.3 스타일링
TipTap TaskList는 기본 스타일이 없으므로 CSS 추가 필요:
- 체크박스 렌더링 (input[type="checkbox"])
- 완료 항목 취소선(line-through) + 반투명 처리
- list-style-type: none (기본 불릿 제거)

### 3.4 읽기 전용 뷰 호환성
- 저장된 HTML을 `dangerouslySetInnerHTML`로 렌더링하는 읽기 전용 뷰에서는 체크박스가 `<input type="checkbox">` 로 자동 렌더링됨
- 읽기 전용에서는 체크 상태 변경 불필요 (disabled 또는 CSS로 처리)

## 4. 구현 계획

### Step 1: 패키지 설치
```bash
npm install -w @amb/web @tiptap/extension-task-list @tiptap/extension-task-item
```

### Step 2: RichTextEditor.tsx 수정
1. **Import 추가**
   - `import TaskList from '@tiptap/extension-task-list'`
   - `import TaskItem from '@tiptap/extension-task-item'`
   - Lucide 아이콘: `ListChecks` (체크리스트 버튼용)

2. **Extensions 배열에 추가**
   ```typescript
   TaskList,
   TaskItem.configure({ nested: true }),
   ```

3. **툴바 버튼 추가**
   - Heading2 버튼 옆 (구분선 뒤)에 배치
   - `editor.chain().focus().toggleTaskList().run()`
   - `editor.isActive('taskList')` 로 활성화 상태 표시

### Step 3: CSS 스타일 추가
EditorContent의 className에 TailwindCSS 유틸리티 또는 글로벌 CSS로 추가:
```css
/* 에디터 내 체크리스트 스타일 */
ul[data-type="taskList"] {
  list-style: none;
  padding-left: 0;
}
ul[data-type="taskList"] li {
  display: flex;
  align-items: flex-start;
  gap: 0.5rem;
}
ul[data-type="taskList"] li > label {
  flex-shrink: 0;
  margin-top: 0.25rem;
}
ul[data-type="taskList"] li[data-checked="true"] > div > p {
  text-decoration: line-through;
  color: #9ca3af;
}
```

### Step 4: 읽기 전용 뷰 스타일
Meeting Notes 상세보기 등에서 HTML을 표시하는 영역에도 동일한 CSS 적용하여 체크리스트 렌더링 지원

## 6. 사이드 임팩트 분석

| 항목 | 영향 |
|------|------|
| **기존 데이터** | 영향 없음. 기존 HTML에 taskList 관련 마크업이 없으므로 무관 |
| **RichTextEditor 사용처 7곳** | 추가 확장이 에디터에 자동 포함됨. 모든 사용처에서 체크리스트 사용 가능 (의도된 동작) |
| **번들 크기** | task-list + task-item = ~10KB gzip. 미미한 증가 |
| **백엔드** | 변경 없음. HTML string 그대로 저장 |
| **i18n** | 툴바 버튼은 title 속성만 사용 (영문 "Checklist" 또는 아이콘만) |

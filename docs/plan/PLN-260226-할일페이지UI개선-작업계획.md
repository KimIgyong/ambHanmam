## 2. Phase 1: UI 즉시 개선

### 2.1 TodoItem.tsx 변경

**변경 내용**:
1. 제목 텍스트 → 클릭 가능하게 변경 (`cursor-pointer`, `hover:underline`)
2. 클릭 시 `onEdit(todo)` 호출
3. 연필(Pencil) 아이콘 버튼 제거

**변경 전**:
```tsx
<div className="text-sm font-medium text-gray-900">
  {todo.title}
</div>
// ...
<button onClick={() => onEdit(todo)}>
  <Pencil className="h-4 w-4" />
</button>
```

**변경 후**:
```tsx
<button type="button" onClick={() => onEdit(todo)}
  className="text-sm font-medium text-gray-900 cursor-pointer hover:underline text-left">
  {todo.title}
</button>
```

### 2.2 TodoPage.tsx 3컬럼 레이아웃

**변경 내용**:
기존 2섹션(overdue/today) → 3컬럼(완료/오늘할일/예정) 칸반 레이아웃

**분류 로직 변경**:
```typescript
// AS-IS: overdueTodos (dueDate < today && !COMPLETED), todayTodos (나머지)
// TO-BE:
const completedTodos = todos.filter(t => t.status === 'COMPLETED');
const todayTodos = todos.filter(t => t.status !== 'COMPLETED' && t.dueDate <= today);
const scheduledTodos = todos.filter(t => t.status !== 'COMPLETED' && t.dueDate > today);
```

**레이아웃**: `max-w-3xl` → `max-w-6xl`, 3컬럼 grid
```
grid grid-cols-1 md:grid-cols-3 gap-4
```

각 컬럼:
- 완료 (초록색 헤더, 완료 스타일)
- 오늘 할일 (파란색 헤더, 기한초과 강조)
- 예정 (회색 헤더)

## 3. Phase 2: 관련자 등록 기능

### 3.1 DB 테이블

```sql
CREATE TABLE amb_todo_participants (
  tpt_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tdo_id UUID NOT NULL REFERENCES amb_todos(tdo_id),
  usr_id UUID NOT NULL REFERENCES amb_users(usr_id),
  tpt_role VARCHAR(20) NOT NULL DEFAULT 'WATCHER',
  tpt_created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_tpt_todo ON amb_todo_participants(tdo_id);
CREATE INDEX idx_tpt_user ON amb_todo_participants(usr_id);
```

### 3.2 백엔드

**신규 파일**:
- `todo-participant.entity.ts` - TypeORM 엔티티
  
**변경 파일**:

- `todo.service.ts` - 관련자 CRUD + 할일 조회시 participants 포함
- `todo.controller.ts` - 관련자 추가/삭제 API
- `todo.mapper.ts` - participants 매핑
- `todo.module.ts` - 새 엔티티 등록
- `create-todo.request.ts` - participants 필드
- `update-todo.request.ts` - participants 필드

**API 엔드포인트**:
- `POST /todos/:id/participants` - 관련자 추가
- `DELETE /todos/:id/participants/:userId` - 관련자 삭제

**간소화 방안**: 할일 생성/수정 시 `participant_ids` 배열을 함께 전달하여 sync 방식으로 처리

## 2. 상세 구현 계획

### Step 1: 데이터베이스 (0.5일)

**`sql/notification.sql`** 작성:
```sql
CREATE TABLE amb_notifications (
    ntf_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    ntf_type VARCHAR(30) NOT NULL,
    ntf_title VARCHAR(500) NOT NULL,
    ntf_message TEXT,
    ntf_recipient_id UUID NOT NULL REFERENCES amb_users(usr_id),
    ntf_sender_id UUID NOT NULL REFERENCES amb_users(usr_id),
    ntf_resource_type VARCHAR(30) NOT NULL,
    ntf_resource_id UUID NOT NULL,
    ntf_is_read BOOLEAN DEFAULT FALSE,
    ntf_read_at TIMESTAMP WITH TIME ZONE,
    ent_id UUID NOT NULL REFERENCES amb_entities(ent_id),
    ntf_created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    ntf_deleted_at TIMESTAMP WITH TIME ZONE
);
```

### Step 3: 도메인 서비스 이벤트 emit (1일)

**수정 대상 + 이벤트 발행 지점:**

| 서비스 | 메서드 | 이벤트 |
|--------|--------|--------|
| `TodoService` | `createTodo()` → `syncParticipants()` 후 | `assignee.assigned` (참여자 전원) |
| `TodoService` | `updateTodo()` → `syncParticipants()` 후 | `assignee.assigned` (신규 추가 참여자만) |
| `IssueService` | `createIssue()` → `issAssigneeId` 설정 후 | `assignee.assigned` (담당자) |
| `IssueService` | `updateIssue()` → `assignee_id` 변경 시 | `assignee.assigned` (새 담당자) |
| `MeetingNoteService` | `createMeetingNote()` | `assignee.assigned` (담당자 + 참여자) |
| `MeetingNoteService` | `updateMeetingNote()` | `assignee.assigned` (신규 추가 대상) |

**이벤트 페이로드:**
```typescript
this.eventEmitter.emit('assignee.assigned', {
  type: 'TODO_ASSIGNED',
  resourceType: 'TODO',
  resourceId: todo.tdoId,
  resourceTitle: todo.tdoTitle,
  senderId: currentUser.usrId,
  senderName: currentUser.usrName,
  recipientIds: newParticipantIds.filter(id => id !== currentUser.usrId),
  entityId: currentUser.entId,
  role: 'PARTICIPANT',
});
```

| Method | Endpoint | 설명 | Request | Response |
|--------|----------|------|---------|----------|

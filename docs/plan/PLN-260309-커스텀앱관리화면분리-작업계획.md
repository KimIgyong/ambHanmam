# 작업 계획서: 커스텀앱 관리 화면 분리 (USER_LEVEL / ADMIN_LEVEL)

- **문서번호**: PLAN-커스텀앱관리화면분리-작업계획-20260309
- **작성일**: 2026-03-09
- **참조**: REQ-커스텀앱관리화면분리-20260309

---

## 1. 현황 요약

- **USER_LEVEL**: `/entity-settings/custom-apps` 에서 자기 법인 커스텀앱 관리 → **구현 완료, 변경 없음**
- **ADMIN_LEVEL**: 커스텀앱 관리 화면 없음 → **신규 구현 필요**
- **백엔드 API**: `resolveEntityId()`로 `entity_id` 쿼리 파라미터 지원 → **변경 불필요**

---

## 2. 구현 계획

### Step 1: ADMIN 커스텀앱 관리 페이지 생성

**신규 파일**: `apps/web/src/domain/settings/pages/AdminCustomAppsPage.tsx`

**기능**:
- 헤더: 뒤로가기 버튼 + 타이틀 + 법인 선택 드롭다운
- 법인 선택 시 해당 법인의 커스텀앱 목록 조회
- 기존 `EntityCustomAppsPage.tsx`와 동일한 CRUD 기능
- API 호출 시 `entity_id` 쿼리 파라미터 전달

**구현 방식**:
- 기존 EntityCustomAppsPage의 로직을 참조하되, ADMIN 전용 페이지로 신규 작성
- 법인 목록: 기존 `GET /hr/entities` API 활용 (useEntities 훅)
- 커스텀앱 API: 기존 entity-settings.service.ts에 entityId 파라미터 추가

### Step 2: 프론트엔드 서비스/훅 수정

**수정 파일**: `apps/web/src/domain/entity-settings/service/entity-settings.service.ts`

**변경 내용**:
- `getCustomApps(entityId?: string)` — entity_id 쿼리 파라미터 옵션 추가
- `createCustomApp(dto, entityId?: string)` — 동일
- `updateCustomApp(id, dto, entityId?: string)` — 동일
- `deleteCustomApp(id, entityId?: string)` — 동일
- `healthCheckApp(id, entityId?: string)` — 동일

**수정 파일**: `apps/web/src/domain/entity-settings/hooks/useEntitySettings.ts`

**변경 내용**:
- `useCustomApps(entityId?: string)` — 쿼리 키에 entityId 포함, API 호출 시 전달
- mutation 훅들: entityId 파라미터 지원

### Step 3: SettingsPage에 카드 추가

**수정 파일**: `apps/web/src/domain/settings/pages/SettingsPage.tsx`

**변경 내용**:
- SETTINGS_CARDS 배열에 커스텀앱 관리 카드 추가
- icon: AppWindow, path: `/admin/custom-apps`
- 메뉴코드: `ENTITY_CUSTOM_APPS` 활용 (ADMIN은 기본 접근 가능)

### Step 4: 라우트 추가

**수정 파일**: `apps/web/src/router/index.tsx`

**변경 내용**:
```tsx
{
  path: 'admin/custom-apps',
  element: <AdminGuard><AdminCustomAppsPage /></AdminGuard>,
}
```

### Step 5: i18n 번역 추가

**수정 파일**: `apps/web/src/locales/{en,ko,vi}/settings.json`

**추가 키**:
```json
"customApps": {
  "title": "Custom Apps Management",
  "description": "Manage custom apps for each entity"
}
```

**수정 파일**: `apps/web/src/locales/{en,ko,vi}/common.json`

**추가 키**:
```json
"settingsPage": {
  "customApps": {
    "title": "Custom Apps",
    "description": "Manage custom apps registered by entities"
  }
}
```

## 4. 구현 순서

```
Step 2 (서비스/훅 수정) ─── entityId 파라미터 추가
         ↓
Step 1 (AdminCustomAppsPage 생성) ─── 법인 선택 + CRUD UI
         ↓
Step 3 (SettingsPage 카드 추가) ─── 메뉴 진입점
Step 4 (라우트 추가) ─── 네비게이션 연결
Step 5 (i18n) ─── 번역 키
```

Step 3, 4, 5는 독립적이므로 병렬 가능.

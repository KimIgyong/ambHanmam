## 1. AS-IS 현황

### 1.1 현재 LandingPage.tsx
- 간단한 Hero + 3개 서비스 카드 구조 (113줄)
- CMS 모드/STATIC 모드 토글 로직 포함 (`cmsApi.getPage('home')`)
- `react-i18next`의 `useTranslation()` 사용 (기존 `common.json` 키)
- `PublicLayout.tsx`의 `<Outlet />`으로 렌더링 (공용 헤더/푸터 감싸)

### 1.2 현재 PublicLayout.tsx
- 고정 헤더: 로고, Pricing 링크, 언어 전환, Login/Register 버튼
- 간단 푸터: `© 2026 Amoeba Company. All rights reserved.`
- **AmoebaLanding.jsx는 자체 네비게이션/푸터를 내장** → 충돌 이슈 있음

### 1.3 현재 i18n
- `en/common.json`, `ko/common.json`, `vi/common.json` — 단일 네임스페이스
- 랜딩 관련 키: `hero.*`, `services.*`, `nav.*`, `footer.*`

### 1.4 현재 Tailwind Config
- `primary` 컬러: indigo 계열 (#6366F1)
- 폰트: Pretendard
- AmoebaLanding은 **`orange`** 계열 + `Plus Jakarta Sans` + `Noto Sans KR` 사용

| **Footer** | 다크 4컬럼 푸터 | 698~722 |

### 2.2 의존성
- **React**: useState, useEffect, useRef, useCallback
- **lucide-react**: Globe, ChevronDown, Menu, X, ArrowRight 등 16개 아이콘
- **TailwindCSS**: 모든 스타일링
- **커스텀 CSS**: `@keyframes` 6개 (amb-float, amb-pulse, amb-orbit 등) + Google Fonts CDN
- **외부 라이브러리 없음**: react-i18next 미사용, react-router 미사용

### 2.3 핵심 특징
- **자체 i18n**: `translations` 객체 + `t(obj)` 헬퍼 함수 (i18next 미사용)
- **자체 네비게이션**: 고정 헤더, 스크롤 기반 앵커 내비
- **자체 푸터**: 다크 4컬럼 풀 푸터
- **스크롤 애니메이션**: IntersectionObserver 기반 `Reveal` 래퍼
- **CSS 동적 주입**: `useEffect`로 `<style>` 태그 삽입

## 3. 갭 분석 및 결정사항

| 항목 | AmoebaLanding | portal-web 현재 | 결정 |
|------|---------------|-----------------|------|
| **i18n** | 인라인 `translations` 객체 | `react-i18next` + JSON | **인라인 유지** — 랜딩 전용이므로 분리가 합리적 |
| **네비게이션** | 자체 고정 헤더 (앵커 스크롤) | PublicLayout 공용 헤더 | **랜딩 전용 레이아웃** — PublicLayout 외부로 분리 |
| **푸터** | 자체 다크 4컬럼 푸터 | PublicLayout 공용 단순 푸터 | **랜딩 내 자체 푸터 사용** |
| **라우팅** | 없음 (앵커 스크롤) | react-router | **CTA 링크만 react-router Link로 교체** |
| **컬러** | orange (#F97316) | primary/indigo (#6366F1) | **orange 하드코딩 유지** — 브랜드 컬러 |
| **폰트** | Plus Jakarta Sans + Noto Sans KR (CDN) | Pretendard | **CDN 유지** — 랜딩 전용 |
| **CSS 애니메이션** | `<style>` 동적 주입 | 없음 | **globals.css로 이전** |
| **TypeScript** | JSX (타입 없음) | TSX | **TSX 변환 필요** |
| **CMS 모드** | 없음 | cmsApi 연동 | **제거** — 항상 이 랜딩 표시 |

## 4. 작업 계획

### Phase 1: 라우터 구조 변경
**목적**: 랜딩 페이지가 PublicLayout 밖에서 독립적으로 렌더링되도록 분리

| # | 작업 | 파일 | 설명 |
|---|------|------|------|
| 1-1 | 라우터 수정 | `router/index.tsx` | `/` 경로를 PublicLayout 외부 독립 라우트로 분리. 나머지(`/pricing`, `/login` 등)는 PublicLayout 유지 |

### Phase 2: 랜딩 페이지 교체
**목적**: AmoebaLanding.jsx를 TSX로 변환하여 LandingPage.tsx를 교체

| # | 작업 | 파일 | 설명 |
|---|------|------|------|
| 2-1 | CSS 애니메이션 추가 | `styles/globals.css` | `@keyframes` 6개 + `.amb-*` 유틸 클래스 추가 |
| 2-2 | LandingPage 교체 | `pages/public/LandingPage.tsx` | AmoebaLanding.jsx → TSX 변환, 전체 교체 |

### Phase 3: TSX 변환 시 적용할 변경사항

| # | JSX 원본 | TSX 변환 |
|---|----------|----------|
| 3-1 | `export default function AmoebaLanding()` | `export function LandingPage()` (기존 export 유지) |
| 3-2 | `<button onClick={() => scrollTo("contact")}>`(CTA) | `<Link to="/register">` (회원가입으로 연결) |
| 3-3 | `<a href="mailto:...">` (Contact Sales) | `<a href="mailto:contact@amoeba.group">` 유지 |
| 3-4 | `<a href="#">` (푸터 링크들) | `<Link to="/pricing">` 등 실제 라우트로 연결 |
| 3-5 | `<style>` 동적 주입 `useEffect` | `globals.css`로 이전, `useEffect` 제거 |
| 3-6 | 타입 미정의 변수들 | TypeScript 타입 어노테이션 추가 |
| 3-7 | `useCallback((obj) => ...)` | `useCallback((obj: Record<string, string>) => ...)` |
| 3-8 | `useRef(null)` | `useRef<HTMLDivElement>(null)` |

### Phase 4: 빌드 및 배포

| # | 작업 | 설명 |
|---|------|------|
| 4-1 | 빌드 검증 | `npm run build` 전체 빌드 성공 확인 |
| 4-2 | 커밋 | `feat: portal landing page 교체 (AmoebaLanding 적용)` |
| 4-3 | Push & Deploy | `git push` + 스테이징 배포 |

## 6. 사이드 임팩트 분석

| 영향 범위 | 리스크 | 대응 |
|-----------|--------|------|
| PublicLayout 공용 헤더 | 랜딩에서 중복 헤더 표시 | 랜딩을 PublicLayout 외부 라우트로 분리 |
| PublicLayout 공용 푸터 | 랜딩에서 중복 푸터 표시 | 동일 — 독립 라우트 |
| 기존 i18n 키 | `hero.*`, `services.*` 키 미사용 | 삭제하지 않음 (다른 CMS 모드에서 사용 가능) |
| `/pricing`, `/login` 등 링크 | 랜딩의 CTA에서 이동 필요 | react-router `Link` 사용 |
| 폰트 CDN 로딩 | LCP에 영향 가능 | `preconnect` 힌트 추가 검토 |
| tailwind `primary` 컬러 | 랜딩은 orange 직접 사용 | 충돌 없음 — 기존 primary는 다른 페이지에서 유지 |

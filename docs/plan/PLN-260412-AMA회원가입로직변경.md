#### 3.3.2 register() 서비스 변경

**핵심 변경점**:
1. 비밀번호: 사용자 입력 → 임시 UUID 비밀번호 자동 생성 (bcrypt 해시)
2. 이름: `email.split('@')[0]` 로 추출 (기본값)
3. 회사명: `email.split('@')[1]` 도메인 또는 `'My Company'` 기본값
4. 국가: `'VN'` 기본값 (normalizeCountryCode fallback)
5. Client 생성: 최소 정보로 생성

```typescript
async register(dto: RegisterRequest) {
  // verify_token 검증 (기존 동일)
  // 이메일 중복 확인 (기존 동일)
  // 약관 동의 확인 (기존 동일)

  // 임시 비밀번호 자동 생성
  const tempPassword = uuidv4().replace(/-/g, '').slice(0, 16);
  const hashedPassword = await bcrypt.hash(tempPassword, PORTAL_CONSTANTS.BCRYPT_SALT_ROUNDS);

  // 이름: 이메일 로컬파트에서 추출
  const defaultName = dto.email.split('@')[0];
  // 회사명: 이메일 도메인 사용
  const emailDomain = dto.email.split('@')[1] || 'company';
  const defaultCompany = emailDomain.split('.')[0] || 'My Company';

  // Client 생성 (최소 정보)
  const client = new SvcClientEntity();
  client.cliCode = await this.generateClientCode();
  client.cliType = 'COMPANY';
  client.cliCompanyName = defaultCompany;
  client.cliCountry = 'VN';   // 기본값, AMA에서 변경 가능
  client.cliStatus = 'PROSPECT';
  client.cliPortalSource = 'PORTAL';
  const savedClient = await this.clientRepo.save(client);

  // Customer 생성
  const customer = new PortalCustomerEntity();
  customer.pctEmail = dto.email;
  customer.pctPassword = hashedPassword;
  customer.pctName = defaultName;
  customer.pctCompanyName = defaultCompany;
  customer.pctCountry = 'VN';
  customer.pctEmailVerified = true;
  customer.pctCliId = savedClient.cliId;
  customer.pctTermsAgreedAt = dto.terms_agreed ? now : undefined;
  customer.pctPrivacyAgreedAt = dto.privacy_agreed ? now : undefined;
  customer.pctMarketingAgreedAt = dto.marketing_agreed ? now : undefined;
  customer.pctTermsVersion = 'v1.0';
  customer.pctPrivacyVersion = 'v1.0';
  const saved = await this.customerRepo.save(customer);

  // Auto-Provision 호출 (기존 동일)
  const provisionResult = await this.triggerAutoProvision(saved.pctId);

  // ★ 자동 로그인 토큰 요청 (Phase 2에서 구현)
  let autoLoginToken: string | null = null;
  try {
    autoLoginToken = await this.requestAutoLoginToken(saved.pctId);
  } catch {
    // 실패해도 가입은 완료
  }

  return {
    customerId: saved.pctId,
    email: saved.pctEmail,
    entityCode: provisionResult?.entityCode,
    entityName: provisionResult?.entityName,
    autoLoginToken,   // ★ 새 필드
  };
}
```

#### 4.4.1 AMA-API: 자동 로그인 토큰 생성

```typescript
// portal-bridge.service.ts
async generateAutoLoginToken(pctId: string): Promise<string> {
  // 포탈 고객 확인
  const customer = await this.customerRepo.findOne({ where: { pctId } });
  if (!customer) throw new NotFoundException('Portal customer not found');

  // 매핑된 AMA 사용자 확인
  const mapping = await this.mappingRepo.findOne({
    where: { pctId, pumStatus: 'ACTIVE' },
    relations: ['user'],
  });
  if (!mapping?.user) throw new BadRequestException('No active user mapping found');

  const user = mapping.user;
  const entity = await this.entityRepo.findOne({ where: { entId: user.usrCompanyId } });

  // One-Time Token 생성 (5분 만료)
  const token = this.jwtService.sign(
    {
      type: 'portal_auto_login',
      sub: user.usrId,
      email: user.usrEmail,
      entityCode: entity?.entCode,
      pctId,
    },
    { expiresIn: '5m' },
  );

  return token;
}
```

#### 4.4.2 AMA-API: 자동 로그인 검증

```typescript
// auth.service.ts
async autoLogin(token: string, ip?: string, userAgent?: string): Promise<LoginResult> {
  let payload: { type?: string; sub?: string; email?: string; entityCode?: string };
  try {
    payload = this.jwtService.verify(token);
  } catch {
    throw new BusinessException('E1030', 'Invalid auto-login token', HttpStatus.UNAUTHORIZED);
  }

  if (payload.type !== 'portal_auto_login' || !payload.sub) {
    throw new BusinessException('E1030', 'Invalid auto-login token type', HttpStatus.UNAUTHORIZED);
  }

  const user = await this.userRepository.findOne({
    where: { usrId: payload.sub },
    relations: ['company'],
  });

  if (!user || ['WITHDRAWN', 'SUSPENDED', 'INACTIVE'].includes(user.usrStatus)) {
    throw new BusinessException('E1030', 'User not found or inactive', HttpStatus.UNAUTHORIZED);
  }

  await this.userRepository.update(user.usrId, { usrLastLoginAt: new Date() });
  this.recordLoginHistory(user, ip, userAgent);
  return { step: 'complete', tokens: await this.generateTokens(user) };
}
```

#### 4.4.3 Portal-API: triggerAutoProvision 후 토큰 요청

```typescript
// portal-auth.service.ts
private async requestAutoLoginToken(pctId: string): Promise<string | null> {
  const amaApiUrl = this.configService.get<string>('AMA_API_URL', 'http://localhost:3009/api/v1');
  const internalToken = this.configService.get<string>('PORTAL_BRIDGE_INTERNAL_TOKEN', '');

  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), 5000);

  try {
    const response = await fetch(
      `${amaApiUrl}/portal-bridge/auto-login-token`,
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'x-portal-bridge-token': internalToken,
        },
        body: JSON.stringify({ pctId }),
        signal: controller.signal,
      },
    );

    if (!response.ok) return null;
    const result = await response.json();
    return result?.data?.token || null;
  } catch {
    return null;
  } finally {
    clearTimeout(timeout);
  }
}
```

#### 5.4.2 initialSetup() 서비스

```typescript
async initialSetup(userId: string, dto: InitialSetupRequest): Promise<LoginResult> {
  const user = await this.userRepository.findOne({
    where: { usrId: userId },
    relations: ['company'],
  });
  if (!user) throw new BusinessException(...);

  // 비밀번호 변경
  user.usrPassword = await bcrypt.hash(dto.password, BCRYPT_SALT_ROUNDS);
  user.usrMustChangePw = false;
  user.usrName = dto.name;
  await this.userRepository.save(user);

  // 회사명/국가 변경 → Entity 업데이트
  if (user.usrCompanyId && (dto.company_name || dto.country_code)) {
    const entity = await this.entityRepository.findOne({ where: { entId: user.usrCompanyId } });
    if (entity) {
      if (dto.company_name) {
        entity.entName = dto.company_name;
        entity.entNameEn = dto.company_name;
      }
      if (dto.country_code) {
        entity.entCountry = dto.country_code;
      }
      await this.entityRepository.save(entity);
    }
  }

  // 새 토큰 발급 (mustChangePw: false)
  return { step: 'complete', tokens: await this.generateTokens(user) };
}
```

### 6.1 Google OAuth 변경

Google OAuth 플로우도 동일한 간소화 적용:

```
[Google 버튼 클릭]
  ↓ Google OAuth popup
[Google Callback]
  ↓ onboarding_token
[Google Onboarding Page] ← 변경 필요
  ├─ 비밀번호 설정 제거 (기존: 비밀번호 + 회사 + 국가 + 약관)
  ├─ 약관 동의만 수집
  └─ Submit
  ↓
[자동 로그인 (autoLoginToken) → AMA]
  ↓
[InitialSetupPage] (비밀번호 + 이름 + 회사 + 국가)
```

## 7. DB 변경 사항

### 7.1 스키마 변경 (마이그레이션 불필요)

기존 nullable 컬럼들이 이미 null 허용이므로 스키마 변경 없음:
- `amb_svc_portal_customers.pct_company_name` — 기존 `@Column({ nullable: true })` 
- `amb_svc_portal_customers.pct_country` — 기존 `@Column({ nullable: true })`
- `amb_svc_portal_customers.pct_name` — 기존 `@Column({ nullable: true })`

다만 `resolveOrCreateEntityForPortalCustomer()`에서 `companyName` 빈값 체크를 완화해야 함:

```typescript
// AS-IS
const companyName = (customer.pctCompanyName || customer.pctName || '').trim();
if (!companyName) {
  throw new BadRequestException('Portal customer company name is required');
}

// TO-BE
const companyName = (customer.pctCompanyName || customer.pctName || customer.pctEmail.split('@')[0] || 'Company').trim();
// 빈값이면 fallback, 에러 제거
```

### 7.2 이메일 변경

| 변경 | 상세 |
|------|------|
| 임시 비밀번호 이메일 | **발송 유지** — 자동 로그인 토큰 실패 시 fallback |
| 이메일 템플릿 | 웰컴 메시지 추가 + 자동 로그인 실패 시 안내 |

## 시스템 현황 분석 요약

### 현행 인증/인가 구조
- **JWT 전략**: `jwt.strategy.ts` — 쿠키(`access_token`) 또는 `Bearer` 토큰에서 추출
- **전역 가드**: `JwtAuthGuard` (APP_GUARD), `DataScopeInterceptor` (APP_INTERCEPTOR) — `auth.module.ts`에서 등록
- **레벨/역할 가드**: `LevelRoleGuard` (exact match), `RolesGuard` (hierarchy 기반) — 별도 시스템
- **JWT payload**: `sub`, `email`, `level`, `role`, `status`, `companyId`, `isHq`, `mustChangePw`, `entityId`, `timezone`, `locale`
- **로그인 흐름**: `entity_code` 유무에 따라 법인 지정/미지정 로그인 분기

### 현행 사용자 구조
- **UserEntity** (`amb_users`): Unique = `(usr_email, usr_company_id)` WHERE `usr_deleted_at IS NULL`
- **레벨**: `ADMIN_LEVEL` (SUPER_ADMIN, ADMIN), `USER_LEVEL` (MASTER, MANAGER, MEMBER, VIEWER)
- **가입**: `JOIN_METHOD` — SEED, REGISTER, INVITE, PORTAL
- **CLIENT 관련 필드 없음**: `usr_cli_id` 컬럼 미존재

### 현행 프로젝트/이슈 구조
- **ProjectController**: `@Controller('project/projects')`, `@UseGuards(EntityGuard)` — `entityId` 기반 필터
- **ProjectMemberEntity**: `(pjt_id, usr_id)` unique, `usr_id` FK → `UserEntity`
- **IssueEntity**: `iss_reporter_id` FK → `UserEntity`, `iss_visibility` (ENTITY/CELL/PRIVATE)
- **프로젝트-고객사 연결 없음**: `project-client.entity.ts` 미존재

### 현행 초대 구조
- **InvitationEntity** (`amb_invitations`): `inv_` prefix, 토큰 기반, 만료/상태 관리
- 패턴 참고 가능: `inv_token`, `inv_expires_at`, `inv_status`, `inv_auto_approve`

}

// intercept 메서드 내부

  request.dataScope = { scope: 'ALL', companyId: null, cliId: null, isHq: true };

export class ClientGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const { user } = context.switchToHttp().getRequest();
    if (!user || user.level !== 'CLIENT_LEVEL') {
      throw new ForbiddenException('Client access only');
    }
    return true;
  }
}

## 빌드 및 배포

### 빌드 검증
```bash
# 타입 패키지 빌드
npm run -w @amb/types build

# 백엔드 빌드
npm run -w @amb/api build

# 프론트엔드 빌드
npm run -w @amb/web build
```

### 스테이징 배포
```bash
# 1. Git push
git add . && git commit -m "feat: 고객사 포털 이슈 관리 시스템 구현" && git push origin main

# 2. 스테이징 서버에서 DB 마이그레이션
ssh amb-staging "docker exec -i amb-postgres-staging psql -U amb_user -d db_amb" < migration.sql

# 3. 배포 스크립트 실행
ssh amb-staging "cd ~/ambManagement && git pull origin main && bash docker/staging/deploy-staging.sh"
```

### 배포 후 검증
1. `/client/login` 페이지 접속 확인
2. 내부 PM 계정으로 프로젝트에 고객사 연결 + 초대 발송
3. 초대 링크로 고객 가입 → 로그인
4. 고객 대시보드, 프로젝트 조회, 이슈 등록 확인
5. 내부 사용자 측에서 고객 등록 이슈 확인

# 작업 계획서: 커스텀앱 등록 400 Bad Request 오류 수정

- **문서번호**: PLAN-커스텀앱등록400오류-작업계획-20260309
- **작성일**: 2026-03-09
- **참조**: REQ-커스텀앱등록400오류-20260309

---

## 1. 현황 요약

- `/entity-settings/custom-apps` 페이지에서 커스텀앱 등록 시 400 오류
- 원인: `resolveEntityId()`에서 `user.entityId`가 없어 실패
- 대상: ADMIN_LEVEL 사용자 또는 entity 미할당 MASTER 사용자

---

## 2. 구현 계획

### Step 1: EntityCustomAppsPage — entityId 미존재 시 방어 처리

**수정 파일**: `apps/web/src/domain/entity-settings/pages/EntityCustomAppsPage.tsx`

**변경 내용**:
- `useAuthStore`에서 현재 사용자의 `isAdmin()` 확인
- ADMIN_LEVEL인 경우 → `/admin/custom-apps`로 자동 리다이렉트
- entityId 미존재(entity 미할당 MASTER)인 경우 → 안내 메시지 표시 (법인 배정 필요)

```tsx
// ADMIN은 전용 관리 페이지로 리다이렉트
const isAdmin = useAuthStore((s) => s.isAdmin);
if (isAdmin()) return <Navigate to="/admin/custom-apps" replace />;
```

### Step 2: 프론트엔드 폼 유효성 검사 강화

**수정 파일**: `apps/web/src/domain/entity-settings/pages/EntityCustomAppsPage.tsx`

**변경 내용**:
- URL 입력 시 `https://` 프리픽스 자동 체크/안내
- code 필드 입력 시 이미 `.toLowerCase().replace()` 처리 중 → 충분
- Submit 전 URL이 `https://`로 시작하는지 프론트 검증 추가

### Step 3: API 에러 메시지 토스트 표시

**수정 파일**: `apps/web/src/domain/entity-settings/pages/EntityCustomAppsPage.tsx`

**변경 내용**:
- `createMutation.mutateAsync()` 호출 시 try-catch 추가
- 400 에러의 validation 메시지를 사용자에게 toast 또는 alert로 표시
- 기존에 에러 처리 없이 조용히 실패하는 문제 해결

### Step 4: i18n 번역 키 추가

**수정 파일**: `apps/web/src/locales/{en,ko,vi}/entitySettings.json`

**추가 키**:
```json
"customApps": {
  "noEntity": "No entity assigned. Please contact your administrator.",
  "urlHttpsRequired": "URL must start with https://",
  "createError": "Failed to create app: {{message}}",
  "updateError": "Failed to update app: {{message}}"
}
```


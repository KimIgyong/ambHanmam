# PLAN-이슈리플라이멘션이모지알림-작업계획-20260402

## 1. 시스템 개발 현황 분석

### 기존 인프라
- **알림 이벤트 파이프라인**: EventEmitter2 → NotificationListener → DB → SSE → Web Push (검증 완료)
- **백엔드 멘션 파싱**: `extractMentionedUserIds()` (정규식 기반)
- **프론트엔드 멘션 UI**: `RichTextEditor` + `MentionPortalDropdown` (Portal 기반)
- **리액션 시스템**: `toggleCommentReaction()` + `buildCommentReactionMap()` (답글 ID 포함)
- **별점 시스템**: `ContentRatingService.upsertIssueRating()` (EventEmitter2 미연결)

### DB 스키마 변경
- **없음** — `amb_notifications.ntf_type`은 varchar이므로 새 알림 타입 추가 시 DDL 불필요

## 2. 단계별 구현 계획

### Phase 1: 프론트엔드 — 답글 멘션/이모지 UI (사이드 임팩트: 낮음)

**변경 파일**:
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `apps/web/src/domain/issues/components/IssueDetailModal.tsx` | 수정 | 답글 에디터에 `enableMention={true}` 추가, 답글에 이모지 버튼 추가 |

**상세**:
1. 답글 `RichTextEditor`에 `enableMention={true}` prop 추가 (L963)
2. 답글(reply) 렌더링에 이모지 리액션 버튼 추가 (L925~L955 영역)
   - 부모 코멘트와 동일한 이모지 UI 패턴 복사
   - `reply.reactions` 데이터는 이미 `getIssueComments` 응답에 포함됨

### Phase 2: 백엔드 — 답글 자동 멘션 알림 (사이드 임팩트: 중간)

**변경 파일**:
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `apps/api/src/domain/issues/service/issue.service.ts` | 수정 | `addIssueComment()` 답글 알림 로직 분기 |

**상세**:
- `addIssueComment()` 내 멘션 미감지+답글인 경우:
  - 기존: assignee + participants 전원 알림
  - 변경: **원래 코멘트 작성자(`parentComment.iscAuthorId`)에게만 알림** 
  - 조건: `parentId` 존재 + 멘션 미감지 + 작성자 ≠ 원래 작성자
- 이미 `parentComment`는 L783에서 조회하고 있으므로 추가 쿼리 불필요

### Phase 3: 백엔드 — 이모지 리액션 + 별점 알림 (사이드 임팩트: 중간)

**변경 파일**:
| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `apps/api/src/domain/notification/constant/notification-type.constant.ts` | 수정 | REACTION_EVENT, RATING_EVENT 이벤트 + 타입 + 인터페이스 추가 |
| `apps/api/src/domain/notification/service/notification.listener.ts` | 수정 | `handleReaction()`, `handleRating()` 핸들러 추가 |
| `apps/api/src/domain/issues/service/issue.service.ts` | 수정 | `toggleCommentReaction()`에 이벤트 emit 추가 |
| `apps/api/src/domain/activity-index/service/content-rating.service.ts` | 수정 | `upsertIssueRating()`에 이벤트 emit 추가, EventEmitter2/UserRepo DI 추가 |
| `apps/api/src/domain/activity-index/activity-index.module.ts` | 수정 | UserEntity import 추가 (DI) |

**상세**:

#### 3a. 이모지 리액션 알림
- `toggleCommentReaction()` — 리액션 **추가**된 경우에만:
  - comment의 `iscAuthorId` 조회 (이미 `findOne`으로 조회 중)
  - 작성자 ≠ 리액터가면 → `REACTION_EVENT` emit
  - 페이로드: `{ resourceType, resourceId, commentId, reactionType, senderId, senderName, recipientId, entityId }`
- `NotificationListener.handleReaction()`:
  - 알림 메시지: `"{senderName}님이 코멘트에 {emoji}를 눌렀습니다."`
  - emoji 매핑: LIKE→👍, CHECK→✅, PRAY→🙏, GRIN→😀, LOVE→❤️

#### 3b. 별점 알림
- `ContentRatingService.upsertIssueRating()`:
  - 이슈 조회 시 이미 `issue.issReporterId` 접근 가능
  - 별점 저장 후 → `RATING_EVENT` emit
  - sender 이름은 UserRepository에서 조회
- `NotificationListener.handleRating()`:
  - 알림 메시지: `"{senderName}님이 이슈에 ⭐{rating}개를 주셨습니다."`

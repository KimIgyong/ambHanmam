### 2.4 변경 후 상태 흐름

**open_mode = 'new_tab' (명시적 새탭 설정)**
```
진입 → 설명 페이지 표시 + "Open in new tab" 버튼
       (iframe 시도 없음)
```

**open_mode = 'iframe' (iframe 허용 설정)**
```
진입 → 'landing' (설명 + 로딩 버튼 + 백그라운드 감지 시작)
         ├─ onLoad 발생 (3초 내) → 'iframe' (전체화면 자동 전환)
         └─ 타임아웃 (3초 후) → 'blocked' (설명 + "Open in new tab" 버튼)
```

## 4. 단계별 구현 계획

### Phase 1 — DB 엔티티 컬럼 추가

**파일**: `apps/api/src/domain/entity-settings/entity/entity-custom-app.entity.ts`

```typescript
@Column({ name: 'eca_open_mode', type: 'varchar', length: 10, default: 'iframe' })
ecaOpenMode: string;  // 'iframe' | 'new_tab'

@Column({ name: 'eca_api_key_enc', type: 'text', nullable: true })
ecaApiKeyEnc: string | null;
```

> 개발 환경은 `synchronize: true`로 자동 컬럼 추가. 스테이징은 마이그레이션 SQL 직접 실행.
>
> ```sql
> ALTER TABLE amb_entity_custom_apps
>   ADD COLUMN IF NOT EXISTS eca_open_mode VARCHAR(10) NOT NULL DEFAULT 'iframe',
>   ADD COLUMN IF NOT EXISTS eca_api_key_enc TEXT NULL;
> ```

---

### Phase 2 — Backend DTO 수정

**파일**: `apps/api/src/domain/entity-settings/dto/custom-app.dto.ts`

`CreateCustomAppRequest` 및 `UpdateCustomAppRequest` 모두에 추가:

```typescript
@IsOptional()
@IsIn(['iframe', 'new_tab'])
open_mode?: string;  // 미지정 시 기본값 'iframe'

@IsOptional()
@IsString()
api_key?: string;  // auth_mode = 'api_key' 일 때 사용
```

`CustomAppResponse`에 추가:

```typescript
open_mode: string;        // 'iframe' | 'new_tab'
api_key_masked?: string;  // '••••••••Ab3X' 형태 (4자리 노출)
```

---

### Phase 3 — Backend Service 수정

**파일**: `apps/api/src/domain/entity-settings/service/entity-custom-app.service.ts`

#### 3.1 암호화 유틸

```typescript
// AES-256-GCM 사용, 환경변수 ENCRYPTION_KEY (32바이트 hex)
private encrypt(text: string): string { ... }
private decrypt(encrypted: string): string { ... }
```

#### 3.2 `create()` 수정

```typescript
app.ecaOpenMode = dto.open_mode ?? 'iframe';
if (dto.api_key) {
  app.ecaApiKeyEnc = this.encrypt(dto.api_key);
}
```

#### 3.3 `update()` 수정

```typescript
app.ecaOpenMode = dto.open_mode ?? app.ecaOpenMode;
if (dto.api_key) {
  // 값이 있을 때만 갱신 (빈칸 = 기존 키 유지)
  app.ecaApiKeyEnc = this.encrypt(dto.api_key);
}
```

#### 3.4 `toResponse()` 수정

```typescript
open_mode: app.ecaOpenMode,
api_key_masked: app.ecaApiKeyEnc
  ? '••••••••' + this.decrypt(app.ecaApiKeyEnc).slice(-4)
  : undefined,
```

#### 3.5 `generateAppToken()` 수정

```typescript
case 'api_key':
  if (!app.ecaApiKeyEnc) throw new Error('API Key not configured');
  const apiKey = this.decrypt(app.ecaApiKeyEnc);
  // URL에 api_key 파라미터로 포함하거나 헤더 방식 중 선택
  // 현재 설계: URL 파라미터 방식
  return { url: `${app.ecaUrl}?api_key=${apiKey}`, token: null };
```

### Phase 5 — `CustomAppHostPage.tsx` 리팩터링

#### 5.1 상태 타입

```typescript
type AppPhase = 'loading' | 'landing' | 'iframe' | 'blocked';
```

| 상태 | 설명 |
|------|------|
| `loading` | 앱 정보/토큰 로딩 중 |
| `landing` | 설명 카드 표시 (iframe 감지 진행 중) |
| `iframe` | 전체화면 iframe 표시 |
| `blocked` | iframe 차단 또는 new_tab 모드 → 설명 + 새탭 버튼 |

#### 5.2 `open_mode` 기반 분기

```typescript
// 앱 정보 로드 완료 후
if (app.openMode === 'new_tab') {
  setPhase('blocked');  // 즉시 새탭 안내 페이지
} else {
  setPhase('landing');  // iframe 감지 시작
  timeoutRef.current = setTimeout(() => setPhase('blocked'), 3000);
}
```

#### 5.3 숨겨진 iframe (iframe 모드에서만)

```tsx
{phase === 'landing' && (
  <iframe
    src={iframeSrc}
    className="hidden"
    onLoad={() => {
      clearTimeout(timeoutRef.current);
      setPhase('iframe');
    }}
  />
)}
```

#### 5.4 UI 렌더링 분기

```
phase === 'loading'  → 전체화면 스피너
phase === 'landing'  → 설명 카드 + "앱 연결 중..." 버튼 + 숨겨진 iframe
phase === 'iframe'   → 상단 바 + 전체화면 iframe
phase === 'blocked'  → 설명 카드 + "Open in new tab" 버튼
```

## 6. 사이드 임팩트 분석

| 영역 | 임팩트 | 조치 사항 |
|------|--------|---------|
| 라우터 변경 | 없음 (`/apps/:appCode` 경로 유지) | — |
| API 변경 | Request/Response DTO에 필드 추가 (하위호환) | 프론트 API 타입 동기화 필요 |
| DB 변경 | 컬럼 2개 추가 (DEFAULT 있어 기존 데이터 영향 없음) | 스테이징 배포 전 ALTER SQL 실행 |
| `ENCRYPTION_KEY` 환경변수 | 신규 필요 (32바이트 hex) | 개발/스테이징/프로덕션 `.env` 모두 추가 |
| 기존 커스텀앱 레코드 | `eca_open_mode = 'iframe'` 기본값 적용 | 기존 UX 유지됨 |
| 기존 iframe 동작 | 변경: 즉시 iframe → 설명 페이지 먼저 | 의도된 UX 변경 |
| 자동 전환 지연 | 최대 3초 (감지 타임아웃) | 수용 가능, 감지 완료 즉시 전환 |
| new_tab 설정 앱 | iframe 시도 없이 즉시 새탭 안내 | 3초 대기 없이 즉각 응답 |
| API Key 노출 위험 | URL 파라미터 방식은 로그에 노출 가능 | 로그 마스킹 처리 또는 Authorization 헤더 방식 검토 필요 |

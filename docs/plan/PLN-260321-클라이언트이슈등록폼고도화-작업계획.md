# PLAN-클라이언트이슈등록폼고도화-작업계획-20260321

## 1. 시스템 개발 현황 분석

### 1.1 재사용 가능한 기존 컴포넌트
| 컴포넌트 | 경로 | 용도 |
|----------|------|------|
| RichTextEditor | `domain/meeting-notes/components/RichTextEditor.tsx` | TipTap 기반 리치텍스트 에디터 |
| TranslationPanel | `domain/translations/components/TranslationPanel.tsx` | 다국어 번역 패널 |
| AssigneeSelector | `domain/issues/components/AssigneeSelector.tsx` | 담당자 선택 |
| ParticipantSelector | `domain/issues/components/ParticipantSelector.tsx` | 관련자 다중 선택 |

### 1.2 인증 호환성 확인
- 클라이언트 JWT는 글로벌 `JwtStrategy`에서 정상 검증됨 (`CLIENT_LEVEL`)
- 파일 업로드 `POST /files/upload` → 글로벌 JWT 가드 → 클라이언트 토큰으로 접근 가능
- 파일 다운로드 `GET /files/:filename` → `@Public()` → 인증 불필요

### 1.3 제약사항
- `AssigneeSelector`/`ParticipantSelector`는 내부 멤버 API 사용 (`useAuthStore`, `useMemberList`)
- 클라이언트 포털에서는 `useClientAuthStore` 사용 → 프로젝트 멤버 조회 API 필요
- `TranslationPanel`은 이슈 ID 필요 → 이슈 저장 후 상세 페이지에서만 동작

## 2. 단계별 구현 계획

### Phase 1: 백엔드 API 확장

#### 1-1. `CreateClientIssueRequest` DTO 확장
**파일**: `apps/api/src/domain/client-portal/dto/create-client-issue.request.ts`
- 추가 필드: `parent_issue_id`, `assignee_id`, `participant_ids`, `start_date`, `due_date`, `google_drive_link`

#### 1-2. `ClientIssueService.createIssue()` 확장
**파일**: `apps/api/src/domain/client-portal/service/client-issue.service.ts`
- 새 필드 매핑: `issParentId`, `issAssigneeId`, `issStartDate`, `issDueDate`, `issGoogleDriveLink`
- participant_ids 처리: IssueParticipant 엔티티 생성

#### 1-3. 프로젝트 멤버 조회 API 추가
**파일**: `apps/api/src/domain/client-portal/controller/client-project.controller.ts`
- `GET /client/projects/:id/members` → 프로젝트 멤버(내부 직원) 목록 반환
- 서비스: `ClientProjectService.findProjectMembers()`

#### 1-4. 프로젝트 이슈 검색 API (관련이슈용)
- 기존 `GET /client/projects/:id/issues`에 `search` 파라미터 이미 존재 → 별도 추가 불필요

### Phase 2: 프론트엔드 이슈 등록 폼 재구성

#### 2-1. `ClientIssueCreatePage.tsx` 전면 재작성
**현재**: 120줄, 단순 form 4필드
**목표**: ~500줄, 풀 기능 form 12필드

**폼 레이아웃 (2열 그리드)**:
```
Row 1: [프로젝트명 *] [관련이슈]
Row 2: [유형 *]       [긴급도 *]
Row 3: [제목 *] (전체폭)
Row 4: [내용 - RichTextEditor] (전체폭)
Row 5: [담당자]       [관련자]
Row 6: [파일 업로드 / 링크] (전체폭)
Row 7: [시작일]       [종료예정일]
─── 버튼 ───
[등록] [취소]
```

#### 2-2. 컴포넌트 재사용 전략
- **RichTextEditor**: 직접 재사용 (동일한 `apiClient` 인증)
- **파일 첨부**: `IssueFormModal`의 드래그드롭 로직을 별도 컴포넌트로 추출 또는 인라인 구현
- **관련이슈 검색**: 프로젝트 이슈 검색 API 호출 + 디바운스 자동완성
- **담당자/관련자**: 프로젝트 멤버 API 기반 커스텀 셀렉터 (내부 AssigneeSelector 로직 참조)

### Phase 3: 이슈 상세 페이지에 번역 패널 추가

#### 3-1. `ClientIssueDetailPage.tsx` 수정
- 이슈 설명 하단에 `TranslationPanel` 추가
- props: `sourceType="ISSUE"`, `sourceId={issue.id}`, `sourceFields={['title', 'description']}`

### Phase 4: i18n 다국어 처리

#### 4-1. `clientPortal.json` 키 추가 (en/ko/vi)

# My Page UI Restructure - Work Plan

## 1. 시스템 개발 현황 분석

### 1.1 프론트엔드
- `MyPage.tsx`: 메인 페이지 (섹션 순서 결정)
- `ProfileSection.tsx`: 기본정보 (프로필 이미지 + 이름/이메일/Unit/역할/회사이메일/가입일/비상연락처)
- `PasswordChangeSection.tsx`, `TimezoneSection.tsx`: 하단 컴포넌트 (변경 없음)
- 라우트: `/attendance` ✅, `/leave-requests` (HR 하위) → `/hr/leave-requests`

### 1.2 백엔드
- `PATCH /api/v1/users/me/profile` - 현재 `{ name?, phone? }` 만 허용
- `usr_company_email` 컬럼: VARCHAR(200), unique, nullable - DB에 존재
- `company_email` 필드를 DTO에 추가하면 기존 validation 패턴 따라 구현 가능

### 1.3 서비스/훅
- `my-page.service.ts`: `updateProfile(data)` → `PATCH /users/me/profile`
- `useMyPage.ts`: `useUpdateProfile()` mutation

## 2. 단계별 구현 계획

### Phase 1: i18n 키 추가
- `myPage.json` (ko/en/vi)에 카드뷰 라벨, 회사이메일 편집 관련 키 추가
- 사이드 임팩트: 없음

### Phase 2: 백엔드 - 회사이메일 수정 지원
- `user.controller.ts`의 updateProfile에 `company_email` 필드 추가
- Validation: nullable, max 200자, 이메일 형식 검증
- 사이드 임팩트: 기존 name/phone 수정에 영향 없음 (optional 필드 추가)

### Phase 3: ProfileSection 리디자인
- 레이아웃을 좌우 2영역으로 변경:
  - 좌: 프로필 사진 + 사진변경/삭제
  - 우: 정보 항목 2컬럼 그리드
- 정보 항목 배치:
  - Row 1: 이름 ✏️ | 이메일 (읽기전용)
  - Row 2: Unit·부서 (읽기전용) | 역할 (읽기전용)
  - Row 3: 회사이메일 ✏️ | 가입일
  - Row 4: 비상연락처 ✏️
- 회사이메일 인라인 편집 추가 (phone과 동일 패턴)
- 사이드 임팩트: ProfileSection의 시각적 변경. props/데이터 동일.

### Phase 4: MyPage 레이아웃 재구성
1. 기본정보 (ProfileSection)
2. **카드뷰 퀵링크** (신규 - 출퇴근/내휴가/지출결의)
3. 연차현황 (Leave Balance) - 기존 그대로
4. 지출결의서 (Expense Requests) - 기존 그대로
5. 소속법인 (Entities) - 기존 그대로
6. Unit 현황 - 기존 그대로
7. Cell 배정현황 - 기존 그대로
8. 비밀번호 변경 - 기존 그대로
9. 타임존 - 기존 그대로
- 사이드 임팩트: 순서만 변경, 각 섹션 내부 변경 없음

### Phase 5: 빌드 검증
- `npm run -w @amb/web build`
- `npm run -w @amb/api build`

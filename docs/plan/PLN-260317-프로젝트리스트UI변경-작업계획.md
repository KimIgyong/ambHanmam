## 단계별 구현 계획

### 단계 1: 백엔드 − memberCount 서브쿼리 추가

**파일**: `apps/api/src/domain/project/service/project.service.ts`

**작업 내용**:
- `findAll` 메서드의 QueryBuilder에 memberCount 서브쿼리 추가
  ```sql
  (SELECT COUNT(*) FROM amb_project_members pm
   WHERE pm.pjt_id = p.pjt_id AND pm.pmb_is_active = true)
  ```
- `ProjectMapper.toResponse()`에서 memberCount 매핑 (issueCount과 동일 패턴)

**파일**: `packages/types/src/domain.types.ts`

**작업 내용**:
- `ProjectResponse` 인터페이스에 `memberCount: number` 필드 추가

---

### 단계 2: 프론트엔드 − ProjectLayout 변경 (사이드바 → 상단 탭)

**파일**: `apps/web/src/domain/project/pages/ProjectLayout.tsx`

**변경 내용**:
- 기존 좌측 사이드바(`aside`) 제거
- 기존 모바일 가로 탭을 모든 해상도에 적용되는 상단 탭으로 변환
- 탭 메뉴 우측에 `+New Proposal` 버튼 배치
- 레이아웃 구조: 상단 탭바 + 전체 너비 콘텐츠(`<Outlet />`)

**변경 후 구조**:
```
┌──────────────────────────────────────────┐
│ [프로젝트]  [제안서]        [+New Proposal] │  ← 상단 탭 + 버튼
├──────────────────────────────────────────┤
│                                          │
│              <Outlet />                  │  ← 전체 너비 콘텐츠
│                                          │
└──────────────────────────────────────────┘
```

---

### 단계 3: 프론트엔드 − ProjectListPage 카드뷰 변경

**파일**: `apps/web/src/domain/project/pages/ProjectListPage.tsx`

**변경 내용**:
- 기존 페이지 헤더(`h1 "프로젝트 목록"`) 제거 (ProjectLayout 탭에서 처리)
- 데스크톱 테이블 뷰 제거
- 모바일 카드뷰를 기반으로 반응형 그리드 카드뷰 구성
  - 모바일: 1열 (space-y-3)
  - 태블릿: 2열 그리드 (md:grid-cols-2)
  - 데스크톱: 3열 그리드 (lg:grid-cols-3)
- 카드 내용:
  - 프로젝트명 (메인 타이틀)
  - 프로젝트코드 (서브텍스트)
  - 상태 뱃지 (ProjectStatusBadge)
  - 카테고리
  - 매니저(PM)
  - 멤버수 (Users 아이콘 + 숫자)
  - 이슈건수 (Bug/AlertCircle 아이콘 + 숫자)
  - **예산 비노출**

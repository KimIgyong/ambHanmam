# PLAN-이슈삭제권한정책정리-작업계획-20260331

## 1. 시스템 개발 현황 분석

### 현재 이슈 삭제 구조
- **단일 엔드포인트**: `DELETE /api/v1/issues/:id` → `@Roles('MANAGER')` → `softRemove()`
- **프론트**: 삭제 버튼은 `isManager || isInvolved`에 표시되나, 확인 모달에서 `isMaster`만 활성
- **보안 누락**: `OwnEntityGuard` 미적용, userId 미전달

### 관련 하위 테이블 (Hard Delete 시 캐스케이드 대상)
| 테이블 | FK | 설명 |
|--------|------|------|
| `amb_issue_comments` | `iss_id` | 이슈 코멘트 |
| `amb_issue_comment_reactions` | `icm_id` (via comment) | 코멘트 리액션 |
| `amb_issue_status_logs` | `iss_id` | 상태 변경 이력 |
| `amb_issue_participants` | `iss_id` | 이슈 참여자 |
| `amb_issue_sequences` | - | 이슈 채번 (삭제 불필요) |

### 프로젝트 매니저 판별 필드
- `kms_projects.pjt_manager_id`: 프로젝트 매니저 (1명)
- `kms_project_members`: `pjt_id` + `usr_id` + `pmb_role = 'MANAGER'`

## 2. 단계별 구현 계획

### Phase 1: 백엔드 — 삭제 API 권한 로직 개선

**목표**: 기존 단일 softDelete를 역할별 분기 처리로 변경하고, 영구 삭제 엔드포인트 추가

#### 1-1. 서비스 수정 — `issue.service.ts`

**softDeleteIssue(id, userId, userRole)** 메서드 추가:
```
1. 이슈 조회 (with project relation)
2. 권한 검증:
   a. MASTER/ADMIN → 무조건 허용
   b. Reporter(생성자) → assigneeId가 NULL일 때만 허용
   c. Project Manager → 이슈의 pjtId에 해당하는 프로젝트의 PM 여부 확인
   d. 그 외 → 403 Forbidden
3. softRemove() 실행
```

**permanentDeleteIssue(id)** 메서드 추가 (MASTER 전용):
```
1. 이슈 조회 (withDeleted: true — soft delete된 것도 포함)
2. 하위 데이터 삭제 (트랜잭션):
   a. amb_issue_comment_reactions (comment의 reaction 먼저)
   b. amb_issue_comments
   c. amb_issue_status_logs
   d. amb_issue_participants
3. 이슈 본체 hard delete (repository.remove())
```

**사이드 임팩트**: 기존 deleteIssue() 호출부를 softDeleteIssue()로 변경 필요

#### 1-2. 컨트롤러 수정 — `issue.controller.ts`

```typescript
// 기존 DELETE /:id 수정
@Delete(':id')
@Auth()
@UseGuards(OwnEntityGuard)
async softDeleteIssue(
  @Param('id') id: string,
  @CurrentUser() user: UserPayload,
  @Query('entity_id') queryEntityId: string | undefined,
) {
  const entityId = resolveEntityId(queryEntityId, user);
  await this.issueService.softDeleteIssue(id, user.userId, user.role, entityId);
}

// 신규 DELETE /:id/permanent (MASTER 전용)
@Delete(':id/permanent')
@Auth()
@UseGuards(RolesGuard)
@Roles('MASTER')
@UseGuards(OwnEntityGuard)
async permanentDeleteIssue(
  @Param('id') id: string,
  @CurrentUser() user: UserPayload,
  @Query('entity_id') queryEntityId: string | undefined,
) {
  const entityId = resolveEntityId(queryEntityId, user);
  await this.issueService.permanentDeleteIssue(id, entityId);
}
```

**사이드 임팩트**: 기존 `@Roles('MANAGER')` 제거 → 서비스 레이어에서 세분화된 권한 검증으로 대체

## 1. 시스템 개발 현황 분석

### 1.1 현재 개발 상태

- `/entity-settings/permissions`는 Unit/Cell/개인 권한만 관리
- `amb_menu_config`는 전역 메뉴 기본 설정만 저장
- 사이드메뉴 카테고리는 프론트 상수(`WORK_TOOL_CODES`)로 분기
- 법인별 메뉴 순서/카테고리 커스터마이징 기능은 미구현

### 1.2 기존 재사용 가능 구성요소

- Entity 범위 보안 패턴: `@Auth()` + `@UseGuards(OwnEntityGuard)` + `resolveEntityId()`
- 권한 페이지 탭 UI 패턴: `EntityPermissionPage.tsx`
- 메뉴 API 응답 구조: `/api/v1/menus` (`menuCode`, `labelKey`, `icon`, `path`, `sortOrder`)
- 메뉴 설정 엔티티: `MenuConfigEntity` (`amb_menu_config`)

## 2. 단계별 구현 계획

## Phase 1: DB 및 백엔드 기반 추가

### 목표
법인별 메뉴 설정을 저장/조회할 수 있는 저장소 및 API 기반 마련

### 작업
1. 신규 엔티티 추가: `EntityMenuConfigEntity` (`amb_entity_menu_configs`)
2. SQL 마이그레이션 작성: 테이블/인덱스/유니크 제약
3. entity-settings permission API 확장
   - `GET /api/v1/entity-settings/permissions/menu-config`
   - `PUT /api/v1/entity-settings/permissions/menu-config`
   - `DELETE /api/v1/entity-settings/permissions/menu-config` (초기화)
4. 메뉴 조회 API(`/api/v1/menus`)에 entity별 override 적용

### 산출물
- 엔티티/DTO/서비스/컨트롤러 코드
- `sql/` 마이그레이션 파일

### 사이드 임팩트
- 메뉴 API 응답 정렬/카테고리 로직 변경
- 기존 권한 API(Unit/Cell/개인)는 영향 없음

## 5. DB 마이그레이션

### 필요 여부
- **필요 (신규 테이블 추가)**

### 대상 테이블
- `amb_entity_menu_configs`

### 스키마 초안

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `emc_id` | UUID PK | |
| `ent_id` | UUID NOT NULL | 법인 ID |
| `emc_menu_code` | VARCHAR(50) NOT NULL | 메뉴 코드 |
| `emc_category` | VARCHAR(30) NOT NULL | `WORK_TOOL` / `WORK_MODULE` |
| `emc_sort_order` | INT NOT NULL | 메뉴 표시 순서 |
| `emc_updated_at` | TIMESTAMP | 수정 시각 |
| `emc_updated_by` | UUID NULL | 수정 사용자 |

### 제약/인덱스
- UNIQUE (`ent_id`, `emc_menu_code`)
- INDEX (`ent_id`, `emc_sort_order`)

### 운영 반영 주의
- staging/production은 `synchronize: false`
- 코드 배포 전 SQL 선적용 필수

## 2. 구현 계획

### 단계 1: Presence 훅 전역화 (코드 변경 최소화)

**변경 파일**: 2개

| 파일 | 변경 내용 |
|------|----------|
| `apps/web/src/layouts/MainLayout.tsx` | `usePresenceHeartbeat()` + `usePresenceSSE()` 호출 추가 |
| `apps/web/src/domain/amoeba-talk/pages/AmoebaTalkPage.tsx` | 중복 호출 제거 |

**변경 상세**:

1. **MainLayout.tsx**: import 추가 + 두 훅 호출
   - 모든 인증된 페이지에서 하트비트/SSE 활성화
   - 기존 전역 훅들 (`useNotificationSSE`, `usePushSubscription` 등)과 동일 패턴

2. **AmoebaTalkPage.tsx**: `usePresenceHeartbeat()`, `usePresenceSSE()` 호출 제거
   - 이미 MainLayout에서 전역으로 호출되므로 중복 제거
   - `useTalkSSE()`는 채팅 전용이므로 유지

## 3. 사이드 임팩트 분석

| 영향 항목 | 위험도 | 분석 |
|----------|--------|------|
| SSE 연결 수 증가 | 낮음 | 기존: 채팅 페이지 접속 시만 SSE 연결. 변경 후: 로그인 시 항상 연결. 동시접속 50명 이하이므로 부담 없음 |
| 하트비트 API 부하 | 낮음 | 기존과 동일 주기(30초). 유휴/비활성 시 자동 중단 로직 유지 |
| 기존 채팅 기능 | 없음 | AmoebaTalkPage의 `useTalkSSE()` 채널별 SSE는 그대로 유지 |
| 모바일/PWA | 없음 | 동일한 MainLayout 경유하므로 자동 적용 |

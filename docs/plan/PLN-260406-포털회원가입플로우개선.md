## 2. 단계별 구현 계획 (Phased Implementation)

### Phase 1: BE — 이메일 인증 API (send-verify-code, verify-email-code)

**목표**: 인라인 이메일 인증을 위한 2개 API 엔드포인트 구현

#### 2.1.1 인메모리 인증코드 저장소

```typescript
// 파일: apps/portal-api/src/domain/auth/service/email-verify.store.ts (신규)
// 구조: Map<email, { code, expiresAt, attempts, lastSentAt, sendCount }>
// TTL: 10분, 최대 발송: 5회/시간, 최대 검증 시도: 5회
```

#### 2.1.2 이메일 발송 서비스

```typescript
// 파일: apps/portal-api/src/domain/auth/service/portal-mail.service.ts (구현 완료)
// DB Gmail SMTP 설정 우선 사용 (amb_smtp_settings 테이블)
// AES-256-GCM 복호화로 DB 저장된 SMTP 비밀번호 사용
// DB 설정 없거나 실패 시 환경변수 SMTP_HOST/PORT/USER/PASS fallback
// 메서드: sendVerificationCode(email, code) → HTML 템플릿 이메일 발송
```

#### 2.1.3 컨트롤러 + 서비스 메서드

| 엔드포인트 | 로직 |
|-----------|------|
| `POST /portal/auth/send-verify-code` | 이메일 중복 확인 → 4자리 코드 생성 → 인메모리 저장 (10분 TTL) → 이메일 발송 |
| `POST /portal/auth/verify-email-code` | 코드 일치 확인 → verify_token (JWT, 10분 TTL) 발급 → 인메모리 코드 삭제 |

#### 2.1.4 DTO

```typescript
// send-verify-code.request.ts (신규)
{ email: string }

// verify-email-code.request.ts (신규)
{ email: string, code: string }

// verify-email-code.response.ts
{ success: true, verify_token: string }
```

#### 2.1.5 에러 코드

| 코드 | 상황 |
|------|------|
| E1024 | 이메일 중복 |
| E1025 | 인증코드 불일치 |
| E1026 | 인증코드 만료 |

**산출물**: 2개 API 동작 확인 (Postman/curl 테스트)  
**사이드 임팩트**: 없음 (신규 API)

### Phase 2: BE — Register API 변경

**목표**: 기존 register 엔드포인트에 verify_token 검증 추가

#### 2.2.1 Register DTO 변경

```typescript
// AS-IS
{ email, password, name, company_name, country, phone?, terms_agreed, privacy_agreed, marketing_agreed? }

// TO-BE
{ email, password, name, company_name, country_code, phone?,
  verify_token, terms_agreed, privacy_agreed, marketing_agreed? }
```

- `country` → `country_code` (명확한 네이밍, ISO 3166-1 alpha-2)
- `verify_token` 추가 (필수)

#### 2.2.2 Register Service 변경

1. verify_token JWT 검증 (이메일 일치 + 만료 확인)
2. 기존 로직 유지: bcrypt → Client 생성 → Customer 생성 → Auto Provision
3. `pct_email_verified` = **true** 즉시 설정 (코드 인증 완료 상태)
4. `pct_email_verify_token` = null (링크 인증 불필요)

**산출물**: Register API가 verify_token 필수로 동작  
**사이드 임팩트**: 
- ⚠️ 기존 RegisterPage (FE)와 호환 깨짐 → Phase 3에서 FE 동시 교체
- Google OAuth 경로는 영향 없음 (별도 엔드포인트)

### 4.1 영향 범위

| 영향 대상 | 심각도 | 설명 | 대응책 |

| Portal 로그인 | 🟢 None | 별도 경로 | 변경 없음 |

### 4.2 배포 순서 의존성

```
Phase 1 (BE 이메일 인증 API)  ← 독립, 먼저 배포 가능
    ↓
Phase 2 (BE Register DTO 변경) + Phase 3 (FE Step 1) + Phase 4 (FE Step 2)
    ↑ 이 3개는 반드시 동시 배포 (BE DTO 변경 ↔ FE 호출 싱크)

    ↓

```

- 라우팅: `/register` 경로 유지 → 컴포넌트만 교체이므로 롤백 용이

### Phase 2 (후속 작업 — AMA 비밀번호 분리)

ALTER TABLE amb_users
  ADD COLUMN IF NOT EXISTS usr_ama_pw_set_required BOOLEAN NOT NULL DEFAULT false;

CREATE INDEX IF NOT EXISTS idx_amb_users_ama_pw_set_required
  ON amb_users (usr_ama_pw_set_required)
  WHERE usr_ama_pw_set_required = true;

### Phase 2 (후속 작업 — 약관 이력 테이블)

```sql
CREATE TABLE IF NOT EXISTS amb_user_agreements (
  agr_id          UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  usr_id          UUID        NOT NULL,
  agr_type        VARCHAR(30) NOT NULL,
  agr_agreed      BOOLEAN     NOT NULL DEFAULT true,
  agr_agreed_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  agr_ip          VARCHAR(45),
  agr_created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT fk_user_agreements_users FOREIGN KEY (usr_id) REFERENCES amb_users (usr_id),
  CONSTRAINT ck_agr_type CHECK (agr_type IN ('TOS', 'PRIVACY', 'MARKETING')),
  CONSTRAINT uq_user_agreements_usr_type UNIQUE (usr_id, agr_type)
);

CREATE INDEX IF NOT EXISTS idx_amb_user_agreements_usr_id ON amb_user_agreements (usr_id);
```

## 6. 환경변수

### Portal-API SMTP 설정

이메일 발송은 **DB Gmail SMTP 설정을 우선 사용**하며, DB 설정이 없을 때만 환경변수 fallback:

- **DB 우선**: `amb_smtp_settings` 테이블 → AMA Admin에서 설정한 Gmail SMTP (AES-256-GCM 암호화)
- **환경변수 fallback** (DB 없을 때):

```env
# fallback SMTP (DB 설정이 없을 때만 사용)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=
SMTP_PASS=
SMTP_FROM=noreply@amoeba.site

# 복호화 키 (DB SMTP 비밀번호 복호화용, AMA API와 동일 값)
API_KEY_ENCRYPTION_SECRET=<shared-secret>
```

### 인증코드 설정

```env
# 인증코드 설정
VERIFY_CODE_TTL_MINUTES=10
VERIFY_CODE_MAX_SEND_PER_HOUR=5
```

## 시스템 현황 분석

### 핵심 변경 사항
1. 사이드바에서 SITE_MANAGEMENT, SERVICE_MANAGEMENT 제거
2. 설정 카드뷰에 2개 카드 추가
3. 사이트관리 진입점을 대시보드+법인 목록으로 변경
4. CMS 백엔드 법인 격리 (resolveEntityId 적용)
5. React Query 캐시 키에 entityId 추가

### 기존 인프라 현황
| 구성요소 | 상태 | 설명 |
|---------|------|------|
| `EntitySelector` 컴포넌트 | 완료 | 헤더에서 법인 선택 UI |
| `entity.store.ts` (Zustand) | 완료 | 선택 법인 상태 관리 + localStorage |
| `api-client.ts` 인터셉터 | 완료 | 모든 요청에 `X-Entity-Id` 헤더 자동 주입 |
| `resolveEntityId()` 유틸리티 | 완료 | X-Entity-Id 헤더 → JWT entityId fallback |
| `SettingsPage.tsx` 카드뷰 | 완료 | 16개 카드, `useMyPermissions()` 기반 필터링 |
| CMS 엔티티 `entId` 컬럼 | 완료 | 모든 CMS 테이블에 법인 격리 컬럼 존재 |

### 5단계: 프론트엔드 React Query 캐시 키에 entityId 추가

#### 5-1. useMenus.ts 수정
**파일**: `apps/web/src/domain/site-management/hooks/useMenus.ts`

```typescript
import { useEntityStore } from '@/domain/hr/store/entity.store';

const menuKeys = {
  all: (entityId?: string) => ['cms-menus', entityId] as const,
  tree: (entityId?: string) => [...menuKeys.all(entityId), 'tree'] as const,
};

export const useMenuTree = () => {
  const entityId = useEntityStore((s) => s.currentEntity?.entityId);
  return useQuery({
    queryKey: menuKeys.tree(entityId),
    queryFn: () => cmsMenuService.getMenuTree(),
    enabled: !!entityId,
  });
};
```

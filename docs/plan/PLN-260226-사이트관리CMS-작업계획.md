## 2. 사이드바 메뉴 등록 + 권한 설정 (신규 모듈 필수 작업)

> **사용자 요청**: 새 모듈 추가 시 사이드메뉴 추가, 어드민 그룹 권한 부여, 어드민 설정 메뉴에 추가, 메뉴 접근 권한 추가가 반영되어야 함.

### 2.1 사이드바 메뉴 등록 (`DEFAULT_MENU_CONFIGS`)

**파일**: `apps/api/src/domain/settings/service/menu-config.service.ts`

`DEFAULT_MENU_CONFIGS` 배열에 SITE_MANAGEMENT 항목 추가:

```typescript
// 기존 ASSET_MANAGEMENT (sortOrder: 1600) 다음에 추가
{ menuCode: 'SITE_MANAGEMENT', labelKey: 'common:sidebar.siteManagement', icon: 'Globe', path: '/site', category: 'MODULE', sortOrder: 1700 },
```

- **동작**: 서버 시작 시 `onModuleInit()` → DB에 `SITE_MANAGEMENT` menuCode가 없으면 자동 INSERT
- **어드민 설정 메뉴 자동 노출**: `DEFAULT_MENU_CONFIGS`에 등록되면 **추가 코드 없이** `설정 > 메뉴 설정` 탭(MenuConfigTab)에 자동으로 표시됨 (활성화/순서 관리 가능)

### 2.2 메뉴 코드 상수 등록 (`MENU_CODES`)

**파일**: `apps/api/src/global/constant/menu-code.constant.ts`

`MENU_CODES` 객체에 추가:

```typescript
// 기존 ASSET_MANAGEMENT 다음에 추가
SITE_MANAGEMENT: 'SITE_MANAGEMENT',
```

`MENU_NAMES` 객체에 추가:

```typescript
SITE_MANAGEMENT: 'Site Management',
```

### 2.3 역할별 기본 접근 권한 설정 (`DEFAULT_PERMISSIONS`)

**파일**: `apps/api/src/global/constant/menu-code.constant.ts`

`DEFAULT_PERMISSIONS`의 각 역할에 `SITE_MANAGEMENT` 키 추가:

| 역할 | 접근 허용 | 근거 |
|------|----------|------|
| `SUPER_ADMIN` | `true` (자동) | `ALL_MENU_CODES.map(code => [code, true])` — MENU_CODES에 추가하면 자동 포함 |
| `ADMIN` | `true` (자동) | 동일 — 자동 포함 |
| `MANAGER` | `true` | CMS 콘텐츠 관리 역할 (NFR-CMS-006 요구사항) |
| `MEMBER` | `false` | CMS는 관리자/매니저 전용 기능 |
| `VIEWER` | `false` | CMS 접근 불가 |
| `USER` | `false` | 하위 호환 (MEMBER과 동일 수준) |

```typescript
// MANAGER 객체 내 추가
SITE_MANAGEMENT: true,

// MEMBER 객체 내 추가
SITE_MANAGEMENT: false,

// VIEWER 객체 내 추가
SITE_MANAGEMENT: false,

// USER 객체 내 추가
SITE_MANAGEMENT: false,
```

- **동작**: 서버 시작 시 `MenuPermissionService.onModuleInit()` → DB에 해당 (menuCode, role) 조합이 없으면 자동 INSERT
- **어드민 설정 권한 자동 노출**: `DEFAULT_PERMISSIONS`에 등록되면 **추가 코드 없이** `설정 > 역할별 권한` 탭(RolePermissionTab)에 SITE_MANAGEMENT 행이 자동 표시됨 (ADMIN_GROUP은 전체 ON으로 비활성, USER_GROUP은 MANAGER/MEMBER/VIEWER 별 토글 가능)

### 2.7 전체 동작 흐름 요약

```
[서버 시작]
  │
  ├─ MenuConfigService.onModuleInit()
  │   └─ DEFAULT_MENU_CONFIGS에서 'SITE_MANAGEMENT' → amb_menu_config INSERT
  │      → 설정 > 메뉴 설정 탭에 자동 노출
  │
  ├─ MenuPermissionService.onModuleInit()
  │   └─ DEFAULT_PERMISSIONS에서 각 역할별 SITE_MANAGEMENT → amb_menu_permissions INSERT
  │      → 설정 > 역할별 권한 탭에 자동 노출
  │
  └─ [런타임]
      ├─ GET /settings/permissions/me → ADMIN/MANAGER에게 SITE_MANAGEMENT 반환
      ├─ MainLayout.tsx → useMyMenus() → ICON_MAP[Globe] → 사이드바에 "사이트관리" 렌더링
      └─ /site 라우트 진입 → MenuGuard("SITE_MANAGEMENT") → 권한 확인 후 접근 허용
```

## 8. 검증 방법

| 단계 | 검증 항목 | 방법 |
|------|----------|------|
| 1 | 메뉴/권한 자동 등록 | `npm run dev:api` → DB 확인: `amb_menu_config`에 SITE_MANAGEMENT 행, `amb_menu_permissions`에 5개 역할별 행 |
| 2 | 어드민 설정 노출 | 설정 > 메뉴 설정 탭에 "Site Management" 행 표시, 활성화 토글/순서 변경 가능 |
| 3 | 역할별 권한 노출 | 설정 > 역할별 권한 탭에 SITE_MANAGEMENT 행 표시, ADMIN_GROUP 전체 ON(잠금), USER_GROUP MANAGER:ON/MEMBER:OFF/VIEWER:OFF |
| 4 | 사이드바 렌더링 | `npm run dev:web` → ADMIN 로그인 → 사이드바에 Globe 아이콘 + "사이트관리" 메뉴 표시 |
| 5 | 라우트 접근 | `/site` 접근 → SiteLayout + 서브메뉴 표시 → `/site/menus` 메뉴 관리 화면 |
| 6 | 권한 가드 | MEMBER 로그인 → 사이드바에 "사이트관리" 미표시 → `/site` 직접 접근 시 리다이렉트 |
| 7 | DB 테이블 | 서버 시작 후 `amb_cms_menus`, `amb_cms_pages` 등 9개 테이블 존재 확인 |
| 8 | API 동작 | POST `/api/v1/cms/menus` → 메뉴 + 페이지 생성 → GET 확인 |
| 9 | E2E 플로우 | 메뉴 생성 → 페이지 편집 → 발행 → Public API 조회 |
| 10 | 빌드 | `npm run build` 성공 |
| 11 | 린트 | `npm run lint` 통과 |

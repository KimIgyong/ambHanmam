#### 2.1.1 신규 DB 테이블

##### `amb_sub_plan_features` (plf_) — 플랜 비교 항목
> Portal Comparison Table 14행 + 향후 확장 행

```sql
CREATE TABLE amb_sub_plan_features (
  plf_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plf_feature_key VARCHAR(50) NOT NULL UNIQUE,   -- 'monthly_price', 'storage_base' 등
  plf_label_i18n_key VARCHAR(100) NOT NULL,       -- i18n 키: 'pricing.comparison.monthly_price'
  plf_value_free VARCHAR(200),                    -- Free 플랜 값 (문자열)
  plf_value_basic VARCHAR(200),                   -- Basic 플랜 값
  plf_value_premium VARCHAR(200),                 -- Premium 플랜 값
  plf_is_check BOOLEAN DEFAULT FALSE,             -- ✓/✕ 비교 모드 여부
  plf_highlight BOOLEAN DEFAULT FALSE,            -- 강조 행 여부
  plf_sort_order INT DEFAULT 0,                   -- 표시 순서
  plf_is_active BOOLEAN DEFAULT TRUE,
  plf_created_at TIMESTAMP DEFAULT NOW(),
  plf_updated_at TIMESTAMP DEFAULT NOW()
);
```

##### `amb_sub_plan_tiers` (plt_) — BASIC 티어 상세
> Portal BASIC Tier Detail Table 9행

```sql
CREATE TABLE amb_sub_plan_tiers (
  plt_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  pln_id UUID NOT NULL REFERENCES amb_sub_plans(pln_id),  -- 소속 플랜 (BASIC)
  plt_tier_number INT NOT NULL,                   -- 티어 번호 (1~9)
  plt_users_min INT NOT NULL,                     -- 사용자 최소 (6, 11, 16…)
  plt_users_max INT NOT NULL,                     -- 사용자 최대 (10, 15, 20…)
  plt_monthly_price NUMERIC(10,2) NOT NULL,       -- 월간 요금 ($10, $20…)
  plt_annual_price NUMERIC(10,2) NOT NULL,        -- 연간 요금 ($100, $200…)
  plt_savings NUMERIC(10,2) DEFAULT 0,            -- 연간 절감액
  plt_tokens_monthly INT NOT NULL,                -- 월간 토큰량 (100000, 200000…)
  plt_sort_order INT DEFAULT 0,
  plt_is_active BOOLEAN DEFAULT TRUE,
  plt_created_at TIMESTAMP DEFAULT NOW(),
  plt_updated_at TIMESTAMP DEFAULT NOW()
);
```

##### `amb_sub_plan_addons` (pla_) — ADD-ON 상품
> ADD-ON Pricing Table 3행 + 향후 확장

```sql
CREATE TABLE amb_sub_plan_addons (
  pla_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  pla_addon_key VARCHAR(50) NOT NULL UNIQUE,      -- 'token_addon', 'storage_addon', 'referral_reward'
  pla_label_i18n_key VARCHAR(100) NOT NULL,       -- i18n 키
  pla_value_free VARCHAR(200),                    -- Free 플랜 설정값
  pla_value_basic VARCHAR(200),                   -- Basic 플랜 설정값
  pla_unit VARCHAR(50),                           -- 단위 (10K tokens, 1GB…)
  pla_price VARCHAR(50),                          -- 단가 ($1.00, Free…)
  pla_sort_order INT DEFAULT 0,
  pla_is_active BOOLEAN DEFAULT TRUE,
  pla_created_at TIMESTAMP DEFAULT NOW(),
  pla_updated_at TIMESTAMP DEFAULT NOW()
);
```

## 5. DB 마이그레이션

### 5.1 마이그레이션 SQL

**파일**: `sql/005-create-plan-features-tiers-addons.sql`

- 3개 테이블 CREATE (amb_sub_plan_features, amb_sub_plan_tiers, amb_sub_plan_addons)
- 시드 데이터 INSERT (features 14행 + tiers 9행 + addons 3행)
- 인덱스 생성

### 5.2 적용 순서

1. 로컬 개발: TypeORM `synchronize: true` 자동 반영
2. 스테이징: `docker exec -i amb-postgres-staging psql -U amb_user -d db_amb < sql/005-...`
3. 프로덕션: `docker exec -i amb-postgres-production psql -U amb_user -d db_amb < sql/005-...`

### 5.3 롤백

```sql
DROP TABLE IF EXISTS amb_sub_plan_addons;
DROP TABLE IF EXISTS amb_sub_plan_tiers;
DROP TABLE IF EXISTS amb_sub_plan_features;
```

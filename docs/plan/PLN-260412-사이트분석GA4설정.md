## 2. 시스템 현황 요약

### 이미 구현된 것
- Admin Site Analytics 페이지 (`/admin/site-analytics`) — 2탭 (Analytics / GA Settings)
- Portal Analytics 대시보드 — 서머리 카드 4개 + 트렌드 + 리퍼러 + 페이지뷰 + 트래픽 소스
- Backend Analytics API 12개 엔드포인트 (`/api/v1/analytics/*`)
- `amb_site_event_logs` 테이블 + 엔티티
- GA Settings 페이지 — Portal Measurement ID 1개 설정
- Portal-web GA4 클라이언트 트래킹 — 16개 이벤트 함수 + gtag 설정

### 핵심 갭
1. **데이터 수집 미연결**: Portal-web이 GA4 전용 → 백엔드 이벤트 API 미호출 → `amb_site_event_logs` 비어있음
2. **GA Settings 1개만 지원**: App(ama.amoeba.site) Measurement ID 저장 컬럼/UI 없음
3. **서머리 카드에 퍼널 지표 미포함**: `ama_navigation` (AMA 이동) 이벤트 타입 없음

### 4.2 Portal-API 이벤트 수집 서비스

```typescript
// apps/portal-api/src/domain/analytics/analytics.service.ts
@Injectable()
export class PortalAnalyticsService {
  constructor(
    @InjectRepository(SiteEventLogEntity)
    private readonly eventLogRepo: Repository<SiteEventLogEntity>,
  ) {}

  async recordEvent(dto: CreateSiteEventRequest, ip?: string, userAgent?: string) {
    const entity = this.eventLogRepo.create({
      selSite: 'portal',          // portal-api에서 오는 이벤트는 무조건 portal
      selEventType: dto.event_type,
      selPagePath: dto.page_path || null,
      selReferrer: dto.referrer || null,
      selIpAddress: ip || null,
      selUserAgent: userAgent?.substring(0, 500) || null,
      selMetadata: dto.metadata || {},
    });
    await this.eventLogRepo.save(entity).catch(() => {}); // fire-and-forget
  }
}
```

### 4.3 Portal-web 서버 이벤트 유틸

```typescript
// apps/portal-web/src/lib/server-events.ts
const API_BASE = import.meta.env.VITE_API_URL || '';

export function trackServerEvent(params: {
  event_type: string;
  page_path?: string;
  referrer?: string;
  metadata?: Record<string, unknown>;
}) {
  // fire-and-forget: 실패해도 사용자 경험에 영향 없음
  fetch(`${API_BASE}/analytics/events`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ site: 'portal', ...params }),
  }).catch(() => {});
}
```

### 4.4 ga-events.ts 병렬 전송 패턴

```typescript
// 기존
export function trackRegisterPageView(step: number | string) {

// 변경 후 (서버 이벤트 병렬 추가)
import { trackServerEvent } from './server-events';

export function trackRegisterPageView(step: number | string) {
  window.gtag?.('event', 'register_page_view', { step: String(step) });
  trackServerEvent({ event_type: 'register_visit', page_path: '/register', metadata: { step } });
}
```

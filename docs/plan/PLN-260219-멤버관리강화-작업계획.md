## 2. 단계별 작업 계획

### STEP 1: 백엔드 - API 확장

#### 1-1. MemberService.findOne() 확장

**파일**: `apps/api/src/domain/members/service/member.service.ts`

기존 `findOne(id)` 메서드의 응답에 법인/부서/HR 직원 정보를 추가.

```
변경 전: { userId, email, name, department, role, groups, createdAt }
변경 후: + entityRoles[], deptRoles[], hrEmployees[]
```

구현:
- EntityUserRoleEntity JOIN HrEntityEntity 조회
- UserDeptRoleEntity JOIN DepartmentEntity 조회
- EmployeeEntity에서 해당 법인들의 직원 이름 매칭 조회

#### 1-2. MemberController에 법인 역할 관리 엔드포인트 추가

**파일**: `apps/api/src/domain/members/controller/member.controller.ts`

| 메서드 | 경로 | 설명 | 권한 |
|--------|------|------|------|
| POST | `/members/:id/entity-roles` | 법인 역할 배정 | ADMIN |
| DELETE | `/members/:id/entity-roles/:eurId` | 법인 역할 해제 | ADMIN |

> 부서 역할은 기존 `POST /departments/roles`, `DELETE /departments/roles/:id` 재사용

#### 1-3. members.module.ts에 엔티티 추가

**파일**: `apps/api/src/domain/members/members.module.ts`

추가할 엔티티:
- `EntityUserRoleEntity`
- `HrEntityEntity`
- `UserDeptRoleEntity`
- `DepartmentEntity`
- `EmployeeEntity`

#### 1-4. DTO 추가

**신규 파일**: `apps/api/src/domain/members/dto/request/assign-entity-role.request.ts`

```typescript
{ entity_id: string (UUID), role: string }
```

## 3. 사이드 임팩트 분석

### 3.1 안전 (영향 없음)

| 영역 | 이유 |
|------|------|
| 기존 멤버 목록 API | `GET /members` 변경 없음, `findAll()` 미변경 |
| 기존 그룹/초대 기능 | 완전히 독립적, 코드 변경 없음 |
| 기존 부서 API | 기존 엔드포인트 재사용, 변경 없음 |
| 기존 HR API | 조회만 수행, 변경 없음 |
| DB 스키마 | 신규 테이블/컬럼 없음 |

### 3.2 주의 필요

| 영역 | 리스크 | 대응 |
|------|--------|------|
| `members.module.ts` | 엔티티 import 증가로 모듈 의존성 확대 | 모든 엔티티가 이미 다른 모듈에서 사용 중이므로 안전 |
| `MemberService.findOne()` | 응답 구조 변경 | 기존 필드 유지 + 신규 필드 추가만 수행. 기존 호출처(MemberRoleChangeModal)는 기존 필드만 사용하므로 하위 호환 |
| `MemberListTab.tsx` | 사용자명 클릭 동작 추가 | 기존 텍스트 → 링크 변경. 기능적 차이 없음 |
| 부서 배정 API | 기존 `POST /departments/roles` 재사용 | 동일 API 사용, 프론트에서 호출 경로만 다름 |

### 3.3 코딩 컨벤션 준수 확인

| 규칙 | 적용 |
|------|------|
| DB 컬럼 네이밍 | 기존 엔티티 활용, 신규 컬럼 없음 |
| API 규칙 | Request: `snake_case`, Response: `camelCase` |
| 파일 네이밍 | 컴포넌트: PascalCase, 서비스: kebab-case |
| 표준 응답 | `{ success, data, timestamp }` |
| i18n | 번역 키 사용, 하드코딩 금지 |

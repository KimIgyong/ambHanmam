## 시스템 현황 요약

### 핵심 문제점
1. **`USER` vs `MEMBER` 역할 혼용** — 초대 폼은 `USER` 사용, User Entity 기본값은 `MEMBER`
2. **`SUPER_ADMIN` 관리자 체크 누락** — GroupListTab, NoticesPage, NoticeDetailPage
3. **메뉴 카테고리 오분류** — SCHEDULE, PROJECT_MANAGEMENT이 MODULE에 있으나 WORK_TOOL이어야 함
4. **CHAT 메뉴가 Role Permissions에 표시** — 별도 관리인데 노출됨
5. **RoleBadge 색상 누락** — SUPER_ADMIN, MEMBER, VIEWER에 대한 색상 미정의
6. **초대 폼에 VIEWER 옵션 없음**

## 단계별 구현 계획

### Phase 1: 역할(Role) 통일 — `USER` → `MEMBER`

**대상 파일**: InvitationFormModal.tsx, MemberRoleChangeModal.tsx

#### 1-1. InvitationFormModal.tsx
- role 드롭다운 옵션: `USER` → `MEMBER` 변경
- `VIEWER` 옵션 추가
- "Function" 라벨을 "Function (AI Agent)" 로 명확화

#### 1-2. MemberRoleChangeModal.tsx
- role 옵션: `USER` → `MEMBER` 변경
- `VIEWER` 옵션 추가

#### 1-3. DB 즉시 마이그레이션 (배포 후 실행)

**문제 발견**: `ROLES` 상수에 `USER`가 없어 `initDefaults()`에서 `USER` 역할의 메뉴 권한 레코드가 시드되지 않음.
→ `usr_role = 'USER'`인 사용자는 `getMyMenus()` 호출 시 `WHERE mpmRole = 'USER'`로 조회해도 **레코드 0건** → 사이드바 메뉴 비정상.

**마이그레이션 SQL** (배포 후 스테이징 DB에서 실행):
```sql
-- 1. usr_role = 'USER' → 'MEMBER' 일괄 변환
UPDATE amb_users SET usr_role = 'MEMBER' WHERE usr_role = 'USER';

-- 2. 불필요한 USER 역할 권한 레코드 정리 (있을 경우)
DELETE FROM amb_menu_permissions WHERE mpm_role = 'USER';
```

**사이드 임팩트**: 없음 — `MEMBER`는 이미 `ROLES`에 포함되어 `initDefaults()`에서 권한 레코드가 존재하므로, 변환 후 메뉴 권한이 정상 적용됨.

### Phase 2: SUPER_ADMIN 체크 누락 수정

**대상 파일**: GroupListTab.tsx, NoticesPage.tsx, NoticeDetailPage.tsx

#### 2-1. GroupListTab.tsx
- `user?.role === 'ADMIN'` → `user?.role === 'SUPER_ADMIN' || user?.role === 'ADMIN'`

#### 2-2. NoticesPage.tsx
- `user?.role === 'ADMIN'` → `user?.role === 'SUPER_ADMIN' || user?.role === 'ADMIN'`

#### 2-3. NoticeDetailPage.tsx
- `user?.role === 'ADMIN'` → `user?.role === 'SUPER_ADMIN' || user?.role === 'ADMIN'`

**사이드 임팩트**: 없음 — 권한 확장만 발생

---

### Phase 3: RoleBadge 색상 맵 완성

**대상 파일**: MemberListTab.tsx

RoleBadge의 색상 맵에 누락된 역할 추가:
- `SUPER_ADMIN`: 빨강 계열 (최고 관리자 강조)
- `MEMBER`: 초록 계열 (기존 USER와 동일)
- `VIEWER`: 회색 계열 (최소 권한)

---

### Phase 4: CHAT 메뉴 숨김 + 카테고리 재분류

#### 4-1. RolePermissionTab.tsx (Frontend)
- `CHAT_` prefix 메뉴를 Role Permissions 목록에서 필터링하여 숨김

#### 4-2. menu-config.service.ts (Backend)
- `SCHEDULE`: `category: 'MODULE'` → `category: 'WORK_TOOL'`, `sortOrder: 750`
- `PROJECT_MANAGEMENT`: `category: 'MODULE'` → `category: 'WORK_TOOL'`, `sortOrder: 760`

**사이드 임팩트**:
- DB에 이미 시드된 데이터는 `onModuleInit`에서 `upsert` 로직에 의해 업데이트됨
- 사이드바 그룹핑이 변경될 수 있음 (WORK_TOOL 섹션에 표시)

#### 4-3. menu-code.constant.ts (Backend)
- DEFAULT_PERMISSIONS에서 `USER` 항목 **완전 삭제** (DB 마이그레이션으로 USER 사용자가 없으므로 불필요)
- `RolesGuard`의 `USER: 2` 매핑도 제거 (더 이상 호환 유지 불필요)

| 리스크 | 대응 |

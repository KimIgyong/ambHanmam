#### Step 2.4: URL 자동 하이퍼링크 유틸 (D-4)
**파일**: `apps/web/src/global/util/autoLinkUrls.ts` (🆕 신규)

**함수 시그니처**:
```typescript
export function autoLinkUrls(html: string, maxLength?: number): string
```

**처리 로직**:
1. HTML 문자열에서 `<a>` 태그를 임시 플레이스홀더로 치환
2. 나머지 텍스트에서 URL 패턴(`https?://[^\s<>"]+`) 매칭
3. 매칭된 URL을 `<a href="..." target="_blank" rel="noopener noreferrer">표시텍스트</a>`로 변환
4. 표시 텍스트는 maxLength(기본 80) 초과 시 말줄임
5. 플레이스홀더 복원

**적용 위치**:
- `IssueDetailModal.tsx` — Description 렌더링: `sanitizeHtml(description)` → `autoLinkUrls(sanitizeHtml(description))`
- `IssueDetailModal.tsx` — Comment content 렌더링: 동일 적용

#### Step 3.1: DB 스키마 + Entity (D-3)
**파일**: `apps/api/src/domain/issues/entity/issue-comment-reaction.entity.ts` (🆕)

**테이블**: `amb_issue_comment_reactions`

```typescript
@Entity('amb_issue_comment_reactions')
@Unique(['iscId', 'usrId', 'icrType'])
export class IssueCommentReactionEntity {
  @PrimaryGeneratedColumn('uuid', { name: 'icr_id' })
  icrId: string;

  @Column({ name: 'isc_id', type: 'uuid' })
  iscId: string;             // 코멘트 ID

  @Column({ name: 'usr_id', type: 'uuid' })
  usrId: string;             // 사용자 ID

  @Column({ name: 'icr_type', length: 20 })
  icrType: string;           // LIKE | CHECK | PRAY | GRIN | LOVE

  @CreateDateColumn({ name: 'icr_created_at' })
  icrCreatedAt: Date;
}
```

**SQL 마이그레이션** (스테이징/프로덕션용):
```sql
CREATE TABLE amb_issue_comment_reactions (
  icr_id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  isc_id UUID NOT NULL REFERENCES amb_issue_comments(isc_id) ON DELETE CASCADE,
  usr_id UUID NOT NULL,
  icr_type VARCHAR(20) NOT NULL,
  icr_created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE (isc_id, usr_id, icr_type)
);

CREATE INDEX idx_icr_comment ON amb_issue_comment_reactions(isc_id);
```

#### Step 3.2: 리액션 API (D-3)
**파일**: `issue.controller.ts`, `issue.service.ts`

**엔드포인트**:
```
POST /api/v1/issues/:id/comments/:commentId/reactions
Body: { type: 'LIKE' | 'CHECK' | 'PRAY' | 'GRIN' | 'LOVE' }
Response: { success: true, data: { action: 'added' | 'removed', reactions: ReactionSummary } }
```

**로직**: Talk 리액션과 동일한 토글 방식
1. 이미 해당 type 리액션이 있으면 삭제 (removed)
2. 없으면 추가 (added)
3. 갱신된 리액션 요약(ReactionSummary) 반환

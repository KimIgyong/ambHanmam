| 파일 | 역할 | 현재 상태 |

### 1.3 핵심 데이터 흐름

```
Frontend                          Backend
─────────                         ─────────
useCellToday() ─────────────────→ GET /today/cell → getCellToday()
                                   └─ UserCellEntity → 소속 Cell 전체 멤버 → MemberTodaySummary[]
useAllToday()  ─────────────────→ GET /today/all  → getAllToday()
                                   └─ EntityUserRole → 엔티티 전체 멤버 → MemberTodaySummary[]

MemberTodaySummary = { userId, userName, department?, unitName?, position?, ... }
```

#### 4-2. UnitSection 서브 컴포넌트 (부서 서브탭 콘텐츠)

```tsx
function UnitSection({ units, unitGroups, selectedUnitId, onSelectUnit }) {
  return (
    <div>
      {/* 부서 칩 필터 */}
      <div className="flex flex-wrap gap-2 mb-4">
        <Chip active={!selectedUnitId} onClick={() => onSelectUnit(null)}>전체부서</Chip>
        {units?.map(u => (
          <Chip key={u.unitId} active={selectedUnitId === u.unitId}
                onClick={() => onSelectUnit(u.unitId)}>
            {u.unitName}
          </Chip>
        ))}
      </div>

      {/* 부서별 카드뷰 */}
      <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
        {filteredUnits.map(({ unitId, unitName, manager, members }) => (
          <UnitCard key={unitId} unitName={unitName} manager={manager}
                    memberCount={members.length} onClick={() => onSelectUnit(unitId)} />
        ))}
      </div>

      {/* 특정 부서 선택 시: 해당 부서 멤버 Today 리스트 */}
      {selectedUnitId && <UnitMemberTodayList unitId={selectedUnitId} members={...} />}
    </div>
  );
}
```

#### 4-3. CellSection 서브 컴포넌트 (셀 서브탭 콘텐츠)

```tsx
function CellSection({ cells, allMembers, selectedCellIds, onToggleCell }) {
  return (
    <div>
      {/* 셀 칩 필터 */}
      <div className="flex flex-wrap gap-2 mb-4">
        <Chip active={selectedCellIds.length === 0} onClick={() => setSelectedCellIds([])}>전체셀</Chip>
        {cells?.map(c => (
          <Chip key={c.cellId} active={selectedCellIds.includes(c.cellId)}
                onClick={() => onToggleCell(c.cellId)}>
            {c.cellName}
          </Chip>
        ))}
      </div>

      {/* 셀별 카드뷰 (중복 선택 가능) */}
      <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
        {filteredCells.map(c => (
          <CellCard key={c.cellId} cellName={c.cellName}
                    members={getCellMembers(allMembers, c.cellId)}
                    isSelected={selectedCellIds.includes(c.cellId)}
                    onClick={() => onToggleCell(c.cellId)} />
        ))}
      </div>
    </div>
  );
}
```

#### 4-4. 조직 API 훅 추가

**파일**: `apps/web/src/domain/today/hooks/useMyToday.ts` 또는 별도 훅 파일

```typescript
export const useOrganizationUnits = () => {
  const entityId = useEntityStore((s) => s.currentEntity?.entityId);
  return useQuery({
    queryKey: ['organization', 'units', entityId],
    queryFn: () => apiClient.get('/entity-settings/organization/units').then(r => r.data.data),
    staleTime: 1000 * 60 * 5,
    enabled: !!entityId,
  });
};

export const useOrganizationCells = () => {
  const entityId = useEntityStore((s) => s.currentEntity?.entityId);
  return useQuery({
    queryKey: ['organization', 'cells', entityId],
    queryFn: () => apiClient.get('/entity-settings/organization/cells').then(r => r.data.data),
    staleTime: 1000 * 60 * 5,
    enabled: !!entityId,
  });
};
```

## 4. 사이드 임팩트 분석

### 4.1 ViewScopeTab 영향 범위

| 사용처 | 영향 | 대응 |
|--------|------|------|
| TodayPage | 직접 대상 | org 스코프 활용 |
| 기타 모듈 (이슈, 할일 등) | `scopes` prop 미지정 → 기존 4개+org 모두 표시 | `scopes` prop으로 기존 4개만 지정하거나, 기존 모듈은 `'org'` 무시하도록 onScopeChange에서 처리 |

**해결 전략**: ViewScopeTab에 `scopes` 옵셔널 prop 추가. 미지정 시 기존 4개만 표시 (backward compatible), TodayPage에서만 5개 지정

### 4.2 ReportsModal 제거 영향

- TodayPage에서만 사용 → TodayPage에서 import/렌더링 제거
- `TodayAiAnalysis.tsx`의 `ReportsModal` export는 유지 (다른 곳에서 사용 가능성)
- 실제로 import하는 곳이 TodayPage뿐이면 추후 정리

### 4.3 백엔드 CellToday 응답 확장

- 기존 `{ members: MemberTodaySummary[] }` → `{ cells: [], members: [] }` (필드 추가)
- 기존 프론트에서 `data.members`로 접근 → 호환성 유지 (members 그대로)
- 새 `cells` 필드와 `cellIds`는 추가 필드이므로 breaking change 없음

### 4.4 성능 고려

- 조직별 탭: `useAllToday()` + `useOrganizationUnits()` + `useOrganizationCells()` 3개 쿼리
- staleTime을 적절히 설정 (organization 5분, allToday 2분)
- 프론트 그룹핑은 `useMemo`로 메모이제이션

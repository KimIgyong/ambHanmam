## 1. 구현 단계

### Phase 1: 백엔드 — Presence 서비스

#### Step 1.1: PresenceService 생성
**새 파일**: `apps/api/src/domain/amoeba-talk/service/presence.service.ts`

- **인메모리 Map**: `Map<string, { lastHeartbeat: Date; status: 'online' | 'offline' }>`
- `heartbeat(userId)` — 하트비트 수신, 상태 online 업데이트
- `getStatus(userIds: string[])` — 여러 사용자 상태 일괄 조회
- `getUserStatus(userId)` — 단일 사용자 상태 조회
- `checkIdleUsers()` — 60초 이상 하트비트 없는 사용자를 offline 전환, 변경된 사용자 목록 반환
- `@Cron('*/30 * * * * *')` — 30초마다 `checkIdleUsers()` 실행

상태 변경 시 → `TalkSseService` 로 `presence:update` 이벤트 발행

#### Step 1.2: Presence Controller 생성
**새 파일**: `apps/api/src/domain/amoeba-talk/controller/presence.controller.ts`

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/talk/presence/heartbeat` | 하트비트 수신 |
| GET | `/talk/presence/status` | 사용자 상태 조회 (query: `user_ids`) |
| SSE | `/talk/presence/events` | 전역 Presence SSE 스트림 |

#### Step 1.3: TalkSseService 확장
**수정 파일**: `apps/api/src/domain/amoeba-talk/service/talk-sse.service.ts`

- 기존 채널별 Subject에 더해 **전역 Presence Subject** 추가
- `emitPresence(userId, status)` — 전역 Presence 이벤트 발행
- `subscribePresence()` — Presence 이벤트 스트림 구독

#### Step 1.4: Module 업데이트
**수정 파일**: `apps/api/src/domain/amoeba-talk/amoeba-talk.module.ts`

- PresenceService, PresenceController 등록
- `ScheduleModule` 의존 확인 (Cron용)

## 3. 사이드 임팩트 분석

### 3.1 안전
- 기존 SSE 스트림에 영향 없음 (별도 Presence SSE 스트림)
- 기존 채팅 기능에 영향 없음
- 인메모리 Map이므로 DB 마이그레이션 불필요

### 3.2 주의사항
- **서버 재시작 시**: 모든 사용자 offline (30초 후 하트비트로 복구)
- **SSE 연결 수**: 기존 채널별 SSE + 전역 Presence SSE = 동시 2개 연결
- **네트워크 부하**: 하트비트 30초 주기 — 경량 POST 요청이므로 미미

## 4. 검증 방법

1. `npm run build` — 빌드 성공 확인
2. Amoeba Talk 페이지 접속 → 본인 상태 online 확인
3. MemberPanel 열기 → 온라인 사용자 초록 점, 오프라인 회색 점
4. DM 채널 선택 → 헤더에 "Active now" 또는 "Offline" 표시
5. 브라우저 탭 전환 후 5분 대기 → 상태 offline 전환 확인
6. 탭 복귀 → 상태 online 복귀 확인
7. 브라우저 닫기 → 60초 후 offline 전환 확인
8. 스테이징 배포 후 2명 이상 동시 접속 테스트

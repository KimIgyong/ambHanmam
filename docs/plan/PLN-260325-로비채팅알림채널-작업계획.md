**작성일**: 2026-03-25

## 1. 구현 개요

로비 채팅(Amoeba Talk)의 ChannelList 최상단에 "알림 센터" 가상 채널을 추가하여, 사용자가 받은 모든 알림을 대화 형식으로 확인할 수 있는 기능 구현.

**핵심 원칙:**  
- 백엔드 변경 없음 (기존 Notification API 활용)
- 프론트엔드 가상 채널 방식 (DB 채널 생성 불필요)
- 기존 컴포넌트/로직 최소 변경

---

## 2. 단계별 구현 계획

### STEP 1: 상수 및 훅 준비

**1-1. 가상 채널 ID 상수**
- 파일: `apps/web/src/domain/amoeba-talk/store/talk.store.ts`
- 내용: `NOTIFICATION_CHANNEL_ID = '__NOTIFICATION_CENTER__'` 상수 export

**1-2. useNotifications 훅 신규 생성**
- 파일: `apps/web/src/domain/amoeba-talk/hooks/useNotifications.ts` (신규)
- 기능:
  - `useNotificationList()` - React Query infinite query로 알림 페이지네이션 조회
  - `useNotificationUnread()` - 미읽음 카운트 조회
  - `useMarkNotificationRead()` - 단건 읽음 처리 mutation
  - `useMarkAllNotificationsRead()` - 전체 읽음 처리 mutation
- API: 기존 `/notifications` 엔드포인트 사용

### STEP 2: ChannelList 알림 채널 아이템 추가

**2-1. ChannelList.tsx 수정**
- 채널 섹션/DM 섹션 위에 알림 센터 아이템 렌더링
- 알림 센터 아이템 구성:
  - Bell 아이콘 (indigo-500 배경)
  - "알림 센터" 타이틀
  - 미읽음 배지 (기존 notification unreadCount)
  - 최신 알림 미리보기
- 클릭 시 `setCurrentChannelId(NOTIFICATION_CHANNEL_ID)`

### STEP 3: NotificationChatView 컴포넌트 신규 생성

**3-1. NotificationChatView.tsx** (신규)
- 파일: `apps/web/src/domain/amoeba-talk/components/NotificationChatView.tsx`
- 구성:
  - **헤더**: 🔔 알림 센터 제목 + "모두 읽음" 버튼
  - **알림 목록**: 무한 스크롤 (useInfiniteQuery)
  - **알림 아이템**: 타입 아이콘 + 발신자/제목 + 상세 메시지 + 시간 + 읽음 상태
  - **클릭 네비게이션**: 알림 타입별 리소스 페이지 이동 (기존 NotificationBell의 handleNotificationClick 로직 재사용)
- 읽기 전용 (메시지 입력 영역 없음)

### STEP 4: AmoebaTalkPage 통합

**4-1. AmoebaTalkPage.tsx 수정**
- `currentChannelId === NOTIFICATION_CHANNEL_ID` 분기 추가
- 알림 센터 선택 시:
  - ChannelHeader, TalkMessageList, TalkMessageInput 대신 NotificationChatView 렌더링
  - MemberPanel 숨김
  - Talk SSE 연결 스킵

### STEP 5: i18n 키 추가

**5-1. talk.json 수정 (ko/en/vi)**
- `notificationCenter`: "알림 센터" / "Notifications" / "Thông báo"
- `markAllRead`: "모두 읽음" / "Mark all read" / "Đánh dấu tất cả đã đọc"
- `noNotificationsDesc`: "알림이 없습니다" / "No notifications" / "Không có thông báo"
- `notificationGoTo`: "바로가기" / "Go to" / "Đi tới"

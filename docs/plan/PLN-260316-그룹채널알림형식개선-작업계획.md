## 2. 구현 계획

### 단계 1: 백엔드 - `sendMessagePush()` 수정

**파일**: `apps/api/src/domain/amoeba-talk/service/message.service.ts`

**현재 코드** (L232-280):
```typescript
private async sendMessagePush(
  channelId: string,
  senderId: string,
  senderName: string,
  content: string,
  mentionUserIds?: string[],
): Promise<void> {
  // ...
  // SSE 알림
  ntfTitle: isMentioned ? `${senderName} mentioned you` : senderName,
  ntfMessage: preview,
  
  // Web Push
  title: senderName,
  body: preview,
}
```

**수정 내용**:
1. 채널 정보 조회 추가: `this.channelRepo.findOne({ where: { chnId: channelId } })`
2. 채널 타입에 따라 title/body 분기:
   - `DIRECT`: 현행 유지 (`title: senderName`, `body: preview`)
   - `PUBLIC/PRIVATE`: 
     - `title: channelName`
     - `body: "${senderName}: ${preview}"`
3. 멘션 알림도 동일 로직 적용:
   - `DIRECT`: `title: "${senderName} mentioned you"`, `body: preview`
   - `PUBLIC/PRIVATE`: `title: channelName`, `body: "${senderName} mentioned you: ${preview}"`

**수정 코드 (예시)**:
```typescript
private async sendMessagePush(...): Promise<void> {
  try {
    // 채널 정보 조회 (추가)
    const channel = await this.channelRepo.findOne({ where: { chnId: channelId } });
    const isGroupChannel = channel && channel.chnType !== 'DIRECT';
    const channelName = channel?.chnName || 'Chat';

    // ... (기존 멤버 조회, 멘션 변환 로직 동일) ...

    // 알림 title/body 결정
    const pushTitle = isGroupChannel ? channelName : senderName;
    const pushBody = isGroupChannel ? `${senderName}: ${preview}` : preview;

    // SSE 알림
    for (const recipientId of recipientIds) {
      const isMentioned = mentionSet.has(recipientId);
      let ntfTitle: string;
      let ntfMessage: string;

      if (isGroupChannel) {
        ntfTitle = channelName;
        ntfMessage = isMentioned
          ? `${senderName} mentioned you: ${preview}`
          : `${senderName}: ${preview}`;
      } else {
        ntfTitle = isMentioned ? `${senderName} mentioned you` : senderName;
        ntfMessage = preview;
      }

      this.notificationSseService.emit(recipientId, { ntfTitle, ntfMessage, ... });
    }

    // Web Push
    await this.pushService.sendPushToMany(recipientIds, {
      title: pushTitle,
      body: pushBody,
      ...
    });
  }
}
```

### 단계 2: 프론트엔드 - `NotificationModal` SSE 모달 타이틀 수정

**파일**: `apps/web/src/global/components/NotificationModal.tsx`

**현재 코드** (L155-158):
```typescript
function getDisplayTitle(notification, t) {
  const sender = notification.senderName || notification.ntfTitle;
  if (notification.ntfType === 'TALK_MESSAGE' || notification.ntfType === 'TALK_MENTION') {
    return t('talkMessage', { sender });
  }
  return notification.ntfTitle;
}
```

**수정 내용**:
- 그룹 채널 알림인 경우 `ntfTitle`(=채널명)을 직접 사용
- `ntfTitle`과 `senderName`이 다르면 그룹 채널로 판단

```typescript
function getDisplayTitle(notification, t) {
  if (notification.ntfType === 'TALK_MESSAGE' || notification.ntfType === 'TALK_MENTION') {
    // ntfTitle이 채널명인 경우 (그룹 채널) → 채널명 직접 표시
    return notification.ntfTitle;
  }
  return notification.ntfTitle;
}
```

→ 사실상 그룹/DM 모두 `ntfTitle`을 그대로 사용하면 됨 (백엔드에서 이미 분기 처리). 따라서 `getDisplayTitle()`을 단순화.

## 3. 사이드 임팩트 분석

| 항목 | 영향 | 대응 |
|------|------|------|
| DM 알림 | 변경 없음 (DIRECT 분기) | - |
| 멘션 알림 | 그룹 채널에서 채널명 + 멘션 메시지 | 포맷 조정 |
| 채널 DB 조회 추가 | 1건 추가 쿼리 | 이미 channelId 인덱스, 성능 영향 미미 |
| push-sw.js | 수정 불필요 | `data.title`/`data.body` 그대로 사용 |
| 모바일 앱 | Web Push 페이로드 변경이 그대로 반영 | 추가 작업 불필요 |
| 알림 센터 목록 | TALK_MESSAGE는 DB 미저장 | 영향 없음 |

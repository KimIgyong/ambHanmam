## 2. Phase 1: PC 화면 중앙 모달 알림

### Step 1-1: NotificationModal 컴포넌트 생성

**신규 파일**: `apps/web/src/global/components/NotificationModal.tsx`

**기능**:
- Zustand store의 알림 큐 구독
- 화면 중앙에 모달 팝업 표시
- 알림 타입별 아이콘 + 제목 + 메시지
- 자동 닫힘 (5초) + 수동 닫힘 (X 버튼)
- "바로가기" 버튼 → 리소스 이동
- 알림 큐 FIFO 처리 (한번에 1개 표시)
- 반투명 배경 (작업 차단하지 않음, pointer-events-none)

**UI 구성**:
```
┌──────────────────────────────┐
│  [아이콘] 알림 제목      [X] │
│  메시지 내용 미리보기         │
│                  [바로가기]   │
│  ████████████████ 진행바     │
└──────────────────────────────┘
```
- 상단 중앙에 슬라이드 다운 애니메이션
- 하단 프로그레스 바 (5초 카운트다운)
- 같은 알림이 쌓이면 카운트 표시 (+3건)

### Step 1-2: 알림 큐 Store 확장

**수정 파일**: `apps/web/src/global/store/notification.store.ts`

**추가 필드**:
```typescript
modalQueue: NotificationItem[]      // 모달 표시 대기열
currentModal: NotificationItem | null // 현재 표시 중
addToModalQueue: (item: NotificationItem) => void
dismissModal: () => void
```

### Step 1-3: SSE 훅 수정

**수정 파일**: `apps/web/src/global/hooks/useNotificationSSE.ts`

**변경 내용**:
- 기존 sonner toast 대신 (또는 추가로) modalQueue에 추가
- 현재 경로 확인하여 해당 리소스 페이지에 있으면 모달 미표시
  - 예: Talk 페이지에서 채팅 알림 → 모달 미표시

### Step 1-4: MainLayout에 모달 마운트

**수정 파일**: `apps/web/src/layouts/MainLayout.tsx`

**변경 내용**:
```tsx
<NotificationModal />  // 최상위에 추가
```

### Step 1-5: i18n 번역 추가

**수정 파일**: `apps/web/src/locales/{en,ko,vi}/notifications.json`

**추가 키**:
```json
"modal": {
  "goTo": "Go to",
  "dismiss": "Dismiss",
  "moreNotifications": "+{{count}} more"
}
```

## 3. Phase 2: 채팅 메시지 SSE 알림 추가

### Step 2-1: 채팅 메시지 알림 이벤트 발행

**수정 파일**: `apps/api/src/domain/amoeba-talk/service/message.service.ts`

**변경 내용**:
- `sendMessage()` 에서 기존 Push 발송에 추가로 SSE 이벤트 발행
- `TALK_MESSAGE` 타입 활용 (이미 정의됨, 미사용 중)
- 수신자: 채널 멤버 중 발신자 제외
- DB 저장은 하지 않음 (채팅은 빈번하므로 알림 DB에 쌓지 않음)

```typescript
// SSE로 실시간 알림 전송 (DB 저장 없이)
this.notificationSseService.emitToMany(recipientIds, {
  type: 'TALK_MESSAGE',
  title: senderName,
  message: messagePreview,
  resourceType: 'TALK',
  resourceId: channelId,
});
```

### Step 2-2: SSE 훅에서 TALK_MESSAGE 처리

**수정 파일**: `apps/web/src/global/hooks/useNotificationSSE.ts`

**변경 내용**:
- `TALK_MESSAGE` 타입 수신 시 모달 큐에 추가
- 현재 Talk 페이지에서 해당 채널 열려 있으면 → 모달 미표시
- navigateToResource에 TALK 타입 추가: `/amoeba-talk?channel={id}`

### Step 2-3: 알림벨에 TALK_MESSAGE 아이콘 추가

**수정 파일**: `apps/web/src/global/components/NotificationBell.tsx`

**변경 내용**:
- TALK_MESSAGE 타입 아이콘 추가 (💬 MessageCircle)

## 4. Phase 3: 모바일 Push Notification

### Step 3-1: Capacitor Push 플러그인 설치

```bash
cd apps/mobile
npm install @capacitor/push-notifications
npx cap sync
```

### Step 3-2: FCM 설정 (Android)

- `google-services.json` 배치 (android/app/)
- Firebase 프로젝트 설정

### Step 3-3: Push 서비스 생성

**신규 파일**: `apps/mobile/src/services/push.service.ts`

**기능**:
- 앱 시작 시 Push 권한 요청
- FCM 토큰 취득
- 백엔드 `/push/subscribe` 에 토큰 등록
- 알림 수신 핸들러
- 알림 클릭 → 화면 이동 핸들러

### Step 3-4: 백엔드 Push 서비스 확장

**수정 파일**: `apps/api/src/domain/notification/service/push.service.ts`

**변경 내용**:
- `push-subscription` 테이블에 `psb_device_type` 컬럼 추가 ('web' | 'android' | 'ios')
- Web Push: 기존 web-push 라이브러리 사용
- 모바일 Push: firebase-admin SDK 또는 FCM HTTP v1 API 사용
- `sendPush()`에서 device_type 분기 처리

### Step 3-5: DB 마이그레이션

```sql
ALTER TABLE amb_push_subscriptions
ADD COLUMN psb_device_type VARCHAR(10) DEFAULT 'web' NOT NULL,
ADD COLUMN psb_fcm_token VARCHAR(500);
```

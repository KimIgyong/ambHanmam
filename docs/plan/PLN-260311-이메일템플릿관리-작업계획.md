## 2. 시스템 현황 분석

### 2.1 이메일 발송 흐름 (AS-IS)

```
[포탈 계정 생성]
Portal Bridge → POST /portal-bridge/customers/:id/create-account
  → PortalBridgeService.createInternalAccount()
  → MailService.sendRawEmail()  ← 하드코딩된 HTML, 로그인 URL 없음

[법인 초대]
Invitation → POST /invitations
  → InvitationService.create()
  → MailService.sendInvitationEmail()  ← styled HTML, 법인 브랜딩 포함
    ← HrEntityEntity에서 entEmailDisplayName/Color/LogoUrl 조회
```

### 2.2 기존 설정 인프라 패턴

- `amb_smtp_settings` → ADMIN 전역 설정 (SmtpSettingsService/Controller + `settings/smtp`)
- `amb_site_settings` → 법인별 설정 (`entId` FK + SiteSettingsService/Controller)
- `hr_entities.ent_email_display_name/brand_color/logo_url` → 법인별 이메일 브랜딩 (이미 존재)
- `entity-settings/organization` 컨트롤러 → `@Auth() + @UseGuards(OwnEntityGuard)` 패턴 (MASTER 접근)

## 3. DB 설계

### 3.1 신규 테이블: `amb_email_templates`

```sql
CREATE TABLE amb_email_templates (
  emt_id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ent_id         UUID REFERENCES amb_hr_entities(ent_id) ON DELETE CASCADE,
  emt_code       VARCHAR(50) NOT NULL,   -- 'ACCOUNT_CREATED' | 'INVITATION'
  emt_subject    VARCHAR(500) NOT NULL,
  emt_body       TEXT NOT NULL,          -- HTML 템플릿 (변수: {{userName}} 등)
  emt_updated_at TIMESTAMPTZ DEFAULT now(),
  emt_updated_by UUID,
  UNIQUE (ent_id, emt_code)
);

-- ADMIN용 전역 템플릿: ent_id = NULL, emt_code = 'ACCOUNT_CREATED'
-- 법인별 초대 템플릿: ent_id = 법인UUID, emt_code = 'INVITATION'
```

**네이밍 규칙 준수**: 테이블 prefix `amb_`, 컬럼 prefix `emt_` (email template)

### 3.2 지원 변수

| 변수 | 설명 | ACCOUNT_CREATED | INVITATION |
|------|------|:--------------:|:---------:|
| `{{userName}}` | 수신자 이름 | ✅ | ✅ |
| `{{userEmail}}` | 수신자 이메일 | ✅ | - |
| `{{tempPassword}}` | 임시 비밀번호 | ✅ | - |
| `{{loginUrl}}` | 로그인 URL | ✅ | - |
| `{{entityName}}` | 법인명 | ✅ | ✅ |
| `{{role}}` | 사용자 역할 | ✅ | ✅ |
| `{{inviterName}}` | 초대자 이름 | - | ✅ |
| `{{department}}` | 부서 | - | ✅ |
| `{{inviteLink}}` | 초대 수락 링크 | - | ✅ |

### 4.2 EmailTemplateEntity

```typescript
// apps/api/src/domain/settings/entity/email-template.entity.ts
@Entity('amb_email_templates')
export class EmailTemplateEntity {
  @PrimaryGeneratedColumn('uuid', { name: 'emt_id' })
  emtId: string;

  @Column({ name: 'ent_id', type: 'uuid', nullable: true })
  entId: string | null;  // null = 전역 (ADMIN용)

  @Column({ name: 'emt_code', length: 50 })
  emtCode: string;  // 'ACCOUNT_CREATED' | 'INVITATION'

  @Column({ name: 'emt_subject', length: 500 })
  emtSubject: string;

  @Column({ name: 'emt_body', type: 'text' })
  emtBody: string;

  @UpdateDateColumn({ name: 'emt_updated_at' })
  emtUpdatedAt: Date;

  @Column({ name: 'emt_updated_by', type: 'uuid', nullable: true })
  emtUpdatedBy: string | null;
}
```

### 4.3 EmailTemplateService

```typescript
// 주요 메서드
getTemplate(code: string, entId?: string | null): Promise<EmailTemplateEntity | null>
upsertTemplate(code: string, dto, userId, entId?): Promise<EmailTemplateEntity>
deleteTemplate(code: string, entId?): Promise<void>
renderTemplate(template: string, variables: Record<string, string>): string  // {{변수}} 치환
getDefaultTemplate(code: string): { subject: string; body: string }  // 하드코딩 기본값
```

### 4.4 ADMIN용 컨트롤러 (`settings/email-templates`)

```
GET    /api/v1/settings/email-templates/:code     → 전역 템플릿 조회
PUT    /api/v1/settings/email-templates/:code     → 전역 템플릿 저장/수정
DELETE /api/v1/settings/email-templates/:code     → 전역 템플릿 리셋 (기본값으로)
GET    /api/v1/settings/email-templates/:code/preview  → 샘플 데이터로 렌더링 미리보기
```
- 가드: `AdminGuard` (ADMIN_LEVEL 전용)
- 대상 템플릿 코드: `ACCOUNT_CREATED`

### 4.5 MASTER용 컨트롤러 (`entity-settings/email-templates`)

```
GET    /api/v1/entity-settings/email-templates/:code       → 법인별 템플릿 조회
PUT    /api/v1/entity-settings/email-templates/:code       → 법인별 템플릿 저장/수정
DELETE /api/v1/entity-settings/email-templates/:code       → 법인별 템플릿 리셋
GET    /api/v1/entity-settings/email-templates/:code/preview  → 미리보기
```
- 가드: `@Auth() + @UseGuards(OwnEntityGuard)` (기존 entity-settings 패턴)
- 대상 템플릿 코드: `INVITATION`
- `query.entity_id` + `resolveEntityId()` 패턴으로 법인 ID 처리

### 4.6 MailService 수정

`sendAccountCreatedEmail()` 신규 메서드 추가:
```typescript
async sendAccountCreatedEmail(params: {
  to: string;
  userName: string;
  userEmail: string;
  tempPassword: string;
  entityName: string;
  role: string;
  customTemplate?: { subject: string; body: string } | null;
}): Promise<boolean>
```
- `FRONTEND_URL` 환경변수로 `loginUrl` 조합: `${frontendUrl}/login`
- DB 템플릿이 있으면 사용, 없으면 하드코딩 기본값 사용
- 기본 템플릿은 기존 `sendRawEmail` HTML을 개선 (로그인 URL 버튼 포함, styled)

`sendInvitationEmail()` 수정: 법인별 DB 템플릿 지원 파라미터 추가
- 기존 시그니처 유지 + `customTemplate?: { subject, body }` 추가

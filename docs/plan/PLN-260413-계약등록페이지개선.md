### Phase 2: 결제 스케줄 — DB + Backend

**목표**: 결제 유형/스케줄 저장 인프라 구축

#### DB 변경 (Migration SQL)

```sql
-- 1. 기존 테이블에 결제 유형 + 정기 청구 금액 추가
ALTER TABLE amb_bil_contracts
  ADD COLUMN ctr_payment_type VARCHAR(20) DEFAULT NULL,
  ADD COLUMN ctr_billing_amount DECIMAL(15,2) DEFAULT NULL;

-- 2. 비정기 결제 스케줄 테이블 신규 생성
CREATE TABLE amb_bil_payment_schedules (
  pms_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ctr_id UUID NOT NULL REFERENCES amb_bil_contracts(ctr_id) ON DELETE CASCADE,
  pms_seq INT NOT NULL,
  pms_billing_date DATE NOT NULL,
  pms_billing_period VARCHAR(20),
  pms_amount DECIMAL(15,2) NOT NULL DEFAULT 0,
  pms_status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  pms_created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  pms_updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_pms_ctr_id ON amb_bil_payment_schedules(ctr_id);
```

#### 구현 상세

**PaymentScheduleEntity**:
```typescript
@Entity('amb_bil_payment_schedules')
export class PaymentScheduleEntity {
  @PrimaryGeneratedColumn('uuid', { name: 'pms_id' })
  pmsId: string;

  @Column({ name: 'ctr_id', type: 'uuid' })
  ctrId: string;

  @Column({ name: 'pms_seq', type: 'int' })
  pmsSeq: number;

  @Column({ name: 'pms_billing_date', type: 'date' })
  pmsBillingDate: string;

  @Column({ name: 'pms_billing_period', length: 20, nullable: true })
  pmsBillingPeriod: string;

  @Column({ name: 'pms_amount', type: 'decimal', precision: 15, scale: 2, default: 0 })
  pmsAmount: number;

  @Column({ name: 'pms_status', length: 20, default: 'PENDING' })
  pmsStatus: string;

  @ManyToOne(() => ContractEntity, (c) => c.paymentSchedules)
  @JoinColumn({ name: 'ctr_id' })
  contract: ContractEntity;
}
```

**ContractEntity 추가**:
```typescript
@Column({ name: 'ctr_payment_type', length: 20, nullable: true })
ctrPaymentType: string;

@Column({ name: 'ctr_billing_amount', type: 'decimal', precision: 15, scale: 2, nullable: true })
ctrBillingAmount: number;

@OneToMany(() => PaymentScheduleEntity, (s) => s.contract, { cascade: true })
paymentSchedules: PaymentScheduleEntity[];
```

**서비스 로직**: milestone 처리 패턴 동일 적용
- create: 스케줄 배열 → `scheduleRepo.create()` + `scheduleRepo.save()`
- update: `scheduleRepo.delete({ ctrId })` → 재생성 (전체 교체)

#### 사이드 임팩트
- 기존 `ctr_billing_day`, `ctr_billing_period` 필드는 유지 (삭제하지 않음)
- `payment_type=null`인 기존 계약은 기존 로직 그대로 동작
- `payment_type='REGULAR'`인 경우: 기존 `billing_day`/`billing_period` 필드 활용 + `billing_amount` 추가
- `payment_type='IRREGULAR'`인 경우: `payment_schedules` 테이블 활용
- `findAll`, `findById` 쿼리에 `paymentSchedules` relation 추가 필요

#### 구현 상세

```typescript
// 신규 등록 시 임시 파일 보관
const [pendingFiles, setPendingFiles] = useState<{ docType: string; file: File }[]>([]);

// handleSubmit 수정
const handleSubmit = async () => {
  // ... 기존 payload 구성
  if (isNew) {
    const created = await createMutation.mutateAsync(payload);
    // 파일 업로드 (계약 생성 후)
    if (pendingFiles.length > 0) {
      for (const pf of pendingFiles) {
        await uploadDocument({ refType: 'CONTRACT', refId: created.contractId, docType: pf.docType, file: pf.file });
      }
    }
    navigate(`/billing/contracts/${created.contractId}`);  // 수정 페이지로 이동
  } else {
    await updateMutation.mutateAsync({ id: id!, data: payload });
    navigate('/billing/contracts');
  }
};
```

**파일 선택 UI** (신규 등록 시):
- 간소화된 파일 선택 영역 (Full DocumentManager 대신)
- 문서 타입 선택 + 파일 선택 + 파일 목록 표시
- 저장 전까지 로컬 상태에만 보관

**수정 시**: 기존 DocumentManager 그대로 사용 (변경 없음)

#### 사이드 임팩트
- `createContract` API 반환값에서 `contractId` 필요 → 현재 `BilContractResponse` 반환하므로 OK
- 파일 업로드 실패 시 계약은 이미 생성됨 → 에러 토스트 표시 + 수정 페이지에서 재업로드 안내

### 스키마 변경

```sql
-- amb_bil_contracts: 2개 컬럼 추가

-- amb_bil_payment_schedules: 신규 테이블
CREATE TABLE amb_bil_payment_schedules (

  ctr_id UUID NOT NULL REFERENCES amb_bil_contracts(ctr_id) ON DELETE CASCADE,
  pms_seq INT NOT NULL,
  pms_billing_date DATE NOT NULL,
  pms_billing_period VARCHAR(20),
  pms_amount DECIMAL(15,2) NOT NULL DEFAULT 0,
  pms_status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  pms_created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  pms_updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_pms_ctr_id ON amb_bil_payment_schedules(ctr_id);
```

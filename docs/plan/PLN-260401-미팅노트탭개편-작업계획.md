### 1.2 Daily Note 현황

- `amb_meeting_notes.mtn_type` = `'MEMO'`로 저장됨
- 제목 패턴: `📅 Daily Note YYYY-MM-DD`
- `DAILY_NOTE` 타입으로 전환 시 기존 데이터 UPDATE 필요
- DB 컬럼 타입 `VARCHAR(20)`이므로 스키마 변경 없음

### 1.3 사이드 임팩트 사전 분석

| 영향 대상 | 영향 내용 | 대응 |
|-----------|----------|------|
| `daily-note.service.ts` | `type: 'MEMO'` → `type: 'DAILY_NOTE'` | 코드 수정 |
| 기존 Daily Note 데이터 | MEMO 타입 → DAILY_NOTE 타입 | UPDATE SQL |
| `NoteSearch` (Cmd+K 검색) | 타입 필터 없이 전체 검색 → Daily Note도 검색됨 | 현재 그대로 유지 |
| `MeetingNoteFormModal` | DAILY_NOTE 타입 선택 불가해야 함 (자동 생성 전용) | type 선택 UI에서 제외 |
| `kms` 자동 태깅/이벤트 | `MeetingNoteCreated` 이벤트 → KMS 태깅 로직 | `DAILY_NOTE`는 제외 고려 |
| 관계 그래프 (`NoteGraphPage`) | 타입 기반 표시 → `DAILY_NOTE` 노드 추가 | 그대로 표시 (변경 없음) |

### Phase 2: 데이터 마이그레이션 SQL 작성

파일: `sql/migration_daily_note_type.sql`

```sql
-- Daily Note 타입 마이그레이션
-- 제목 패턴 '📅 Daily Note%'인 MEMO 타입 노트를 DAILY_NOTE로 변경
UPDATE amb_meeting_notes
SET mtn_type = 'DAILY_NOTE',
    mtn_updated_at = NOW()
WHERE mtn_type = 'MEMO'
  AND mtn_title LIKE '📅 Daily Note%'
  AND mtn_deleted_at IS NULL;
```

### 5.1 Daily Note 타입 변경 영향

| 영향 대상 | 설명 | 대응 방안 |
|-----------|------|---------|
| **기존 Daily Note 노트** | MEMO 타입으로 Meeting Note 목록에 보임 | 마이그레이션 SQL 적용 후 DAILY_NOTE로 분류됨 |

## 6. DB 마이그레이션

### 6.1 대상 환경
- **로컬**: TypeORM synchronize 없음 (데이터만 변경)
- **스테이징**: `docker exec amb-postgres-staging psql` 로 직접 실행 필요
- **프로덕션**: 배포 전 수동 실행 필요

### 6.2 마이그레이션 SQL (`sql/migration_daily_note_type.sql`)

```sql
-- Meeting Notes Daily Note 타입 마이그레이션
-- 목적: MEMO 타입 중 '📅 Daily Note%' 제목 패턴인 노트를 DAILY_NOTE 타입으로 변경
-- 실행 환경: 스테이징, 프로덕션

BEGIN;

-- 변경 대상 확인
SELECT COUNT(*) as target_count
FROM amb_meeting_notes
WHERE mtn_type = 'MEMO'
  AND mtn_title LIKE '📅 Daily Note%'
  AND mtn_deleted_at IS NULL;

-- 타입 변경
UPDATE amb_meeting_notes
SET mtn_type = 'DAILY_NOTE',
    mtn_updated_at = NOW()
WHERE mtn_type = 'MEMO'
  AND mtn_title LIKE '📅 Daily Note%'
  AND mtn_deleted_at IS NULL;

-- 변경 결과 확인
SELECT mtn_type, COUNT(*) as count
FROM amb_meeting_notes
WHERE mtn_deleted_at IS NULL
GROUP BY mtn_type
ORDER BY mtn_type;

COMMIT;
```

### 6.3 롤백 SQL

```sql
-- 롤백: DAILY_NOTE 타입을 다시 MEMO로 변경
UPDATE amb_meeting_notes
SET mtn_type = 'MEMO',
    mtn_updated_at = NOW()
WHERE mtn_type = 'DAILY_NOTE'
  AND mtn_deleted_at IS NULL;
```

```
│  rounded rounded                     rounded rounded-full      │
│  hover:  Calendar icon + text        hover:  text-indigo-600   │
│  bg-gray                             bg-gray                   │
│  -100                                -100                      │
└─────────────────────────────────────────────────────────────────┘
```

**상세 스타일링:**

```tsx
{/* DateNavigator - 탭 바와 보드 사이 */}
<div className="mb-4 flex items-center justify-center gap-3">
  {/* 이전 날짜 */}
  <button
    onClick={goToPrevDay}
    className="rounded-lg p-2 text-gray-500 hover:bg-gray-100 hover:text-gray-700"
    title={t('todos:datePicker.prevDay')}
  >
    <ChevronLeft className="h-5 w-5" />
  </button>

  {/* 날짜 표시 (클릭 시 date input 열기) */}
  <button
    onClick={openDatePicker}
    className="flex items-center gap-2 rounded-lg px-4 py-2 text-lg font-semibold text-gray-900 hover:bg-gray-50"
  >
    <Calendar className="h-5 w-5 text-gray-400" />
    <span>{formatDate(selectedDate)}</span>  {/* 2026-03-01 (토) */}
  </button>

  {/* 다음 날짜 */}
  <button
    onClick={goToNextDay}
    className="rounded-lg p-2 text-gray-500 hover:bg-gray-100 hover:text-gray-700"
    title={t('todos:datePicker.nextDay')}
  >
    <ChevronRight className="h-5 w-5" />
  </button>

  {/* 오늘 버튼 - 선택일이 오늘이 아닐 때만 표시 */}
  {!isToday && (
    <button
      onClick={goToToday}
      className="rounded-full bg-indigo-50 px-3 py-1 text-sm font-medium text-indigo-600 hover:bg-indigo-100"
    >
      {t('todos:datePicker.today')}
    </button>
  )}
</div>
```

**날짜 표시 포맷:**
- 한국어: `2026-03-01 (토)`
- 영어: `Mar 01, 2026 (Sat)`
- 베트남어: `01/03/2026 (Th 7)`

**숨겨진 date input:**
```tsx
<input
  ref={dateInputRef}
  type="date"
  className="absolute opacity-0 h-0 w-0"
  value={selectedDate}
  onChange={(e) => setSelectedDate(e.target.value)}
/>
```

**Todo 컬럼 코드:**

```tsx
{/* Column 1: Todo (할 일 + 기한초과) */}
<div className="rounded-xl border border-gray-200 bg-gray-50 p-4">
  <h2 className="mb-3 flex items-center gap-2 text-sm font-semibold tracking-wider text-gray-600">
    <span className="inline-block h-2.5 w-2.5 rounded-full bg-gray-400" />
    {t('todos:columns.todo')} ({todoItems.length + overdueItems.length})
  </h2>

  {/* 기한초과 섹션 */}
  {overdueItems.length > 0 && (
    <div className="mb-3">
      <div className="mb-2 flex items-center gap-1.5">
        <AlertTriangle className="h-3.5 w-3.5 text-red-500" />
        <span className="text-xs font-medium text-red-600">
          {t('todos:columns.overdue')} ({overdueItems.length})
        </span>
      </div>
      <div className="space-y-2">
        {overdueItems.map((todo) => (
          <TodoItem key={todo.todoId} todo={todo} isOverdue={true} ... />
        ))}
      </div>
    </div>
  )}

  {/* 구분선 (기한초과 있을 때) */}
  {overdueItems.length > 0 && todoItems.length > 0 && (
    <div className="my-3 border-t border-gray-200" />
  )}

  {/* 일반 할일 */}
  <div className="space-y-2">
    {todoItems.map((todo) => (
      <TodoItem key={todo.todoId} todo={todo} isOverdue={false} ... />
    ))}
  </div>

  {/* 빈 상태 */}
  {overdueItems.length === 0 && todoItems.length === 0 && (
    <div className="rounded-lg border border-dashed border-gray-300 p-6 text-center text-xs text-gray-400">
      {t('todos:noTodos')}
    </div>
  )}
</div>
```

## 5. 데이터 분류 로직

### 5.1 useMemo 분류 함수

```typescript
const { todoItems, overdueItems, inProgressItems, doneItems, scheduledItems } = useMemo(() => {
  const todo: TodoResponse[] = [];
  const overdue: TodoResponse[] = [];
  const inProgress: TodoResponse[] = [];
  const done: TodoResponse[] = [];
  const scheduled: TodoResponse[] = [];

  for (const item of todos) {
    if (item.status === 'COMPLETED') {
      // Done 컬럼: completedAt 날짜가 선택일과 동일
      const completedDate = item.completedAt
        ? new Date(item.completedAt).toISOString().split('T')[0]
        : null;
      if (completedDate === selectedDate) {
        done.push(item);
      }
      // 선택일과 다른 완료 건은 표시하지 않음
    } else if (item.dueDate < selectedDate) {
      // Overdue: 예정일 < 선택일 + 미완료
      overdue.push(item);
    } else if (item.dueDate === selectedDate) {
      // 예정일 == 선택일
      if (item.status === 'IN_PROGRESS') {
        inProgress.push(item);
      } else {
        todo.push(item);  // SCHEDULED 상태
      }
    } else {
      // 예정일 > 선택일 + 미완료
      scheduled.push(item);
    }
  }

  // Overdue는 오래된 순 (dueDate 오름차순)
  overdue.sort((a, b) => a.dueDate.localeCompare(b.dueDate));

  return { todoItems: todo, overdueItems: overdue, inProgressItems: inProgress, doneItems: done, scheduledItems: scheduled };
}, [todos, selectedDate]);
```

### 5.2 분류 규칙 요약표

| 조건 | 컬럼 | 배지/스타일 |
|------|------|-----------|
| `status === 'COMPLETED' && completedAt 날짜 === 선택일` | Done | 취소선, opacity-60 |
| `status === 'COMPLETED' && completedAt 날짜 ≠ 선택일` | **미표시** | - |
| `status ≠ 'COMPLETED' && dueDate < 선택일` | Todo (Overdue 섹션) | 🔴 빨간 테두리, Overdue 배지 |
| `status === 'SCHEDULED' && dueDate === 선택일` | Todo (일반 섹션) | 일반 스타일 |
| `status === 'IN_PROGRESS' && dueDate === 선택일` | In Progress | 일반 스타일 |
| `status ≠ 'COMPLETED' && dueDate > 선택일` | Scheduled | 일반 스타일 |

## 12. 인터랙션 플로우

### 12.1 날짜 변경

```
사용자 ◀ 클릭
  → selectedDate = selectedDate - 1일
  → useMemo 재실행 → 4컬럼 재분류
  → UI 즉시 갱신 (API 재호출 없음, 클라이언트 필터링)
```

### 12.2 상태 변경 (TodoItem 클릭)

```
사용자 ○ 아이콘 클릭 (SCHEDULED → IN_PROGRESS)
  → onStatusChange(todoId, 'IN_PROGRESS')
  → API PUT /todos/:id { status: 'IN_PROGRESS' }
  → React Query invalidation → todos 배열 갱신
  → useMemo 재실행 → 해당 카드가 Todo → In Progress 컬럼으로 이동
```

### 12.3 완료 처리

```
사용자 🕐 아이콘 클릭 (IN_PROGRESS → COMPLETED)
  → onStatusChange(todoId, 'COMPLETED')
  → API PUT /todos/:id { status: 'COMPLETED' }
  → 백엔드에서 completedAt = new Date() 설정
  → React Query invalidation → 갱신된 todo에 completedAt 포함
  → useMemo: completedAt 날짜 == selectedDate → Done 컬럼에 표시
```

## 13. 엣지 케이스 처리

| 케이스 | 처리 방법 |
|--------|----------|
| Overdue 건이 많을 경우 (10건 이상) | Overdue 섹션을 `max-h-60 overflow-y-auto`로 스크롤 처리 |
| completedAt이 null인 COMPLETED 건 | `updatedAt`을 폴백으로 사용: `completedDate = completedAt \|\| updatedAt` |
| 선택일이 미래인 경우 | Overdue 섹션에 표시될 건이 더 많아짐 (정상 동작) |
| 모든 컬럼이 비어있을 경우 | 각 컬럼에 개별 "항목 없음" 빈 상태 표시 |
| ScopedTodoList 검색 + 날짜 조합 | 먼저 API에서 검색 필터 적용한 결과를 받고, 프론트에서 날짜 기준 컬럼 분류 |

## Phase 1: 로비챗 모바일 반응형 레이아웃 (핵심)

### 1-1. AmoebaTalkPage.tsx — 모바일/데스크톱 조건부 레이아웃

**변경 내용**:
- 모바일(`md` 이하): `currentChannelId` 유무에 따라 채널 리스트 OR 채팅 뷰 단일 표시
- 데스크톱(`md` 이상): 기존 3단 레이아웃 유지

```
모바일:
  currentChannelId === null → <ChannelList className="w-full" />
  currentChannelId !== null → <ChannelHeader /> + <MessageList /> + <MessageInput />

데스크톱:
  <ChannelList w-72 /> + <ChatArea flex-1 /> + <MemberPanel w-64 />
```

**핵심 CSS 패턴**:
- 채널 사이드바: `hidden md:flex md:w-72` (모바일 숨김) + 조건부 `flex w-full md:hidden` (모바일 전체화면)
- 채팅 영역: `hidden md:flex` (모바일 숨김) + 조건부 `flex w-full md:hidden` (모바일 전체화면)

### 1-2. ChannelHeader.tsx — 모바일 뒤로 가기 버튼

**변경 내용**:
- `md:hidden` 뒤로 가기 버튼 추가
- 클릭 시 `setCurrentChannelId(null)` → 채널 리스트로 복귀

### 1-3. MemberPanel.tsx — 모바일 오버레이

**변경 내용**:
- 모바일: `fixed inset-0 z-40` 오버레이 + 우측에서 슬라이드 인
- 데스크톱: 기존 `w-64 border-l` 유지

### 1-4. 패딩/간격 반응형 조정

## Phase 3: 이슈 AI 검토 — 백엔드

### 3-1. AI 검토 엔드포인트

**파일**: `apps/api/src/domain/issues/controller/issue.controller.ts`

```
POST /api/v1/issues/:id/ai-review
- Auth: EntityGuard
- 권한: 누구나 (해당 법인 소속)
- Response: SSE 스트리밍
```

### 3-2. AI 검토 서비스 로직

**파일**: `apps/api/src/domain/issues/service/issue.service.ts`

**로직**:
1. 이슈 정보 조회 (제목, 설명, 타입, 심각도, 모듈, 기존 코멘트)
2. (선택) 유사 이슈 검색 — 같은 법인의 최근 이슈 중 제목 유사도 비교
3. Claude API 프롬프트 구성:
   - 시스템: "이슈 검토 전문가. 한국어로 응답."
   - 사용자: 이슈 정보 + 분석 요청 (요약, 심각도 평가, 영향 범위, 해결 방안, 승인 권고)
4. SSE 스트리밍으로 응답 전송
5. 완료 시 `iss_ai_analysis` 필드 업데이트

**프롬프트 템플릿**:
```
다음 이슈를 검토해주세요:

제목: {title}
유형: {type}
심각도: {severity}
설명: {description}
관련 모듈: {affected_modules}
보고자: {reporter_name}

다음 항목으로 분석해주세요:
1. **요약**: 핵심 내용 1-2문장
2. **심각도 평가**: 설정된 심각도가 적절한지
3. **영향 범위**: 영향받는 기능/모듈
4. **해결 방안**: 가능한 접근 방법
5. **승인 권고**: APPROVE / REJECT / NEED_MORE_INFO
```

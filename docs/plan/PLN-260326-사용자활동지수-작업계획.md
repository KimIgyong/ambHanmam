#### 1-2. 테이블 DDL

**amb_issue_ratings**
```sql
CREATE TABLE amb_issue_ratings (
  isr_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  iss_id UUID NOT NULL REFERENCES amb_issues(iss_id) ON DELETE CASCADE,
  usr_id UUID NOT NULL REFERENCES amb_users(usr_id) ON DELETE CASCADE,
  isr_rating SMALLINT NOT NULL CHECK (isr_rating >= 1 AND isr_rating <= 5),
  isr_created_at TIMESTAMPTZ DEFAULT now(),
  isr_updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (iss_id, usr_id)
);
CREATE INDEX idx_issue_ratings_iss ON amb_issue_ratings(iss_id);
```

**amb_todo_ratings**
```sql
CREATE TABLE amb_todo_ratings (
  tdr_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  tdr_rating SMALLINT NOT NULL CHECK (tdr_rating >= 1 AND tdr_rating <= 5),
  tdr_created_at TIMESTAMPTZ DEFAULT now(),
  tdr_updated_at TIMESTAMPTZ DEFAULT now(),
```

#### 2-3. 핵심 서비스 로직

**ContentRatingService**:
- `upsertRating(targetType, targetId, userId)`: 자기평가 방지 체크 → upsert
- `deleteRating(targetType, targetId, userId)`: 평점 삭제
- `getRatingStats(targetType, targetIds)`: `{ avg, count }` 벌크 조회
- `getMyRatings(targetType, targetIds, userId)`: 내 평점 벌크 조회
- 자기평가 판별: 이슈→`iss_reporter_id`, 태스크→`usr_id`, 코멘트→`*_author_id`

**ActivitySnapshotService**:
- `@Cron('0 1 * * *')` (매일 01:00) — 전일 데이터 집계
- 각 법인(entity)별 > 각 사용자별:
  1. 콘텐츠 작성 건수 카운트 (created_at 기준)
  2. 가중치 조회 (법인별, 없으면 기본값)
  3. 활동 점수 = Σ(건수 × 가중치), 채팅은 cap 적용
  4. 받은 별점 합계/건수 조회
  5. 호응도 점수 = Σ(별점합 × 호응도가중치) + 이모지반응 처리
  6. 종합 점수 = 활동 + 호응도
  7. UPSERT into `amb_daily_activity_stats`

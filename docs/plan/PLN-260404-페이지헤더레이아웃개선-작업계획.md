## 구현 전략

### 핵심 설계 원칙

- **페이지 컴포넌트 변경 최소화**: Context + 공통 컴포넌트로 팁 버튼 기능을 캡슐화
- **점진적 적용 가능**: `PageTitle` 컴포넌트 도입 후 페이지별로 순차 교체
- **기존 팁 패널 동작 유지**: 슬라이드 패널 UX는 그대로 보존
- **라우팅 구조 유지**: HR/Billing URL 구조 변경 없이 레이아웃만 탭으로 전환

### 아키텍처

```
MenuGuideContext (MainLayout 제공)
  ├── 현재 경로 매칭 메뉴 정보 (menuCode, labelKey, icon)
  ├── 팁 데이터 (useMenuTipForPage)
  ├── canOpen: 항상 true (REQ-02)
  └── 팁 패널 열기 함수 (openTip)

PageTitle 컴포넌트 (페이지에서 사용)
  ├── props: children (제목 텍스트), className?
  ├── useMenuGuide() 훅으로 Context 소비
  └── 렌더: [아이콘] <h1>{children}</h1> [📖 팁버튼]
```

### Step 2: MenuGuideProvider + TipPanel [REQ-01, 02]

**파일**: `apps/web/src/global/contexts/MenuGuideContext.tsx` (Step 1과 동일 파일)

```tsx
export function MenuGuideProvider({ menus, children }) {
  const [open, setOpen] = useState(false);
  const matched = useMemo(...);
  const { data: tip } = useMenuTipForPage(matched?.menuCode);
  const tipContent = tip?.content || null;
  const tipTitle = tip?.title || t(matched?.labelKey, ...);

  // REQ-02: 콘텐츠 없어도 패널 열기 가능 (canOpen 제거)
  return (
    <MenuGuideContext.Provider value={{ ..., openTip: () => setOpen(true) }}>
      {children}
      {open && (
        <TipSlidePanel
          title={tipTitle}
          content={tipContent}   // null이면 패널 내부에서 "준비중" 렌더
          onClose={() => setOpen(false)}
        />
      )}
    </MenuGuideContext.Provider>
  );
}
```

// content가 null이면 준비중 메시지 표시

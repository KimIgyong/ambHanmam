## Step 1: Backend — Entity Settings 클라이언트 관리 API

### 목적
Entity Settings에서 현재 Entity 소속 고객사를 CRUD 관리하는 API 제공.

### 신규 파일

#### `apps/api/src/domain/entity-settings/controller/entity-client.controller.ts`

```
GET    /api/v1/entity-settings/clients
         → 현재 사용자 Entity의 고객사 목록 (cli_ent_id 필터)
         → 응답: 회사명, 상태, 담당자, 고객 사용자 수, 초대 수

GET    /api/v1/entity-settings/clients/:cliId
         → 고객사 상세 (사용자 목록 + 초대 목록 포함)

POST   /api/v1/entity-settings/clients
         → 고객사 생성 (cli_ent_id = 현재 Entity 자동 설정)
         → 필수: company_name
         → 선택: type, country, industry, note

PATCH  /api/v1/entity-settings/clients/:cliId
         → 고객사 정보 수정

DELETE /api/v1/entity-settings/clients/:cliId
         → 고객사 소프트 삭제

POST   /api/v1/entity-settings/clients/:cliId/invite
         → 고객 담당자 초대 (이메일 + 이름 + 역할)
         → 초대 토큰 생성 + 이메일 발송

GET    /api/v1/entity-settings/clients/:cliId/invitations
         → 해당 고객사 초대 목록

POST   /api/v1/entity-settings/clients/:cliId/invitations/:civId/resend
         → 초대 재발송

DELETE /api/v1/entity-settings/clients/:cliId/invitations/:civId
         → 초대 취소
```

#### `apps/api/src/domain/entity-settings/service/entity-client.service.ts`

- `findClients(entId, query)` — Entity 소속 고객사 목록
- `findClientDetail(entId, cliId)` — 상세 (사용자+초대 포함)
- `createClient(entId, dto)` — 생성 (`cli_ent_id` 자동 설정)
- `updateClient(entId, cliId, dto)` — 수정 (Entity 범위 검증)
- `deleteClient(entId, cliId)` — 소프트 삭제
- `inviteClient(entId, cliId, dto, invitedBy)` — 초대+메일 발송
- `getInvitations(cliId)` — 초대 목록
- `resendInvitation(civId)` — 재발송
- `cancelInvitation(civId)` — 취소

### 수정 파일

#### `apps/api/src/domain/entity-settings/entity-settings.module.ts`
- EntityClientController, EntityClientService 등록
- SvcClientEntity, ClientInvitationEntity 리포지토리 추가

### 보안 고려

- **역할 제한**: MASTER/ADMIN/MANAGER만 접근 (기존 Entity Settings 가드 활용)

**login() 메서드 변경:**
```diff
- async login(email: string, password: string) {
+ async login(entityCode: string, email: string, password: string) {
+   // 1. Entity Code로 Entity 조회
+   const entity = await this.entityRepository.findOne({
+     where: { entCode: entityCode, entStatus: 'ACTIVE' },
+   });
+   if (!entity) {
+     throw new UnauthorizedException('Invalid entity code');
+   }
+
+   // 2. CLIENT_LEVEL 사용자 조회 (이메일 + 클라이언트 Entity 일치 검증)
    const user = await this.userRepository.findOne({
      where: { usrEmail: email, usrLevelCode: 'CLIENT_LEVEL' },
-     relations: ['client'],
+     relations: ['client', 'client.hrEntity'],
    });
-
-   if (!user || !(await bcrypt.compare(password, user.usrPassword))) {
+
+   // 3. 사용자 존재 + 비밀번호 검증 + Entity 일치 검증
+   if (!user || !user.client || user.client.cliEntId !== entity.entId) {
+     throw new UnauthorizedException('Invalid credentials');
+   }
+   if (!(await bcrypt.compare(password, user.usrPassword))) {
```

**검증 로직 흐름:**
1. `entity_code` → `HrEntity` 조회 (entCode + ACTIVE)
2. `email` + `CLIENT_LEVEL` → 사용자 조회
3. `user.client.cliEntId === entity.entId` 일치 검증
4. `bcrypt.compare(password, usrPassword)` 비밀번호 검증
5. 성공 시 JWT 발급 + Entity 정보 포함 응답

- "Powered by {Entity Name}" 또는 Entity 브랜딩 영역

### USER_LEVEL LoginPage.tsx와의 차이점
| 항목 | USER_LEVEL | CLIENT_LEVEL (신규) |
|------|------------|------------------|
| Entity 검색 | `EntitySearchModal` 재사용 | 동일 (재사용) |
| 최근 Entity | localStorage 기반 | 동일 (재사용) |
| URL 파라미터 | `entityCode` + `entityName` | 동일 |
| Login API | `POST /api/v1/auth/login` | `POST /api/v1/client-auth/login` |
| 로그인 후 라우팅 | `/select-entity` | `/client` |
| 조직 찾기 메일 | 신청 가능 | 미제공 (초대 메일 내 Entity Code 안내) |

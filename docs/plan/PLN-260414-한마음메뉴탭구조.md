# PLN-260414 한마음 시스템 메뉴 탭 구조 작업 계획서

## 1. 시스템 개발 현황 분석

### 1.1 현재 프론트엔드 구조

| 항목 | 현황 |
|------|------|
| **레이아웃** | `MainLayout.tsx` — GNB(헤더) + LNB(사이드바 w-52) + Content(Outlet) |
| **라우터** | `router/index.tsx` — `createBrowserRouter`, 평탄한 route 구조 |
| **메뉴** | `useMyMenus()` → Work Tools/Modules 2분류, `menuConfig.ts`에 아이콘/색상 매핑 |
| **i18n** | 50 네임스페이스, 3언어(ko/en/vi), `i18n.ts`에 등록 |
| **공통 컴포넌트** | Button, Modal, Toast, Table 등 기존 UI 컴포넌트 |
| **도메인 구조** | `src/domain/{name}/pages/`, `src/domain/{name}/components/` |

### 1.2 영향 받는 파일

| 파일 | 변경 유형 |
|------|----------|
| `router/index.tsx` | 수정 — `/hm/*` 라우트 그룹 추가 |
| `i18n.ts` | 수정 — `hanmam` 네임스페이스 등록 |

## 4. 사이드 임팩트 분석

| 영역 | 영향 | 위험도 |
|------|------|--------|
| 기존 AMA 기능 | `/hm` prefix로 완전 격리 → 영향 없음 | 🟢 낮음 |
| 라우터 | `router/index.tsx`에 새 route 그룹 추가만. 기존 route 변경 없음 | 🟢 낮음 |
| i18n | `hanmam` 네임스페이스만 추가. 기존 50개 NS 영향 없음 | 🟢 낮음 |
| 번들 크기 | 41개 페이지 추가로 번들 증가. lazy import로 코드 분할 권장 | 🟡 중간 |
| 빌드 시간 | 55개 파일 추가로 빌드 시간 소폭 증가 | 🟢 낮음 |
| 인증/권한 | `AuthGuard`만 적용. 목업 단계에서 `MenuGuard` 불필요 | 🟢 낮음 |

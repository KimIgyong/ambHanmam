## Phase 0: DB 마이그레이션 [백엔드/DB]

### 0-1. 신규 테이블: `amb_menu_group_permissions`
메뉴-그룹 권한 매핑 테이블. `amb_groups`를 참조.

```sql
-- scripts/migration-group-permission.sql

-- 1. 메뉴-그룹 권한 테이블
CREATE TABLE IF NOT EXISTS amb_menu_group_permissions (
  mgp_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  mgp_menu_code VARCHAR(50) NOT NULL,
  grp_id UUID NOT NULL REFERENCES amb_groups(grp_id) ON DELETE CASCADE,
  mgp_accessible BOOLEAN NOT NULL DEFAULT true,
  mgp_created_at TIMESTAMP DEFAULT now(),
  mgp_updated_at TIMESTAMP DEFAULT now(),
  UNIQUE(mgp_menu_code, grp_id)
);

CREATE INDEX idx_mgp_menu_code ON amb_menu_group_permissions(mgp_menu_code);
CREATE INDEX idx_mgp_grp_id ON amb_menu_group_permissions(grp_id);
```

### 0-2. 멤버 편집용 + 초대 발송로그 컬럼 추가
```sql
-- 2. 직무 컬럼 추가
ALTER TABLE amb_users ADD COLUMN IF NOT EXISTS usr_job_title VARCHAR(100) DEFAULT NULL;

-- 3. 초대 메일 발송 로그 컬럼 추가
ALTER TABLE amb_invitations ADD COLUMN IF NOT EXISTS inv_last_sent_at TIMESTAMP DEFAULT NULL;
ALTER TABLE amb_invitations ADD COLUMN IF NOT EXISTS inv_send_count INTEGER DEFAULT 0;
```

## 사이드 임팩트 분석

| 영역 | 영향 | 대응 |
|------|------|------|
| 사이드바 메뉴 | getMyMenus() 4단계 추가 → 그룹 미배정 사용자 메뉴 누락 가능 | 그룹 설정 없으면 기존과 동일하게 통과 |
| ADMIN_GROUP | 변경 없음 | 그룹 필터 대상에서 제외 |
| 사용자별 개별 권한 | 변경 없음 | 개별 권한은 그룹 필터보다 우선 |
| 그룹 삭제 시 | CASCADE → `amb_menu_group_permissions` 자동 삭제 | ON DELETE CASCADE 설정됨 |
| 법인 관리 페이지 | 탭 추가 → 기존 법인 CRUD 영향 없음 | 독립 탭으로 분리 |
| 멤버 상세 | 읽기전용→편집 변경 → 기존 동작 유지 | PATCH API로 각 필드 독립 수정 |
| `amb_departments` | 부서 관리 전용, 권한과 무관 | 법인 관리에서 부서 탭으로 CRUD |
| 성능 | getMyMenus()에 그룹 조회 추가 | 인덱스 활용, 소규모 데이터 |
| RegisterPage 가입 후 | JWT 즉시 로그인 → 성공 안내 화면으로 변경 | PENDING 상태 사용자도 기존 login은 가능 (AuthGuard가 /pending으로 이동) |
| InvitationListTab | 테이블 열 2개 추가 | 기존 열 영향 없음, 반응형 고려 |
| MemberListTab | 상태 컬럼 + 승인/거부 버튼 추가 | 기존 역할변경 기능 유지 |
| 비밀번호 보기 토글 | 양쪽 페이지에 Eye 아이콘 추가 | lucide-react Eye/EyeOff 아이콘 사용 |

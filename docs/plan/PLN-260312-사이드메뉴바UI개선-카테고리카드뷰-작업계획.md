## 1. 시스템 현황 분석

### 1-1. 영향 받는 현재 구현

**MainLayout.tsx** (`apps/web/src/layouts/MainLayout.tsx`)
- `renderMenuItems(inDrawer)` 함수: 사이드바 전체 메뉴 렌더링 담당
- 카테고리 레이블: `<p>` 태그로 구현되어 클릭 불가
- `workTools`, `workModules`, `customApps` 데이터가 이미 분리되어 있어 재사용 가능
- 사이드바 전체 collapsed 상태: `useState(false)` 존재 — 카테고리별 상태는 별도 추가 필요

**라우터** (`apps/web/src/router/index.tsx`)
- MainLayout 하위 children으로 경로 등록
- `/menu/*` 경로군 없음 → 신규 추가 필요

**i18n** (`apps/web/src/locales/{ko,en,vi}/common.json`)
- `sidebar.workTools`, `sidebar.workModules`, `sidebar.customApps` 번역 키 존재
- 카드뷰 페이지용 추가 번역 키 필요

### 1-2. 재사용 가능한 기존 구현

| 훅/컴포넌트 | 재사용 방식 |
|------------|----------|
| `useMyMenus()` | 카드뷰 페이지에서 workTools/workModules 데이터 조회 |
| `useMyCustomApps()` | 카드뷰 페이지에서 customApps 데이터 조회 |
| `ICON_MAP`, `COLOR_MAP` | 카드뷰 페이지 아이콘/색상 렌더링 (MainLayout에서 추출 or 공통화) |
| `WORK_TOOL_CODES` | 카드뷰에서 workTools/workModules 분리 |

## 2. 구현 계획

### Phase 1: 공통 상수/유틸리티 추출 (선택적)

현재 `MainLayout.tsx`에 인라인 정의된 `ICON_MAP`, `COLOR_MAP`, `WORK_TOOL_CODES`를  
카드뷰 페이지에서도 사용하기 위해 공통 파일로 추출한다.

**파일**: `apps/web/src/domain/menu/constants/menuConfig.ts`
```ts
export const WORK_TOOL_CODES = [...];
export const ICON_MAP = {...};
export const COLOR_MAP = {...};
```

> ⚠️ MainLayout.tsx에서 import 변경 필요

### Phase 2: 카드뷰 페이지 컴포넌트 구현

**파일**: `apps/web/src/domain/menu/pages/MenuCategoryPage.tsx`

```tsx
type CategoryType = 'workTools' | 'workModules' | 'customApps';

interface Props {
  type: CategoryType;
}

export default function MenuCategoryPage({ type }: Props)
```

**구현 내용**:

1. **데이터 로딩**:
   - `workTools`, `workModules`: `useMyMenus()` + `WORK_TOOL_CODES` 필터링
   - `customApps`: `useMyCustomApps()`

2. **레이아웃 구조**:
```
MenuCategoryPage
├── 페이지 헤더
│   ├── 뒤로가기 버튼 (← 화살표 아이콘)
│   ├── 카테고리 제목 (Work Tools / Work Modules / Custom Apps)
│   └── 메뉴 개수 뱃지 (예: "8 menus")
└── 카드 그리드
    ├── 카드 (반복)
    │   ├── 아이콘 (colored)
    │   ├── 메뉴명
    │   └── (hover 시 강조 효과)
    └── 빈 상태 메시지 (메뉴 없을 때)
```

3. **카드 반응형 그리드**:
   - Mobile: `grid-cols-2`
   - Tablet (md): `grid-cols-3`
   - Desktop (lg): `grid-cols-4`
   - Wide (xl): `grid-cols-5`

4. **카드 디자인**:
```
┌─────────────────────┐
│  🔷                 │   ← 아이콘 (크게, 컬러)
│                     │
│  Todo               │   ← 메뉴명
└─────────────────────┘
```
- 기본: 흰 배경, 연한 border
- hover: shadow-md, indigo border
- active(현재 경로): indigo-50 배경, indigo border

### Phase 3: MainLayout 사이드바 카테고리 헤더 개선

**파일**: `apps/web/src/layouts/MainLayout.tsx`

#### 3-1. 상태 추가
```tsx
const [workToolsExpanded, setWorkToolsExpanded] = useState(true);
const [workModulesExpanded, setWorkModulesExpanded] = useState(true);
const [customAppsExpanded, setCustomAppsExpanded] = useState(true);
```

#### 3-2. `renderMenuItems` 함수 내 카테고리 헤더 교체

**Before** (현재):
```tsx
<p className="px-3 pb-1 pt-3 text-xs font-semibold uppercase tracking-wider text-gray-400">
  {t('common:sidebar.workTools')}
</p>
```

**After** (변경):
```tsx
<div className="flex items-center justify-between px-3 pb-1 pt-3">
  <button
    onClick={() => navigate('/menu/work-tools')}
    className="text-xs font-semibold uppercase tracking-wider text-gray-400 hover:text-gray-600 transition-colors"
  >
    {t('common:sidebar.workTools')}
  </button>
  <button
    onClick={() => setWorkToolsExpanded(prev => !prev)}
    className="text-gray-400 hover:text-gray-600 transition-colors p-0.5 rounded"
  >
    {workToolsExpanded ? <ChevronUp className="h-3.5 w-3.5" /> : <ChevronDown className="h-3.5 w-3.5" />}
  </button>
</div>
{workToolsExpanded && workTools.map((item) => renderMenuItem(item, inDrawer))}
```

#### 3-3. 아이콘 import 추가
- `ChevronUp`, `ChevronDown` (lucide-react)

|------|------|------|

│  📝 Meeting Notes                   │

│  ...                                │

[접힘 상태]
┌─────────────────────────────────────┐

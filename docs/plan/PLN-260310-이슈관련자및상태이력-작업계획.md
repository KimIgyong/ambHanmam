# 작업 계획서: 이슈 관련자(Participant) 및 상태 이력 개선

**작성일**: 2026-03-10
**요구사항 분석서**: `docs/analysis/REQ-이슈관련자및상태이력-20260310.md`

## 구현 단계

### Step 1: 관련자 엔티티 + DB 테이블
**수정 파일**: `apps/api/src/domain/issues/entity/issue-participant.entity.ts` (신규)

- `amb_issue_participants` 테이블 생성
  - `isp_id` (PK, UUID), `iss_id` (FK), `usr_id` (FK)
  - `isp_role` (varchar 20, default 'PARTICIPANT') — PARTICIPANT, FORMER_ASSIGNEE
  - `isp_created_at` (timestamp)
- 복합 유니크: `(iss_id, usr_id)`
- IssueModule에 엔티티 등록

### Step 2: 백엔드 API — 관련자 CRUD + 자동 전환
**수정 파일**:
- `apps/api/src/domain/issues/service/issue.service.ts`
  - `createIssue()`: `participant_ids` 처리
  - `updateIssue()`: `participant_ids` 처리 + 담당자 변경 시 이전 담당자 자동 관련자 전환
  - `getIssueById()`: 관련자 목록 포함
  - `getIssues()`: 관련자 정보 포함
  - `getMyIssues()`: scope 'my_involved'에 관련자 포함
  - 권한 체크: 관련자도 조회/코멘트/상태변경 가능
- `apps/api/src/domain/issues/controller/issue.controller.ts`
  - 기존 엔드포인트에 관련자 데이터 포함
- `apps/api/src/domain/issues/dto/create-issue.request.ts` — `participant_ids` 추가
- `apps/api/src/domain/issues/dto/update-issue.request.ts` — `participant_ids` 추가
- `apps/api/src/domain/issues/mapper/issue.mapper.ts` — participants 매핑
- `apps/api/src/domain/issues/issue.module.ts` — IssueParticipantEntity 등록

### Step 3: 공유 타입
**수정 파일**: `packages/types/src/domain.types.ts`
- `IssueResponse`에 `participants?: IssueParticipantResponse[]` 추가
- `IssueParticipantResponse` 인터페이스 추가

### Step 4: 프론트엔드 — 관련자 선택 UI + 상태 이력 개선
**수정 파일**:
- `apps/web/src/domain/issues/components/IssueFormModal.tsx` — 관련자 멀티셀렉터 추가
- `apps/web/src/domain/issues/components/IssueDetailModal.tsx`
  - 관련자 목록 표시
  - 상태 이력 기본 펼침(`showHistory` 초기값 true)
- `apps/web/src/domain/issues/service/issue.service.ts` — participant_ids 전달
- `apps/web/src/domain/issues/components/ParticipantSelector.tsx` (신규) — 멀티셀렉터 컴포넌트

### Step 5: 번역 키 추가
**수정 파일**: `locales/{en,ko,vi}/issues.json`

- `amb_issue_participants` 테이블 신규 → DB 마이그레이션 필요 (스테이징 수동 CREATE TABLE)
- 기존 IssueResponse에 optional 필드 추가 — 하위호환
- `scope: 'my_involved'` 쿼리 변경 — 관련자도 포함하도록 WHERE 조건 확장
- 알림 시스템: 관련자에게도 이슈 변경 알림 발송 (기존 ASSIGNEE_EVENT 확장)

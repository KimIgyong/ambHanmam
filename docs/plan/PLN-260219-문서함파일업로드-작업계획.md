## STEP 1: 백엔드 - Drive 컨트롤러에 업로드 엔드포인트 추가

### 대상 파일
- `apps/api/src/domain/drive/controller/drive.controller.ts`
- `apps/api/src/domain/drive/service/drive.service.ts`

### 작업 내용
1. `DriveController`에 `POST /drive/upload` 엔드포인트 추가
   - `@UseInterceptors(FileInterceptor('file'))` 사용
   - `@UploadedFile()` 데코레이터로 파일 수신
   - `@Body('folder_id')` 로 대상 폴더 ID 수신
   - Multer 옵션: 50MB 제한
2. `DriveService`에 `uploadFile(folderId, file, userId)` 메서드 추가
   - `GoogleDriveService.uploadFile()` 호출
   - 업로드 결과 반환 (fileId, name, webViewLink)

### 사이드 임팩트
- 없음. 기존 읽기 전용 엔드포인트에 영향 없음
- `GoogleDriveService.uploadFile()`은 이미 구현되어 있어 인프라 변경 없음

## STEP 2: 프론트엔드 - API 서비스 및 React Query 훅

### 대상 파일

- `apps/web/src/domain/documents/hooks/useDrive.ts`

### 작업 내용

2. `useDrive.ts`에 `useUploadFile()` mutation 추가
   - 성공 시 파일 목록 캐시 무효화 (`driveKeys.files`)

### 사이드 임팩트

## STEP 3: 프론트엔드 - DocumentsPage 업로드 UI

### 대상 파일
- `apps/web/src/domain/documents/pages/DocumentsPage.tsx`

### 작업 내용
1. 업로드 버튼 추가 (폴더 내부 탐색 중일 때만 표시)
   - 브레드크럼 옆 또는 헤더 영역
   - `Upload` 아이콘 + 텍스트
2. 숨김 `<input type="file" multiple>` 추가
3. 파일 선택 시 순차 업로드 처리
4. 업로드 진행 상태 표시 (토스트 또는 인라인)
   - 파일명, 진행 상태 (대기/업로드 중/완료/실패)
5. 전체 업로드 완료 시 파일 목록 자동 갱신

### 사이드 임팩트
- DocumentsPage UI 변경 (업로드 버튼 추가)
- 기존 파일 탐색/검색 기능에는 영향 없음

## STEP 4: i18n 번역 파일 업데이트

- `apps/web/src/locales/ko/documents.json`
- `apps/web/src/locales/en/documents.json`
- `apps/web/src/locales/vi/documents.json`

### 작업 내용
업로드 관련 번역 키 추가:
- `upload`: 업로드 버튼 텍스트
- `uploadProgress`: 업로드 진행 중 메시지
- `uploadSuccess`: 업로드 완료 메시지
- `uploadFailed`: 업로드 실패 메시지
- `uploadSizeLimit`: 파일 크기 제한 안내
- `uploading`: 업로드 중 상태

## 2. 단계별 구현 계획

### Phase 1: 스타일 기반 준비
**목표**: 새 디자인에 필요한 폰트·CSS 변수·애니메이션을 globals.css에 추가

**작업 내역**:
1. `globals.css`에 Google Fonts import 추가 (Sora, DM Sans, JetBrains Mono)
2. CSS 커스텀 변수 정의 (새 디자인의 Navy/Blue/Slate 컬러)
3. 슬라이더 트랜지션, fade-up 애니메이션, typing dots 등 keyframe 추가
4. 유틸리티 클래스 추가 (`.sora`, `.mono`, `.dm-sans` 등)

**사이드 임팩트**:
- 폰트 추가는 전역이지만, 클래스로 적용하므로 기존 페이지에 영향 없음
- 기존 `amb-*` 애니메이션과 충돌 없도록 네이밍 구분

### Phase 2: LandingPage 번역 데이터 구현
**목표**: HTML `T` 객체의 ko/en/vi 번역 데이터를 TypeScript `translations` 객체로 변환

**작업 내역**:
1. 기존 `translations` 객체 전체 교체
2. HTML의 ~200개 키를 React 컴포넌트에서 사용하기 편한 구조로 재구성
3. 배열 데이터 (tags, footer links) 포함
4. `I18nText` 타입 유지

### Phase 3: LandingPage 섹션 구현 — Hero (Pain Slider)
**목표**: Navy 배경의 Hero + 4장 카드 자동 슬라이더

**작업 내역**:
1. Hero 헤더 (섹션 라벨 + 타이틀 + 서브)
2. Slider 구현:
   - `useState`로 현재 슬라이드 인덱스 관리
   - `useEffect`로 4.2초 자동 재생 (hover 시 정지)
   - `useRef`로 viewport 크기 감지 (1/2/3 카드 반응형)
   - prev/next 버튼 + dot 네비게이션
3. Pain Card 컴포넌트 (번호, 아이콘, 제목, 설명, 해결 방안)

### Phase 4: LandingPage 섹션 구현 — Features
**목표**: 4개 feature 블록 (좌우 교차 레이아웃 + 시각적 컴포넌트)

**작업 내역**:
1. Feature 블록 공통 구조 (2열 그리드, flip 모드)
2. **F1 Mission Flow**: 4단계 수직 흐름 (아이콘 + 연결선 + 배지)
3. **F2 Chat Bubbles**: 좌/우 메시지 버블 + 번역 배지
4. **F3 Report Card**: 헤더(3 dots) + 본문(3 rows) + AI 타이핑 애니메이션
5. **F4 App Grid**: 2×2 앱 카드 (아이콘 + 이름 + 사용 횟수)
6. Feature tags (동적 렌더링)

### Phase 5: LandingPage 섹션 구현 — Cycle, Stats, Tech, CTA
**목표**: 나머지 섹션 구현

**작업 내역**:
1. **Cycle**: 5단계 순환 노드 (아이콘 + 라벨 + 화살표) + 결과 카드
2. **Stats**: 5개 통계 카드 (숫자/아이콘 + 라벨)
3. **Tech**: 4개 기술 카드 (Frontend, Backend, Database, AI)
4. **CTA**: Navy 배경 + 데모 신청/문의 버튼
5. Fade-Up 애니메이션 적용 (기존 `Reveal` 컴포넌트 패턴 재사용)

### Phase 6: 반응형 + 최종 점검
**목표**: 반응형 레이아웃 점검 및 빌드 검증

**작업 내역**:
1. 960px 이하: 슬라이더 2장, Feature 1열, nav links 숨김
2. 640px 이하: 슬라이더 1장, Cycle 세로 정렬, App Grid 1열
3. CmsHeader와의 조화 확인 (투명 배경 모드 활용 여부)
4. **빌드 검증**: `npm run -w @amb/portal-web build`

## 6. 구현 전략 요약

```
globals.css 폰트/애니메이션 추가
  ↓
LandingPage.tsx 전면 재작성
  - 번역 데이터 (translations 객체)
  - Hero/Pain Slider
  - Features (4블록 + 시각 컴포넌트)
  - Cycle, Stats, Tech, CTA
  - Reveal 애니메이션 + 반응형
  ↓
빌드 검증
```

**핵심 원칙**:
- HTML 디자인의 시각적 충실도를 최우선으로 구현
- lucide-react 아이콘 활용 가능한 곳은 활용, 맞는 아이콘 없으면 인라인 SVG
- 기존 PublicLayout (CmsHeader + CmsFooter) 구조 유지
- 단일 파일 패턴 유지 (LandingPage.tsx 내에 모든 것 포함)

# PLAN-커스텀앱-앱스토어-메뉴분리-작업계획-20260330

## 1. 시스템 개발 현황 분석

### 1.1 현행 아키텍처

현재 커스텀앱과 앱스토어가 `EntityCustomAppsTabPage`에 3탭으로 통합되어 있으며, "앱스토어에서 가져오기" 기능이 앱스토어앱을 커스텀앱 테이블에 복제하는 방식으로 도메인 경계를 위반하고 있다.

**현행 프론트엔드 구조**:
```
EntityCustomAppsTabPage.tsx (3탭 통합)
├── EntityCustomAppsPage.tsx (커스텀앱 CRUD + Import 버튼)
│   └── ImportFromStoreModal.tsx (파트너앱 → 커스텀앱 복제)
├── iframe: apps.amoeba.site (공인 앱스토어)
└── iframe: stg-apps.amoeba.site (테스트 앱스토어)
```

**현행 백엔드 구조**:
```
entity-custom-app.controller.ts
├── GET /          → findAll (관리 목록)
├── GET /my        → findMyApps (사이드바)
├── POST /         → create (등록)
├── PATCH /:id     → update (수정)
├── DELETE /:id    → remove (삭제)
├── POST /:id/token  → generateAppToken (토큰)
├── POST /:id/health → healthCheck (연결 테스트)
├── GET /importable   → getImportableApps ← 제거 대상
└── POST /import/:papId → importFromStore ← 제거 대상
```

**Service 의존성**:
```
EntityCustomAppService
├── customAppRepo (EntityCustomAppEntity)
├── partnerAppRepo (PartnerAppEntity)     ← import 전용, 제거 가능
├── installRepo (PartnerAppInstallEntity)  ← import 전용, 제거 가능
└── jwtService (JwtService)
```

### 1.2 목표 아키텍처

```
[커스텀앱] — /entity-settings/custom-apps
```

#### 2.1.2 AppStore 프록시 서비스 신설

`apps/api/src/domain/entity-settings/service/app-store.service.ts` (신규):

```typescript
@Injectable()
export class AppStoreService {
  constructor(private configService: ConfigService) {}

  async getSubscriptions(entityId: string): Promise<AppStoreSubscription[]> {
    const baseUrl = this.configService.get('APP_STORE_API_URL');
    const url = `${baseUrl}/api/v1/platform/subscriptions/entity/${entityId}`;
    
    const response = await fetch(url, { 
      signal: AbortSignal.timeout(10000) 
    });
    
    if (!response.ok) {
      throw new InternalServerErrorException('Failed to fetch app store subscriptions');
    }
    
    const { data } = await response.json();
    return data.apps;
  }
}
```

#### 2.1.3 AppStore 프록시 컨트롤러 신설

`apps/api/src/domain/entity-settings/controller/app-store.controller.ts` (신규):

```typescript
@Controller('entity-settings/app-store')
export class AppStoreController {
  @Get('subscriptions')
  @Auth()
  @UseGuards(OwnEntityGuard)
  async getSubscriptions(
    @Query('entity_id') queryEntityId: string | undefined,
    @CurrentUser() user: UserPayload,
  ) {
    const entityId = resolveEntityId(queryEntityId, user);
    return this.appStoreService.getSubscriptions(entityId);
  }
}
```

```typescript
// AppStore Subscription 타입
export interface AppStoreApp {
  appSlug: string;
  appName: string;
  appNameEn: string | null;
  appStatus: 'ACTIVE' | 'COMING_SOON';
  appIconUrl: string | null;
  subscription: {
    subId: string;
    status: 'PENDING' | 'ACTIVE' | 'SUSPENDED';
    requestedAt: string;
    approvedAt: string | null;
    expiresAt: string | null;
  } | null;

// API 호출
async getAppStoreSubscriptions(entityId?: string): Promise<AppStoreApp[]> {
  const params: Record<string, string> = {};
  if (entityId) params.entity_id = entityId;
  const { data } = await apiClient.get('/entity-settings/app-store/subscriptions', { params });
  return data;
}
```

#### 2.3.2 React Query 훅 추가

```typescript
export function useAppStoreSubscriptions(entityId?: string) {

    queryKey: ['entity-settings', 'app-store-subscriptions', entityId || 'own'],
    queryFn: () => entitySettingsService.getAppStoreSubscriptions(entityId),
    staleTime: 60 * 1000, // 1분 (외부 API이므로 약간 길게)

}
```

| 기존 Import된 커스텀앱 | `eca_source_pap_id` 값이 있는 앱들이 DB에 존재 | 그대로 유지, 커스텀앱으로 계속 취급 |
| ImportCustomAppRequest DTO | 더 이상 사용 안 됨 | 제거 |
| PartnerApp/Install 엔티티 | Service에서 직접 참조 제거 | 다른 서비스(EntityAppsService)에서 사용 시 Module 등록 유지 |
| 단위 테스트 | 20개 중 Import 관련 테스트 존재 | 해당 테스트 제거 |

### 4.3 새 앱스토어 페이지 영향

| 영향 범위 | 내용 | 대응 |
|-----------|------|------|
| 법인설정 메뉴 | 신규 메뉴 아이템 필요 | menuCode 등록 여부 확인 후 추가 |
| MenuGuard | 새 라우트에 MenuGuard 적용 필요 | 적절한 menuCode 배정 |
| 라우터 코드 스플리팅 | 새 페이지 lazy import 필요 | React.lazy 적용 |

### 4.4 사이드바 변경 영향 (Phase 4)

| 영향 범위 | 내용 | 대응 |
|-----------|------|------|
| MainLayout 렌더링 | 새 섹션 추가로 사이드바 길이 증가 | 스크롤 가능 영역 내에 배치 |
| 외부 API 호출 | 사이드바 로드마다 Subscription API 호출 | staleTime 길게 설정 (2~5분) |
| 앱스토어 앱 실행 방식 | 커스텀앱처럼 `/apps/:code`로 호스팅할 수 없음 (CustomAppHostPage는 내부 DB 조회) | 새탭으로 앱스토어 앱 URL 직접 오픈 (`APP_STORE_URL/{appSlug}/`) |

## 5. DB 마이그레이션

### 5.1 마이그레이션 필요 여부: 없음

현 Phase에서는 DB 스키마 변경이 없다.

- `eca_source_pap_id` 컬럼: 삭제하지 않음 (기존 데이터 보존)
- `eca_source_app_slug` 컬럼: 이번 작업에서는 추가하지 않음 (Import 기능 자체를 제거하므로 불필요)
- 앱스토어 구독 데이터: 외부 API에서 실시간 조회 (로컬 DB 저장 안 함)

> **참고**: 향후 앱스토어 앱을 사이드바에서 직접 실행(커스텀앱처럼 iframe 호스팅)하려면 별도 테이블 설계가 필요할 수 있으나, 현재 범위에서는 새탭 오픈으로 처리.

## 단계 1: DB 스키마 + Entity 변경

### 1.1 `amb_issues` 테이블에 `iss_ref_number` 컬럼 추가

```sql
ALTER TABLE amb_issues ADD COLUMN iss_ref_number VARCHAR(50) DEFAULT NULL;
CREATE INDEX idx_issues_ref_number ON amb_issues(iss_ref_number) WHERE iss_ref_number IS NOT NULL;
```

### 1.2 IssueEntity 수정

**파일**: `apps/api/src/domain/issues/entity/issue.entity.ts`

```typescript
@Column({ name: 'iss_ref_number', type: 'varchar', length: 50, nullable: true })
issRefNumber: string | null;
```

### 1.3 `iss_created_at` 컬럼 변경

현재 `@CreateDateColumn` → 일반 `@Column`으로 변경하여 Redmine `created_on` 값 직접 입력 가능하게:

```typescript
@Column({ name: 'iss_created_at', type: 'timestamp', default: () => 'CURRENT_TIMESTAMP' })
issCreatedAt: Date;
```

## 단계 3: Redmine 가져오기 로직 개선

**파일**: `apps/api/src/domain/migration/service/redmine-migration.service.ts`

### 3.1 참고용 이슈넘버 + 생성일 입력

```typescript
const issue = this.issueRepo.create({
  // ... 기존 필드 ...
  issRefNumber: `RM-${ri.id}`,            // 참고용 이슈넘버
  issCreatedAt: new Date(ri.created_on),   // Redmine 원본 생성일
});
```

### 3.2 STATUS_MAP — Redmine REQUEST → AMA APPROVED 매핑

```typescript
const STATUS_MAP: Record<number, string> = {
  1: 'OPEN',         // New
  2: 'IN_PROGRESS',  // In Progress
  3: 'RESOLVED',     // Resolved
  4: 'OPEN',         // Feedback
  5: 'CLOSED',       // Closed
  6: 'REJECTED',     // Rejected
  7: 'APPROVED',     // Request → APPROVED
};
```
> Redmine REQUEST 상태(status_id 확인 필요)를 AMA APPROVED로 매핑

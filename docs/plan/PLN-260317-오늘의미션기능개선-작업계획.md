## 2. 사이드 임팩트 분석

### 2.1 직접 영향

| 영역 | 영향 내용 | 위험도 | 대응 |
|------|---------|-------|------|
| **check API Breaking Change** | `PATCH /mission/:date/check` Body가 `{result,score}` → `{lines[]}` 로 변경 | 🔴 높음 | FE/BE 동시 배포. 모바일 앱 미사용 확인됨 |
| **msn_check_result 값 변경** | 기존: `HALF`/`PARTIAL`/`ALL_DONE`/`EXCEED` → 신규: `ZERO`/`PARTIAL`/`GOOD`/`PERFECT` | 🟡 중간 | FE에서 기존+신규 값 모두 매핑 처리 |
| **msn_check_score 범위 변경** | 기존: `50/75/100/110` (고정 4종) → 신규: `0~100` (연속) | 🟢 낮음 | SMALLINT 타입 변경 불필요 (범위 내), 팀 통계 `avgScore` 계산에 영향 없음 (이미 number 연산) |

### 2.2 간접 영향

| 영역 | 파일 | 영향 | 대응 |
|------|------|------|------|
| **History 캘린더** | `TodayHistoryCalendar.tsx` | `SCORE_COLORS` 맵에 기존 4종만 있음 → 신규 4종 추가 필요 | 기존+신규 매핑 추가 |
| **History 상세 페이지** | `TodayHistoryPage.tsx` | `checkScore%` 표시 → 0~100 범위로 변경되어도 정상 동작 | 변경 불필요 |

### 2.3 하위 호환 전략

#### msn_check_result 값 전환

```
기존 데이터 (읽기 전용)              신규 데이터 (쓰기)
─────────────────────────          ─────────────────────
HALF      (50%)                    ZERO     (0%)
PARTIAL   (75%)                    PARTIAL  (1~74%)
ALL_DONE  (100%)                   GOOD     (75~99%)
EXCEED    (110%)                   PERFECT  (100%)
```

**FE 처리 전략:**
- `SCORE_COLORS` 맵에 기존 4종 + 신규 4종 총 8종 매핑
- 라벨 표시 시 `msnCheckScore` 값 기준 `N%`로 통일 (result 라벨 사용 안함)
- History 캘린더 dot 색상: 점수 범위 기반 색상 분기로 통일 (result 문자열 의존 제거도 가능하나, 과도한 변경 방지를 위해 매핑 추가 방식 채택)

#### API 호환성

- check API는 **Breaking Change** — 이전 DTO 지원 불가
- 이유: 구조가 `{result, score}` → `{lines[]}` 로 완전히 다름
- 대응: FE/BE 동시 배포 (모노레포 단일 배포 파이프라인)

  if (score >= 75) return 'GOOD';
  if (score > 0) return 'PARTIAL';
  return 'ZERO';
}

## 5. 핵심 기술 결정 사항

| # | 결정 | 이유 |
|---|------|------|
| 1 | TodoEntity Repository 직접 사용 (TodoService 미사용) | Mission 체크에서의 Todo 생성은 단순 INSERT만 필요. TodoService의 computed status 로직 등 불필요한 복잡성 회피 |
| 2 | TypeORM `DataSource.transaction()` 사용 | 라인별 Todo INSERT + 미션 UPDATE를 원자적으로 처리. 부분 실패 시 전체 롤백 |
| 3 | 라인 파싱은 프론트엔드 DOMParser | HTML 구조를 서버로 전달할 필요 없음. 클라이언트에서 파싱 후 결과만 서버 전송 |
| 4 | carry-over는 DB 저장 (`msn_carry_over_text`) | 초안의 스테이트 전용 제안 대비 새로고침 유실 방지. 오늘 미션 저장 시 자동 clear 불필요 (이력 보존) |
| 5 | `SCORE_COLORS` 매핑 추가 (점수 기반 통일 불채택) | History 캘린더 기존 구조 최소 변경. 기존+신규 result 값 모두 색상 매핑 |
| 6 | check API Breaking Change 허용 | 모노레포 단일 배포이므로 FE/BE 동시 반영. 모바일 앱 미사용 확인됨 |
| 7 | `tdo_source` 컬럼 Phase 2 연기 | 미션→Todo 역추적은 현 시점에서 불필요. 향후 분석 기능 추가 시 별도 작업 |

## 2. 구현 계획

### 단계 1: 공용 `useDebounce` 훅 생성

**파일**: `apps/web/src/hooks/useDebounce.ts` (신규)

```typescript
import { useState, useEffect } from 'react';

export function useDebounce<T>(value: T, delay = 500): T {
  const [debouncedValue, setDebouncedValue] = useState(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);
  return debouncedValue;
}
```

### 단계 3: Debounce 미적용 페이지에 추가 (4건)

이 페이지들은 `search` state를 직접 API 훅에 전달하여 매 키입력마다 API를 호출합니다.
검색어를 `searchInput` (로컬) + `debouncedSearch` (API용)로 분리합니다.

| # | 파일 | 변경 |
|---|------|------|
| 3-1 | `domain/project/pages/ProjectListPage.tsx` | `search` → `searchInput` + `useDebounce` |
| 3-2 | `domain/service-management/pages/ClientListPage.tsx` | `search` → `searchInput` + `useDebounce` |
| 3-3 | `domain/billing/pages/ContractListPage.tsx` | `search` → `searchInput` + `useDebounce` |
| 3-4 | `domain/billing/pages/PartnerListPage.tsx` | `search` → `searchInput` + `useDebounce` |

**변경 패턴** (각 페이지 동일):
```typescript
// Before
const [search, setSearch] = useState('');
const { data } = useXxxList({ search: search || undefined });

// After
const [searchInput, setSearchInput] = useState('');
const debouncedSearch = useDebounce(searchInput, 500);
const { data } = useXxxList({ search: debouncedSearch || undefined });
```

## 4. 사이드 임팩트 분석

| 항목 | 영향도 | 설명 |
|------|--------|------|
| 검색 응답 지연 체감 | 낮음 | 300ms → 500ms (200ms 차이, 체감 미미) |
| 드롭다운/체크박스 필터 | 없음 | 텍스트 입력 필드만 대상, 선택형 필터는 즉시 반영 유지 |
| 기존 테스트 | 없음 | 동작 방식 동일, 타이밍만 변경 |
| API 호출 횟수 | 감소 | Debounce 미적용 페이지에서 불필요한 API 호출 대폭 감소 |

## 5. 검증 계획

1. 이슈 페이지 검색 필드에서 한글 입력 중 포커스 유지 확인
2. 입력 완료 후 500ms 뒤 검색 결과 갱신 확인
3. 프로젝트/고객/계약/파트너 목록 검색에서 동일 동작 확인
4. 글로벌 검색바에서 입력 중 수정 가능 확인
5. 드롭다운 필터(유형, 상태 등) 즉시 반영 동작 확인

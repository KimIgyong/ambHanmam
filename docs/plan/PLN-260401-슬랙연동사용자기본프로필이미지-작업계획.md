# PLAN-슬랙연동사용자기본프로필이미지-작업계획-20260401

## 1. 시스템 개발 현황 분석
- 현재 사용자 생성 경로 전반에서 `usr_profile_image` 기본값이 설정되지 않음.
- 프로필 이미지 조회 API는 이미지 미존재 시 404 반환.
- Amoeba Talk `UserAvatar`는 `/users/{id}/profile-image`를 직접 호출하므로 이미지 없는 사용자는 404가 누적됨.
- 슬랙 inbound 메시지는 시스템 사용자 ID(`00000000-0000-0000-0000-000000000000`)를 sender로 사용하여 일반 사용자 프로필 체계와 분리되어 있음.

## 4. 사이드 임팩트 분석
- 저장 용량 증가
  - 모든 사용자에 이미지 저장으로 DB 용량 증가
  - 대응: 압축된 PNG/JPEG 생성, 최대 크기 제한
- 성능 영향
  - 생성 시 CPU 부하 증가 가능
  - 대응: 경량 생성 로직 및 배치 처리
- 호환성 영향
  - 기존 업로드/삭제 API와 충돌 가능
  - 대응: 사용자 업로드 이미지는 기본 이미지보다 우선 유지
- 슬랙 연동 영향
  - 시스템 사용자 처리 정책 미정 시 404가 잔존 가능
  - 대응: 시스템 ID fetch 스킵 규칙 적용

## 5. DB 마이그레이션
- 기본안(권장): 없음
  - PNG/JPEG 바이너리 저장 유지
- 대안안(SVG 원본 저장): 필요
  - 예: `usr_profile_image_mime` 컬럼 추가
  - 주의: staging/production 수동 SQL 필수

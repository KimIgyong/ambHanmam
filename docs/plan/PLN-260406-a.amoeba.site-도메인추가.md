## 2. 단계별 구현 계획

### Phase 1: nginx 설정 추가 (서버 작업)

**작업 내용**: `a.amoeba.site.conf` 생성

**실행 명령**:
```bash
# 1) ama.amoeba.site.conf 복사 → a.amoeba.site.conf
ssh amb-production "sudo cp /etc/nginx/conf.d/ama.amoeba.site.conf /etc/nginx/conf.d/a.amoeba.site.conf"

# 2) server_name 변경
ssh amb-production "sudo sed -i 's/ama\.amoeba\.site/a.amoeba.site/g' /etc/nginx/conf.d/a.amoeba.site.conf"

# 3) 주석 헤더 변경
ssh amb-production "sudo sed -i 's/ama\.amoeba\.site - AMA/a.amoeba.site - AMA/g' /etc/nginx/conf.d/a.amoeba.site.conf"

# 4) 로그 파일 경로 변경
# (sed에서 ama.amoeba.site → a.amoeba.site 이미 처리됨)

# 5) 설정 검증
ssh amb-production "sudo nginx -t"

# 6) nginx 리로드 (중단 없음)
ssh amb-production "sudo nginx -s reload"
```

**생성될 파일 내용** (예상):
```nginx
server {
    listen 80;
    listen [::]:80;
    server_name a.amoeba.site;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name a.amoeba.site;
    ssl_certificate /etc/nginx/ssl/amoeba.site.crt;
    ssl_certificate_key /etc/nginx/ssl/amoeba.site.key;
    # ... (ama.amoeba.site.conf와 동일)

    error_log /var/log/nginx/a.amoeba.site-error.log;
    access_log /var/log/nginx/a.amoeba.site-access.log;
}
```

**검증**: `curl -I https://a.amoeba.site` → 200 OK

### Phase 2: CORS 설정 추가 (환경변수)

**작업 내용**: `.env.production`에 `a.amoeba.site` CORS origin 추가

**변경 전**:
```
CORS_ORIGINS=https://ama.amoeba.site,https://stg-apps.amoeba.site,https://apps.amoeba.site
PORTAL_CORS_ORIGINS=https://www.amoeba.site,https://ama.amoeba.site
```

**변경 후**:
```
CORS_ORIGINS=https://ama.amoeba.site,https://a.amoeba.site,https://stg-apps.amoeba.site,https://apps.amoeba.site
PORTAL_CORS_ORIGINS=https://www.amoeba.site,https://ama.amoeba.site,https://a.amoeba.site
```

**실행 명령**:
```bash
# 1) 백업
ssh amb-production "cp ~/ambManagement/docker/production/.env.production ~/ambManagement/docker/production/.env.production.bak.$(date +%Y%m%d_%H%M%S)"

# 2) CORS_ORIGINS에 a.amoeba.site 추가
ssh amb-production "sed -i 's|CORS_ORIGINS=https://ama.amoeba.site,|CORS_ORIGINS=https://ama.amoeba.site,https://a.amoeba.site,|' ~/ambManagement/docker/production/.env.production"

# 3) PORTAL_CORS_ORIGINS에 a.amoeba.site 추가
ssh amb-production "sed -i 's|PORTAL_CORS_ORIGINS=https://www.amoeba.site,https://ama.amoeba.site|PORTAL_CORS_ORIGINS=https://www.amoeba.site,https://ama.amoeba.site,https://a.amoeba.site|' ~/ambManagement/docker/production/.env.production"

# 4) 변경 확인
ssh amb-production "grep -E 'CORS_ORIGINS|PORTAL_CORS' ~/ambManagement/docker/production/.env.production"
```

### Phase 3: Docker 컨테이너 재시작

**작업 내용**: CORS 변경 적용을 위해 API 컨테이너만 재시작

```bash
# API 컨테이너만 재시작 (약 10초 다운타임)
ssh amb-production "cd ~/ambManagement && docker compose -f docker/production/docker-compose.production.yml --env-file docker/production/.env.production restart amb-api amb-portal-api"

# 상태 확인
ssh amb-production "docker ps --format 'table {{.Names}}\t{{.Status}}' | grep amb"
```

> ⚠️ `amb-web`, `amb-portal-web`은 재시작 불필요 (VITE 빌드 시점 변수 없음, 정적 파일만 서빙)

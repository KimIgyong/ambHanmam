#### 변경 파일

|------|------|------|
| `apps/api/src/domain/portal-bridge/service/portal-bridge.service.ts` | 수정 | `getDeletionPreview()` 메서드 추가 |
| `apps/api/src/domain/portal-bridge/controller/portal-bridge.controller.ts` | 수정 | `GET /customers/:pctId/deletion-preview` 엔드포인트 추가 |

#### 구현 상세

**`getDeletionPreview(pctId: string)`** 반환 구조:

```typescript
{
  customer: {
    pctId: string;
    pctEmail: string;
    pctName: string;
    pctCompanyName: string;
  };
  counts: {
    mappings: number;        // amb_portal_user_mappings 건수
    payments: number;        // amb_svc_portal_payments 건수
    client: {                // amb_svc_clients (nullable)
      cliId: string;
      cliCompanyName: string;
    } | null;
    subscriptions: number;   // amb_svc_subscriptions 건수 (cliId 기준)
    usageRecords: number;    // amb_svc_usage_records 건수
  };
  warnings: {
    hasActiveSubscription: boolean; // Level 2 차단 조건
    hasCompletedPayments: boolean;  // 완료된 결제 존재 여부 (경고용)
  };
}
```

**주의**: portal-api 소속 테이블(`amb_svc_portal_payments`)은 api에 읽기전용 엔티티로 등록 필요. 현재 `PortalCustomerReadonlyEntity`와 동일한 패턴으로 `PortalPaymentReadonlyEntity` 추가.

### Phase 2: 백엔드 — 완전 삭제 API

**목표**: Level 1/2 Hard Delete 실행 엔드포인트 제공

#### 변경 파일

| 파일 | 유형 | 내용 |
|------|------|------|
| `apps/api/src/domain/portal-bridge/service/portal-bridge.service.ts` | 수정 | `deleteCustomerPermanent()` 메서드 추가 |
| `apps/api/src/domain/portal-bridge/controller/portal-bridge.controller.ts` | 수정 | `DELETE /customers/:pctId/permanent` 엔드포인트 추가 |
| `apps/api/src/domain/portal-bridge/dto/` | 신규 | `PermanentDeleteRequest` DTO |
| `apps/api/src/domain/portal-bridge/entity/portal-payment-readonly.entity.ts` | 신규 | 결제 읽기전용 엔티티 |
| `apps/api/src/domain/portal-bridge/portal-bridge.module.ts` | 수정 | 엔티티 등록 |

#### 구현 상세

**`deleteCustomerPermanent(pctId, dto, adminUserId)`**:

```typescript
// dto: { level: 1 | 2, confirmEmail: string }

// 1. 고객 조회 (withDeleted: true — soft-deleted도 완전 삭제 가능)
// 2. 이메일 확인 검증 (dto.confirmEmail === customer.pctEmail)
// 3. Level 2인 경우: 활성 구독 확인 → 있으면 400 에러
// 4. 트랜잭션 내에서 삭제 실행:

await manager.transaction(async (tx) => {
  // Level 2 추가 삭제 (고객사/구독/사용기록)
  if (level === 2 && cliId) {
    const subIds = await tx.find(SvcSubscriptionEntity, 
      { where: { cliId }, select: ['subId'], withDeleted: true });
    if (subIds.length > 0) {
      await tx.delete(SvcUsageRecordEntity, { subId: In(subIds.map(s => s.subId)) });
    }
    await tx.delete(SvcSubscriptionEntity, { cliId });  // hard delete
    await tx.delete(SvcClientEntity, { cliId });         // hard delete
  }

  // Level 1 기본 삭제
  await tx.delete(PortalUserMappingEntity, { pctId });
  await tx.delete(PortalPaymentReadonlyEntity, { ppmCustomerId: pctId });
  await tx.delete(PortalCustomerReadonlyEntity, { pctId });
});

// 5. Logger로 감사 로그 기록
```

**Request DTO**:

```typescript
class PermanentDeleteRequest {
  @IsIn([1, 2])
  level: number;

  @IsEmail()
  confirm_email: string;
}
```

#### 검증

```bash
npm run -w @amb/api build
```

## 3. 사이드 임팩트 분석

### 3.1 영향 없는 영역

| 영역 | 이유 |
|------|------|
| 기존 소프트 삭제 | 엔드포인트/로직 변경 없음, 완전 분리 |
| Portal 로그인/회원가입 | 삭제 후 이메일로 재가입만 가능해짐 (양방향 영향 없음) |
| Google OAuth | 삭제된 이메일은 재가입 대상이 됨 (기존 `email_exists` 로직 무관) |
| 구독 결제 흐름 | Level 2 삭제 시 활성 구독 차단하므로 진행 중 결제 보호됨 |

### 3.2 주의 영역

| 영역 | 영향 | 대응 |
|------|------|------|
| **User 매핑 내부 사용자** | Level 1 삭제 시 매핑만 제거, 내부 UserEntity는 보존 | 내부 사용자는 별도 관리 (삭제하지 않음) |
| **Stripe 고객 연동** | 고객에 `pct_stripe_customer_id` 있을 수 있음 | DB 삭제만 수행, Stripe 측 정리는 별도 (향후 확장) |
| **자동 프로비저닝 재실행** | Hard delete 후 같은 이메일로 재가입 시 새 고객으로 auto-provision됨 | 정상 동작 (새 법인/사용자 생성) |
| **결제 감사 추적** | Level 1에서 결제 이력 삭제됨 | 삭제 미리보기에서 결제 건수 표시하여 관리자 인지 유도 |

### 3.3 DB 마이그레이션

**마이그레이션 불필요** — 기존 테이블 구조 변경 없음. 새 엔티티(`PortalPaymentReadonlyEntity`)는 기존 테이블의 읽기전용 뷰.

## 5. 구현 순서 및 의존성

```

    │   · GET 엔드포인트 추가
    │   · 빌드 검증
    │
    ▼
Phase 2: 완전 삭제 API (백엔드)
    │   · PermanentDeleteRequest DTO 생성
    │   · deleteCustomerPermanent() 구현
    │   · DELETE 엔드포인트 추가
    │   · 빌드 검증
    │
    ▼
Phase 3: 프론트엔드 연동 + UI
    │   · API 서비스 + Hook 추가
    │   · 드롭다운 메뉴 변경
```

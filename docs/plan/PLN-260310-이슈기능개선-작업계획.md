## 2. 단계별 구현 계획

### Phase 1: Toast 알림 (IssuesPage.tsx)

**위치**: `handleFormSubmit` → `createIssue.mutate.onSuccess` / `updateIssue.mutate.onSuccess`  
**추가**: `handleDeleteConfirm` → `deleteIssue.mutate.onSuccess`

```ts
// 등록 성공
onSuccess: () => {
  toast.success(t('issues:messages.created'));
  setShowFormModal(false);
}

// 수정 성공
onSuccess: () => {
  toast.success(t('issues:messages.updated'));
  setShowFormModal(false);
  setEditingIssue(null);
}

// 삭제 성공
onSuccess: () => {
  toast.success(t('issues:messages.deleted'));
  setDeletingIssue(null);
}
```

**영향 범위**: IssuesPage.tsx만 변경 (toast import 추가)

#### 2-2. 프론트엔드: IssueFormModal 번역 패널

**상태 추가**:
```ts
const [translateLang, setTranslateLang] = useState<'ko' | 'en' | 'vi' | null>(null);
```

### Phase 3: AI 검토 영어 기본

#### 3-1. 백엔드 language 파라미터 수신

**파일**: `apps/api/src/domain/issues/controller/issue.controller.ts`
```ts
@Post(':id/ai-review')
async aiReview(
  @Param('id') id: string,
  @CurrentUser() user: UserPayload,
  @Req() req: any,
  @Body() body: { language?: string },  // 추가
  @Res() res: Response,
) {
  const stream$ = this.issueService.aiReview(id, user.userId, req?.entityId, body.language || 'en');
```

**파일**: `apps/api/src/domain/issues/service/issue.service.ts`
```ts
aiReview(issueId: string, userId: string, entityId?: string, language: string = 'en') {
  // systemPrompt 수정:
  const langInstruction = language === 'ko'
    ? 'Respond only in Korean (한국어로만 응답하세요).'
    : language === 'vi'
      ? 'Respond only in Vietnamese (Chỉ trả lời bằng tiếng Việt).'
      : 'Respond only in English.';
  
  const systemPrompt = `You are an expert issue reviewer...
${langInstruction}
Be concise and actionable.`;
```

#### 3-2. 프론트엔드: language 전달 + 번역 버튼

**파일**: `apps/api/src/domain/issues/components/IssueDetailModal.tsx`

**AI 검토 요청**:
```ts
body: JSON.stringify({ language: 'en' })
```

**번역 버튼** (AI 분석 결과 박스 우측 하단):
```tsx
<div className="mt-2 flex gap-1">
  {['ko', 'vi'].map((lang) => (
    <button onClick={() => translateAiReview(lang)}>
      {lang === 'ko' ? '🇰🇷 KO' : '🇻🇳 VI'}
    </button>
  ))}
</div>
```

**번역 로직**: 기존 `translateComment` 패턴과 동일하게 SSE 번역 처리 (`ISSUE` source_type, source_id = issue.issueId)

## 3. 사이드 임팩트 분석

| 변경 | 영향 범위 | 리스크 |
|------|-----------|--------|
| toast 추가 | IssuesPage만 | 없음 |
| TranslateRequest source_id optional | 기존 번역 API 모두 | 기존 호출은 source_id를 전달하므로 무영향 |
| ai-review body 파라미터 추가 | 하위 호환성 유지 (기본값 'en') | 없음 |
| IssueFormModal 번역 패널 | 폼 모달 UI | 레이아웃 높이 증가 가능 |

---

## 4. 구현 순서

1. `i18n 키 추가` (ko/en/vi)
2. `toast 알림` (IssuesPage.tsx)
3. `백엔드 ai-review language 파라미터` (controller + service)
4. `AI 검토 번역 버튼` (IssueDetailModal.tsx)
5. `백엔드 TranslateRequest source_id optional`
6. `IssueFormModal 번역 패널`

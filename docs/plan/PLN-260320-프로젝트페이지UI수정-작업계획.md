### 단계 2: Issues 탭 — 다중 필터 + 페이징 (`ProjectIssuesTab.tsx`)

**변경 파일**: `apps/web/src/domain/project/components/ProjectIssuesTab.tsx`

#### 2-1. 상태/타입 필터 변경

```
Before:
  const [statusFilter, setStatusFilter] = useState<string>('');
  const [typeFilter, setTypeFilter] = useState<string>('');
  
After:
  // 기본: CLOSED, REJECTED 제외
  const DEFAULT_STATUSES = new Set(['OPEN', 'APPROVED', 'IN_PROGRESS', 'RESOLVED']);
  const ALL_TYPES = new Set(['BUG', 'FEATURE_REQUEST', 'TASK', 'OPINION', 'OTHER']);
  const [statusFilter, setStatusFilter] = useState<Set<string>>(DEFAULT_STATUSES);
  const [typeFilter, setTypeFilter] = useState<Set<string>>(ALL_TYPES);
```

**필터링 로직 변경**:
```
Before: if (statusFilter) { if (i.status !== statusFilter) ... }
After:  if (!statusFilter.has(i.status)) return false;
        if (!typeFilter.has(i.type)) return false;
```

**UI 변경**: `<select>` → `<MultiSelectFilter>` 2개

#### 2-2. 클라이언트 사이드 페이징

```
신규 상태:
  const [currentPage, setCurrentPage] = useState(1);
  const PAGE_SIZE = 20;

계산:
  const totalPages = Math.ceil(issues.length / PAGE_SIZE);
  const pagedIssues = issues.slice((currentPage - 1) * PAGE_SIZE, currentPage * PAGE_SIZE);

필터 변경 시:
  setCurrentPage(1); // 페이지 리셋
```

**페이지네이션 UI**: 테이블 하단에 `이전 | 1 2 3 ... | 다음` + `N~M / 전체 T건` 표시

**사이드 임팩트**: 
- `selectedIssue`, `editingIssue` 등 모달 상태는 페이징과 독립 → 영향 없음
- 검색/필터 변경 시 page 리셋만 필요

**라벨 영역 동적 계산**:

### 단계 5: WBS 탭 — 그룹별 Closed 토글

**변경 파일**: `apps/web/src/domain/project/components/ProjectWbsTab.tsx`

#### 5-1. 데이터 전략 변경

```
Before:
  const { data: wbsTree } = useWbsTree(projectId, { include_closed: showClosed });
  // showClosed false → CLOSED 이슈 미포함

After:
  // 항상 전체 데이터 로드 (include_closed: true)
  const { data: wbsTree } = useWbsTree(projectId, { include_closed: true });
  
  // 프론트에서 그룹별 필터링
  const [closedVisibleGroups, setClosedVisibleGroups] = useState<Set<string>>(new Set());
  // 글로벌 showClosed → closedVisibleGroups에 모든 그룹 추가/제거
```

#### 5-2. 필터링 로직

```
const filterIssues = (issues: IssueResponse[], groupKey: string) => {
  if (showClosed || closedVisibleGroups.has(groupKey)) return issues;
  return issues.filter(i => i.status !== 'CLOSED');
};

// Closed 이슈 카운트 (그룹별)
const closedCount = (issues: IssueResponse[]) => 
  issues.filter(i => i.status === 'CLOSED').length;
```

#### 5-3. UI 변경

각 그룹 헤더에 추가:
```
<span className="font-medium">{epic.title}</span>
...
{closedCount(epic.issues) > 0 && !showClosed && !closedVisibleGroups.has(`epic-${epic.epicId}`) && (
  <button onClick={() => toggleClosedGroup(`epic-${epic.epicId}`)}>
    [closed {closedCount(epic.issues)}건 보기]
  </button>
)}
{closedVisibleGroups.has(`epic-${epic.epicId}`) && (
  <button onClick={() => toggleClosedGroup(`epic-${epic.epicId}`)}>
    [closed 숨기기]
  </button>
)}
```

동일하게 Component, Unassigned 그룹에도 적용

#### 5-4. 글로벌 토글 연동

```
글로벌 showClosed ON → 모든 그룹 CLOSED 표시 (closedVisibleGroups 무시)
글로벌 showClosed OFF → closedVisibleGroups에 따라 개별 토글
```

**사이드 임팩트**:
- 항상 `include_closed: true`로 변경 → API 응답 데이터 증가 (성능 영향 미미)
- `issueCount`/`doneIssueCount`는 서버에서 계산되므로 진행률 표시에 영향 없음
- 드래그앤드롭: CLOSED 이슈가 숨겨진 상태에서는 드래그 대상이 아님 → 문제 없음

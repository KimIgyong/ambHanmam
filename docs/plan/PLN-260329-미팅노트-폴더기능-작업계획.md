## 1. 구현 단계

### Phase 1: 백엔드 (Entity + Service + Controller)

- **파일**: `apps/api/src/domain/meeting-notes/dto/request/update-meeting-note.request.ts` (수정) — `folder_id` 추가

#### 1-4. Service에 폴더 CRUD + 목록 필터 추가
- **파일**: `apps/api/src/domain/meeting-notes/service/meeting-note.service.ts` (수정)

  - `updateFolder(folderId, req, userId)`

#### 1-5. Controller에 폴더 엔드포인트 추가

  - `DELETE /meeting-notes/folders/:id`

#### 1-6. Module 업데이트
- **파일**: `apps/api/src/domain/meeting-notes/meeting-notes.module.ts` (수정)

#### 1-7. Mapper 업데이트

## 3. 사이드 임팩트 분석

| 영향 범위 | 위험도 | 설명 |
|-----------|--------|------|
| 기존 노트 데이터 | 🟢 낮음 | `folder_id NULL` = 미분류로 자연 호환 |
| `GET /meeting-notes` API | 🟢 낮음 | 기존 파라미터 유지, `folder_id` optional 추가만 |
| CreateDTO / UpdateDTO | 🟢 낮음 | `folder_id` optional로 추가, 기존 필드 변경 없음 |
| MeetingNoteResponse 타입 | 🟡 중간 | `folderId`, `folderName` 추가 (기존 필드 영향 없음) |
| MeetingNotesPage 레이아웃 | 🟡 중간 | 사이드바 추가로 레이아웃 변경 |
| MeetingNoteFormModal | 🟢 낮음 | 폴더 드롭다운 1개 추가 |
| DB | 🟢 낮음 | 새 테이블 + 기존 테이블 nullable 컬럼 추가 |

#### 1-2. 컨트롤러 엔드포인트

```typescript
// asset.controller.ts
@Get('import/template')
@ApiOperation({ summary: 'Download asset import Excel template' })
async downloadImportTemplate(@Res() res: Response) {
  const buffer = await this.assetService.generateImportTemplate();
  res.set({
    'Content-Type': 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
    'Content-Disposition': 'attachment; filename=asset_import_template.xlsx',
  });
  res.send(buffer);
}
```

### Phase 2: 백엔드 — 엑셀 벌크 업로드 API

**파일**: `asset.service.ts`, `asset.controller.ts`

#### 2-1. 엑셀 파싱 및 벌크 생성 메서드

```typescript
// asset.service.ts
async importFromExcel(
  buffer: Buffer,
  userId: string,

): Promise<{
  totalRows: number;

}> {
  // 1. 파일 파싱
  const wb = new ExcelJS.Workbook();
  await wb.xlsx.load(buffer);
  const ws = wb.getWorksheet(1);

  // 2. 헤더 검증
  // 3. 행별 데이터 추출 및 검증
  // 4. 유효한 행 배치 생성 (50건 단위)
  //    - generateAssetCode() 호출
  //    - 변경이력 CREATE 기록
  // 5. 결과 리포트 반환
}
```

**검증 로직**:
- 필수값: `asset_name`, `asset_category`, `ownership_type`
- Enum 검증: `asset_category` ∈ {IT_EQUIPMENT, SUPPLY, FACILITY, MEETING_ROOM}
- Enum 검증: `ownership_type` ∈ {PURCHASE, LEASE, OTHER}
- Enum 검증: `status` ∈ {STORED, IN_USE, RESERVED} (비어있으면 STORED)
- Enum 검증: `currency` ∈ {USD, KRW, VND}
- 숫자 검증: `quantity`, `purchase_amount`, `depreciation_years`, `residual_value`
- 날짜 검증: `purchase_date` → YYYY-MM-DD 형식
- 행 수 제한: 500건 초과 시 에러

#### 2-2. 컨트롤러 엔드포인트

```typescript
// asset.controller.ts
@Post('import/excel')
@ApiOperation({ summary: 'Bulk import assets from Excel' })
@ApiConsumes('multipart/form-data')
@UseInterceptors(FileInterceptor('file', { limits: { fileSize: 10 * 1024 * 1024 } }))
async importExcel(
  @UploadedFile() file: Express.Multer.File,
  @CurrentUser() user: UserPayload,
  @Req() req: any,
) {
  const role = req.entityRole || user.role;
  const data = await this.assetService.importFromExcel(
    file.buffer, user.userId, role, req.entityId,
  );
  return { success: true, data, timestamp: new Date().toISOString() };
}
```

## 5. 주의사항

1. **라우트 순서**: `import/template`, `import/excel` 경로는 `:id` 파라미터 라우트보다 **위에** 선언해야 함 (NestJS는 선언 순서대로 매칭)
2. **자산 코드 동시성**: `generateAssetCode()`가 날짜 기반 순번이므로, 벌크 생성 시 순차적으로 호출하여 중복 방지
3. **회의실 제외**: MEETING_ROOM 카테고리는 벌크 등록 시 기본 필드만 생성 (room_* 필드는 단건 수정으로 보완)
4. **에러 처리**: 전체 롤백이 아닌 행별 처리 — 성공한 행은 저장, 실패한 행은 에러 리포트

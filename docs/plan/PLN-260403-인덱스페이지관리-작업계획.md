# PLAN-인덱스페이지관리-작업계획-20260403

## 1. 시스템 개발 현황 분석

### 1.1 기존 인프라
| 구성요소 | 현황 | 활용 |
|----------|------|------|
| `amb_site_settings` 테이블 | 법인별 사이트 설정 (포탈URL, IP허용 등) | 컬럼 2개 추가 |
| `SiteSettingsEntity` | TypeORM 엔티티 (8 필드) | 필드 2개 추가 |
| `SiteSettingsService` | get/update 메서드 | 확장 |
| `SiteSettingsController` | `GET/PUT /settings/site` (AdminGuard) | 확장 불필요 |
| `SiteSettingsPage.tsx` | 관리자 사이트 설정 UI | 인덱스 섹션 추가 |
| `useSiteSettings.ts` | React Query hook | DTO 확장만 |
| `/api/v1/open/*` 패턴 | 공개 API 엔드포인트 | 인덱스 페이지 API 추가 |
| React Router | SPA 라우팅 (AuthGuard 포함) | IndexGate 컴포넌트 추가 |

### 1.2 변경 영향도
- **DB**: 기존 테이블에 nullable 컬럼 추가 → 기존 기능 영향 없음
- **API**: 기존 엔드포인트 DTO 확장 + 신규 공개 API 1개
- **Frontend**: SiteSettingsPage 확장 + IndexGate 컴포넌트 추가
- **Nginx**: 변경 불필요 (SPA fallback 그대로 활용)

## 2. 단계별 구현 계획

### Phase 1: Backend — DB 컬럼 + Entity + DTO 확장

**목표**: 인덱스 설정 데이터 저장/조회 가능하도록 백엔드 확장

**작업 내용**:
1. `SiteSettingsEntity`에 필드 추가
   - `stsIndexEnabled: boolean` (default: false)
   - `stsIndexHtml: string | null` (text)
2. `UpdateSiteSettingsRequest` DTO에 필드 추가
   - `index_enabled?: boolean`
   - `index_html?: string` (MaxLength: 1,000,000)
3. `SiteSettingsService` 수정
   - `updateSettings()`에 새 필드 매핑
   - `toResponse()`에 새 필드 포함
4. `SiteSettingsResponse` 타입 확장 (packages/types)
   - `indexEnabled: boolean`
   - `indexHtml: string | null`
5. 스테이징 DB 마이그레이션 SQL 준비

**사이드 임팩트**: 기존 GET/PUT API 응답에 필드 추가뿐 → BW 호환

### Phase 2: Backend — 공개 API 엔드포인트

**목표**: 비인증 사용자가 인덱스 설정 조회 가능

**작업 내용**:
1. `OpenController` (기존 `/api/v1/open`) 또는 신규 컨트롤러에 엔드포인트 추가
   - `GET /api/v1/open/index-page`
   - 응답: `{ enabled: boolean, html: string | null }`
   - 인증 불필요
2. `SiteSettingsService`에 `getIndexPage()` 메서드 추가
   - 첫 번째 Entity의 설정에서 `indexEnabled`, `indexHtml`만 반환

**사이드 임팩트**: 공개 API 추가 → 기존 API 영향 없음

### Phase 3: Frontend — 관리자 설정 UI 확장

**목표**: 관리자가 인덱스 페이지 ON/OFF 및 HTML 편집 가능

**작업 내용**:
1. `SiteSettingsPage.tsx` 확장
   - "인덱스 페이지" 섹션 추가 (기존 설정 아래)
   - ON/OFF 토글 스위치
   - HTML 코드 에디터 (monospace textarea + 라인 번호)
   - "미리보기" 버튼 (새 탭에서 HTML 렌더링)
   - "기본 템플릿" 버튼 (예시 HTML 삽입)
2. `useSiteSettings` 훅 → DTO 확장에 따라 자동 호환
3. i18n 번역 키 추가 (ko/en/vi)

**사이드 임팩트**: 기존 SiteSettingsPage 레이아웃에 섹션 추가

### Phase 4: Frontend — IndexGate 라우팅

**목표**: 루트(`/`) 접속 시 조건부 인덱스 페이지 표시

**작업 내용**:
1. `IndexGate.tsx` 컴포넌트 생성
   - 앱 로드 시 `GET /api/v1/open/index-page` 호출
   - `enabled === true` → 커스텀 HTML 렌더링 (전체 화면)
   - `enabled === false` → children 렌더 (기존 AuthGuard 흐름)
   - 로딩 중 → splash/loading 표시
   - 에러 시 → fallback to 기존 흐름
2. `router/index.tsx` 수정
   - 루트 경로에 IndexGate 래핑
   - 인덱스 페이지 내 "로그인" 링크 → `/login` 이동
3. `settings.service.ts`에 공개 API 호출 함수 추가

**사이드 임팩트**: 
- 루트 접속 시 추가 API 호출 1회 (캐시로 최소화)
- 기존 인증 사용자 → IndexGate 통과 후 기존 흐름 유지

| 영역 | 임팩트 | 대응 |
|------|--------|------|

```sql

  ADD COLUMN IF NOT EXISTS sts_index_enabled BOOLEAN DEFAULT false,
  ADD COLUMN IF NOT EXISTS sts_index_html TEXT;
```

## 1. 시스템 개발 현황 분석

### 1.1 현재 구현 상태

| 기능 | Cell 범위 조회 | Unit 범위 조회 | 비고 |
|------|:---:|:---:|------|
| **Todos** | ✅ 완료 | ❌ 미구현 | `GET /todos/cell` 존재 |

### Phase 5: Meeting Notes 페이지 (탭 재구성)

#### Task 5.1: 백엔드 - 뷰 스코프 필터 추가
- **파일**: `apps/api/src/domain/meeting-notes/service/meeting-note.service.ts`
- **작업**: 기존 `visibility` 파라미터 외에 `scope` 파라미터 추가
  - `scope=mine`: 내가 작성한 노트만
  - `scope=unit`: 내 Unit에서 작성된 노트 (mtnUnit 기준)
  - `scope=cell`: 내 Cell에서 작성된 노트 (mtnCellId 기준)
  - `scope=all`: Entity 전체
- **주의**: 기존 `visibility` (PRIVATE/CELL/ENTITY)는 노트의 공개 범위로 유지, `scope`는 뷰 필터로 분리

#### Task 5.2: 프론트엔드 - Notes 탭 재구성
- **파일**: `apps/web/src/domain/meeting-notes/pages/MeetingNotesPage.tsx`
- **변경**: 
  - 기존 visibility 버튼 → `ViewScopeTab` 컴포넌트 적용 (뷰 범위)
  - visibility 필터는 추가 필터로 이동 또는 제거 검토

- **파일**: `apps/web/src/domain/today/pages/TodayPage.tsx`

## 3. 사이드 임팩트 분석

### 3.1 영향 범위

| 영향 영역 | 영향도 | 설명 |
|-----------|--------|------|
| Todos 페이지 | ⚠️ 중 | 탭 구조 변경, 제목 변경 |
| Issues 페이지 | ⚠️ 중 | 스코프 필터 구조 변경, my_involved 분리 |
| Calendar 페이지 | ⚠️ 중 | 필터 모드 확장, SHARED 분리 |
| Meeting Notes 페이지 | ⚠️ 중 | 필터→탭 전환, visibility와 scope 분리 |
| Today 페이지 | 🔴 고 | 3-패널→단일 패널 리팩토링, 권한 해제 |
| 사이드바 메뉴 | ✅ 없음 | 이미 변경 완료 (나의 할일) |
| 모바일 앱 | ✅ 없음 | 웹 전용 변경 |

### 3.2 호환성 위험

1. **Today TeamTodayPanel/AllTodayPanel**: 리팩토링 시 기존 기능 손실 위험
   - 대응: 기존 패널의 데이터 조회 로직을 보존하면서 래핑
2. **Calendar SHARED 모드**: 4-탭에서 제외 시 기존 사용자 혼란
   - 대응: 별도 토글 UI로 SHARED 유지
3. **Issues my_involved**: 자주 사용되는 필터인 경우 접근성 저하
   - 대응: 추가 필터 옵션으로 유지
4. **Meeting Notes visibility**: 뷰 범위(scope)와 노트 공개 범위(visibility)가 혼동 가능
   - 대응: UI에서 명확히 분리 (탭 = 뷰 범위, 드롭다운 = 공개 범위)

### 3.3 데이터 무결성
- Cell/Unit 미소속 사용자 → 해당 탭 데이터 비어있음 (에러는 아님)
- 소속 변경 시 실시간 반영 (매 API 호출마다 소속 조회)

## 6. 테스트 고려사항

### 6.1 테스트 시나리오
1. Cell/Unit 모두 소속된 사용자 → 4개 탭 모두 데이터 표시
2. Cell만 소속된 사용자 → Unit 탭 빈 결과
3. Unit만 소속된 사용자 → Cell 탭 빈 결과
4. 둘 다 미소속 → mine과 all만 데이터 표시
5. Today 페이지 일반 사용자 → 4개 탭 모두 접근 가능
6. 각 페이지 탭 전환 시 API 호출 확인
7. 페이지 새로고침 시 탭 상태 유지 여부

### 6.2 성능 테스트
- Unit/Cell 멤버 수가 많은 경우(50명+) → 응답 시간 확인
- 회사 전체 보기 → 대량 데이터 페이지네이션 동작 확인

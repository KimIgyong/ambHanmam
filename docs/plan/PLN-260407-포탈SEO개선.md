## 2. 단계별 구현 계획

### Phase 1: 기반 작업 — react-helmet-async + OG 이미지 (Critical)

**목표**: 페이지별 동적 meta 인프라 구축, 소셜 공유 이미지 수정

#### 1-1. react-helmet-async 설치 및 Provider 설정
- `npm install react-helmet-async` (portal-web)
- `App.tsx` 또는 최상위에 `<HelmetProvider>` 래핑
- **사이드 임팩트**: 없음 (추가적 래핑만)

#### 1-2. SEO 공통 컴포넌트 생성
- `src/components/seo/PageHead.tsx` — 재사용 가능한 Helmet 래퍼
  ```typescript
  interface PageHeadProps {
    title?: string;
    description?: string;
    path?: string;          // canonical 경로
    ogImage?: string;       // OG 이미지 URL
    jsonLd?: object;        // 구조화 데이터
    noindex?: boolean;      // 인덱싱 제어
  }
  ```
- 기본값: index.html의 현재 meta를 fallback으로 사용
- canonical 자동 생성: `https://www.amoeba.site{path}`
- hreflang 자동 포함: ko, en, vi, x-default

#### 1-3. OG 이미지 PNG 변환
- `og-image.svg` → `og-image.png` (1200×630) 변환
- index.html 및 PageHead에서 PNG 경로 참조
- **사이드 임팩트**: 소셜 공유 미리보기 즉시 개선

#### 1-4. index.html AMA→Ạ/에이닷(A-dot) 리브랜딩 + hreflang
- title, description, og, twitter, JSON-LD의 "AMA" → EN: "Ạ/A-dot", KO: "Ạ/에이닷" (7개소)
- `<link rel="alternate" hreflang>` 4개 추가
- `<link rel="manifest">` 추가 (Phase 3에서 파일 생성)
- **사이드 임팩트**: 검색 결과 표시명 변경

**산출물**: react-helmet-async 설치, PageHead 컴포넌트, 수정된 index.html, og-image.png

### Phase 3: sitemap · manifest · 캐시 최적화 (Medium)

**목표**: 크롤러 가이드 강화, PWA 기본 지원, 성능 개선

#### 3-1. sitemap.xml 재작성

**기존 10 URL → 20+ URL 확장:**

```xml
<!-- 메인 페이지 -->
/                               priority=1.0  weekly
/pricing                        priority=0.8  monthly
/register                       priority=0.7  monthly

<!-- 가이드 SPA 경로 (신규) -->
/guide/user-guide               priority=0.6  monthly
/guide/user-manual              priority=0.6  monthly
/guide/smart-todo               priority=0.6  monthly
/guide/work-tools               priority=0.6  monthly
/guide/google-drive             priority=0.5  monthly
/guide/app-install              priority=0.5  monthly
/guide/car-manager              priority=0.4  monthly

<!-- CMS 페이지 (신규) -->
/page/about                     priority=0.5  monthly
/page/terms                     priority=0.3  yearly
/page/privacy                   priority=0.3  yearly

<!-- 제거: 구 HTML 직접 경로 (SPA 경로로 대체) -->
<!-- /service/ama/*.html → 제거 -->
```

- `lastmod` 태그 추가 (현재 날짜)
- `/login`, `/forgot-password`, `/reset-password` 제외 (noindex 페이지)
- **사이드 임팩트**: 구 HTML 직접 URL은 sitemap에서 제거하되, 파일은 유지 (기존 북마크 호환)

#### 3-2. site.webmanifest 생성
```json
{
  "name": "Ạ/A-dot — AI Management Assistant",
  "short_name": "Ạ/A-dot",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#08122A",
  "theme_color": "#2563EB",
  "icons": [
    { "src": "/favicon.svg", "type": "image/svg+xml", "sizes": "any" },
    { "src": "/amoeba-symbol.png", "type": "image/png", "sizes": "192x192" }
  ]
}
```
- index.html에 `<link rel="manifest" href="/site.webmanifest" />`

#### 3-3. nginx 캐시 정책 세분화
```nginx
# HTML → no-cache (항상 최신 meta)
location ~* \.html$ {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
}

# 정적 자산 → 1y immutable (기존 유지)
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```
- **사이드 임팩트**: 정적 HTML(가이드)이 매번 재검증됨 → 실제 영향 미미 (304)

**산출물**: 확장된 sitemap.xml, site.webmanifest, 수정된 nginx.conf

## 4. 사이드 임팩트 분석

| 변경 | 영향 | 심각도 | 대응 |
|------|------|--------|------|
| react-helmet-async 추가 | 번들 사이즈 약 +8KB gzipped | 낮음 | 성능 영향 무시 가능 |
| HelmetProvider 래핑 | 기존 컴포넌트 동작 불변 | 없음 | - |
| og-image SVG→PNG | 기존 SVG 미사용 시 다운로더 약간 느림 (PNG 용량) | 낮음 | PNG 최적화 (100KB 이하) |
| sitemap 구 HTML 경로 제거 | 기존 Google 인덱스에서 점진적 제거 | 낮음 | SPA 경로로 대체되므로 콘텐츠 유지 |
| 404 catch-all 라우트 | 정의된 경로만 정상, 나머지 404 표시 | 낮음 | 기존 페이지 경로 모두 포함 확인 |
| nginx HTML no-cache | 첫 방문 시 HTML 항상 서버 검증 | 낮음 | 304 응답으로 속도 영향 미미 |
| title 변경 (AMA→Ạ/에이닷(A-dot)) | 검색결과 표시명 변경 | 중간 | 리브랜딩 의도된 변경 |

## 7. 참고 — SPA SEO 한계와 향후 고려사항

### 현재 접근 (이번 작업)
- react-helmet-async로 Googlebot에 대한 동적 meta 제공 (Googlebot은 JS 실행 지원)
- 소셜 크롤러(Facebook, LinkedIn, Twitter)는 JS 미실행 → index.html 정적 태그에 의존
- **효과**: Google 검색 최적화 높음, 소셜 공유는 랜딩 기본값 사용

### 향후 검토 (별도 프로젝트)
| 접근법 | 효과 | 복잡도 | 비고 |
|--------|------|--------|------|
| **prerender.io / rendertron** | 소셜 크롤러에도 동적 meta | 중간 | 별도 서비스 필요 |
| **vite-plugin-ssr / vite-plugin-ssg** | 주요 페이지 정적 생성 | 중간 | 빌드 시간 증가 |
| **Next.js 마이그레이션** | 완전한 SSR/ISR | 매우 높음 | 대규모 리팩토링 |

### 1.2 핵심 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| AMA 로그인 URL 형식 | `/{entityCode}/login` (path param) | EntityLoginPage가 이미 path param 지원, entity 자동 조회/표시됨 |
| Auto-Provision 응답 | entityCode 추가 | Portal-API가 파싱하여 register 응답에 포함 |
| 이메일 loginUrl | `/{entityCode}/login` 형식 | 이메일에서 클릭 시 entity 자동 선택 |
| 실패 시 Fallback | entityCode 없으면 기존 화면 유지 | auto-provision 실패 시 graceful degradation |

### Phase 1: 백엔드 — Auto-Provision 응답 확장 (apps/api)

**목표**: auto-provision 응답에 `entityCode` 추가, 이메일에 entityCode 포함

#### 2.1.1 portal-bridge.service.ts 수정

**파일**: `apps/api/src/domain/portal-bridge/service/portal-bridge.service.ts`

1. `createInternalAccount()` 반환에 `entityCode` 추가:
```typescript
return {
  userId: user.usrId,
  email: user.usrEmail,
  name: user.usrName,
  role: user.usrRole,
  entityId: dto.entity_id,
  entityName: entity.entName,
  entityCode: entity.entCode,    // ← 추가
  mappingId: mapping.pumId,
};
```

2. 이메일 변수에 `entityCode` 추가 + `loginUrl` 개선:
```typescript
const variables: Record<string, string> = {
  userName: customer.pctName,
  userEmail: customer.pctEmail,
```

**사이드 임팩트**: 기존 createInternalAccount 호출부(관리자 매핑 생성 등)가 entityCode 필드 추가를 인식해야 하지만, 필드 추가이므로 하위 호환성 유지.

### Phase 2: 백엔드 — Portal-API 응답 확장 (apps/portal-api)

**목표**: portal-auth.service.ts가 auto-provision 응답에서 entityCode를 추출하여 register 응답에 포함

#### 2.2.1 portal-auth.service.ts 수정

**파일**: `apps/portal-api/src/domain/auth/service/portal-auth.service.ts`

1. `triggerAutoProvision()` 반환 타입을 void → `{ entityCode, entityName }` 으로 변경:
```typescript
private async triggerAutoProvision(pctId: string): Promise<{ entityCode?: string; entityName?: string }> {
  // ...기존 fetch 로직...
  if (!response.ok) { throw ... }
  const result = await response.json();
  return {
    entityCode: result?.data?.entityCode,
    entityName: result?.data?.entityName,
  };
}
```

2. `register()` 응답에 entityCode, entityName 추가:
```typescript
// triggerAutoProvision 호출
const provisionResult = await this.triggerAutoProvision(saved.pctId);

return {
  customerId: saved.pctId,
  email: saved.pctEmail,
  name: saved.pctName,
  emailVerified: saved.pctEmailVerified,
  entityCode: provisionResult?.entityCode,    // ← 추가
  entityName: provisionResult?.entityName,    // ← 추가
};
```

**사이드 임팩트**: register 응답 JSON에 필드 추가. 기존 프론트엔드에서 unused 필드이므로 하위 호환성 유지.

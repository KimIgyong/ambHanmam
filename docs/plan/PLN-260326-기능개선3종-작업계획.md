## 단계 1: 칸반뷰 카드 작성자 이름 추가

### 현황 분석
- `TodoKanbanView.tsx`에서 카드 렌더링 시 `todo.title`, `todo.dueDate`, `todo.tags`는 표시하지만 **`todo.userName`은 미표시**
- `TodoResponse.userName`은 백엔드에서 이미 반환됨 (unit/cell/company 쿼리 모두 user JOIN 포함)
- ViewScopeTab은 이미 모든 뷰 모드에 적용되어 있으므로 칸반뷰에서 scope 필터는 이미 동작

### 작업 내용

#### 1-1. TodoKanbanView.tsx — 카드에 작성자 이름 표시
- **위치**: 카드 제목 아래, 마감일 위 (또는 제목 옆)
- **표시 형식**: 작성자 아이콘 + 이름 (예: `👤 홍길동`)
- **조건**: `userName`이 존재할 때만 표시
- **스타일**: `text-xs text-gray-500` (보조 정보 스타일)

```tsx
{/* 카드 제목 아래 */}
{todo.userName && (
  <div className="text-xs text-gray-500 flex items-center gap-1">
    <User className="h-3 w-3" />
    <span>{todo.userName}</span>
  </div>
)}
```

## 단계 2: 알림 전체보기/바로가기/미리보기 버튼

### 현황 분석
- NotificationBell: 알림 항목 전체 클릭 → `handleNotificationClick()` → `window.location.href` 이동
- 드롭다운에 최근 10건만 표시되며, 전체 알림 목록으로 이동하는 경로 없음
- 로비챗에 `NotificationChatView` 컴포넌트가 전체 알림 리스트를 렌더링하는 기능 이미 존재 (`NOTIFICATION_CHANNEL_ID = '__NOTIFICATION_CENTER__'`)
- 라우팅: `ntfResourceType` 기반 switch 문 (TODO/ISSUE/MEETING_NOTE/CALENDAR/TALK)
- 리소스 상세 API: `useTodoDetail()`, `useIssueDetail()` 등 이미 존재

### 작업 내용

#### 2-0. NotificationBell.tsx — 전체 알림보기 버튼 추가
- 알림 목록 하단(footer)에 "전체 알림보기" 링크 버튼 추가
- 클릭 시: 드롭다운 닫고 `/amoeba-talk?channel=__NOTIFICATION_CENTER__` 로 이동
- 로비챗 알림센터(`NotificationChatView`)에서 전체 알림 리스트 확인 가능

#### 2-1. NotificationBell.tsx — 바로가기/미리보기 버튼 분리
- 기존: 알림 항목 전체 클릭 → 페이지 이동
- 변경: 알림 항목에 2개 버튼 추가
  - 🔗 **바로가기**: 기존 라우팅 로직 수행 (handleNotificationClick)
  - 👁️ **미리보기**: 사이드 패널 열기 (setPreviewNotification)
- 항목 본체 클릭: 읽음 처리 + 미리보기 (가장 자연스러운 동작)


#### 2-2. NotificationPreviewPanel.tsx — 미리보기 사이드 패널 신규 생성
- **위치**: 화면 오른쪽에서 슬라이드인 (width: 400~480px)
- **구조**:
  - 헤더: 리소스 타입 라벨 + 바로가기 링크 + X 닫기 버튼
  - 바디: 리소스 유형별 상세 내용

- **리소스별 미리보기 컨텐츠**:

| 리소스 | 표시 항목 |
|--------|----------|
| TODO | 제목, 설명, 상태, 마감일, 태그, 댓글 목록 |
| ISSUE | 제목, 설명, 상태, 우선순위, 담당자, 댓글 목록 |

| CALENDAR | 이벤트명, 일시, 장소, 참석자 |

- **데이터 페칭**: 기존 서비스 API 활용
  - TODO → `todoService.getTodoDetail(id)`
  - ISSUE → `issueService.getIssue(id)`
  - MEETING_NOTE → `meetingNoteService.getMeetingNote(id)`

## 단계 3: 메시지 삭제 UX 통합 개선

### 현황 분석
- 현재 두 개의 삭제 관련 아이콘이 분리되어 있음:
  1. 🗑️ (Trash2): 본인 메시지 → 직접 `handleDelete()` 호출 (confirm 포함)
  2. 👁️‍🗨️ (EyeOff): 드롭다운 토글 → "나만 삭제" / "모두에게 삭제"
- 1시간 이후: "모두에게 삭제" 버튼 사라짐 (안내 메시지 없음)

### 작업 내용

#### 3-1. TalkMessageList.tsx — 삭제 UI 통합
- **제거**: 독립 Trash2 아이콘 버튼 (직접 handleDelete)
- **통합**: EyeOff 아이콘 하나만 남기고, 삭제 드롭다운으로 통일
- **드롭다운 아이콘**: Trash2로 변경 (삭제 의미 명확)

#### 3-2. 시간 기반 삭제 옵션 분기

```
[본인 메시지, 1시간 이내]
┌─────────────────────────────────┐
│ 👁️‍🗨️ 내 대화창에서 삭제         │
│ 🗑️ 모두에게 삭제                │
└─────────────────────────────────┘

[본인 메시지, 1시간 이후]
┌─────────────────────────────────┐
│ ⓘ 발송 후 1시간이 지나 모두에게  │
│   삭제할 수 없습니다             │
│ ─────────────────────────       │
│ 👁️‍🗨️ 내 대화창에서 삭제         │
└─────────────────────────────────┘

[타인 메시지]
┌─────────────────────────────────┐
│ 👁️‍🗨️ 내 대화창에서 삭제         │
└─────────────────────────────────┘
```

#### 3-3. 구현 상세

```tsx
{/* 통합 삭제 드롭다운 */}
<div className="relative">
  <button onClick={() => toggleDeleteDropdown(msg.id)}>
    <Trash2 className="h-3.5 w-3.5" />
  </button>
  {deleteDropdownId === msg.id && (
    <div className="dropdown">
      {/* 본인 메시지 + 1시간 이후: 안내 메시지 */}
      {isOwn && !canDeleteForAll(msg) && (
        <div className="px-3 py-2 text-xs text-gray-400 italic border-b">
          <Info className="inline h-3 w-3 mr-1" />
          {t('talk:deleteExpiredNotice')}
        </div>
      )}
      {/* 내 대화창에서 삭제 (항상) */}
      <button onClick={() => handleHide(msg.id)}>
        <EyeOff className="h-3 w-3" />
        {t('talk:deleteForMe')}
      </button>
      {/* 모두에게 삭제 (본인 + 1시간 이내) */}
      {canDeleteForAll(msg) && (
        <button onClick={() => handleDelete(msg.id)} className="text-red-600">
          <Trash2 className="h-3 w-3" />
          {t('talk:deleteForAll')}
        </button>
      )}
    </div>
  )}
</div>
```

#### 3-4. i18n 번역 키 추가/수정
- `talk:deleteExpiredNotice` — "발송 후 1시간이 지나 모두에게 삭제할 수 없습니다" / "Cannot delete for everyone after 1 hour" / "Không thể xoá cho tất cả sau 1 giờ"
- 기존 키 유지: `talk:deleteForMe`, `talk:deleteForAll`, `talk:deleteMessage`

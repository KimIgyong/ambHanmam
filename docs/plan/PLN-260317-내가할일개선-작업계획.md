#### Step 1-1. DB 마이그레이션

**목적**: `tdo_start_date` 컬럼 추가, `tdo_due_date` NULL 허용 변경

```sql
-- 1) 시작일 컬럼 추가
ALTER TABLE amb_todos
  ADD COLUMN IF NOT EXISTS tdo_start_date DATE;

-- 2) 마감일 NULL 허용
ALTER TABLE amb_todos
  ALTER COLUMN tdo_due_date DROP NOT NULL;

-- 3) 인덱스 생성
CREATE INDEX IF NOT EXISTS idx_todos_start_date ON amb_todos (tdo_start_date) WHERE tdo_deleted_at IS NULL;
CREATE INDEX IF NOT EXISTS idx_todos_due_date ON amb_todos (tdo_due_date) WHERE tdo_deleted_at IS NULL;
CREATE INDEX IF NOT EXISTS idx_todos_completed ON amb_todos (tdo_completed_at) WHERE tdo_deleted_at IS NULL;
```

#### Step 1-2. computeTodoStatus() 유틸 구현

**목적**: 날짜 기반 상태 자동 계산 함수 신규 작성

**신규 파일**: `apps/api/src/domain/todo/utils/compute-todo-status.util.ts`

**구현 내용**:
```typescript
// 타입 정의
export type TodoComputedStatus =
  | 'COMPLETED' | 'OVERDUE' | 'DUE_TODAY'
  | 'IN_PROGRESS' | 'UPCOMING_SOON' | 'UPCOMING';

// 핵심 함수
export function computeTodoStatus(todo, today, upcomingSoonDays = 3): TodoComputedStatus

// 보조 함수 — D-day 계산
export function computeDaysFromToday(date, today): number | null
```

**판단 트리**:
1. `completed_at IS NOT NULL` → COMPLETED
2. `due_date < today` → OVERDUE
3. `due_date = today` → DUE_TODAY
4. `start_date ≤ today (or null)` → IN_PROGRESS
5. `start_date ≤ today + 3일` → UPCOMING_SOON
6. 그 외 → UPCOMING

**사이드 임팩트**: 없음 — 신규 유틸 파일, 기존 코드에 영향 없음

#### Step 1-3. 공유 타입 수정

**목적**: `TodoComputedStatus` 타입을 공유 패키지에 추가, `TodoResponse` 인터페이스 확장

**변경 파일**: `packages/types/src/domain.types.ts`

**변경 내용**:
```typescript
// 기존 유지
export const TODO_STATUS = { SCHEDULED, IN_PROGRESS, COMPLETED };

// 신규 추가
export const TODO_COMPUTED_STATUS = {
  COMPLETED: 'COMPLETED',
  OVERDUE: 'OVERDUE',
  DUE_TODAY: 'DUE_TODAY',
  IN_PROGRESS: 'IN_PROGRESS',
  UPCOMING_SOON: 'UPCOMING_SOON',
  UPCOMING: 'UPCOMING',
};
export type TodoComputedStatus = typeof TODO_COMPUTED_STATUS[keyof typeof TODO_COMPUTED_STATUS];

// TodoResponse 확장
export interface TodoResponse {
  // ... 기존 필드 유지
  startDate: string | null;           // 신규 — 시작 예정일 (ISO date)
  computedStatus: TodoComputedStatus; // 신규 — 자동 계산 상태
  daysUntilDue: number | null;       // 신규 — 음수=지연, 0=오늘, 양수=잔여일
  daysUntilStart: number | null;     // 신규 — UPCOMING_SOON/UPCOMING 전용
}
```

**사이드 임팩트**: TodoResponse에 필드 추가이므로 기존 프론트엔드 코드에서 오류 없음 (optional 추가)

#### Step 1-5. Mapper 수정

**목적**: TodoEntity → TodoResponse 매핑에 computed status 추가

**변경 파일**: `todo.mapper.ts`

**변경 내용**:
```typescript
static toResponse(entity: TodoEntity): TodoResponse {
  // 기존 매핑 유지 + 추가
  const computedStatus = computeTodoStatus({
    tdo_start_date: entity.tdoStartDate,
    tdo_due_date: entity.tdoDueDate,
    tdo_completed_at: entity.tdoCompletedAt,
  });
  return {
    ...기존필드,
    startDate: entity.tdoStartDate?.toISOString()?.split('T')[0] || null,
    computedStatus,
    daysUntilDue: computeDaysFromToday(entity.tdoDueDate),
    daysUntilStart: computeDaysFromToday(entity.tdoStartDate),
    status: entity.tdoStatus,  // 레거시 호환 유지
  };
}
```

**사이드 임팩트**: 모든 Todo 응답에 `computedStatus` 추가 — 프론트에서 점진적 전환 가능

#### Step 1-7. TodayService 쿼리 전환

**목적**: Today 페이지의 할일 쿼리를 `tdo_status` 기반 → 날짜 기반으로 전환

**변경 파일**: `today.service.ts`

**변경 내용**:

| 쿼리 | 현재 | 변경 후 |
|------|------|---------|
| 오늘 마감 | `WHERE tdo_due_date = today AND tdo_status != 'COMPLETED'` | `WHERE tdo_due_date = today AND tdo_completed_at IS NULL` |

**사이드 임팩트**: Today 카드뷰의 todayDueCount, overdueCount 등 집계값이 날짜 기반으로 정확해짐

## 6. 테스트 계획

### 6.1 백엔드 단위 테스트

| 테스트 | 검증 내용 |
|--------|----------|
| `computeTodoStatus()` | 6개 상태 × 엣지 케이스 (null 날짜, 경계일, 과거/미래) |
| `computeDaysFromToday()` | 양수/음수/0 반환 검증 |
| Mapper | computed status + daysUntilDue/Start 정확성 |
| Validation | `start_date > due_date` 오류 반환 |

### 6.2 통합 테스트

| 시나리오 | 검증 내용 |
|----------|----------|
| 할일 생성 (시작일만) | computed status = IN_PROGRESS or UPCOMING |
| 할일 생성 (마감일만) | computed status = DUE_TODAY or IN_PROGRESS |
| 할일 생성 (둘 다 없음) | computed status = IN_PROGRESS |
| 완료 토글 | `completed_at` 설정/해제 + 반복 할일 자동 생성 |
| 날짜 경과 | 동일 할일이 UPCOMING → IN_PROGRESS → DUE_TODAY → OVERDUE 자동 전환 |

### 6.3 프론트 수동 테스트

| 시나리오 | 검증 내용 |
|----------|----------|
| 5섹션 렌더링 | 각 섹션에 올바른 할일 분류 |
| 배지 표시 | D+N, D-Day, D-N, N일 후 시작 정확성 |
| 폼 — 상태 제거 | 상태 드롭다운 없이 생성/수정 가능 |
| 폼 — 시작일 | 날짜 선택 동작, 비우기 가능 |
| 체크박스 토글 | 완료/되돌리기 동작 |
| UPCOMING 접힘 | 기본 접힘 → 토글 펼침 |

# 업무지원 - 설계문서 관리 기능명세서

## 1. 개요
시스템 설계 항목별 진행 상태(준비→설계→개발→검증→완성)를 트리 구조로 계층적으로 관리합니다.

## 2. 화면 구성도 (Wireframe)
```
┌──────────────────────────────────────────────────────────────────────┐
│  설계문서 관리                                                          │
│  설계서 종류: [화면설계서▼]                                             │
├────────────────────┬─────────────────────────────────────────────────┤
│  [트리 네비게이션]  │  [우측 상태 요약 패널]                            │
│                    │                                                   │
│  ▼ 시스템 전체     │  등록: 45  오류: 2  설계중: 10                    │
│    ▼ 경영정보      │  개발중: 15  검증중: 5  완성: 13                   │
│      ▷ 회의관리    │                                                   │
│      ▷ 프로젝트    │  ──────────────────────────────                  │
│    ▼ 영업업무      │  선택된 항목: 경영정보 > 회의관리                   │
│      ▷ 고객사관리  │  상태: 개발중 ●                                    │
│                    │  담당자: 김철수                                     │
└────────────────────┴─────────────────────────────────────────────────┘
```

## 3. 입력 필드
| 필드명 | 유형 | 설명 |
|--------|------|------|
| 설계서 종류 | Select | 화면설계서, DB설계서, API명세 등 |
| 트리 노드 | Click | 항목 선택 |

## 4. 출력 필드
| 영역 | 설명 |
|------|------|
| 트리 | 계층형 기능 목록 |
| 상태 요약 | 등록/오류/설계중/개발중/검증중/완성 건수 |
| 우측 패널 | 선택 항목 상세 상태, 담당자 |

## 5. 액션 버튼
- 트리 `[+]`/`[-]` : 확장/축소
- 상태 수치 클릭 → 해당 상태 항목 필터링 목록

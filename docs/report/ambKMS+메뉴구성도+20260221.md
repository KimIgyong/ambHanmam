## 6. 메뉴 권한 DB 구조

메뉴 권한 관리를 위한 3개의 테이블로 구성된다.

### 6.1 테이블 ERD

```
amb_menu_config (메뉴 설정)
├── 1:N ── amb_menu_permissions (메뉴별 역할 권한)
└── 1:N ── amb_user_menu_permissions (사용자별 메뉴 권한)
```

### 6.2 amb_menu_config (메뉴 설정 마스터)

메뉴의 기본 구성 정보를 관리하는 마스터 테이블이다.

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|---------|------|
| `mnc_id` | UUID | PK, DEFAULT uuid_generate_v4() | 메뉴 설정 ID |
| `mnc_code` | VARCHAR(50) | UNIQUE, NOT NULL | 메뉴 코드 (예: `CHAT_MANAGEMENT`) |
| `mnc_label_key` | VARCHAR(100) | NOT NULL | i18n 번역 키 |
| `mnc_icon` | VARCHAR(50) | NOT NULL | lucide-react 아이콘명 |
| `mnc_path` | VARCHAR(200) | NOT NULL | 라우팅 경로 |
| `mnc_category` | VARCHAR(20) | NOT NULL | 카테고리 (`chat`, `main`, `settings`) |
| `mnc_color` | VARCHAR(30) | NULL | Tailwind 색상 클래스 |
| `mnc_sort_order` | INTEGER | NOT NULL, DEFAULT 0 | 정렬 순서 |
| `mnc_is_active` | BOOLEAN | NOT NULL, DEFAULT true | 활성화 여부 |
| `mnc_parent_id` | UUID | FK → amb_menu_config.mnc_id, NULL | 상위 메뉴 ID |
| `mnc_entity_id` | UUID | FK → amb_entities.ent_id, NOT NULL | 법인 ID |
| `mnc_created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW() | 생성일시 |
| `mnc_updated_at` | TIMESTAMP | NOT NULL, DEFAULT NOW() | 수정일시 |
| `mnc_deleted_at` | TIMESTAMP | NULL | 삭제일시 (Soft Delete) |

**인덱스**:
- `IDX_menu_config_code` — `mnc_code` (UNIQUE)
- `IDX_menu_config_category` — `mnc_category`
- `IDX_menu_config_entity` — `mnc_entity_id`
- `IDX_menu_config_sort` — `mnc_sort_order`

### 6.3 amb_menu_permissions (메뉴별 역할 권한)

역할(Role) 기반의 메뉴 접근 권한을 정의한다.

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|---------|------|
| `mnp_id` | UUID | PK, DEFAULT uuid_generate_v4() | 메뉴 권한 ID |
| `mnp_menu_id` | UUID | FK → amb_menu_config.mnc_id, NOT NULL | 메뉴 ID |
| `mnp_role` | VARCHAR(20) | NOT NULL | 역할 (`ADMIN`, `MANAGER`, `USER`) |
| `mnp_access_level` | VARCHAR(10) | NOT NULL | 접근 수준 (`FULL`, `READ`, `NONE`) |
| `mnp_entity_id` | UUID | FK → amb_entities.ent_id, NOT NULL | 법인 ID |
| `mnp_created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW() | 생성일시 |
| `mnp_updated_at` | TIMESTAMP | NOT NULL, DEFAULT NOW() | 수정일시 |

**인덱스**:
- `IDX_menu_perm_menu_role` — `(mnp_menu_id, mnp_role)` (UNIQUE)
- `IDX_menu_perm_entity` — `mnp_entity_id`

**접근 수준 정의**:

| access_level | 설명 | 매핑 |
|-------------|------|------|
| `FULL` | 읽기/쓰기 모두 가능 | O |
| `READ` | 읽기 전용 | R |
| `NONE` | 접근 불가 | X |

### 6.4 amb_user_menu_permissions (사용자별 메뉴 권한)

역할 기반 권한을 사용자 단위로 오버라이드할 때 사용한다.

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|---------|------|
| `ump_id` | UUID | PK, DEFAULT uuid_generate_v4() | 사용자 메뉴 권한 ID |
| `ump_user_id` | UUID | FK → amb_users.usr_id, NOT NULL | 사용자 ID |
| `ump_menu_id` | UUID | FK → amb_menu_config.mnc_id, NOT NULL | 메뉴 ID |
| `ump_access_level` | VARCHAR(10) | NOT NULL | 접근 수준 (`FULL`, `READ`, `NONE`) |
| `ump_granted_by` | UUID | FK → amb_users.usr_id, NOT NULL | 권한 부여자 ID |
| `ump_expires_at` | TIMESTAMP | NULL | 권한 만료일시 (NULL이면 무기한) |
| `ump_entity_id` | UUID | FK → amb_entities.ent_id, NOT NULL | 법인 ID |
| `ump_created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW() | 생성일시 |
| `ump_updated_at` | TIMESTAMP | NOT NULL, DEFAULT NOW() | 수정일시 |

**인덱스**:
- `IDX_user_menu_perm_user_menu` — `(ump_user_id, ump_menu_id)` (UNIQUE)
- `IDX_user_menu_perm_user` — `ump_user_id`
- `IDX_user_menu_perm_entity` — `ump_entity_id`

## 7. 프론트엔드 메뉴 가드 체계

프론트엔드에서 메뉴 접근을 제어하는 가드 시스템을 설명한다.

### 7.1 가드 아키텍처 개요

```
Router
 └── AuthGuard (인증 확인)
      └── MenuGuard (메뉴 권한 확인)
           ├── ChatMenuGuard (채팅 메뉴 전용 가드)
           └── SettingsMenuGuard (설정 메뉴 전용 가드)
```

### 7.2 MenuGuard 컴포넌트

일반 메뉴에 대한 접근 권한을 확인하는 래퍼 컴포넌트이다.

```tsx
// apps/web/src/components/guards/MenuGuard.tsx

interface MenuGuardProps {
  menuCode: string;
  children: React.ReactNode;
  fallback?: React.ReactNode;
  requiredLevel?: 'FULL' | 'READ';
}

const MenuGuard: React.FC<MenuGuardProps> = ({
  menuCode,
  children,
  fallback = <AccessDeniedPage />,
  requiredLevel = 'READ',
}) => {
  const { user } = useAuth();
  const { checkMenuAccess } = useMenuPermissions();

  const accessLevel = checkMenuAccess(menuCode, user.role);

  // 사용자별 오버라이드 확인
  const userOverride = useUserMenuPermission(user.id, menuCode);
  const effectiveLevel = userOverride?.accessLevel ?? accessLevel;

  if (effectiveLevel === 'NONE') {
    return <>{fallback}</>;
  }

  if (requiredLevel === 'FULL' && effectiveLevel === 'READ') {
    return <>{fallback}</>;
  }

  return <>{children}</>;
};
```

### 7.3 ChatMenuGuard 컴포넌트

채팅 메뉴에 대한 접근 권한을 확인하며, 부서별 에이전트 접근을 추가로 제어한다.

```tsx
// apps/web/src/components/guards/ChatMenuGuard.tsx

interface ChatMenuGuardProps {
  chatCode: string;          // 예: 'CHAT_ACCOUNTING'
  children: React.ReactNode;
  fallback?: React.ReactNode;
}

const ChatMenuGuard: React.FC<ChatMenuGuardProps> = ({
  chatCode,
  children,
  fallback = <ChatAccessDeniedPage />,
}) => {
  const { user } = useAuth();
  const { checkMenuAccess } = useMenuPermissions();
  const { agents } = useAgentStore();

  // 1단계: 메뉴 접근 권한 확인
  const accessLevel = checkMenuAccess(chatCode, user.role);
  if (accessLevel === 'NONE') {
    return <>{fallback}</>;
  }

  // 2단계: 해당 에이전트 활성화 여부 확인
  const agentType = chatCodeToAgentType(chatCode);
  const agent = agents.find(a => a.type === agentType);
  if (!agent || !agent.isActive) {
    return <AgentInactivePage agentType={agentType} />;
  }

  // 3단계: 읽기 전용 모드 판별
  const isReadOnly = accessLevel === 'READ';

  return (
    <ChatContext.Provider value={{ isReadOnly }}>
      {children}
    </ChatContext.Provider>
  );
};
```

### 7.4 useMenuPermissions 훅

메뉴 권한 상태를 관리하는 커스텀 훅이다.

```tsx
// apps/web/src/hooks/useMenuPermissions.ts

interface UseMenuPermissionsReturn {
  menuPermissions: MenuPermission[];
  checkMenuAccess: (menuCode: string, role: string) => AccessLevel;
  getAccessibleMenus: (role: string, category?: string) => MenuConfig[];
  isLoading: boolean;
}

const useMenuPermissions = (): UseMenuPermissionsReturn => {
  const { data: menuPermissions, isLoading } = useQuery({
    queryKey: ['menuPermissions'],
    queryFn: () => menuApi.getPermissions(),
    staleTime: 5 * 60 * 1000, // 5분 캐시
  });

  const checkMenuAccess = useCallback(
    (menuCode: string, role: string): AccessLevel => {
      const permission = menuPermissions?.find(
        p => p.menuCode === menuCode && p.role === role,
      );
      return permission?.accessLevel ?? 'NONE';
    },
    [menuPermissions],
  );

  const getAccessibleMenus = useCallback(
    (role: string, category?: string): MenuConfig[] => {
      return menuPermissions
        ?.filter(p => p.role === role && p.accessLevel !== 'NONE')
        ?.filter(p => !category || p.menu.category === category)
        ?.map(p => p.menu)
        ?.sort((a, b) => a.sortOrder - b.sortOrder) ?? [];
    },
    [menuPermissions],
  );

  return { menuPermissions: menuPermissions ?? [], checkMenuAccess, getAccessibleMenus, isLoading };
};
```

### 7.7 권한 판별 흐름도

```
사용자 요청
    │
    ▼
[AuthGuard] ── 미인증 ──▶ 로그인 페이지
    │ 인증됨
    ▼
[MenuGuard] ── 사용자별 오버라이드 확인
    │              │
    │          있음 ▼
    │         오버라이드 권한 적용
    │              │
    │          없음 ▼
    │         역할 기반 권한 확인
    │              │
    ▼              ▼
  NONE ──────▶ 접근 거부 페이지 (403)
    │
  READ ──────▶ 읽기 전용 모드
    │
  FULL ──────▶ 전체 접근 허용
```

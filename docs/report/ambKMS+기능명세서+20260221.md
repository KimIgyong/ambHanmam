# AMB Management 기능명세서

- **작성일**: 2026-02-21
- **시스템명**: AMB Management (아메바 경영관리 시스템)
- **버전**: 1.0
- **작성목적**: 전체 시스템 기능 명세 정리
- **기술스택**: React 18 + NestJS 10 + PostgreSQL 15 + Claude AI
- **API Base Path**: `/api/v1`

## 6. 기능별 상세 명세

### FN-AUTH-001: 로그인

- **기능명**: 로그인
- **설명**: 이메일/비밀번호로 JWT 토큰 발급, HttpOnly 쿠키로 토큰 설정
- **선행조건**: 유효한 계정 존재
- **후행조건**: access_token, refresh_token 쿠키 설정
- **API**: POST `/api/v1/auth/login`
  - Request: `{ email, password }`
  - Response: `{ accessToken, refreshToken, user: { userId, email, name, role } }`
- **화면**: `LoginPage.tsx`
- **권한**: Public
- **비즈니스 규칙**: Rate limit 5회/분, bcrypt 비밀번호 비교
- **에러 코드**: E1002 (Invalid credentials)

### FN-AUTH-002: 회원가입

- **기능명**: 회원가입
- **설명**: 새 계정 생성, 초대 토큰 기반 또는 자유 가입
- **선행조건**: 미등록 이메일
- **후행조건**: 계정 생성, 자동 로그인
- **API**: POST `/api/v1/auth/register`
  - Request: `{ email, password, name, department?, invitation_token? }`
  - Response: `{ accessToken, refreshToken, user }`
- **화면**: `RegisterPage.tsx`
- **권한**: Public
- **비즈니스 규칙**: Rate limit 3회/분, 초대 토큰이 있으면 검증 후 역할 부여
- **에러 코드**: E2002 (Email already registered)

### FN-CVS-005: 메시지 전송 (AI 채팅)

- **기능명**: AI 에이전트에게 메시지 전송
- **설명**: 대화에 메시지를 전송하면 부서별 AI 에이전트가 SSE 스트리밍으로 응답
- **선행조건**: 유효한 대화 존재, 사용자가 대화 소유자
- **후행조건**: 사용자 메시지 + AI 응답 메시지 저장
- **API**: POST `/api/v1/conversations/:id/messages`
  - Request: `{ content }`
  - Response: SSE 스트림 (`data: { text?, done?, error? }`)
- **화면**: `ChatPage.tsx`, `MessageInput.tsx`, `MessageList.tsx`
- **권한**: USER
- **비즈니스 규칙**: Accept-Language 헤더로 AI 응답 언어 제어, 부서별 시스템 프롬프트 적용
- **에러 코드**: E3001 (Conversation not found), E3002 (Access denied), E9002 (AI service error)

## 7. 비기능 요구사항

### 7.1 보안

| 항목 | 설명 |

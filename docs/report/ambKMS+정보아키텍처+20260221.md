# AMB Management 정보 아키텍처 (Information Architecture)

**작성일**: 2026-02-21
**시스템**: AMB Management - 아메바 회사 종합 관리 시스템

---

## 1. 시스템 개요

AMB Management는 아메바 회사의 9개 부서별 전문 AI 에이전트를 갖춘 종합 경영 관리 시스템이다.
- 기술 스택: React 18 + NestJS 10 + PostgreSQL 15 + Anthropic Claude API
- 모노레포: apps/api (백엔드), apps/web (프론트엔드), packages/types, packages/common
- 실시간: SSE (Server-Sent Events)
- 다국어: EN, KO, VI (25개 i18n 네임스페이스)

## 5. 콘텐츠 유형 (Content Types)

| 콘텐츠 | 형식 | 가시성 | 영속성 |
|--------|------|--------|--------|

| 할일 | 텍스트 + 태그 | 개인/팀 | Soft Delete |
| 송장/계약 | 구조화 데이터 | 엔티티 단위 | 영구 |
| 직원 정보 | 구조화 데이터 | 엔티티 단위 | Soft Delete |

| 아메바톡 | 텍스트/파일/시스템 | 채널 멤버 | 영구 |

## 6. 네비게이션 패턴

### 6.1 글로벌 네비게이션
- **사이드바 메뉴**: 권한 기반 동적 생성, 축소/확장 가능
- **글로벌 검색**: 모듈별 통합 검색 (상단 헤더)
- **엔티티 선택기**: 다중 법인 전환
- **언어 선택기**: EN/KO/VI 즉시 전환

### 6.2 로컬 네비게이션
- **서브 레이아웃**: HR, Billing, Mail, Project, Service — 좌측 서브메뉴
- **채팅 레이아웃**: 좌측 대화 목록 + 우측 채팅 화면
- **KMS**: 상단 탭 (태그 클라우드, 태그 관리, 지식 그래프)

### 6.3 컨텍스트 네비게이션
- **AI 어시스턴트**: 하단 우측 FAB 버튼 → 모달 (환영 → 부서 선택 → 채팅)
- **브레드크럼**: 목록 → 상세 → 편집 패턴

### 6.4 모바일 적응
- **768px 이하**: 햄버거 메뉴 → 드로어 네비게이션
- **서브레이아웃**: 수평 스크롤 탭
- **채팅**: 목록 ↔ 채팅 토글 (뒤로가기 버튼)
- **AI 어시스턴트**: 전체 화면

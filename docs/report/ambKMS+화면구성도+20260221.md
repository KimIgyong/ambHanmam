## 1. 레이아웃 구조

시스템은 4가지 주요 레이아웃으로 구성되며, 사용자 인증 상태와 현재 위치에 따라 적절한 레이아웃이 적용된다.

### 1.1 MainLayout (인증 후 기본 레이아웃)

로그인된 사용자에게 표시되는 기본 레이아웃이다. 좌측 사이드바와 상단 헤더, 중앙 콘텐츠 영역으로 구성된다.

```
┌──────────────────────────────────────────────────────────────┐
│ [☰] AMB        [🔍 검색바]        [엔티티] [언어] [이름] [↪] │ ← 헤더 (h-14/h-16)
├──────────┬───────────────────────────────────────────────────┤
│          │                                                   │
│  할일    │                                                   │
│  에이전트│            메인 콘텐츠 영역                         │
│  회의    │            (React Router Outlet)                   │
│  아메바  │                                                   │
│  근무    │                                                   │
│  공지    │                                                   │
│  문서    │                                                   │
│  회계    │                                                   │
│  HR      │                                                   │
│  청구    │                                             [🤖]  │ ← AI FAB 버튼
│  메일    │                                                   │
│  KMS     │                                                   │
│  ────────│                                                   │
│  My      │                                                   │
│  설정    │                                                   │
│          │                                                   │
└──────────┴───────────────────────────────────────────────────┘
   w-60                           flex-1
 (축소 시 w-16,
  아이콘만 표시)
```

**헤더 구성 요소:**
| 영역 | 요소 | 설명 |
|------|------|------|
| 좌측 | 햄버거 메뉴 [☰] | 사이드바 축소/확장 토글 |
| 좌측 | AMB 로고 | 클릭 시 대시보드 이동 |
| 중앙 | 검색바 | 전체 시스템 검색 (Ctrl+K) |
| 우측 | 엔티티 선택 | 소속 엔티티(법인) 전환 |
| 우측 | 언어 선택 | ko / en / vi 전환 |
| 우측 | 사용자 이름 | 프로필 드롭다운 |
| 우측 | 로그아웃 [↪] | 세션 종료 |

**사이드바 메뉴 구조:**
| 메뉴 | 라우트 | 아이콘 | 비고 |
|------|--------|--------|------|
| 할일 | /todos | ☑ | 배지: 미완료 건수 |
| 에이전트 | /chat | 💬 | 배지: 안읽은 메시지 |
| 회의 | /meetings | 📅 | - |
| 아메바 | /amoeba | 🏢 | - |
| 근무 | /attendance | ⏰ | - |
| 공지 | /notices | 📢 | 배지: 새 공지 |
| 문서 | /documents | 📄 | - |
| 회계 | /accounting | 💰 | 서브메뉴 |
| HR | /hr | 👥 | 서브메뉴 |
| 청구 | /billing | 🧾 | 서브메뉴 |
| 메일 | /mail | ✉️ | 배지: 안읽은 메일 |
| KMS | /kms | 📚 | - |
| ── 구분선 ── | | | |
| My | /my | 👤 | 개인 설정 |
| 설정 | /settings | ⚙️ | 관리자 전용 |

### 1.2 SubMenuLayout - Chat (에이전트 대화)

에이전트(Claude AI) 대화 화면은 좌측에 대화 목록, 우측에 채팅 영역으로 분할된다.

```
┌──────────────────────────────┬───────────────────────────────────┐
│ [+ 새 대화]                  │  에이전트 이름    [📋] [⚙️]       │
│──────────────────────────────│───────────────────────────────────│
│                              │                                   │
│ ● 대화 제목 1         2/20  │  ┌─────────────────────────────┐  │
│   마지막 메시지 미리보기...   │  │ 🤖 에이전트 응답              │  │
│                              │  │ 마크다운 렌더링 영역          │  │
│   대화 제목 2         2/19  │  │ (코드블록, 테이블, 목록 등)   │  │
│   마지막 메시지 미리보기...   │  └─────────────────────────────┘  │
│                              │                                   │
│   대화 제목 3         2/18  │  ┌─────────────────────────────┐  │
│   마지막 메시지 미리보기...   │  │ 👤 사용자 메시지              │  │
│                              │  └─────────────────────────────┘  │
│                              │                                   │
│                              │  ┌─────────────────────────────┐  │
│                              │  │ 🤖 스트리밍 응답 중...  ███▒  │  │
│                              │  │ (SSE 실시간 렌더링)          │  │
│                              │  └─────────────────────────────┘  │
│                              │                                   │
│                              │ ┌─────────────────────────┬───┐  │
│                              │ │ 메시지를 입력하세요...    │ ⬆ │  │
│                              │ └─────────────────────────┴───┘  │
└──────────────────────────────┴───────────────────────────────────┘
          w-280 (고정)                        flex-1
```

**대화 목록 요소:**
- 새 대화 생성 버튼
- 날짜별 그룹핑 (오늘 / 어제 / 이번 주 / 이전)
- 대화 항목: 제목 + 날짜 + 미리보기
- 우클릭 컨텍스트 메뉴: 이름 변경 / 삭제

**채팅 영역 요소:**
- 메시지 버블 (사용자 / 에이전트 구분)
- 마크다운 렌더링 (코드 하이라이팅, 테이블, 수식)
- SSE 스트리밍 애니메이션
- 입력창: 자동 높이 조절 (textarea)
- 전송 버튼 / Enter 전송 (Shift+Enter 줄바꿈)

### 1.3 SubMenuLayout - 서브 레이아웃 (HR, Billing, Mail, Project, Service)

좌측 서브 탭 메뉴와 우측 콘텐츠로 구성되는 2단 레이아웃이다.

```
┌─────────────┬────────────────────────────────────────────────┐
│             │                                                │
│  서브 메뉴   │                                                │
│  (세로 탭)   │           서브 콘텐츠 영역                      │
│             │           (Nested Outlet)                      │
│  ● 메뉴1    │                                                │
│    메뉴2    │                                                │
│    메뉴3    │                                                │
│    메뉴4    │                                                │
│             │                                                │
└─────────────┴────────────────────────────────────────────────┘
     w-48                          flex-1
```

**적용 대상:**
| 모듈 | 서브 메뉴 항목 |
|------|---------------|
| HR | 직원 목록, 계약 관리, 급여, 근태 |
| 청구 | 인보이스, 결제, 매출 리포트 |
| 메일 | 받은메일함, 보낸메일함, 임시저장, 휴지통 |
| 프로젝트 | 프로젝트 목록, 타임라인, 리소스 |
| 서비스 | 서비스 목록, 구독 관리, 모니터링 |

### 1.4 AuthLayout (로그인 / 회원가입)

인증 전 사용자에게 표시되는 중앙 정렬 레이아웃이다.

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│                          AMB                                 │
│                    아메바 관리 시스템                           │
│                                                              │
│              ┌──────────────────────────┐                    │
│              │                          │                    │
│              │        로그인             │                    │
│              │                          │                    │
│              │  ┌────────────────────┐  │                    │
│              │  │ 📧 이메일           │  │                    │
│              │  └────────────────────┘  │                    │
│              │                          │                    │
│              │  ┌────────────────────┐  │                    │
│              │  │ 🔒 비밀번호         │  │                    │
│              │  └────────────────────┘  │                    │
│              │                          │                    │
│              │  ☐ 로그인 상태 유지      │                    │
│              │                          │                    │
│              │  ┌────────────────────┐  │                    │
│              │  │      로그인        │  │  ← primary 버튼    │
│              │  └────────────────────┘  │                    │
│              │                          │                    │
│              │  계정이 없으신가요?       │                    │
│              │  → 회원가입              │                    │
│              │                          │                    │
│              └──────────────────────────┘                    │
│                       max-w-md                               │
│                                                              │
│              ⓒ 2026 Amoeba Inc.                              │
└──────────────────────────────────────────────────────────────┘
```

**인증 플로우:**
1. 로그인 → 이메일 + 비밀번호 → JWT 발급
2. 회원가입 → 이메일 + 비밀번호 + 이름 + 엔티티 선택
3. 비밀번호 찾기 → 이메일 인증 → 재설정

3. 해당 부서 에이전트와 채팅 시작
4. "전체 채팅으로 이동" 클릭 시 ChatPage로 라우팅
5. 모달 닫기: X 버튼 / 배경 클릭 / ESC 키

```
│  │ └──────────────────────────────────────────────────────┘ ││

```

**폼 필드 유형:**
| 유형 | 컴포넌트 | 용도 |
|------|---------|------|
| Text | `<Input>` | 단일 행 텍스트 |
| Textarea | `<Textarea>` | 여러 줄 텍스트 |
| Select | `<Select>` | 드롭다운 선택 |
| DatePicker | `<DatePicker>` | 날짜 선택 |
| Checkbox | `<Checkbox>` | 체크박스 |
| Radio | `<RadioGroup>` | 라디오 버튼 |
| FileUpload | `<FileUpload>` | 파일 업로드 |
| TagInput | `<TagInput>` | 태그 입력 (이메일 수신자 등) |

**폼 상태:**
- 기본: 회색 테두리
- 포커스: primary 색상 테두리 + 그림자
- 에러: 빨간 테두리 + 에러 메시지
- 비활성: 회색 배경, 입력 불가
- 필수: 라벨 옆 * 표시 (빨간색)

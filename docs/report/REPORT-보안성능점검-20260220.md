## 2. CRITICAL 발견사항 (즉시 조치 필요)

### 2.1 하드코딩된 암호화 시크릿 키
- **파일**: `apps/api/src/domain/settings/service/crypto.service.ts:17-21`
- **내용**: `API_KEY_ENCRYPTION_SECRET` 환경변수 미설정 시 기본값 `'amb_api_key_encryption_32byte_k'`이 사용됨
- **영향**: 소스코드에 노출된 기본 키로 암호화된 모든 API 키가 복호화 가능
- **조치**: 프로덕션 환경에서 강력한 랜덤 시크릿 필수 설정, 기본값 제거

### 2.2 하드코딩된 JWT 시크릿
- **파일**: `apps/api/src/domain/auth/strategy/jwt.strategy.ts:29`, `apps/api/src/domain/auth/auth.module.ts:37`
- **내용**: `JWT_SECRET` 미설정 시 기본값 `'amb_jwt_secret'` 사용
- **영향**: 기본 시크릿을 알면 누구나 유효한 JWT 토큰 위조 가능 (인증 우회)
- **조치**: 기본값 제거, 환경변수 미설정 시 서버 시작 실패 처리

### 2.3 하드코딩된 DB 크레덴셜
- **파일**: `apps/api/src/app.module.ts:129-130`
- **내용**: DB 사용자명/비밀번호 기본값 `'amb_user'`/`'amb_password'`
- **영향**: 소스코드에서 DB 접근 크레덴셜 노출
- **조치**: 기본값 제거, 시크릿 매니저(AWS Secrets Manager, Vault 등) 사용

## 3. HIGH 발견사항 (조기 조치 필요)

### 3.1 localStorage 토큰 저장 (XSS 취약)
- **파일**: `apps/web/src/domain/auth/store/auth.store.ts:46-51`
- **내용**: accessToken, refreshToken을 localStorage에 저장 (Zustand persist)
- **영향**: XSS 공격으로 JavaScript가 실행되면 토큰 탈취 가능
- **조치**: httpOnly + secure + sameSite 쿠키로 전환

### 3.2 토큰 해지(Revocation) 메커니즘 부재
- **파일**: `apps/api/src/domain/auth/controller/user.controller.ts`
- **내용**: 로그아웃 엔드포인트 없음, 토큰 블랙리스트 없음
- **영향**: 탈취된 토큰은 만료까지 유효 (access 15분, refresh 7일)
- **조치**: Redis 기반 토큰 블랙리스트 구현, 로그아웃 API 추가

### 3.3 N+1 쿼리 - 계정 잔액 조회
- **파일**: `apps/api/src/domain/accounting/service/accounting.service.ts:26-41`
- **내용**: 각 은행 계정별 `getAccountBalanceInfo()` 개별 호출 (루프 내 N+1)
- **영향**: 계정 수 증가 시 쿼리 수 선형 증가, 응답 시간 저하
- **조치**: 배치 쿼리로 전환 (GROUP BY로 한 번에 조회)

### 3.4 N+1 쿼리 - KMS 배치 동기화
- **파일**: `apps/api/src/domain/kms/service/batch-sync.service.ts:69-112`
- **내용**: 각 todo 항목별 `workItemRepo.findOne()` + `safeAutoTag()` 개별 호출
- **영향**: 대량 동기화 시 쿼리 폭증
- **조치**: 배치 조회 + 벌크 insert 패턴으로 전환

### 3.5 무제한 쿼리 결과 반환 (Pagination 미적용)
- **파일**:
  - `apps/api/src/domain/billing/service/contract.service.ts:63` - 모든 계약 반환
  - `apps/api/src/domain/billing/service/invoice.service.ts:66` - 모든 송장 반환
  - `apps/api/src/domain/notices/service/notice.service.ts:63` - 모든 뉴스 반환
  - `apps/api/src/domain/project/service/project.service.ts:56` - 모든 프로젝트 반환
- **영향**: 데이터 증가 시 메모리 과부하 및 응답 시간 급증
- **조치**: 페이지네이션 필수 적용 (`skip`/`take` + `getManyAndCount`)

## 4. MEDIUM 발견사항

### 4.1 비밀번호 정책 불일치 및 약한 요구사항
- **파일**: `apps/api/src/domain/auth/dto/request/login.request.ts:13` (6자), `reset-password.request.ts:13` (8자)
- **내용**: 로그인/등록은 6자, 비밀번호 재설정은 8자 최소 → 불일치. 복잡성 요구사항 없음
- **조치**: 최소 12자 또는 대/소/숫자/특수문자 복잡성 규칙 통일

### 4.2 리프레시 토큰 로테이션 미구현
- **파일**: `apps/api/src/domain/auth/service/auth.service.ts:151-174`
- **내용**: 새 리프레시 토큰 발급 시 기존 토큰 무효화 없음
- **조치**: 리프레시 토큰 로테이션 구현 (발급 시 기존 토큰 폐기)

### 4.3 세션 타임아웃 미구현
- **파일**: `apps/web/src/domain/auth/store/auth.store.ts`
- **내용**: 유휴 상태 자동 로그아웃 없음, localStorage에 무기한 세션 유지
- **조치**: 프론트엔드 유휴 감지 + 자동 로그아웃 구현

### 4.4 HTML 살균(Sanitization) 미적용
- **파일**:
  - `apps/api/src/domain/webmail/service/mail-send.service.ts:83` - 메일 본문
  - `apps/api/src/domain/notices/service/notice.service.ts:149-150` - 공지사항
  - `apps/api/src/domain/chat/service/chat.service.ts:177-181` - 채팅 메시지
  - `apps/api/src/domain/webmail/service/postal-webhook.service.ts:63-65` - 수신 메일
- **내용**: 사용자/외부 입력 HTML을 그대로 저장 → XSS 위험
- **조치**: DOMPurify 등으로 HTML 살균 후 저장

### 4.5 웹훅 DTO 검증 비활성화
- **파일**: `apps/api/src/domain/webmail/controller/webmail.controller.ts:123`
- **내용**: `@UsePipes()`로 ValidationPipe 비활성화 → 비정상 페이로드 수신 가능
- **조치**: 최소한의 페이로드 검증 로직 추가

### 4.6 약한 암호화 키 파생 방식
- **파일**: `apps/api/src/domain/settings/service/crypto.service.ts`
- **내용**: `secret.padEnd(32, '0').slice(0, 32)` → 단순 제로 패딩
- **조치**: PBKDF2 또는 HKDF 키 파생 함수 사용

### 4.7 DB 인덱스 누락
- **대상 엔티티 및 권장 인덱스**:

| 엔티티 | 컬럼 | 권장 인덱스 |
|--------|------|------------|
| User | `usrEmail`, `usrCompanyEmail` | `@Index(['usrEmail'])`, `@Index(['usrCompanyEmail'])` |
| Employee | `entId` + `empStatus` | `@Index(['entId', 'empStatus'])` |
| Invoice | `entId` + `invStatus`, `invDate` | `@Index(['entId', 'invStatus'])`, `@Index(['invDate'])` |
| Contract | `entId` + `ctrStatus` | `@Index(['entId', 'ctrStatus'])` |
| Message | `cvsId` + `msgOrder` | `@Index(['cvsId', 'msgOrder'])` |
| WorkItem | `entId` + `witModule` | `@Index(['entId', 'witModule'])` |
| Notice | `ntcVisibility` + `ntcCreatedAt` | `@Index(['ntcVisibility', 'ntcCreatedAt'])` |

### 4.8 DB 연결 풀 설정 미지정
- **파일**: `apps/api/src/app.module.ts:122-138`
- **내용**: TypeORM 연결 풀 설정(max, min, timeout) 미지정 → 기본값 사용
- **조치**: `extra: { max: 20, min: 5, idleTimeoutMillis: 30000, connectionTimeoutMillis: 2000 }` 추가

### 4.9 메일 동기화 동시성 문제
- **파일**: `apps/api/src/domain/webmail/service/mail-sync-cron.service.ts:18`
- **내용**: 5분마다 전체 계정 순차 동기화, 메모리 플래그 기반 중복 방지 (분산 환경 부적합)
- **조치**: Redis 기반 분산 락 또는 DB 기반 락 적용

### 4.10 Cron 작업 분산 환경 미대응
- **파일**: `apps/api/src/domain/billing/service/billing-automation.service.ts:31-40`
- **내용**: 메모리 `isCronRunning` 플래그로만 중복 실행 방지
- **조치**: 다중 인스턴스 환경 대비 Redis/DB 기반 락 필요

## 5. LOW 발견사항

### 5.1 프론트엔드 번들 최적화 부재
- **파일**: `apps/web/vite.config.ts`
- **내용**: 코드 분할(code splitting) 설정 없음, 2MB+ 번들 크기
- **조치**: `rollupOptions.manualChunks`로 vendor/tiptap/markdown 등 분리

### 5.2 synchronize: true 프로덕션 위험
- **파일**: `apps/api/src/app.module.ts:133`
- **내용**: TypeORM `synchronize: true`가 환경 구분 없이 적용될 수 있음
- **조치**: `synchronize: configService.get('NODE_ENV') === 'development'`로 변경

- **파일**: `apps/api/src/domain/auth/entity/user.entity.ts:33-34`
- **내용**: `usrPostalPassword` 평문 저장 (자동 생성 UUID이나 암호화 권장)
- **조치**: 암호화 후 저장

## 7. 우선순위별 조치 로드맵

### Phase 1: 즉시 조치 (CRITICAL + HIGH)
1. 하드코딩된 시크릿 기본값 제거 → 환경변수 필수화
2. localStorage 토큰 → httpOnly 쿠키 전환
3. 토큰 해지(로그아웃) API 구현
4. 주요 목록 API에 페이지네이션 적용
5. N+1 쿼리 배치 최적화

### Phase 2: 단기 조치 (MEDIUM)
6. 비밀번호 정책 강화 (12자 + 복잡성)
7. 리프레시 토큰 로테이션
8. HTML 살균 라이브러리 도입
9. DB 인덱스 추가 (마이그레이션)
10. 연결 풀 설정

### Phase 3: 중기 조치 (LOW + 개선)
11. 프론트엔드 번들 코드 분할
12. Cron 분산 락 (Redis)
13. 세션 유휴 타임아웃
14. synchronize 환경 분리

## 8. 참고사항

- 현재 프로덕션 미배포 상태이므로, 하드코딩된 시크릿은 개발/스테이징 환경에만 영향
- 프로덕션 배포 전 Phase 1 항목은 반드시 해결 필요
- 인덱스 추가는 TypeORM 마이그레이션으로 안전하게 적용 권장
- 모든 시크릿은 환경변수 기반으로 관리하고, 미설정 시 서버 시작 실패 처리 권장

## 1. 검토 개요

### 1.1 배경

AMA 서비스를 **유럽연합(EU)** 및 **미국** 지역에 제공할 경우, 다음 법규를 준수해야 합니다:

| 지역 | 법규 | 핵심 요건 |
|------|------|----------|
| **EU** | GDPR (General Data Protection Regulation) | 개인정보 처리 동의, 데이터 주체 권리, DPO, 72시간 침해 통보 |
| **미국 캘리포니아** | CCPA/CPRA (California Consumer Privacy Act) | 알 권리, 삭제 요청, 판매 거부(opt-out), 민감 정보 제한 |
| **미국 연방** | ADA Section 508 | 장애인 웹접근성 보장 |
| **국제** | WCAG 2.1 Level AA | 웹 콘텐츠 접근성 지침 |

### 1.2 검토 방법

- 소스 코드 정적 분석 (백엔드/프론트엔드 전체)
- DB 스키마 및 엔티티 분석
- 보안 미들웨어 설정 확인
- ARIA 속성 및 시맨틱 HTML 사용 현황 조사
- 기존 보안 점검 보고서 교차 검증

### 1.3 점검 대상 앱

| 앱 | 경로 | 설명 |
|----|------|------|
| apps/api | NestJS 백엔드 | REST API, 인증, 데이터 처리 |
| apps/web | React 프론트엔드 | 내부 직원용 웹앱 |
| apps/portal-api | NestJS 포털 API | B2B 고객/파트너 API |
| apps/portal-web | React 포털 프론트엔드 | 고객/파트너 포털 |

### 7.2 주요 발견사항

**강점:**
1. AES-256-GCM 암호화와 bcrypt 해싱 등 **기술적 보안이 매우 우수**
2. 멀티테넌시 격리(OwnEntityGuard)로 **Privacy by Design 원칙 이미 구현**
3. Helmet, CORS, Rate Limiting 등 **보안 미들웨어 완비**
4. 감사 로그(amb_access_audit_log) **PII 접근 추적 가능**

**약점:**
1. **GDPR 핵심 요건 미충족**: 동의 관리, DSR, 침해 통보 등 사용자 권리 보장 체계 부재
2. **웹접근성 기본 요소 미비**: 스킵 내비게이션, 폼 접근성(aria-invalid/required), 테이블 접근성
3. **데이터 라이프사이클 관리 부재**: 보유 기간 미정의, 자동 파기 없음
4. **프라이버시 정책 실질 부재**: 링크만 있고 실제 내용 없음

### 7.3 적격성 판단

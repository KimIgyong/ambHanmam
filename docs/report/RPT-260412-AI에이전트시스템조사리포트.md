## 1. 시스템 개요

AMB Management는 **다법인(Multi-Tenant)** 환경에서 **6개 부서별 AI 에이전트**를 제공하며, Anthropic Claude API로 실시간 SSE 스트리밍 채팅을 수행한다.

### 아키텍처 구조

```
Frontend (React)                 Backend (NestJS)                Claude API
─────────────────               ─────────────────               ─────────────
AgentsPage.tsx   ──HTTP REST──► AgentController
ChatPage.tsx     ──SSE Stream─► ChatController
                                    │
                                ChatService
                                    │
                                AgentFactoryService
                                    │
                         ┌──────────┼──────────┐
                   LegalAgent  AccountingAgent  ...
                         └──────────┼──────────┘
                                    │
                            BaseAgentService
                                    │
                             ClaudeService  ──REST──► Anthropic API
                                    │                 claude-sonnet-4-20250514
                             AiUsageService
                                    │
                              PostgreSQL DB
```

### 4.2 핵심 서비스 역할

| 서비스 | 역할 |
|--------|------|
| **BaseAgentService** | 추상 클래스. `unitCode`, `getSystemPrompt()` 정의. `chat()` (비스트리밍), `chatStream()` (SSE) 제공 |
| **AgentFactoryService** | unitCode → 실제 에이전트 인스턴스 매핑 (Map 패턴) |
| **AgentConfigService** | 에이전트 설정 CRUD. 커스텀 프롬프트 관리, 활성화/비활성화 |
| **ChatService** | 대화 생성/조회/삭제, 메시지 전송, AI 응답 스트리밍, 지식/이슈/노트 변환 |
| **ClaudeService** | Anthropic SDK 래퍼. API 키 관리 (DB→ENV→Mock), 쿼터 확인, 토큰 사용 기록 |
| **AiUsageService** | 토큰 사용량 기록, 일일/월간 쿼터 확인 |

### 4.3 메시지 처리 흐름

```
1. 사용자 메시지 → ChatController (SSE endpoint)
2. ChatService.sendMessage()
   ├─ 사용자 메시지 저장 (amb_messages, role='user')
   ├─ 대화 히스토리 로드 (최근 N개)
   ├─ AgentFactoryService.getAgent(unitCode)
   └─ agent.chatStream(messages, language, usageContext)
3. BaseAgentService.chatStream()
   ├─ 시스템 프롬프트 + 언어 지시문 결합
   └─ ClaudeService.streamMessage()
4. ClaudeService.streamMessage()
   ├─ checkQuotaIfNeeded() — 쿼터 확인 (초과 시 E4010/E4011)
   ├─ claude.messages.stream() — Anthropic API 호출
   ├─ Observable<ClaudeStreamEvent> 반환
   └─ recordUsageIfContext() — 토큰 사용 기록 (async, fire-and-forget)
5. AI 응답 완료
   ├─ AI 메시지 저장 (amb_messages, role='assistant')
   └─ KMS 이벤트 발행 (지식 파이프라인)
```

## 7. Claude API 통합

### 7.1 모델

- **모델**: `claude-sonnet-4-20250514`
- **SDK**: `@anthropic-ai/sdk`
- **통신**: REST (비스트리밍) + SSE (스트리밍)

### 7.2 API 키 우선순위

```
1. DB (EntityAiConfig / ApiKeyService) — 5분 캐시
2. 환경변수 CLAUDE_API_KEY — 폴백
3. Mock Mode — 개발/테스트용
```

### 7.3 스트리밍 이벤트

```typescript
interface ClaudeStreamEvent {
  type: 'content' | 'done' | 'error';
  content?: string;       // 부분 텍스트
  stopReason?: string;    // 'end_turn' | 'max_tokens'
  usage?: { inputTokens, outputTokens };
  error?: string;
}
```

### 7.4 다언어 지원

`Accept-Language` 헤더 → 에이전트 응답 언어 결정:
- `ko` → 한국어 응답
- `en` → 영어 응답  
- `vi` → 베트남어 응답

## 8. 토큰 쿼터 관리

### 8.1 쿼터 체크 흐름

```
메시지 전송 → ClaudeService.checkQuotaIfNeeded()
    ├─ entId 기반 일일/월간 사용량 조회
    ├─ 한도 초과 시 → BusinessException (E4010/E4011)
    └─ 허용 시 → Claude API 호출 → 사용량 기록
```

### 8.2 에러 코드

| 코드 | 상황 | HTTP |
|------|------|------|
| E4010 | 일일 토큰 한도 초과 | 403 |
| E4011 | 월간 토큰 한도 초과 | 403 |

### 8.3 충전 (Top-up)

- Megapay IPN 결제 이벤트 연동
- `QUOTA_PURCHASE:{productId}:{tokenAmount}` 형태로 상품 인코딩
- 결제 성공 시 `eac_monthly_token_limit` 증가, 환불 시 롤백

## 9. 대화 → 업무 자산 변환

ChatService에서 제공하는 3가지 변환:

| 변환 대상 | API | 설명 |
|-----------|-----|------|
| **KMS 지식** | `convert-to-knowledge` | 대화를 지식 자산으로 축적 |
| **업무 이슈** | `convert-to-issue` | 대화에서 발견된 이슈를 태스크화 |
| **회의 노트** | `convert-to-note` | 대화를 회의록으로 변환 |

→ 업무 성장 흐름: 채팅 → 태스크/이슈 → 프로젝트 → KMS 자산

## 10. 미구현/향후 계획

### 10.1 Today AI Analysis (미구현)

CLAUDE.md 정의: "일일 운영 최적화 — 매일의 실행 결과를 분석하고 다음 업무 계획으로 연결"

예상 구현:
- 일일 스케줄러 (NestJS `@nestjs/schedule`)
- 당일 미션/태스크/이슈 데이터 수집
- Claude AI 분석 → 성과 리포트 + 다음 계획 생성

### 10.2 지식 컨텍스트 주입 (부분 구현)

IT 에이전트 프롬프트에 `knowledgeContext` 파라미터가 존재하나, 실제 KMS 문서 로드 로직은 미완성.  
향후 RAG (Retrieval-Augmented Generation) 패턴으로 구현 예상.

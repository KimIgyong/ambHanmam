## 3. 엔티티 상세 명세

### 3.1 Auth 도메인

#### amb_users
| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| usr_id | uuid PK | auto-gen | 사용자 ID |
| usr_email | varchar(200) | UNIQUE | 로그인 이메일 |
| usr_password | varchar(200) | NOT NULL | bcrypt 해시 |
| usr_name | varchar(50) | NOT NULL | 이름 |
| usr_department | varchar(30) | NOT NULL | 부서코드 |
| usr_role | varchar(20) | default 'USER' | USER/MANAGER/ADMIN |
| usr_company_email | varchar(200) | UNIQUE, nullable | 사내 메일 |
| usr_postal_password | varchar(200) | nullable | Postal 비밀번호 |
| usr_token_version | int | default 0 | 토큰 회전 버전 |
| usr_signature_image | bytea | nullable | 서명 이미지 |
| usr_created_at | timestamp | auto | 생성일 |
| usr_updated_at | timestamp | auto | 수정일 |
| usr_deleted_at | timestamp | nullable | 소프트삭제 |

#### amb_password_resets
| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| prs_id | uuid PK | auto-gen | 리셋 ID |
| usr_id | uuid FK | → amb_users | 사용자 |
| prs_token | varchar(100) | UNIQUE | 리셋 토큰 |
| prs_expires_at | timestamp | NOT NULL | 만료시각 |
| prs_used_at | timestamp | nullable | 사용시각 |
| prs_created_at | timestamp | auto | 생성일 |

### 5.1 공통 컬럼 패턴

| 패턴 | 설명 | 적용 테이블 |
|------|------|------------|
| `{prefix}_id` (uuid PK) | 모든 테이블의 기본키 | 전체 79개 |
| `{prefix}_created_at` | CreateDateColumn | 전체 |
| `{prefix}_updated_at` | UpdateDateColumn | 대부분 |
| `{prefix}_deleted_at` | Soft Delete | 약 30개 테이블 |
| `ent_id` (uuid FK) | 법인 격리 (Multi-Tenancy) | 약 30개 테이블 |

### 5.2 암호화 패턴 (AES-256-GCM)

| 테이블 | 암호화 컬럼 |
|--------|------------|
| amb_api_keys | apk_key_encrypted + apk_key_iv + apk_key_tag |
| amb_smtp_settings | sms_pass_encrypted + sms_pass_iv + sms_pass_tag |
| amb_mail_accounts | mac_encrypted_password + mac_encrypted_iv + mac_encrypted_tag |
| amb_hr_employees_kr | ekr_resident_no (단일 컬럼 암호화) |
| amb_hr_freelancers | frl_resident_no (단일 컬럼 암호화) |

### 5.3 테이블 네이밍 규칙

- **Prefix**: `amb_` (Amoeba), `kms_` (Knowledge Management)
- **도메인 구분**: `amb_hr_`, `amb_bil_`, `amb_svc_`, `amb_kms_`, `amb_talk_`, `amb_mail_`
- **컬럼 Prefix**: 3자 약어 (usr, cvs, msg, emp, ptn, ctr, inv, etc.)
- **FK 패턴**: 참조 대상의 PK명과 동일 (예: `ent_id`, `usr_id`, `emp_id`)

### 5.4 인덱스 전략

| 테이블 | 인덱스 | 용도 |
|--------|--------|------|
| amb_messages | [cvs_id, msg_order] | 대화별 메시지 조회 |
| amb_hr_employees | [ent_id, emp_status] | 법인별 직원 필터링 |
| amb_bil_contracts | [ent_id, ctr_status] | 법인별 계약 필터링 |
| amb_bil_invoices | [ent_id, inv_status], [inv_date] | 인보이스 조회 |
| amb_notices | [ntc_visibility, ntc_created_at] | 공지 목록 |
| amb_transactions | [bac_id, txn_date] | 계좌별 거래 |
| amb_work_items | [ent_id, wit_module] | 법인별 모듈 필터링 |
| amb_mail_messages | [account_id, folder], [account_id, external_id] UNIQUE | 메일 조회 |
| amb_hr_tax_simple_table | [ent_id, year, salary_from, salary_to, dependents] | 세금 조회 |

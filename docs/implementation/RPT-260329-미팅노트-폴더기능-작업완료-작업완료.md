## 1. 구현 내용

### 백엔드
- **MeetingNoteFolderEntity** 신규 생성 (`amb_meeting_note_folders` 테이블)
  - UUID PK + ent_id 법인 격리 + soft delete
- **MeetingNoteEntity**에 `mtn_folder_id` nullable 컬럼 + ManyToOne 관계 추가
- **폴더 CRUD API**: GET/POST/PATCH/DELETE `/meeting-notes/folders`
- **노트 목록에 folder_id 필터** 추가 (`?folder_id=<uuid>` 또는 `?folder_id=uncategorized`)
- **Create/Update DTO**에 `folder_id` optional 필드 추가
- **Mapper/Response**에 `folderId`, `folderName` 필드 추가

### 프론트엔드
- **NoteFolderSidebar** 컴포넌트: 폴더 목록, 인라인 생성/수정/삭제, 전체/미분류 기본 항목
- **MeetingNotesPage**: 좌측 사이드바 통합, 폴더 선택 시 filter 적용
- **MeetingNoteFormModal**: 폴더 선택 드롭다운 추가
- **React Query 훅**: `useMeetingNoteFolders`, `useCreateFolder`, `useUpdateFolder`, `useDeleteFolder`
- **API 서비스**: 폴더 CRUD 메서드 + `folder_id` 필터 파라미터
- **i18n**: 한국어/영어/베트남어 `folder` 섹션 추가

### 공유 타입
- `MeetingNoteResponse`에 `folderId`, `folderName` 추가
- `MeetingNoteFolderResponse` 인터페이스 신규

### DB 마이그레이션
- `sql/migration_meeting_note_folders.sql` 제공

## 4. 배포 전 필수 사항

스테이징/프로덕션 배포 전 DB 마이그레이션 실행 필요:

```bash
# 스테이징
ssh amb-staging "docker exec amb-postgres-staging psql -U amb_user -d db_amb -f /dev/stdin" < sql/migration_meeting_note_folders.sql

# 또는 직접
ssh amb-staging
docker exec -i amb-postgres-staging psql -U amb_user -d db_amb < ~/ambManagement/sql/migration_meeting_note_folders.sql
```

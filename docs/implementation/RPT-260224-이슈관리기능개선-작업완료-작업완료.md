## 2. Phase별 구현 상세

### Phase 1: 백엔드 스키마/DTO/서비스 개선

**핵심 해결**: PATCH `/api/v1/issues/:id` 400 에러 — `type` 필드가 `UpdateIssueRequest` DTO에 누락

| 항목 | 내용 |
|------|------|
| SQL 마이그레이션 | `iss_assignee_id` UUID FK 컬럼 추가, 기존 `iss_assignee` 이름 → UUID 데이터 마이그레이션 |
| Entity | `issAssigneeId` UUID + `@ManyToOne assignee: UserEntity` 추가 |
| UpdateIssueRequest | `type` (`@IsIn(['BUG','FEATURE','ENHANCEMENT'])`) + `assignee_id` (`@IsUUID()`) 추가 |
| CreateIssueRequest | `assignee_id` (`@IsUUID() @IsOptional()`) 추가 |
| IssueService | `relations: ['reporter', 'assignee']` 로딩, `getMyIssues` → `issAssigneeId = :userId` 직접 매칭 |
| IssueMapper | `assigneeId`, `assigneeName`, `visibility`, `groupId`, `originalLang` 5개 필드 추가 |
| IssueResponse | 공유 타입에 5개 필드 추가 |

### Phase 2: 담당자 선택 UI

- `AssigneeSelector` 컴포넌트: `useMemberList` 훅 연동, 이름/이메일/부서 검색, 외부 클릭 닫기
- `IssueFormModal`: 텍스트 입력 → `AssigneeSelector` 교체, 신규 생성 시 현재 사용자 기본 선택
- FE 서비스/훅: `assignee` → `assignee_id` UUID 기반으로 전환

### Phase 3: 탭 UI + 리스트 뷰 분리

- 3탭 UI: 리스트 / 칸반 / 간트 (아이콘: `List`, `Columns3`, `BarChart3`)
- `IssueListView` 분리: 기존 `IssuesPage`에서 목록 렌더링 + 페이지네이션 추출
- 칸반/간트: `React.lazy()` 비동기 로딩, 리스트는 PAGE_SIZE=20, 나머지는 size=999

### Phase 4: 칸반 보드

- 라이브러리: `@dnd-kit/core@^6.3.1`, `@dnd-kit/sortable@^10.0.0`, `@dnd-kit/utilities@^3.2.2`
- `IssueKanbanCard`: `useDraggable` 훅, 유형 뱃지, 심각도 dot, 우선순위, 담당자명
- `IssueKanbanColumn`: `useDroppable` 훅, 상태별 색상, 드롭 가능 영역 하이라이트
- `IssueKanbanView`: `DndContext` + `DragOverlay`, `ISSUE_VALID_TRANSITIONS` 기반 상태 전환 검증
- **권한 제어**: `APPROVED`, `REJECTED`, `CLOSED` 전환은 MANAGER 이상만 가능

### Phase 5: 간트 차트

- 커스텀 SVG 기반 간트 차트 (외부 라이브러리 미사용)
- 고정 좌측 레이블 영역 (240px) + 스크롤 가능 타임라인
- 주간 마커, 오늘 빨간 점선, 상태별 색상 바
- 호버 툴팁 (이슈 상세 정보), 클릭 시 상세 모달 열기
- 기간: `createdAt` → `resolvedAt` (또는 오늘)

### Phase 6: i18n + 빌드 검증

- 한국어/영어/베트남어 3개 언어 번역 키 추가: `tabs`, `kanban`, `assignee`, `gantt` 섹션

## 4. 기술적 결정 사항

### 하위 호환성
- `issAssignee` (VARCHAR 이름 컬럼) 유지, `@deprecated` 표시
- `IssueMapper`에서 `assignee` 필드: `entity.assignee?.usrName || entity.issAssignee || null` (새 릴레이션 우선, 폴백)
- SQL 마이그레이션에서 기존 이름 데이터를 UUID로 자동 변환

### 성능 최적화
- 칸반/간트 뷰: `React.lazy()` + `Suspense`로 코드 분할
- 간트 차트: 외부 라이브러리 없이 순수 SVG 구현 (번들 사이즈 최소화)
- `@dnd-kit`: React 18 호환, 접근성 지원

### 권한 제어
- 칸반 드래그앤드롭 시 `ISSUE_VALID_TRANSITIONS` 맵 기반 상태 전환 검증
- `MANAGER_ONLY_STATUSES` (`APPROVED`, `REJECTED`, `CLOSED`): MANAGER 이상 권한 필요
- 권한 부족 시 해당 컬럼 비활성화 표시

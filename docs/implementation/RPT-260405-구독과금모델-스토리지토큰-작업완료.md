# RPT-260405-구독과금모델-스토리지토큰-작업완료

## 1. 구현 요약

SaaS 구독 과금 모델(FREE/BASIC/PREMIUM)을 신규 구현하였다. 토큰 지갑(BASE/ADDON/REFERRAL), 스토리지 쿼터, Polar.sh 결제 연동을 포함한 전체 백엔드 + 프론트엔드 구현이 완료되었다.

**핵심 원칙**: 기존 테이블(`amb_svc_*`)을 변경하지 않고, 새로운 `amb_sub_*` / `amb_pg_*` 도메인으로 독립 구현.

## 2. Phase별 구현 내역

### Phase 1: DB Schema + Entity Layer
- SQL DDL 2개 파일 + 8개 TypeScript 엔티티/열거형 파일 생성
- 7개 테이블: `amb_sub_plans`, `amb_sub_subscriptions`, `amb_sub_token_wallets`, `amb_sub_token_ledgers`, `amb_sub_storage_quotas`, `amb_pg_subscriptions`, `amb_pg_webhook_events`

### Phase 2: Subscription Module Core
- SubscriptionService, TokenWalletService, StorageQuotaService, PolarService
- SubscriptionController, PolarWebhookController
- TokenBalanceGuard, Request/Response DTOs
- SubscriptionModule 등록 (AppModule에 추가)
- 에러 코드 E29xxx 체계 추가

### Phase 3: Token System Integration
- TokenBalanceGuard → ChatController `sendMessage`에 적용
- AiUsageService에서 `ai-usage.recorded` 이벤트 발행
- TokenWalletService에서 `@OnEvent('ai-usage.recorded')` 리스너로 자동 차감
- Polar webhook 서명 검증용 rawBody 미들웨어 추가 (main.ts)

### Phase 4: Storage Quota Integration
- FileController: `uploadFile`에 `checkUpload` + `addUsage` 연동
- DriveController: `uploadFile`에 `checkUpload` + `addUsage` 연동
- FileModule, DriveModule에 SubscriptionModule import 추가

### Phase 5: Frontend UI
- 3개 언어 i18n JSON 파일 (ko/en/vi)
- subscription.service.ts (API 클라이언트)
- useSubscription.ts (React Query 훅 7개)
- SubscriptionPage.tsx (탭 기반 통합 페이지)
- PlanSection.tsx, TokenSection.tsx, StorageSection.tsx (섹션 컴포넌트)
- entity-settings 라우터에 `/entity-settings/subscription` 경로 추가
- EntitySettingsPage 메뉴에 구독 관리 항목 추가

## 5. 배포 전 필수 작업

### 5.1 DB 마이그레이션 (스테이징/프로덕션 수동 실행)

```bash
# 스테이징
docker exec amb-postgres-staging psql -U amb_user -d db_amb -f /path/to/001-create-subscription-schema.sql
docker exec amb-postgres-staging psql -U amb_user -d db_amb -f /path/to/002-seed-plans.sql

# 프로덕션
docker exec amb-postgres-production psql -U amb_user -d db_amb -f /path/to/001-create-subscription-schema.sql
docker exec amb-postgres-production psql -U amb_user -d db_amb -f /path/to/002-seed-plans.sql
```

### 5.2 환경변수 추가

```
POLAR_ACCESS_TOKEN=<Polar API 토큰>
POLAR_WEBHOOK_SECRET=<Polar 웹훅 시크릿>
POLAR_ORGANIZATION_ID=<Polar 조직 ID>
POLAR_PRODUCT_BASIC_MONTHLY=<Polar 상품 ID>
POLAR_PRODUCT_BASIC_ANNUAL=<Polar 상품 ID>
POLAR_PRODUCT_TOKEN_ADDON=<Polar 상품 ID>
POLAR_PRODUCT_STORAGE_ADDON=<Polar 상품 ID>
```

### 5.3 Polar 설정
1. Polar 대시보드에서 상품 4개 생성 (BASIC Monthly/Annual, Token Addon, Storage Addon)
2. 웹훅 엔드포인트 등록: `https://{domain}/api/v1/webhooks/polar`
3. 웹훅 이벤트 구독: `subscription.*`, `order.created`

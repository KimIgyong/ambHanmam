## 2. 구현 내용 요약

### Phase 0: 프로젝트 인프라 구성
- `packages/portal-shared/` — 공유 상수/타입 (JWT, 구독 상태, 역할)
- `apps/portal-api/` — NestJS 백엔드 (포트 3010), 7개 shared-entities, 글로벌 필터/인터셉터
- `apps/portal-web/` — Vite+React 프론트엔드 (포트 5180), TailwindCSS, i18n (EN/KO/VI)
- 환경설정: `env/portal-backend/.env.development`, `env/portal-frontend/.env.development`
- Docker: `docker/staging/docker-compose.staging.yml`에 portal-api, portal-web 서비스 추가
- 루트 `package.json`에 `dev:portal`, `dev:portal-api`, `dev:portal-web`, `dev:all` 스크립트 추가

### Phase 1: 엔티티 확장 + 고객 인증
- **기존 엔티티 확장** (6개, 모두 nullable 컬럼):
  - `service-plan.entity.ts`: +5 Stripe 컬럼 (monthly/annual price ID, trial_days, overage)
  - `subscription.entity.ts`: +5 Stripe 컬럼 (subscription/price ID, period start/end, cancel_at_period_end)
  - `client.entity.ts`: +3 컬럼 (stripe_customer_id, email_verified, portal_source)
  - `partner.entity.ts`: +2 컬럼 (stripe_customer_id, source)
  - `contract.entity.ts`: +1 컬럼 (stripe_subscription_id)
  - `invoice.entity.ts`: +2 컬럼 (stripe_invoice_id, stripe_payment_intent_id)
- **포탈 고객 엔티티**: `amb_svc_portal_customers` 테이블 (14개 비즈니스 컬럼)
- **인증 모듈** (portal-api):
  - 7개 엔드포인트: register, verify-email, login, refresh, forgot-password, reset-password, me
  - JWT 전략 (`portal-jwt`), bcrypt 해싱, UUID 토큰
  - Rate limiting: register 5/min, login 10/min, forgot-password 3/min
- **카탈로그 모듈**: 공개 서비스/플랜 API (2개 엔드포인트)

### Phase 2: 공개 마케팅 사이트
- **PublicLayout**: 반응형 헤더 (모바일 햄버거 메뉴), 푸터, 언어 선택기
- **LandingPage**: Hero 섹션, 서비스 카드 3개 (서비스 상세 페이지 링크)
- **PricingPage**: API 연동 요금제 비교 (가격 포맷팅, 무료 체험 표시)
- **ServiceDetailPage**: 서비스 상세 + 플랜 목록
- **인증 페이지 6개**: Login, Register, ForgotPassword, ResetPassword, VerifyEmail, VerifyEmailSent
- **Auth Store**: zustand + localStorage persist, 401 자동 refresh 인터셉터
- **i18n**: EN/KO/VI 3개 언어 152개 번역 키

### Phase 3: Stripe 연동
- **StripeService**: Checkout Session 생성, Billing Portal, Webhook 처리
- **Webhook 이벤트 처리**: checkout.session.completed, subscription.created/updated/deleted, invoice.paid/payment_failed
- **Stripe → 구독 상태 매핑**: trialing→TRIAL, active→ACTIVE, past_due→SUSPENDED, canceled→CANCELLED
- **Raw body 처리**: Express 미들웨어로 webhook 경로에만 raw body 버퍼 유지

### Phase 4-5: 구독 관리 + 고객 포탈
- **SubscriptionModule**: 고객별 구독 목록/상세 API (2개 엔드포인트)
- **PortalLayout**: 인증 가드, 사이드바 (데스크톱) + 하단 탭 (모바일)
- **DashboardPage**: 환영 메시지, 이메일 미인증 경고, 통계 카드, 빠른 작업
- **SubscriptionsPage**: 구독 목록 (API 연동), 상태별 아이콘/색상, 빈 상태
- **BillingPage**: 빈 상태 (Phase 6 예정)
- **SettingsPage**: 프로필 편집, 보안 설정

### Phase 6: 사용량 기반 과금 (Usage Billing)
- **SvcUsageRecordEntity**: `amb_svc_usage_records` 테이블 (구독별 메트릭, 수량, 단가, 기간, Stripe 동기화)
- **UsageModule**:
  - `UsageService`: 사용량 기록 생성, Stripe Metered Billing 동기화, 메트릭별 요약 조회
  - `UsageController`: 내부 API — POST /portal/usage/record, POST /portal/usage/record/batch
  - `UsageReportController`: 고객 API — GET /portal/usage/summary, GET /portal/usage/subscriptions/:id/current
- **UsagePage**: 프론트엔드 사용량 대시보드 (구독 선택, 메트릭별 카드, 미니 바 차트)

### Phase 7: 관리자 콘솔 + 지역별 PG
- **PortalPaymentEntity**: `amb_svc_portal_payments` 테이블 (게이트웨이, 거래ID, 금액, 통화, 상태, 국가)
- **PaymentGatewayModule**:
  - `PaymentProvider` 인터페이스: createPayment, verifyPayment, refundPayment
  - `VnpayProvider`: VNPay (베트남) — HMAC-SHA512 서명, VND ×100 변환, sandbox URL
  - `VnptepayProvider`: VNPTePay (베트남) — HMAC-SHA256 서명
  - `TossProvider`: TossPayments (한국) — Basic Auth, confirm/cancel API
  - `PaymentGatewayService`: 국가/통화 기반 자동 게이트웨이 라우팅
    - VN/VND → VNPay, VNPTePay
    - KR/KRW → TossPayments
    - 기본 → Stripe (글로벌)
  - `PaymentGatewayController`: POST /portal/payments/create, GET /portal/payments/callback/:gateway, GET /portal/payments/history, GET /portal/payments/gateways
- **AdminModule**:
  - `AdminDashboardService`: 대시보드 통계, 게이트웨이별 결제 분석
  - `AdminDashboardController`: GET /portal/admin/dashboard, GET /portal/admin/payments/by-gateway
  - `AdminCustomerController`: GET /portal/admin/customers (페이지네이션+검색), GET /portal/admin/customers/:id (상세)

## 5. Phase 6-7 구현 완료

| Phase | 내용 | 상태 |
|-------|------|------|
| Phase 6 | 사용량 기반 과금 (Usage Billing) | ✅ 완료 — UsageModule + Stripe Metered Billing 동기화 |
| Phase 7 | 관리자 콘솔 + 지역별 PG | ✅ 완료 — VNPay, VNPTePay, TossPayments + Admin API |

### 배포 전 필수 작업
1. Stripe 키 설정: `STRIPE_SECRET_KEY`, `STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET`
2. PG 키 설정 (지역별):
   - VNPay: `VNPAY_TMN_CODE`, `VNPAY_HASH_SECRET`
   - VNPTePay: `VNPTEPAY_MERCHANT_ID`, `VNPTEPAY_SECRET_KEY`
   - TossPayments: `TOSS_CLIENT_KEY`, `TOSS_SECRET_KEY`
3. PostgreSQL `synchronize: true`로 초기 테이블 생성 (또는 마이그레이션 실행)
4. `amb_svc_services`, `amb_svc_plans` 테이블에 서비스/플랜 시드 데이터 입력
5. 이메일 발송 서비스 연동 (SMTP 설정)

## 6. 아키텍처 요약

```
                    www.amoeba.site (portal-web :5180)
                           │
                    portal-api :3010
                    ├── AuthModule (JWT, bcrypt)
                    ├── CatalogModule (public)
                    ├── StripeModule (checkout, webhook)
                    ├── SubscriptionModule (customer portal)
                    ├── UsageModule (metered billing)
                    ├── PaymentGatewayModule (VNPay, VNPTePay, Toss, Stripe)
                    └── AdminModule (dashboard, customers)
                           │
                    ┌──────┴──────┐
                    │  PostgreSQL  │ (db_amb — 기존 DB 공유)
                    └──────┬──────┘
                           │
                    internal API :3009 (기존)
```

- **DB 공유**: portal-api와 기존 api가 동일한 PostgreSQL DB(`db_amb`)를 공유
- **엔티티 확장**: 기존 테이블에 nullable 컬럼 추가 방식으로 Stripe 연동 (85% 자산 재사용)
- **인증 분리**: 내부 사용자(`amb_users`)와 포탈 고객(`amb_svc_portal_customers`) 완전 분리
- **결제 게이트웨이 라우팅**: 국가/통화 기반 자동 PG 선택 (VN→VNPay/VNPTePay, KR→Toss, 글로벌→Stripe)
- **사용량 과금**: Stripe Metered Billing 동기화, 메트릭별 사용량 추적

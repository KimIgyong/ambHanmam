## 2. 요구사항 및 문제 현황 (AS-IS)

### 문제 1 — 알림 드롭다운 모바일 좌측 잘림
- 헤더 알림 벨 클릭 시 드롭다운이 `absolute right-0 w-80` 고정으로 표시
- 좁은 화면(모바일)에서 드롭다운 좌측이 뷰포트 밖으로 넘침 → 컨텐츠 잘림

### 문제 2 — 팝업 알림 iOS 노치 영역 표시
- `index.html`에 `viewport-fit=cover` + `apple-mobile-web-app-status-bar-style: black-translucent` 설정
- `NotificationModal`이 `pt-20`(80px 고정)으로 위치 계산 → 기기 별 노치 높이 차이 미고려
- iPhone 14 Pro(Dynamic Island 48px), iPhone X(노치 44px) 등에서 팝업이 상태바 영역에 겹쳐 닫기 버튼 클릭 불가

## 3. 수정 내용 (TO-BE)

### 수정 1 — `NotificationBell.tsx`

**변경 위치**: 드롭다운 컨테이너 className

| 구분 | 변경 내용 |
|------|----------|
| AS-IS | `absolute right-0 top-full mt-2 w-80` |
| TO-BE | 모바일: `fixed top-16 left-1/2 -translate-x-1/2 w-[calc(100vw-2rem)] max-w-sm`<br>데스크탑(`sm` 이상): `sm:absolute sm:top-full sm:right-0 sm:translate-x-0 sm:w-80` |

- 모바일 (`sm` 미만): 화면 폭 기준 중앙 고정 배치, 좌우 1rem 여백 확보
- 태블릿/데스크탑: 기존 우측 정렬 absolute 방식 유지

### 수정 2 — `NotificationModal.tsx`

**변경 위치**: 최상위 컨테이너 `div` padding-top

| 구분 | 변경 내용 |
|------|----------|
| AS-IS | `pt-20` (고정 80px) |
| TO-BE | `style={{ paddingTop: 'max(calc(env(safe-area-inset-top, 0px) + 8px), 16px)' }}` |

- `env(safe-area-inset-top)`: iOS 기기별 노치/Dynamic Island 실제 높이를 런타임에 읽음
- `+ 8px`: 노치 바로 아래 최소 여백
- `max(..., 16px)`: safe-area 미지원 기기(Android 등) 최소 상단 여백 보장

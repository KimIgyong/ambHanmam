## 1. 작업 개요

문서함(`/documents`) 페이지에 대한 4가지 기능개선을 수행하였다.

| 번호 | 기능 | 설명 |
|------|------|------|
| 1 | 업로드 버튼 위치 변경 | 헤더에서 브레드크럼 영역으로 이동, 폴더 내부 진입 시 항상 표시 |
| 2 | 관리자 하위폴더 생성 | 관리자 사용자가 FolderPlus 버튼과 `window.prompt` 다이얼로그를 통해 하위폴더 생성 |
| 3 | 카드/그리드 뷰 지원 | 리스트/그리드 뷰 토글 기능 추가, `localStorage`를 통한 뷰 상태 영속성, `FileGridItem` 컴포넌트(썸네일 포함) |
| 4 | KMS 태그 자동 분석 | 파일 업로드 시 `EventEmitter2`를 통해 `module.data.created` 이벤트 발행, KMS 자동 태깅 연동 |

## 3. 상세 구현 내용

### 3.1 업로드 버튼 위치 변경

- 기존 헤더 영역에 배치되었던 업로드 버튼을 브레드크럼 영역으로 이동하였다.
- 사용자가 폴더 내부에 진입한 경우에만 업로드 버튼이 표시되도록 조건부 렌더링을 적용하였다.
- 이를 통해 사용자가 현재 위치를 인식하면서 바로 파일을 업로드할 수 있도록 UX를 개선하였다.

### 3.2 관리자 하위폴더 생성

- 브레드크럼 영역에 `FolderPlus` 아이콘 버튼을 추가하였다. 관리자(admin) 권한이 있는 사용자에게만 노출된다.
- 버튼 클릭 시 `window.prompt` 다이얼로그를 통해 폴더명을 입력받는다.
- 백엔드에 `POST /api/v1/drive/create-folder` 엔드포인트를 신규 추가하였으며, `drive.service.ts`에 `createSubfolder` 메서드를 구현하였다.
- 프론트엔드에서는 `useCreateSubfolder` 훅을 통해 API를 호출하고 폴더 목록을 자동으로 갱신한다.

### 3.3 카드/그리드 뷰 지원

- `DocumentsPage.tsx`에 리스트/그리드 뷰 토글 버튼을 추가하였다.
- 뷰 상태는 `localStorage`에 저장되어 페이지 재방문 시에도 이전 선택이 유지된다.
- 그리드 뷰를 위한 `FileGridItem.tsx` 컴포넌트를 신규 생성하였다. 이미지 파일의 경우 실제 썸네일을 표시하고, 비이미지 파일은 파일 유형에 맞는 아이콘을 표시한다.
- 다국어 지원을 위해 `viewList`, `viewGrid` 번역 키를 3개 언어(ko, en, vi)에 추가하였다.

### 3.4 KMS 태그 자동 분석

- `drive.service.ts`에서 파일 업로드 완료 시 `EventEmitter2`를 통해 `module.data.created` 이벤트를 발행하도록 구현하였다.
- KMS 모듈이 해당 이벤트를 수신하여 업로드된 파일에 대한 자동 태깅 분석을 수행한다.
- 이벤트 기반 비동기 처리로 구현되어, 태깅 실패가 파일 업로드 자체에 영향을 미치지 않는다.

## 6. 비고

- `FileGridItem.tsx`는 신규 생성 파일이며, 프로젝트의 파일 네이밍 컨벤션(PascalCase 컴포넌트)을 준수한다.
- API 엔드포인트는 `/api/v1` base path 규칙을 준수한다.
- 다국어 키는 하드코딩 없이 `useTranslation()` 훅의 `t()` 함수를 통해 사용되며, i18n 규칙을 준수한다.
- KMS 이벤트 연동은 기존 `module.data.created` 이벤트 체계를 활용하여 모듈 간 결합도를 최소화하였다.

# RPT-일정관리-작업완료보고-20250224

## 1. 구현 개요

일정관리(캘린더) 시스템의 백엔드 API + 프론트엔드 UI 전체 구현을 완료하였습니다.
요구사항 분석서(REQ-일정관리-20260224)와 작업계획서(PLAN-일정관리-작업계획-20260224)에 따라 6단계 작업을 수행하였습니다.

## 2. 구현 내용

### Phase 1: 엔티티 & DB 마이그레이션

| 항목 | 내용 |
|------|------|
| 엔티티 | 5개 (schedule, schedule-recurrence, schedule-exception, schedule-participant, schedule-notification) |
| 테이블 | amb_schedules, amb_schedule_recurrences, amb_schedule_exceptions, amb_schedule_participants, amb_schedule_notifications |
| 인덱스 | 7개 (entityId+startDatetime, ownerId, category, participant userId, notification schId+type 등) |
| CHECK 제약 | enum 컬럼별 CHECK 제약 조건 (category, visibility, priority, frequency, response_status 등) |
| DTO | 5개 (create, update, exception, participants, respond) |

### Phase 2: CRUD + ACL 서비스

| 항목 | 내용 |
|------|------|
| CRUD | create, findAll, findById, update, remove (soft delete) |
| 낙관적 잠금 | current_updated_at 비교 → 409 Conflict |
| ACL | PRIVATE/SHARED/DEPARTMENT/ENTITY 4단계 가시성 |
| 부서 계층 | getUserDeptIds() → getDeptIdsWithChildren() 재귀 탐색 |
| 매니저 접근 | SUPER_ADMIN/ADMIN/MANAGER는 부서원 일정 접근 가능 |

### Phase 3: 리마인더 서비스

| 항목 | 내용 |
|------|------|
| 스케줄 | @Cron('0 */5 * * * *') - 5분 간격 |
| 수신자 | 소유자 + ACCEPTED/TENTATIVE 참석자 |
| 채널 | TALK / EMAIL (추후 구현 플레이스홀더) |

### Phase 4: Google Calendar 연동 (Stub)

| 항목 | 내용 |
|------|------|
| 구조 | 3회 재시도 + 지수 백오프 |
| 메서드 | connectGoogle, disconnectGoogle, pushToGoogle, deleteFromGoogle |
| 동기화 상태 | NONE → SYNCING → SYNCED / FAILED |

### Phase 5: 프론트엔드 캘린더 UI

| 항목 | 내용 |
|------|------|
| 서비스 | ScheduleService (18개 API 메서드) |
| 스토어 | useScheduleStore (Zustand - 뷰 타입, 필터, 선택 상태) |
| 훅 | useScheduleList, useScheduleDetail, useCreateSchedule, useUpdateSchedule, useDeleteSchedule, useRespondSchedule 등 8개 |
| 뷰 | 월간/주간/일간 3개 캘린더 뷰 |
| 컴포넌트 | CalendarToolbar, CalendarMonthView, CalendarWeekView, CalendarDayView, ScheduleFormModal, ScheduleDetailPanel |
| i18n | 3개 언어 (ko/en/vi) schedule 네임스페이스 등록 |
| 라우트 | /schedule (MenuGuard menuCode="SCHEDULE") |

### Phase 6: AI 엔드포인트 (Stub)

| 항목 | 내용 |
|------|------|
| generateToday | 오늘 일정 자동 생성 |
| generateWbs | WBS 기반 마일스톤 일정 생성 |
| optimizeWeek | 주간 일정 최적화 제안 |
| analyzeTeam | 팀 일정 분석 |

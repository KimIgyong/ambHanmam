## 1. 구현 요약

ambKMS(mng.amoeba.site)에서 portal-web(www.amoeba.site)의 페이지를 코드 배포 없이 관리할 수 있는 헤드리스 CMS 모듈의 Phase 1(핵심 기능)을 구현하였다.

| Phase | 내용 | 상태 |
|------|------|------|
| Phase 1-Day 1 | 기반 구조 + 엔티티 9개 + 메뉴/권한 등록 | ✅ 완료 |
| Phase 1-Day 2 | DTO 13개 + Mapper 5개 + Service 6개 | ✅ 완료 |
| Phase 1-Day 3 | Controller 6개 (Admin 5 + Public 1) | ✅ 완료 |
| Phase 1-Day 4 | 프론트엔드 기반 구조 + 타입 + i18n + 메뉴/라우터 | ✅ 완료 |
| Phase 1-Day 5 | API 서비스 + React Query 훅 + Zustand 스토어 | ✅ 완료 |
| Phase 1-Day 6 | 메뉴 관리 + 페이지 목록 화면 | ✅ 완료 |
| Phase 1-Day 7 | 페이지 편집기 (STATIC TipTap + 메타/SEO + 발행 + 버전) | ✅ 완료 |

### 2.4 메뉴/권한 시스템 등록

- **MENU_CODES**: `SITE_MANAGEMENT` 추가
- **DEFAULT_PERMISSIONS**: MANAGER 기본 true, MEMBER/VIEWER/USER 기본 false
- **SUPER_ADMIN/ADMIN**: `ALL_MENU_CODES.map()` 패턴으로 자동 포함
- **DEFAULT_MENU_CONFIGS**: `{ code: 'SITE_MANAGEMENT', icon: 'Globe', path: '/site', category: 'MODULE', sortOrder: 1700 }`
- **자동 등록**: `MenuConfigService.onModuleInit()`에서 DB 미등록 시 자동 INSERT

## 6. 알려진 이슈

1. **토스트 알림 없음**: 프로젝트에 글로벌 토스트 라이브러리(react-hot-toast 등) 미설치. 현재 success/error 피드백이 제한적.
2. **Public API CORS**: `main.ts` CORS 설정에 `www.amoeba.site` 추가 필요 (별도 배포 시점에 적용).
3. **게시글/구독자 라우트**: 라우터에 placeholder 등록만 완료. 실제 UI는 Phase 2에서 구현.
4. **이미지 업로드**: StaticEditor에서 이미지 URL 직접 입력만 지원. Google Drive 연동 업로드는 별도 구현 필요.

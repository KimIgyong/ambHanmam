## 2. 검증 결과

### 2.1 3단계 이메일 구분 체계

| 컨텍스트 | 구현 상태 | 검증 내용 |
|----------|-----------|-----------|
| 아메바 포털 이메일 | 정상 | `PortalCustomerEntity.pctEmail` — 독립 시스템, 매니지먼트와 분리 |
| 아메바 서비스 계정 이메일 | 정상 | `UserEntity.usrEmail` — 글로벌 UNIQUE, 다법인 소속 지원 |
| 법인별 초대 이메일 | 정상 | `InvitationEntity.invEmail` — 법인 단위 중복 체크 |

### 2.2 법인 초대 중복 방지

- 같은 이메일 + 같은 법인 → 차단 (정상)
- 같은 이메일 + 다른 법인 → 허용 (정상)
- `invitation.service.ts`에서 `invEmail + invCompanyId` 복합 조건으로 체크

### 2.3 포털 브릿지 계정 생성

- `portal-bridge.service.ts`에 `EntityUserRoleEntity` 생성 로직 구현 완료 (200-215행)
- `portal-bridge.service.ts`에 `EmployeeEntity` 자동 생성 로직 구현 완료 (217-245행)
- `portal-bridge.module.ts`에 `EntityUserRoleEntity`, `EmployeeEntity` 의존성 등록 완료

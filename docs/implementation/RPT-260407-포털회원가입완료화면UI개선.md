### 테스트 플로우 (실행 순서)

```
1. POST /api/v1/auth/send-verify-code (pm@amoeba.group)
   → 200 OK: "Verification code sent"

2. POST /api/v1/auth/verify-email-code (code: 5726)
   → 200 OK: verify_token JWT 발급

3. POST /api/v1/auth/register (company: "Test Company 0407", plan: "free_trial")
   → 200 OK: { entityCode: "VN9257", entityName: "Test Company 0407" }

4. 이메일 수신 확인
   → Company Code: VN9257 표시, Login Now URL에 /VN9257/login 포함

5. POST stg-ama.amoeba.site/api/v1/auth/login (entity_code: VN9257, temp password)
   → 200 OK: { mustChangePw: true, accessToken, refreshToken }

6. POST stg-ama.amoeba.site/api/v1/auth/change-password (old → new)
   → 200 OK: "Password changed successfully"

7. POST stg-ama.amoeba.site/api/v1/auth/login (entity_code: VN9257, new password)
   → 200 OK: { mustChangePw: false, accessToken, refreshToken }
```

## 5. 이슈 및 해결

| 이슈 | 해결 |
|------|------|
| AMA 로그인 DTO 필드명 | `entityCode` (camelCase) → `entity_code` (snake_case) 로 수정. Request DTO는 snake_case 규칙 |
| 이메일 인증 코드 DB 조회 불가 | 인증 코드는 in-memory Map에 저장됨 (DB 테이블 없음). 실제 이메일 수신 필요 |

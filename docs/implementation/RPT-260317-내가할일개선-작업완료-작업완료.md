## 1. 구현 내용 요약

### 1.1 핵심 변경

기존 수동 3단계 상태 관리(SCHEDULED → IN_PROGRESS → COMPLETED)를 **날짜 기반 자동 6단계 분류**로 전환했다.

| 구분 | AS-IS | TO-BE |
|------|-------|-------|
| 상태 분류 | 3종 (수동 순환) | 6종 (날짜 기반 자동 계산) |
| 완료 처리 | 3-way 아이콘 클릭 순환 | **체크박스 토글** (완료/재개) |
| 마감일 | 필수 (NOT NULL) | **선택** (NULL 허용) |
| 시작 예정일 | 미존재 | **신규 추가** (`tdo_start_date`) |
| UI 구조 | 4-column 보드 (날짜 네비게이션) | **5-section 세로 리스트** |
| D-day 표시 | Overdue일 때만 D+N | 전 상태 배지 (D+N, D-Day, D-N, N일 후 시작) |

### 1.2 Computed Status 6단계

| 상태 | 조건 | 배지 색상 |
|------|------|----------|
| `COMPLETED` | `tdo_completed_at IS NOT NULL` | 🟢 초록 |
| `OVERDUE` | 미완료 + 마감일 < 오늘 | 🔴 빨강 |
| `DUE_TODAY` | 미완료 + 마감일 = 오늘 | 🟠 주황 |
| `IN_PROGRESS` | 미완료 + 시작일 ≤ 오늘 (또는 시작일 없음) + 마감일 > 오늘 | 🔵 파랑 |
| `UPCOMING_SOON` | 미완료 + 시작일 ≤ 오늘+3일 | 🟡 노랑 |
| `UPCOMING` | 미완료 + 시작일 > 오늘+3일 (또는 마감일 없음) | ⚪ 회색 |

## 4. 레거시 호환성

### 4.1 `tdo_status` 컬럼 유지

- `tdo_status` 컬럼은 **삭제하지 않고** 읽기 전용+자동 동기화로 전환
- Computed status → Legacy status 매핑:
  - `COMPLETED` → `'COMPLETED'`
  - `OVERDUE`, `DUE_TODAY`, `IN_PROGRESS` → `'IN_PROGRESS'`
  - `UPCOMING_SOON`, `UPCOMING` → `'SCHEDULED'`
- 할일 생성/수정/완료/재개 시 자동으로 레거시 상태 동기화

### 4.2 하위 호환 보장

- `invoice-todo.service.ts` (청구서 자동 할일): DTO의 `status` 필드를 `@deprecated`로 표시하되, 전송해도 무시됨 (날짜 기반 계산이 우선)
- `today.service.ts`: 쿼리 조건을 `tdo_completed_at IS NULL` 기반으로 전환하여 computed status와 일관성 유지
- API Response에서 기존 `status` 필드도 여전히 포함 (점진적 마이그레이션)

## 5. 핵심 구현 설계

### 5.1 상태 계산 유틸 (`compute-todo-status.util.ts`)

```
computeTodoStatus(todo, today?, upcomingSoonDays=3)
├── completedAt != null → COMPLETED
├── dueDate < today → OVERDUE
├── dueDate == today → DUE_TODAY
├── startDate ≤ today (or null) → IN_PROGRESS
├── startDate ≤ today + 3 → UPCOMING_SOON
└── else → UPCOMING
```

- **순수 함수**: 사이드이펙트 없음, 테스트 용이
- `upcomingSoonDays` 파라미터로 "곧 시작" 기준일 조정 가능

### 5.2 완료/재개 전용 엔드포인트

| 엔드포인트 | 동작 |
|-----------|------|
| `PATCH /todos/:id/complete` | `tdo_completed_at = NOW()`, 레거시 상태 동기화, 반복 할일 생성 |
| `PATCH /todos/:id/reopen` | `tdo_completed_at = NULL`, 날짜 기반 레거시 상태 재계산 |

### 5.3 UI 구조 변경

**TodoPage / ScopedTodoList**: 5-section 세로 리스트
```
┌─ 🔴 지연 (OVERDUE) ─────────────────┐
│   [☐] 마감 지난 할일들 (D+N 배지)     │
├─ 🟠 오늘 마감 (DUE_TODAY) ───────────┤
│   [☐] 오늘 마감 할일들 (D-Day 배지)    │
├─ 🔵 진행 중 (IN_PROGRESS) ───────────┤
│   [☐] 현재 진행 할일들 (D-N 배지)      │
├─ 🟡 곧 시작 (UPCOMING_SOON) ─────────┤
│   [☐] 3일 내 시작 할일들               │
├─ ⚪ 예정 (UPCOMING) ─────────────────┤
│   [☐] 미래 할일들                      │
├─ 🟢 완료 (COMPLETED) ▶ 접기/펼치기 ──┤
│   [☑] 완료된 할일들 (기본 접힘)         │
└───────────────────────────────────────┘
```

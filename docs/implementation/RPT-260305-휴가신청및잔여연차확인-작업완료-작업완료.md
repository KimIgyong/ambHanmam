## 1. 구현 내용 요약

### 1-1. 백엔드 — LeaveRequest 엔티티 + 서비스 + 컨트롤러

**LeaveRequestEntity** (`leave-request.entity.ts`)
- 테이블: `amb_hr_leave_requests` (TypeORM synchronize 자동 생성)
- 주요 컬럼: `lrq_id`(UUID PK), `emp_id`(FK), `usr_id`(FK), `ent_id`(FK nullable), `lrq_type`, `lrq_start_date`, `lrq_end_date`, `lrq_days`(DECIMAL 4,1), `lrq_reason`, `lrq_status`, `lrq_approved_by`, `lrq_rejected_reason`, soft delete 지원

**CreateLeaveRequestRequest / RejectLeaveRequestRequest** (DTO)
- 신청: type(`@IsIn`), start_date, end_date, reason(optional)
- 반려: reason(`@IsNotEmpty`)

**LeaveRequestService** (`leave-request.service.ts`)
- `getMyLeaveBalance(userId, year)` — usr_id → emp_id 매핑, LeaveBalance 조회 + 근속연수 계산
- `getMyLeaveRequests(userId, year)` — 해당 연도 본인 신청 내역
- `createLeaveRequest(userId, entId, dto)` — 영업일 수 계산, 잔여 연차 검증(PENDING 포함), 중복 일자 체크
- `cancelLeaveRequest(userId, requestId)` — PENDING만 취소 가능, 본인 확인
- `getLeaveRequests(entId, params)` — 관리자용 목록 (상태/연도 필터, 페이징)
- `approveLeaveRequest(requestId, approvedByUserId)` — DB 트랜잭션으로 상태 변경 + LeaveBalance 차감 + Timesheet 자동 삽입
- `rejectLeaveRequest(requestId, approvedByUserId, reason)` — 반려 사유 기록

**승인 시 자동 처리 (트랜잭션 내):**
1. lrq_status → APPROVED, lrq_approved_by 기록
2. ANNUAL/AM_HALF/PM_HALF만 연차 차감 (lvb_used ↑, lvb_remaining ↓)
3. Timesheet 자동 삽입 — ANNUAL: 'AL'(workHours=0), 반차: 'AH'(workHours=4)
   - 영업일만 삽입 (주말 제외)
   - 공휴일(HolidayEntity) 제외
   - 기존 Timesheet 있으면 UPDATE (Upsert)

**LeaveRequestController** (`leave-request.controller.ts`)

| 메서드 | 경로 | Guard | 설명 |
|--------|------|-------|------|
| GET | `/hr/leave-requests/my/balance` | — | 내 연차 잔여 |
| GET | `/hr/leave-requests/my` | — | 내 휴가 신청 내역 |
| POST | `/hr/leave-requests` | — | 휴가 신청 |
| PATCH | `/hr/leave-requests/:id/cancel` | — | 신청 취소 |
| GET | `/hr/leave-requests` | AdminGuard | 전체 목록 (관리자) |
| PATCH | `/hr/leave-requests/:id/approve` | AdminGuard | 승인 |
| PATCH | `/hr/leave-requests/:id/reject` | AdminGuard | 반려 |

## 3. 아키텍처 설계 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| usr_id → emp_id 매핑 | `EmployeeEntity.usrId` FK 활용 | 이미 FK 존재, 추가 테이블 불필요 |
| 반차 Timesheet 코드 | 'AH' (Annual Half), workHours=4 | 기존 'AL'과 구분, 반일 근무 표현 |
| 병가/특별휴가 차감 | 연차 잔여 차감 안 함 | SICK, SPECIAL은 별도 카운트, 기록 목적 |
| 승인 권한 | AdminGuard (ADMIN/SUPER_ADMIN) | 기존 HR 관리자 권한 패턴 유지 |
| 테이블 생성 | TypeORM synchronize 활용 | 개발/스테이징 자동 생성, 별도 마이그레이션 불필요 |
| 승인 트랜잭션 | DataSource.transaction() | 상태변경 + 잔여차감 + Timesheet 삽입 원자성 보장 |
| 내 휴가 라우트 | `/my-leave` (HR 외부 독립) | 모든 직원이 접근, HR 메뉴 권한 불필요 |
| 관리자 승인 라우트 | `/hr/leave-requests` (HR 하위) | HR 관리자만 접근, 기존 HR 레이아웃 활용 |
| Pending 중복 차감 방지 | 신청 시 PENDING 건의 days 합산 검증 | 동시 다건 신청 시 잔여 초과 방지 |

## 4. 사이드 임팩트

### 4.1 안전한 변경
- 기존 LeaveController / LeaveService 변경 없음 (별도 `leave-request` 경로)
- 기존 LeavePage.tsx 변경 없음 (별도 LeaveRequestAdminPage 신규 생성)
- 기존 Timesheet 데이터에 INSERT/UPDATE만 (삭제 없음)
- 기존 LeaveBalance에 차감만 (승인 시 lvb_used ↑, lvb_remaining ↓)
- DB 마이그레이션 불필요 (TypeORM synchronize 자동 생성)

### 4.2 주의사항
- **승인 취소 기능 없음**: 승인 후 되돌리기 미구현 (Timesheet + LeaveBalance 롤백 필요, 향후 추가)
- **MyPage 비등록 직원**: usr_id → emp_id 매핑 없는 사용자는 연차 카드 미표시 (정상 동작)
- **동시 승인 경합**: 같은 직원의 여러 건을 동시 승인 시 LeaveBalance 정합성은 DB 트랜잭션으로 보호

## 7. 향후 개선 가능사항

1. **승인 취소 기능**: 승인 후 관리자가 되돌릴 수 있는 기능 (Timesheet 삭제 + LeaveBalance 복원)
2. **차감 일수 자동 표시**: 신청 모달에서 영업일 수 실시간 계산하여 미리보기
3. **알림 연동**: 신청/승인/반려 시 알림 (기존 NotificationService 연동)
4. **이메일 알림**: 관리자에게 신청 알림 메일, 직원에게 승인/반려 알림 메일
5. **대시보드 위젯**: 메인 대시보드에 "대기 중 휴가 신청 N건" 표시
6. **ChannelList DM 점**: 내 휴가 페이지에서 바로 채팅으로 관리자에게 문의 연동
7. **연차 자동 부여**: 입사일 기준 매년 자동 LeaveBalance 생성 스케줄러

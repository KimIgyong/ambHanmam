## 핵심 구현 상세

### 1. 메뉴 접근권한 4단계 필터 (`getMyMenus`)
```
1단계: 메뉴 활성화 check (menu_config.enabled)
2단계: ADMIN 바이패스 (SUPER_ADMIN/ADMIN → 전체 접근)
3단계: 사용자 오버라이드 → 역할 기본값
4단계 [NEW]: 그룹 필터
  - 사용자 소속 그룹 ID 조회 (amb_user_groups)
  - 메뉴별 그룹 설정 확인 (amb_menu_group_permissions)
  - 그룹 설정 없는 메뉴: 통과
  - 그룹 설정 있는 메뉴: 소속 그룹 중 하나라도 accessible=true → 통과 (OR 논리)
```

### 2. 회원가입 후 UX 흐름
```
RegisterPage → authService.register() 
  → 성공 시: success 화면 표시 (ACTIVE vs PENDING 분기 메시지)
  → "로그인 페이지로" 버튼 클릭 → /login 이동
  → 관리자가 MemberListTab에서 PENDING 멤버 승인/거부
```

### 3. 법인별 그룹/부서 관리
- EntityManagementPage 각 법인 카드에 ▼ 버튼으로 확장 영역 추가
- 2개 서브탭: 그룹 관리 / 부서 관리
- 기존 Group/Department CRUD API + FE 훅 재활용

### 4. RolePermissionTab 그룹 열
- USER_GROUP 서브탭: 기존 MANAGER/MEMBER/VIEWER 토글 옆에 그룹별 체크박스 열 추가
- 법인코드별 그룹 소팅, 가로 스크롤 지원
- 역할 토글(switch)과 그룹 체크박스(checkbox) 시각적 구분
- 저장 시 역할 권한 + 그룹 권한 동시 전송

## 1. 구현 요약

이메일 템플릿 관리 기능을 ADMIN/MASTER 권한 체계에 맞춰 구현하고, 실제 메일 발송 경로(`ACCOUNT_CREATED`, `INVITATION`)에 템플릿 적용을 연결하였다.

- ADMIN 전역 템플릿 관리 API/화면 추가 (`ACCOUNT_CREATED`)
- MASTER 법인별 템플릿 관리 API/화면 추가 (`INVITATION`)
- 포탈 계정 생성/법인 초대 메일 발송 로직에 DB 템플릿 해석 적용
- 메뉴코드/라우팅/i18n 연동
- 스테이징 배포 후 런타임 이슈(엔티티 메타데이터 누락) 핫픽스 반영

## 4. 배포 및 검증 결과

### 4.1 스테이징 배포

- `main` push 후 스테이징 서버에서 pull + deploy 완료
- API 컨테이너 정상 기동 확인

### 4.2 API 동작 검증

- ADMIN 템플릿 API (`/api/v1/settings/email-templates/:code`) 조회/저장/미리보기/삭제 정상
- MASTER 템플릿 API (`/api/v1/entity-settings/email-templates/:code`) 조회/저장/미리보기/삭제 정상
- 메뉴 권한 노출 확인
  - ADMIN: `SETTINGS_EMAIL_TEMPLATES`, `ENTITY_EMAIL_TEMPLATE`
  - MASTER: `ENTITY_EMAIL_TEMPLATE`

### 4.3 실발송 경로 검증

- `ACCOUNT_CREATED` 경로
  - 포탈 고객 계정 생성 API 호출 시 템플릿 해석 후 raw mail 발송 로그 확인
- `INVITATION` 경로
  - 법인 초대 생성 API 호출 시 템플릿 해석 후 raw mail 발송 로그 확인
- DB 사이드이펙트 검증
  - 초대 `inv_send_count` 증가 및 `inv_last_sent_at` 기록
  - 포탈 유저 매핑 생성 확인

### 4.4 테스트 데이터 정리

- 검증 중 생성한 테스트 데이터 정리 완료
  - `stg.invite.*@example.com` 초대 2건 삭제
  - `[STG-FLOW-*]` 템플릿 2건 삭제
  - 검증용 포탈 매핑 1건 삭제
- 연계 사용자 1건(`contact@amoeba.site`)은 운영 영향 고려하여 유지

## 5. 이슈 및 해결

### 이슈

- 배포 직후 템플릿 관련 API 500 에러 발생
- 원인: `EmailTemplateEntity`가 글로벌 TypeORM 엔티티 목록에서 누락

### 조치

- `apps/api/src/app.module.ts`에 `EmailTemplateEntity` import + entities 등록
- 핫픽스 커밋/재배포 후 동일 증상 재현되지 않음

## 7. 결론

이메일 템플릿 관리 기능은 계획 범위 내에서 구현/배포/실발송 검증까지 완료되었다. 배포 중 발견된 TypeORM 메타데이터 이슈는 핫픽스로 해결되었고, 현재 스테이징에서 ADMIN/MASTER 템플릿 편집과 메일 발송 경로 적용이 정상 동작한다.

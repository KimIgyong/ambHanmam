## 1. 작업 개요

기존 Department/Group 기반 조직 구조를 Unit/Cell/Team 모델로 전체 리네이밍.
DB 스키마, 백엔드, 프론트엔드, 공유 타입을 일관되게 변경.

### 리네이밍 매핑

| 기존 | 변경 | 범위 |
|------|------|------|
| Department | Unit | 테이블, 컬럼, 코드, UI |
| Group | Cell | 테이블, 컬럼, 코드, UI |
| user_dept_roles | user_unit_roles | 테이블, 엔티티 |
| user_groups | user_cells | 테이블, 엔티티 |
| menu_group_permissions | menu_cell_permissions | 테이블, 엔티티 |
| DEPARTMENT_HEAD | UNIT_HEAD | 데이터 값 |
| DEPARTMENTS (메뉴코드) | UNITS | 데이터 값 |

## 5. 스테이징/프로덕션 배포 시 주의사항

1. **DB 마이그레이션 먼저 실행** (TypeORM synchronize 의존하지 않고 직접 실행 권장)
   ```bash
   psql -U ambmanager -d ambmanagement -f sql/migration_org_structure.sql
   ```
2. **롤백 필요 시**
   ```bash
   psql -U ambmanager -d ambmanagement -f sql/migration_org_structure_rollback.sql
   ```
3. TypeORM `synchronize: true`가 개발 환경에서 자동으로 누락된 컬럼/테이블을 생성하지만, 프로덕션에서는 반드시 마이그레이션 SQL로 실행할 것

## 2. 구현 내용

### 2.1 백엔드 (NestJS)

#### 새 엔티티 (5개)
| 엔티티 | 테이블명 | 용도 |
|--------|----------|------|
| `MeetingNoteProjectEntity` | `amb_meeting_note_projects` | 노트-프로젝트 연결 (M:N) |
| `MeetingNoteParticipantEntity` | `amb_meeting_note_participants` | 노트-관련자 연결 (M:N) |
| `MeetingNoteIssueEntity` | `amb_meeting_note_issues` | 노트-이슈 연결 (M:N) |
| `MeetingNoteCommentEntity` | `amb_meeting_note_comments` | 코멘트 (soft delete) |
| `MeetingNoteRatingEntity` | `amb_meeting_note_ratings` | 별점 (1~5, 유니크 user+note) |

#### 엔티티 수정
- `MeetingNoteEntity`: `mtn_assignee_id` (UUID nullable) 컬럼 추가, `assignee` ManyToOne 관계, 5개 OneToMany 관계 추가

#### 새 API 엔드포인트
| Method | Path | 용도 |
|--------|------|------|
| `GET` | `/meeting-notes/:id/comments` | 코멘트 목록 조회 |
| `POST` | `/meeting-notes/:id/comments` | 코멘트 작성 |
| `DELETE` | `/meeting-notes/:id/comments/:commentId` | 코멘트 삭제 (본인만) |
| `PUT` | `/meeting-notes/:id/rating` | 별점 등록/수정 (upsert) |

#### 서비스 주요 변경
- `syncParticipants()`, `syncProjects()`, `syncIssues()`: delete-all + re-insert 패턴
- `getRatingStats()`, `getCommentCounts()`, `getMyRatings()`: 집계 쿼리
- 목록/상세 조회 시 `commentCount`, `avgRating`, `ratingCount`, `myRating` 포함

#### DTO 수정
- `CreateMeetingNoteRequest`: `assignee_id`, `participant_ids`, `project_ids`, `issue_ids` 추가
- `UpdateMeetingNoteRequest`: 동일 필드 추가 (모두 optional)
- `CreateMeetingNoteCommentRequest` (신규): `content` 필드

## 5. DB 마이그레이션 참고

TypeORM `synchronize: true` 설정 시 자동 생성되는 테이블:
- `amb_meeting_note_projects` (Unique: mtn_id + pjt_id)
- `amb_meeting_note_participants` (Unique: mtn_id + usr_id)
- `amb_meeting_note_issues` (Unique: mtn_id + iss_id)
- `amb_meeting_note_comments` (Soft delete: mnc_deleted_at)
- `amb_meeting_note_ratings` (Unique: mtn_id + usr_id, Check: mnr_score 1~5)
- `amb_meeting_notes` 테이블에 `mtn_assignee_id` 컬럼 추가

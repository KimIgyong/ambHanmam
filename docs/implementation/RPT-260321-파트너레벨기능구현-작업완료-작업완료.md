## 2. 구현 내용

### 2.1 아키텍처 결정
- 기존 CLIENT_LEVEL 패턴을 참고하여 동일한 아키텍처로 구현
- `apps/web` + `apps/api` 단일 앱 내 PARTNER_LEVEL 역할 추가
- `/partner/*` 라우트로 격리된 Partner Portal UI 제공
- PARTNER_LEVEL 전용 JWT 토큰(`partnerId` 포함)으로 데이터 접근 범위 제한

### 2.2 Phase 1: 기반 인프라 (타입 시스템, 인증/인가)

#### 타입 시스템 확장 (packages/types)
- `user-level.types.ts`: `PARTNER_LEVEL`, `PARTNER_ADMIN`, `PARTNER_MEMBER` 역할, `PARTNER_INVITE` 가입방식 추가
- `permission.types.ts`: `DATA_SCOPE.PARTNER`, `ROLE_DATA_SCOPE`에 파트너 역할 매핑 추가
- `partner-app.types.ts`: 파트너 앱 상태(`DRAFT`/`SUBMITTED`/`IN_REVIEW`/`APPROVED`/`REJECTED`/`PUBLISHED`/`SUSPENDED`), 카테고리, 인증방식, 오픈모드 타입 정의

#### 인증/인가 체계
- `UserEntity`에 `usr_partner_id` (파트너사 FK) 컬럼 추가
- JWT Payload에 `partnerId` 포함
- `DataScopeInterceptor`에 PARTNER 스코프 처리 분기 추가
- `RolesGuard` ROLE_HIERARCHY에 `PARTNER_ADMIN: 0`, `PARTNER_MEMBER: 0` 추가
- `auth.decorator.ts`에 `PartnerOnly()`, `PartnerAdminOnly()` 데코레이터 추가

### 2.3 Phase 2: Admin 파트너 관리 및 초대

#### 파트너 조직 모듈 (apps/api/src/domain/partner/)
- `PartnerOrganizationEntity` (`amb_partners`) — 파트너사 정보 (코드, 이름, 국가, 연락처, 상태)
- `PartnerService` — 파트너사 CRUD
- `PartnerController` (`/api/v1/admin/partners/*`) — Admin 전용 API

#### 초대 시스템 확장
- `InvitationEntity`에 `inv_partner_id` 컬럼 추가 (파트너사 연결)
- `InvitationService`에 PARTNER_LEVEL 초대/수락 로직 추가
- `EntityMemberService`에 PARTNER_LEVEL 초대 분기 추가

#### Admin UI
- `PartnerManagementPage.tsx` — 파트너사 관리 페이지 (목록, 등록, 수정)
- `InvitePartnerUserModal` — PartnerManagementPage 내 파트너 사용자 초대 모달

### 2.4 Phase 3: Partner Portal (인증, 레이아웃, 페이지)

## 3. 배포 이슈 및 해결

### 3.1 TypeORM Object type 에러
- **현상**: API 시작 시 `Data type "Object" in "PartnerAppEntity.papIcon" is not supported by "postgres"` 에러
- **원인**: TypeScript `string | null` 유니온 타입이 decorator metadata에서 `Object`로 컴파일됨
- **해결**: nullable string 컬럼에 explicit `type: 'varchar'` 지정 (커밋 `8b7caa1`)
- **영향 파일**: `partner-app.entity.ts`, `partner-app-version.entity.ts`, `partner-app-install.entity.ts`

### 3.2 PartnerOrganizationEntity 동일 이슈
- **해결**: Phase 1+2 커밋(`e716b92`) 에서 동일 패턴 수정

## 2. Phase 1 구현 내역 (프로덕션 배포 전 필수)

### P1. XSS — DOMPurify 적용
- `dompurify` + `@types/dompurify` 설치
- `sanitizeHtml()` 유틸 함수 생성 (`apps/web/src/global/util/sanitize.ts`)
- `dangerouslySetInnerHTML` 사용 11개 파일에 DOMPurify.sanitize() 적용
- **변경 파일**: `NoticeDetailPage`, `TodayHistoryPage`, `TeamTodayPanel`, `AllTodayPanel`, `MemberDetailModal`, `MissionSection`, `TodoDetailModal`, `IssueDetailModal`, `TranslationPanel`, `MeetingNoteDetailPage`, `NoteCommentSection`, `ClientIssueDetailPage`

### P2. SSE 채널 접근 권한 검증
- `message.controller.ts` SSE 엔드포인트에 채널 멤버십 검증 추가
- `channel.service.ts`에 `isMember()` 메서드 활용
- 비멤버 요청 시 ForbiddenException 반환

### P3. IDOR — Todo 소유자 검증
- `todo.service.ts` `getTodoById()`에 소유자/참여자/공개여부 검증 로직 추가
- PRIVATE: 소유자 + 참여자만 조회 가능
- SHARED: 같은 Entity 소속만 조회 가능

### P4. 파일 다운로드 인증
- `file.controller.ts`에서 `@Public()` 데코레이터 제거
- JWT 인증 적용, 서명된 임시 URL 메커니즘 추가
- 프론트엔드 다운로드 요청에 인증 헤더 포함 확인

### P5. Docker non-root 실행
- staging Dockerfile 4개에 non-root user(nodejs:1001) 추가
- `docker-entrypoint-api.sh` 스크립트로 uploads 볼륨 권한 영구 해결
- 후속 패치 2건: 권한 오류 수정 (`6bc0a36`), entrypoint 방식 전환 (`8e184e1`)

### P6. Nginx 보안 헤더
- `nginx.conf`, `nginx-portal.conf`에 보안 헤더 추가:
  - `Strict-Transport-Security` (HSTS)
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: SAMEORIGIN` (portal iframe 호환)
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Permissions-Policy: camera=(), microphone=(), geolocation=()`
  - `Content-Security-Policy: frame-ancestors *.amoeba.site`
- 후속 패치: DENY → SAMEORIGIN + CSP frame-ancestors 조정 (`4aea255`)

### P7. 비밀번호 복잡도 강화
- 4개 DTO에 `@Matches()` 정규식 추가 (대/소문자, 숫자, 특수문자 1개 이상 필수)
  - `change-password.request.ts`, `register.request.ts`, `reset-password.request.ts` (api + portal-api)

### P8. 세션 유휴 타임아웃
- 확인 결과 JWT accessToken TTL(4시간) + refreshToken 로테이션으로 기존 구현됨
- 프론트엔드 `useIdleTimeout` 추가 구현은 Phase 3에서 선택적으로 진행

### P9. Path Traversal URL 인코딩 방어
- `file.controller.ts`에 `decodeURIComponent()` 후 재검증 로직 추가
- null byte(`%00`), 이중 인코딩 패턴 차단

## 3. Phase 2 구현 내역 (단기 보안 강화)

### P10. usrPostalPassword 컬럼 제거
- `user.entity.ts`에서 `usrPostalPassword` 필드 완전 제거
- 코드베이스 전수 검색 확인 → 미사용 확인 후 제거 결정

### P11. 출석 서비스 트랜잭션 래핑
- `attendance.service.ts` `createAttendances()`를 `dataSource.transaction()` 래핑
- 다건 save 중 부분 실패 시 전체 롤백 보장

### P12. Postal 웹훅 서명 검증 강화
- `webmail.controller.ts`에 `timingSafeEqual()` 적용
- 타이밍 사이드채널 공격 방어

### P13. 파일 매직 바이트 검증
- `file-type@16.5.4` 라이브러리 추가 (`apps/api/package.json`)
- `file.service.ts`에 업로드 파일 헤더 검증 로직 추가
- MIME 타입과 매직 바이트 불일치 시 업로드 거부

### P14. 계정 잠금 로직
- `user.entity.ts`에 `usrFailedLoginCount` (int, default 0), `usrLockedUntil` (timestamp, nullable) 컬럼 추가
- `auth.service.ts`에 로그인 실패 카운터 + 5회 연속 실패 시 15분 잠금 로직 구현
- 잠금 해제 후 자동 카운터 리셋

### P15. DB 자동 백업 스크립트
- `scripts/db-backup.sh` 생성 (pg_dump + gzip, 30일 보관)
- 컨테이너 자동 감지 (production > staging 우선순위)
- 백업 파일 최소 1KB 검증
- 스테이징 서버 crontab 등록: `0 2 * * *`
- 수동 테스트 완료 (5.9MB 백업 성공)

### P16. Access Token tokenVersion 검증
- `auth.service.ts` → accessToken payload에 `tokenVersion` 포함
- `jwt.strategy.ts` `validate()` → DB tokenVersion과 대조 검증
- 불일치 시 UnauthorizedException → 강제 로그아웃

### P17. 데이터 수정/삭제 감사 로그
- `data-audit-log.entity.ts`: `amb_data_audit_log` 테이블 + 3개 인덱스
- `data-audit.subscriber.ts`: TypeORM EntitySubscriber로 10개 주요 엔티티 변경 감시
  - 대상: User, Conversation, Todo, Notice, Issue, Project, Contract, Invoice, Partner, Calendar
- `request-context.ts`: AsyncLocalStorage로 요청 컨텍스트 전파
- `audit-context.interceptor.ts`: NestJS Interceptor로 userId를 AsyncLocalStorage에 설정
- `main.ts`: REQUEST_CONTEXT 미들웨어 + AuditContextInterceptor 글로벌 등록
- `acl.module.ts`: DataAuditLogEntity + DataAuditSubscriber 등록

## 1. 구현 내용 요약

Meeting Notes(메모/회의록) 모듈에 Obsidian 스타일의 지식 관리 기능을 추가하였다. 노트 간 양방향 링크(`[[Wiki Link]]`), 전문 검색(Full-text Search), 그래프 시각화, 데일리 노트 자동 생성을 포함한 6개 Phase를 전체 구현하였다.

| Phase | 구현 내용 | 상태 |
|-------|----------|------|
| **Phase 1** | DB 스키마 — 노트 링크/노트↔태스크 테이블 + FTS tsvector 컬럼 | ✅ 완료 |
| **Phase 2** | WikiLink Extension + 백링크 패널 + 자동완성 | ✅ 완료 |
| **Phase 3** | PostgreSQL FTS 전문검색 + Cmd+K 커맨드 팔레트 | ✅ 완료 |
| **Phase 4** | 노트 그래프 시각화 (react-force-graph-2d) | ✅ 완료 |
| **Phase 5** | 데일리 노트 자동생성 + 미션/체크 연동 | ✅ 완료 |
| **Phase 6** | i18n 다국어 (ko/en/vi) + 라우터 등록 | ✅ 완료 |

## 2. 기능 상세

### 2.1 양방향 링크 (WikiLink)

**사용 방법**: 리치텍스트 에디터에서 `[[` 입력 → 자동완성 드롭다운 표시 → 노트/미션/태스크/이슈/프로젝트 선택

| 항목 | 구현 내용 |
|------|----------|
| 트리거 | `[[` 문자 입력 시 감지 (TipTap `onUpdate` + `onKeyDown` 핸들링) |
| 자동완성 | debounce 300ms, API `GET /meeting-notes/link-autocomplete?q=&type=` 호출 |
| 렌더링 | 인라인 칩(chip) — 타입별 색상 구분 (노트: indigo, 허브: emerald, 이슈: red, 프로젝트: green, 미션: amber) |
| 타입 프리픽스 | `[[미션:`, `[[태스크:`, `[[이슈:`, `[[프로젝트:` 지원 |
| 저장 | HTML `<span data-type="wikiLink" data-link-text="..." data-target-type="..." data-target-id="...">` 형태로 직렬화 |
| 백엔드 동기화 | 노트 저장/수정 시 `parseAndSaveLinks()` → `amb_note_links` 테이블에 UPSERT |

### 2.2 백링크 패널

**위치**: 노트 상세 페이지(`MeetingNoteDetailPage`) 하단, 댓글 섹션 아래

| 항목 | 구현 내용 |
|------|----------|
| API | `GET /meeting-notes/:id/backlinks` |
| 표시 내용 | 해당 노트를 참조하는 소스 노트 제목 + 링크 텍스트 + 주변 콘텍스트 스니펫 |
| 네비게이션 | 클릭 시 소스 노트 상세 페이지로 이동 |
| 캐싱 | React Query — `['meeting-notes', 'backlinks', noteId]` 키 |

### 2.3 전문 검색 (Full-text Search)

**방식**: PostgreSQL `tsvector` + GIN 인덱스 기반

| 항목 | 구현 내용 |
|------|----------|
| 인덱스 | `mtn_search_vector` 컬럼 (tsvector) + GIN 인덱스 |
| 업데이트 타이밍 | 노트 생성/수정 시 `updateSearchVector()` — HTML 태그 제거 후 `to_tsvector('simple', title + content)` |
| 검색 API | `GET /meeting-notes/search?q=&scope=MY|ENTITY` |
| 매칭 | `plainto_tsquery('simple', query)` + `ts_rank` 점수 정렬 |
| 스니펫 | `ts_headline`으로 매칭 구간 하이라이트 |
| 기존 노트 | 마이그레이션 SQL로 106건 일괄 업데이트 완료 |

### 2.4 커맨드 팔레트 (NoteSearch)

**단축키**: `Cmd+K` (macOS) / `Ctrl+K` (Windows/Linux)

| 항목 | 구현 내용 |
|------|----------|
| UI | 모달 오버레이 + 검색 입력 + 결과 리스트 |
| 키보드 네비게이션 | `↑` / `↓` 항목 이동, `Enter` 선택, `Escape` 닫기 |
| debounce | 300ms |
| 결과 표시 | 제목 + 스니펫(하이라이트) + 작성일 + 점수 |
| 접근 | MeetingNotesPage 상단 검색 아이콘 버튼 또는 키보드 단축키 |

### 2.5 그래프 뷰

**라이브러리**: `react-force-graph-2d` v1.29.1 (D3 force simulation 기반)

| 항목 | 구현 내용 |
|------|----------|
| 경로 | `/meeting-notes/graph` |
| 노드 타입 | NOTE (indigo), TASK (purple), ISSUE (red), PROJECT (green), MISSION (amber) |
| 노드 크기 | 연결 수 비례 (최소 4, 최대 16) |
| 엣지 | WikiLink 기반 (`amb_note_links`) |
| API | `GET /meeting-notes/graph?scope=MY|ENTITY` |
| 필터 | 타입별 토글 필터 + MY/ENTITY 스코프 전환 |
| 인터랙션 | 노드 클릭 → 노트 상세 페이지 이동 |
| 코드 스플릿 | `React.lazy` 동적 import → 별도 178KB 청크 (메인 번들 비영향) |
| 접근 | MeetingNotesPage 상단 그래프 아이콘 버튼 |

### 2.6 데일리 노트 + 미션 연동

| 항목 | 구현 내용 |
|------|----------|
| 자동 생성 | `saveMission()` 실행 시 `getOrCreateDailyNote()` → 해당일 데일리 노트 없으면 자동 생성 |
| 폴더 배정 | "데일리 노트" 폴더 자동 생성/배정 |
| 미션 내용 동기화 | 미션 저장 → 데일리 노트에 "오늘의 미션" 섹션 HTML 추가 |
| 체크 결과 반영 | `saveCheck()` 완료 → 데일리 노트에 "미션 점검 결과" 섹션 추가 (✅완료/🔄진행중/⏭️해당없음) |
| 에러 격리 | try-catch로 데일리 노트 실패가 미션 저장을 블로킹하지 않음 |

## 4. DB 변경사항

### 4.1 신규 테이블

#### `amb_note_links` — 노트 간 WikiLink 관계

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `nlk_id` | UUID PK | 링크 고유 ID |
| `ent_id` | UUID NOT NULL | 법인 ID (데이터 격리) |
| `nlk_source_note_id` | UUID FK → amb_meeting_notes | 소스 노트 |
| `nlk_target_note_id` | UUID FK → amb_meeting_notes (nullable) | 타겟 노트 (미존재 시 NULL) |
| `nlk_link_text` | VARCHAR(500) | 링크 텍스트 |
| `nlk_target_type` | VARCHAR(20) DEFAULT 'NOTE' | NOTE/MISSION/TASK/ISSUE/PROJECT |
| `nlk_target_ref_id` | UUID (nullable) | 타입별 참조 ID (non-NOTE) |
| `nlk_context` | TEXT (nullable) | 주변 텍스트 스니펫 |
| `nlk_created_at` | TIMESTAMPTZ | 생성일 |

**인덱스**: `idx_note_links_source`, `idx_note_links_target`, `idx_note_links_ent`, `idx_note_links_unique` (source_note_id + link_text + target_type UNIQUE)

#### `amb_meeting_note_todos` — 노트↔태스크 M:N 연결

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `mnt_id` | UUID PK | 연결 고유 ID |
| `mtn_id` | UUID FK → amb_meeting_notes | 노트 ID |
| `tdo_id` | UUID FK → amb_todos | 태스크 ID |
| `mnt_created_at` | TIMESTAMPTZ | 생성일 |

**인덱스**: `idx_mnt_unique` (mtn_id + tdo_id UNIQUE)

| F3 | 프로덕션 배포 | 높음 | 스테이징 검증 완료 후 프로덕션 DB 마이그레이션 + 배포 |
| F4 | KMS 태그 연동 | 낮음 | 그래프 뷰에 KMS 태그 노드 추가 |
| F5 | 마크다운 이중 저장 | 낮음 | HTML + Markdown 동시 저장 (선택적) |

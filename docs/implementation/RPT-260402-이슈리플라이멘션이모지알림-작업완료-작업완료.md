# RPT-이슈리플라이멘션이모지알림-작업완료보고-20260402

## 1. 구현 내용 요약

### R1: 리플라이 에디터 멘션/이모지 지원
- IssueDetailModal.tsx 리플라이 RichTextEditor에 `enableMention={true}` 추가
- 리플라이 하단에 이모지 리액션 버튼(👍✅🙏😀❤️) 렌더링 추가

### R2: 리플라이 자동 멘션 알림
- issue.service.ts `addIssueComment`에서 parentComment를 함수 스코프로 추출
- 멘션 없는 리플라이 시 원래 코멘트 작성자에게 자동 MENTION_EVENT 발행
- 본인 리플라이(자기 코멘트에 답글) 시 알림 제외

### R3: 이모지 리액션 알림
- notification-type.constant.ts에 `COMMENT_REACTION` 타입, `REACTION_EVENT`, `CommentReactionEvent` 인터페이스 추가
- issue.service.ts `toggleCommentReaction`에서 리액션 **추가** 시 `REACTION_EVENT` emit (제거 시 미발행)
- notification.listener.ts에 `handleCommentReaction` 핸들러 추가 (이모지 매핑: LIKE→👍, CHECK→✅, PRAY→🙏, GRIN→😀, LOVE→❤️)
- 본인 코멘트 리액션 시 알림 제외

### R4: 이슈 별점 알림
- notification-type.constant.ts에 `CONTENT_RATING` 타입, `RATING_EVENT`, `ContentRatingEvent` 인터페이스 추가
- content-rating.service.ts에 `EventEmitter2`, `UserEntity` 주입, `upsertIssueRating` 후 `RATING_EVENT` emit
- notification.listener.ts에 `handleContentRating` 핸들러 추가
- 이슈 작성자(issReporterId)에게 "X님이 이슈에 ⭐N개를 주셨습니다." 알림

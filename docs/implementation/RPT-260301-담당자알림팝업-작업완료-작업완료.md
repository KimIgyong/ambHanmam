## 1. 구현 내용 요약

할일(Todo), 이슈(Issue), 회의록(MeetingNote)에서 담당자/참여자로 지정된 사용자에게 실시간 알림 팝업을 표시하는 기능을 완전히 구현했습니다.

### 1.1 백엔드

| 구현 항목 | 설명 |
|-----------|------|
| DDL | `amb_notifications` 테이블 생성 (UUID PK, 인덱스 3개) |
| Entity | `NotificationEntity` - TypeORM 엔티티 + ManyToOne 관계 |
| Service | `NotificationService` - 알림 CRUD (생성, 조회, 읽음 처리) |
| SSE Service | `NotificationSseService` - RxJS Subject 기반 실시간 스트림 |
| Listener | `NotificationListener` - `@OnEvent('assignee.assigned')` 핸들러 |
| Controller | REST API 5개 엔드포인트 + SSE 스트림 |
| 이벤트 emit | Todo/Issue/MeetingNote 서비스에 EventEmitter2 호출 추가 |

### 1.2 프론트엔드

| 구현 항목 | 설명 |
|-----------|------|
| Toaster | sonner 라이브러리 + App.tsx에 Toaster 컴포넌트 설정 |
| SSE Hook | `useNotificationSSE` - EventSource 연결 + 토스트 표시 |
| Store | `notification.store.ts` - Zustand 기반 상태 관리 |
| NotificationBell | 벨 아이콘 + 미읽음 배지 + 드롭다운 목록 + 읽음 처리 |
| i18n | ko/en/vi 3개 언어 번역 파일 |
| Layout 통합 | MainLayout 헤더에 알림벨 + SSE 훅 통합 |

## 7. 배포 가이드

### DDL 마이그레이션 (필수)
스테이징/프로덕션 배포 전 DB에 DDL을 적용해야 합니다:

```bash
# 스테이징
ssh amb-staging
cd ~/ambManagement
psql -U amb_user -d db_amb -f sql/migration_notification.sql

# 프로덕션
ssh amb-production
cd ~/ambManagement
psql -U amb_user -d db_amb -f sql/migration_notification.sql
```

### 배포 절차
1. PR #12 머지 (main 브랜치)
2. 스테이징 서버에서 DDL 실행
3. 스테이징 배포: `bash docker/staging/deploy-staging.sh`
4. 스테이징 테스트 완료 후 `main → production` PR 생성
5. 프로덕션 서버에서 DDL 실행
6. 프로덕션 배포: `bash docker/production/deploy-production.sh`

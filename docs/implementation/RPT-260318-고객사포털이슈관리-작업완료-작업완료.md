## 2. 구현 내용

### 2.1 아키텍처 결정
- **Option C 채택**: 기존 `apps/web` + `apps/api`에 CLIENT_LEVEL 역할 추가
- 별도 앱 분리 없이 `/client/*` 라우트로 격리된 포털 UI 제공
- CLIENT_LEVEL 전용 JWT 토큰으로 데이터 접근 범위 제한

### 2.2 백엔드 (Phase 1 + Phase 2)

#### 타입 시스템 확장 (packages/types)
- `USER_LEVEL_CODE`에 `CLIENT_LEVEL` 추가
- `USER_ROLE`에 `CLIENT_ADMIN`, `CLIENT_MEMBER` 추가
- `JOIN_METHOD`에 `CLIENT_INVITE` 추가
- `ROLE_DATA_SCOPE`에 CLIENT 역할 데이터 범위 (`OWN_ONLY`) 추가

#### 인증/인가 체계
- `UserEntity`에 `usr_cli_id` (고객사 FK) 컬럼 추가
- JWT Payload에 `cliId` 포함
- `DataScopeInterceptor`에 CLIENT scope 추가
- `RolesGuard` ROLE_HIERARCHY에 CLIENT 역할 추가
- `ClientGuard` — CLIENT_LEVEL 전용 접근 제어 가드

#### 엔티티
- `ProjectClientEntity` (`kms_project_clients`) — 프로젝트-고객사 매핑
- `ClientInvitationEntity` (`amb_client_invitations`) — 초대 관리

#### 서비스
- `ClientAuthService` — 로그인, 초대 검증, 회원가입, 프로필 관리, 비밀번호 변경
- `ClientProjectService` — 고객사별 프로젝트 조회, 이슈 수 집계
- `ClientIssueService` — 이슈 등록(BUG/REQUEST/INQUIRY), 조회, 댓글, 해결 확인

#### 컨트롤러
- `ClientAuthController` (`/api/v1/client-auth/*`) — 로그인, 회원가입, 프로필
- `ClientProjectController` (`/api/v1/client/projects/*`) — 프로젝트 목록/상세
- `ClientIssueController` (`/api/v1/client/issues/*`) — 이슈 CRUD, 댓글, 확인
- `ClientManagementController` (`/api/v1/client-management/*`) — 관리자용 초대
- `ProjectController`에 프로젝트-고객사 매핑 엔드포인트 3개 추가

#### 모듈
- `ClientPortalModule` — 7개 엔티티, 3개 서비스, 4개 컨트롤러 등록
- `AppModule`에 ClientPortalModule 등록

### 2.4 DB 마이그레이션
```sql
-- amb_users 테이블에 고객사 FK 추가
ALTER TABLE amb_users ADD COLUMN IF NOT EXISTS usr_cli_id UUID REFERENCES amb_svc_clients(cli_id);
CREATE INDEX IF NOT EXISTS idx_users_cli_id ON amb_users(usr_cli_id) WHERE usr_cli_id IS NOT NULL;

-- 프로젝트-고객사 매핑 테이블
CREATE TABLE IF NOT EXISTS kms_project_clients (
  pcl_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  pcl_pjt_id UUID NOT NULL REFERENCES kms_projects(pjt_id),
  pcl_cli_id UUID NOT NULL REFERENCES amb_svc_clients(cli_id),
  pcl_assigned_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  pcl_assigned_by UUID REFERENCES amb_users(usr_id),
  UNIQUE(pcl_pjt_id, pcl_cli_id)
);

-- 고객사 초대 테이블
CREATE TABLE IF NOT EXISTS amb_client_invitations (
  civ_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  civ_cli_id UUID NOT NULL REFERENCES amb_svc_clients(cli_id),
  civ_email VARCHAR(255) NOT NULL,
  civ_name VARCHAR(100),
  civ_token VARCHAR(255) NOT NULL UNIQUE,
  civ_status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  civ_role VARCHAR(50) NOT NULL DEFAULT 'CLIENT_MEMBER',
  civ_token_expires TIMESTAMPTZ NOT NULL,
  civ_invited_by UUID REFERENCES amb_users(usr_id),
  civ_accepted_at TIMESTAMPTZ,
  civ_created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  civ_updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

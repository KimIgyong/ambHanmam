## 3. DB 변경

### 마이그레이션 (sql/260413-contract-payment-schedule.sql)

1. `amb_bil_contracts` 테이블: `ctr_payment_type VARCHAR(20)`, `ctr_billing_amount DECIMAL(15,2)` 컬럼 추가
2. `amb_bil_payment_schedules` 테이블 신규 생성
   - PK: `pms_id` (UUID)
   - FK: `ctr_id` → `amb_bil_contracts.ctr_id` ON DELETE CASCADE
   - 컬럼: `pms_seq`, `pms_billing_date`, `pms_billing_period`, `pms_amount`, `pms_status`
   - 인덱스: `idx_bil_pms_contract` (ctr_id)

> **배포 시 SQL 실행 필요**: 스테이징/프로덕션 DB에서 마이그레이션 SQL 실행 후 배포

## 4. 설계 결정

- **별도 테이블(amb_bil_payment_schedules)**: 마일스톤과 독립적으로 관리 (마일스톤은 완료율 기반, 스케줄은 날짜 기반)
- **delete + recreate 패턴**: update 시 기존 스케줄 전체 삭제 후 재생성 (마일스톤과 동일 패턴)
- **하위 호환성**: `ctr_payment_type=NULL`은 기존 계약에 영향 없음
- **신규 등록 파일 첨부**: 계약 생성 후 contractId로 순차 업로드 (기존 DocumentManager는 편집 모드 전용)

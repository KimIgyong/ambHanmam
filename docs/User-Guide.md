## 1. 오늘의할일 (TODO)

> **경로**: 사이드바 > 오늘의할일 (`/todos`)

### 1.1 개요

개인 및 팀 단위의 할일을 관리하는 칸반 기반 도구입니다. 마감일 기준으로 **완료 / 오늘 / 예정**의 3개 컬럼으로 자동 분류되며, 개인 · 그룹 · 전사 범위로 할일을 조회할 수 있습니다.

### 1.2 화면 구성

| 영역 | 설명 |
|------|------|
| **탭 바** | `내 할일` · `그룹` · `전사` 3개 탭으로 조회 범위 전환 |
| **칸반 보드** | 3개 컬럼 — 완료(초록), 오늘(파랑), 예정(주황) |
| **할일 카드** | 제목, 마감일, 상태 배지, 태그, 참여자 아바타 표시 |
| **+ 할일 추가** | 우상단 버튼 (내 할일 탭에서만 표시) |

### 1.3 주요 기능

#### 할일 생성
1. 우상단 **[+ 할일 추가]** 버튼 클릭
2. 입력 항목:
   - **제목** (필수)
   - **설명** (선택, 텍스트)
   - **마감일** (필수, 날짜)
   - **태그** (선택, 쉼표로 구분)
   - **연결 이슈** (선택, 기존 이슈 연결)
   - **참여자** (선택, 멤버 다중 선택)
3. 저장 시 자동 번역 대화상자가 표시됩니다 (한→영/베트남어)

#### 상태 관리
할일 카드의 체크 아이콘을 클릭하면 상태가 순환됩니다.

| 상태 | 의미 | 배지 색상 |
|------|------|-----------|
| `SCHEDULED` | 예정 | 주황 |
| `IN_PROGRESS` | 진행중 | 파랑 |
| `COMPLETED` | 완료 | 초록 |

#### 자동 분류 규칙
- **완료 컬럼**: 상태가 `COMPLETED`인 할일
- **오늘 컬럼**: 마감일이 오늘 이하 (기한 초과 포함)인 진행중/예정 할일
- **예정 컬럼**: 마감일이 내일 이후인 미완료 할일

#### 상세보기 / 수정 / 삭제
- 카드 클릭 → **상세 모달** (제목, 설명, 상태, 참여자, 연결 이슈 표시)
- 카드 메뉴 → **수정 / 삭제** 선택
- **상태 변경 이력** 조회 가능 (히스토리 아이콘 클릭)

#### 그룹 / 전사 탭
- **그룹 탭**: 같은 부서 소속 멤버들의 할일 조회
- **전사 탭**: 전체 회사의 할일 조회
- 조회 전용이며, 다른 사람의 할일을 수정할 수 없습니다

## 2. 노트 (회의록/메모)

> **경로**: 사이드바 > 노트 (`/meeting-notes`)

### 2.1 개요

회의록(Meeting Note)과 메모(Memo) 두 가지 유형의 노트를 작성· 관리하는 도구입니다. 공개 범위를 설정하여 개인 메모부터 전사 공유 회의록까지 다양한 용도로 활용할 수 있습니다.

### 2.2 화면 구성

| 영역 | 설명 |
|------|------|
| **유형 필터** | `전체` · `회의록` · `메모` 버튼 |
| **공개범위 필터** | `전체` · `비공개` · `그룹` · `법인` · `공개` 버튼 |
| **카드 그리드** | 2열 카드 레이아웃 (모바일 1열) |
| **+ 새 노트** | 우상단 생성 버튼 |

### 2.3 주요 기능

#### 노트 생성
1. **[+ 새 노트]** 버튼 클릭
2. 입력 항목:
   - **유형**: 회의록 / 메모
   - **제목** (필수)
   - **내용** (리치 텍스트 에디터, HTML 서식 지원)
   - **회의 일자** (회의록 유형 시 표시)
   - **공개 범위**: PRIVATE / GROUP / ENTITY / PUBLIC
   - **부서** (GROUP 범위 선택 시)
3. 저장 후 목록에 카드로 표시됩니다

#### 공개 범위

| 범위 | 조회 대상 |
|------|-----------|
| `PRIVATE` | 작성자 본인만 |
| `GROUP` | 지정된 부서/그룹 멤버 |
| `ENTITY` | 같은 법인 소속 전체 |
| `PUBLIC` | 전사 공개 |

#### 상세 페이지
- 카드 클릭 → 상세 페이지(`/meeting-notes/:id`)로 이동
- HTML 형식 노트 내용 렌더링
- **번역 패널**: 다국어(한/영/베트남어) 자동 번역 지원
- **수정 / 삭제** 버튼 (작성자만 표시)

## 3. 출근스케줄

> **경로**: 사이드바 > 출근스케줄 (`/work-schedule`)

### 3.1 개요

팀원들의 주간 근무 일정을 한 눈에 확인하고 등록하는 도구입니다. 주간 단위 표 형식으로 각 멤버의 월~금 근무 유형과 시간이 표시됩니다.

### 3.2 화면 구성

| 영역 | 설명 |
|------|------|
| **주간 네비게이션** | `<` `>` 버튼으로 이전/다음 주 이동. 기본은 다음 주 월요일 |
| **스케줄 테이블** | 행: 멤버, 열: 월~금. 본인이 항상 최상단 표시 |
| **셀 내용** | 근무유형 배지 + 시작~종료 시간 |
| **편집 폼** | 셀 클릭 시 하단에 근무유형·시작시간 선택 폼 |
| **NOTE 컬럼** | 비고란 (재택 근무일 등 표시) |

### 3.3 근무 유형

| 유형 | 표시 | 시간 자동 계산 |
|------|------|---------------|
| `WORK` | 출근 (회색) | 시작시간 + 9시간 |
| `REMOTE` | 재택 (초록) | 시작시간 + 9시간 |
| `DAY_OFF` | 휴가 (빨강) | 시간 표시 없음 |
| `AM_HALF` | 오전반차 (노랑) | 13:00 ~ 17:00 (오후만 근무) |
| `PM_HALF` | 오후반차 (주황) | 시작시간 ~ 12:00 (오전만 근무) |

### 3.4 주요 기능

#### 일정 등록/수정
1. 본인 행의 셀 클릭 (편집 가능한 날짜만 선택 가능)
2. 하단 폼에서 **근무유형**과 **시작시간** 선택
3. 종료시간은 유형에 따라 자동 계산
4. **[저장]** 버튼 클릭

#### 일정 삭제
- 기존 일정이 있는 셀 선택 → **[삭제]** 버튼 클릭

#### 주간 일괄 등록
- **[일괄 등록]** 버튼으로 월~금 5일치를 한 번에 등록
- 각 요일별 근무유형·시작시간을 개별 설정 가능

#### 편집 가능 범위
- **본인 행만** 수정 가능 (다른 멤버 행은 조회만)
- **편집 가능 날짜**: 다음 주 월요일부터 오늘 기준 1개월 후까지
- 이미 지난 주의 일정은 수정 불가

#### 시작시간 옵션
`06:00` ~ `12:00` 사이 30분 간격으로 선택 가능

## 4. 공지사항

> **경로**: 사이드바 > 공지사항 (`/notices`)

### 4.1 개요

관리자(ADMIN)가 작성하는 전사 공지사항입니다. 중요 공지를 상단 고정(Pin) 할 수 있으며, 파일을 첨부할 수 있습니다.

### 4.2 화면 구성

| 영역 | 설명 |
|------|------|
| **헤더** | 제목 + 관리자만 보이는 [+ 공지 작성] 버튼 |
| **필터** | `전체` · `고정` 2개 필터 |
| **목록** | 카드 형식, 최대 너비 3xl, 고정 공지는 📌 아이콘 표시 |

### 4.3 주요 기능

#### 공지 작성 (관리자 전용)
1. **[+ 공지 작성]** 버튼 클릭 (ADMIN만 표시)
2. 입력 항목:
   - **제목** (필수)
   - **내용** (리치 텍스트 에디터)
   - **상단 고정** (토글)
   - **부서 지정** (선택, 특정 부서만 노출)
   - **첨부파일** (FormData 방식 업로드)
3. 저장 시 전사 또는 지정 부서에 공지 노출

#### 공지 상세 조회
- 카드 클릭 → 상세 페이지(`/notices/:id`)로 이동
- HTML 형식 내용 렌더링
- **첨부파일 다운로드** 가능
- **번역 패널**: 한/영/베트남어 자동 번역
- **조회수** 표시
- 관리자는 **수정 / 삭제** 가능

#### 상단 고정 필터
- **전체**: 모든 공지 표시 (고정 공지가 먼저)
- **고정**: 상단 고정된 공지만 표시

## 5. 문서함

> **경로**: 사이드바 > 문서함 (`/documents`)

### 5.1 개요

Google Drive와 연동하여 팀 공유 문서를 관리하는 도구입니다. 관리자가 공유 드라이브의 폴더를 등록하면, 팀원들이 해당 폴더의 파일을 조회·검색·업로드할 수 있습니다.

### 5.2 사전 설정

> ⚠️ Google Drive 연동이 필요합니다. 설정 > Drive 설정에서 서비스 계정을 연결하세요.

### 5.3 화면 구성

| 영역 | 설명 |
|------|------|
| **상단 바** |검색 입력란, 뷰모드 토글(리스트/그리드), 정렬 버튼, 업로드/새 폴더 버튼 |
| **브레드크럼** | 현재 폴더 경로 표시, 클릭으로 상위 폴더 이동 |
| **루트 뷰** | 등록된 폴더 카드 목록, 관리자용 [+ 폴더 등록] 버튼 |
| **폴더 내부 뷰** | 파일/서브폴더 목록 (무한 스크롤) |
| **파일 상세 모달** | 파일 클릭 시 상세 정보 표시 |

### 5.4 주요 기능

#### 폴더 등록/해제 (관리자 전용)
1. 루트 화면에서 **[+ 폴더 등록]** 클릭
2. Google Drive의 공유 드라이브 또는 폴더 ID 입력
3. 설명과 함께 등록 → 팀원들이 접근 가능
4. 등록 해제: 폴더 카드의 **[해제]** 버튼 (파일 삭제 없이 연결만 해제)

#### 파일 탐색
1. 등록된 폴더 카드 클릭 → 폴더 내부 진입
2. **서브폴더** 클릭으로 하위 구조 탐색
3. **브레드크럼** 경로 클릭으로 상위 폴더 이동
4. 무한 스크롤로 대량 파일 로딩

#### 파일 검색
- 상단 검색 입력란에 키워드 입력 (300ms 디바운스)
- 현재 등록된 모든 폴더를 대상으로 검색
- 검색 결과도 무한 스크롤 지원

#### 파일 업로드
1. 폴더 내부에서 **[업로드]** 버튼 클릭
2. 파일 선택 (다중 선택 가능)
3. **최대 파일 크기**: 50MB
4. 업로드 큐에서 진행 상태 확인 (대기/업로드 중/완료/오류)

#### 서브폴더 생성
- 폴더 내부에서 **[새 폴더]** 버튼 클릭
- 폴더명 입력 후 생성

#### 뷰 모드 / 정렬
- **리스트 뷰**: 파일명, 크기, 수정일, 소유자 컬럼
- **그리드 뷰**: 파일 아이콘/미리보기 카드
- **정렬 기준**: 수정일 · 크기 · 유형 · 소유자
- **정렬 방향**: 오름차순/내림차순 토글
- 뷰 모드와 정렬 설정은 브라우저에 저장 (localStorage)

#### 파일 상세
- 파일 클릭 → 상세 모달에서 파일 정보 확인
- Google Drive 링크로 원본 파일 열기 가능

## 6. 이슈

> **경로**: 사이드바 > 이슈 (`/issues`)

### 6.1 개요

버그, 기능 요청, 의견, 작업 등 다양한 유형의 이슈를 등록하고 추적하는 모듈입니다. 리스트 · 칸반 · 간트 3가지 뷰를 제공하며, 프로젝트별 필터링과 담당자 배정이 가능합니다.

### 6.2 화면 구성

| 영역 | 설명 |
|------|------|
| **헤더** | 제목 + [+ 이슈 추가] 버튼 |
| **뷰 전환 탭** | `리스트` · `칸반` · `간트` 3개 뷰 |
| **필터 바** | 유형 · 상태 · 심각도 · 우선순위 · 프로젝트 · 검색어 |
| **본문** | 선택된 뷰에 따른 이슈 표시 |

### 6.3 이슈 유형

| 유형 | 코드 | 설명 |
|------|------|------|
| 버그 | `BUG` | 소프트웨어 결함 보고 |
| 기능 요청 | `FEATURE_REQUEST` | 새로운 기능 개발 요청 |
| 의견 | `OPINION` | 개선 의견이나 제안 |
| 작업 | `TASK` | 일반적인 업무 작업 |
| 기타 | `OTHER` | 분류 외 이슈 |

### 6.4 이슈 상태

| 상태 | 의미 |
|------|------|
| `OPEN` | 신규 등록 |
| `IN_PROGRESS` | 진행 중 |
| `RESOLVED` | 해결 완료 |
| `CLOSED` | 종료 |
| `REOPENED` | 재오픈 |
| `DEFERRED` | 연기 |

### 6.5 주요 기능

#### 이슈 생성
1. **[+ 이슈 추가]** 버튼 클릭
2. 입력 항목:
   - **유형** (필수): BUG / FEATURE_REQUEST / OPINION / TASK / OTHER
   - **제목** (필수)
   - **설명** (리치 텍스트 에디터, 이미지 삽입 가능)
   - **심각도**: LOW / MEDIUM / HIGH
   - **우선순위**: 1(최저) ~ 5(최고)
   - **담당자** (선택, 멤버 검색으로 배정)
   - **영향 모듈** (선택, 다중 선택)
   - **프로젝트 연결** (선택)
   - **Google Drive 링크** (선택)
3. 저장 시 이슈 번호(`ISS-XXXX`) 자동 부여

#### 3가지 뷰

**리스트 뷰**
- 테이블 형식으로 이슈 목록 표시
- 페이지네이션 (20건/페이지)
- 정렬: 이슈 번호, 생성일 기준

**칸반 뷰**
- 상태별 컬럼으로 이슈 카드 배치
- 컬럼: OPEN → IN_PROGRESS → RESOLVED → CLOSED
- 전체 이슈 로딩 (최대 999건)

**간트 뷰**
- 타임라인 바 차트
- 시작일~마감일 기준으로 이슈 기간 시각화
- 전체 이슈 로딩

#### 필터링
- **유형 필터**: 이슈 유형별 필터
- **상태 필터**: 이슈 상태별 필터
- **심각도 필터**: LOW / MEDIUM / HIGH
- **우선순위 필터**: 1 ~ 5
- **프로젝트 필터**: 연결된 프로젝트별
- **검색**: 제목 키워드 검색
- 필터 변경 시 페이지가 1으로 리셋됩니다

#### 이슈 상세 / 수정
- 이슈 클릭 → **상세 모달**에서 전체 정보 확인
- **상태 변경 이력** 조회
- **수정** 버튼으로 모든 필드 편집 가능
- **삭제**: 삭제 확인 모달 후 소프트 삭제

## 7. 프로젝트

> **경로**: 사이드바 > 프로젝트 (`/project`)

### 7.1 개요

프로젝트 제안서(Proposal)부터 프로젝트 실행 관리까지의 전체 라이프사이클을 지원하는 모듈입니다. AI를 활용한 제안서 초안 생성과 분석 기능이 탑재되어 있습니다.

### 7.2 화면 구성

| 영역 | 설명 |
|------|------|
| **사이드바** | 2개 서브메뉴: `프로젝트 목록` · `제안서` |
| **프로젝트 목록** | 테이블 + 필터 (상태/유형/검색) |
| **프로젝트 상세** | 4탭: 개요 · 이슈 · 칸반 · 간트 |
| **제안서 목록** | 제안서 카드 리스트 |
| **제안서 상세** | 제안서 내용 + AI 분석 + 유사 프로젝트 + 리뷰 이력 |

### 7.3 프로젝트 관리

#### 프로젝트 목록 (`/project/projects`)
- 전체 프로젝트 테이블 보기
- 필터: 상태, 유형, 검색어
- 상태 배지로 현황 파악

#### 프로젝트 상세 (`/project/projects/:id`)

**개요 탭**
- 프로젝트 기본 정보 (제목, 설명, 기간, 상태)
- 멤버 목록 및 역할
- 프로젝트 관련 통계 요약

**이슈 탭**
- 해당 프로젝트에 연결된 이슈 목록
- 이슈 상세 조회 및 필터링

**칸반 탭**
- 이슈를 상태별 칸반 보드로 표시

**간트 탭**
- 이슈 타임라인을 간트 차트로 시각화

#### 멤버 관리
- 프로젝트 상세에서 멤버 추가/제거
- 멤버별 역할 설정

### 7.4 제안서 (Proposal)

#### 제안서 작성 (`/project/proposals/new`)
1. 프로젝트 기본 정보 입력
2. **AI 초안 생성** (AiDraftPanel):
   - 프로젝트 개요를 입력하면 AI가 제안서 초안 자동 생성
   - 생성된 초안을 기반으로 수정 가능
3. 저장 시 `DRAFT` 상태로 생성

#### 제안서 워크플로우

| 단계 | 상태 | 설명 |
|------|------|------|
| 1 | `DRAFT` | 초안 작성 중 (수정 가능) |
| 2 | `SUBMITTED` | 검토 요청 (제출됨) |
| 3 | `REVIEW` | 검토 중 |
| 4 | `APPROVED` | 승인 완료 → 프로젝트 생성 가능 |
| 4' | `REJECTED` | 반려 → 수정 후 재제출 가능 |

#### 제안서 상세 (`/project/proposals/:id`)
- **AI 분석 패널** (AiAnalysisPanel): AI가 제안서 품질과 리스크 분석
- **유사 프로젝트** (SimilarProjectsList): 과거 유사 프로젝트 자동 추천
- **리뷰 이력**: 검토 의견과 승인/반려 기록
- **리뷰 액션 패널** (ReviewActionPanel): 승인/반려 의견 작성

## 8. 일정관리

> **경로**: 사이드바 > 일정관리 (`/schedule`)

### 8.1 개요

개인 및 팀 일정을 캘린더 기반으로 관리하는 모듈입니다. 월간/주간/일간 3가지 뷰를 제공하며, 반복 일정과 카테고리별 색상 구분을 지원합니다.

### 8.2 화면 구성

| 영역 | 설명 |
|------|------|
| **캘린더 툴바** | 뷰 전환(월/주/일), 날짜 이동(`<` 오늘 `>`), 필터모드, 카테고리 필터 |
| **캘린더 본문** | 선택된 뷰에 따른 일정 표시 |
| **일정 상세 패널** | 일정 클릭 시 우측에 사이드 패널 표시 |

### 8.3 캘린더 뷰

| 뷰 | 설명 |
|-------|------|
| **월간** (Month) | 달력 형태의 월별 일정 표시. 각 날짜 셀에 일정 제목이 색상 배지로 표시 |
| **주간** (Week) | 7일 타임라인 뷰. 시간대별 일정 배치 |
| **일간** (Day) | 1일 상세 타임라인. 시간 단위로 일정 배치 |

### 8.4 일정 카테고리

| 카테고리 | 코드 | 용도 |
|----------|------|------|
| 업무 | `WORK` | 일반 업무 일정 |
| 회의 | `MEETING` | 회의/미팅 |
| 개인 | `PERSONAL` | 개인 일정 |
| 프로젝트 | `PROJECT` | 프로젝트 관련 일정 |
| 공휴일 | `HOLIDAY` | 공휴일/기념일 |
| 기타 | `ETC` | 분류 외 |

### 8.5 주요 기능

#### 일정 생성
1. 캘린더의 빈 영역 클릭 또는 **[+]** 버튼 클릭
2. 입력 항목:
   - **제목** (필수)
   - **카테고리** (필수, 6가지 중 선택)
   - **시작/종료 일시** (필수)
   - **종일 여부** (토글)
   - **설명** (선택)
   - **색상**: 7가지 색상 팔레트에서 선택
   - **공개범위**: 비공개 / 공유 / 법인
   - **반복 설정** (선택)
3. 저장 시 캘린더에 즉시 반영

#### 반복 일정
| 반복 주기 | 코드 | 예시 |
|-----------|------|------|
| 없음 | `NONE` | 단일 일정 |
| 매일 | `DAILY` | 매일 반복 |
| 매주 | `WEEKLY` | 매주 같은 요일 반복 |
| 매월 | `MONTHLY` | 매월 같은 날 반복 |
| 매년 | `YEARLY` | 매년 같은 날짜 반복 |

- 반복 종료 조건: 종료일 지정 또는 반복 횟수 지정

#### 필터 모드
| 모드 | 표시 범위 |
|------|-----------|
| `ALL` | 모든 일정 |
| `MY` | 내가 만든 일정만 |
| `DEPARTMENT` | 내 부서의 일정 |
| `SHARED` | 나에게 공유된 일정 |

#### 일정 상세
- 캘린더의 일정 클릭 → 우측 사이드 패널에 상세 표시
- 수정 / 삭제 가능
- 반복 일정의 경우 단일 건 또는 전체 반복 수정 선택

## 9. 거래처관리

> **경로**: 사이드바 > 거래처관리 (`/billing`)

### 9.1 개요

거래처, 계약, 작업명세서(SOW), 청구서, 수금까지의 전체 거래 흐름을 관리하는 종합 모듈입니다. 대시보드에서 전체 거래 현황을 한눈에 파악하고, 각 하위 메뉴에서 세부 관리를 수행합니다.

### 9.2 화면 구성 (사이드바 메뉴)

| 서브메뉴 | 경로 | 설명 |
|----------|------|------|
| **대시보드** | `/billing/dashboard` | 거래 현황 종합 차트 |
| **거래처** | `/billing/partners` | 거래처 목록 및 상세 |
| **계약** | `/billing/contracts` | 계약서 관리 |
| **SOW** | `/billing/sow` | 작업명세서 관리 |
| **청구서** | `/billing/invoices` | 청구서 발행 및 관리 |
| **수금** | `/billing/payments` | 수금/결제 관리 |
| **설정** | `/billing/settings` | 관리자 전용 설정 |

### 9.3 주요 기능

#### 대시보드 (`/billing/dashboard`)
- **요약 카드**: 총 거래처 수, 활성 계약 수, 미수금 금액, 이번 달 청구액
- **월별 차트**: 청구/수금 추이 차트
- **청구 캘린더**: 월별 청구 일정 표시
- **연체 알림**: 미수금 연체 건 경고 표시

#### 거래처 관리 (`/billing/partners`)
- **목록**: 거래처 테이블 (사업자번호, 대표자, 연락처)
- **신규 등록**: [+ 새 거래처] 버튼
- **상세 보기**: 거래처 클릭 → 4개 탭
  - **기본정보**: 사업자 정보, 연락처, 담당자
  - **계약**: 해당 거래처의 계약 목록
  - **청구서**: 해당 거래처의 청구서 이력
  - **SOW**: 관련 작업명세서

#### 계약 관리 (`/billing/contracts`)
- **목록**: 계약 테이블 + 상태 배지
- **상세**:
  - 계약 기본 정보 (기간, 금액, 상태)
  - **마일스톤** 관리 (단계별 진행 상황)
  - **변경 이력** 로그
  - **첨부 문서** (Google Drive 연동)
  - 계약 상태 배지 (활성/만료/해지 등)

#### SOW 관리 (`/billing/sow`)
- 작업명세서 목록 및 상세 관리
- 계약과 연결하여 작업 범위 정의

#### 청구서 관리 (`/billing/invoices`)
- **목록**: 청구서 테이블 + 상태 필터
- **신규 발행**: [+ 새 청구서] 버튼
- **상세**:
  - **청구 항목** 편집기 (품목, 수량, 단가)
  - **승인 워크플로우** (승인 요청/승인/반려)
  - **전자세금계산서** 연동 (E-Invoice)
  - **NTS 섹션** (국세청 연동 정보)
  - **PDF 출력**
  - **이메일 발송** (거래처에 청구서 메일)
- **일괄 생성**: 여러 거래처/계약에 대한 청구서 일괄 발행

#### 수금 관리 (`/billing/payments`)
- 수금/결제 내역 관리
- 청구서와 연결하여 수금 현황 추적

## 10. 자산관리

> **경로**: 사이드바 > 자산관리 (`/assets`)

### 10.1 개요

회사 자산(IT 장비, 소모품, 시설, 회의실 등)의 등록·배정·반납·폐기까지의 전체 라이프사이클을 관리하는 모듈입니다. 자산 사용 신청과 승인 워크플로우를 포함합니다.

### 10.2 화면 구성 (탭)

| 탭 | 설명 |
|----|------|
| **자산 목록** | 전체 자산 조회, 등록, 상세 보기 |
| **내 신청** | 내가 요청한 자산 사용/반납 신청 이력 |
| **승인 관리** | 내가 승인해야 할 요청 목록 (관리자) |
| **대시보드** | 자산 현황 통계 및 차트 |

### 10.3 자산 카테고리

| 카테고리 | 코드 | 예시 |
|----------|------|------|
| IT 장비 | `IT_EQUIPMENT` | 노트북, 모니터, 키보드 |
| 소모품 | `SUPPLY` | 문구, 토너, 소모성 부품 |
| 시설 | `FACILITY` | 사무실, 창고, 주차 공간 |
| 회의실 | `MEETING_ROOM` | 회의실, 세미나실 |

### 10.4 자산 상태

| 상태 | 코드 | 의미 |
|------|------|------|
| 사용중 | `IN_USE` | 현재 배정되어 사용 중 |
| 보관중 | `STORED` | 반납 후 보관 |
| 수리중 | `REPAIRING` | 수리/정비 중 |
| 폐기 대기 | `DISPOSAL_PENDING` | 폐기 승인 대기 |
| 폐기 완료 | `DISPOSED` | 폐기 처리 완료 |
| 예약됨 | `RESERVED` | 사용 예약 상태 |

### 10.5 주요 기능

#### 자산 등록 (자산 목록 탭)
1. **[+ 자산 등록]** 버튼 클릭
2. 입력 항목:
   - 자산명, 카테고리, 현재 상태
   - 구입일, 구입가
   - 시리얼 번호 등 상세 정보
3. 저장 후 목록에 추가

#### 자산 상세
- 목록에서 자산 클릭 → 상세 모달
- 자산 정보, 배정 이력, 수리 이력 확인

#### 자산 사용 신청 (내 신청 탭)
1. **[+ 사용 신청]** 버튼 클릭
2. 사용할 자산 선택 + 사유 입력
3. 신청 후 승인자에게 알림

#### 승인 관리 (승인 탭)
- 승인 대기 목록 조회
- 각 요청에 대해 **승인** 또는 **반려** 처리
- 승인 시 자산 상태 자동 변경

#### 대시보드
- **요약 카드**: 총 자산 수, 사용중, 보관중, 수리중 현황
- **카테고리별 차트**: 카테고리별 자산 분포
- **리스크 분석**: 노후 자산, 정기점검 필요 자산 등

## 11. 웹메일

> **경로**: 사이드바 > 웹메일 (`/mail`)

### 11.1 개요

사내 메일 서버(Postal)와 연동되는 웹메일 클라이언트입니다. 수신함·보낸메일·임시저장·휴지통 4개 폴더로 메일을 관리하며, 메일 작성·답장·전달·전체답장 기능을 제공합니다.

### 11.2 최초 설정

웹메일을 처음 사용할 때 **메일 계정 설정**이 필요합니다.

1. 메일 메뉴 접근 시 자동으로 설정 폼 표시
2. **이메일 주소** 입력 (회사 메일 주소)
3. **표시 이름** 입력 (선택)
4. [설정 완료] 클릭 → 메일 사용 시작

### 11.3 화면 구성

| 영역 | 설명 |
|------|------|
| **사이드바** | 4개 폴더 + [새 메일] 버튼, 읽지않은 메일 배지 |
| **메일 목록** | 선택된 폴더의 메일 리스트 |
| **메일 상세** | 메일 클릭 시 상세 내용 표시 |

### 11.4 폴더

| 폴더 | 아이콘 | 설명 |
|------|--------|------|
| **받은편지함** | 📥 | 수신된 메일 목록 |
| **보낸편지함** | 📤 | 발송 완료된 메일 목록 |
| **임시저장** | 📝 | 작성 중 저장된 임시 메일 |
| **휴지통** | 🗑️ | 삭제된 메일 (복구 가능) |

### 11.5 주요 기능

#### 메일 작성 (`/mail/compose`)
1. 사이드바의 **[새 메일]** 버튼 클릭
2. 입력 항목:
   - **받는 사람 (To)** (필수, 다중 입력 가능)
   - **참조 (CC)** (선택)
   - **숨은참조 (BCC)** (선택)
   - **제목** (필수)
   - **본문** (리치 텍스트 에디터)
   - **첨부파일** (선택)
3. **[보내기]** 클릭 → 발송 완료
4. **[임시저장]** 클릭 → 임시저장 폴더에 저장

#### 메일 읽기
- 목록에서 메일 클릭 → 상세 페이지(`/mail/message/:id`)
- HTML 형식 본문 렌더링 (DOMPurify로 XSS 방어)
- 첨부파일 확인 및 다운로드
- 발신자, 수신자, CC, 날짜 등 헤더 정보 표시

#### 답장 / 전체답장 / 전달
- 메일 상세 페이지에서 액션 버튼:
  - **답장**: 발신자에게만 답장
  - **전체 답장**: 발신자 + CC 전체에 답장
  - **전달**: 새 수신자에게 메일 전달
- 원본 메일 본문이 인용 형태로 자동 포함

#### 중요 표시 / 삭제
- **별표 토글**: 메일 목록에서 ⭐ 아이콘 클릭으로 중요 표시/해제
- **삭제**: 휴지통으로 이동 (소프트 삭제)

#### 읽지 않은 메일 알림
- 사이드바의 각 폴더 이름 옆에 읽지 않은 메일 수 **배지** 표시
- 받은편지함의 미읽 건수가 실시간으로 업데이트

- 지불유형: `사업소득` / `용역비`
- 상태: `활성` / `완료` / `해지`

### 12.5 근태관리 (`/hr/timesheet`)

- **월별 보기** (연-월 네비게이션)
- 요약 카드: 월별 출근 통계 (출근일수, 지각, 결근 등)
- 근태 그리드: 직원×날짜 매트릭스로 출퇴근 코드 입력
- 관리자: 일괄 등록(Batch Upsert) 가능

### 12.6 초과근무 (`/hr/overtime`)

- **월별 보기** (월 네비게이션)
- 요약: 전체 초과근무 시간 통계
- 초과근무 테이블: 직원별 초과근무 내역
- 관리자: 모달 폼으로 초과근무 기록 추가

### 12.7 휴가관리 (`/hr/leave`)

- **연도별 보기** (연도 네비게이션)
- 요약: 전체 휴가 사용 현황
- 잔여일수 테이블: 직원별 휴가 유형·발생·사용·잔여일
- 직원 행 클릭 → 상세 모달 (개별 휴가 이력)
- 관리자: 연간 휴가 잔여일 재계산 기능

### 12.8 급여관리 (`/hr/payroll`)

#### 급여 기간 목록 (`/hr/payroll`)
- 급여 산정 기간 리스트
- 관리자: 새 급여 기간 생성 (연도, 월, 지급일 지정)
- 기간 클릭 → 상세 페이지 이동

#### 급여 상세 (`/hr/payroll/:periodId`)
- 기간 제목, 상태 배지, 직원 수, 지급일 표시
- **급여 처리 패널**: 단계별 처리 (산출 → 1차 승인 → 2차 승인 → 확정)
- **급여 요약 테이블**: 직원별 급여 내역 (기본급, 수당, 공제, 실수령액)
- 직원 행 클릭 → 슬라이드 오버 패널에서 개별 급여 상세

| 급여 상태 | 의미 |
|-----------|------|
| `CALCULATED` | 산출 완료 |
| `APPROVED_L1` | 1차 승인 |
| `APPROVED_L2` | 2차 승인 |
| `FINALIZED` | 최종 확정 |

## 13. 회계관리

### 13.3 주요 기능

#### 계좌 관리
1. **계좌 등록**: [+ 계좌 추가] 클릭
   - 은행명, 지점, 계좌번호, 별칭
   - 통화 (KRW/VND/USD 등)
   - 기초 잔액, 개시일
2. **계좌 수정**: 카드의 수정 버튼
3. **계좌 삭제**: 확인 대화상자 후 삭제

- 계좌 카드 클릭 → 거래 내역 페이지 이동
- **계좌 헤더**: 은행명, 지점, 계좌번호, 현재 잔액
- **거래 테이블**: 페이지네이션 (20/50/100/200건 선택)
- **필터**: 날짜 범위, 거래처명, 적요 검색
- **정렬**: 날짜 오름차순/내림차순 토글

#### 거래 등록/수정
1. [+ 거래 추가] 버튼 또는 행 클릭
2. 입력 항목:
   - 거래일, 프로젝트명
   - 순 금액 (Net Value)
   - 부가세 (VAT)
   - 수수료 (Bank Charge)
   - 거래처명, 적요

#### Excel 임포트
1. [Excel 임포트] 버튼 클릭
2. Excel 파일 업로드
3. 처리 결과 표시: 생성된 계좌 수, 가져온 거래 수, 처리된 시트 수
4. 계좌별 개별 거래 임포트도 가능

## 14. 지식관리 (KMS)

> **경로**: 사이드바 > 지식관리 (`/kms`)

### 14.1 개요

조직의 지식 자산을 체계적으로 관리하고 AI 기반 분석을 제공하는 모듈입니다. 문서 생성, 5A 매트릭스 분석, 태그 기반 분류, 지식 그래프 시각화 기능을 포함합니다.

### 14.2 서브메뉴 구성

| 서브메뉴 | 경로 | 설명 |
|----------|------|------|
| **문서 빌더** | `/kms/doc-builder` | AI 기반 문서 자동 생성 |
| **5A 매트릭스** | `/kms/ddd` | 비즈니스 성숙도 대시보드 |
| **5A 설정** | `/kms/ddd-settings` | 매트릭스 지표/목표/가중치 설정 |
| **태그 클라우드** | `/kms/tag-cloud` | 태그 기반 시각화 |
| **태그 관리** | `/kms/tags` | 태그 계층 구조 CRUD |
| **지식 그래프** | `/kms/knowledge-graph` | 노드 기반 지식 연결 시각화 |

### 14.3 문서 빌더 (`/kms/doc-builder`)

3개 탭으로 구성됩니다.

**기초 데이터 탭 (Base Data)**
- 카테고리 목록과 완성도 표시 (퍼센트 게이지)
- 기밀 등급: `PUBLIC` / `INTERNAL` / `CONFIDENTIAL`
- 카테고리 클릭 → 편집 패널에서 기초 데이터 입력/수정
- 전체 완성도 퍼센트 표시

**문서 생성 탭 (Generate)**
- AI가 기초 데이터를 기반으로 문서 초안 자동 생성
- 생성된 문서 미리보기 및 다운로드
- 다국어 지원 (한/영/베트남어)

**동기화 탭 (Sync)**
- 외부 모듈 데이터 동기화: DDD, 거래처(Billing), 인사(HR)
- 동기화 상태 표시 및 오래된 데이터 경고
- 데이터 없을 경우 초기 설정 마법사(`InitialSetupWizard`) 자동 표시

### 14.4 5A 매트릭스 (`/kms/ddd`)

비즈니스 프로세스를 5단계로 분석하는 성숙도 대시보드입니다.

| 단계 | 영문 | 설명 |
|------|------|------|
| 인지 | Advertise | 시장 인지도 |
| 획득 | Acquisition | 고객 획득 |
| 활성화 | Activation | 서비스 활성화 |
| 가속 | Accelerate | 성장 가속 |
| 옹호 | Advocate | 고객 충성도 |

주요 기능:
- **분기별 기간 선택** (YYYY-QN 형식)
- **가중 점수 게이지**: 프로세스(30%) + 역량(40%) + 품질(30%)
- **AI 인사이트**: 각 대시보드에 대한 AI 분석 및 추천
  - 인사이트 실행 완료 표시 기능
- **스냅샷 데이터 입력**: 현재 수치 입력 패널
- **데이터 수집 & 분석**: 자동 수집 후 AI 분석 실행

### 14.5 5A 설정 (`/kms/ddd-settings`)

3개 탭으로 구성:

| 탭 | 설명 |
|----|------|
| **지표** (Metrics) | 5A 단계별 KPI 지표 설정 |
| **목표** (Targets) | 단계별 목표 값 설정 |
| **가중치** (Gauges) | 프로세스/역량/품질 가중치 설정 (합계 100% 검증) |

### 14.6 태그 클라우드 (`/kms/tag-cloud`)

- **범위 선택**: 나(`MY`) / 팀(`TEAM`) / 법인(`ENTITY`)
- **레벨 필터**: 전체 / 도메인 / 토픽 / 컨텍스트
- 인터랙티브 태그 클라우드 시각화 (사용 빈도에 따른 크기)
- 태그 클릭 → 범위별 비교 패널
- 인사이트 사이드바: 태그 사용 패턴 분석

### 14.7 태그 관리 (`/kms/tags`)

- **계층형 태그 트리** 구조로 표시
- 태그 CRUD (생성, 편집, 삭제)
- 입력 항목: 이름, 현지 이름, 레벨(Domain/Topic/Context), 부모 태그, 색상

### 14.8 지식 그래프 (`/kms/knowledge-graph`)

- 레벨별 그룹으로 노드 시각화: 도메인 → 토픽 → 컨텍스트
- 노드 크기: 사용 횟수(weight)에 비례
- 노드 호버 시 연결(connection) 관계 표시
- 최소 사용 횟수 1, 최대 노드 100개

## 15. 서비스관리

> **경로**: 사이드바 > 서비스관리 (`/service`)

### 15.1 개요

회사가 제공하는 서비스 상품, 고객, 구독의 전체 라이프사이클을 관리하는 모듈입니다. 서비스 카탈로그 등록부터 고객 가입, 구독 상태 추적까지 지원합니다.

### 15.2 서브메뉴 구성

| 서브메뉴 | 경로 | 설명 |
|----------|------|------|
| **대시보드** | `/service/dashboard` | 서비스 현황 종합 |
| **서비스** | `/service/services` | 서비스 상품 관리 |
| **고객** | `/service/clients` | 고객 관리 |
| **구독** | `/service/subscriptions` | 구독/계약 관리 |

### 15.3 대시보드 (`/service/dashboard`)
- **4개 통계 카드**: 총 서비스 수, 총 고객 수, 활성 구독 수, 30일 내 만료 예정
- **서비스 개요 카드**: 각 서비스별 상태 배지, 요금제 수, 활성 구독자 수

### 15.4 서비스 관리 (`/service/services`)

#### 서비스 목록
- 카드 그리드 레이아웃
- 필터: 상태, 카테고리
- 카드 클릭 → 상세 페이지

#### 서비스 상태

| 상태 | 의미 |
|------|------|
| `ACTIVE` | 활성 (판매 중) |
| `INACTIVE` | 비활성 |
| `DEPRECATED` | 단종 |

#### 서비스 카테고리

| 카테고리 | 설명 |
|----------|------|
| `COMMUNICATION` | 커뮤니케이션 서비스 |
| `COMMERCE` | 커머스 서비스 |
| `MARKETING` | 마케팅 서비스 |
| `PACKAGE` | 패키지 상품 |
| `OTHER` | 기타 |

#### 서비스 등록
- 코드, 이름, 카테고리, 설명, 색상 입력

#### 서비스 상세 (`/service/services/:id`)
- 서비스 정보 조회 및 인라인 수정
- **요금제(Plan) 관리**:
  - 요금제 목록 조회
  - 새 요금제 추가 (코드, 이름, 가격, 과금 주기)
  - 요금제 삭제
- 서비스 삭제 (확인 대화상자)

### 15.5 고객 관리 (`/service/clients`)

#### 고객 목록
- 테이블 형식 (회사명, 코드, 유형, 국가, 상태, 구독 수)
- 필터: 텍스트 검색, 상태, 유형
- 행 클릭 → 상세 페이지

#### 고객 유형

| 유형 | 설명 |
|------|------|
| `COMPANY` | 법인 고객 |
| `INDIVIDUAL` | 개인 고객 |

#### 고객 상태

| 상태 | 의미 |
|------|------|
| `ACTIVE` | 활성 고객 |
| `INACTIVE` | 비활성 |
| `PROSPECT` | 잠재 고객 |
| `CHURNED` | 이탈 고객 |

#### 고객 상세 (`/service/clients/:id`)
4개 탭으로 구성:

| 탭 | 설명 |
|----|------|
| **정보** | 고객 기본 정보 인라인 수정 |
| **연락처** | 담당자 CRUD (이름, 이메일, 전화, 직책, 부서, 대표 담당자 여부) |
| **구독** | 해당 고객의 구독 목록 |
| **노트** | 메모 CRUD (유형, 제목, 내용) |

### 15.6 구독 관리 (`/service/subscriptions`)

#### 구독 목록
- 테이블 형식 (고객명, 서비스, 요금제, 가격, 종료일, 상태)
- 필터: 상태, 서비스

#### 구독 상태

| 상태 | 색상 | 의미 |
|------|------|------|
| `TRIAL` | 파랑 | 체험 기간 |
| `ACTIVE` | 초록 | 활성 구독 |
| `SUSPENDED` | 노랑 | 일시 정지 |
| `EXPIRING` | 주황 | 만료 임박 |
| `EXPIRED` | 빨강 | 만료됨 |
| `CANCELLED` | 빨강 | 해지됨 |

#### 구독 등록
- 고객, 서비스, 시작/종료일, 과금주기, 가격, 통화, 자동갱신 여부, 메모

#### 구독 상세 (`/service/subscriptions/:id`)
- 고객-서비스-요금제 정보 표시
- 상태 배지 + 인라인 수정 모드
- **라이프사이클 액션**:
  - 해지 (사유 입력 필수)
  - 정지
  - 재개
  - 갱신
- **구독 이력 로그**: 상태 변경 기록

# 관리자 설정

## 16. 설정 (관리자)

> **경로**: 사이드바 하단 > 설정 (`/settings`)  
> ⚠️ **관리자(ADMIN) 권한 필요**

### 16.1 개요

시스템 전반의 설정을 관리하는 관리자 전용 영역입니다. 카드형 대시보드에서 각 설정 항목으로 이동하며, 사용자의 메뉴 권한에 따라 접근 가능한 카드만 표시됩니다.

### 16.2 설정 항목 목록

| 설정 | 경로 | 설명 |
|------|------|------|
| **멤버 관리** | `/settings/members` | 사용자 관리, 그룹, 초대 |
| **API 키** | `/settings/api-keys` | 외부 API 키 관리 |
| **SMTP** | `/settings/smtp` | 메일 서버 설정 |
| **메뉴 권한** | `/settings/permissions` | 메뉴별 접근 권한 |
| **Drive** | `/settings/drive` | Google Drive 연동 |
| **법인 관리** | `/settings/entities` | 법인(Entity) 관리 |
| **부서 관리** | `/settings/departments` | 부서 계층 관리 |
| **대화 관리** | `/settings/conversations` | AI 대화 이력 관리 |
| **메일 계정** | `/settings/mail-accounts` | 메일 계정 관리 |
| **에이전트 설정** | `/settings/agents` | AI 에이전트 설정 |
| **용어집** | `/settings/glossary` | 번역 용어집 |

### 16.3 내 서명
설정 메인 페이지 하단에 **내 서명** 섹션이 포함되어 있습니다.
- 서명 이미지 업로드 (개인 전자 서명용)
- 기존 서명 조회 및 삭제

### 16.4 멤버 관리 (`/settings/members`)

3개 탭으로 구성:

| 탭 | 설명 |
|----|------|
| **멤버** | 전체 사용자 목록, 검색, 역할/상태 확인 |
| **그룹** | 업무 그룹 관리 (생성/수정/삭제) |
| **초대** | 멤버 초대 발송 및 대기 목록 관리 |

#### 멤버 상세 (`/settings/members/:id`)
- 기본 정보: 회사 이메일, 이름, 직책
- **법인 역할 배정**: 어떤 법인에 어떤 역할로 소속
- **부서 역할 배정**: 부서별 역할 설정
- **그룹 배정**: 업무 그룹 소속 관리
- **직원 연결**: HR 직원 레코드와 연결/해제
- **가입 승인/거절**: 신규 가입 요청 처리

### 16.5 API 키 관리 (`/settings/api-keys`)
- API 키 카드 목록 (키 이름, 생성일, 마지막 사용일)
- 키 생성: 이름 입력 → 키 자동 생성
- 키 삭제: 확인 대화상자 후 삭제
- **주의**: 생성 시 표시되는 키는 다시 확인할 수 없으므로 즉시 복사

### 16.6 SMTP 설정 (`/settings/smtp`)
- SMTP 서버 연결 설정
- 입력 항목: 호스트, 포트, 사용자, 비밀번호, 보내는 주소, 보안(TLS) 토글
- **[저장]**: 설정 저장
- **[연결 테스트]**: SMTP 연결 테스트 실행 + 성공/실패 결과 표시

### 16.7 메뉴 권한 (`/settings/permissions`)

3개 탭으로 구성:

| 탭 | 설명 |
|----|------|
| **메뉴 구성** | 시스템에서 사용 가능한 메뉴 항목 ON/OFF |
| **역할별 권한** | 역할(ADMIN/MANAGER/USER 등)별 메뉴 접근 권한 설정 |
| **사용자별 권한** | 특정 사용자에 대한 개별 권한 오버라이드 |

### 16.8 Drive 설정 (`/settings/drive`)
- Google Drive 서비스 계정 연동 설정
- 입력 항목:
  - 위임 이메일 (Impersonate Email)
  - 거래처 루트 폴더 ID/이름
- **공유 드라이브 탐색**: 연결된 공유 드라이브 목록 + 폴더 트리 네비게이션
- **[저장]**: 설정 저장
- **[연결 테스트]**: Drive 연결 테스트
- **설정 가이드**: 하단에 설정 절차 안내 문서 표시

### 16.9 법인 관리 (`/settings/entities`)
- 법인(Entity) CRUD
- 등록 항목:
  - 코드, 법인명, 영문명
  - 국가 (KR/VN)
  - 통화 (KRW/VND/USD)
  - 사업자등록번호, 주소, 대표자, 연락처, 이메일
  - 급여 지급일, 상태
- 법인별 하위 설정:
  - **그룹 탭**: 해당 법인의 그룹 관리
  - **부서 탭**: 해당 법인의 부서 관리
- **법인 직인** 이미지 업로드/관리

### 16.10 부서 관리 (`/settings/departments`)
- **계층형 부서 트리** 구조로 표시
- 부서 CRUD (이름, 현지명, 상위 부서, 활성 여부, 정렬 순서)
- 하위 부서 추가 (자식 노드)
- 부서별 소속 **멤버 목록** 조회
- 부서에 사용자 배정 / 역할 변경 / 역할 제거

### 16.11 대화 관리 (`/settings/conversations`)
- 전체 AI 대화 이력 관리자 열람
- 필터: 부서, 검색어, 날짜 범위
- 페이지네이션 (20건/페이지)
- 대화 클릭 → 상세 모달에서 전체 메시지 확인

### 16.12 메일 계정 관리 (`/settings/mail-accounts`)

2개 탭으로 구성:

| 탭 | 설명 |
|----|------|
| **계정** | 메일 계정 CRUD (카드형 목록, 검색) |
| **웹메일 설정** | 웹메일 시스템 전반 설정 |

### 16.13 에이전트 설정 (`/settings/agents`)

2개 탭으로 구성:

| 탭 | 설명 |
|----|------|
| **채팅 에이전트** | AI 에이전트 활성/비활성 토글, 설정 편집 |
| **프롬프트 설정** | 에이전트별 시스템 프롬프트 관리 |

#### 전용 에이전트 코드

| 코드 | 부서 | 채팅 메뉴 |
|------|------|-----------|
| `LEGAL` | 법무 | `CHAT_LEGAL` |
| `ACCOUNTING` | 경리 | `CHAT_ACCOUNTING` |
| `TRANSLATION` | 번역 | `CHAT_TRANSLATION` |
| `PM` | 프로젝트관리 | `CHAT_PM` |
| `DEVELOPMENT` | 개발 | `CHAT_DEVELOPMENT` |

#### 에이전트 설정 편집
- 시스템 프롬프트: AI 에이전트의 역할과 행동을 정의하는 프롬프트
- 설명: 에이전트 소개 텍스트
- 공개 범위: 특정 부서 또는 전체 공개

## 공통 기능

### 다국어 지원
- 전체 UI가 **한국어 / 영어 / 베트남어** 3개 언어 지원

- 헤더 중앙의 검색바로 전체 시스템 통합 검색

- 우하단 플로팅 버튼(💬)으로 AI 어시스턴트 모달 열기

- 업무 관련 질문, 데이터 조회, 작업 지원 가능

### 자동 로그아웃
- 30분간 미활동 시 자동 로그아웃
